Next.jsはコンフィグレーションなしで**モダンブラウザ**をサポートしています。

- Chrome 64+
- Edge 79+
- Firefox 67+
- Opera 51+
- Safari 12+

## Browserslist {#browserslist}

特定のブラウザや機能をターゲットにしたい場合、Next.jsは`package.json`ファイルで[Browserslist](https://browsersl.ist)の設定をサポートしています。Next.jsはデフォルトで以下のBrowserslist設定を使用します：

```json title="package.json"
{
  "browserslist": [
    "chrome 64",
    "edge 79",
    "firefox 67",
    "opera 51",
    "safari 12"
  ]
}
```

## Polyfills {#polyfills}

[広く使用されているポリフィル](https://github.com/vercel/next.js/blob/canary/packages/next-polyfill-nomodule/src/index.js)を注入します。以下を含みます：

- [**fetch()**](https://developer.mozilla.org/docs/Web/API/Fetch_API) — 置き換え：`whatwg-fetch`と`unfetch`
- [**URL**](https://developer.mozilla.org/docs/Web/API/URL) — 置き換え：[Node.js APIの`url`パッケージ](https://nodejs.org/api/url.html)
- [**Object.assign()**](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Object/assign) — 置き換え：`object-assign`、`object.assign`、および`core-js/object/assign`

もし依存ライブラリがこれらのポリフィルを含んでいる場合、重複を避けるためにプロダクションビルドから自動的に取り除かれます。

さらに、バンドルサイズを縮小するために、Next.jsはこれらのポリフィルを必要とするブラウザのみに読み込みます。グローバルでのウェブトラフィックの大半はこれらのポリフィルをダウンロードしません。

### カスタムポリフィル {#custom-polyfills}

あなた自身のコードや外部のnpm依存がターゲットブラウザ（たとえばIE 11）でサポートされていない機能を必要とする場合、自分でポリフィルを追加する必要があります。

この場合、必要な**特定のポリフィル**を[カスタム`<App>`](https://nextjs.org/docs/canary/pages/building-your-application/routing/custom-app)または個々のコンポーネントでトップレベルのインポートとして追加すべきです。

## JavaScriptの言語機能 {#javascript-language-features}

Next.jsでは最新のJavaScript機能をそのまま使用できます。[ES6の機能](https://github.com/lukehoban/es6features)に加え、Next.jsは以下もサポートしています：

- [Async/await](https://github.com/tc39/ecmascript-asyncawait) (ES2017)
- [オブジェクトのRest/Spreadプロパティ](https://github.com/tc39/proposal-object-rest-spread) (ES2018)
- [動的`import()`](https://github.com/tc39/proposal-dynamic-import) (ES2020)
- [オプショナルチェイニング](https://github.com/tc39/proposal-optional-chaining) (ES2020)
- [Nullish Coalescing](https://github.com/tc39/proposal-nullish-coalescing) (ES2020)
- [クラスフィールド](https://github.com/tc39/proposal-class-fields)と[静的プロパティ](https://github.com/tc39/proposal-static-class-features) (ES2022)
- その他多数

### TypeScriptの機能 {#typescript-features}

Next.jsには組み込みのTypeScriptサポートがあります。[詳細はこちら](https://nextjs.org/docs/canary/pages/api-reference/config/typescript)をご覧ください。

### Babel設定のカスタマイズ（高度な機能） {#customizing-babel-config-advanced}

Babelの設定をカスタマイズできます。[詳細はこちら](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)をご覧ください。
