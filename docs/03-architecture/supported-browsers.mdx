Next.jsは、**モダンブラウザ**をゼロコンフィグレーションでサポートしています。

- Chrome 64+
- Edge 79+
- Firefox 67+
- Opera 51+
- Safari 12+

## Browserslist {#browserslist}

特定のブラウザや機能をターゲットにしたい場合、Next.jsは`package.json`ファイルでの[Browserslist](https://browsersl.ist)設定をサポートしています。Next.jsはデフォルトで以下のBrowserslist設定を使用します：

```json title="package.json"
{
  "browserslist": [
    "chrome 64",
    "edge 79",
    "firefox 67",
    "opera 51",
    "safari 12"
  ]
}
```

## Polyfills {#polyfills}

Next.jsは、以下を含む[広く使用されているポリフィル](https://github.com/vercel/next.js/blob/canary/packages/next-polyfill-nomodule/src/index.js)を注入します：

- [**fetch()**](https://developer.mozilla.org/docs/Web/API/Fetch_API) — 置き換え：`whatwg-fetch`と`unfetch`
- [**URL**](https://developer.mozilla.org/docs/Web/API/URL) — 置き換え：[`url`パッケージ（Node.js API）](https://nodejs.org/api/url.html)
- [**Object.assign()**](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Object/assign) — 置き換え：`object-assign`、`object.assign`、`core-js/object/assign`

これらのポリフィルが依存関係に含まれている場合、重複を避けるためにプロダクションビルドから自動的に削除されます。

さらに、バンドルサイズを削減するため、Next.jsはこれらのポリフィルを必要とするブラウザにのみロードします。世界中のウェブトラフィックの大部分はこれらのポリフィルをダウンロードしません。

### カスタムポリフィル {#custom-polyfills}

あなたのコードや外部のnpm依存関係がターゲットブラウザでサポートされていない機能（例えばIE 11）を必要とする場合、ポリフィルを自分で追加する必要があります。

この場合、[カスタム`<App>`](https://nextjs.org/docs/canary/pages/building-your-application/routing/custom-app)や個々のコンポーネントで必要な**特定のポリフィル**をトップレベルでインポートするべきです。

## JavaScriptの言語機能 {#javascript-language-features}

Next.jsは、最新のJavaScript機能をそのまま使用できるようにしています。[ES6機能](https://github.com/lukehoban/es6features)に加えて、Next.jsは以下もサポートしています：

- [Async/await](https://github.com/tc39/ecmascript-asyncawait) (ES2017)
- [Object Rest/Spread Properties](https://github.com/tc39/proposal-object-rest-spread) (ES2018)
- [Dynamic `import()`](https://github.com/tc39/proposal-dynamic-import) (ES2020)
- [Optional Chaining](https://github.com/tc39/proposal-optional-chaining) (ES2020)
- [Nullish Coalescing](https://github.com/tc39/proposal-nullish-coalescing) (ES2020)
- [Class Fields](https://github.com/tc39/proposal-class-fields)と[Static Properties](https://github.com/tc39/proposal-static-class-features) (ES2022)
- その他多数！
