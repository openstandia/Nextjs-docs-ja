Next.jsはゼロコンフィギュレーションで**モダンブラウザ**をサポートします。

- Chrome 64+
- Edge 79+
- Firefox 67+
- Opera 51+
- Safari 12+

## Browserslist {#browserslist}

特定のブラウザや機能をターゲットにしたい場合は、Next.jsは `package.json` ファイルでの [Browserslist](https://browsersl.ist) 設定をサポートしています。Next.jsはデフォルトで以下のBrowserslist設定を使用します：

```json title="package.json"
{
  "browserslist": [
    "chrome 64",
    "edge 79",
    "firefox 67",
    "opera 51",
    "safari 12"
  ]
}
```

## ポリフィル {#polyfills}

私たちは、以下を含む[広く使用されるポリフィル](https://github.com/vercel/next.js/blob/canary/packages/next-polyfill-nomodule/src/index.js)を挿入します：

- [**fetch()**](https://developer.mozilla.org/docs/Web/API/Fetch_API) — 置換対象：`whatwg-fetch`および`unfetch`
- [**URL**](https://developer.mozilla.org/docs/Web/API/URL) — 置換対象：[`url` パッケージ (Node.js API)](https://nodejs.org/api/url.html)
- [**Object.assign()**](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Object/assign) — 置換対象：`object-assign`、`object.assign`、および`core-js/object/assign`

これらのポリフィルを含んでいる依存関係がある場合は、それらが重複しないように、プロダクションビルドから自動的に削除されます。

さらに、バンドルサイズを削減するため、Next.jsはこれらのポリフィルを必要とするブラウザにのみロードします。世界のウェブトラフィックの大部分はこれらのポリフィルをダウンロードしません。

### カスタムポリフィル {#custom-polyfills}

あなた自身のコードや外部のnpm依存関係がターゲットブラウザ（例：IE 11）でサポートされていない機能を必要とする場合、ポリフィルを自分で追加する必要があります。

この場合、必要な**特定のポリフィル**を[カスタム `<App>`](https://nextjs.org/docs/canary/pages/building-your-application/routing/custom-app)または個々のコンポーネントでトップレベルのインポートを追加するべきです。

## JavaScript言語機能 {#javascript-language-features}

Next.jsは最新のJavaScript機能をそのまま使用できるようにします。[ES6の機能](https://github.com/lukehoban/es6features)に加えて、Next.jsは以下もサポートしています：

- [Async/await](https://github.com/tc39/ecmascript-asyncawait) (ES2017)
- [Object Rest/Spread Properties](https://github.com/tc39/proposal-object-rest-spread) (ES2018)
- [Dynamic `import()`](https://github.com/tc39/proposal-dynamic-import) (ES2020)
- [Optional Chaining](https://github.com/tc39/proposal-optional-chaining) (ES2020)
- [Nullish Coalescing](https://github.com/tc39/proposal-nullish-coalescing) (ES2020)
- [Class Fields](https://github.com/tc39/proposal-class-fields) および [Static Properties](https://github.com/tc39/proposal-static-class-features) (ES2022)
- その他！
