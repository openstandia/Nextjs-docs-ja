ソースマップは、開発中はデフォルトで有効になっています。本番ビルド中は、クライアントにソースを漏らさないために、特定の設定フラグでオプトインしない限り無効です。

Next.jsは、本番ビルド中にブラウザー用のソースマップ生成を有効にするための設定フラグを提供しています：

```js title="next.config.js"
module.exports = {
  productionBrowserSourceMaps: true,
}
```

`productionBrowserSourceMaps` オプションが有効にされると、ソースマップは JavaScript ファイルと同じディレクトリに出力されます。Next.js は、要求があった場合にこれらのファイルを自動的に提供します。

- ソースマップを追加すると、`next build` の時間が増加します
- `next build` 中のメモリ使用量が増加します
