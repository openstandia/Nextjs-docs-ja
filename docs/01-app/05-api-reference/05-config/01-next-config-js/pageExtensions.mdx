<AppOnly>

デフォルトでは、Next.jsは次の拡張子を持つファイルを受け入れます：`.tsx`、`.ts`、`.jsx`、`.js`。これを変更して、markdown（`.md`、`.mdx`）のような他の拡張子を許可することができます。

```js title="next.config.js"
const withMDX = require('@next/mdx')()

/** @type {import('next').NextConfig} */
const nextConfig = {
  pageExtensions: ['js', 'jsx', 'ts', 'tsx', 'md', 'mdx'],
}

module.exports = withMDX(nextConfig)
```

</AppOnly>

<PagesOnly>

Next.jsで使用されるデフォルトのページ拡張子（`.tsx`、`.ts`、`.jsx`、`.js`）を拡張することができます。`next.config.js`内で、`pageExtensions`設定を追加してください：

```js title="next.config.js"
module.exports = {
  pageExtensions: ['mdx', 'md', 'jsx', 'js', 'tsx', 'ts'],
}
```

これらの値を変更すると、次のようなすべてのNext.jsページに影響します：

- [`middleware.js`](https://nextjs.org/docs/canary/pages/building-your-application/routing/middleware)
- [`instrumentation.js`](https://nextjs.org/docs/canary/pages/guides/instrumentation)
- `pages/_document.js`
- `pages/_app.js`
- `pages/api/`

例えば、`.ts`ページ拡張子を`.page.ts`に再設定した場合、`middleware.page.ts`、`instrumentation.page.ts`、`_app.page.ts`のようにページをリネームする必要があります。

## `pages`ディレクトリに非ページファイルを含める {#including-non-page-files-in-the-pages-directory}

`pages`ディレクトリにテストファイルやコンポーネントで使用される他のファイルを配置することができます。`next.config.js`内で、`pageExtensions`設定を追加してください：

```js title="next.config.js"
module.exports = {
  pageExtensions: ['page.tsx', 'page.ts', 'page.jsx', 'page.js'],
}
```

その後、ページを`.page`を含むファイル拡張子にリネームします（例：`MyPage.tsx`を`MyPage.page.tsx`にリネーム）。上記で言及したファイルを含むすべてのNext.jsページをリネームすることを確認してください。

</PagesOnly>
