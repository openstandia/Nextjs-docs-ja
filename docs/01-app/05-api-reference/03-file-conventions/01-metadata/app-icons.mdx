`favicon`、`icon`、または`apple-icon`ファイルの規約を使用すると、アプリケーションのアイコンを設定できます。

これらは、Webブラウザのタブ、スマートフォンのホーム画面、検索エンジンの結果などに表示されるアプリのアイコンを追加するのに役立ちます。

アプリのアイコンを設定する方法は2つあります：

- [画像ファイルを使用する方法（.ico, .jpg, .png）](#image-files-ico-jpg-png)
- [コードを使用してアイコンを生成する方法（.js, .ts, .tsx）](#generate-icons-using-code-js-ts-tsx)

## 画像ファイルを使用する方法（.ico, .jpg, .png） {#image-files-ico-jpg-png}

`/app`ディレクトリ内に`favicon`、`icon`、または`apple-icon`の画像ファイルを配置することで、アプリのアイコンを設定します。
`favicon`画像は`app/`のトップレベルにのみ配置できます。

Next.jsはファイルを評価し、アプリの`<head>`要素に適切なタグを自動的に追加します。

| ファイル規約                | サポートされているファイルタイプ        | 有効な場所 |
| --------------------------- | --------------------------------------- | ---------- |
| [`favicon`](#favicon)       | `.ico`                                  | `app/`     |
| [`icon`](#icon)             | `.ico`, `.jpg`, `.jpeg`, `.png`, `.svg` | `app/**/*` |
| [`apple-icon`](#apple-icon) | `.jpg`, `.jpeg`, `.png`                 | `app/**/*` |

### `favicon` {#favicon}

`favicon.ico`画像ファイルをroot `/app`ルートセグメントに追加します。

```html title="<head> output"
<link rel="icon" href="/favicon.ico" sizes="any" />
```

### `icon` {#icon}

`icon.(ico|jpg|jpeg|png|svg)`画像ファイルを追加します。

```html title="<head> output"
<link
  rel="icon"
  href="/icon?<generated>"
  type="image/<generated>"
  sizes="<generated>"
/>
```

### `apple-icon` {#apple-icon}

`apple-icon.(jpg|jpeg|png)`画像ファイルを追加します。

```html title="<head> output"
<link
  rel="apple-touch-icon"
  href="/apple-icon?<generated>"
  type="image/<generated>"
  sizes="<generated>"
/>
```

> **Good to know**:
>
> - ファイル名に番号のサフィックスを追加することで、複数のアイコンを設定できます。例えば、`icon1.png`、`icon2.png`などです。番号付きファイルは辞書順にソートされます。
> - Faviconsはroot `/app`セグメントにのみ設定できます。より細かい設定が必要な場合は、[`icon`](#icon)を使用できます。
> - 適切な`<link>`タグと属性（`rel`、`href`、`type`、`sizes`など）は、アイコンの種類と評価されたファイルのメタデータによって決まります。
> - 例えば、32x32pxの`.png`ファイルには`type="image/png"`と`sizes="32x32"`属性が付きます。
> - 拡張子が`.svg`の場合やファイルの画像サイズが決定されていない場合、`sizes="any"`がアイコンに追加されます。詳細はこの[favicon handbook](https://evilmartians.com/chronicles/how-to-favicon-in-2021-six-files-that-fit-most-needs)をご覧ください。

## コードを使用してアイコンを生成する方法（.js, .ts, .tsx） {#generate-icons-using-code-js-ts-tsx}

[実際の画像ファイル](#image-files-ico-jpg-png)を使用することに加えて、コードを使用してプログラム的にアイコンを**生成**することができます。

`icon`または`apple-icon`ルートを作成し、デフォルトで関数をエクスポートすることでアプリのアイコンを生成します。

| ファイル規約 | サポートされているファイルタイプ |
| ------------ | -------------------------------- |
| `icon`       | `.js`, `.ts`, `.tsx`             |
| `apple-icon` | `.js`, `.ts`, `.tsx`             |

アイコンを生成する最も簡単な方法は、`next/og`の[`ImageResponse`](/docs/app/api-reference/functions/image-response) APIを使用することです。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/icon.tsx" switcher
import { ImageResponse } from 'next/og'

// 画像メタデータ
export const size = {
  width: 32,
  height: 32,
}
export const contentType = 'image/png'

// 画像生成
export default function Icon() {
  return new ImageResponse(
    (
      // ImageResponse JSX要素
      <div
        style={{
          fontSize: 24,
          background: 'black',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
          color: 'white',
        }}
      >
        A
      </div>
    ),
    // ImageResponseオプション
    {
      // 便利のため、エクスポートされたアイコンのサイズメタデータを再利用して
      // ImageResponseの幅と高さを設定します。
      ...size,
    }
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">


```html title="<head> output"
<link rel="icon" href="/icon?<generated>" type="image/png" sizes="32x32" />
```

> **Good to know**:
>
> - デフォルトでは、生成されたアイコンは[**静的に最適化**](/docs/app/building-your-application/rendering/server-components#static-rendering-default)されています（ビルド時に生成され、キャッシュされます）。ただし、[Dynamic APIs](/docs/app/building-your-application/rendering/server-components#server-rendering-strategies#dynamic-apis)やキャッシュされていないデータを使用する場合は除きます。
> - [`generateImageMetadata`](/docs/app/api-reference/functions/generate-image-metadata)を使用して、同じファイル内で複数のアイコンを生成できます。
> - `favicon`アイコンを生成することはできません。代わりに[`icon`](#icon)または[favicon.ico](#favicon)ファイルを使用してください。
> - アプリのアイコンは特別なRoute Handlersであり、[Dynamic API](/docs/app/deep-dive/caching#dynamic-apis)や[dynamic config](/docs/app/deep-dive/caching#segment-config-options)オプションを使用しない限り、デフォルトでキャッシュされます。

### Props {#props}

デフォルトのエクスポート関数は、以下のpropsを受け取ります：

#### `params`（オプション） {#params-optional}

rootセグメントから`icon`または`apple-icon`が配置されているセグメントまでの[動的ルートパラメータ](/docs/app/building-your-application/routing/dynamic-routes)オブジェクトを含むオブジェクトです。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/[slug]/icon.tsx" switcher
export default function Icon({ params }: { params: { slug: string } }) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/[slug]/icon.js" switcher
export default function Icon({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

| ルート                          | URL         | `params`                  |
| ------------------------------- | ----------- | ------------------------- |
| `app/shop/icon.js`              | `/shop`     | `undefined`               |
| `app/shop/[slug]/icon.js`       | `/shop/1`   | `{ slug: '1' }`           |
| `app/shop/[tag]/[item]/icon.js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |

### Returns {#returns}

デフォルトのエクスポート関数は、`Blob` | `ArrayBuffer` | `TypedArray` | `DataView` | `ReadableStream` | `Response`を返す必要があります。

> **Good to know**: `ImageResponse`はこの戻り値の型を満たしています。

### Config exports {#config-exports}

`icon`または`apple-icon`ルートから`size`と`contentType`変数をエクスポートすることで、アイコンのメタデータをオプションで設定できます。

| オプション                    | 型                                                                                                             |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------- |
| [`size`](#size)               | `{ width: number; height: number }`                                                                            |
| [`contentType`](#contenttype) | `string` - [画像MIMEタイプ](https://developer.mozilla.org/docs/Web/HTTP/Basics_of_HTTP/MIME_types#image_types) |

#### Route Segment Config {#route-segment-config}

`icon`と`apple-icon`は、PagesやLayoutsと同じ[ルートセグメント設定](/docs/app/api-reference/file-conventions/route-segment-config)オプションを使用できる特別な[Route Handlers](/docs/app/building-your-application/routing/route-handlers)です。
