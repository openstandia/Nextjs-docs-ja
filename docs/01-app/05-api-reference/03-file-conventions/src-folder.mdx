プロジェクトの root に特別な Next.js の `app` または `pages` ディレクトリを持つ代わりに、Next.js はアプリケーションコードを `src` フォルダに配置する一般的なパターンもサポートしています。

これは、アプリケーションコードをプロジェクトの設定ファイルから分離するもので、多くの個人やチームに好まれています。

`src` フォルダを使用するには、`app` Router フォルダまたは `pages` Router フォルダをそれぞれ `src/app` または `src/pages` に移動します。

> **Good to know**:
>
> - `/public` ディレクトリはプロジェクトの root に残す必要があります
> - `package.json`、`next.config.js`、`tsconfig.json` などの設定ファイルはプロジェクトの root に残す必要があります
> - `.env.*` ファイルはプロジェクトの root に残す必要があります
> - `app` または `pages` が root ディレクトリに存在する場合、`src/app` または `src/pages` は無視されます
> - `src` を使用している場合、他のアプリケーションフォルダ（例：`/components` や `/lib`）も移動することが多いです
> - Middleware を使用している場合は、`src` フォルダ内に配置することを確認してください
> - Tailwind CSS を使用している場合、`tailwind.config.js` ファイルの[content セクション](https://tailwindcss.com/docs/content-configuration)に `/src` プレフィックスを追加する必要があります
> - `@/*` のような TypeScript パスをインポートに使用している場合、`tsconfig.json` の `paths` オブジェクトに `src/` を含めるように更新する必要があります
