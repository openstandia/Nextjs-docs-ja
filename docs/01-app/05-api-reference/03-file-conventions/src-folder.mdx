プロジェクトのrootに特別なNext.jsの`app`または`pages`ディレクトリを持つ代わりに、Next.jsは`src`フォルダの下にアプリケーションコードを配置する一般的なパターンもサポートしています。

これは、アプリケーションコードをプロジェクトの設定ファイルから分離するもので、多くの個人やチームに好まれています。

`src`フォルダを使用するには、`app` Router フォルダまたは`pages` Router フォルダをそれぞれ`src/app`または`src/pages`に移動します。

> **Good to know**:
>
> - `/public`ディレクトリはプロジェクトのrootに残す必要があります。
> - `package.json`、`next.config.js`、`tsconfig.json`などの設定ファイルはプロジェクトのrootに残す必要があります。
> - `.env.*`ファイルはプロジェクトのrootに残す必要があります。
> - `app`または`pages`がrootディレクトリに存在する場合、`src/app`または`src/pages`は無視されます。
> - `src`を使用している場合、他のアプリケーションフォルダ（例：`/components`や`/lib`）も移動することが多いです。
> - Middlewareを使用している場合は、`src`フォルダ内に配置してください。
> - Tailwind CSSを使用している場合、`tailwind.config.js`ファイルの[contentセクション](https://tailwindcss.com/docs/content-configuration)に`/src`プレフィックスを追加する必要があります。
> - `@/*`のようなTypeScriptのインポートパスを使用している場合、`tsconfig.json`の`paths`オブジェクトを更新して`src/`を含める必要があります。
