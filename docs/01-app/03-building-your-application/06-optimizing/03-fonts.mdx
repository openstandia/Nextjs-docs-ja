[**`next/font`**](/docs/app/api-reference/components/font)は、フォント（カスタムフォントを含む）を自動的に最適化し、外部ネットワークリクエストを削除してプライバシーとパフォーマンスを改善します。

> **🎥 視聴ください：** `next/font`の使用方法について詳しく学びましょう → [YouTube（6分）](https://www.youtube.com/watch?v=L8_98i_bMMA)。

`next/font`には、*任意の*フォントファイルの**組み込みの自動セルフホスティング**が含まれています。これにより、CSSの`size-adjust`プロパティを使用してレイアウトシフトなしでWebフォントを最適に読み込むことができます。

この新しいフォントシステムにより、便利にGoogleフォントをすべて使用できるようになり、パフォーマンスとプライバシーを考慮した設計が可能です。CSSとフォントファイルはビルド時にダウンロードされ、静的アセットの残りと一緒にセルフホストされます。**ブラウザからGoogleに対してリクエストは送信されません。**

## Google Fonts {#google-fonts}

任意のGoogleフォントを自動でセルフホストします。フォントはデプロイメントに含まれ、デプロイメントの同じドメインから提供されます。**ブラウザからGoogleに対してリクエストは送信されません。**

`next/font/google`から使用したいフォントを関数としてインポートすることから始めます。最良のパフォーマンスと柔軟性のために[可変フォント](https://fonts.google.com/variablefonts)を使用することをお勧めします。

<AppOnly>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher
import { Inter } from 'next/font/google'

// 可変フォントを読み込む場合、フォントの太さを指定する必要はありません
const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
})

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  )
}
```

複数の太さやスタイルを指定するには、配列を使用します：

```jsx title="app/layout.js"
const roboto = Roboto({
  weight: ['400', '700'],
  style: ['normal', 'italic'],
  subsets: ['latin'],
  display: 'swap',
})
```

### `<head>`でフォントを適用する {#apply-the-font-in-head}

フォントをラッパーや`className`を使わずに以下のように`<head>`内に挿入して使用することもできます：

```jsx title="pages/_app.js"
import { Inter } from 'next/font/google'

const inter = Inter({ subsets: ['latin'] })

export default function MyApp({ Component, pageProps }) {
  return (
    <>
      <style jsx global>{`
        html {
          font-family: ${inter.style.fontFamily};
        }
      `}</style>
      <Component {...pageProps} />
    </>
  )
}
```

## ローカルフォント {#local-fonts}

`next/font/local`をインポートし、ローカルフォントファイルの`src`を指定します。最良のパフォーマンスと柔軟性のために[可変フォント](https://fonts.google.com/variablefonts)を使用することをお勧めします。

## Tailwind CSSと併用する {#with-tailwind-css}

`next/font`は、[CSS変数](/docs/app/api-reference/components/font#css-variables)を介して[Tailwind CSS](https://tailwindcss.com/)で使用できます。

以下の例では、`next/font/google`（Googleまたはローカルフォントのいずれのフォントも使用可能）からフォント`Inter`を使用します。`variable`オプションを使用してCSS変数名を定義し、それを`inter`に割り当てます。その後、`inter.variable`を使用してHTMLドキュメントにCSS変数を追加します。

<AppOnly>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher

const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
  variable: '--font-inter',
})

const roboto_mono = Roboto_Mono({
  subsets: ['latin'],
  display: 'swap',
  variable: '--font-roboto-mono',
})

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={`${inter.variable} ${roboto_mono.variable}`}>
      <body>{children}</body>
    </html>
  )
}
```

</TabItem>

最後に、[Tailwind CSSの設定](/docs/app/building-your-application/styling/tailwind-css#configuring-tailwind)にCSS変数を追加します：

```js title="tailwind.config.js"
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    './pages/**/*.{js,ts,jsx,tsx}',
    './components/**/*.{js,ts,jsx,tsx}',
    './app/**/*.{js,ts,jsx,tsx}',
  ],
  theme: {
    extend: {
      fontFamily: {
        sans: ['var(--font-inter)'],
        mono: ['var(--font-roboto-mono)'],
      },
    },
  },
  plugins: [],
}
```

これで、`font-sans`と`font-mono`ユーティリティクラスを使用して要素にフォントを適用できるようになります。

## プリロード {#preloading}

<AppOnly>
サイトのページでフォント関数が呼び出されると、それはグローバルには提供されず、すべてのルートでプリロードされるわけではありません。むしろ、フォントは使用されるファイルの種類に基づいて関連するルートでのみプリロードされます：

- [ユニークページ](/docs/app/api-reference/file-conventions/page)の場合、そのページのユニークルートでプリロードされます
- [layout](/docs/app/building-your-application/routing/layouts-and-templates#layouts)に含まれている場合、そのlayoutでラップされているすべてのルートでプリロードされます
- [root レイアウト](/docs/app/building-your-application/routing/layouts-and-templates#root-layout-required)にある場合、すべてのルートでプリロードされます

</AppOnly>

<PagesOnly>

サイトのページでフォント関数が呼び出されると、それはグローバルには提供されず、すべてのルートでプリロードされるわけではありません。むしろ、フォントは使用されるファイルの種類に基づいて関連するルートでのみプリロードされます：

- [ユニークページ](https://nextjs.org/docs/canary/pages/building-your-application/routing/pages-and-layouts)なら、そのページのユニークルートでプリロードされます
- [カスタム App](https://nextjs.org/docs/canary/pages/building-your-application/routing/custom-app)にある場合、サイトのすべてのルート（`/pages`以下）がプリロードされます

</PagesOnly>

## フォントの再利用 {#reusing-fonts}

`localFont`またはGoogleフォント関数を呼び出すたびに、そのフォントがアプリケーションで1つのインスタンスとしてホストされます。そのため、複数のファイルで同じフォント関数を読み込むと、同じフォントの複数のインスタンスがホストされます。この状況では、以下を行うことをお勧めします：

- 共有ファイルでフォントローダー関数を呼び出す
- それを定数としてエクスポートする
- このフォントを使用したい各ファイルで定数をインポートする
