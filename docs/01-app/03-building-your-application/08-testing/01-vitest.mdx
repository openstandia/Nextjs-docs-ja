ViteとReact Testing Libraryはよく一緒に使われ、**ユニットテスト**に用いられます。このガイドでは、Next.jsでVitestをセットアップし、最初のテストを書く方法を紹介します。

> **Good to know:** React エコシステムにおいて `async` Server Components は新しいため、Vitestは現在これをサポートしていません。同期的なServer Component とClient Componentの**ユニットテスト**を実行することは可能ですが、`async` componentには**E2Eテスト**を使うことをお勧めします。

## クイックスタート {#quickstart}

`create-next-app`を使って、Next.jsの[with-vitest](https://github.com/vercel/next.js/tree/canary/examples/with-vitest)の例を使用することで、すぐに始められます：

```bash title="Terminal"
npx create-next-app@latest --example with-vitest with-vitest-app
```

## 手動セットアップ {#manual-setup}

Vitestを手動でセットアップするには、`vitest`と以下のパッケージを開発依存としてインストールします：

```bash title="Terminal"
# TypeScriptを使用する場合 {#using-typescript}
npm install -D vitest @vitejs/plugin-react jsdom @testing-library/react @testing-library/dom vite-tsconfig-paths
# JavaScriptを使用する場合 {#using-javascript}
npm install -D vitest @vitejs/plugin-react jsdom @testing-library/react @testing-library/dom
```

プロジェクトのrootに`vitest.config.mts|js`ファイルを作成し、以下のオプションを追加します：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="vitest.config.mts" switcher
import { defineConfig } from 'vitest/config'
import react from '@vitejs/plugin-react'
import tsconfigPaths from 'vite-tsconfig-paths'

export default defineConfig({
  plugins: [tsconfigPaths(), react()],
  test: {
    environment: 'jsdom',
  },
})
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="vitest.config.js" switcher
import { defineConfig } from 'vitest/config'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  test: {
    environment: 'jsdom',
  },
})
```

</TabItem>
</Tabs>

Vitestの設定に関する詳細については、[Vitest Configuration](https://vitest.dev/config/#configuration)ドキュメントをご参照ください。

次に、`package.json`に`test`スクリプトを追加します：

```json title="package.json"
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "test": "vitest"
  }
}
```

`npm run test`を実行すると、Vitestはプロジェクト内の変更を**監視**します。

## 最初のVitestユニットテストを作成する {#creating-your-first-vitest-unit-test}

`<Page />` componentが見出しを正常にレンダリングできるかをチェックするテストを作成して、すべてが正常に動作するか確認しましょう。

<AppOnly>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <div>
      <h1>Home</h1>
      <Link href="/about">About</Link>
    </div>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.jsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <div>
      <h1>Home</h1>
      <Link href="/about">About</Link>
    </div>
  )
}
```

</TabItem>
</Tabs>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="__tests__/page.test.tsx" switcher
import { expect, test } from 'vitest'
import { render, screen } from '@testing-library/react'
import Page from '../app/page'

test('Page', () => {
  render(<Page />)
  expect(screen.getByRole('heading', { level: 1, name: 'Home' })).toBeDefined()
})
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="__tests__/page.test.jsx" switcher
import { expect, test } from 'vitest'
import { render, screen } from '@testing-library/react'
import Page from '../app/page'

test('Page', () => {
  render(<Page />)

})
```

</TabItem>
</Tabs>

> **Good to know**: 上記の例では一般的な`__tests__`規約を使用していますが、テストファイルは`app router`内に配置することもできます。

## テストを実行する {#running-your-tests}

次のコマンドを実行してテストを実行します：

```bash title="Terminal"
npm run test
# または {#or}
yarn test
# または {#or}
pnpm test
# または {#or}
bun test
```
