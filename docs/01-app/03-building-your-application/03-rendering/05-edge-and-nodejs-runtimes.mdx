Next.jsでは、アプリケーションで使用できる2つのサーバーランタイムがあります：

- **Node.js Runtime**（デフォルト）: すべてのNode.js APIとエコシステムからの互換パッケージにアクセスできます
- **Edge Runtime**: より制限された[APIのセット](/docs/app/api-reference/edge)を含んでいます

## Use Cases {#use-cases}

- Node.js Runtimeは、アプリケーションのレンダリングに使用されます
- Edge Runtimeは、ミドルウェア（リダイレクト、書き換え、ヘッダーの設定などのルーティングルール）に使用されます

## Caveats {#caveats}

- Edge RuntimeはすべてのNode.js APIをサポートしていません。いくつかのパッケージは期待どおりに動作しない可能性があります。サポートされていないAPIについては、[Edge Runtime](/docs/app/api-reference/edge#unsupported-apis)で詳細を確認してください
- Edge RuntimeはIncremental Static Regeneration（ISR）をサポートしていません
- 両方のランタイムは、デプロイインフラストラクチャに応じて[ストリーミング](/docs/app/building-your-application/routing/loading-ui-and-streaming)をサポートできます
