Route Handlersを使用すると、Web [Request](https://developer.mozilla.org/docs/Web/API/Request)および[Response](https://developer.mozilla.org/docs/Web/API/Response) APIを使用して、特定のルートに対するカスタムリクエストハンドラーを作成できます。

<Image
  alt="Route.js Special File"
  srcLight="/docs/light/route-special-file.png"
  srcDark="/docs/dark/route-special-file.png"
  width="1600"
  height="444"
/>

> **Good to know**: Route Handlersは`app`ディレクトリ内でのみ利用可能です。これらは、`pages`ディレクトリ内の[API Routes](https://nextjs.org/docs/canary/pages/building-your-application/routing/api-routes)の相当部分であるため、API RoutesとRoute Handlersを一緒に使用する必要は**ありません**。

## 規約 {#convention}

Route Handlersは、`app`ディレクトリ内の[`route.js|ts`ファイル](/docs/app/api-reference/file-conventions/route)で定義されます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/route.ts" switcher
export async function GET(request: Request) {}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/api/route.js" switcher
export async function GET(request) {}
```

</TabItem>
</Tabs>

Route Handlersは、`page.js`や`layout.js`と同様に、`app`ディレクトリ内でネストできます。ただし、`page.js`と同じルートセグメントレベルに`route.js`ファイルを置くことは**できません**。

### サポートされているHTTPメソッド {#supported-http-methods}

次の[HTTPメソッド](https://developer.mozilla.org/docs/Web/HTTP/Methods)がサポートされています：`GET`、`POST`、`PUT`、`PATCH`、`DELETE`、`HEAD`、および `OPTIONS`。サポートされていないメソッドが呼び出された場合、Next.jsは`405 Method Not Allowed`の応答を返します。

### 拡張されたNextRequestとNextResponseのAPI {#extended-nextrequest-and-nextresponse-apis}

Next.jsは、ネイティブな[Request](https://developer.mozilla.org/docs/Web/API/Request)および[Response](https://developer.mozilla.org/docs/Web/API/Response) APIのサポートに加えて、[`NextRequest`](/docs/app/api-reference/functions/next-request)および[`NextResponse`](/docs/app/api-reference/functions/next-response)を拡張して、より高度なユースケース向けの便利なヘルパーを提供します。

## 振る舞い {#behavior}

### キャッシング {#caching}

Route Handlersはデフォルトでキャッシュされません。ただし、`GET`メソッドについてはキャッシングを選択することができます。他のサポートされているHTTPメソッドは**キャッシュされません**。`GET`メソッドをキャッシュするには、Route Handlerファイル内で`export const dynamic = 'force-static'`のような[route config option](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を使用してください。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/items/route.ts" switcher
export const dynamic = 'force-static'

export async function GET() {
  const res = await fetch('https://data.mongodb-api.com/...', {
    headers: {
      'Content-Type': 'application/json',
      'API-Key': process.env.DATA_API_KEY,
    },
  })
  const data = await res.json()

  return Response.json({ data })
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/items/route.js" switcher
export const dynamic = 'force-static'

export async function GET() {
  const res = await fetch('https://data.mongodb-api.com/...', {
    headers: {
      'Content-Type': 'application/json',
      'API-Key': process.env.DATA_API_KEY,
    },
  })
  const data = await res.json()

  return Response.json({ data })
}
```

</TabItem>
</Tabs>

> **Good to know**: 他のサポートされているHTTPメソッドは**キャッシュされません**。たとえ同じファイル内でキャッシュされた`GET`メソッドと一緒に配置されてもです。

### 特別なRoute Handlers {#special-route-handlers}

[`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap)、[`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons)のような特別なRoute Handlersおよびその他の[メタデータファイル](/docs/app/api-reference/file-conventions/metadata)は、Dynamic APIや動的な設定オプションを使用しない限り、デフォルトでは静的に保たれます。

### ルートの解決 {#route-resolution}

`route`は最下位レベルのルーティングプリミティブと考えられます。

- これらは、`page`のようにレイアウトやクライアントサイドナビゲーションには**参加しません**。
- `page.js`と同じルートに`route.js`ファイルを置くことは**できません**。

| Page                 | Route              | Result                       |
| -------------------- | ------------------ | ---------------------------- |
| `app/page.js`        | `app/route.js`     | <Cross size={18} /> Conflict |
| `app/page.js`        | `app/api/route.js` | <Check size={18} /> Valid    |
| `app/[user]/page.js` | `app/api/route.js` | <Check size={18} /> Valid    |

各`route.js`または`page.js`ファイルは、そのルートに対するすべてのHTTP動詞を引き受けます。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/page.ts" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}

// ❌ Conflict
// `app/route.ts`
export async function POST(request: Request) {}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}

// ❌ Conflict
// `app/route.js`
export async function POST(request) {}
```

</TabItem>
</Tabs>

### Cookie {#cookies}

`next/headers`の[`cookies`](/docs/app/api-reference/functions/cookies)を使用して、cookieを読み取ったり設定したりできます。このサーバ関数はRoute Handler内部で直接呼び出すことも、他の関数内にネストして呼び出すことも可能です。

あるいは、[`Set-Cookie`](https://developer.mozilla.org/docs/Web/HTTP/Headers/Set-Cookie)ヘッダーを使用して新しい`Response`を返すこともできます。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/route.ts" switcher
import { cookies } from 'next/headers'

export async function GET(request: Request) {
  const cookieStore = await cookies()
  const token = cookieStore.get('token')

  return new Response('Hello, Next.js!', {
    status: 200,
    headers: { 'Set-Cookie': `token=${token.value}` },
  })
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/api/route.js" switcher
import { cookies } from 'next/headers'

export async function GET(request) {
  const cookieStore = await cookies()
  const token = cookieStore.get('token')

  return new Response('Hello, Next.js!', {
    status: 200,
    headers: { 'Set-Cookie': `token=${token}` },
  })
}
```

</TabItem>
</Tabs>

リクエストからcookieを読み取るために、基盤となるWeb APIを使用することも可能です（[`NextRequest`](/docs/app/api-reference/functions/next-request)）：


</TabItem>

### ヘッダー {#headers}

`next/headers`の[`headers`](/docs/app/api-reference/functions/headers)を使用してヘッダーを読み取ることができます。このサーバ関数はRoute Handlerで直接呼び出したり、他の関数内にネストして呼び出すことも可能です。

この`headers`インスタンスは読み取り専用です。ヘッダーを設定するには、新しい`headers`で新しい`Response`を返す必要があります。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/route.ts" switcher
import { headers } from 'next/headers'

export async function GET(request: Request) {
  const headersList = await headers()
  const referer = headersList.get('referer')

  return new Response('Hello, Next.js!', {
    status: 200,
    headers: { referer: referer },
  })
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/api/route.js" switcher
import { headers } from 'next/headers'

export async function GET(request) {

    headers: { referer: referer },
  })
```
```js title="app/api/route.js" switcher
}
```

</TabItem>
</Tabs>

### 動的ルートセグメント {#dynamic-route-segments}

Route Handlersは、[Dynamic Segments](/docs/app/building-your-application/routing/dynamic-routes)を使用して動的データからリクエストハンドラーを作成できます。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/items/[slug]/route.ts" switcher
export async function GET(
  request: Request,
  { params }: { params: Promise<{ slug: string }> }
) {
  const slug = (await params).slug // 'a', 'b', or 'c'
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/items/[slug]/route.js" switcher
export async function GET(request, { params }) {
  const slug = (await params).slug // 'a', 'b', or 'c'
}
```

</TabItem>
</Tabs>

| Route                       | Example URL | `params`                 |
| --------------------------- | ----------- | ------------------------ |
| `app/items/[slug]/route.js` | `/items/a`  | `Promise<{ slug: 'a' }>` |
| `app/items/[slug]/route.js` | `/items/b`  | `Promise<{ slug: 'b' }>` |
| `app/items/[slug]/route.js` | `/items/c`  | `Promise<{ slug: 'c' }>` |

### URLクエリパラメーター {#url-query-parameters}

Route Handlerに渡されるリクエストオブジェクトは`NextRequest`インスタンスであり、クエリパラメーターを簡単に処理するための[いくつかの追加の便利なメソッド](/docs/app/api-reference/functions/next-request#nexturl)を備えています。

### ストリーミング {#streaming}

ストリーミングは、OpenAIのようなLarge Language Models（LLMs）とAI生成コンテンツを組み合わせる場合によく使用されます。[AI SDK](https://sdk.vercel.ai/docs/introduction)について詳しく学べます。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/chat/route.ts" switcher
import { openai } from '@ai-sdk/openai'
import { StreamingTextResponse, streamText } from 'ai'

export async function POST(req: Request) {
  const { messages } = await req.json()
  const result = await streamText({
    model: openai('gpt-4-turbo'),
    messages,
  })

  return new StreamingTextResponse(result.toAIStream())
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/api/chat/route.js" switcher

    model: openai('gpt-4-turbo'),
    messages,
  })

  return new StreamingTextResponse(result.toAIStream())
}
```

</TabItem>
</Tabs>

これらの抽象化は、Web APIを使用してストリームを作成します。基盤となるWeb APIを直接使用することもできます。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/route.ts" switcher
// https://developer.mozilla.org/docs/Web/API/ReadableStream#convert_async_iterator_to_stream
function iteratorToStream(iterator: any) {
  return new ReadableStream({
    async pull(controller) {
      const { value, done } = await iterator.next()

      if (done) {
        controller.close()
      } else {
        controller.enqueue(value)
      }
    },
  })
}

function sleep(time: number) {
  return new Promise((resolve) => {
    setTimeout(resolve, time)
  })
}

const encoder = new TextEncoder()

async function* makeIterator() {
  yield encoder.encode('<p>One</p>')
  await sleep(200)
  yield encoder.encode('<p>Two</p>')
  await sleep(200)
  yield encoder.encode('<p>Three</p>')
}

export async function GET() {
  const iterator = makeIterator()
  const stream = iteratorToStream(iterator)

  return new Response(stream)
}
```

</TabItem>
