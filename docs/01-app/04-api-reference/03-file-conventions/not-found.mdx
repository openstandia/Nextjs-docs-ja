**not-found** ファイルは、ルートセグメント内で [`notFound`](/docs/app/api-reference/functions/not-found) 関数がスローされたときに UI をレンダリングするために使用されます。カスタム UI を提供するだけでなく、Next.js はストリームレスポンスには`200` HTTP ステータスコードを、非ストリームレスポンスには`404`を返します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

## リファレンス {#reference}

### Props {#props}

`not-found.js` コンポーネントは、どの props も受け取りません。

> **Good to know**: 予期される `notFound()` エラーをキャッチすることに加えて、root `app/not-found.js` ファイルはアプリケーション全体の一致しない URL も処理します。つまり、アプリが対処しない URL にアクセスしたユーザーには、`app/not-found.js` ファイルによってエクスポートされた UI が表示されます。

## 例 {#examples}

### データフェッチング {#data-fetching}

デフォルトで、`not-found` は Server Component です。データを取得して表示するために `async` としてマークできます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

</TabItem>
