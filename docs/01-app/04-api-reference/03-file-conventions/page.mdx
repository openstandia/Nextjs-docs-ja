`page`ファイルは、ルートに**固有**のUIを定義することを可能にします。ファイルからコンポーネントをデフォルトでエクスポートすることで、ページを作成できます；

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  return <h1>My Page</h1>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default function Page({ params, searchParams }) {
  return <h1>My Page</h1>
}
```

</TabItem>
</Tabs>

## Good to know {#good-to-know}

- `page`には`.js`、`.jsx`、または`.tsx`の拡張子を使用できます
- `page`は常にルートsubtreeの**leaf**です
- `page`ファイルは、ルートセグメントを**公開アクセス可能**にするために必要です
- ページはデフォルトで[Server Components](https://react.dev/reference/rsc/server-components)ですが、[Client Component](https://react.dev/reference/rsc/use-client)に設定することができます

## Reference {#reference}

### Props {#props}

#### `params`（オプション） {#params-optional}

rootセグメントからそのページまでの[動的ルートパラメータ](/docs/app/building-your-application/routing/dynamic-routes)を含むオブジェクトに解決するプロミス；

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const slug = (await params).slug
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/[slug]/page.js" switcher
export default async function Page({ params }) {
  const slug = (await params).slug
}
```

</TabItem>
</Tabs>

| 例ルート                             | URL         | `params`                                |
| ------------------------------------ | ----------- | --------------------------------------- |
| `app/shop/[slug]/page.js`            | `/shop/1`   | `Promise<{ slug: '1' }>`                |
| `app/shop/[category]/[item]/page.js` | `/shop/1/2` | `Promise<{ category: '1', item: '2' }>` |
| `app/shop/[...slug]/page.js`         | `/shop/1/2` | `Promise<{ slug: ['1', '2'] }>`         |

- `params`はプロミスであるため、値にアクセスするには`async/await`やReactの[`use`](https://react.dev/reference/react/use)関数を使う必要があります
  - バージョン14以前では、`params`は同期的なpropでした。後方互換性のために、Next.js 15では同期的にアクセスできますが、この動作は将来非推奨になる予定です

#### `searchParams`（オプション） {#searchparams-optional}

現在のURLの[検索パラメータ](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)を含むオブジェクトに解決するプロミス；

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const filters = (await searchParams).filters
}
```

</TabItem>
</Tabs>

| 例URL           | `searchParams`                |
| --------------- | ----------------------------- |
| `/shop?a=1`     | `Promise<{ a: '1' }>`         |
| `/shop?a=1&b=2` | `Promise<{ a: '1', b: '2' }>` |
| `/shop?a=1&a=2` | `Promise<{ a: ['1', '2'] }>`  |

- `searchParams`はプロミスであるため、値にアクセスするには`async/await`やReactの[`use`](https://react.dev/reference/react/use)関数を使う必要があります
  - バージョン14以前では、`searchParams`は同期的なpropでした。後方互換性のために、Next.js 15では同期的にアクセスできますが、この動作は将来非推奨になる予定です
- `searchParams`は、その値を事前に知ることができない**[ダイナミックAPI](/docs/app/building-your-application/rendering/server-components#server-rendering-strategies#dynamic-apis)**です。これを使用すると、ページはリクエスト時に**[ダイナミックレンダリング](/docs/app/building-your-application/rendering/server-components#dynamic-rendering)**に切り替わります
- `searchParams`は`URLSearchParams`インスタンスではなく、単なるJavaScriptオブジェクトです

## Examples {#examples}

### `params`に基づいたコンテンツの表示 {#displaying-content-based-on-params}

[動的ルートセグメント](/docs/app/building-your-application/routing/dynamic-routes)を使用して、`params`propに基づいた特定のコンテンツを表示または取得することができます；

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

</TabItem>
</Tabs>

### `searchParams`によるフィルタ処理の対応 {#handling-filtering-with-searchparams}

`searchParams`propを使用して、URLのクエリ文字列に基づいてフィルタリング、ページング、またはソートを行うことができます；

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

</TabItem>
</Tabs>

### クライアントコンポーネントでの`searchParams`と`params`の読み取り {#reading-searchparams-and-params-in-client-components}

クライアントコンポーネントでは`async`が使えないため、`searchParams`と`params`をReactの[`use`](https://react.dev/reference/react/use)関数でプロミスを読み取ります；

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
'use client'

import { use } from 'react'

export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/page.js" switcher
'use client'

import { use } from 'react'

export default function Page({ params, searchParams }) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

</TabItem>
</Tabs>
