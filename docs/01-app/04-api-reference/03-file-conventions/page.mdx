`page` ファイルは、ルートに**固有**のUIを定義することを可能にします。ファイルからコンポーネントをデフォルトエクスポートすることでページを作成できます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  return <h1>My Page</h1>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default function Page({ params, searchParams }) {
  return <h1>My Page</h1>
}
```

</TabItem>
</Tabs>

## Good to know {#good-to-know}

- `page` には `.js`、`.jsx`、または `.tsx` ファイル拡張子を使用できます
- `page` は常にルート subtree の**leaf**です
- ルートセグメントを**公にアクセス可能**にするには、`page` ファイルが必要です
- ページはデフォルトで[Server Components](https://react.dev/reference/rsc/server-components)ですが、[Client Component](https://react.dev/reference/rsc/use-client) に設定可能です

## Reference {#reference}

### Props {#props}

#### `params`（オプション） {#params-optional}

root セグメントからそのページまでの[動的ルートパラメータ](/docs/app/building-your-application/routing/dynamic-routes)を含むオブジェクトに解決される Promise です。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const slug = (await params).slug
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/[slug]/page.js" switcher
export default async function Page({ params }) {
  const slug = (await params).slug
}
```

</TabItem>
</Tabs>

| 例のルート                           | URL         | `params`                                |
| ------------------------------------ | ----------- | --------------------------------------- |
| `app/shop/[slug]/page.js`            | `/shop/1`   | `Promise<{ slug: '1' }>`                |
| `app/shop/[category]/[item]/page.js` | `/shop/1/2` | `Promise<{ category: '1', item: '2' }>` |
| `app/shop/[...slug]/page.js`         | `/shop/1/2` | `Promise<{ slug: ['1', '2'] }>`         |

- `params` prop は Promise であるため、値にアクセスするには `async/await` または React の [`use`](https://react.dev/reference/react/use) 関数を使用する必要があります
  - バージョン14以前では、`params` は同期 prop でした。下位互換性を考慮して、Next.js 15 ではまだ同期的にアクセスできますが、この動作は将来的に廃止される予定です

#### `searchParams`（オプション） {#searchparams-optional}

現在の URL の[検索パラメータ](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)を含むオブジェクトに解決される Promise です。例えば：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const filters = (await searchParams).filters
}
```

</TabItem>
</Tabs>

| 例のURL         | `searchParams`                |
| --------------- | ----------------------------- |
| `/shop?a=1`     | `Promise<{ a: '1' }>`         |
| `/shop?a=1&b=2` | `Promise<{ a: '1', b: '2' }>` |
| `/shop?a=1&a=2` | `Promise<{ a: ['1', '2'] }>`  |

- `searchParams` prop は Promise であるため、値にアクセスするには `async/await` または React の [`use`](https://react.dev/reference/react/use) 関数を使用する必要があります
  - バージョン14以前では、`searchParams` は同期 prop でした。下位互換性を考慮して、Next.js 15 ではまだ同期的にアクセスできますが、この動作は将来的に廃止される予定です
- `searchParams` は、その値を事前に知ることができない**[Dynamic API](/docs/app/building-your-application/rendering/server-components#server-rendering-strategies#dynamic-apis)**です。これを使用すると、リクエスト時にページを**[動的レンダリング](/docs/app/building-your-application/rendering/server-components#dynamic-rendering)** するようオプトインされます
- `searchParams` は単なる JavaScript オブジェクトであり、`URLSearchParams` インスタンスではありません

## Examples {#examples}

### `params` に基づくコンテンツ表示 {#displaying-content-based-on-params}

[動的ルートセグメント](/docs/app/building-your-application/routing/dynamic-routes)を使用して、`params` prop に基づいて特定のコンテンツをページに表示または取得することができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

</TabItem>
</Tabs>

### `searchParams` を使用したフィルタリングの処理 {#handling-filtering-with-searchparams}

URL のクエリ文字列に基づいてフィルタリング、ページネーション、またはソートを処理するために `searchParams` prop を使用できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

</TabItem>
</Tabs>

### Client Componentでの `searchParams` と `params` の読み取り {#reading-searchparams-and-params-in-client-components}

Client Component では`searchParams` および `params` (非 `async` 可能) を使用するために、React の [`use`](https://react.dev/reference/react/use) 関数を使用して、その Promise を読み取ります：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
'use client'

import { use } from 'react'

export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/page.js" switcher
'use client'

import { use } from 'react'

export default function Page({ params, searchParams }) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

</TabItem>
</Tabs>
