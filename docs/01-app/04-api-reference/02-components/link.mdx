`<Link>` は、[プレフェッチ](/docs/app/building-your-application/routing/linking-and-navigating#2-prefetching)とルート間のクライアントサイドナビゲーションを提供するために HTML の `<a>` 要素を拡張した React コンポーネントです。これは Next.js でルート間をナビゲートするための主要な方法です。

基本的な使用法：

```tsx title="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

## リファレンス {#reference}

以下の props を `<Link>` コンポーネントに渡すことができます：

<PagesOnly>

| Prop                                | 例                      | 型                    | 必須 |
| ----------------------------------- | ----------------------- | --------------------- | ---- |
| [`href`](#href-required)            | `href="/dashboard"`     | String または Object  | はい |
| [`replace`](#replace)               | `replace={false}`       | Boolean               | -    |
| [`scroll`](#scroll)                 | `scroll={false}`        | Boolean               | -    |
| [`prefetch`](#prefetch)             | `prefetch={false}`      | Boolean               | -    |
| [`legacyBehavior`](#legacybehavior) | `legacyBehavior={true}` | Boolean               | -    |
| [`passHref`](#passhref)             | `passHref={true}`       | Boolean               | -    |
| [`shallow`](#shallow)               | `shallow={false}`       | Boolean               | -    |
| [`locale`](#locale)                 | `locale="fr"`           | String または Boolean | -    |

</PagesOnly>

<AppOnly>

| Prop                     | 例                  | 型                   | 必須 |
| ------------------------ | ------------------- | -------------------- | ---- |
| [`href`](#href-required) | `href="/dashboard"` | String または Object | はい |
| [`replace`](#replace)    | `replace={false}`   | Boolean              | -    |
| [`scroll`](#scroll)      | `scroll={false}`    | Boolean              | -    |
| [`prefetch`](#prefetch)  | `prefetch={false}`  | Boolean または null  | -    |

</AppOnly>

> **Good to know**: `className` や `target="_blank"` などの `<a>` タグ属性は `<Link>` に props として追加でき、基礎となる `<a>` 要素に渡されます。

### `scroll` {#scroll}

**デフォルトは `true` です。** Next.js の `<Link>` のデフォルトのスクロール動作は **スクロール位置を保持すること** であり、これはブラウザが前後のナビゲーションを扱う方法に似ています。新しい[ページ](/docs/app/api-reference/file-conventions/page)にナビゲートすると、ページがビューポート内に表示され続ける限り、スクロール位置は保持されます。ただし、ページがビューポートに表示されていない場合、Next.js は最初のページ要素の上部にスクロールします。

`scroll = {false}` の場合、Next.js は最初のページ要素にスクロールしようとはしません。

> **Good to know**: Next.js は `scroll: false` であることを確認してからスクロール動作を管理します。スクロールが有効な場合、それはナビゲーションのための関連する DOM ノードを識別し、各トップレベル要素を検査します。すべてのスクロール不可能な要素およびレンダリングされていない HTML の要素はスキップされます。これには、`getBoundingClientRect` で計算される要素や非表示の要素など、sticky や固定位置の要素が含まれます。Next.js は次にビューポート内に表示されるスクロール可能な要素を識別するまで兄弟要素を続けます。

```tsx title="app/page.tsx" switcher
  return (
    <Link href="/dashboard" scroll={false}>
      Dashboard
    </Link>
  )
}
```

</TabItem>

### `prefetch` {#prefetch}

<AppOnly>

プレフェッチは、`<Link />` コンポーネントがユーザーのビューポートに入ったとき（最初またはスクロールによって）に発生します。Next.js は、リンクされたルート（`href` で示される）とそのデータをバックグラウンドで事前に取得して、クライアントサイドのナビゲーションのパフォーマンスを向上させます。ユーザーが `<Link />` をホバーした時点でプレフェッチされたデータが期限切れの場合、Next.js は再びプレフェッチを試みます。**プレフェッチは本番環境でのみ有効です。**

`prefetch` props に渡すことのできる値は以下のとおりです：

- **`null` （デフォルト）**: プレフェッチの動作は、ルートが静的か動的かに依存します。静的ルートの場合、完全なルートがプレフェッチされます（すべてのデータを含む）。動的ルートの場合、[`loading.js`](/docs/app/building-your-application/routing/loading-ui-and-streaming#instant-loading-states) 境界までの部分ルートがプレフェッチされます。
- `true`: 静的および動的両方のルートで完全なルートがプレフェッチされます。
- `false`: ビューポートに入る際、またはホバーの際にプレフェッチは一切行われません。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</TabItem>
</Tabs>

</AppOnly>

<PagesOnly>

プレフェッチは、`<Link />` コンポーネントがユーザーのビューポートに入ったとき（最初またはスクロールによって）に発生します。Next.js は、リンクされたルート（`href` で示される）とデータをバックグラウンドで事前にロードして、クライアントサイドナビゲーションのパフォーマンスを向上させます。**プレフェッチは本番環境でのみ有効です。**

`prefetch` props に渡すことのできる値は以下のとおりです：

- **`true` （デフォルト）**: 完全なルートとそのデータがプレフェッチされます。
- `false`: ビューポートに入る際のプレフェッチは行われませんが、ホバー時に行われます。ホバー時の取得も完全に削除したい場合は、`<a>` タグを使用するか、App Router を[段階的に導入](/docs/app/building-your-application/upgrading/app-router-migration)することで、ホバー時のプレフェッチも無効にできます。

### `legacyBehavior` {#legacybehavior}

`<a>` 要素は、もはや `<Link>` の子として必須ではありません。レガシーな動作を使用するために `legacyBehavior` prop を追加するか、コードをアップグレードするために `<a>` 要素を削除できます。自動的にコードをアップグレードするための [codemod が利用可能](/docs/app/building-your-application/upgrading/codemods#new-link)です。

> **Good to know**: `legacyBehavior` が `true` に設定されていない場合、`className`、`onClick` など、すべての[`アンカー`](https://developer.mozilla.org/docs/Web/HTML/Element/a)タグプロパティを `next/link` に渡すことが可能です。

### `passHref` {#passhref}

`Link` にその子に `href` プロパティを強制的に送信させます。デフォルトは `false` です。[機能的なコンポーネントを渡す](#nesting-a-functional-component)例を参照してください。

### `locale` {#locale}

アクティブなロケールは自動的に先頭に付きます。`locale` は異なるロケールを指定することを可能にします。`false` の場合、デフォルトの動作が無効になっているので `href` はロケールを含む必要があります。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="pages/index.tsx" switcher
import Link from 'next/link'

export default function Home() {
  return (
    <>
      {/* デフォルトの動作：ロケールが先頭につく */}
      <Link href="/dashboard">Dashboard (with locale)</Link>

      {/* ロケールの先頭付きを無効にする */}
      <Link href="/dashboard" locale={false}>
        Dashboard (without locale)
      </Link>

      {/* 異なるロケールを指定する */}
      <Link href="/dashboard" locale="fr">
        Dashboard (French)
      </Link>
    </>
  )
}
```

### アクティブラリンクの確認 {#checking-active-links}

[`usePathname()`](/docs/app/api-reference/functions/use-pathname)を使用して、リンクがアクティブかどうかを確認できます。たとえば、アクティブなリンクにクラスを追加するには、現在の `pathname` がリンクの `href` と一致するかどうかを確認します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/ui/nav-links.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'
import Link from 'next/link'

export function Links() {
  const pathname = usePathname()

  return (
    <nav>
      <Link className={`link ${pathname === '/' ? 'active' : ''}`} href="/">
        Home
      </Link>

      <Link
        className={`link ${pathname === '/about' ? 'active' : ''}`}
        href="/about"
      >
        About
      </Link>
    </nav>
  )
}
```

</TabItem>

### `id` にスクロール {#scrolling-to-an-id}

ナビゲーションで特定の `id` にスクロールしたい場合、URL に `#` ハッシュリンクを追加するか、ハッシュリンクを `href` prop に渡すことができます。これは、`<Link>` が `<a>` 要素にレンダリングされるため、可能です。

```jsx
<Link href="/dashboard#settings">Settings</Link>

// 出力
<a href="/dashboard#settings">Settings</a>
```

> **Good to know**:
>
> - Next.js は、ナビゲーション時にビューポートに見えていない場合、[ページ](/docs/app/api-reference/file-conventions/page)までスクロールします。

</AppOnly>

### 子が `<a>` タグをラップするカスタムコンポーネントの場合 {#if-the-child-is-a-custom-component-that-wraps-an-a-tag}

<AppOnly>

`Link` の子が `<a>` タグをラップするカスタムコンポーネントである場合は、`Link` に `passHref` を追加する必要があります。これは、たとえば[styled-components](https://styled-components.com/)のようなライブラリを使用する場合に必要です。これがないと、`<a>` タグに `href` 属性がつかず、サイトのアクセシビリティを損ない、SEOに影響を与える可能性があります。[ESLint](https://nextjs.org/docs/canary/pages/api-reference/config/eslint)を使用している場合は、`passHref` の正しい使用を保証するための組み込みルール `next/link-passhref` があります。

</AppOnly>

### 機能的なコンポーネントを入れ子にする {#nesting-a-functional-component}

`Link` の子が機能的なコンポーネントである場合、`passHref` と `legacyBehavior` を使用することに加えて、コンポーネントを [`React.forwardRef`](https://react.dev/reference/react/forwardRef)でラップする必要があります：

<AppOnly>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import Link from 'next/link'
import React from 'react'

// MyButton の props タイプを定義する
interface MyButtonProps {
  onClick?: React.MouseEventHandler<HTMLAnchorElement>
  href?: string
}

// 転送された ref を適切に型付けするために React.ForwardRefRenderFunction を使用する
const MyButton: React.ForwardRefRenderFunction<
  HTMLAnchorElement,
  MyButtonProps
> = ({ onClick, href }, ref) => {
  return (
    <a href={href} onClick={onClick} ref={ref}>
      Click Me
    </a>
  )
}

// React.forwardRef を使用してコンポーネントをラップする
const ForwardedMyButton = React.forwardRef(MyButton)

export default function Page() {
  return (
    <Link href="/about" passHref legacyBehavior>
      <ForwardedMyButton />
    </Link>
  )
}
```

</TabItem>

### ページの上部へのスクロールを無効にする {#disable-scrolling-to-the-top-of-the-page}

<AppOnly>

Next.js での `<Link>` のデフォルトのスクロール動作は **スクロール位置を保持すること** であり、これはブラウザが前後のナビゲーションを扱う方法に似ています。新しい[ページ](/docs/app/api-reference/file-conventions/page)にナビゲートすると、ページがビューポート内に表示され続ける限り、スクロール位置は保持されます。

ただし、ページがビューポートに表示されていない場合、Next.js は最初のページ要素の上部にスクロールします。この動作を無効にしたい場合、`<Link>` コンポーネントに `scroll={false}` を渡すか `router.push()` または `router.replace()` に `scroll: false` を渡せます。

<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher

      Disables scrolling to the top
    </Link>
  )
}
```

</TabItem>


### ミドルウェアでのリンクのプレフェッチ {#prefetching-links-in-middleware}

ユーザーを別のページにリダイレクトするような認証やその他の目的で[ミドルウェア](/docs/app/building-your-application/routing/middleware)を使用することは一般的です。ミドルウェアを介したリライトで `<Link />` コンポーネントがリンクを適切にプレフェッチするためには、Next.js に表示する URL とプレフェッチする URL の両方を教える必要があります。これはミドルウェアに正しいルートをプレフェッチするために不要なフェッチを避ける必要があります。

たとえば、認証済みおよび訪問者ビューを持つ `/dashboard` ルートを提供したい場合、ミドルウェアでユーザーを正しいページにリダイレクトするために次のコードを追加できます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="middleware.ts" switcher
import { NextResponse } from 'next/server'

export function middleware(request: Request) {
  const nextUrl = request.nextUrl
  if (nextUrl.pathname === '/dashboard') {
    if (request.cookies.authToken) {
      return NextResponse.rewrite(new URL('/auth/dashboard', request.url))
    } else {
      return NextResponse.rewrite(new URL('/public/dashboard', request.url))
    }
  }
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

