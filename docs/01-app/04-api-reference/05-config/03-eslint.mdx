Next.jsは、Next.jsアプリケーションで一般的な問題をキャッチするためのESLintプラグイン、[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next) を基本設定にすでに含んでいます。

## リファレンス {#reference}

`eslint-config-next`には、次のESLintプラグインの推奨ルールセットがすべて使用されています：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これは `next.config.js` の設定よりも優先されます。

## 例 {#examples}

### カスタムディレクトリとファイルのリント {#linting-custom-directories-and-files}

デフォルトでは、Next.jsは`pages/`、`app/`、`components/`、`lib/`、および`src/`ディレクトリ内のすべてのファイルに対してESLintを実行します。ただし、プロダクションビルドのための`next.config.js`内の`eslint`設定で`dirs`オプションを使用して、特定のディレクトリを指定することもできます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // プロダクションビルド中（next build）は、'pages'と'utils'ディレクトリのみでESLintを実行
  },
}
```

同様に、`next lint`に`--dir`および`--file`フラグを使用して、特定のディレクトリおよびファイルをリントすることもできます：

```bash title="Terminal"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内でのrootディレクトリの指定 {#specifying-a-root-directory-within-a-monorepo}

Next.jsがインストールされていないrootディレクトリで`eslint-plugin-next`を使用するプロジェクトでは、`.eslintrc`内の`settings`プロパティを使用して、Next.jsアプリケーションがどこにあるかを`eslint-plugin-next`に伝えることができます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  }),
]

export default eslintConfig
```

`rootDir`は、パス（相対または絶対）、グロブ（例：`"packages/*/"`）、またはパス/グロブの配列のいずれかとなります。

### キャッシュを無効にする {#disabling-the-cache}

パフォーマンス向上のため、ESLintが処理したファイルの情報はデフォルトでキャッシュされます。これは`.next/cache`または定義した[ビルドディレクトリ](/docs/app/api-reference/config/next-config-js/distDir)に保存されます。キャッシュを無効にする必要がある場合は、`next lint`で`--no-cache`フラグを使用します：

```bash title="Terminal"
next lint --no-cache
```

### ルールの無効化 {#disabling-rules}

サポートされているプラグイン（`react`、`react-hooks`、`next`）によって提供されるルールを変更または無効にしたい場合は、直接`.eslintrc`の`rules`プロパティを使用して変更できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  }),
]

export default eslintConfig
```

### コア Web Vitals と共に使用する場合 {#with-core-web-vitals}

最初に`next lint`を実行し、**strict**オプションを選択すると、`next/core-web-vitals`ルールセットが有効になります。

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals'],
  }),
]

export default eslintConfig
```

`next/core-web-vitals`は、`eslint-plugin-next`を更新して、[Core Web Vitals](https://web.dev/vitals/)に影響を与えるデフォルトで警告となるいくつかのルールをエラーに設定します。

> `next/core-web-vitals`エントリポイントは、[Create Next App](/docs/app/api-reference/cli/create-next-app)で作成された新しいアプリケーションに自動的に含まれています。

### TypeScript と共に使用する場合 {#with-typescript}

Next.js ESLintルールに加えて、`create-next-app --typescript`はTypeScript専用のリントルールを`next/typescript`で設定に追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals', 'next/typescript'],
  }),
]

export default eslintConfig
```

これらのルールは[`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)に基づいています。
詳細については、[typescript-eslint > Configs](https://typescript-eslint.io/linting/configs)を参照してください。

### Prettier と共に使用する場合 {#with-prettier}

ESLintにはコードフォーマットルールも含まれており、既存の[Prettier](https://prettier.io/)設定と競合する可能性があります。ESLintとPrettierを共存させるために、[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier)をESLint設定に含めることをお勧めします。

まず、依存関係をインストールします：

```bash title="Terminal"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

次に、`prettier`を既存のESLint設定に追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next', 'prettier'],
  }),
]

export default eslintConfig
```

### ステージングされたファイルに対するリントの実行 {#running-lint-on-staged-files}

[lint-staged](https://github.com/okonet/lint-staged)と共に`next lint`を使用してステージングされたGitファイルに対してリンターを実行する場合、`--file`フラグを使用するために、プロジェクトのrootにある`.lintstagedrc.js`ファイルに次の内容を追加します：

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

  eslint: {

  },

### 既存の設定の移行 {#migrating-existing-config}

すでにアプリケーションでESLintを設定している場合、いくつかの条件が満たされない限り、`eslint-config-next`を含めるのではなく、このプラグインから直接拡張することをお勧めします。

#### 推奨プラグインルールセット {#recommended-plugin-ruleset}

次の条件がすべて真である場合：

- 既に次のプラグインのいずれかを、別個にまたは`airbnb`や`react-app`など別の設定を通じてインストールしている：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- BabelがNext.js内で設定されている方法とは異なる、特定の`parserOptions`を定義している（[Babelの設定をカスタマイズした場合](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)を除き、これは推奨されません）
- `eslint-plugin-import`をNode.jsおよび/またはTypeScriptの解決に使用しており、[解決策](https://github.com/benmosher/eslint-plugin-import#resolvers)をインポートの処理に定義している

この場合、次のいずれかを推奨します：これらのプロパティの設定方法が[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js)で適切であればこれらの設定を削除するか、または直接Next.js ESLintプラグインから拡張します：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

このプラグインは、プロジェクトに通常通りインストールでき、`next lint`を実行する必要はありません：

```bash title="Terminal"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これにより、複数の設定にわたって同じプラグインやパーサーをインポートすることによって発生する衝突やエラーのリスクを排除します。

#### 追加設定 {#additional-configurations}

すでに別のESLint設定を使用していて、`eslint-config-next`を含める場合は、他の設定の後に最後に拡張されるようにします。例えば：

```mjs title="eslint.config.mjs"
import js from '@eslint/js'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で利用可能
  baseDirectory: import.meta.dirname,
  recommendedConfig: js.configs.recommended,
})

const eslintConfig = [
  ...compat.config({
    extends: ['eslint:recommended', 'next'],
  }),
]

export default eslintConfig
```

`next`設定は、`parser`、`plugins`、`settings`プロパティのデフォルト値を設定する役割を果たしています。特定の使用ケースに対して異なる設定が必要でない限り、これらのプロパティを手動で再宣言する必要はありません。

他の共有設定を含める場合、**これらのプロパティが上書きされたり変更されたりしないようにする必要があります**。そうでない場合、上記で述べた通り、共有動作を`next`設定で持つ設定を削除するか、直接Next.js ESLintプラグインから拡張することをお勧めします。
