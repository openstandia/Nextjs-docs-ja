Next.jsは、Next.jsアプリケーションの一般的な問題をキャッチすることを可能にする、基本設定にすでにバンドルされたESLintプラグイン[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)を提供しています。

## 参照 {#reference}

以下のESLintプラグインから推奨されるルールセットはすべて`eslint-config-next`内で使用されています：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これは`next.config.js`からの設定よりも優先されます。

## 例 {#examples}

### カスタムディレクトリとファイルのリント {#linting-custom-directories-and-files}

デフォルトでは、Next.jsは`pages/`、`app/`、`components/`、`lib/`、`src/`ディレクトリ内のすべてのファイルに対してESLintを実行します。ただし、プロダクションビルド用に`next.config.js`の`eslint`設定の中で`dirs`オプションを使用して特定のディレクトリを指定できます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // プロダクションビルド中に'pages'と'utils'ディレクトリでのみESLintを実行します（next build）
  },
}
```

同様に、特定のディレクトリとファイルをリントするために、`next lint`で`--dir`と`--file`フラグを使用できます：

```bash title="Terminal"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内でrootディレクトリを指定 {#specifying-a-root-directory-within-a-monorepo}

もしNext.jsがrootディレクトリにインストールされていないプロジェクト（例えばモノレポ）の中で`eslint-plugin-next`を使用している場合は、`.eslintrc`内の`settings`プロパティを使って`eslint-plugin-next`にNext.jsアプリケーションがどこにあるかを指示できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  }),
]

export default eslintConfig
```

`rootDir`はパス（相対または絶対）、グロブ（例`"packages/*/"`）、またはパスおよび/またはグロブの配列にすることができます。

### キャッシュの無効化 {#disabling-the-cache}

パフォーマンス向上のため、ESLintによって処理されたファイルの情報はデフォルトでキャッシュされます。これは`.next/cache`または定義された[ビルドディレクトリ](/docs/app/api-reference/config/next-config-js/distDir)に保存されます。単一のソースファイルの内容以上のものに依存するESLintルールを含めており、キャッシュを無効にする必要がある場合は、`next lint`で`--no-cache`フラグを使用します。

```bash title="Terminal"
next lint --no-cache
```

### ルールの無効化 {#disabling-rules}

サポートされているプラグイン（`react`、`react-hooks`、`next`）によって提供されるいずれかのルールを変更または無効化したい場合は、`.eslintrc`の`rules`プロパティを使用して直接変更できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  }),
]

export default eslintConfig
```

### Core Web Vitalsと共に {#with-core-web-vitals}

`next/core-web-vitals`ルールセットは、`next lint`が初めて実行されるときに有効になり、**strict**オプションが選択されます。

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals'],
  }),
]

export default eslintConfig
```

`next/core-web-vitals`は、デフォルトで警告となるいくつかのルールを[Core Web Vitals](https://web.dev/vitals/)に影響を与える場合はエラーに更新します。

> [Create Next App](/docs/app/api-reference/cli/create-next-app)で構築された新しいアプリケーションでは、`next/core-web-vitals`エントリポイントが自動的に含まれます。

### TypeScriptと共に {#with-typescript}

Next.js ESLintルールに加えて、`create-next-app --typescript`はESLint設定に`next/typescript`と共にTypeScript固有のリントルールも追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals', 'next/typescript'],
  }),
]

export default eslintConfig
```

これらのルールは[`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)に基づいています。
詳細は[TypeScript-ESLint > Configs](https://typescript-eslint.io/linting/configs)を参照してください。

### Prettierと共に {#with-prettier}

ESLintはコードフォーマットルールも含んでおり、既存の[Prettier](https://prettier.io/)のセットアップと競合する可能性があります。ESLintとPrettierを連携させるために、[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier)を含めることを推奨します。

まずは依存関係をインストールします：

```bash title="Terminal"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

それから、既存のESLint設定に`prettier`を追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next', 'prettier'],
  }),
]

export default eslintConfig
```

### ステージングされたファイルでのリントの実行 {#running-lint-on-staged-files}

[lint-staged](https://github.com/okonet/lint-staged)を使用してステージングされたgitファイルでリントを実行するために`next lint`を使用したい場合は、`.lintstagedrc.js`ファイルをプロジェクトのrootに追加し、`--file`フラグの使用を指定する必要があります。

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## プロダクションビルド中のリントの無効化 {#disabling-linting-during-production-builds}

`next build`中にESLintが実行されないようにするには、`next.config.js`の`eslint.ignoreDuringBuilds`オプションを`true`に設定します：

### 既存の設定の移行 {#migrating-existing-config}

アプリケーションにすでにESLintが設定されている場合、いくつかの条件が満たされていない限り、`eslint-config-next`を含めるのではなく、このプラグインから直接拡張することを推奨します。

#### 推奨プラグインのルールセット {#recommended-plugin-ruleset}

次の条件が真である場合：

- 次のプラグインの1つ以上がすでにインストールされています（別々に、または`airbnb`や`react-app`などの異なる設定を介して）：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- BabelがNext.jsでどのように設定されているかとは異なる特定の`parserOptions`を定義しています（[カスタムのBabel設定](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)を行っていない限り推奨されません）
- `eslint-plugin-import`をNode.jsおよび/またはTypeScript [リゾルバ](https://github.com/benmosher/eslint-plugin-import#resolvers)が定義されている状態でインストールしています

その場合、これらのプロパティが[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js)内でどのように設定されているのか気に入った場合にはこれらの設定を削除するか、代わりにNext.js ESLintプラグインから直接拡張することを推奨します：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

このプラグインは通常、`next lint`を実行することなくプロジェクトにインストールできます：

```bash title="Terminal"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これは、複数の設定で同じプラグインやパーサーをインポートすることによる衝突やエラーのリスクを排除します。

#### 追加の設定 {#additional-configurations}

すでに別のESLint設定を使用しており、`eslint-config-next`を含めたい場合は、他の設定の後にそれを最後に拡張するようにしてください。例：

```mjs title="eslint.config.mjs"
import js from '@eslint/js'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能
  baseDirectory: import.meta.dirname,
  recommendedConfig: js.configs.recommended,
})

const eslintConfig = [
  ...compat.config({
    extends: ['eslint:recommended', 'next'],
  }),
]

export default eslintConfig
```

`next`設定はすでに`parser`、`plugins`、および`settings`プロパティのデフォルト値を設定します。これらのプロパティを手動で再宣言する必要はありません。ただし、特定のユースケースに対して異なる設定が必要な場合は別です。

他の共有可能な設定を含める場合は、**これらのプロパティが上書きされないまたは変更されないことを確認する必要があります**。そうでない場合は、上記のとおり、`next`設定とともに動作を共有するための設定を削除するか、直接Next.js ESLintプラグインから拡張することをお勧めします。
