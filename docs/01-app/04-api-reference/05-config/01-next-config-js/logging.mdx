## オプション {#options}

### フェッチ {#fetching}

Next.jsを開発モードで実行する際に、ログのレベルとフルURLをコンソールに記録するかどうかを設定できます。

現在、`logging`は`fetch` APIを使用したデータフェッチにのみ適用されます。他のNext.js内のログにはまだ適用されていません。

```js title="next.config.js"
module.exports = {
  logging: {
    fetches: {
      fullUrl: true,
    },
  },
}
```

[Server Components HMR cache](/docs/app/api-reference/config/next-config-js/serverComponentsHmrCache)から復元された`fetch`リクエストはデフォルトでは記録されません。ただし、`logging.fetches.hmrRefreshes`を`true`に設定することで有効にできます。

```js title="next.config.js"
module.exports = {
  logging: {
    fetches: {
      hmrRefreshes: true,
    },
  },
}
```

### 受信リクエスト {#incoming-requests}

デフォルトでは、開発中にすべての受信リクエストがコンソールに記録されます。`incomingRequests`オプションを使用して、無視するリクエストを決定できます。
これは開発中にのみ記録されるため、このオプションは本番ビルドには影響しません。

```js title="next.config.js"
module.exports = {
  logging: {
    incomingRequests: {
      ignore: [/\api\/v1\/health/],
    },
  },
}
```

または、`incomingRequests`を`false`に設定することで、受信リクエストのログを無効にすることもできます。

```js title="next.config.js"
module.exports = {
  logging: {
    incomingRequests: false,
  },
}
```

### ログの無効化 {#disabling-logging}

さらに、`logging`を`false`に設定することで、開発中のログを無効にすることができます。

```js title="next.config.js"
module.exports = {
  logging: false,
}
```
