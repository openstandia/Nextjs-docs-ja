Next.js CLIを使用すると、アプリケーションを開発、構築、起動などができます。

基本的な使用法：

```bash title="Terminal"
npx next [command] [options]
```

## リファレンス {#reference}

以下のオプションが利用可能です：

| オプション              | 説明                                     |
| ----------------------- | ---------------------------------------- |
| `-h` または `--help`    | 利用可能なすべてのオプションを表示します |
| `-v` または `--version` | Next.jsのバージョン番号を出力します      |

### コマンド {#commands}

以下のコマンドが利用可能です：

| コマンド                               | 説明                                                                                                                                                                                                                            |
| -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [`dev`](#next-dev-options)             | 開発モードで Next.js を起動し、Hot Module Reloading、エラーレポートなどを利用できます。                                                                                                                                         |
| [`build`](#next-build-options)         | アプリケーションの最適化されたプロダクションビルドを作成します。各ルートについての情報を表示します。                                                                                                                            |
| [`start`](#next-start-options)         | プロダクションモードで Next.js を起動します。アプリケーションは事前に `next build` でコンパイルされている必要があります。                                                                                                       |
| [`info`](#next-info-options)           | 現在のシステムに関連する詳細を出力して、Next.js のバグ報告に使用できます。                                                                                                                                                      |
| [`lint`](#next-lint-options)           | `/src`、`/app`、`/pages`、`/components`、および `/lib` ディレクトリ内のすべてのファイルに対してESLintを実行します。ESLintがまだ設定されていない場合、必要な依存関係をインストールするためのガイド付きセットアップも提供します。 |
| [`telemetry`](#next-telemetry-options) | 完全に匿名のテレメトリーコレクションを有効または無効にすることができます。                                                                                                                                                      |

> **Good to know**: コマンドなしで `next` を実行すると、`next dev` のエイリアスになります。

### `next dev`オプション {#next-dev-options}

`next dev` は、Hot Module Reloading（HMR）、エラーレポートなどを伴う開発モードでアプリケーションを起動します。`next dev` を実行する際に利用可能なオプションは次のとおりです：

| オプション                               | 説明                                                                                                                                              |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `-h, --help`                             | 利用可能なすべてのオプションを表示します。                                                                                                        |
| `[directory]`                            | アプリケーションをビルドするディレクトリです。指定しない場合、現在のディレクトリが使用されます。                                                  |
| `--turbopack`                            | [Turbopack](/docs/app/api-reference/turbopack) を使用して開発モードを開始します。                                                                 |
| `-p` または `--port <port>`              | アプリケーションを開始するポート番号を指定します。デフォルト：3000、環境変数：PORT                                                                |
| `-H` または `--hostname <hostname>`      | アプリケーションを開始するホスト名を指定します。ネットワーク上の他のデバイスでアプリケーションを利用可能にする場合に便利です。デフォルト：0.0.0.0 |
| `--experimental-https`                   | HTTPSでサーバーを起動し、自己署名証明書を生成します。                                                                                             |
| `--experimental-https-key <path>`        | HTTPSキーのファイルパスです。                                                                                                                     |
| `--experimental-https-cert <path>`       | HTTPS証明書ファイルのパスです。                                                                                                                   |
| `--experimental-https-ca <path>`         | HTTPS証明書認証局ファイルのパスです。                                                                                                             |
| `--experimental-upload-trace <traceUrl>` | デバッグトレースのサブセットをリモートHTTP URLに報告します。                                                                                      |

### `next build`オプション {#next-build-options}

`next build` はアプリケーションの最適化されたプロダクションビルドを作成します。出力には各ルートの情報が表示されます。例えば：

```bash title="Terminal"
Route (app)                              Size     First Load JS
┌ ○ /_not-found                          0 B               0 kB
└ ƒ /products/[id]                       0 B               0 kB

○  (静的)   静的コンテンツとしてプリレンダリング
ƒ  (動的)  リクエストに応じてサーバーでレンダリングされます
```

- **Size**：クライアント側でページにナビゲートする際にダウンロードされるアセットのサイズです。各ルートのサイズにはその依存関係のみが含まれます。
- **First Load JS**：サーバーからページにアクセスする際にダウンロードされるアセットのサイズです。すべてのJavaScriptの共有部分は別のメトリックとして示されます。

これらの値は両方とも [**gzipで圧縮されます**](/docs/app/api-reference/config/next-config-js/compress)。最初のロードは緑、黄、または赤で示されます。パフォーマンスの良いアプリケーションを目指して緑を目指しましょう。

`next build` コマンドに利用可能なオプションは次のとおりです：

| オプション                         | 説明                                                                                                                                                              |
| ---------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `-h, --help`                       | 利用可能なすべてのオプションを表示します。                                                                                                                        |
| `[directory]`                      | アプリケーションをビルドするディレクトリです。指定しない場合、現在のディレクトリが使用されます。                                                                  |
| `-d` または `--debug`              | ビルドの詳細な出力を有効にします。このフラグを有効にすると、リライター、リダイレクト、ヘッダーなどの追加のビルド出力が表示されます。                              |
|                                    |                                                                                                                                                                   |
| `--profile`                        | 本番環境での[Reactのプロファイリング](https://react.dev/reference/react/Profiler)を有効にします。                                                                 |
| `--no-lint`                        | Lintの実行を無効にします。                                                                                                                                        |
| `--no-mangling`                    | [Name Mangling](https://en.wikipedia.org/wiki/Name_mangling) を無効にします。これはパフォーマンスに影響を及ぼす可能性があり、デバッグ目的にのみ使用してください。 |
| `--experimental-app-only`          | app routerのルートのみをビルドします。                                                                                                                            |
| `--experimental-build-mode [mode]` | 実験的なビルドモードを使用します。（選択肢："compile"、"generate"、デフォルト："default"）                                                                        |

### `next start`オプション {#next-start-options}

`next start` はアプリケーションをプロダクションモードで起動します。事前に[`next build`](#next-build-options)でアプリケーションをコンパイルしておく必要があります。

`next start` コマンドに利用可能なオプションは以下のとおりです：

| オプション                              | 説明                                                                                                               |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `-h` または `--help`                    | 利用可能なすべてのオプションを表示します。                                                                         |
| `[directory]`                           | アプリケーションを開始するディレクトリです。ディレクトリが指定されていない場合は現在のディレクトリが使用されます。 |
| `-p` または `--port <port>`             | アプリケーションを開始するポート番号を指定します。（デフォルト：3000、環境変数：PORT）                             |
| `-H` または `--hostname <hostname>`     | アプリケーションを開始するホスト名を指定します。（デフォルト：0.0.0.0）                                            |
| `--keepAliveTimeout <keepAliveTimeout>` | アイナクティブな接続を閉じるまでの最大ミリ秒数を指定します。                                                       |

### `next info`オプション {#next-info-options}

`next info` は、[GitHub issue](https://github.com/vercel/next.js/issues) を開く際に Next.js のバグを報告するために使用できる現在のシステムに関する関連詳細を出力します。この情報には、オペレーティングシステムのプラットフォーム/アーキ/バージョン、バイナリ（Node.js、npm、Yarn、pnpm）、パッケージのバージョン（`next`、`react`、`react-dom`）などが含まれます。

出力は次のようになります：

```bash title="Terminal"
Operating System:
  Platform: darwin
  Arch: arm64
  Version: Darwin Kernel Version 23.6.0
  Available memory (MB): 65536
  Available CPU cores: 10
Binaries:
  Node: 20.12.0
  npm: 10.5.0
  Yarn: 1.22.19
  pnpm: 9.6.0
Relevant Packages:
  next: 15.0.0-canary.115 // 使用可能な最新バージョンが検出されます（15.0.0-canary.115）
  eslint-config-next: 14.2.5
  react: 19.0.0-rc
  react-dom: 19.0.0
  typescript: 5.5.4
Next.js Config:
  output: N/A
```

`next info` コマンドに利用可能なオプションは以下のとおりです：

| オプション           | 説明                                     |
| -------------------- | ---------------------------------------- |
| `-h` または `--help` | 利用可能なすべてのオプションを表示します |
| `--verbose`          | デバッグ用の追加情報を収集します。       |

### `next lint`オプション {#next-lint-options}

`next lint` は、`pages/`、`app/`、`components/`、`lib/`、および `src/` ディレクトリ内のすべてのファイルに対してESLintを実行します。ESLintがアプリケーションにまだ設定されていない場合、必要な依存関係をインストールするためのガイド付きセットアップも提供します。

`next lint` コマンドに利用可能なオプションは以下のとおりです：

| オプション                                            | 説明                                                                                                                     |
| ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `[directory]`                                         | アプリケーションをLintするためのベースディレクトリです。指定しない場合は現在のディレクトリが使用されます。               |
| `-d, --dir, <dirs...>`                                | ESLintを実行するディレクトリを含めます。                                                                                 |
| `--file, <files...>`                                  | ESLintを実行するファイルを含めます。                                                                                     |
| `--ext, [exts...]`                                    | JavaScriptファイルの拡張子を指定します。（デフォルト：[ ".js", ".mjs", ".cjs", ".jsx", ".ts", ".mts", ".cts", ".tsx" ]） |
| `-c, --config, <config>`                              | すべての他の構成オプションをオーバーライドして、この構成ファイルを使用します。                                           |
| `--resolve-plugins-relative-to, <rprt>`               | プラグインを解決する場所を指定します。                                                                                   |
| `--strict`                                            | Next.jsの厳格な構成を使用して `.eslintrc.json` ファイルを作成します。                                                    |
| `--rulesdir, <rulesdir...>`                           | このディレクトリ（複数可）から追加の規則を使用します。                                                                   |
| `--fix`                                               | Lintの問題を自動的に修正します。                                                                                         |
| `--fix-type <fixType>`                                | 適用する修正の種類を指定します（例：問題、提案、レイアウト）。                                                           |
| `--ignore-path <path>`                                | 無視するファイルを指定します。                                                                                           |
| `--no-ignore <path>`                                  | `--ignore-path` オプションを無効にします。                                                                               |
| `--quiet`                                             | エラーのみを報告します。                                                                                                 |
| `--max-warnings [maxWarnings]`                        | 非ゼロの終了コードをトリガーする前に許容する警告の数を指定します。（デフォルト：-1）                                     |
| `-o, --output-file, <outputFile>`                     | レポートを出力するファイルを指定します。                                                                                 |
| `-f, --format, <format>`                              | 特定の出力フォーマットを使用します。                                                                                     |
| `--no-inline-config`                                  | コメントによる設定や規則の変更を防ぎます。                                                                               |
| `--report-unused-disable-directives-severity <level>` | 使用されていないeslint-disable指令の深刻度レベルを指定します。（選択肢："error"、"off"、"warn"）                         |
| `--no-cache`                                          | キャッシュを無効にします。                                                                                               |
| `--cache-location, <cacheLocation>`                   | キャッシュの場所を指定します。                                                                                           |
| `--cache-strategy, [cacheStrategy]`                   | キャッシュで変更されたファイルを検出するための戦略を指定します。（デフォルト："metadata"）                               |
| `--error-on-unmatched-pattern`                        | 未一致のファイルパターンがあるときにエラーを報告します。                                                                 |
| `-h, --help`                                          | このメッセージを表示します。                                                                                             |

### `next telemetry`オプション {#next-telemetry-options}

Next.jsは、一般的な使用について**完全に匿名の**テレメトリーデータを収集します。この匿名プログラムへの参加はオプションであり、情報を共有したくない場合はオプトアウトすることができます。

`next telemetry` コマンドに利用可能なオプションは以下のとおりです：

| オプション   | 説明                                                 |
| ------------ | ---------------------------------------------------- |
| `-h, --help` | 利用可能なすべてのオプションを表示します。           |
| `--enable`   | Next.js のテレメトリーのコレクションを有効にします。 |
| `--disable`  | Next.js のテレメトリーのコレクションを無効にします。 |

[Telemetry](https://nextjs.org/telemetry) についてもっと学びましょう。

## 例 {#examples}

### デフォルトポートの変更 {#changing-the-default-port}

デフォルトでは、Next.js は開発中および `next start` を使用するときに `http://localhost:3000` を使用します。`-p` オプションを使用してデフォルトのポートを変更できます：

```bash title="Terminal"
next dev -p 4000
```

または、`PORT` 環境変数を使用します：

```bash title="Terminal"
PORT=4000 next dev
```

> **Good to know**: `PORT` は `.env` 内で設定することはできません。HTTP サーバーの起動は他のコードが初期化される前に行われるためです。

### 開発中のHTTPS使用 {#using-https-during-development}

Webhookや認証のような特定のユースケースでは、[HTTPS](https://developer.mozilla.org/en-US/docs/Glossary/HTTPS) を使用して `localhost` 上で安全な環境を確保することができます。Next.js は `--experimental-https` フラグを使用して、自己署名証明書を生成する `next dev` を実行できます：

```bash title="Terminal"
next dev --experimental-https
```

生成された証明書を使用すると、Next.js の開発サーバーは `https://localhost:3000` に存在します。ポートが `-p`、`--port`、または `PORT` で指定されない限り、デフォルトのポート `3000` が使用されます。

`--experimental-https-key` および `--experimental-https-cert` を使用してカスタム証明書およびキーを提供することもできます。オプションとして、`--experimental-https-ca` を使用してカスタム CA 証明書を提供することもできます。

```bash title="Terminal"
next dev --experimental-https --experimental-https-key ./certificates/localhost-key.pem --experimental-https-cert ./certificates/localhost.pem
```

`next dev --experimental-https` は開発専用で、[`mkcert`](https://github.com/FiloSottile/mkcert) を使用してローカルに信頼された証明書を作成します。プロダクションでは、信頼できる認証局から正しく発行された証明書を使用してください。

> **Good to know**: Vercel へのデプロイ時には、Next.js アプリケーションのためにHTTPSが [自動的に設定されます](https://vercel.com/docs/security/encryption)。

### ダウンストリームプロキシ用のタイムアウト設定 {#configuring-a-timeout-for-downstream-proxies}

Next.js をダウンストリームプロキシ（例えば AWS ELB/ALB のようなロードバランサ）でデプロイする場合、Nextの基礎となるHTTPサーバーをダウンストリームプロキシのタイムアウトより _長い_ [keep-alive timeout](https://nodejs.org/api/http.html#http_server_keepalivetimeout) に設定することが重要です。そうしないと、指定されたTCP接続の keep-alive timeout が到達した場合、Node.js はダウンストリームプロキシに通知することなくその接続を直ちに終了します。この結果、Node.jsが既に終了した接続を再利用しようとしたときに、プロキシエラーが発生します。

本番環境での Next.js サーバーのタイムアウト値を設定するには、`next start` に `--keepAliveTimeout`（ミリ秒単位）を渡します：

```bash title="Terminal"
next start --keepAliveTimeout 70000
```

### Node.js引数を渡す {#passing-node-js-arguments}

任意の [Node 引数](https://nodejs.org/api/cli.html#cli_node_options_options) を `next` コマンドに渡すことができます。例えば：

```bash title="Terminal"
NODE_OPTIONS='--throw-deprecation' next
NODE_OPTIONS='-r esm' next
NODE_OPTIONS='--inspect' next
```
