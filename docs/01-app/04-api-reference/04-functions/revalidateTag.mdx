---
title: 'revalidateTag'
description: 'revalidateTag 関数のAPIリファレンス'
---

`revalidateTag` を使用すると、特定のキャッシュタグに対してオンデマンドで[キャッシュされたデータ](/docs/app/building-your-application/caching)をパージできます。

> **Good to know**:
>
> - `revalidateTag` は、[Node.js と Edge runtime](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `revalidateTag` は、パスが次に訪問されるときにのみキャッシュを無効化します。つまり、`revalidateTag` を dynamic route segment と共に呼び出しても、すぐに多くの再検証が同時にトリガーされることはありません。無効化は、パスが次に訪問されたときにのみ発生します。

## Parameters {#parameters}

```tsx
revalidateTag(tag: string): void;
```

- `tag`: 再検証したいデータに関連付けられたキャッシュタグを表す文字列。256文字以下である必要があります。この値は大文字・小文字を区別します。

`fetch` にタグを以下のように追加できます：

```tsx
fetch(url, { next: { tags: [...] } });
```

## Returns {#returns}

`revalidateTag` は値を返しません。

### Route Handler {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { revalidateTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')
  revalidateTag(tag)
  return Response.json({ revalidated: true, now: Date.now() })
}
```
