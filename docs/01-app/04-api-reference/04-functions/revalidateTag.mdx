---
title: 'revalidateTag'
description: 'revalidateTag 関数のAPIリファレンス。'
---

`revalidateTag`は、特定のキャッシュタグに対して、オンデマンドで[キャッシュされたデータ](/docs/app/building-your-application/caching)をパージすることを可能にします。

> **Good to know**:
>
> - `revalidateTag`は、[Node.js および Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `revalidateTag`は、そのパスが次に訪問されたときにのみキャッシュを無効化します。つまり、`revalidateTag`をdynamic route segmentで呼び出しても、すぐに多くの再検証が一度にトリガーされるわけではありません。無効化は、そのパスが次に訪問されたときにのみ発生します。

## パラメータ {#parameters}

```tsx
revalidateTag(tag: string): void;
```

- `tag`: 再検証したいデータに関連付けられたキャッシュタグを表す文字列。256文字以下でなければなりません。この値は大文字と小文字を区別します。

`fetch`にタグを次のように追加できます：

```tsx
fetch(url, { next: { tags: [...] } });
```

## 戻り値 {#returns}

`revalidateTag`は値を返しません。

## 例 {#examples}

### Server Action {#server-action}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions.ts" switcher
'use server'

// `next/cache`からrevalidateTagをインポート
import { revalidateTag } from 'next/cache'

// 非同期のサーバーアクション
export default async function submit() {
  await addPost() // 投稿を追加する関数を呼び出し
  revalidateTag('posts') // `'posts'`タグのキャッシュを再検証
}
```

### ルートハンドラ {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
// リクエスト型のインポート
import type { NextRequest } from 'next/server'
// `next/cache`からrevalidateTagをインポート
import { revalidateTag } from 'next/cache'

// 非同期のルートハンドラ
export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag') // URLからタグを取得
  revalidateTag(tag) // 取得したタグのキャッシュを再検証
  return Response.json({ revalidated: true, now: Date.now() }) // JSONレスポンスを返す
}
```
