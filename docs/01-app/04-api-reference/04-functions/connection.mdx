title: 'connection'

`connection()` 関数は、レンダリングを続行する前に、ユーザーリクエストが来るのを待つべきであることを示すために使用します。

これは、コンポーネントが [Dynamic APIs](/docs/app/building-your-application/rendering/server-components#dynamic-apis) を使用しない場合でも、ビルド時に静的にレンダリングされるのではなく、実行時に動的にレンダリングされるようにしたい場合に便利です。例えば、`Math.random()` や `new Date()` のような外部情報にアクセスして、レンダリング結果を意図的に変更したい場合に通常発生します。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/page.tsx" switcher
import { connection } from 'next/server'

export default async function Page() {
  await connection()
  // これ以降のすべては事前レンダリングから除外されます
  const rand = Math.random()
  return <span>{rand}</span>
}
```

</TabItem>

## リファレンス {#reference}

### 型 {#type}

```jsx
function connection(): Promise<void>
```

### パラメータ {#parameters}

- この関数はパラメータを受け取りません。

### 戻り値 {#returns}

- この関数は `void` の Promise を返します。消費することを目的としていません。

## Good to know {#good-to-know}

- `connection` は、Next.js の将来とより良く一致するように、[`unstable_noStore`](/docs/app/api-reference/functions/unstable_noStore) を置き換えます。
- 関数は、動的なレンダリングが必要で、一般的な Dynamic APIs が使用されていない場合にのみ必要です。
