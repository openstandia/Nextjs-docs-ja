`forbidden` 関数は、Next.js の 403 エラーページをレンダリングするエラーをスローします。これは、アプリケーション内での認可エラーの処理に役立ちます。UI は [`forbidden.js` ファイル](/docs/app/api-reference/file-conventions/forbidden)を使用してカスタマイズできます。

`forbidden`を使用開始するには、`next.config.js` ファイルで実験的な [`authInterrupts`](/docs/app/api-reference/config/next-config-js/authInterrupts) 設定オプションを有効にします：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    authInterrupts: true,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
module.exports = {
  experimental: {
    authInterrupts: true,
  },
}
```

</TabItem>
</Tabs>

`forbidden` は、[Server Components](/docs/app/building-your-application/rendering/server-components)、[Server Actions](/docs/app/building-your-application/data-fetching/server-actions-and-mutations)、および [Route Handlers](/docs/app/building-your-application/routing/route-handlers) で呼び出すことができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/auth/page.tsx" switcher
import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'

export default async function AdminPage() {
  const session = await verifySession()

  // ユーザーが'admin' ロールを持っているかチェックします
  if (session.role !== 'admin') {
    forbidden()
  }

  // 認可されたユーザーに対してadminページをレンダリングします
  return <></>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/auth/page.js" switcher

```

</TabItem>
</Tabs>

## Good to know {#good-to-know}

- `forbidden` 関数は [root レイアウト](/docs/app/building-your-application/routing/layouts-and-templates#root-layout-required)で呼び出すことはできません

## Examples {#examples}

### Role-based route protection {#role-based-route-protection}

`forbidden` を使用して、ユーザーのロールに基づいて特定のルートへのアクセスを制限できます。これにより、認証済みだが必要な権限を持たないユーザーがルートにアクセスするのを防ぎます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/admin/page.tsx" switcher
import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'

export default async function AdminPage() {
  const session = await verifySession()

  // ユーザーが'admin' ロールを持っているかチェックします
  if (session.role !== 'admin') {
    forbidden()
  }

  // 認可されたユーザーに対してadminページをレンダリングします
  return (
    <main>
      <h1>Admin Dashboard</h1>
      <p>Welcome, {session.user.name}!</p>
    </main>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/admin/page.js" switcher

      <h1>Admin Dashboard</h1>

    </main>

}
```

</TabItem>
</Tabs>

### Mutations with Server Actions {#mutations-with-server-actions}

Server Actions における更新の実装時に、`forbidden` を使用することで、特定のロールを持つユーザーだけが機密データを更新できるようにすることができます。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions/update-role.ts" switcher
'use server'

import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'
import db from '@/app/lib/db'

export async function updateRole(formData: FormData) {
  const session = await verifySession()

  // 管理者のみがロールを更新できることを保証します
  if (session.role !== 'admin') {
    forbidden()
  }

  // 認可されたユーザーに対してロールを更新します
  // ...
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/actions/update-role.js" switcher

  }

  // ...

```
