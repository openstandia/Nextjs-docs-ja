`unstable_expireTag`は、特定のキャッシュタグに対して、[キャッシュデータ](/docs/app/building-your-application/caching)を随時パージすることを可能にします。

> **Good to know**:
>
> - `unstable_expireTag`は、[Node.js と Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `unstable_expireTag`は、パスが次に訪問されたときにのみキャッシュを無効化します。これは、`unstable_expireTag`をdynamic route segmentで呼び出しても、即座に多くの有効期限切れが発生することはありません。無効化はパスが次に訪問されたときにのみ行われます。

## リファレンス {#reference}

### パラメータ {#parameters}

```tsx
unstable_expireTag(...tags: string[]): void;
```

- `tags`: 再検証したいデータに関連付けられているキャッシュタグを表す文字列引数。各文字列は256文字以下である必要があります。この値は大文字と小文字を区別します。

`fetch`にタグを追加することができます：

```tsx
fetch(url, { next: { tags: [...] } });
```

### 戻り値 {#returns}

`unstable_expireTag`は値を返しません。

## 例 {#examples}

### サーバーアクション {#server-action}

Server Actionで`unstable_expireTag`を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions.ts" switcher
'use server'

import { unstable_expireTag } from 'next/cache'

export default async function submit() {
  await addPost()
  unstable_expireTag('posts', 'blog')
}
```

### ルートハンドラー {#route-handler}

Route Handlerで`unstable_expireTag`を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { unstable_expireTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')
  unstable_expireTag(tag)
  return Response.json({ revalidated: true, now: Date.now() })
}
```

</TabItem>
