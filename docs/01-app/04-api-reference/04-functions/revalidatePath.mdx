`revalidatePath` は特定のパスに対して[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドで消去することができます。

> **Good to know**:
>
> - `revalidatePath` は、[Node.jsおよびEdgeランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で使用可能です。
> - `revalidatePath` は、指定されたパスが次に訪問されたときにのみキャッシュを無効化します。これにより、`revalidatePath` を動的ルートセグメントで呼び出しても、一度に多くの再検証が即座にトリガーされることはありません。無効化は、そのパスが次に訪問された際にのみ行われます。
> - 現在、`revalidatePath` は、サーバーアクションで使用された場合、[クライアント側のRouter Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートを無効化します。この動作は一時的なものであり、将来的には特定のパスにのみ適用されるように更新される予定です。
> - `revalidatePath` を使用することで、[サーバー側のRoute Cache](/docs/app/building-your-application/caching#full-route-cache)の**特定のパスのみ**を無効化します。

## パラメータ {#parameters}

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 再検証したいデータに関連するファイルシステムパス（例: `/product/[slug]/page`）か、リテラルルートセグメント（例: `/product/123`）を表す文字列です。1024文字以内である必要があります。この値は大文字と小文字を区別します。
- `type`: （オプション）再検証するパスの種類を変更するための`'page'`または`'layout'`の文字列です。`path`に動的セグメントが含まれている場合（例: `/product/[slug]/page`）、このパラメータは必須です。パスがリテラルルートセグメント（例: 動的ページ`/product/[slug]/page`では`/product/1`）を指す場合、`type`を指定するべきではありません。

## 戻り値 {#returns}

`revalidatePath` は値を返しません。

## 例 {#examples}

### 特定のURLを再検証する {#revalidating-a-specific-url}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

これにより、次回のページ訪問時に特定のURLが再検証されます。

### ページパスを再検証する {#revalidating-a-page-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// あるいはルートグループを使用
revalidatePath('/(main)/blog/[slug]', 'page')
```

これにより、次回のページ訪問時に指定された`page`ファイルに一致するURLが再検証されます。これは特定のページ以下のページを無効化するものではありません。例えば、`/blog/[slug]`は`/blog/[slug]/[author]`を無効化しません。

### レイアウトパスを再検証する {#revalidating-a-layout-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// あるいはルートグループを使用
revalidatePath('/(main)/post/[slug]', 'layout')
```

これにより、次回のページ訪問時に指定された`layout`ファイルに一致するURLが再検証されます。同じレイアウトを持つ以下のページは次回の訪問時に再検証が行われます。例えば、上記の場合では`/blog/[slug]/[another]`も次回の訪問時に再検証されます。

### すべてのデータを再検証する {#revalidating-all-data}

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

これにより、クライアント側のRouter Cacheが消去され、次回のページ訪問時にData Cacheが再検証されます。

### ルートハンドラー {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
