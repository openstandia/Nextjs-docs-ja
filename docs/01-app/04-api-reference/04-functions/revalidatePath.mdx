`revalidatePath` は特定のパスに対してオンデマンドで[キャッシュされたデータ](/docs/app/building-your-application/caching)を削除することができます。

> **Good to know**:
>
> - `revalidatePath` は [Node.js や Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で使用可能です。
> - `revalidatePath` は、含まれるパスが次に訪問された時にのみキャッシュを無効化します。つまり、`revalidatePath` を dynamic route segment とともに呼び出しても、即座に多数のリバリデーションをトリガーすることはありません。無効化は次にパスが訪問されたときだけ発生します。
> - 現時点では、`revalidatePath` は server action内で使用されると、[クライアントサイドの Router Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートを無効化します。この動作は一時的なもので、将来的には特定のパスにのみ適用されるように更新されます。
> - `revalidatePath` を使用すると、[サーバーサイドの Route Cache](/docs/app/building-your-application/caching#full-route-cache)内の**特定のパスのみ**が無効化されます。

## パラメータ {#parameters}

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 再バリデーションしたいデータに対応するファイルシステムパス（例：`/product/[slug]/page`）またはリテラルな route segment（例：`/product/123`）を表す文字列。1024文字未満である必要があります。この値は大文字と小文字を区別します。
- `type`: （オプション）再バリデートするパスの種類を変更するための `'page'` または `'layout'` の文字列。`path` が dynamic segment を含む場合（例：`/product/[slug]/page`）、このパラメータが必要です。pathがリテラルな route segment を指している場合（例：dynamic page の `/product/1`）、`type` を指定する必要はありません。

## 戻り値 {#returns}

`revalidatePath` は値を返しません。

## 例 {#examples}

### 特定の URL を再バリデーションする {#revalidating-a-specific-url}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

これは次のページ訪問時に特定の URL を再バリデーションします。

### ページパスを再バリデーションする {#revalidating-a-page-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// または、route group を使用して
revalidatePath('/(main)/blog/[slug]', 'page')
```

これは次のページ訪問時に、指定された `page` ファイルに一致するURLを再バリデートします。これは特定のページの下位ページを無効化することはありません。例えば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効化しません。

### レイアウトパスを再バリデーションする {#revalidating-a-layout-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// または、route group を使用して
revalidatePath('/(main)/post/[slug]', 'layout')
```

これは次のページ訪問時に、指定された `layout` ファイルに一致するURLを再バリデートします。同じレイアウトが適用される下位ページも次の訪問時に再バリデートされます。例えば、上述の場合、`/blog/[slug]/[another]` も次の訪問時に再バリデートされます。

### すべてのデータを再バリデーションする {#revalidating-all-data}

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

これはクライアントサイドの Router Cache を削除し、次のページ訪問時に Data Cache を再バリデートします。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
