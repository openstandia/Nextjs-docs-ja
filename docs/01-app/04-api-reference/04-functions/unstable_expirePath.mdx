`unstable_expirePath` は、特定のパスに対してキャッシュされた[データ](/docs/app/building-your-application/caching)をオンデマンドで消去することを可能にします。

> **Good to know**:
>
> - `unstable_expirePath` は、[Node.js とEdge runtimes](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes) の両方で利用可能です。
> - `unstable_expirePath` は、指定したパスが次にアクセスされたときにのみキャッシュを無効化します。したがって、`unstable_expirePath` を dynamic route segment とともに呼び出しても、すぐに多くの無効化がトリガーされるわけではありません。無効化は、そのパスが次に訪問されたときにのみ発生します。
> - 現在、`unstable_expirePath` は server action で使用されるとき、[クライアント側Router Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートを無効化します。この動作は一時的なものであり、将来は特定のパスにのみ適用されるように更新される予定です。
> - `unstable_expirePath` を使用すると、[サーバー側 Route Cache](/docs/app/building-your-application/caching#full-route-cache)内の**特定のパスのみ**が無効化されます。

## リファレンス {#reference}

### パラメーター {#parameters}

```tsx
unstable_expirePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 期限切れにしたいデータに関連付けられたファイルシステムのパスを示す文字列（例：`/product/[slug]/page`）、またはリテラルルートセグメント（例：`/product/123`）のいずれかです。文字数は1024 文字未満でなければなりません。この値は大文字と小文字を区別します。
- `type`: （オプション）期限切れにするパスの種類を変更するための`'page'`または`'layout'` の文字列です。`path` に動的セグメントが含まれる場合（例：`/product/[slug]/page`）、このパラメーターが必要です。path が動的ページ（例：`/product/[slug]/page`）のリテラルルートセグメント（例：`/product/1`）を参照する場合、このパラメーターを指定しないでください。

### 戻り値 {#returns}

`unstable_expirePath` は値を返しません。

## 例 {#examples}

### 特定のURL を期限切れにする {#expiring-a-specific-url}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/post-1')
```

これにより、次のページ訪問時に特定の一つのURL のキャッシュが消去されます。

### ページパスを期限切れにする {#expiring-a-page-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'page')
// またはルートグループとともに
unstable_expirePath('/(main)/blog/[slug]', 'page')
```

これにより、次のページ訪問時に、指定された`page`ファイルに一致するすべてのURLのキャッシュが消去されます。特定のページの下にあるページは無効化されません。例えば、`/blog/[slug]`は`/blog/[slug]/[author]`を無効化しません。

### レイアウトパスを期限切れにする {#expiring-a-layout-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'layout')
// またはルートグループとともに
unstable_expirePath('/(main)/post/[slug]', 'layout')
```

これにより、次のページ訪問時に、指定された`layout`ファイルに一致するすべてのURLのキャッシュが消去されます。同じレイアウトを持つ下位のページが次回訪問時に再検証されることになります。例えば、上記の場合、`/blog/[slug]/[another]` も次回訪問時に再検証されます。

### すべてのデータを期限切れにする {#expiring-all-data}

```ts
import { unstable_expirePath } from 'next/cache'

unstable_expirePath('/', 'layout')
```

これにより、次のページ訪問時にデータキャッシュが消去されます。

### サーバーアクション {#server-action}

Server Action 内で `unstable_expirePath` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions.ts" switcher
'use server'

import { unstable_expirePath } from 'next/cache'

export default async function submit() {
  await submitForm()
  unstable_expirePath('/')
}
```

</TabItem>
</Tabs>

### ルートハンドラ {#route-handler}

Route Handler 内で `unstable_expirePath` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/expire/route.ts" switcher
import { unstable_expirePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    unstable_expirePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    expired: false,
    now: Date.now(),
    message: 'Missing path to expire',
  })
}
```

</TabItem>
