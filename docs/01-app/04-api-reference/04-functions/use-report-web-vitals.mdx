`useReportWebVitals`フックを使用することで、[Core Web Vitals](https://web.dev/vitals/)をレポートすることができます。これを分析サービスと組み合わせて使用することが可能です。

<PagesOnly>

```jsx title="pages/_app.js"
import { useReportWebVitals } from 'next/web-vitals'

function MyApp({ Component, pageProps }) {
  useReportWebVitals((metric) => {
    console.log(metric)
  })

  return <Component {...pageProps} />
}
```

</PagesOnly>

<AppOnly>

```jsx title="app/_components/web-vitals.js"
'use client'

import { useReportWebVitals } from 'next/web-vitals'

export function WebVitals() {
  useReportWebVitals((metric) => {
    console.log(metric)
  })

  return null
}
```

```jsx title="app/layout.js"
import { WebVitals } from './_components/web-vitals'

export default function Layout({ children }) {
  return (
    <html>
      <body>
        <WebVitals />
        {children}
      </body>
    </html>
  )
}
```

> `useReportWebVitals`フックは`"use client"`ディレクティブを必要とするため、最もパフォーマンスが良いアプローチはroot レイアウトがインポートする別のコンポーネントを作成することです。これにより、クライアントの境界は`WebVitals`コンポーネントに限定されます。

</AppOnly>

## useReportWebVitals {#usereportwebvitals}

フックの引数として渡される`metric`オブジェクトは様々なプロパティで構成されています；

- `id`: 現在のページ読み込みコンテキストにおけるメトリックのユニーク識別子
- `name`: パフォーマンスメトリックの名前。可能な値には、Webアプリケーションに特有な[Web Vitals](#web-vitals)メトリック（TTFB、FCP、LCP、FID、CLS）の名前が含まれます
- `delta`: 現在の値とメトリックの前回の値の差。通常、ミリ秒単位で表され、メトリックの値の時間における変化を示します
- `entries`: メトリックに関連する[Performance Entries](https://developer.mozilla.org/docs/Web/API/PerformanceEntry)の配列。これらのエントリは、メトリックに関連するパフォーマンスイベントに関する詳細情報を提供します
- `navigationType`: メトリックのコレクションをトリガーした[ナビゲーションのタイプ](https://developer.mozilla.org/docs/Web/API/PerformanceNavigationTiming/type)を示します。可能な値には`"navigate"`、`"reload"`、`"back_forward"`、および`"prerender"`が含まれます
- `rating`: メトリック値の質的評価で、パフォーマンスの評価を提供します。可能な値は`"good"`、`"needs-improvement"`、`"poor"`です。この評価は、許容可能なパフォーマンスまたは劣っているパフォーマンスを示す事前に定義されたしきい値に対してメトリックの値を比較することによって決定されます
- `value`: 通常ミリ秒単位のパフォーマンスエントリの実際の値または持続時間。これは、追跡されるメトリックの量的測定値を提供します；値のソースは特定の測定メトリックに依存し、さまざまな[Performance API](https://developer.mozilla.org/docs/Web/API/Performance_API)から取得されます

## Web Vitals {#web-vitals}

[Web Vitals](https://web.dev/vitals/)は、Webページのユーザー体験を捉えることを目的とした一連の有用なメトリックです。以下のWeb Vitalsがすべて含まれています：

- [Time to First Byte](https://developer.mozilla.org/docs/Glossary/Time_to_first_byte) （TTFB）
- [First Contentful Paint](https://developer.mozilla.org/docs/Glossary/First_contentful_paint) （FCP）
- [Largest Contentful Paint](https://web.dev/lcp/) （LCP）
- [First Input Delay](https://web.dev/fid/) （FID）
- [Cumulative Layout Shift](https://web.dev/cls/) （CLS）
- [Interaction to Next Paint](https://web.dev/inp/) （INP）

これらのメトリックの結果はすべて`name`プロパティを使用して処理できます。

<PagesOnly>

```jsx title="pages/_app.js"
import { useReportWebVitals } from 'next/web-vitals'

function MyApp({ Component, pageProps }) {
  useReportWebVitals((metric) => {
    switch (metric.name) {
      case 'FCP': {
        // FCPの結果を処理します
      }
      case 'LCP': {
        // LCPの結果を処理します
      }
      // ...
    }
  })

  return <Component {...pageProps} />
}
```

</PagesOnly>

## カスタムメトリックス {#custom-metrics}

上記のコアメトリックに加え、ページがハイドレートしレンダリングする時間を測定する追加のカスタムメトリックがあります：

- `Next.js-hydration`: ページがハイドレーションを開始し終了するまでにかかる時間（ミリ秒単位）
- `Next.js-route-change-to-render`: ルート変更後、ページがレンダリングを開始するまでにかかる時間（ミリ秒単位）
- `Next.js-render`: ルート変更後、ページがレンダリングを終了するまでにかかる時間（ミリ秒単位）

これらのメトリックの結果を個別に処理することができます：

```jsx title="pages/_app.js"
import { useReportWebVitals } from 'next/web-vitals'

function MyApp({ Component, pageProps }) {
  useReportWebVitals((metric) => {
    switch (metric.name) {
      case 'Next.js-hydration':
        // ハイドレーションの結果を処理します
        break
      case 'Next.js-route-change-to-render':
        // ルート変更からレンダリングまでの結果を処理します
        break
      case 'Next.js-render':
        // レンダリングの結果を処理します
        break
      default:
        break
    }
  })

  return <Component {...pageProps} />
}
```

これらのメトリックは、[User Timing API](https://caniuse.com/#feat=user-timing)をサポートするすべてのブラウザで動作します。

</PagesOnly>

## Vercelでの使用 {#usage-on-vercel}

[Vercel Speed Insights](https://vercel.com/docs/speed-insights/quickstart)は`useReportWebVitals`を使用せず、代わりに`@vercel/speed-insights`パッケージを使用します。
`useReportWebVitals`フックはローカル開発で役立ちます、またはWeb Vitalsを収集するために別のサービスを使用する場合に便利です。

## 結果の外部システムへの送信 {#sending-results-to-external-systems}

サイトでの実ユーザーのパフォーマンスを測定・追跡するために結果を任意のエンドポイントに送信できます。例：

```js
useReportWebVitals((metric) => {
  const body = JSON.stringify(metric)
  const url = 'https://example.com/analytics'

  // `navigator.sendBeacon()`を使用可能であれば使用し、フォールバックとして`fetch()`を使用します。
  if (navigator.sendBeacon) {
    navigator.sendBeacon(url, body)
  } else {
    fetch(url, { body, method: 'POST', keepalive: true })
  }
})
```

> **Good to know**: [Google Analytics](https://analytics.google.com/analytics/web/)を使用する場合、
> `id`値を使用するとメトリックの分布を手動で作成できます（パーセンタイルを計算するなど）

> ```js
> useReportWebVitals(metric => {
>   // Google Analyticsを初期化した場合、この例のように`window.gtag`を使います：
>   // https://github.com/vercel/next.js/blob/canary/examples/with-google-analytics
>   window.gtag('event', metric.name, {
>     value: Math.round(metric.name === 'CLS' ? metric.value * 1000 : metric.value), // 値は整数でなければなりません
>     event_label: metric.id, // 現在のページ読み込みに特有のid
>     non_interaction: true, // バウンス率に影響を与えない
>   });
> }
> ```
>
> [Google Analyticsへの結果送信](https://github.com/GoogleChrome/web-vitals#send-the-results-to-google-analytics)について詳しく読むことができます。
