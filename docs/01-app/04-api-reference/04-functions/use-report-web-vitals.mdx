`useReportWebVitals`フックは、[Core Web Vitals](https://web.dev/vitals/)を報告するためのものであり、分析サービスと組み合わせて使用できます。

<PagesOnly>

```jsx title="pages/_app.js"
import { useReportWebVitals } from 'next/web-vitals'

function MyApp({ Component, pageProps }) {
  useReportWebVitals((metric) => {
    console.log(metric)
  })

  return <Component {...pageProps} />
}
```

</PagesOnly>

<AppOnly>

```jsx title="app/_components/web-vitals.js"
'use client'

import { useReportWebVitals } from 'next/web-vitals'

export function WebVitals() {
  useReportWebVitals((metric) => {
    console.log(metric)
  })

  return null
}
```

```jsx title="app/layout.js"
import { WebVitals } from './_components/web-vitals'

export default function Layout({ children }) {
  return (
    <html>
      <body>
        <WebVitals />
        {children}
      </body>
    </html>
  )
}
```

> `useReportWebVitals`フックは`"use client"`ディレクティブを必要とするため、最も効率的なアプローチは、root レイアウトでインポートする個別のコンポーネントを作成することです。これにより、クライアント境界が`WebVitals`コンポーネントに限定されます。

</AppOnly>

## useReportWebVitals {#usereportwebvitals}

フックの引数として渡される`metric`オブジェクトには、いくつかのプロパティが含まれています：

- `id`: 現在のページロードのコンテキストにおけるメトリックの一意識別子
- `name`: パフォーマンスメトリックの名前。Webアプリケーションに特化した[Web Vitals](#web-vitals)メトリックの名前（TTFB、FCP、LCP、FID、CLS）が含まれる可能性があります。
- `delta`: メトリックの現在の値と以前の値の差。通常ミリ秒単位でメトリックの値の時間的変化を表します。
- `entries`: メトリックに関連する[Performance Entries](https://developer.mozilla.org/docs/Web/API/PerformanceEntry)の配列。これらのエントリーは、メトリック関連のパフォーマンスイベントに関する詳細を提供します。
- `navigationType`: メトリック収集をトリガーした[ナビゲーションの種類](https://developer.mozilla.org/docs/Web/API/PerformanceNavigationTiming/type)を示します。可能な値には、`"navigate"`、`"reload"`、`"back_forward"`、`"prerender"`があります。
- `rating`: メトリック値の定性的評価で、パフォーマンスの評価を提供します。可能な値は`"good"`、`"needs-improvement"`、`"poor"`です。この評価は、プリセットの閾値と比較して、許容可能かどうかを判断します。
- `value`: パフォーマンスエントリの実際の値または期間で、通常はミリ秒単位です。この値は、追跡されるパフォーマンス側面の定量的な評価を提供します。値のソースは測定される具体的なメトリックに依存し、さまざまな[Performance API](https://developer.mozilla.org/docs/Web/API/Performance_API)から取得される可能性があります。

## Web Vitals {#web-vitals}

[Web Vitals](https://web.dev/vitals/)は、ウェブページのユーザーエクスペリエンスを把握するための一連の有益なメトリックです。以下のWeb Vitalsが含まれています：

- [Time to First Byte](https://developer.mozilla.org/docs/Glossary/Time_to_first_byte) (TTFB)
- [First Contentful Paint](https://developer.mozilla.org/docs/Glossary/First_contentful_paint) (FCP)
- [Largest Contentful Paint](https://web.dev/lcp/) (LCP)
- [First Input Delay](https://web.dev/fid/) (FID)
- [Cumulative Layout Shift](https://web.dev/cls/) (CLS)
- [Interaction to Next Paint](https://web.dev/inp/) (INP)

これらのメトリックの結果はすべて`name`プロパティを使って処理できます。

<PagesOnly>

```jsx title="pages/_app.js"
import { useReportWebVitals } from 'next/web-vitals'

function MyApp({ Component, pageProps }) {
  useReportWebVitals((metric) => {
    switch (metric.name) {
      case 'FCP': {
        // FCPの結果を処理
      }
      case 'LCP': {
        // LCPの結果を処理
      }
      // ...
    }
  })

  return <Component {...pageProps} />
}
```

</PagesOnly>

## カスタムメトリック {#custom-metrics}

上記の主要なメトリックに加え、ページのハイドレーションとレンダリングにかかる時間を測定する追加のカスタムメトリックもあります：

- `Next.js-hydration`: ページの開始からハイドレーションが終了するまでの長さ（ミリ秒単位）
- `Next.js-route-change-to-render`: ルート変更後にページがレンダリングを開始するまでの時間の長さ（ミリ秒単位）
- `Next.js-render`: ルート変更後にページのレンダリングが完了するまでの時間の長さ（ミリ秒単位）

これらのメトリックの結果はすべて個別に処理できます：

```jsx title="pages/_app.js"
import { useReportWebVitals } from 'next/web-vitals'

function MyApp({ Component, pageProps }) {
  useReportWebVitals((metric) => {
    switch (metric.name) {
      case 'Next.js-hydration':
        // ハイドレーション結果を処理
        break
      case 'Next.js-route-change-to-render':
        // ルート変更からレンダリングの結果を処理
        break
      case 'Next.js-render':
        // レンダリング結果を処理
        break
      default:
        break
    }
  })

  return <Component {...pageProps} />
}
```

これらのメトリックは[User Timing API](https://caniuse.com/#feat=user-timing)をサポートするすべてのブラウザで動作します。

</PagesOnly>

## Vercelでの使用 {#usage-on-vercel}

[Vercel Speed Insights](https://vercel.com/docs/speed-insights/quickstart)は`useReportWebVitals`を使用せず、代わりに`@vercel/speed-insights`パッケージを使用します。
`useReportWebVitals`フックは、ローカル開発時または他のサービスを使用してWeb Vitalsを収集している場合に便利です。

## 結果を外部システムへ送信 {#sending-results-to-external-systems}

本サイトでの実際のユーザーパフォーマンスを測定し、追跡するために、結果を任意のエンドポイントに送信できます。例えば：

```js
useReportWebVitals((metric) => {
  const body = JSON.stringify(metric)
  const url = 'https://example.com/analytics'

  // `navigator.sendBeacon()`が利用可能な場合は使用し、利用できない場合は`fetch()`にフォールバック
  if (navigator.sendBeacon) {
    navigator.sendBeacon(url, body)
  } else {
    fetch(url, { body, method: 'POST', keepalive: true })
  }
})
```

> **Good to know**: [Googleアナリティクス](https://analytics.google.com/analytics/web/)を使用している場合、
> `id`値を使用してメトリックの分布を手動で構築することが可能です（パーセンタイルを計算するなど）

> ```js
> useReportWebVitals(metric => {
>   // Googleアナリティクスを次の例のように初期化した場合は`window.gtag`を使用してください：
>   // https://github.com/vercel/next.js/blob/canary/examples/with-google-analytics
>   window.gtag('event', metric.name, {
>     value: Math.round(metric.name === 'CLS' ? metric.value * 1000 : metric.value), // 値は整数である必要があります
>     event_label: metric.id, // 現在のページロードに一意のid
>     non_interaction: true, // バウンスレートへの影響を避けます。
>   });
> }
> ```
>
> [Google Analyticsへの結果の送信]について詳しく読んでください。(https://github.com/GoogleChrome/web-vitals#send-the-results-to-google-analytics)。
