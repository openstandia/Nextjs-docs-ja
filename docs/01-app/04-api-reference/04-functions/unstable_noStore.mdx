**バージョン15では、`unstable_noStore`の代わりに[`connection`](/docs/app/api-reference/functions/connection)を使用することを推奨します。**

`unstable_noStore`は、静的レンダリングを使用しないことを指定し、特定のコンポーネントがキャッシュされないように宣言的に設定するために使用できます。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```

> **Good to know**:
>
> - `unstable_noStore` は、`fetch` の `cache: 'no-store'` と同等です
> - `unstable_noStore` は、より細かく利用可能でコンポーネントごとに使用できるため、`export const dynamic = 'force-dynamic'` よりも推奨されます

- [`unstable_cache`](/docs/app/api-reference/functions/unstable_cache) の中で `unstable_noStore` を使用しても静的生成をオプトアウトすることはありません。代わりに、キャッシュの設定を参照して結果をキャッシュするかどうかを決定します。

## 使用法 {#usage}

`fetch` に `cache: 'no-store'` や `next: { revalidate: 0 }` といった追加のオプションを渡したくない場合や、`fetch` が利用できないケースでは、これらのすべてのユースケースの代替として `noStore()` を使用できます。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```
