**バージョン15では、`unstable_noStore`の代わりに[`connection`](/docs/app/api-reference/functions/connection)を使用することをお勧めします；**

`unstable_noStore`は、静的レンダリングを宣言的に除外し、特定のコンポーネントをキャッシュしないことを示すために使用できます。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```

> **Good to know**:
>
> - `unstable_noStore`は、`fetch`の`cache: 'no-store'`と同等です
> - `unstable_noStore`は、`export const dynamic = 'force-dynamic'`よりも細かい制御が可能で、コンポーネントごとに使用できます

- [`unstable_cache`](/docs/app/api-reference/functions/unstable_cache)内で`unstable_noStore`を使用しても静的生成が無効にはなりません；代わりに、キャッシュ設定に従ってキャッシュするかどうかが決定されます

## 使用法 {#usage}

`fetch`に`cache: 'no-store'`や`next: { revalidate: 0 }`のような追加オプションを渡したくない場合や、`fetch`が利用できない場合には、これらすべてのケースの代わりとして`noStore()`を使用できます。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```
