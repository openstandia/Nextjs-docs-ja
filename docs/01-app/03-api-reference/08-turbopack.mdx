[Turbopack](https://turbo.build/pack)は、JavaScriptとTypeScript用に最適化された、Rustで書かれたインクリメンタルバンドラーで、Next.jsに組み込まれています。Turbopackは、Next.jsで`pages`ディレクトリと`app`ディレクトリの両方で使用することができ、ローカル開発をより迅速にします。

Turbopackを有効にするには、Next.jsの開発サーバーを実行するときに`--turbopack`フラグを使用します。

```json title="package.json" highlight={3}
{
  "scripts": {
    "dev": "next dev --turbopack",
    "build": "next build",
    "start": "next start",
    "lint": "next lint"
  }
}
```

## リファレンス {#reference}

### サポートされている機能 {#supported-features}

Next.jsのTurbopackは、ほとんどのユーザー向けにゼロコンフィグレーションであり、より高度なユースケースに対して拡張可能です。Turbopackで現在サポートされている機能について詳しくは、[APIリファレンス](/docs/app/api-reference/config/next-config-js/turbo)を参照してください。

### サポートされていない機能 {#unsupported-features}

Turbopackは現在、`next dev`のみをサポートしており、`next build`はサポートしていません。安定性に向けて進む中で、ビルドのサポートに取り組んでいます。

現在サポートされていない機能は次のとおりです：

- Turbopackは、[Lightning CSS](https://lightningcss.dev/)を利用しており、一部の使用頻度が低いCSSモジュール機能をサポートしていません
  - 関数バリアントのみをサポートする`:local`および`:global`をスタンドアロンの擬似クラスとしてます。例：`:global(a)`。
  - CSS変数に取って代わられた@valueルール
  - `:import`および`:export`のICSSルール
- [無効なCSSコメント構文](https://stackoverflow.com/questions/51843506/are-double-slash-comments-valid-in-css) 例： `//`
  - CSSコメントは、仕様に従って`/* comment */`と書くべきです。
  - Sassなどのプリプロセッサは、この代替構文をサポートしています。
- `next.config.js`での[`webpack()`](/docs/app/api-reference/config/next-config-js/webpack)設定
  - TurbopackはWebpackを置き換えます。これは、webpack設定がサポートされないことを意味します。
  - Turbopackの設定については、[ドキュメント](/docs/app/api-reference/config/next-config-js/turbo)をご覧ください。
  - Turbopackでは、[Webpackローダー](/docs/app/api-reference/config/next-config-js/turbo#configuring-webpack-loaders)のサブセットがサポートされています。
- Babel (`.babelrc`)
  - Turbopackはすべてのトランスパイルと最適化に[SWC](/docs/architecture/nextjs-compiler#why-swc)コンパイラを利用しています。つまり、Babelはデフォルトで含まれていません。
  - `.babelrc`ファイルがある場合、Next.jsが有効にできる一般的なBabelプラグインをSWC変換として含んでいるため、必要ないかもしれません。詳細は、[コンパイラのドキュメント](/docs/architecture/nextjs-compiler#supported-features)を参照してください。
  - 特定のユースケースがカバーされていないことを確認した上でBabelを使用する必要がある場合、Turbopackの[カスタムwebpackローダーのサポート](/docs/app/api-reference/config/next-config-js/turbo#configuring-webpack-loaders)を活用して`babel-loader`を含めることができます。
- App Routerでroot レイアウトを自動的に作成すること
  - この動作は入力ファイルを変更するため現在サポートされていません。その代わり、エラーが表示され、適切な場所に手動でroot レイアウトを追加することが求められます。
- `@next/font`（レガシーフォントサポート）
  - `@next/font`は`next/font`に移行され、非推奨となりました。[`next/font`](/docs/app/building-your-application/optimizing/fonts)は、Turbopackで完全にサポートされています。
- [Relay変換](/docs/architecture/nextjs-compiler#relay)
  - 将来的にこれを実装する予定です。
- `pages/_document.tsx`での`.css`インポートをブロックすること
  - 現在Webpackを使用して、Next.jsは`pages/_document.tsx`の`.css`ファイルのインポートをブロックしています。
  - 将来的にこの警告を実装することを計画しています。
- [`experimental.typedRoutes`](/docs/app/api-reference/config/next-config-js/typedRoutes)
  - 将来的にこれを実装する予定です。
- `experimental.nextScriptWorkers`
  - 将来的にこれを実装する予定です。
- `experimental.sri.algorithm`
  - 将来的にこれを実装する予定です。
- `experimental.fallbackNodePolyfills`
  - 将来的にこれを実装する予定です。
- `experimental.esmExternals`
  - Next.jsでのレガシーesmExternals設定のサポートは現在計画していません。
- [AMP](https://nextjs.org/docs/canary/pages/building-your-application/configuring/amp)
  - Next.jsでTurbopackを使用する際のAMPのサポートは現在計画していません。
- Yarn PnP
  - Next.jsでのYarn PnPのサポートは現在計画していません。
- [`experimental.urlImports`](/docs/app/api-reference/config/next-config-js/urlImports)
  - Next.jsでの`experimental.urlImports`のサポートは現在計画していません。
- [`:import`と`:export`のICSSルール](https://github.com/css-modules/icss)
  - Turbopackが使用するCSSパーサーである[Lightning CSS](https://lightningcss.dev/css-modules.html)がこれらのルールをサポートしていないため、Next.jsでの`:import`と`:export`のICSSルールのサポートは現在計画していません。
- edge runtimeでの`unstable_allowDynamic`設定

## 例 {#examples}

### トレースファイルの生成 {#generating-trace-files}

トレースファイルは、Next.jsチームがパフォーマンス指標とメモリ使用量を調査し改善するのに役立ちます。トレースファイルを生成するには、`next dev --turbopack`コマンドに`NEXT_TURBOPACK_TRACING=1`を追加してください。これにより、`.next/trace.log`ファイルが生成されます。

Turbopackのパフォーマンスおよびメモリ使用量に関連する問題を報告する際には、[GitHub](https://github.com/vercel/next.js)のイシューでトレースファイルを含めてください。
