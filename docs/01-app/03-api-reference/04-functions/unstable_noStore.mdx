バージョン15では、`unstable_noStore`の代わりに[`connection`](/docs/app/api-reference/functions/connection)を使用することをお勧めします。

`unstable_noStore`を使用すると、特定のコンポーネントをキャッシュしないように宣言的に静的レンダリングから除外することができます。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```

> **Good to know**:
>
> - `unstable_noStore`は、`fetch`の`cache: 'no-store'`と同等です
> - より細かくコンポーネントごとに使用できるため、`unstable_noStore`は`export const dynamic = 'force-dynamic'`よりも推奨されます

- [`unstable_cache`](/docs/app/api-reference/functions/unstable_cache)内で`unstable_noStore`を使用しても、静的生成からは除外されません。代わりに、キャッシュの設定によって結果をキャッシュするかどうかが決まります。

## 使用法 {#usage}

`fetch`に対して`cache: 'no-store'`、`next: { revalidate: 0 }`などの追加オプションを渡したくない場合や、`fetch`が利用できない状況では、これらのすべてのユースケースに対して`noStore()`を代わりに使用できます。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```
