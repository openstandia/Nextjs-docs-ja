`forbidden`関数はエラーをスローし、Next.jsの403エラーページをレンダリングします。これは、アプリケーション内で認可エラーを処理するのに便利です。UIは[`forbidden.js` ファイル](/docs/app/api-reference/file-conventions/forbidden)を使用してカスタマイズできます。

`forbidden`を使用するには、`next.config.js`ファイルで実験的な[`authInterrupts`](/docs/app/api-reference/config/next-config-js/authInterrupts)構成オプションを有効にします：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    authInterrupts: true,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
module.exports = {
  experimental: {
    authInterrupts: true,
  },
}
```

</TabItem>
</Tabs>

`forbidden`は[server component](/docs/app/building-your-application/rendering/server-components)、[server actions](/docs/app/building-your-application/data-fetching/server-actions-and-mutations)、および[route handler](/docs/app/building-your-application/routing/route-handlers)で呼び出すことができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/auth/page.tsx" switcher
import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'

export default async function AdminPage() {
  const session = await verifySession()

  // ユーザーが'admin'ロールを持っているか確認
  if (session.role !== 'admin') {
    forbidden()
  }

  // 認可されたユーザー用の管理ページをレンダリング
  return <></>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/auth/page.js" switcher

```

</TabItem>
</Tabs>

## Good to know {#good-to-know}

- `forbidden`関数は[root レイアウト](/docs/app/building-your-application/routing/layouts-and-templates#root-layout-required)では呼び出せません。

## Examples {#examples}

### ロールベースのルート保護 {#role-based-route-protection}

`forbidden`を使用して、ユーザーのロールに基づいて特定のルートへのアクセスを制限できます。これにより、認証されたが必要な権限を持たないユーザーがルートにアクセスできないようになります。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/admin/page.tsx" switcher
import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'

export default async function AdminPage() {
  const session = await verifySession()

  // ユーザーが'admin'ロールを持っているか確認
  if (session.role !== 'admin') {
    forbidden()
  }

  // 認可されたユーザー用の管理ページをレンダリング
  return (
    <main>
      <h1>Admin Dashboard</h1>
      <p>Welcome, {session.user.name}!</p>
    </main>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/admin/page.js" switcher

import { forbidden } from 'next/navigation'

      <h1>Admin Dashboard</h1>
      <p>Welcome, {session.user.name}!</p>
    </main>
  )
}
```

</TabItem>
</Tabs>

### サーバーアクションによるミューテーション {#mutations-with-server-actions}

Server Actionsでミューテーションを実装する際、特定のロールを持つユーザーにのみ機密データの更新を許可するために`forbidden`を使用できます。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions/update-role.ts" switcher
'use server'

import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'
import db from '@/app/lib/db'

export async function updateRole(formData: FormData) {
  const session = await verifySession()

  // 管理者だけがロールを更新できるようにする
  if (session.role !== 'admin') {
    forbidden()
  }

  // 認可されたユーザーのためのロールの更新を実行
  // ...
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/actions/update-role.js" switcher
'use server'

import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'
import db from '@/app/lib/db'

  }

  // 認可されたユーザーのためのロールの更新を実行
  // ...
}
```

</TabItem>
</Tabs>
