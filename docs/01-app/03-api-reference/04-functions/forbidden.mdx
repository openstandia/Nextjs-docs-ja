`forbidden`関数は、Next.jsの403エラーページを表示するエラーを投げます。これはアプリケーション内での認証エラーを処理する際に便利です。[`forbidden.js`ファイル](/docs/app/api-reference/file-conventions/forbidden)を使用してUIをカスタマイズできます。

`forbidden`を使用するには、`next.config.js`ファイルの中で実験的な[`authInterrupts`](/docs/app/api-reference/config/next-config-js/authInterrupts)設定オプションを有効にしてください：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    authInterrupts: true,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
module.exports = {
  experimental: {
    authInterrupts: true,
  },
}
```

</TabItem>
</Tabs>

`forbidden`は[Server Components](/docs/app/building-your-application/rendering/server-components)、[Server Actions](/docs/app/building-your-application/data-fetching/server-actions-and-mutations)、および[Route Handlers](/docs/app/building-your-application/routing/route-handlers)内で呼び出すことができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/auth/route.tsx" switcher
import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'

export default async function AdminPage() {
  const session = await verifySession()

  // ユーザーが 'admin' ロールを持っているか確認
  if (session.role !== 'admin') {
    forbidden()
  }

  // 認可されたユーザーに対して管理ページをレンダリング
  return <></>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/auth/route.js" switcher

```

</TabItem>
</Tabs>

## Good to know {#good-to-know}

- `forbidden`関数は[root レイアウト](/docs/app/building-your-application/routing/layouts-and-templates#root-layout-required)内で呼び出すことはできません

## Examples {#examples}

### ロールベースのルート保護 {#role-based-route-protection}

ユーザーロールに基づいて特定のルートへのアクセスを制限するために`forbidden`を使用できます。これにより認証されているが必要な権限を持っていないユーザーがルートにアクセスできないようになります。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/admin/page.tsx" switcher
import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'

export default async function AdminPage() {
  const session = await verifySession()

  // ユーザーが 'admin' ロールを持っているか確認
  if (session.role !== 'admin') {
    forbidden()
  }

  // 認可されたユーザーに対して管理ページをレンダリング
  return (
    <main>
      <h1>Admin Dashboard</h1>
      <p>Welcome, {session.user.name}!</p>
    </main>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/admin/page.js" switcher

import { forbidden } from 'next/navigation'

  return (
    <main>
      <h1>Admin Dashboard</h1>
      <p>Welcome, {session.user.name}!</p>
    </main>
  )
}
```

</TabItem>
</Tabs>

### Server Actionsを使ったミューテーション {#mutations-with-server-actions}

Server Actionsでミューテーションを実装する際に、特定の役割を持つユーザーだけが機密データを更新できるようにするために`forbidden`を使用できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/auth/route.tsx" switcher
'use server'

import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'
import db from '@/app/lib/db'

export async function updateRole(formData: FormData) {
  const session = await verifySession()

  // 管理者のみがロールを更新できることを確認
  if (session.role !== 'admin') {
    forbidden()
  }

  // 認可されたユーザーに対してロールの更新を実施
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/auth/route.js" switcher
'use server'

import { verifySession } from '@/app/lib/dal'
import { forbidden } from 'next/navigation'
import db from '@/app/lib/db'

  }

  // 認可されたユーザーに対してロールの更新を実施
  // ...
}
```

</TabItem>
</Tabs>
