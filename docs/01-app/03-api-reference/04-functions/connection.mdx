`connection()` 関数は、ユーザーリクエストが入ってくるまでレンダリングを待機することを指定する機能を提供します。

この機能は、コンポーネントが [動的 API](/docs/app/building-your-application/rendering/server-components#dynamic-apis)を使用しない場合でも、ランタイムで動的にレンダリングされてビルド時に静的にレンダリングされないようにしたい場合に役立ちます。これは通常、`Math.random()` や `new Date()` など、レンダリングの結果を意図的に変更したい外部情報にアクセスする場合に発生します。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/page.tsx" switcher
import { connection } from 'next/server'

export default async function Page() {
  await connection()
  // 以下はすべてプリレンダリングから除外されます
  const rand = Math.random()
  return <span>{rand}</span>
}
```

</TabItem>

## リファレンス {#reference}

### 型 {#type}

```jsx
function connection(): Promise<void>
```

### パラメータ {#parameters}

- この関数はパラメータを受け取りません

### 戻り値 {#returns}

- 関数は `void` Promise を返します。この Promise は消費することを目的としていません。

## Good to know {#good-to-know}

- `connection` は、Next.js の将来の方向性とより一致するために [`unstable_noStore`](/docs/app/api-reference/functions/unstable_noStore)を置き換えます。
- 動的レンダリングが必要で、一般的な動的 API が使用されていない場合にのみ、この関数は必要です。
