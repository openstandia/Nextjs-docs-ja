`expirePath` を使用すると、特定のパスに対してオンデマンドで[キャッシュされたデータ](/docs/app/building-your-application/caching)をパージできます。

> **Good to know**:
>
> - `expirePath` は、[Node.jsおよびEdgeランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `expirePath` は含まれたパスが次に訪問されたときにのみキャッシュを無効にします。つまり、`expirePath` を動的な route segment で呼び出すと、一度に多くの有効期限切れをトリガーすることはありません。無効化は次にパスが訪問されたときにのみ発生します。
> - 現在、`expirePath` はサーバーアクションで使用する場合、[クライアントサイドのRouter Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートを無効にします。この動作は一時的なもので、将来的には特定のパスにのみ適用されるように更新される予定です。
> - `expirePath` の使用により、[サーバーサイドのRoute Cache](/docs/app/building-your-application/caching#full-route-cache)における**特定のパスのみ**が無効になります。

## リファレンス {#reference}

### パラメータ {#parameters}

```tsx
expirePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 有効期限を設定したいデータに関連付けられたファイルシステムのパスを表す文字列（例：`/product/[slug]/page`）、またはリテラルの route segment（例：`/product/123`）を指定します。1024文字未満である必要があります。この値は大文字と小文字を区別します。
- `type`: （オプション）有効期限を設定するパスの種類を変更するための `'page'` または `'layout'` 文字列です。`path` に動的なセグメントが含まれている場合（例：`/product/[slug]/page`）、このパラメーターは必須です。パスがリテラルの route segment（例：動的ページの `/product/1`（例：`/product/[slug]/page`））を指している場合は、`type` を指定しないでください。

### 戻り値 {#returns}

`expirePath` は値を返しません。

## 例 {#examples}

### 特定のURLの期限を設定する {#expiring-a-specific-url}

```ts
import { expirePath } from 'next/cache'
expirePath('/blog/post-1')
```

これは次のページ訪問時に特定のURLのキャッシュをパージします。

### ページパスの期限を設定する {#expiring-a-page-path}

```ts
import { expirePath } from 'next/cache'
expirePath('/blog/[slug]', 'page')
// または route group を使用
expirePath('/(main)/blog/[slug]', 'page')
```

これは次のページ訪問時に提供された `page` ファイルに一致するURLのキャッシュをパージします。これは、特定のページの下にあるページを無効にしません。例えば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効にしません。

### レイアウトパスの期限を設定する {#expiring-a-layout-path}

```ts
import { expirePath } from 'next/cache'
expirePath('/blog/[slug]', 'layout')
// または route group を使用
expirePath('/(main)/post/[slug]', 'layout')
```

これは次のページ訪問時に提供された `layout` ファイルに一致するURLのキャッシュをパージします。同じレイアウトを持つ下位のページは次の訪問時に再検証されます。前述の例では、`/blog/[slug]/[another]` も次の訪問時に再検証されます。

### すべてのデータの期限を設定する {#expiring-all-data}

```ts
import { expirePath } from 'next/cache'

expirePath('/', 'layout')
```

これは次のページ訪問時にデータキャッシュをパージします。

### サーバーアクション {#server-action}

サーバーアクションで `expirePath` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions.ts" switcher
'use server'

import { expirePath } from 'next/cache'

export default async function submit() {
  await submitForm()

}
```

### ルートハンドラー {#route-handler}

ルートハンドラーで `expirePath` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/expire/route.ts" switcher
import { expirePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    expirePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    expired: false,
    now: Date.now(),
    message: 'Missing path to expire',
  })
}
```

</TabItem>
