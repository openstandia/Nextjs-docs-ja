`expirePath`は、特定のパスの[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドで削除することを可能にします。

> **Good to know**:
>
> - `expirePath`は、[Node.jsとEdgeランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で使用可能です。
> - `expirePath`は、指定されたパスが次に訪問されたときにのみキャッシュを無効にします。これは、`expirePath`を動的なdynamic route segmentで呼び出しても、すぐに多くのキャッシュの無効化は発生しないことを意味します。無効化は次にそのパスが訪問されたときにのみ行われます。
> - 現在、`expirePath`は、サーバーアクションで使用されるとき、[クライアントサイドのRouter Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートを無効にします。この動作は一時的であり、将来は特定のパスにのみ適用されるように更新されます。
> - `expirePath`を使用すると、[サーバーサイドのRoute Cache](/docs/app/building-your-application/caching#full-route-cache)内の特定のパスのみが無効化されます。

## Reference {#reference}

### Parameters {#parameters}

```tsx
expirePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 対象データに関連付けられたファイルシステムパスを表す文字列（例えば、`/product/[slug]/page`）、またはリテラルなroute segment（例えば、`/product/123`）いずれかです。1024文字未満である必要があります。この値は大文字と小文字を区別します。
- `type`: （オプション）`'page'`または`'layout'`の文字列を指定して、expireするパスの種類を指定します。`path`が動的セグメント（例えば、`/product/[slug]/page`）を含む場合、このパラメータは必須です。pathがリテラルなroute segmentを指している場合（例えば、動的ページ`/product/[slug]/page`の場合、`/product/1`）、`type`は指定しないでください。

### Returns {#returns}

`expirePath`は値を返しません。

## Examples {#examples}

### 特定のURLのキャッシュの無効化 {#expiring-a-specific-url}

```ts
import { expirePath } from 'next/cache'
expirePath('/blog/post-1')
```

これにより、次のページ訪問時に特定のURLのキャッシュが削除されます。

### ページパスのキャッシュ無効化 {#expiring-a-page-path}

```ts
import { expirePath } from 'next/cache'
expirePath('/blog/[slug]', 'page')
// またはroute groupを使用して
expirePath('/(main)/blog/[slug]', 'page')
```

これにより、次のページ訪問時に提供された`page`ファイルに一致するすべてのURLのキャッシュが削除されます。ただし、指定したページの下のページは無効化されません。例えば、`/blog/[slug]`は`/blog/[slug]/[author]`を無効化しません。

### レイアウトパスのキャッシュ無効化 {#expiring-a-layout-path}

```ts
import { expirePath } from 'next/cache'
expirePath('/blog/[slug]', 'layout')
// またはroute groupを使用して
expirePath('/(main)/post/[slug]', 'layout')
```

これにより、次のページ訪問時に提供された`layout`ファイルに一致するすべてのURLのキャッシュが削除されます。同じレイアウトを持つ下位のページが次の訪問時に再検証されることになります。例えば、上記の場合、`/blog/[slug]/[another]`も次の訪問時に再検証されます。

### すべてのデータのキャッシュの無効化 {#expiring-all-data}

```ts
import { expirePath } from 'next/cache'

expirePath('/', 'layout')
```

これにより、次のページ訪問時にデータキャッシュが削除されます。

### サーバーアクション {#server-action}

サーバーアクションで`expirePath`を呼び出すことができます：

### ルートハンドラー {#route-handler}

ルートハンドラーで`expirePath`を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/expire/route.ts" switcher
import { expirePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    expirePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    expired: false,
    now: Date.now(),
    message: 'Missing path to expire',
  })
}
```

</TabItem>
