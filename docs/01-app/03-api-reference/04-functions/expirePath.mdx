`expirePath` は特定のパスに対してオンデマンドで[キャッシュデータ](/docs/app/building-your-application/caching)をパージする機能を提供します。

> **Good to know**:
>
> - `expirePath` は[Node.js および Edge 実行環境](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `expirePath` は、指定したパスが次に訪問されたときにのみキャッシュを無効化します。つまり、dynamic route segment を含むパスで `expirePath` を呼び出しても、一度に多くの無効化を即座にトリガーすることはありません。無効化はそのパスが次に訪問されるときにのみ行われます。
> - 現在、サーバーアクションで使用される場合、`expirePath` は [client-side Router Cache](/docs/app/building-your-application/caching#client-side-router-cache) 内のすべてのルートを無効化します。この動作は一時的なもので、将来的には特定のパスにだけ適用されるように更新される予定です。
> - `expirePath` を使用すると、[server-side Route Cache](/docs/app/building-your-application/caching#full-route-cache)内の**その特定のパスだけを**無効化します。

## Reference {#reference}

### Parameters {#parameters}

```tsx
expirePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 期限切れにしたいデータに関連付けられたファイルシステムパスの文字列（例: `/product/[slug]/page`）、または、リテラル route segment（例: `/product/123`）。文字数は1024文字未満である必要があります。この値は大文字と小文字を区別します。
- `type`: （オプション）期限切れにするパスのタイプを変更するための `'page'` または `'layout'` の文字列。`path` に dynamic segment が含まれる場合（例: `/product/[slug]/page`）、このパラメータが必要です。リテラル route segment（例: dynamic page(`/product/[slug]/page`)の場合、`/product/1`）を指す場合、`type` を指定してはいけません。

### Returns {#returns}

`expirePath` は値を返しません。

## Examples {#examples}

### 特定のURLの無効化 {#expiring-a-specific-url}

```ts
import { expirePath } from 'next/cache'
expirePath('/blog/post-1')
```

これにより、次のページ訪問時に特定のURLのキャッシュがパージされます。

### ページパスの無効化 {#expiring-a-page-path}

```ts
import { expirePath } from 'next/cache'
expirePath('/blog/[slug]', 'page')
// または、ルートグループとともに
expirePath('/(main)/blog/[slug]', 'page')
```

これにより、次のページ訪問時に指定された `page` ファイルに一致する任意のURLのキャッシュがパージされます。これは、特定のページの下のページを無効化しません。たとえば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効化しません。

### レイアウトパスの無効化 {#expiring-a-layout-path}

```ts
import { expirePath } from 'next/cache'
expirePath('/blog/[slug]', 'layout')
// または、ルートグループとともに
expirePath('/(main)/post/[slug]', 'layout')
```

これにより、次回のページ訪問時に指定された `layout` ファイルに一致する任意のURLのキャッシュがパージされます。同じレイアウトを持つ下のページも、次の訪問時に再検証されます。たとえば、上記の場合、`/blog/[slug]/[another]` も次の訪問時に再検証されます。

### すべてのデータの無効化 {#expiring-all-data}

```ts
import { expirePath } from 'next/cache'

expirePath('/', 'layout')
```

これにより、次のページ訪問時にデータキャッシュがパージされます。

### サーバーアクション {#server-action}

サーバーアクションで `expirePath` を呼び出すことができます：

<TabItem value="ts" label="TypeScript">

import { expirePath } from 'next/cache'

  await submitForm()

### Route Handler {#route-handler}

Route Handlerで`expirePath`を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/expire/route.ts" switcher
import { expirePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    expirePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    expired: false,
    now: Date.now(),
    message: 'Missing path to expire',
  })
}
```

</TabItem>
