---
title: 'revalidateTag'
description: 'revalidateTag 関数の API リファレンス'
---

`revalidateTag` は、特定のキャッシュタグについて、オンデマンドで[キャッシュされたデータ](/docs/app/building-your-application/caching)をパージすることを可能にします。

> **Good to know**:
>
> - `revalidateTag` は、[Node.js と Edge runtime](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `revalidateTag` は、該当のパスが次回訪問されたときにのみキャッシュを無効にします。これは `revalidateTag` を dynamic route segment とともに呼び出しても、直ちに多くの再検証を一度に引き起こさないことを意味します。無効化は、該当のパスが次回訪問されたときにのみ発生します。

## パラメータ {#parameters}

```tsx
revalidateTag(tag: string): void;
```

- `tag`: 再検証したいデータに関連付けられたキャッシュタグを表す文字列; 256文字以下である必要があります。この値は大文字と小文字を区別します。

`fetch` にタグを以下のように追加できます：

```tsx
fetch(url, { next: { tags: [...] } });
```

## 戻り値 {#returns}

`revalidateTag` は値を返しません。

## 例 {#examples}

### Server Action {#server-action}

'use server'

import { revalidateTag } from 'next/cache'

// 投稿を追加し、キャッシュを無効化する

### Route Handler {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { revalidateTag } from 'next/cache'

// クエリパラメータからタグを取得し、キャッシュを無効化する
export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')
  revalidateTag(tag)
  return Response.json({ revalidated: true, now: Date.now() })
}
```
