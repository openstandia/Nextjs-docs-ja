---
title: 'revalidateTag'
description: 'revalidateTag 関数の API リファレンス'
---

`revalidateTag` は、特定のキャッシュタグに対する[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドでパージできるようにします。

> **Good to know**:
>
> - `revalidateTag` は、[Node.js と Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で使用できます。
> - `revalidateTag` は、パスが次に訪問されたときにのみキャッシュを無効化します。これは、`revalidateTag` を dynamic route segment に対して呼び出しても、一度に多数の再検証が即座にトリガーされるわけではないことを意味します。無効化は、パスが次に訪問されたときにのみ発生します。

## パラメーター {#parameters}

```tsx
revalidateTag(tag: string): void;
```

- `tag`: 再検証したいデータに関連付けられているキャッシュタグを表す文字列です。256文字以下である必要があります。この値は大文字と小文字を区別します。

次のようにして `fetch` にタグを追加できます：

```tsx
fetch(url, { next: { tags: [...] } });
```

## 戻り値 {#returns}

`revalidateTag` は値を返しません。

### Route Handler {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { revalidateTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')
  revalidateTag(tag) // 取得したタグのキャッシュを再検証します
  return Response.json({ revalidated: true, now: Date.now() }) // 再検証されたことを返します
}
```
