---
title: 'revalidateTag'
description: 'revalidateTag 関数の API リファレンス'
---

`revalidateTag` は、特定のキャッシュタグに対応する[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドでクリアすることを可能にします。

> **Good to know**:
>
> - `revalidateTag` は [Node.js および Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes) の両方で利用可能です。
> - `revalidateTag` はパスが次回訪問されたときにのみキャッシュを無効化します。つまり、`revalidateTag` を dynamic route segment と共に呼び出しても、多くの再検証が一度にすぐにトリガーされるわけではありません。無効化はパスが次に訪問されたときのみ発生します。

## パラメーター {#parameters}

```tsx
revalidateTag(tag: string): void;
```

- `tag`: 再検証したいデータに関連付けられているキャッシュタグを表す文字列。256 文字以下である必要があります。この値は大文字小文字を区別します。

以下の方法で `fetch` にタグを追加できます：

```tsx
fetch(url, { next: { tags: [...] } });
```

## 戻り値 {#returns}

`revalidateTag` は値を返しません。

### Route Handler {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { revalidateTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')
  revalidateTag(tag)
  return Response.json({ revalidated: true, now: Date.now() })
}
```
