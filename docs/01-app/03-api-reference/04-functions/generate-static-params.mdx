title: 'generateStaticParams'
description: 'generateStaticParams関数のAPIリファレンスです。'
---

`generateStaticParams`関数は、[動的ルートセグメント](/docs/app/building-your-application/routing/dynamic-routes)と組み合わせて使用することで、リクエスト時ではなくビルド時にルートを[**静的に生成**](/docs/app/building-your-application/rendering/server-components#static-rendering-default)できます。

```jsx title="app/blog/[slug]/page.js"
// [slug]動的セグメントを埋める`params`のリストを返す
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// generateStaticParams`から返された`params`を使って
// このページの複数のバージョンが静的に生成されます
export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

> **Good to know**:
>
> - `generateStaticParams`を使用して生成されなかった動的セグメントが訪問されたときの挙動を制御するために、[`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams)セグメント構成オプションを使用できます。
> - [実行時にパスを再検証するため](/docs/app/building-your-application/rendering/server-components#static-rendering-default)、[`generateStaticParams`から空の配列を返す](#all-paths-at-build-time)か、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を利用する必要があります。
> - `next dev`の間、ルートに移動すると`generateStaticParams`が呼び出されます。
> - `next build`の間、`generateStaticParams`は対応するレイアウトやページが生成される前に実行されます。
> - 再検証（ISR）の際、`generateStaticParams`は再び呼び出されません。
> - `generateStaticParams`は、Pages Routerでの[`getStaticPaths`](https://nextjs.org/docs/canary/pages/api-reference/functions/get-static-paths)関数を置き換えます。

## パラメータ {#parameters}

`options.params`（オプション）

ルートで複数の動的セグメントが`generateStaticParams`を使用する場合、親が生成する各`params`セットに対して子の`generateStaticParams`関数が一度実行されます。

`params`オブジェクトには、親`generateStaticParams`から生成された`params`が含まれており、これを使用して[子セグメントの`params`を生成](#multiple-dynamic-segments-in-a-route)できます。

## 戻り値 {#returns}

`generateStaticParams`は、各オブジェクトが単一のルートの動的セグメントを表すオブジェクトの配列を返さなければなりません。

- オブジェクト内の各プロパティは、そのルートのために埋めるべき動的セグメントです。
- プロパティの名前はセグメントの名前で、プロパティの値がそのセグメントに埋めるべきものです。

| ルート例                         | `generateStaticParams`の戻り値の型        |
| -------------------------------- | ----------------------------------------- |
| `/product/[id]`                  | `{ id: string }[]`                        |
| `/products/[category]/[product]` | `{ category: string, product: string }[]` |
| `/products/[...slug]`            | `{ slug: string[] }[]`                    |

## 例 {#examples}

### 静的レンダリング {#static-rendering}

#### ビルド時にすべてのパス {#all-paths-at-build-time}

ビルド時にすべてのパスを静的にレンダリングするには、パスの完全なリストを`generateStaticParams`に提供します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

#### ビルド時にパスのサブセット {#subset-of-paths-at-build-time}

ビルド時にパスのサブセットを静的にレンダリングし、残りを実行時に初めて訪れたときにレンダリングするには、部分的なパスのリストを返します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // 最初の10個の投稿をビルド時にレンダリング
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

次に、[`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams)セグメント構成オプションを使用することで、`generateStaticParams`を使用して生成されなかった動的セグメント訪問時の挙動を制御できます。

```jsx title="app/blog/[slug]/page.js"
// 上位10個以外のすべての投稿は404に
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### 実行時にすべてのパス {#all-paths-at-runtime}

初めて訪問したときにすべてのパスを静的にレンダリングするには、空の配列を返す（ビルド時にパスはレンダリングされません）か、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を利用してください：

```jsx title="app/blog/[slug]/page.js"
export async function generateStaticParams() {
  return []
}
```

> **Good to know:** `generateStaticParams`からは常に配列を返す必要があります；たとえそれが空であってもです。そうしないと、ルートは動的にレンダリングされます。

```jsx title="app/changelog/[slug]/page.js"
export const dynamic = 'force-static'
```

### 未指定パスのレンダリングを無効にする {#disable-rendering-for-unspecified-paths}

未指定のパスが実行時に静的にレンダリングされるのを防ぐには、ルートセグメントで`export const dynamicParams = false`オプションを追加します。この構成オプションを使用する場合、`generateStaticParams`によって提供されたパスのみがサービスされ、未指定のルートは404または一致します（[catch-all routes](/docs/app/building-your-application/routing/dynamic-routes#catch-all-segments)の場合）。

### ルートにおける複数の動的セグメント {#multiple-dynamic-segments-in-a-route}

現在のレイアウトやページの上の動的セグメントのためにparamsを生成できますが、 **下のセグメントのためには生成できません**。例えば、`app/products/[category]/[product]`ルートの場合：

- `app/products/[category]/[product]/page.js`は`[category]` および `[product]`の両方のparamsを生成できます。
- `app/products/[category]/layout.js`は`[category]`のparamsしか生成できません。

複数の動的セグメントを持つルートのためのparams生成には2つのアプローチがあります：

#### 下から上へのparams生成 {#generate-params-from-the-bottom-up}

子ルートセグメントから複数の動的セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// [category]と[product]の両方のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher

  const products = await fetch('https://.../products').then((res) => res.json())

  // ...
}
```

</TabItem>
</Tabs>

#### 上から下へのparams生成 {#generate-params-from-the-top-down}

親セグメントを最初に生成し、その結果を使用して子セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/layout.tsx" switcher
// [category]セグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({
  params,
}: {
  params: Promise<{ category: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/layout.js" switcher
// [category]セグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

親の`generateStaticParams`が生成する各セグメントに対して、子ルートセグメントの`generateStaticParams`関数が一度実行されます。

子の`generateStaticParams`は、親`generateStaticParams`関数から返された`params`を使用して、自分の独自のセグメントを動的に生成できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// 親セグメントの`generateStaticParams`関数から渡された
// `params`を利用して、[product]セグメントを生成
export async function generateStaticParams({
  params: { category },
}: {
  params: { category: string }
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// 親セグメントの`generateStaticParams`関数から渡された
// `params`を利用して、[product]セグメントを生成
export async function generateStaticParams({ params: { category } }) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

> **Good to know**: `fetch`リクエストは、すべての`generate`の接頭辞が付いた関数、レイアウト、ページ、およびサーバーコンポーネントで同じデータについて自動的に[メモ化](/docs/app/building-your-application/caching#request-memoization)されます。React [`cache` can be used](/docs/app/building-your-application/caching#react-cache-function) `fetch`が利用できない場合に使用できます。
