title: 'generateStaticParams'
description: 'generateStaticParams 関数のAPIリファレンスです。'
---

`generateStaticParams` 関数は、[dynamic route segments](/docs/app/building-your-application/routing/dynamic-routes)と組み合わせて、リクエスト時にオンデマンドで行うのではなくビルド時にルートを[**静的に生成**](/docs/app/building-your-application/rendering/server-components#static-rendering-default)するために使用できます。

```jsx title="app/blog/[slug]/page.js"
// `params` のリストを返して [slug] 動的セグメントを埋めます
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// `generateStaticParams` によって返された `params` を使用して
// このページの複数のバージョンが静的に生成されます
export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

> **Good to know**:
>
> - [`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams) セグメント設定オプションを使用して `generateStaticParams` で生成されなかった動的セグメントに訪問した際の挙動を制御できます。
> - ビルド時に全経路を再検証（ISR）するためには、[ `generateStaticParams` から空の配列を返す](#all-paths-at-build-time)か、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を利用しなければなりません。
> - `next dev` 実行中に、ルートに移動すると `generateStaticParams` が呼び出されます。
> - `next build` 実行中に、対応するLayoutsやPagesが生成される前に `generateStaticParams` が実行されます。
> - 再検証（ISR）の間、再度 `generateStaticParams` は呼び出されません。
> - `generateStaticParams` はPages Router における [`getStaticPaths`](https://nextjs.org/docs/canary/pages/api-reference/functions/get-static-paths) 関数の代わりになります。

## パラメータ {#parameters}

`options.params` （オプション）

1つのルートに複数の動的セグメントが `generateStaticParams` を使用している場合、親が生成する `params` のセットごとに1回、子の `generateStaticParams` 関数が実行されます。

`params` オブジェクトには、親の `generateStaticParams` から出力された `params` が含まれており、[子セグメントの`params`を生成する](#multiple-dynamic-segments-in-a-route)のに使用できます。

## 戻り値 {#returns}

`generateStaticParams` は、各オブジェクトが単一路線の動的セグメントを埋めたものを表すようなオブジェクトの配列を返すべきです。

- オブジェクトの各プロパティは、ルートで埋める動的セグメントです
- プロパティの名前がセグメントの名前であり、プロパティの値がそのセグメントに含めるべき内容です

| 例のルート                       | `generateStaticParams` の戻り値の型       |
| -------------------------------- | ----------------------------------------- |
| `/product/[id]`                  | `{ id: string }[]`                        |
| `/products/[category]/[product]` | `{ category: string, product: string }[]` |
| `/products/[...slug]`            | `{ slug: string[] }[]`                    |

## 例 {#examples}

### 静的レンダリング {#static-rendering}

#### ビルド時にすべてのパス {#all-paths-at-build-time}

ビルド時にすべてのパスを静的にレンダリングするには、`generateStaticParams`にパスの完全なリストを提供します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

#### ビルド時のパスのサブセット {#subset-of-paths-at-build-time}

ビルド時に一部のパスを静的にレンダリングし、残りはランタイムで初めてアクセスしたときにレンダリングするには、部分的なパスリストを返します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // 最初の10件の投稿をビルド時にレンダリング
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

その後、[`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams) セグメント設定オプションを使用して、`generateStaticParams` で生成されなかった動的セグメントに訪問した際の挙動を制御できます。

```jsx title="app/blog/[slug]/page.js"
// 最初の10件以外の投稿は404となります
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### ランタイムでのすべてのパス {#all-paths-at-runtime}

訪問された初回にすべてのパスを静的にレンダリングするには、空の配列を返します（ビルド時にパスはレンダリングされません）または[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を利用します：

```jsx title="app/blog/[slug]/page.js"
export async function generateStaticParams() {
  return []
}
```

> **Good to know:** たとえ空であっても `generateStaticParams` からは常に配列を返さなくてはなりません。そうでない場合、ルートは動的にレンダリングされます。

```jsx title="app/changelog/[slug]/page.js"
export const dynamic = 'force-static'
```

### 指定されていないパスのレンダリングを無効化する {#disable-rendering-for-unspecified-paths}

指定されていないパスをランタイムで静的にレンダリングするのを防ぐために、ルートのセグメントに `export const dynamicParams = false` オプションを追加します。この設定オプションを使用すると、`generateStaticParams` が提供するパスだけがサービスされ、指定されていないルートは404になります（[catch-all routes](/docs/app/building-your-application/routing/dynamic-routes#catch-all-segments) の場合は一致します）。

### ルート内の複数の動的セグメント {#multiple-dynamic-segments-in-a-route}

現在のレイアウトまたはページの上の動的セグメントのために `params` を生成することができますが、**下**のためにはできません。例えば、`app/products/[category]/[product]` のルートを考えた場合：

- `app/products/[category]/[product]/page.js` は `[category]` と `[product]` の両方のために `params` を生成できます。
- `app/products/[category]/layout.js` は `[category]` のために**のみ** `params` を生成できます。

複数の動的セグメントを持つルートのために `params` を生成するには、以下の2つのアプローチがあります：

#### ボトムアップで `params` を生成する {#generate-params-from-the-bottom-up}

子ルートセグメントから複数の動的セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// [category] と [product] の両方のセグメントを生成します
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// [category] と [product] の両方のセグメントを生成します
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

#### トップダウンで `params` を生成する {#generate-params-from-the-top-down}

親のセグメントを先に生成し、その結果を用いて子のセグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/layout.tsx" switcher
// [category] のセグメントを生成します
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({
  params,
}: {
  params: Promise<{ category: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/layout.js" switcher
// [category] のセグメントを生成します
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

親の `generateStaticParams` が生成する各セグメントに対して、子プライマリートの `generateStaticParams` 関数が1回実行されます。

子の `generateStaticParams` 関数は、親の `generateStaticParams` 関数から出力された `params` を使用して、自分のセグメントを動的に生成することができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// 親セグメントの `generateStaticParams` 関数から渡された `params` を使用して
// [product] のセグメントを生成します
export async function generateStaticParams({
  params: { category },
}: {
  params: { category: string }
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// 親セグメントの `generateStaticParams` 関数から渡された `params` を使用して
// [product] のセグメントを生成します
export async function generateStaticParams({ params: { category } }) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

> **Good to know**: `fetch` リクエストは、すべての `generate` プレフィックス関数、Layouts、Pages、およびサーバーコンポーネントの間で同じデータに対して自動的に[memoized](/docs/app/building-your-application/caching#request-memoization)されます。React の[`cache` を使用することができます](/docs/app/building-your-application/caching#react-cache-function) （`fetch` が利用できない場合）。
