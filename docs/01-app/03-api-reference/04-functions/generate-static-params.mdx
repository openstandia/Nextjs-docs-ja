title: 'generateStaticParams'

---

`generateStaticParams` 関数は、[dynamic route segments](/docs/app/building-your-application/routing/dynamic-routes)と組み合わせて、リクエスト時ではなくビルド時にルートを[**静的に生成**](/docs/app/building-your-application/rendering/server-components#static-rendering-default)するために使用できます。

```jsx title="app/blog/[slug]/page.js"
// [slug] dynamic segmentを埋めるための `params` のリストを返す
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// `generateStaticParams` によって返される `params` を使用して
// このページの複数のバージョンが静的に生成されます
export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

> **Good to know**:
>
> - `generateStaticParams`で生成されたことがないdynamic segmentを訪問した場合の動作を制御するために、[`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams)セグメント設定オプションを使用できます。
> - [実行時にパスを再検証](#all-paths-at-runtime)（ISR）するためには、[`generateStaticParams`](#all-paths-at-build-time)から空の配列を返すか、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を利用する必要があります。
> - `next dev`中は、ルートへのナビゲート時に`generateStaticParams`が呼び出されます。
> - `next build`中は、対応するLayoutsやPagesが生成される前に、`generateStaticParams`が実行されます。
> - 再検証（ISR）の間、`generateStaticParams`は再び呼び出されることはありません。
> - `generateStaticParams`はPages Routerにおける[`getStaticPaths`](https://nextjs.org/docs/canary/pages/api-reference/functions/get-static-paths)関数の代わりとなります。

## パラメータ {#parameters}

`options.params`（オプション）

ルート内の複数のdynamic segmentが`generateStaticParams`を使用すると、親が生成した`params`セットごとに一度、子の`generateStaticParams`関数が実行されます。

`params`オブジェクトには、親の`generateStaticParams`で生成された`params`が含まれており、それは[子セグメントで`params`を生成するために](#multiple-dynamic-segments-in-a-route)使用できます。

## 戻り値 {#returns}

`generateStaticParams`は、各オブジェクトが単一のルートの埋め込まれたdynamic segmentを表すオブジェクトの配列を返す必要があります。

- オブジェクト内の各プロパティは、ルートに埋め込むためのdynamic segmentです。
- プロパティ名はセグメントの名前であり、プロパティの値はそのセグメントに埋め込むべき内容です。

| ルートの例                       | `generateStaticParams` の戻り値の型       |
| -------------------------------- | ----------------------------------------- |
| `/product/[id]`                  | `{ id: string }[]`                        |
| `/products/[category]/[product]` | `{ category: string, product: string }[]` |
| `/products/[...slug]`            | `{ slug: string[] }[]`                    |

## 例 {#examples}

### 静的レンダリング {#static-rendering}

#### ビルド時にすべてのパス {#all-paths-at-build-time}

すべてのパスをビルド時に静的レンダリングするには、`generateStaticParams` に完全なパスのリストを渡します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

#### ビルド時に一部のパス {#subset-of-paths-at-build-time}

ビルド時にいくつかのパスを静的レンダリングし、残りは実行時に初めて訪問されたときにレンダリングするには、パスの部分的なリストを返します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // 最初の10件の投稿をビルド時にレンダリング
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

その後、[`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams)セグメント設定オプションを使用して、`generateStaticParams`で生成されていないdynamic segmentが訪問された場合の動作を制御できます。

```jsx title="app/blog/[slug]/page.js"
// トップ10以外のすべての投稿は404になる
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### 実行時にすべてのパス {#all-paths-at-runtime}

すべてのパスを最初に訪問されたときに静的レンダリングするには、空の配列を返す（ビルド時にはパスはレンダリングされません）か、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を利用します：

```jsx title="app/blog/[slug]/page.js"
export async function generateStaticParams() {
  return []
}
```

> **Good to know:** たとえそれが空であっても`generateStaticParams`から常に配列を返さなければなりません。そうしないと、そのルートは動的にレンダリングされます。

```jsx title="app/changelog/[slug]/page.js"
export const dynamic = 'force-static'
```

### 未指定のパスのレンダリングを無効化する {#disable-rendering-for-unspecified-paths}

未指定のパスが実行時に静的にレンダリングされることを防ぐには、ルートセグメントで`export const dynamicParams = false`オプションを追加します。この設定オプションが使用されると、`generateStaticParams`で提供されるパスのみが提供され、未指定のルートは404または（[catch-all routes](/docs/app/building-your-application/routing/dynamic-routes#catch-all-segments)の場合）一致します。

### ルート内の複数の Dynamic Segment {#multiple-dynamic-segments-in-a-route}

現在のレイアウトやページの上位にあるdynamic segmentのためのパラメータを生成できますが、それを下回ることはできません。 例えば、`app/products/[category]/[product]`ルートにおいて：

- `app/products/[category]/[product]/page.js`は、`[category]`と`[product]`の両方のパラメータを生成できます。
- `app/products/[category]/layout.js`は`[category]`のみのパラメータを生成できます。

複数のdynamic segmentを持つルートのためのパラメータを生成するには、二つのアプローチがあります：

#### ボトムアップでパラメータを生成する {#generate-params-from-the-bottom-up}

子ルートセグメントから複数のdynamic segmentを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// [category]と[product]の両方のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// [category]と[product]の両方のセグメントを生成

  const products = await fetch('https://.../products').then((res) => res.json())

    category: product.category.slug,
    product: product.id,
  }))
}

  // ...
}
```

</TabItem>
</Tabs>

#### トップダウンでパラメータを生成する {#generate-params-from-the-top-down}

親セグメントをまず生成し、その結果を使用して子セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/layout.tsx" switcher
// [category]のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({
  params,
}: {
  params: Promise<{ category: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/layout.js" switcher
// [category]のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

親の`generateStaticParams`が生成する各セグメントに対して、子ルートセグメントの`generateStaticParams`関数が1回実行されます。

子の`generateStaticParams`関数は、親の`generateStaticParams`関数から返される`params`を使用して、自身のセグメントを動的に生成できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// 親セグメントの`generateStaticParams`関数から渡された`params`を使用して
// [product]のセグメントを生成します
export async function generateStaticParams({
  params: { category },
}: {
  params: { category: string }
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// 親セグメントの`generateStaticParams`関数から渡された`params`を使用して
// [product]のセグメントを生成します
export async function generateStaticParams({ params: { category } }) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

> **Good to know**: `fetch` リクエストは、`generate` で始まるすべての関数、Layouts、Pages、Server Components にわたって同じデータに対して自動的に[メモ化](/docs/app/building-your-application/caching#request-memoization)されます。 `fetch` が利用できない場合、React[`cache` を使用](/docs/app/building-your-application/caching#react-cache-function)することができます。
