title: 'generateStaticParams'

`generateStaticParams` 関数は、[動的ルートセグメント](/docs/app/building-your-application/routing/dynamic-routes)と組み合わせて、リクエスト時ではなくビルド時にルートを[**静的に生成**](/docs/app/building-your-application/rendering/server-components#static-rendering-default)するために使用できます。

```jsx title="app/blog/[slug]/page.js"
// [slug] 動的セグメントを埋めるための `params` のリストを返します
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// `generateStaticParams` によって返された `params` を使用して
// このページの複数のバージョンが静的に生成されます
export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

> **Good to know**:
>
> - [`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams) セグメント構成オプションを使用して、`generateStaticParams` で生成されなかった動的セグメントを訪問したときの挙動を制御できます。
> - ビルド時にすべてのルートをリバリデート（ISR）するには、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を使用するか、 [`generateStaticParams`](#all-paths-at-build-time) から空の配列を返す必要があります。
> - `next dev` の間、ルートにナビゲートすると `generateStaticParams` が呼び出されます。
> - `next build` の間、`generateStaticParams` は対応する Layouts や Pages が生成される前に実行されます。
> - リバリデーション（ISR）の間、`generateStaticParams` は再び呼び出されません。
> - `generateStaticParams` は Pages Router の [`getStaticPaths`](https://nextjs.org/docs/canary/pages/api-reference/functions/get-static-paths) 関数を置き換えます。

## Parameters {#parameters}

`options.params`（オプション）

ルート内の複数の動的セグメントが `generateStaticParams` を使用している場合、親が生成した各セットの `params` に対して一度だけ子の `generateStaticParams` 関数が実行されます。

`params` オブジェクトは、親の `generateStaticParams` から得た `params` を含み、[子セグメントの `params` を生成する](#multiple-dynamic-segments-in-a-route)のに使用できます。

## Returns {#returns}

`generateStaticParams` は、各オブジェクトが1つのルートの動的セグメントを埋めたものであるオブジェクトの配列を返すべきです。

- オブジェクト内の各プロパティは、ルートに埋めるための動的セグメントです。
- プロパティ名はセグメントの名前で、プロパティ値はそのセグメントに埋めるべきものです。

| 例のルート                       | `generateStaticParams` の返却タイプ       |
| -------------------------------- | ----------------------------------------- |
| `/product/[id]`                  | `{ id: string }[]`                        |
| `/products/[category]/[product]` | `{ category: string, product: string }[]` |
| `/products/[...slug]`            | `{ slug: string[] }[]`                    |

## Examples {#examples}

### Static Rendering {#static-rendering}

#### All paths at build time {#all-paths-at-build-time}

すべてのパスをビルド時に静的にレンダリングするには、`generateStaticParams` に完全なパスのリストを渡します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

#### Subset of paths at build time {#subset-of-paths-at-build-time}

一部のパスをビルド時に静的にレンダリングし、他のパスを実行時に初めて訪れたときにレンダリングするには、一部のパスのリストを返します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // 最初の10記事をビルド時にレンダリング
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

そして、[`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams) セグメント構成オプションを使用して、`generateStaticParams` で生成されていない動的セグメントを訪問したときに何が起こるかを制御できます。

```jsx title="app/blog/[slug]/page.js"
// 上位10記事以外は404
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### All paths at runtime {#all-paths-at-runtime}

初めて訪れたときにすべてのパスを静的にレンダリングするには、空の配列を返す（ビルド時には何もレンダリングされません）、または[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を利用します：

```jsx title="app/blog/[slug]/page.js"
export async function generateStaticParams() {
  return []
}
```

> **Good to know:** `generateStaticParams` からの返り値は常に配列でなければなりません。たとえそれが空であっても。そうしないと、ルートは動的にレンダリングされます。

```jsx title="app/changelog/[slug]/page.js"
export const dynamic = 'force-static'
```

### Disable rendering for unspecified paths {#disable-rendering-for-unspecified-paths}

特定されていないパスを実行時に静的にレンダリングしないようにするには、ルートセグメントに `export const dynamicParams = false` オプションを追加します。この設定オプションを使用すると、`generateStaticParams` が提供するパスのみがサーブされ、指定されていないルートは404か（[catch-all ルート](/docs/app/building-your-application/routing/dynamic-routes#catch-all-segments)の場合は）マッチします。

### Multiple Dynamic Segments in a Route {#multiple-dynamic-segments-in-a-route}

現在のレイアウトやページより上の動的セグメントのために `params` を生成できますが、下のものはできません。たとえば、`app/products/[category]/[product]` ルートで：

- `app/products/[category]/[product]/page.js` は `[category]` と `[product]` の両方に対して `params` を生成できます。
- `app/products/[category]/layout.js` は `[category]` のみの `params` を生成できます。

複数の動的セグメントを持つルートの `params` を生成するには、2つのアプローチがあります：

#### Generate params from the bottom up {#generate-params-from-the-bottom-up}

子ルートセグメントから複数の動的セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// [category] と [product] の両方のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// [category] と [product] の両方のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

#### Generate params from the top down {#generate-params-from-the-top-down}

親セグメントを最初に生成し、その結果を使用して子セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/layout.tsx" switcher
// [category] のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({
  params,
}: {
  params: Promise<{ category: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/layout.js" switcher
// [category] のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

子ルートセグメントの `generateStaticParams` 関数は、親の `generateStaticParams` が生成した各セグメントに対して一度だけ実行されます。

子の `generateStaticParams` 関数は、親の `generateStaticParams` 関数から返される `params` を使用して自分自身のセグメントを動的に生成できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// 親セグメントの `generateStaticParams` 関数から渡された `params` を使用して
// [product] のセグメントを生成
export async function generateStaticParams({
  params: { category },
}: {
  params: { category: string }
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// 親セグメントの `generateStaticParams` 関数から渡された `params` を使用して
// [product] のセグメントを生成
export async function generateStaticParams({ params: { category } }) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

> **Good to know**: `fetch` リクエストは、すべての `generate` プレフィックスが付いた関数、Layouts、Pages、Server Components で同じデータに対して自動的に[memoization](/docs/app/building-your-application/caching#request-memoization)されます。React の [`cache` can be used](/docs/app/building-your-application/caching#react-cache-function) も使用できます。
