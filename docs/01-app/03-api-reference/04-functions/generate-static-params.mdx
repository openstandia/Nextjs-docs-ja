`generateStaticParams` 関数は、[動的ルートセグメント](/docs/app/building-your-application/routing/dynamic-routes)と組み合わせて使用することで、リクエスト時にオンデマンドで行う代わりに、ビルド時にルートを[**静的生成**](/docs/app/building-your-application/rendering/server-components#static-rendering-default)することができます。

```jsx title="app/blog/[slug]/page.js"
// [slug] 動的セグメントを埋めるための `params` のリストを返す
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// このページの複数のバージョンが、
// `generateStaticParams` の `params` によって静的に生成
export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

> **Good to know**:
>
> - `dynamicParams` セグメント設定オプションを使用して、`generateStaticParams` で生成されなかった動的セグメントにアクセスしたときに何が起こるかを制御できます；
> - [ビルド時のすべてのパス](#all-paths-at-build-time)から `generateStaticParams` を空の配列として返すか、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic) を利用して、ランタイムでの[パスを再検証](#all-paths-at-runtime)する必要があります；
> - `next dev` の間、ルートに移動すると `generateStaticParams` が呼び出されます；
> - `next build` の間、対応するレイアウトまたはページが生成される前に `generateStaticParams` が実行されます；
> - 再検証（ISR）の間に、`generateStaticParams` は再び呼び出されません；
> - `generateStaticParams` は Pages Router の[`getStaticPaths`](https://nextjs.org/docs/canary/pages/api-reference/functions/get-static-paths) 関数の代わりとなります；

## パラメーター {#parameters}

`options.params`（オプション）

ルート内で複数の動的セグメントが `generateStaticParams` を使用する場合、子の `generateStaticParams` 関数は、親が生成する各 `params` セットに対して1回実行されます。

`params` オブジェクトには、親の `generateStaticParams` から生成された `params` が含まれており、[子セグメントでの `params` の生成](#multiple-dynamic-segments-in-a-route)に使用できます。

## 戻り値 {#returns}

`generateStaticParams` は、各オブジェクトが単一ルートの動的セグメントを表す、オブジェクトの配列を返す必要があります。

- オブジェクト内の各プロパティは、ルートのために埋められるべき動的セグメントです；
- プロパティ名はセグメントの名前であり、プロパティ値はそのセグメントに埋められるべき値です；

| ルート例                         | `generateStaticParams` 戻り値の型         |
| -------------------------------- | ----------------------------------------- |
| `/product/[id]`                  | `{ id: string }[]`                        |
| `/products/[category]/[product]` | `{ category: string, product: string }[]` |
| `/products/[...slug]`            | `{ slug: string[] }[]`                    |

## 例 {#examples}

### 静的レンダリング {#static-rendering}

#### すべてのパスをビルド時に {#all-paths-at-build-time}

すべてのパスをビルド時に静的レンダリングするには、`generateStaticParams` に完全なパスのリストを提供します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

#### ビルド時のパスのサブセット {#subset-of-paths-at-build-time}

一部のパスをビルド時に静的レンダリングし、残りをランタイムで初めてアクセスしたときにレンダリングするには、部分的なパスのリストを返します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // 最初の10個の投稿をビルド時にレンダリング
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

次に、`dynamicParams` セグメント設定オプションを使用して、`generateStaticParams` で生成されなかった動的セグメントにアクセスしたときに何が起こるかを制御できます；

```jsx title="app/blog/[slug]/page.js"
// 上位10個以外の投稿は404になります
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### ランタイムでのすべてのパス {#all-paths-at-runtime}

初めて訪問されたときにすべてのパスを静的にレンダリングするには、空の配列（ビルド時にパスはレンダリングされません）を返すか、`export const dynamic = 'force-static'` を利用します：

```jsx title="app/blog/[slug]/page.js"
export async function generateStaticParams() {
  return []
}
```

> **Good to know:** `generateStaticParams` からは必ず配列を返す必要があります；たとえそれが空であってもです。そうしない場合、そのルートは動的にレンダリングされます。

```jsx title="app/changelog/[slug]/page.js"
export const dynamic = 'force-static'
```

### 指定されていないパスのレンダリングを無効にする {#disable-rendering-for-unspecified-paths}

指定されていないパスがランタイムで静的にレンダリングされないようにするためには、ルートセグメントに `export const dynamicParams = false` オプションを追加します。この設定オプションが使用されると、`generateStaticParams` で提供されるパスのみが提供され、指定されていないルートは404またはマッチし（キャッチオールルートの場合）、します。

### 複数の動的セグメントを含むルート {#multiple-dynamic-segments-in-a-route}

現在のレイアウトまたはページの上の動的セグメントのパラメーターを生成できますが、下のセグメントのパラメーターは生成できません。たとえば、`app/products/[category]/[product]` ルートの場合：

- `app/products/[category]/[product]/page.js` は `[category]` と `[product]` の両方のパラメーターを生成できます；
- `app/products/[category]/layout.js` は `[category]` のパラメーターのみを生成できます。

複数の動的セグメントを含むルートに対してパラメーターを生成するための2つのアプローチがあります：

#### 下から上へパラメーターを生成 {#generate-params-from-the-bottom-up}

子のルートセグメントから複数の動的セグメントを生成します；

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// [category] と [product] の両方のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher

  // ...
}
```

</TabItem>
</Tabs>

#### 上から下へパラメーターを生成 {#generate-params-from-the-top-down}

親セグメントを最初に生成し、その結果を使用して子セグメントを生成します；

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/layout.tsx" switcher

export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({
  params,
}: {
  params: Promise<{ category: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/layout.js" switcher
// [category] のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

子ルートセグメントの `generateStaticParams` 関数は、親が生成する各セグメントのために1回実行されます。

子 `generateStaticParams` 関数は、親の `generateStaticParams` 関数から返された `params` を使用して独自のセグメントを動的に生成できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// 親セグメントの `generateStaticParams` 関数から渡された `params` を
// 使用して[product]のセグメントを生成
export async function generateStaticParams({
  params: { category },
}: {
  params: { category: string }
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// 親セグメントの `generateStaticParams` 関数から渡された `params` を
// 使用して[product]のセグメントを生成
export async function generateStaticParams({ params: { category } }) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

    product: product.id,
  }))
}

  // ...
}
```

</TabItem>
</Tabs>

> **Good to know**: `fetch` リクエストは、すべての `generate` サフィックスが付いた関数、レイアウト、ページ、およびサーバーコンポーネントにわたって、同じデータに対して自動的に[メモ化](/docs/app/building-your-application/caching#request-memoization)されます；`fetch` が使用できない場合は、React [`cache` が使用可能です](/docs/app/building-your-application/caching#react-cache-function)。
