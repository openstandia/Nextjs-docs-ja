title: 'generateStaticParams'

`generateStaticParams`関数は、[動的ルートセグメント](/docs/app/building-your-application/routing/dynamic-routes)と組み合わせて使用し、リクエスト時のオンデマンドではなく、ビルド時にルートを[**静的に生成**](/docs/app/building-your-application/rendering/server-components#static-rendering-default)することができます。

```jsx title="app/blog/[slug]/page.js"
// `params`のリストを返して、[slug] 動的セグメントを埋める
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// このページの複数のバージョンが、`generateStaticParams`で返された`params`を使用して静的に生成されます
export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

> **Good to know**:
>
> - [`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams)セグメント設定オプションを使用して、`generateStaticParams`で生成されなかった動的セグメントにアクセスしたときの挙動を制御できます。
> - [実行時にパスを再検証する](#all-paths-at-runtime)には、[`generateStaticParams`](#all-paths-at-build-time)から[空の配列を返す](#all-paths-at-build-time)か、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を使用する必要があります。
> - `next dev`の際、`generateStaticParams`はルートに移動する際に呼び出されます。
> - `next build`の際、`generateStaticParams`は対応するレイアウトやページが生成される前に実行されます。
> - 再検証（ISR）の間、`generateStaticParams`は再度呼び出されません。
> - `generateStaticParams`は、Pages Routerの[`getStaticPaths`](https://nextjs.org/docs/canary/pages/api-reference/functions/get-static-paths)関数の代わりに使用されます。

## パラメーター {#parameters}

`options.params` (任意)

ルート内の複数の動的セグメントが`generateStaticParams`を使用すると、親が生成した各セットの`params`に対して、子の`generateStaticParams`関数が1回実行されます。

`params`オブジェクトには、親の`generateStaticParams`から取得した`params`が含まれ、それを使用して[子セグメントの`params`を生成](#multiple-dynamic-segments-in-a-route)することができます。

## 戻り値 {#returns}

`generateStaticParams`は、各オブジェクトが単一ルートの動的セグメントを埋めるオブジェクトの配列を返す必要があります。

- オブジェクト内の各プロパティは、ルート用に埋められるべき動的セグメントです
- プロパティ名はセグメントの名前であり、プロパティの値はそのセグメントが埋められるべき内容です

| ルートの例                       | `generateStaticParams` の戻り値の型       |
| -------------------------------- | ----------------------------------------- |
| `/product/[id]`                  | `{ id: string }[]`                        |
| `/products/[category]/[product]` | `{ category: string, product: string }[]` |
| `/products/[...slug]`            | `{ slug: string[] }[]`                    |

## 例 {#examples}

### 静的レンダリング {#static-rendering}

#### すべてのパスをビルド時にレンダリングする {#all-paths-at-build-time}

すべてのパスをビルド時に静的にレンダリングするには、`generateStaticParams`にすべてのパスのリストを提供します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

#### ビルド時に一部のパスのみをレンダリングする {#subset-of-paths-at-build-time}

ビルド時に一部のパスを静的にレンダリングし、残りは実行時に最初にアクセスされたときにレンダリングするには、一部のパスのリストを返します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // 最初の10個の投稿をビルド時にレンダリングする
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

次に、[`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams)セグメント設定オプションを使用して、`generateStaticParams`で生成されなかった動的セグメントにアクセスした際の挙動を制御できます。

```jsx title="app/blog/[slug]/page.js"
// 上位10件以外のすべての投稿が404になります
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### すべてのパスを実行時にレンダリングする {#all-paths-at-runtime}

最初に訪問されたときにすべてのパスを静的にレンダリングするには、空の配列を返す（ビルド時にはパスはレンダリングされない）か、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を使用します：

```jsx title="app/blog/[slug]/page.js"
export async function generateStaticParams() {
  return []
}
```

> **Good to know:** `generateStaticParams`から常に配列を返す必要があります。たとえそれが空であってもです。さもなければ、ルートが動的にレンダリングされることになります。

```jsx title="app/changelog/[slug]/page.js"
export const dynamic = 'force-static'
```

### 未指定のパスのレンダリングを無効にする {#disable-rendering-for-unspecified-paths}

未指定のパスが実行時に静的にレンダリングされないようにするには、ルートセグメントに`export const dynamicParams = false`オプションを追加します。この設定オプションを使用すると、`generateStaticParams`で指定されたパスのみが提供され、未指定のルートは404になるか（[キャッチオールルート](/docs/app/building-your-application/routing/dynamic-routes#catch-all-segments)のケースでは）マッチします。

### ルート内の複数の動的セグメント {#multiple-dynamic-segments-in-a-route}

現在のレイアウトやページより上の動的セグメントに対して`generateParams`を生成できますが、**その下では**できません。たとえば、`app/products/[category]/[product]`ルートの場合：

- `app/products/[category]/[product]/page.js`は`[category]`と`[product]`の両方に対して`params`を生成できます。
- `app/products/[category]/layout.js`は`[category]`のみに対して`params`を生成できます。

複数の動的セグメントを持つルートの`params`を生成するには2つのアプローチがあります：

#### 末端から生成する {#generate-params-from-the-bottom-up}

子ルートセグメントから複数の動的セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// [category] と [product] の両方に対してセグメントを生成する
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// [category] と [product] の両方に対してセグメントを生成する
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

#### 上位から生成する {#generate-params-from-the-top-down}

最初に親セグメントを生成し、その結果を使用して子セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/layout.tsx" switcher
// [category] に対してセグメントを生成する
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({
  params,
}: {
  params: Promise<{ category: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/layout.js" switcher
// [category] に対してセグメントを生成する
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

子ルートセグメントの`generateStaticParams`関数は、親の`generateStaticParams`が生成する各セグメントに対して1回実行されます。

子の`generateStaticParams`関数は、親の`generateStaticParams`関数から返された`params`を使用して、自身のセグメントを動的に生成することができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// 親セグメントの`generateStaticParams`関数から渡された`params`を使用して[product]のセグメントを生成する
export async function generateStaticParams({
  params: { category },
}: {
  params: { category: string }
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// 親セグメントの`generateStaticParams`関数から渡された`params`を使用して[product]のセグメントを生成する
export async function generateStaticParams({ params: { category } }) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

> **Good to know**: `fetch`のリクエストは、すべての`generate`プレフィックスが付いた関数、レイアウト、ページ、サーバーコンポーネント横断で同じデータに対して自動的に[メモ化](/docs/app/building-your-application/caching#request-memoization)されます。Reactの[`cache`が使用可能](/docs/app/building-your-application/caching#react-cache-function)であれば、`fetch`が使用できない場合に使用できます。
