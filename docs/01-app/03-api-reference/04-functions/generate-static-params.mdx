title: 'generateStaticParams'

`generateStaticParams` 関数は、[動的ルートセグメント](/docs/app/building-your-application/routing/dynamic-routes)と組み合わせて使用することで、要求時ではなくビルド時にルートを[**静的に生成**](/docs/app/building-your-application/rendering/server-components#static-rendering-default)できます。

```jsx title="app/blog/[slug]/page.js"
// [slug] 動的セグメントを埋める `params` のリストを返す
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// `generateStaticParams` によって返された `params` を使用して、このページの複数のバージョンが静的に生成される
export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

> **Good to know**:
>
> - [`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams) セグメント設定オプションを使用して、`generateStaticParams` を使用して生成されていない動的セグメントが訪問された際の動作を制御できます。
> - 実行時にパスを再検証（ISR）するためには、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic) を利用するか、[`generateStaticParams`](#all-paths-at-build-time) から空の配列を返す必要があります。
> - `next dev` の間、ルートにナビゲートすると `generateStaticParams` が呼び出されます。
> - `next build` の間、対応するLayoutsまたはPagesが生成される前に `generateStaticParams` が実行されます。
> - 再検証（ISR）の間、`generateStaticParams` は再び呼び出されません。
> - `generateStaticParams` はPages Routerの[`getStaticPaths`](https://nextjs.org/docs/canary/pages/api-reference/functions/get-static-paths) 関数に置き換わります。

## パラメータ {#parameters}

`options.params` （オプション）

1つのルートの中で複数の動的セグメントが `generateStaticParams` を使用する場合、親が生成する各セットの `params` ごとに子の `generateStaticParams` 関数が1回実行されます。

`params` オブジェクトには親の `generateStaticParams` から取得した `params` が含まれており、それを使用して[子セグメントの`params`を生成](#multiple-dynamic-segments-in-a-route)できます。

## 戻り値 {#returns}

`generateStaticParams` は、各オブジェクトが単一ルートの動的セグメントの入力項目を表すオブジェクトの配列を返すべきです。

- オブジェクト内の各プロパティは、ルートのために埋められるべき動的セグメントです。
- プロパティ名はセグメント名で、プロパティ値はそのセグメントに埋められるべきものです。

| ルート例                         | `generateStaticParams` の戻り値型         |
| -------------------------------- | ----------------------------------------- |
| `/product/[id]`                  | `{ id: string }[]`                        |
| `/products/[category]/[product]` | `{ category: string, product: string }[]` |
| `/products/[...slug]`            | `{ slug: string[] }[]`                    |

## 例 {#examples}

### 静的レンダリング {#static-rendering}

#### ビルド時の全パス {#all-paths-at-build-time}

ビルド時にすべてのパスを静的にレンダリングするには、`generateStaticParams` にパスの完全なリストを提供します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

#### ビルド時の一部のパス {#subset-of-paths-at-build-time}

ビルド時に一部のパスを静的にレンダリングし、残りは実行時に最初に訪問されたときにレンダリングするには、部分的なリストを返します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // 最初の10件の投稿をビルド時にレンダリングする
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

次に、[`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams) セグメント設定オプションを使用して、`generateStaticParams` を使用して生成されていない動的セグメントが訪問された際の動作を制御できます。

```jsx title="app/blog/[slug]/page.js"
// 上位10件以外の投稿は404になる
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### 実行時の全パス {#all-paths-at-runtime}

最初に訪問されたときに全パスを静的にレンダリングするには、空の配列を返す（ビルド時にはどのパスもレンダリングされない）か、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic) を利用します：

```jsx title="app/blog/[slug]/page.js"
export async function generateStaticParams() {
  return []
}
```

> **Good to know:** `generateStaticParams`からは常に配列を返す必要があります。それが空であってもです。そうでないと、ルートは動的にレンダリングされます。

```jsx title="app/changelog/[slug]/page.js"
export const dynamic = 'force-static'
```

### 未指定のパスのレンダリングを無効にする {#disable-rendering-for-unspecified-paths}

未指定のパスが実行時に静的にレンダリングされるのを防ぐには、ルートセグメントで `export const dynamicParams = false` オプションを追加します。この設定オプションが使用されている場合、`generateStaticParams` によって提供されたパスのみが提供され、未指定のルートは404になるか（[catch-all routes](/docs/app/building-your-application/routing/dynamic-routes#catch-all-segments)の場合は一致）します。

### ルート内の複数の動的セグメント {#multiple-dynamic-segments-in-a-route}

現在のレイアウトまたはページより上の動的セグメントのためのパラメータは生成できますが、下のセグメントについては生成できません。たとえば、`app/products/[category]/[product]` ルートがあるとします：

- `app/products/[category]/[product]/page.js` は、[category] と [product] の **両方**のパラメータを生成できます。
- `app/products/[category]/layout.js` は、[category] のみのパラメータを生成できます。

複数の動的セグメントを持つルートのパラメータを生成するには2つのアプローチがあります：

#### 逆順でパラメータを生成する {#generate-params-from-the-bottom-up}

子ルートセグメントから複数の動的セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// [category] と [product] 両方のセグメントを生成する
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// [category] と [product] 両方のセグメントを生成する
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

#### 上位から順にパラメータを生成する {#generate-params-from-the-top-down}

最初に親セグメントを生成し、その結果を使用して子セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/layout.tsx" switcher
// [category] セグメントを生成する
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({
  params,
}: {
  params: Promise<{ category: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/layout.js" switcher
// [category] セグメントを生成する
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

子ルートセグメントの `generateStaticParams` 関数は、親の `generateStaticParams` が生成する各セグメントごとに1回実行されます。

子の `generateStaticParams` 関数は、親の `generateStaticParams` 関数から返される `params` を使用して自身のセグメントを動的に生成できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// 親セグメントの `generateStaticParams` 関数から渡された `params` を使用して [product] セグメントを生成する
export async function generateStaticParams({
  params: { category },
}: {
  params: { category: string }
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// 親セグメントの `generateStaticParams` 関数から渡された `params` を使用して [product] セグメントを生成する
export async function generateStaticParams({ params: { category } }) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

> **Good to know**: `fetch` リクエストは、すべての `generate` プレフィックス付き関数、レイアウト、ページ、およびサーバーコンポーネント間で同じデータに対して自動的に[メモ化](/docs/app/building-your-application/caching#request-memoization)されます。`fetch` が利用できない場合は、Reactの[`cache`を使用することができます](/docs/app/building-your-application/caching#react-cache-function)。
