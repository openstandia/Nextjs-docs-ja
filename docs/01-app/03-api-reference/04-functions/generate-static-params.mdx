title: 'generateStaticParams'

---

`generateStaticParams`関数は、[dynamic route segment](/docs/app/building-your-application/routing/dynamic-routes)と組み合わせて使用することで、リクエスト時ではなくビルド時にルートを[**静的に生成**](/docs/app/building-your-application/rendering/server-components#static-rendering-default)できます。

```jsx title="app/blog/[slug]/page.js"
// [slug] dynamic segmentを埋めるための`params`のリストを返す
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// このページの複数のバージョンが、`generateStaticParams`によって返される`params`を使用して静的に生成されます
export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

> **Good to know**:
>
> - `dynamicParams` segment configオプションを使用することで、`generateStaticParams`で生成されなかったdynamic segmentが訪問されたときに何が起こるかを制御できます。
> - すべての[パスを実行時に再検証](#all-paths-at-runtime)するには、[`generateStaticParams`から空の配列を返す](#all-paths-at-build-time)か、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を利用する必要があります。
> - `next dev`では、ルートに移動したときに`generateStaticParams`が呼び出されます。
> - `next build`では、対応するLayoutsやPagesが生成される前に`generateStaticParams`が実行されます。
> - 再検証（ISR）中に、`generateStaticParams`が再度呼び出されることはありません。
> - `generateStaticParams`は、Pages Routerの[`getStaticPaths`](https://nextjs.org/docs/canary/pages/api-reference/functions/get-static-paths)関数を置換します。

## パラメータ {#parameters}

`options.params`（オプション）

ルート内の複数のdynamic segmentが`generateStaticParams`を使用している場合、子`generateStaticParams`関数は親が生成する`params`のセットごとに一度実行されます。

`params`オブジェクトには、親`generateStaticParams`から生成された`params`が含まれています。これを使用して[子segmentで`params`を生成](#multiple-dynamic-segments-in-a-route)できます。

## 戻り値 {#returns}

`generateStaticParams`は、各オブジェクトが単一のルートのpopulated dynamic segmentを表すオブジェクトの配列を返す必要があります。

- 各オブジェクトのプロパティは、ルートで埋め込むべきdynamic segmentです。
- プロパティ名はsegmentの名前であり、プロパティ値はそのsegmentに埋め込むべき値です。

| Example Route                    | `generateStaticParams` Return Type        |
| -------------------------------- | ----------------------------------------- |
| `/product/[id]`                  | `{ id: string }[]`                        |
| `/products/[category]/[product]` | `{ category: string, product: string }[]` |
| `/products/[...slug]`            | `{ slug: string[] }[]`                    |

## 例 {#examples}

### 静的レンダリング {#static-rendering}

#### ビルド時にすべてのパス {#all-paths-at-build-time}

すべてのパスをビルド時に静的にレンダリングするには、`generateStaticParams`にパスの完全なリストを供給します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

#### ビルド時の一部のパス {#subset-of-paths-at-build-time}

ビルド時に一部のパスを静的レンダリングし、残りは初めて実行時に訪問されたときにレンダリングされるようにするには、部分的なパスのリストを返します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // 最初の10投稿をビルド時にレンダリングします
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

次に、`dynamicParams` segment configオプションを使用することで、`generateStaticParams`で生成されなかったdynamic segmentが訪問されたときに何が起こるかを制御できます。

```jsx title="app/blog/[slug]/page.js"
// 上位10投稿以外は404になります
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### 実行時にすべてのパス {#all-paths-at-runtime}

すべてのパスを初めて訪問したときに静的にレンダリングするには、空の配列を返すか（ビルド時にパスはレンダリングされません）、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)を利用します：

```jsx title="app/blog/[slug]/page.js"
export async function generateStaticParams() {
  return []
}
```

> **Good to know:** たとえ空であっても、`generateStaticParams`からは常に配列を返す必要があります。それ以外の場合、ルートは動的にレンダリングされます。

```jsx title="app/changelog/[slug]/page.js"
export const dynamic = 'force-static'
```

### 未指定のパスのためのレンダリングを無効にする {#disable-rendering-for-unspecified-paths}

指定されていないパスが実行時に静的にレンダリングされるのを防ぐために、ルートセグメントに`export const dynamicParams = false`オプションを追加します。このconfigオプションが使用されると、`generateStaticParams`から提供されたパスのみが提供され、指定されていないルートは404またはマッチ（[catch-all routes](/docs/app/building-your-application/routing/dynamic-routes#catch-all-segments)の場合）します。

### 複数のDynamic Segmentを含むルート {#multiple-dynamic-segments-in-a-route}

現在のレイアウトまたはページの上のdynamic segmentのためにparamsを生成できますが、下のためには生成できません。例えば、`app/products/[category]/[product]`ルートの場合：

- `app/products/[category]/[product]/page.js`は、`[category]`と`[product]`の両方のためにparamsを生成できます。
- `app/products/[category]/layout.js`は、`[category]`のみのためにparamsを生成できます。

複数のdynamic segmentを持つルートのためにparamsを生成するには2つのアプローチがあります：

#### 下から上へparamsを生成する {#generate-params-from-the-bottom-up}

子ルートsegmentから複数のdynamic segmentを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// [category]と[product]の両方のためのsegmentを生成します
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher

</TabItem>

  params,

  params: Promise<{ category: string }>

  // ...

```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/layout.js" switcher

    category: product.category.slug,
  }))

export default function Layout({ params }) {
  // ...

```

</TabItem>

子ルートsegmentの`generateStaticParams`関数は、親`generateStaticParams`が生成する各segmentごとに一度実行されます。

子の`generateStaticParams`関数は、親の`generateStaticParams`関数から返された`params`を使用して、自身のsegmentを動的に生成できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// 親segmentの`generateStaticParams`関数から渡された`params`を使用して[product]のためのsegmentを生成します
export async function generateStaticParams({
  params: { category },
}: {
  params: { category: string }
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// 親segmentの`generateStaticParams`関数から渡された`params`を使用して[product]のためのsegmentを生成します
export async function generateStaticParams({ params: { category } }) {

  ).then((res) => res.json())

    product: product.id,
  }))

  // ...
}
```

</TabItem>
</Tabs>

> **Good to know**: `fetch`リクエストは、すべての`generate`プレフィックスが付いた関数、Layouts、Pages、Server Componentsにわたって同じデータについて自動的に[メモ化](/docs/app/building-your-application/caching#request-memoization)されます。React [`cache function を使用することができます`](/docs/app/building-your-application/caching#react-cache-function) `fetch`が利用できない場合。
