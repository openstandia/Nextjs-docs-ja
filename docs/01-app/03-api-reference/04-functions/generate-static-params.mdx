title: 'generateStaticParams'
description: 'generateStaticParams 関数の API リファレンス'
---

`generateStaticParams` 関数は、[動的ルートセグメント](/docs/app/building-your-application/routing/dynamic-routes)と組み合わせて使用し、リクエスト時ではなくビルド時にルートを[**静的に生成**](/docs/app/building-your-application/rendering/server-components#static-rendering-default)することができます。

```jsx title="app/blog/[slug]/page.js"
// `params` のリストを返し、[slug] 動的セグメントを埋めます
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// このページの複数のバージョンが、`generateStaticParams` によって返された `params` を使用して静的に生成されます
export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

> **知っておくと良いこと**:
>
> - `generateStaticParams` で生成されていない動的セグメントにアクセスしたときに何が起こるかを制御するためには、[`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams) セグメント設定オプションを使用できます。
> - [実行時にパスを再検証（ISR）](#all-paths-at-runtime)するには、`generateStaticParams` から[空の配列を返す](#all-paths-at-build-time)か、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic) を利用する必要があります。
> - `next dev` 中に、`generateStaticParams` は、ルートに移動すると呼び出されます。
> - `next build` 中に、`generateStaticParams` は対応する Layouts または Pages が生成される前に実行されます。
> - 再検証（ISR）中に、`generateStaticParams` は再度呼び出されません。
> - `generateStaticParams` は、Pages Router の [`getStaticPaths`](https://nextjs.org/docs/canary/pages/api-reference/functions/get-static-paths) 関数を置き換えます。

## パラメータ {#parameters}

`options.params` (オプション)

ルート内の複数の動的セグメントが `generateStaticParams` を使用している場合、親が生成した `params` のセットごとに子の `generateStaticParams` 関数が 1 回実行されます。

`params` オブジェクトには、親の `generateStaticParams` からの埋められた `params` が含まれ、[子セグメントで `params` を生成するのに](#multiple-dynamic-segments-in-a-route)使用できます。

## 戻り値 {#returns}

`generateStaticParams` は、それぞれのオブジェクトが単一のルートの埋められた動的セグメントを表すオブジェクトの配列を返す必要があります。

- オブジェクト内の各プロパティは、ルートに埋めるべき動的セグメントです。
- プロパティの名前はセグメントの名前で、プロパティの値はそのセグメントを何で埋めるべきかです。

| 例のルート                       | `generateStaticParams` 戻り値の型         |
| -------------------------------- | ----------------------------------------- |
| `/product/[id]`                  | `{ id: string }[]`                        |
| `/products/[category]/[product]` | `{ category: string, product: string }[]` |
| `/products/[...slug]`            | `{ slug: string[] }[]`                    |

## 例 {#examples}

### 静的レンダリング {#static-rendering}

#### 全てのパスをビルド時にレンダリングする {#all-paths-at-build-time}

ビルド時に全てのパスを静的にレンダリングするには、`generateStaticParams` に全てのパスのリストを提供します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

#### ビルド時に一部のパスのみをレンダリングする {#subset-of-paths-at-build-time}

ビルド時に一部のパスのみを静的にレンダリングし、残りは実行時に最初に訪問されたときにレンダリングするには、部分的なパスのリストを返します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // ビルド時に最初の10記事をレンダリング
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

</TabItem>
</Tabs>

次に、[`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config#dynamicparams) セグメント設定オプションを使って、`generateStaticParams` で生成されていない動的セグメントにアクセスしたときに何が起こるかを制御できます。

```jsx title="app/blog/[slug]/page.js"
// 最初の10記事以外は404になる
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### 実行時に全てのパスをレンダリングする {#all-paths-at-runtime}

全てのパスを最初の訪問時に静的にレンダリングするには、空の配列を返す（ビルド時にレンダリングされるパスはありません）か、[`export const dynamic = 'force-static'`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic) を利用します：

```jsx title="app/blog/[slug]/page.js"
export async function generateStaticParams() {
  return []
}
```

> **知っておくと良いこと**: `generateStaticParams` からは空でも常に配列を返す必要があります。そうでないと、ルートは動的にレンダリングされます。

```jsx title="app/changelog/[slug]/page.js"
export const dynamic = 'force-static'
```

### 未指定のパスのレンダリングを無効化する {#disable-rendering-for-unspecified-paths}

未指定のパスがランタイムに静的にレンダリングされないようにするためには、ルートセグメントに `export const dynamicParams = false` オプションを追加します。この設定オプションが使用されると、`generateStaticParams` で提供されたパスのみが提供され、未指定のルートは404になるか（[キャッチオールルート](/docs/app/building-your-application/routing/dynamic-routes#catch-all-segments)の場合）一致します。

### ルート内の複数の動的セグメント {#multiple-dynamic-segments-in-a-route}

現在のレイアウトやページより上の動的セグメントのパラメータを生成することができますが、**下はできません**。たとえば、`app/products/[category]/[product]` ルートの場合：

- `app/products/[category]/[product]/page.js` は `[category]` と `[product]` の両方のパラメータを生成できます。
- `app/products/[category]/layout.js` は `[category]` のパラメータのみ生成できます。

複数の動的セグメントを持つルートのパラメータを生成するには、次の2つのアプローチがあります：

#### 下から上へパラメータを生成する {#generate-params-from-the-bottom-up}

子ルートセグメントから複数の動的セグメントを生成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// [category] と [product] の両方のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// [category] と [product] の両方のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

#### 上から下へパラメータを生成する {#generate-params-from-the-top-down}

親セグメントを先に生成し、その結果を子セグメントの生成に使用します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/layout.tsx" switcher
// [category] のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({
  params,
}: {
  params: Promise<{ category: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/layout.js" switcher
// [category] のセグメントを生成
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

子ルートセグメントの `generateStaticParams` 関数は、親の `generateStaticParams` が生成するセグメントごとに1回実行されます。

子の `generateStaticParams` 関数は、親の `generateStaticParams` 関数から返された `params` を使用して自分自身のセグメントを動的に生成できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[category]/[product]/page.tsx" switcher
// 親セグメントの `generateStaticParams` 関数から渡された `params` を使用して [product] のセグメントを生成
export async function generateStaticParams({
  params: { category },
}: {
  params: { category: string }
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[category]/[product]/page.js" switcher
// 親セグメントの `generateStaticParams` 関数から渡された `params` を使用して [product] のセグメントを生成
export async function generateStaticParams({ params: { category } }) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

</TabItem>
</Tabs>

> **知っておくと良いこと**: `fetch` リクエストは、全ての `generate` 接頭辞付き関数、Layouts、Pages、Server Components 間で同じデータに対して自動的に[メモ化](/docs/app/building-your-application/caching#request-memoization)されます。`fetch` を利用できない場合は、React [`cache` が使用できます](/docs/app/building-your-application/caching#react-cache-function).
