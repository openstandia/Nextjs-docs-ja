`unauthorized` 関数は、Next.jsで401エラーページを表示するエラーを発生させます。これは、アプリケーション内で認可エラーを処理するのに便利です。UIは[`unauthorized.js` ファイル](/docs/app/api-reference/file-conventions/unauthorized)を使ってカスタマイズできます。

`unauthorized` の利用を始めるには、`next.config.js` ファイルで、実験的な [`authInterrupts`](/docs/app/api-reference/config/next-config-js/authInterrupts) 設定オプションを有効にしてください：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    authInterrupts: true,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
module.exports = {
  experimental: {
    authInterrupts: true,
  },
}
```

</TabItem>
</Tabs>

`unauthorized` は、[Server Components](/docs/app/building-your-application/rendering/server-components)、[Server Actions](/docs/app/building-your-application/data-fetching/server-actions-and-mutations)、および[Route Handlers](/docs/app/building-your-application/routing/route-handlers)で呼び出すことができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/dashboard/page.tsx" switcher
import { verifySession } from '@/app/lib/dal'
import { unauthorized } from 'next/navigation'

export default async function DashboardPage() {
  const session = await verifySession()

  if (!session) {
    unauthorized()
  }

  // 認証されたユーザーに対してダッシュボードを表示します
  return (
    <main>
      <h1>Welcome to the Dashboard</h1>
      <p>Hi, {session.user.name}.</p>
    </main>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/dashboard/page.js" switcher

      <h1>Welcome to the Dashboard</h1>
      <p>Hi, {session.user.name}.</p>
    </main>

```

</TabItem>
</Tabs>

## Good to know {#good-to-know}

- `unauthorized` 関数は、[root レイアウト](/docs/app/building-your-application/routing/layouts-and-templates#root-layout-required)で呼び出すことはできません。

## Examples {#examples}

### ログインUIの表示 {#displaying-login-ui-to-unauthenticated-users}

未認証のユーザーに対して `unauthorized.js` ファイルを表示するために `unauthorized` 関数を使用できます。

import { verifySession } from '@/app/lib/dal'

export default async function DashboardPage() {
  const session = await verifySession()

  if (!session) {
    unauthorized()
  }

  return <div>Dashboard</div>

### Server Actions を用いたミューテーション {#mutations-with-server-actions}

特定のミューテーションを実行できるのは認証されたユーザーのみに限定するために、Server Actions で `unauthorized` を呼び出すことができます。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions/update-profile.ts" switcher
'use server'

import { verifySession } from '@/app/lib/dal'
import { unauthorized } from 'next/navigation'
import db from '@/app/lib/db'

export async function updateProfile(data: FormData) {
  const session = await verifySession()

  // ユーザーが認証されていない場合、401を返します
  if (!session) {
    unauthorized()
  }

  // ミューテーションを進行します
  // ...
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/actions/update-profile.js" switcher

  // ...
}
```

</TabItem>
</Tabs>

### Route Handlers を用いたデータの取得 {#fetching-data-with-route-handlers}

`unauthorized` を Route Handlers で使用することで、認証されたユーザーのみがエンドポイントにアクセスできるようにすることができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/api/profile/route.ts" switcher
import { NextRequest, NextResponse } from 'next/server'
import { verifySession } from '@/app/lib/dal'
import { unauthorized } from 'next/navigation'

export async function GET(req: NextRequest): Promise<NextResponse> {
  // ユーザーのセッションを検証します
  const session = await verifySession()

  // セッションが存在しない場合、401を返し unauthorized.tsx を表示します
  if (!session) {
    unauthorized()
  }

  // データ取得
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/api/profile/route.js" switcher
import { verifySession } from '@/app/lib/dal'
import { unauthorized } from 'next/navigation'

export async function GET() {
  const session = await verifySession()

  // ユーザーが認証されていない場合、401を返し unauthorized.tsx を表示します
  if (!session) {
    unauthorized()
  }

  // データ取得
  // ...
}
```

</TabItem>
</Tabs>
