`unauthorized`関数は、Next.jsの401エラーページを描画するエラーをスローします；アプリケーション内の認証エラーを処理するのに便利です。UIをカスタマイズするには、[`unauthorized.js`ファイル](/docs/app/api-reference/file-conventions/unauthorized)を使用できます。

`unauthorized`を使用するには、`next.config.js`ファイル内で実験的な[`authInterrupts`](/docs/app/api-reference/config/next-config-js/authInterrupts)構成オプションを有効にします：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    authInterrupts: true,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
module.exports = {
  experimental: {
    authInterrupts: true,
  },
}
```

</TabItem>
</Tabs>

`unauthorized`は[Server Component](/docs/app/building-your-application/rendering/server-components)、[Server Actions](/docs/app/building-your-application/data-fetching/server-actions-and-mutations)、および[Route Handlers](/docs/app/building-your-application/routing/route-handlers)で呼び出すことができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/dashboard/page.tsx" switcher
import { verifySession } from '@/app/lib/dal'
import { unauthorized } from 'next/navigation'

export default async function DashboardPage() {
  const session = await verifySession()

  if (!session) {
    unauthorized()
  }

  // 認証されたユーザー向けにダッシュボードをレンダリングする
  return (
    <main>
      <h1>Welcome to the Dashboard</h1>
      <p>Hi, {session.user.name}.</p>
    </main>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/dashboard/page.js" switcher

    <main>
      <h1>Welcome to the Dashboard</h1>
      <p>Hi, {session.user.name}.</p>
    </main>
  )
}
```

</TabItem>
</Tabs>

## Good to know {#good-to-know}

- `unauthorized`関数は、[root レイアウト](/docs/app/building-your-application/routing/layouts-and-templates#root-layout-required)で呼び出すことはできません。

### Server Actionsでのミューテーション {#mutations-with-server-actions}

特定のミューテーションを実行できるのは認証されたユーザーのみであることを確認するために、Server Actionsで`unauthorized`を呼び出すことができます。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions/update-profile.ts" switcher
'use server'

import { verifySession } from '@/app/lib/dal'
import { unauthorized } from 'next/navigation'
import db from '@/app/lib/db'

export async function updateProfile(data: FormData) {
  const session = await verifySession()

  // ユーザーが認証されていない場合は401を返す
  if (!session) {
    unauthorized()
  }

  // ミューテーションを続行
  // ...
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/actions/update-profile.js" switcher

  // ...
}
```

</TabItem>

### Route Handlersでのデータフェッチ {#fetching-data-with-route-handlers}

Route Handlersで`unauthorized`を使用して、エンドポイントにアクセスできるのは認証されたユーザーのみであることを保証できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/api/profile/route.ts" switcher
import { NextRequest, NextResponse } from 'next/server'
import { verifySession } from '@/app/lib/dal'
import { unauthorized } from 'next/navigation'

export async function GET(req: NextRequest): Promise<NextResponse> {
  // ユーザーのセッションを確認する
  const session = await verifySession()

  // セッションが存在しない場合は401を返しunauthorized.tsxを表示する
  if (!session) {
    unauthorized()
  }

  // データをフェッチ
  // ...
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/api/profile/route.js" switcher
import { verifySession } from '@/app/lib/dal'

  const session = await verifySession()

  }

  // データをフェッチ
  // ...
}
```

</TabItem>
