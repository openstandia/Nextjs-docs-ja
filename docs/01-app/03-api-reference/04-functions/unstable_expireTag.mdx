`unstable_expireTag` は、特定のキャッシュタグのために [キャッシュされたデータ](/docs/app/building-your-application/caching) をオンデマンドで削除することを可能にします。

> **Good to know**:
>
> - `unstable_expireTag` は、[Node.jsとEdgeランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes) の両方で使用できます。
> - `unstable_expireTag` は、キャッシュが次にそのパスが訪問されたときにのみ無効にします。これは、`unstable_expireTag`を動的ルートセグメントで呼び出しても、多くの無効化を即座にトリガーしないことを意味します。無効化は次にパスが訪問されたときにのみ発生します。

## 参照 {#reference}

### パラメーター {#parameters}

```tsx
unstable_expireTag(...tags: string[]): void;
```

- `tags`: 再検証したいデータに関連付けられたキャッシュタグを表す文字列の引数。各タグは256文字以内である必要があります。この値は大文字と小文字を区別します。

`fetch`にタグを次のように追加できます：

```tsx
fetch(url, { next: { tags: [...] } });
```

### 戻り値 {#returns}

`unstable_expireTag` は値を返しません。

## 例 {#examples}

### Server Action {#server-action}

`unstable_expireTag` をServer Actionで呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions.ts" switcher
'use server'

import { unstable_expireTag } from 'next/cache'

export default async function submit() {
  await addPost()
  unstable_expireTag('posts', 'blog')
}
```

### Route Handler {#route-handler}

`unstable_expireTag` をRoute Handlerで呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { unstable_expireTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')
  unstable_expireTag(tag)
  return Response.json({ revalidated: true, now: Date.now() })
}
```

</TabItem>
