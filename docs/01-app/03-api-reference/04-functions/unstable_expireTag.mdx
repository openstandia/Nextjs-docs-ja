`unstable_expireTag` は、特定のキャッシュタグの[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドでパージすることができます。

> **Good to know**:
>
> - `unstable_expireTag` は[Node.js および Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `unstable_expireTag` は、次回そのパスが訪問されたときにキャッシュを無効にします。つまり、`unstable_expireTag` を dynamic route segment で呼び出しても、一度に多くの無効化を即座に引き起こすことはありません。無効化は、次にそのパスが訪問されたときにのみ発生します。

## リファレンス {#reference}

### パラメーター {#parameters}

```tsx
unstable_expireTag(...tags: string[]): void;
```

- `tags`: 再検証したいデータに関連するキャッシュタグを表す文字列引数。各文字列は256文字以下である必要があります。この値は大文字小文字を区別します。

`fetch` にタグを追加する方法は次のとおりです:

```tsx
fetch(url, { next: { tags: [...] } });
```

### 戻り値 {#returns}

`unstable_expireTag` は値を返しません。

## 例 {#examples}

### Server Action {#server-action}

`unstable_expireTag` を Server Action で呼び出すことができます:

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions.ts" switcher
'use server'

import { unstable_expireTag } from 'next/cache'

export default async function submit() {
  await addPost()
  unstable_expireTag('posts', 'blog')
}
```

### Route Handler {#route-handler}

`unstable_expireTag` を Route Handler で呼び出すことができます:

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { unstable_expireTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')
  unstable_expireTag(tag)
  return Response.json({ revalidated: true, now: Date.now() })
}
```

</TabItem>
