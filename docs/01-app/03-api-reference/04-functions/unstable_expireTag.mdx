`unstable_expireTag` は、特定のキャッシュタグに対して[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドで消去することができます。

> **Good to know**:
>
> - `unstable_expireTag` は、[Node.js および エッジランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `unstable_expireTag` は、パスが次に訪問されるときにのみキャッシュを無効にします。つまり、動的ルートセグメントで `unstable_expireTag` を呼び出しても、一度に多くの無効化を即座にトリガーすることはありません。無効化は、パスが次に訪問されたときにのみ発生します。

## リファレンス {#reference}

### パラメータ {#parameters}

```tsx
unstable_expireTag(...tags: string[]): void;
```

- `tags`: 無効化したいデータに関連付けられたキャッシュタグを表す文字列引数です。それぞれの長さは256文字以内でなければなりません。この値は大文字と小文字を区別します。

`fetch`にタグを次のように追加できます：

```tsx
fetch(url, { next: { tags: [...] } });
```

### 戻り値 {#returns}

`unstable_expireTag` は値を返しません。

## 例 {#examples}

### サーバーアクション {#server-action}

Server Actionで `unstable_expireTag` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions.ts" switcher
'use server'

import { unstable_expireTag } from 'next/cache'

export default async function submit() {
  await addPost()
  unstable_expireTag('posts', 'blog')
}
```

### Route Handler {#route-handler}

Route Handlerで `unstable_expireTag` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { unstable_expireTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')
  unstable_expireTag(tag)
  return Response.json({ revalidated: true, now: Date.now() })
}
```

</TabItem>
