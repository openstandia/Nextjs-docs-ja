---
title: 'revalidatePath'
description: 'revalidatePath関数のAPIリファレンス'
---

`revalidatePath`を使用すると、特定のパスの[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドで削除できます。

> **Good to know**:
>
> - `revalidatePath`は、[Node.jsとEdgeランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `revalidatePath`は、指定されたパスが次にアクセスされたときにキャッシュを無効にします。つまり、`revalidatePath`をdynamic route segmentで呼び出しても、一度に多数の再検証を即座に引き起こすことはありません。無効化は次にパスが訪問されたときにのみ発生します。
> - 現在、サーバーアクションで使用されると、`revalidatePath`は[クライアント側のRouter Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートを無効化します。この動作は一時的なものであり、将来的には特定のパスにのみ適用されるように更新されます。
> - `revalidatePath`を使用すると、[サーバー側のRoute Cache](/docs/app/building-your-application/caching#full-route-cache)では**特定のパスのみ**が無効化されます。

## パラメータ {#parameters}

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 再検証したいデータに関連付けられたファイルシステムパスを表す文字列（例：`/product/[slug]/page`）またはリテラルのルートセグメント（例：`/product/123`）のいずれか。1024文字未満に収める必要があります。この値は大文字と小文字が区別されます。
- `type`: （オプション）再検証するパスのタイプを変更するための`'page'`または`'layout'`文字列。パスがdynamic segmentを含む場合（例：`/product/[slug]/page`）、このパラメータが必要です。パスがリテラルのルートセグメントを指している場合（例：`/product/1`）、typeを指定しないでください。

## 戻り値 {#returns}

`revalidatePath`は値を返しません。

## 例 {#examples}

### 特定のURLを再検証する {#revalidating-a-specific-url}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

これは、次のページ訪問時に特定のURLを再検証します。

### ページパスを再検証する {#revalidating-a-page-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// またはroute groupsと一緒に
revalidatePath('/(main)/blog/[slug]', 'page')
```

これは、指定された`page`ファイルと一致する任意のURLを次のページ訪問時に再検証します。これにより、特定のページ以下のページは無効化されません。たとえば、`/blog/[slug]`は`/blog/[slug]/[author]`を無効化しません。

### レイアウトパスを再検証する {#revalidating-a-layout-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// またはroute groupsと一緒に
revalidatePath('/(main)/post/[slug]', 'layout')
```

これは、指定された`layout`ファイルと一致する任意のURLを次のページ訪問時に再検証します。これにより、同じレイアウトを持つ下位のページが次の訪問時に再検証されます。たとえば、上記の例では`/blog/[slug]/[another]`も次の訪問時に再検証されます。

### すべてのデータを再検証する {#revalidating-all-data}

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

これにより、クライアント側のRouter Cacheが消去され、次のページ訪問時にデータキャッシュが再検証されます。

### ルートハンドラー {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
