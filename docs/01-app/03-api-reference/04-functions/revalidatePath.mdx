title: 'revalidatePath'
description: 'revalidatePath関数のAPIリファレンス'

`revalidatePath` は、特定のパスに対して[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドで削除することを可能にします。

> **Good to know**:
>
> - `revalidatePath` は [Node.jsとEdgeランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `revalidatePath` は、キャッシュを無効化するのは該当のパスが次に訪問されたときのみです。つまり、`revalidatePath`をdynamic route segmentで呼び出しても、一度に多くの再検証を即座にトリガーすることはありません。無効化は、そのパスが次に訪問されたときにのみ発生します。
> - 現在、`revalidatePath`はサーバーアクションで使用された場合、[クライアントサイドRouter Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートを無効化します。この動作は一時的なものであり、今後は特定のパスにのみ適用されるように更新される予定です。
> - `revalidatePath`の使用によって、[サーバーサイドRoute Cache](/docs/app/building-your-application/caching#full-route-cache)内の**特定のパスのみ**が無効化されます。

## パラメータ {#parameters}

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 再検証したいデータに関連するファイルシステムパス（例えば、`/product/[slug]/page`）、または文字通りのルートセグメント（例えば、`/product/123`）を表す文字列。1024文字未満でなければなりません。この値は大文字小文字を区別します。
- `type`: （オプション）`'page'`または`'layout'`文字列で、再検証するパスのタイプを変更します。`path`にdynamic segment（たとえば、`/product/[slug]/page`）が含まれている場合、このパラメータが必要です。パスが文字通りのルートセグメント、例えばdynamic page（例えば、`/product/[slug]/page`）の`/product/1`を指す場合は、`type`を指定する必要はありません。

## 返り値 {#returns}

`revalidatePath`は値を返しません。

## 例 {#examples}

### 特定のURLの再検証 {#revalidating-a-specific-url}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

これは、次回のページ訪問時に特定のURLを再検証します。

### ページパスの再検証 {#revalidating-a-page-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// あるいはルートグループを使って
revalidatePath('/(main)/blog/[slug]', 'page')
```

これは、次回のページ訪問時に提供された`page`ファイルに一致する任意のURLを再検証します。これにより、特定のページの下のページは無効化されません。たとえば、`/blog/[slug]`は`/blog/[slug]/[author]`を無効化しません。

### レイアウトパスの再検証 {#revalidating-a-layout-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// あるいはルートグループを使って
revalidatePath('/(main)/post/[slug]', 'layout')
```

これは、次回のページ訪問時に提供された`layout`ファイルに一致する任意のURLを再検証します。これにより、同じレイアウトのページは次回訪問時に再検証されます。たとえば、上記の場合、`/blog/[slug]/[another]`も次回訪問時に再検証されます。

### すべてのデータの再検証 {#revalidating-all-data}

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

これは、クライアントサイドRouter Cacheをパージし、次回のページ訪問時にData Cacheを再検証します。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
