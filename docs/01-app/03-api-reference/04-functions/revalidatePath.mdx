---
title: 'revalidatePath'
description: 'revalidatePath 関数のAPIリファレンス'
---

`revalidatePath` は、特定のパスに対して [キャッシュされたデータ](/docs/app/building-your-application/caching) をオンデマンドでパージすることを可能にします。

> **Good to know**:
>
> - `revalidatePath` は、[Node.jsとEdgeランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です；
> - `revalidatePath` は、そのパスが次に訪問されたときにのみキャッシュを無効にします；つまり、`revalidatePath` を dynamic route segment で呼び出しても、即座に多数の再検証がトリガーされるわけではありません；無効化はそのパスが次に訪問されたときにのみ行われます；
> - 現在、`revalidatePath` は、server actions で使用された場合、クライアントサイドの [Router Cache](/docs/app/building-your-application/caching#client-side-router-cache) にあるすべてのルートを無効にします；この動作は一時的なもので、将来的には特定のパスのみに適用されるよう更新されます；
> - `revalidatePath` を使用すると、サーバーサイドの [Route Cache](/docs/app/building-your-application/caching#full-route-cache) で**特定のパスのみ**を無効にします；

## パラメータ {#parameters}

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 再検証したいデータに関連付けられたファイルシステムパスを表す文字列（例: `/product/[slug]/page`）またはリテラルの route segment（例: `/product/123`）のいずれかです；1024文字未満でなければなりません；この値は大文字と小文字を区別します；
- `type`:（オプション）再検証するパスの種類を変更するための `'page'` または `'layout'` の文字列；`path` が dynamic segment を含む場合（例: `/product/[slug]/page`）、このパラメータが必要です；リテラルの route segment を指す場合（例: dynamic page の `/product/[slug]/page` に対して `/product/1` の場合）、`type` を提供しないでください；

## 戻り値 {#returns}

`revalidatePath` は値を返しません；

## 例 {#examples}

### 特定のURLの再検証 {#revalidating-a-specific-url}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

これは、次のページ訪問で特定のURLを再検証します；

### ページパスの再検証 {#revalidating-a-page-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// もしくは、route groupを利用
revalidatePath('/(main)/blog/[slug]', 'page')
```

これは、提供された `page` ファイルに一致するURLを次のページ訪問時に再検証します；これは、特定のページの下のページを無効に**しません**；たとえば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効にしません；

### レイアウトパスの再検証 {#revalidating-a-layout-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// もしくは、route groupを利用
revalidatePath('/(main)/post/[slug]', 'layout')
```

これは、提供された `layout` ファイルに一致するURLを次のページ訪問時に再検証します；これは、同じレイアウトを持つ下のページを次回訪問時に再検証します；たとえば、上記の場合、`/blog/[slug]/[another]` も次回訪問時に再検証されます；

### すべてのデータの再検証 {#revalidating-all-data}

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

これにより、クライアントサイドの Router Cache がパージされ、次のページ訪問時にデータキャッシュが再検証されます；

### ルートハンドラー {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
