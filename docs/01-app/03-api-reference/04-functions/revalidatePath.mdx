`revalidatePath` は、特定のパスに対してオンデマンドで[キャッシュされたデータ](/docs/app/building-your-application/caching)を消去することを可能にします。

> **Good to know**:
>
> - `revalidatePath` は、[Node.js および Edge runtime](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `revalidatePath` は、指定されたパスが次に訪れられたときにのみキャッシュを無効にします。したがって、`revalidatePath` に dynamic route segment を使っても、すぐに多くの再検証が同時にトリガーされるわけではありません。無効化は、そのパスが次に訪れられたときにのみ発生します。
> - 現在、`revalidatePath` は、server action で使用されるときに[クライアントサイド Router Cache](/docs/app/building-your-application/caching#client-side-router-cache)のすべてのルートを無効にします。この動作は一時的なものであり、将来的には特定のパスにのみ適用されるように更新される予定です。
> - `revalidatePath` を使用すると、[サーバーサイド Route Cache](/docs/app/building-your-application/caching#full-route-cache)内の**特定のパスのみ**が無効化されます。

## パラメーター {#parameters}

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 再検証したいデータに関連するファイルシステムパス（例えば、`/product/[slug]/page`）またはリテラルなルートセグメント（例えば、`/product/123`）を表す文字列です。1024文字未満である必要があります。この値は大文字と小文字を区別します。
- `type`: （オプション）再検証するパスのタイプを変更するための `'page'` または `'layout'` 文字列です。`path` に dynamic segment が含まれる場合（例えば、`/product/[slug]/page`）、このパラメーターは必須です。path がリテラルなルートセグメントを指す場合、例えば dynamic page のための `/product/1` の場合、`type` を提供しないでください。

## 戻り値 {#returns}

`revalidatePath` は値を返しません。

## 例 {#examples}

### 特定のURLを再検証する {#revalidating-a-specific-url}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

これは、次のページ訪問時に特定の URL を再検証します。

### ページパスを再検証する {#revalidating-a-page-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// またはルートグループを使用して
revalidatePath('/(main)/blog/[slug]', 'page')
```

これは、次のページ訪問時に指定された `page` ファイルに一致する URL を再検証します。これは、特定のページの下にあるページを無効化するものではありません。例えば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効化しません。

### レイアウトパスを再検証する {#revalidating-a-layout-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// またはルートグループを使用して
revalidatePath('/(main)/post/[slug]', 'layout')
```

これは、次のページ訪問時に指定された `layout` ファイルに一致する URL を再検証します。これは、同じレイアウトを持つ下のページを次の訪問時に再検証させます。例えば、上記の例では、`/blog/[slug]/[another]` も次の訪問時に再検証されます。

### すべてのデータを再検証する {#revalidating-all-data}

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

これは、クライアントサイド Router Cache をパージし、次のページ訪問時に Data Cache を再検証します。

### Route Handler {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
