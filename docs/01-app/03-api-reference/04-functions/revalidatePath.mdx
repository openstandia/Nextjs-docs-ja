---
title: 'revalidatePath'
description: 'revalidatePath関数のAPIリファレンス'
---

`revalidatePath`は、特定のパスに対して[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドでクリアすることができます。

> **Good to know**:
>
> - `revalidatePath`は、[Node.jsとEdgeのランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `revalidatePath`は、指定されたパスが次に訪問されたときにのみキャッシュを無効化します。これは、`revalidatePath`をdynamic route segmentで呼び出しても、すぐに多くの再検証がトリガーされるわけではないことを意味します。無効化はパスが次に訪問されたときにのみ発生します。
> - 現在、`revalidatePath`は、server action内で使用すると[クライアント側のRouter Cache](/docs/app/building-your-application/caching#client-side-router-cache)のすべてのルートを無効化します。この動作は一時的なもので、将来的には特定のパスのみに適用されるように更新される予定です。
> - `revalidatePath`を使用すると、[サーバー側のRoute Cache](/docs/app/building-your-application/caching#full-route-cache)の**特定のパスのみ**を無効化します。

## パラメータ {#parameters}

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 再検証したいデータに関連するファイルシステムパス（例：`/product/[slug]/page`）を表す文字列、またはリテラルルートセグメント（例：`/product/123`）。1024文字未満である必要があります。この値は大文字・小文字を区別します。
- `type`: （オプション）再検証するパスのタイプを変更するための、`'page'`または`'layout'`の文字列。`path`がdynamic segmentを含む場合（例：`/product/[slug]/page`）、このパラメータが必要です。pathがリテラルルートセグメントを指す場合（例：dynamic pageである`/product/1`）、`type`を指定する必要はありません。

## 戻り値 {#returns}

`revalidatePath`は値を返しません。

## 例 {#examples}

### 特定のURLの再検証 {#revalidating-a-specific-url}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

これにより、次のページ訪問時に特定のURLが再検証されます。

### ページパスの再検証 {#revalidating-a-page-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// またはルートグループを使用して
revalidatePath('/(main)/blog/[slug]', 'page')
```

これにより、次のページ訪問時に指定された`page`ファイルに一致する任意のURLが再検証されます。これは、特定のページの下にあるページを無効にすることはありません。たとえば、`/blog/[slug]`は`/blog/[slug]/[author]`を無効にしません。

### レイアウトパスの再検証 {#revalidating-a-layout-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// またはルートグループを使用して
revalidatePath('/(main)/post/[slug]', 'layout')
```

これにより、次のページ訪問時に指定された`layout`ファイルに一致する任意のURLが再検証されます。これにより、同じレイアウトを持つ下位のページは次回訪問時に再検証されます。たとえば、上記のケースでは`/blog/[slug]/[another]`も次回訪問時に再検証されます。

### すべてのデータの再検証 {#revalidating-all-data}

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

これにより、クライアント側のRouter Cacheがクリアされ、次のページ訪問時にData Cacheが再検証されます。

### ルートハンドラー {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
