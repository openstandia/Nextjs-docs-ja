title: 'revalidatePath'
description: 'revalidatePath 関数のAPIリファレンス。'
---

`revalidatePath` を使用すると、特定のパスに対して[キャッシュされたデータ](/docs/app/building-your-application/caching)を随時パージできます。

> **Good to know**:
>
> - `revalidatePath` は、[Node.js および Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes) で利用できます。
> - `revalidatePath` は、含まれているパスが次に訪問されたときにのみキャッシュの無効化を行います。これは、`revalidatePath` を dynamic route segment とともに呼び出しても即座に多くの再検証が行われないことを意味します。無効化は次にパスが訪問されたときにのみ発生します。
> - 現在、`revalidatePath` は、server action で使用される際には [クライアントサイドの Router Cache](/docs/app/building-your-application/caching#client-side-router-cache) 内のすべてのルートを無効にします。この動作は一時的なものであり、将来的には特定のパスのみに適用されるように更新されます。
> - `revalidatePath` を使用すると、[サーバーサイドの Route Cache](/docs/app/building-your-application/caching#full-route-cache) 内で **特定のパスのみ** が無効化されます。

## パラメーター {#parameters}

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 再検証したいデータに関連付けられたファイルシステムのパスを表す文字列（例：`/product/[slug]/page`）、またはリテラルのルートセグメント（例：`/product/123`）を指定します。1024 文字未満である必要があります。この値は大文字と小文字を区別します。
- `type`:（オプション）再検証するパスのタイプを変更するための `'page'` または `'layout'` 文字列。このパラメーターは、`path` が dynamic segment を含む場合に必要です（例：`/product/[slug]/page`）。path がリテラルのルートセグメントを指している場合（例：`/product/1`、dynamic page の場合は `/product/[slug]/page`）、`type` を指定しないでください。

## 戻り値 {#returns}

`revalidatePath` は値を返しません。

## 例 {#examples}

### 特定のURLを再検証する {#revalidating-a-specific-url}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

これにより、次のページ訪問時に特定のURLが再検証されます。

### ページパスを再検証する {#revalidating-a-page-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// または、ルートグループとともに
revalidatePath('/(main)/blog/[slug]', 'page')
```

これにより、次のページ訪問時に提供された `page` ファイルに一致する任意のURLが再検証されます。ただし、特定のページ以下のページは無効化されません。例えば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効化しません。

### レイアウトパスを再検証する {#revalidating-a-layout-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// または、ルートグループとともに
revalidatePath('/(main)/post/[slug]', 'layout')
```

これにより、次のページ訪問時に提供された `layout` ファイルに一致する任意のURLが再検証されます。同じレイアウトを持つ下位のページが次回訪問時に再検証されるようになります。例えば、上記のケースでは、`/blog/[slug]/[another]` も次回訪問時に再検証されます。

### すべてのデータを再検証する {#revalidating-all-data}

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

これにより、クライアントサイドの Router Cache がパージされ、次のページ訪問時に Data Cache が再検証されます。

### Route Handler {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: '再検証するパスがありません',
  })
}
```
