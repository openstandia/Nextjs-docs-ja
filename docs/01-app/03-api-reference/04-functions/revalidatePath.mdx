title: 'revalidatePath'
description: 'revalidatePath 関数の API リファレンス'
---

`revalidatePath` を使用すると、特定のパスに対してオンデマンドで[キャッシュされたデータ](/docs/app/building-your-application/caching)を削除できます。

> **Good to know**:
>
> - `revalidatePath` は、[Node.js および Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で使用可能です。
> - `revalidatePath` は、指定したパスが次にアクセスされるときにのみキャッシュを無効化します。つまり、`revalidatePath` を動的ルートセグメントと共に呼び出しても、多くの再検証が一度に即座にトリガーされることはないということです。無効化は、パスが次に訪問されたときにのみ発生します。
> - 現在、`revalidatePath` はサーバーアクションで使用した場合、[クライアントサイド Router Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートを無効化します。この動作は一時的で、今後は特定のパスにのみ適用されるように変更される予定です。
> - `revalidatePath` を使用することで、[サーバーサイド Route Cache](/docs/app/building-your-application/caching#full-route-cache)において**特定のパスのみ**を無効化します。

## Parameters {#parameters}

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: リバリデートしたいデータに関連するファイルシステムパスを表す文字列（例：`/product/[slug]/page`）またはリテラルルートセグメント（例：`/product/123`）。1024文字未満である必要があります。この値は大文字小文字を区別します。
- `type`: （オプション）`'page'` または `'layout'` 文字列を指定して、リバリデートするパスのタイプを変更します。`path` に動的セグメントが含まれる場合（例：`/product/[slug]/page`）、このパラメータは必須です。パスがリテラルルートセグメントを指す場合（例：動的ページ `/product/[slug]/page`に対する `/product/1`)、`type` を指定すべきではありません。

## Returns {#returns}

`revalidatePath` は値を返しません。

## Examples {#examples}

### 特定のURLの再検証 {#revalidating-a-specific-url}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

これは、次ページ訪問時に特定のURLを再検証します。

### ページパスの再検証 {#revalidating-a-page-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// ルートグループを使用する場合
revalidatePath('/(main)/blog/[slug]', 'page')
```

これは、次ページ訪問時に指定された`page`ファイルに一致する任意のURLを再検証します。これは特定のページ以下のページを無効化しません。例えば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効化しません。

### レイアウトパスの再検証 {#revalidating-a-layout-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// ルートグループを使用する場合
revalidatePath('/(main)/post/[slug]', 'layout')
```

これは、次ページ訪問時に指定された`layout`ファイルに一致する任意のURLを再検証します。これにより、同じレイアウトを使用している下位ページは次の訪問時に再検証されます。たとえば、上記の場合、`/blog/[slug]/[another]` も次回訪問時に再検証されます。

### すべてのデータの再検証 {#revalidating-all-data}

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

これは、クライアントサイドの Router Cache をクリーンアップし、次ページ訪問時に Data Cache を再検証します。

### ルートハンドラー {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
