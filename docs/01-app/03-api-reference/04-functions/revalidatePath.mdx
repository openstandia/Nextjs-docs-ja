title: 'revalidatePath'

---

`revalidatePath` は、特定のパスの[キャッシュ済みデータ](/docs/app/building-your-application/caching)をオンデマンドでクリアすることができます。

> **Good to know**:
>
> - `revalidatePath` は [Node.js および Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `revalidatePath` は、指定したパスが次回訪問されたときにのみキャッシュを無効にします。つまり、`revalidatePath`を dynamic route segment で呼び出しても、一度に多くの再検証を即座に引き起こすわけではありません。無効化は、パスが次回訪問されたときにのみ発生します。
> - 現在、server actions で使用された場合、`revalidatePath` は [client-side Router Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートを無効にします。この動作は一時的なものであり、特定のパスにのみ適用されるように今後変更されます。
> - `revalidatePath` を使用すると、[server-side Route Cache](/docs/app/building-your-application/caching#full-route-cache)内の**特定のパスのみ**が無効化されます。

## Parameters {#parameters}

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 再検証したいデータに関連付けられたファイルシステムパスを表す文字列（例：`/product/[slug]/page`）または、文字通りの route segment（例：`/product/123`）。1024文字未満である必要があります。この値は大文字と小文字を区別します。
- `type`: （オプション）再検証するパスの種類を変更するための `'page'` または `'layout'` 文字列。`path`が dynamic segment を含む場合（例：`/product/[slug]/page`）、このパラメーターは必須です。パスが文字通りの route segment を指している場合、例：dynamic page の場合は `/product/1`（例：`/product/[slug]/page`）、`type` を指定するべきではありません。

## Returns {#returns}

`revalidatePath` は値を返しません。

## Examples {#examples}

### 特定の URL の再検証 {#revalidating-a-specific-url}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

これは、次回ページ訪問時に特定の URL を再検証します。

### ページパスの再検証 {#revalidating-a-page-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// または route groups の場合
revalidatePath('/(main)/blog/[slug]', 'page')
```

これは、次回ページ訪問時に指定された `page` ファイルに一致する任意の URL を再検証します。これは、特定のページの下位ページを無効にするわけではありません。たとえば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効にしません。

### レイアウトパスの再検証 {#revalidating-a-layout-path}

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// または route groups の場合
revalidatePath('/(main)/post/[slug]', 'layout')
```

これは、次回ページ訪問時に指定された `layout` ファイルに一致する任意の URL を再検証します。これは、次回訪問時に同じレイアウトを持つ下位のページも再検証されます。たとえば、上記の場合、`/blog/[slug]/[another]` も次回訪問時に再検証されます。

### すべてのデータの再検証 {#revalidating-all-data}

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

これにより、Client-side Router Cache がクリアされ、次回ページ訪問時に Data Cache が再検証されます。

### Route Handler {#route-handler}

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
