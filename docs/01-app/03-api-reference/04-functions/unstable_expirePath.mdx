`unstable_expirePath` を使用すると、指定されたパスの[キャッシュデータ](/docs/app/building-your-application/caching)をオンデマンドでパージできます。

> **Good to know**:
>
> - `unstable_expirePath` は、[Node.js および Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で使用可能です。
> - `unstable_expirePath` は、含まれるパスが次に訪問されたときにキャッシュを無効にするだけです。これは、`unstable_expirePath` をdynamic route segmentで呼び出しても、一度に多くの無効化を即座にトリガーしないことを意味します。無効化は、パスが次に訪問されたときにのみ行われます。
> - 現在、`unstable_expirePath` をserver actionで使用すると[クライアントサイドrouter cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートが無効化されます。この動作は一時的であり、今後特定のパスにのみ適用されるよう更新される予定です。
> - `unstable_expirePath` を使用すると、[サーバーサイドroute cache](/docs/app/building-your-application/caching#full-route-cache)内の**特定のパスのみが**無効化されます。

## 参考資料 {#reference}

### パラメータ {#parameters}

```tsx
unstable_expirePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 期限切れにしたいデータに関連付けられたファイルシステムパスを表す文字列（例：`/product/[slug]/page`）、またはリテラルなroute segment（例：`/product/123`）。1024文字未満である必要があります。この値は大文字小文字を区別します。
- `type`:（オプション）`'page'`または`'layout'`文字列で、期限切れにするパスのタイプを変更します。もし`path`がdynamic route segmentを含んでいる場合（例：`/product/[slug]/page`）、このパラメータは必須です。`path`がリテラルなroute segmentを指す場合（例：dynamic pageに対する`/product/1`）、`type`を指定する必要はありません。

### 戻り値 {#returns}

`unstable_expirePath` は値を返しません。

## 例 {#examples}

### 特定のURLのキャッシュを無効にする {#expiring-a-specific-url}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/post-1')
```

次回ページアクセス時に、特定のURLのキャッシュをパージします。

### ページパスのキャッシュを無効にする {#expiring-a-page-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'page')
// または、route groupを使用して
unstable_expirePath('/(main)/blog/[slug]', 'page')
```

次回ページアクセス時に、指定された`page`ファイルに一致する任意のURLのキャッシュをパージします。これにより、特定のページの下にあるページは無効になりません。例えば、`/blog/[slug]`は`/blog/[slug]/[author]`を無効にしません。

### レイアウトパスのキャッシュを無効にする {#expiring-a-layout-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'layout')
// または、route groupを使用して
unstable_expirePath('/(main)/post/[slug]', 'layout')
```

次回ページアクセス時に、指定された`layout`ファイルに一致する任意のURLのキャッシュをパージします。これにより、同じレイアウトを持つ下位のページが次回の訪問時に再検証されます。例えば、上記の場合、`/blog/[slug]/[another]`も次回の訪問時に再検証されます。

### すべてのデータを無効にする {#expiring-all-data}

```ts
import { unstable_expirePath } from 'next/cache'

unstable_expirePath('/', 'layout')
```

次回ページアクセス時に、Data Cacheをパージします。

Server Action内で`unstable_expirePath`を呼び出すことができます：

### Route Handler {#route-handler}

Route Handler内で`unstable_expirePath`を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/expire/route.ts" switcher
import { unstable_expirePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    unstable_expirePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    expired: false,
    now: Date.now(),
    message: 'Missing path to expire',
  })
}
```

</TabItem>
