`unstable_expirePath` を使用すると、特定のパスに対する[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドで削除できます。

> **知っておくと良い情報**：
>
> - `unstable_expirePath` は [Node.js および Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes) の両方で利用可能です。
> - `unstable_expirePath` は含まれているパスが次に訪問された際にのみキャッシュを無効にします。つまり、`unstable_expirePath` を dynamic route segment で呼び出しても、多くの無効化が一度に即座にトリガーされることはありません。 無効化はそのパスが次に訪問されたときにのみ発生します。
> - 現在、`unstable_expirePath` は server action で使用するとき、[クライアント側 Router Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートを無効にします。この動作は一時的なものであり、将来的には特定のパスにのみ適用されるように更新されます。
> - `unstable_expirePath` の使用は、[サーバー側ルートキャッシュ](/docs/app/building-your-application/caching#full-route-cache)内の**特定のパスのみ**を無効にします。

## リファレンス {#reference}

### パラメータ {#parameters}

```tsx
unstable_expirePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 期限を設定したいデータに関連付けられたファイルシステムパス（例：`/product/[slug]/page`）またはリテラル route segment（例：`/product/123`）を表す文字列。1024文字未満である必要があります。この値は大文字と小文字を区別します。
- `type`: （オプション）`'page'` または `'layout'` の文字列で、期限を設定するパスのタイプを変更します。`path` に dynamic segment が含まれている場合（例：`/product/[slug]/page`）、このパラメータは必須です。パスがリテラル route segment を指している場合（例：dynamic page の `/product/[slug]/page` 用の `/product/1`）、`type` を提供しないでください。

### 戻り値 {#returns}

`unstable_expirePath` は値を返しません。

## 例 {#examples}

### 特定のURLを無効化する {#expiring-a-specific-url}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/post-1')
```

これは次のページ訪問時に、その特定のURLに対するキャッシュを削除します。

### ページパスを無効化する {#expiring-a-page-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'page')
// または  route group を使用する
unstable_expirePath('/(main)/blog/[slug]', 'page')
```

これは、次のページ訪問時に、指定された `page` ファイルと一致する任意のURLのキャッシュを削除します。これは、特定のページより下のページを無効にしません。たとえば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効にしません。

### レイアウトパスを無効化する {#expiring-a-layout-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'layout')
// または  route group を使用する
unstable_expirePath('/(main)/post/[slug]', 'layout')
```

これは、次のページ訪問時に、指定された `layout` ファイルと一致する任意のURLのキャッシュを削除します。ページ訪問時に、同じ layout を持つ下位ページが再検証されます。たとえば、上記の場合、`/blog/[slug]/[another]` も次の訪問で再検証されます。

### すべてのデータを無効化する {#expiring-all-data}

```ts
import { unstable_expirePath } from 'next/cache'

unstable_expirePath('/', 'layout')
```

これは、次のページ訪問時にデータキャッシュを削除します。

### サーバーアクション {#server-action}

Server Action 内で `unstable_expirePath` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions.ts" switcher
'use server'

import { unstable_expirePath } from 'next/cache'

export default async function submit() {
  await submitForm()
  unstable_expirePath('/')
}
```

</TabItem>
</Tabs>

### ルートハンドラー {#route-handler}

Route Handler 内で `unstable_expirePath` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/expire/route.ts" switcher
import { unstable_expirePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    unstable_expirePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    expired: false,
    now: Date.now(),
    message: 'Missing path to expire',
  })
}
```

</TabItem>
