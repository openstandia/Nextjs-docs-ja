`unstable_expirePath`は、特定のパスの[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドで消去することができます。

> **Good to know**:
>
> - `unstable_expirePath`は、[Node.jsとEdgeランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用可能です。
> - `unstable_expirePath`は、含まれているパスが次に訪問されるときにのみキャッシュを無効にします。これにより、`unstable_expirePath`を動的ルートセグメントと一緒に呼び出しても、多くの失効がすぐにトリガーされるわけではありません。無効化はパスが次に訪問されるときにのみ発生します。
> - 現在、`unstable_expirePath`をサーバーアクションで使用すると、[クライアント側のRouter Cache](/docs/app/building-your-application/caching#client-side-router-cache)内のすべてのルートが無効になります。この動作は一時的なもので、将来は特定のパスのみに適用されるように更新されます。
> - `unstable_expirePath`を使用すると、[サーバー側のRoute Cache](/docs/app/building-your-application/caching#full-route-cache)内の**特定のパスのみ**が無効化されます。

## リファレンス {#reference}

### パラメータ {#parameters}

```tsx
unstable_expirePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 消去したいデータに関連付けられたファイルシステムのパスを示す文字列（例: `/product/[slug]/page`）、または文字通りのルートセグメント（例: `/product/123`）。1024文字未満である必要があります。この値は大文字と小文字を区別します。
- `type`: （オプション）失効させるパスのタイプを `'page'` または `'layout'` として変更するための文字列。`path` に動的セグメントが含まれている場合（例: `/product/[slug]/page`）、このパラメーターは必須です。動的ページのために文字通りのルートセグメント（例: `/product/1`）を指している場合は、`type` を提供しないでください。

### 戻り値 {#returns}

`unstable_expirePath` は値を返しません。

## 例 {#examples}

### 特定のURLを失効させる {#expiring-a-specific-url}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/post-1')
```

これは、次のページ訪問時に特定のURLのキャッシュを消去します。

### ページパスを失効させる {#expiring-a-page-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'page')
// もしくはルートグループを使う場合
unstable_expirePath('/(main)/blog/[slug]', 'page')
```

これは、次のページ訪問時に提供された`page`ファイルに一致する任意のURLを消去します。これは、特定のページの下のページを無効にはしません。たとえば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効にはしません。

### レイアウトパスを失効させる {#expiring-a-layout-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'layout')
// もしくはルートグループを使う場合
unstable_expirePath('/(main)/post/[slug]', 'layout')
```

これは、次のページ訪問時に提供された`layout`ファイルに一致する任意のURLのキャッシュを消去します。これにより、同じレイアウトを持つ下位のページが次の訪問時に再検証されます。たとえば、上記の場合 `/blog/[slug]/[another]` も次の訪問時に再検証されます。

### すべてのデータを失効させる {#expiring-all-data}

```ts
import { unstable_expirePath } from 'next/cache'

unstable_expirePath('/', 'layout')
```

これは、次のページ訪問時にデータキャッシュを消去します。

### サーバーアクション {#server-action}

`unstable_expirePath` をサーバーアクションで呼び出すことができます：

### ルートハンドラー {#route-handler}

`unstable_expirePath` をルートハンドラーで呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/expire/route.ts" switcher
import { unstable_expirePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    unstable_expirePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    expired: false,
    now: Date.now(),
    message: 'Missing path to expire',
  })
}
```

</TabItem>
