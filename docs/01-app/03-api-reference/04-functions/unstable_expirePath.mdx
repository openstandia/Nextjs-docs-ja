`unstable_expirePath` を使用すると、特定のパスに対して[キャッシュされたデータ](/docs/app/building-your-application/caching)をオンデマンドで削除することができます。

> **Good to know**:
>
> - `unstable_expirePath` は [Node.js と Edge runtimes](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes) の両方で利用できます。
> - `unstable_expirePath` は、指定されたパスが次に訪問される際にのみキャッシュを無効化します。したがって、`unstable_expirePath` を動的な route segument で呼び出しても、すぐに多くの無効化がトリガーされるわけではありません。無効化は次にそのパスが訪問されたときにのみ発生します。
> - 現在、`unstable_expirePath` は server action で使用されると、[client-side Router Cache](/docs/app/building-your-application/caching#client-side-router-cache) 内のすべてのルートを無効にします。この動作は一時的なものであり、将来的には特定のパスにのみ適用されるように更新されます。
> - `unstable_expirePath` を使用すると、[server-side Route Cache](/docs/app/building-your-application/caching#full-route-cache) 内の**特定のパスのみ**を無効にします。

## リファレンス {#reference}

### パラメータ {#parameters}

```tsx
unstable_expirePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 無効にしたいデータに関連するファイルシステムのパスを表す文字列（例: `/product/[slug]/page`）、またはリテラルの route segument（例: `/product/123`）。1024文字未満である必要があります。この値は大文字と小文字を区別します。
- `type`: （オプション）無効にするパスのタイプを変更するための `'page'` または `'layout'` の文字列。`path` が動的なセグメントを含む場合（例: `/product/[slug]/page`）、このパラメータが必要です。path がリテラルの route segument を指す場合、たとえば動的ページの `/product/1`（例: `/product/[slug]/page`）、`type` を指定する必要はありません。

### 戻り値 {#returns}

`unstable_expirePath` は値を返しません。

## 例 {#examples}

### 特定のURLを無効化する {#expiring-a-specific-url}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/post-1')
```

これにより、次回のページ訪問時に特定のURLのキャッシュが消去されます。

### ページパスを無効化する {#expiring-a-page-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'page')
// または route group を使用
unstable_expirePath('/(main)/blog/[slug]', 'page')
```

これにより、次回のページ訪問時に指定された `page` ファイルと一致する任意のURLのキャッシュが消去されます。これは、指定されたページの下のページを無効化しません。たとえば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効化しません。

### レイアウトパスを無効化する {#expiring-a-layout-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'layout')
// または route group を使用
unstable_expirePath('/(main)/post/[slug]', 'layout')
```

これにより、次回のページ訪問時に指定された `layout` ファイルと一致する任意のURLのキャッシュが消去されます。これにより、同じレイアウトの下にあるページは次の訪問時に再検証されます。たとえば、上記の場合、`/blog/[slug]/[another]` も次回の訪問時に再検証されます。

### すべてのデータを無効化する {#expiring-all-data}

```ts
import { unstable_expirePath } from 'next/cache'

unstable_expirePath('/', 'layout')
```

これにより、次回のページ訪問時にデータキャッシュが消去されます。

### Server Action {#server-action}

Server Action 内で `unstable_expirePath` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions.ts" switcher
'use server'

import { unstable_expirePath } from 'next/cache'

export default async function submit() {
  await submitForm()
  unstable_expirePath('/')
}
```

</TabItem>
</Tabs>

### Route Handler {#route-handler}

Route Handler 内で `unstable_expirePath` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/expire/route.ts" switcher
import { unstable_expirePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    unstable_expirePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    expired: false,
    now: Date.now(),
    message: 'Missing path to expire',
  })
}
```

</TabItem>
