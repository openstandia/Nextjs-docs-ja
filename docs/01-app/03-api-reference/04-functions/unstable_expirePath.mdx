`unstable_expirePath` は、特定のパスに対して[キャッシュデータ](/docs/app/building-your-application/caching)をオンデマンドで削除することを可能にします。

> **Good to know**:
>
> - `unstable_expirePath`は、[Node.js と Edge ランタイム](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes)の両方で利用できます。
> - `unstable_expirePath`は、指定されたパスが次に訪問されたときにのみキャッシュを無効にします。つまり、`unstable_expirePath` を dynamic route segment とともに呼び出しても、すぐに多くの無効化がトリガーされるわけではありません。無効化は次にそのパスが訪問されたときにのみ発生します。
> - 現在、`unstable_expirePath` はserver actionで使用された場合、[クライアントサイドのRouter Cache](/docs/app/building-your-application/caching#client-side-router-cache)のすべてのルートを無効にします。この動作は一時的であり、将来的に特定のパスにのみ適用されるよう更新される予定です。
> - `unstable_expirePath` を使用すると、[サーバーサイドのルートキャッシュ](/docs/app/building-your-application/caching#full-route-cache)の**特定のパスのみ**が無効化されます。

## 参照 {#reference}

### パラメータ {#parameters}

```tsx
unstable_expirePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 期限切れにしたいデータに関連付けられたファイルシステムパスを表す文字列（例：`/product/[slug]/page`）、またはリテラルのルートセグメント（例：`/product/123`）のいずれか。この値は1024文字未満でなければなりません。この値は大文字小文字を区別します。
- `type`: （オプション）'page' または 'layout' の文字列で、期限切れにするパスのタイプを変更します。`path` にdynamic segment が含まれている場合（例：`/product/[slug]/page`）、このパラメータは必須です。パスがリテラルのルートセグメントを指す場合、たとえば動的ページ（例：`/product/[slug]/page`）での `/product/1`、`type` を指定する必要はありません。

### 戻り値 {#returns}

`unstable_expirePath` は値を返しません。

## 例 {#examples}

### 特定のURLの期限を切る {#expiring-a-specific-url}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/post-1')
```

これにより、次回のページ訪問時に特定のURLのキャッシュが消去されます。

### ページパスの期限を切る {#expiring-a-page-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'page')
// またはルートグループを使用
unstable_expirePath('/(main)/blog/[slug]', 'page')
```

これにより、次のページ訪問時に提供された`page`ファイルに一致する任意のURLのキャッシュが消去されます。これは、特定のページ以下のページを無効にすることはありません。たとえば、`/blog/[slug]` は `/blog/[slug]/[author]` を無効にしません。

### レイアウトパスの期限を切る {#expiring-a-layout-path}

```ts
import { unstable_expirePath } from 'next/cache'
unstable_expirePath('/blog/[slug]', 'layout')
// またはルートグループを使用
unstable_expirePath('/(main)/post/[slug]', 'layout')
```

これにより、次のページ訪問時に提供された`layout`ファイルに一致する任意のURLのキャッシュが消去されます。これにより、同じレイアウトを持つページは次の訪問時に再検証されます。たとえば、上記のケースでは、`/blog/[slug]/[another]` も次の訪問時に再検証されます。

### すべてのデータの期限を切る {#expiring-all-data}

```ts
import { unstable_expirePath } from 'next/cache'

unstable_expirePath('/', 'layout')
```

これにより、次のページ訪問時にData Cacheが消去されます。

### サーバーアクション {#server-action}

Server Action で `unstable_expirePath` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/actions.ts" switcher
'use server'

import { unstable_expirePath } from 'next/cache'

export default async function submit() {
  await submitForm()
  unstable_expirePath('/')
}
```

</TabItem>

### ルートハンドラー {#route-handler}

Route Handlerで `unstable_expirePath` を呼び出すことができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/api/expire/route.ts" switcher
import { unstable_expirePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    unstable_expirePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    expired: false,
    now: Date.now(),
    message: 'Missing path to expire',
  })
}
```

</TabItem>
