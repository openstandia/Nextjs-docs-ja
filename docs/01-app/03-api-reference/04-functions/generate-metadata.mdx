このページでは、`generateMetadata`と静的なメタデータオブジェクトを使用したすべての**設定ベースのメタデータ**オプションについて説明します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

// 静的メタデータ
export const metadata: Metadata = {
  title: '...',
}

// または動的メタデータ
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher
// 静的メタデータ
export const metadata = {
  title: '...',
}

// または動的メタデータ
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
</Tabs>

> **知っておくと良いこと**:
>
> - `metadata`オブジェクトと`generateMetadata`関数のエクスポートは、**Server Componentsでのみサポートされています**。
> - 同じルートセグメントから`metadata`オブジェクトと`generateMetadata`関数を両方エクスポートすることはできません。
> - 初期読み込み時には、`generateMetadata`が完全に解決されるまでストリーミングがブロックされます。これは、`loading.js`からのコンテンツを含みます。

## `metadata`オブジェクト {#the-metadata-object}

静的メタデータを定義するには、`layout.js`または`page.js`ファイルから[`Metadata`オブジェクト](#metadata-fields)をエクスポートします。

<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: '...',
  description: '...',
}

export default function Page() {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher
export const metadata = {
  title: '...',
  description: '...',
}

```

</TabItem>
</Tabs>

サポートされているオプションの完全なリストについては、[Metadata Fields](#metadata-fields)を参照してください。

## `generateMetadata`関数 {#generatemetadata-function}

動的メタデータは、現在のルートパラメータ、外部データ、または親セグメント内の`metadata`などの**動的な情報**に依存し、それは[`Metadata`オブジェクト](#metadata-fields)を返す`generateMetadata`関数をエクスポートすることで設定できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[id]/page.tsx" switcher
import type { Metadata, ResolvingMetadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  // ルートパラメータを読み込み
  const id = (await params).id

  // データを取得
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // 必要に応じて親メタデータをアクセスして拡張（置き換えるのではなく）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }: Props) {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[id]/page.js" switcher
export async function generateMetadata({ params, searchParams }, parent) {
  // ルートパラメータを読み込み
  const id = (await params).id

  // データを取得
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // 必要に応じて親メタデータをアクセスして拡張（置き換えるのではなく）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }) {}
```

</TabItem>
</Tabs>

### パラメータ {#parameters}

`generateMetadata`関数は次のパラメータを受け取ります：

- `props` - 現在のルートのパラメータを含むオブジェクト：

  - `params` - [動的ルートパラメーター](/docs/app/building-your-application/routing/dynamic-routes)オブジェクトを、rootセグメントから`generateMetadata`が呼ばれるセグメントまで含むオブジェクト。例：

    | Route                           | URL         | `params`                  |
    | ------------------------------- | ----------- | ------------------------- |
    | `app/shop/[slug]/page.js`       | `/shop/1`   | `{ slug: '1' }`           |
    | `app/shop/[tag]/[item]/page.js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |
    | `app/shop/[...slug]/page.js`    | `/shop/1/2` | `{ slug: ['1', '2'] }`    |

  - `searchParams` - 現在のURLの[検索パラメータ](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)を含むオブジェクト。例：

    | URL             | `searchParams`       |
    | --------------- | -------------------- |
    | `/shop?a=1`     | `{ a: '1' }`         |
    | `/shop?a=1&b=2` | `{ a: '1', b: '2' }` |
    | `/shop?a=1&a=2` | `{ a: ['1', '2'] }`  |

- `parent` - 親ルートセグメントから解決済みのメタデータのPromise。

### 戻り値 {#returns}

`generateMetadata`は、1つ以上のメタデータフィールドを含む[`Metadata`オブジェクト](#metadata-fields)を返すべきです。

> **知っておくと良いこと**：
>
> - メタデータが実行時情報に依存しない場合は、`generateMetadata`ではなく静的[`metadata`オブジェクト](#the-metadata-object)を使用して定義するべきです。
> - `fetch`リクエストは、同じデータに対する`generateMetadata`、`generateStaticParams`、レイアウト、ページ、およびServer Components間で自動的に[メモ化](/docs/app/building-your-application/caching#request-memoization)されます。`fetch`が利用できない場合、React [`cache`を使用することもでき](/docs/app/building-your-application/caching#react-cache-function)です。
> - `searchParams`は`page.js`セグメントでのみ利用可能です。
> - Next.jsの[`redirect()`](/docs/app/api-reference/functions/redirect)および[`notFound()`](/docs/app/api-reference/functions/not-found)メソッドは、`generateMetadata`内でも使用できます。

## メタデータフィールド {#metadata-fields}

### `title` {#title}

`title`属性は、ドキュメントのタイトルを設定するために使用されます。単純な[文字列](#string)またはオプションの[テンプレートオブジェクト](#template-object)として定義することができます。

#### 文字列 {#string}

```jsx title="layout.js | page.js"
export const metadata = {
  title: 'Next.js',
}
```

```html title="<head> output" hideLineNumbers

```

#### テンプレートオブジェクト {#template-object}

```tsx title="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    template: '...',
    default: '...',
    absolute: '...',
  },
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher

    default: '...',
    template: '...',
    absolute: '...',

```

</TabItem>
</Tabs>

##### デフォルト {#default}

`title.default`は`title`を定義しない子ルートセグメントに **フォールバックタイトル** を提供するために使用できます。

```tsx title="app/layout.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    default: 'Acme',
  },
}
```

```tsx title="app/about/page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {}

// Output: <title>Acme</title>
```

##### テンプレート {#template}

`title.template`は、**子**ルートセグメントで定義された`titles`にプレフィックスまたはサフィックスを追加するために使用できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher

export const metadata: Metadata = {
  title: {
    template: '%s | Acme',

  },
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher

  title: {
    template: '%s | Acme',
    default: 'Acme', // テンプレートを作成するには、デフォルトが必要です
  },

```

</TabItem>
</Tabs>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/about/page.tsx" switcher

  title: 'About',
}

// Output: <title>About | Acme</title>
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/about/page.js" switcher

  title: 'About',
}

// Output: <title>About | Acme</title>
```

</TabItem>

> **知っておくと良いこと**:
>
> - `title.template`は**子**ルートセグメントに適用され、定義されたセグメントには**適用されません**。これは以下の意味があります：
>
>   - `title.default`は、`title.template`を追加するときに**必須**です。
>   - `layout.js`に定義された`title.template`は、同じルートセグメントの`page.js`に定義された`title`には適用されません。
>   - `page.js`に定義された`title.template`は影響を及ぼしません。なぜならページは常に終端セグメント（それには子ルートセグメントがない）だからです。
>
> - ルートが`title`または`title.default`を定義していない場合、`title.template`は**影響を及ぼしません**。

`title.absolute`は親セグメントで設定された`title.template`を**無視**するタイトルを提供するために使うことができます。

```tsx title="app/layout.tsx" switcher
  },
```

> **知っておくと良いこと**:
>
> - `layout.js`
>
>   - `title`（文字列）および`title.default`は、（独自の`title`を定義しない）子セグメントのデフォルトタイトルを定義します。それは、もし存在すれば最も近い親セグメントからの`title.template`を補強します。
>   - `title.absolute`は子セグメントのデフォルトタイトルを定義します。それは親セグメントからの`title.template`を無視します。
>   - `title.template`は子セグメントのための新しいタイトルテンプレートを定義します。
>
> - `page.js`
>   - ページが独自のタイトルを定義していない場合、最も近い親で解決されたタイトルが使用されます。
>   - `title`（文字列）はルートタイトルを定義します。もし存在すれば最も近い親セグメントからの`title.template`を補強します。
>   - `title.absolute`はルートタイトルを定義します。それは親セグメントからの`title.template`を無視します。
>   - ページは常にルートの終端セグメントであるため、`page.js`内の`title.template`は影響を及ぼしません。

### `metadataBase` {#metadatabase}

`metadataBase`は完全修飾URLを必要とする`metadata`フィールドに基本URLのプレフィックスを設定するための便利なオプションです。

- `metadataBase`は**現在のルートセグメントおよびその下**で定義されたURLベースの`metadata`フィールドが、文字通りパス（相対パス）を使用できるようにします。
- フィールドの相対パスは、`metadataBase`と組み合わせられて完全修飾URLを形成します。
- 設定されていない場合、`metadataBase`は[デフォルト値](#default-value)で**自動的に設定**されます。

```jsx title="layout.js | page.js"
export const metadata = {
  metadataBase: new URL('https://acme.com'),
  alternates: {
    canonical: '/',
    languages: {
      'en-US': '/en-US',
      'de-DE': '/de-DE',
    },
  },
  openGraph: {
    images: '/og-image.png',
  },
}
```

```html title="<head> output" hideLineNumbers
<link rel="canonical" href="https://acme.com" />
<link rel="alternate" hreflang="en-US" href="https://acme.com/en-US" />
<link rel="alternate" hreflang="de-DE" href="https://acme.com/de-DE" />
<meta property="og:image" content="https://acme.com/og-image.png" />
```

> **知っておくと良いこと**:
>
> - `metadataBase`は、URLベースの`metadata`フィールドがすべてのルートにわたって適用されるようにrootの`app/layout.js`で設定されることが一般的です。
> - 完全なURLを必要とするすべてのURLベースの`metadata`フィールドは、`metadataBase`オプションで設定することができます。
> - `metadataBase`は、サブドメインを含んでいてもかまいません。例: `https://app.acme.com` または基本パスを含んでいてもかまいません。例: `https://acme.com/start/from/here`
> - もしフィールド自身が完全なURLを提供する場合、`metadataBase`は無視されます。
> - 相対パスをURLベースの`metadata`フィールドで使用し、`metadataBase`を設定しない場合、ビルド時にエラーが発生します。
> - Next.jsは`metadataBase`（例: `https://acme.com/`)とフィールドの相対パス（例: `/path`)の間の重複スラッシュを1つのスラッシュ（例: `https://acme.com/path`)に正規化します。

#### デフォルト値 {#default-value}

設定されていない場合、`metadataBase`には**デフォルト値**があります。

> Vercelでは：
>
> - 本番環境のデプロイメントの場合、`VERCEL_PROJECT_PRODUCTION_URL`が使用されます。
> - プレビュー環境のデプロイメントでは、`VERCEL_BRANCH_URL`が優先され、存在しない場合は`VERCEL_URL`にフォールバックします。
>
> これらの値が存在する場合、それらは`metadataBase`の**デフォルト値**として使用されます。そうでない場合、`http://localhost:${process.env.PORT || 3000}`にフォールバックします。これにより、ローカルビルドとVercelのプレビューおよび本番デプロイメントの両方でOpen Graph画像が機能します。デフォルトを上書きする場合は、URLを計算するために環境変数を使用することをお勧めします。これにより、ローカル開発、ステージング、そして本番環境でURLを構成することができます。
>
> これらの環境変数の詳細については、[システム環境変数のドキュメント](https://vercel.com/docs/concepts/projects/environment-variables/system-environment-variables)を参照してください。

## サポートされていないメタデータ {#unsupported-metadata}

以下のメタデータタイプは現在組み込みのサポートがありませんが、レイアウトやページ内で依然としてレンダリングが可能です。

| メタデータ                    | 推奨事項                                                                                                                                                                                                                                            |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<meta http-equiv="...">`     | HTTPヘッダを使用してください。[`redirect()`](/docs/app/api-reference/functions/redirect)、[ミドルウェア](/docs/app/building-your-application/routing/middleware#nextresponse)、[セキュリティヘッダ](/docs/app/api-reference/next-config-js/headers) |
| `<base>`                      | レイアウトやページ内でタグをレンダリングします。                                                                                                                                                                                                    |
| `<noscript>`                  | レイアウトやページ内でタグをレンダリングします。                                                                                                                                                                                                    |
| `<style>`                     | [Next.jsでのスタイリングについてさらに学ぶ](/docs/app/building-your-application/styling/css)                                                                                                                                                        |
| `<script>`                    | [スクリプトの使用法についてさらに学ぶ](/docs/app/building-your-application/optimizing/scripts)                                                                                                                                                      |
| `<link rel="stylesheet" />`   | レイアウトまたはページ内でスタイルシートを`インポート`します。                                                                                                                                                                                      |
| `<link rel="preload />`       | [ReactDOM preloadメソッド](#link-relpreload)を使用                                                                                                                                                                                                  |
| `<link rel="preconnect" />`   | [ReactDOM preconnectメソッド](#link-relpreconnect)を使用                                                                                                                                                                                            |
| `<link rel="dns-prefetch" />` | [ReactDOM prefetchDNSメソッド](#link-reldns-prefetch)を使用                                                                                                                                                                                         |

### リソースヒント {#resource-hints}

`<link>`要素には、外部リソースが必要とされる可能性があることをブラウザに示すために用いることができる`rel`キーワードがいくつかあります。ブラウザは、この情報を使用してキーワードに応じた事前読み込みの最適化を適用します。

メタデータAPIはこれらのヒントを直接サポートしていませんが、新しい[`ReactDOM`メソッド](https://github.com/facebook/react/pull/26237)を利用して、これらをドキュメントの`<head>`に安全に挿入できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/preload-resources.tsx" switcher
'use client'

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/preload-resources.js" switcher
'use client'

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
</Tabs>

##### `<link rel="preload">` {#link-rel-preload}

ページレンダリング（ブラウザ）ライフサイクル中にリソースの読み込みを早期に開始する。[MDNのドキュメント](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preload)

```tsx
ReactDOM.preload(href: string, options: { as: string })
```

```html title="<head> output" hideLineNumbers
<link rel="preload" href="..." as="..." />
```

##### `<link rel="preconnect">` {#link-rel-preconnect}

外部のオリジンに事前に接続を開始します。[MDNのドキュメント](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preconnect)

```tsx
ReactDOM.preconnect(href: string, options?: { crossOrigin?: string })
```

```html title="<head> output" hideLineNumbers
<link rel="preconnect" href="..." crossorigin />
```

##### `<link rel="dns-prefetch">` {#link-rel-dns-prefetch}

リソースが要求される前にドメイン名を解決しようとします。[MDNのドキュメント](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/dns-prefetch)

```tsx
ReactDOM.prefetchDNS(href: string)
```

```html title="<head> output" hideLineNumbers
<link rel="dns-prefetch" href="..." />
```

> **知っておくと良いこと**:
>
> - これらのメソッドは現在、初回ページ読み込み時にまだサーバー・サイドレンダリングされるクライアントコンポーネントでのみサポートされています。
> - Next.jsの組み込み機能である`next/font`、`next/image`、および`next/script`は、関連するリソースヒントを自動的に扱います。

## タイプ {#types}

`Metadata`タイプを使用することで、メタデータに型安全性を追加できます。もしIDEで[組み込みのTypeScriptプラグイン](/docs/app/api-reference/config/typescript)を使用している場合は、タイプを手動で追加する必要はありませんが、希望する場合は明示的に追加することができます。

### `metadata`オブジェクト {#metadata-object}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Next.js',
}
```

### `generateMetadata`関数 {#generatemetadata-function}

#### 通常の関数 {#regular-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export function generateMetadata(): Metadata {
  return {
    title: 'Next.js',
  }
}
```

#### 非同期関数 {#async-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export async function generateMetadata(): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### セグメントpropsを使用する場合 {#with-segment-props}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export function generateMetadata({ params, searchParams }: Props): Metadata {
  return {
    title: 'Next.js',
  }
}

export default function Page({ params, searchParams }: Props) {}
```

#### 親メタデータを使用する場合 {#with-parent-metadata}

```tsx title="layout.tsx | page.tsx"
import type { Metadata, ResolvingMetadata } from 'next'

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### JavaScriptプロジェクト {#javascript-projects}

JavaScriptプロジェクトの場合、JSDocを使用して型安全性を追加できます。

```js title="layout.js | page.js"
/** @type {import("next").Metadata} */
export const metadata = {
  title: 'Next.js',
}
```
