このページでは、`generateMetadata`と静的メタデータオブジェクトを使ったすべての**設定ベースのメタデータ**オプションを紹介します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

// 静的なメタデータ
export const metadata: Metadata = {
  title: '...',
}

// 動的なメタデータ
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher
// 静的なメタデータ
export const metadata = {
  title: '...',
}

// 動的なメタデータ
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
</Tabs>

> **Good to know**:
>
> - `metadata`オブジェクトと`generateMetadata`関数は**Server Componentでのみサポートされています**。
> - 同一のルートセグメントから`metadata`オブジェクトと`generateMetadata`関数の両方をエクスポートすることはできません。
> - 初回ロード時には、`generateMetadata`が完全に解決されるまで、ストリーミングがブロックされます。これは`loading.js`のコンテンツも含まれます。

## `metadata`オブジェクト {#the-metadata-object}

静的メタデータを定義するには、`layout.js`または`page.js`ファイルから[`Metadata`オブジェクト](#metadata-fields)をエクスポートします。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: '...',
  description: '...',
}

export default function Page() {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher
export const metadata = {
  title: '...',
  description: '...',
}

export default function Page() {}
```

</TabItem>
</Tabs>

サポートされるオプションの完全なリストについては、[Metadata Fields](#metadata-fields)を確認してください。

## `generateMetadata`関数 {#generatemetadata-function}

動的メタデータは、**動的な情報**、例えば現在のルートパラメータ、外部データ、または親セグメントの`metadata`に依存します。これらの情報に依存したメタデータを設定するには、`generateMetadata`関数をエクスポートし、[`Metadata`オブジェクト](#metadata-fields)を返します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[id]/page.tsx" switcher
import type { Metadata, ResolvingMetadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  // ルートパラメータを読み取る
  const id = (await params).id

  // データを取得する
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // 親のメタデータにアクセスして拡張する（置き換えではない）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }: Props) {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[id]/page.js" switcher
export async function generateMetadata({ params, searchParams }, parent) {
  // ルートパラメータを読み取る
  const id = (await params).id

  // データを取得する
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // 親のメタデータにアクセスして拡張する（置き換えではない）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }) {}
```

</TabItem>
</Tabs>

### パラメータ {#parameters}

`generateMetadata`関数は以下のパラメータを受け取ります：

- `props` - 現在のルートのパラメータを含むオブジェクト：

  - `params` - [`dynamic route parameters`](/docs/app/building-your-application/routing/dynamic-routes)オブジェクトを包含します；例：

    | ルート                          | URL         | `params`                  |
    | ------------------------------- | ----------- | ------------------------- |
    | `app/shop/[slug]/page.js`       | `/shop/1`   | `{ slug: '1' }`           |
    | `app/shop/[tag]/[item]/page.js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |
    | `app/shop/[...slug]/page.js`    | `/shop/1/2` | `{ slug: ['1', '2'] }`    |

  - `searchParams` - 現在のURLの[search params](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)オブジェクトを包含します；例：

    | URL             | `searchParams`       |
    | --------------- | -------------------- |
    | `/shop?a=1`     | `{ a: '1' }`         |
    | `/shop?a=1&b=2` | `{ a: '1', b: '2' }` |
    | `/shop?a=1&a=2` | `{ a: ['1', '2'] }`  |

- `parent` - 親ルートセグメントから解決されたメタデータのPromise。

### リターン {#returns}

`generateMetadata`は、1つ以上のメタデータフィールドを含む[`Metadata`オブジェクト](#metadata-fields)を返す必要があります。

> **Good to know**:
>
> - メタデータが実行時の情報に依存しない場合は、静的な[`metadata`オブジェクト](#the-metadata-object)を使用して定義するべきです。
> - `fetch`リクエストは、`generateMetadata`、`generateStaticParams`、Layouts、Pages、およびServer Components全体で同じデータのために自動的に[メモ化](/docs/app/building-your-application/caching#request-memoization)されます。`fetch`が利用できない場合は、React[`cache`を使用することもできます](/docs/app/building-your-application/caching#react-cache-function)。
> - `searchParams`は`page.js`セグメントでのみ利用可能です。
> - Next.jsの[`redirect()`](/docs/app/api-reference/functions/redirect) および[`notFound()`](/docs/app/api-reference/functions/not-found)メソッドも`generateMetadata`内で使用できます。

## メタデータフィールド {#metadata-fields}

### `title` {#title}

`title`属性は、文書のタイトルを設定するために使用されます。これは単純な[文字列](#string)またはオプションの[テンプレートオブジェクト](#template-object)として定義できます。

#### 文字列 {#string}

```jsx title="layout.js | page.js"
export const metadata = {
  title: 'Next.js',
}
```

```html title="<head> output" hideLineNumbers

```

```tsx title="app/layout.tsx" switcher
    template: '...',

  },

```

</TabItem>
<TabItem value="jsx" label="JavaScript">

</TabItem>

##### デフォルト {#default}

`title.default`は、`title`を定義していない子ルートセグメントに対して**フォールバックタイトル**を提供するために使用できます。

```tsx title="app/layout.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    default: 'Acme',
  },
}
```

```tsx title="app/about/page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {}

// 出力: <title>Acme</title>
```

##### テンプレート {#template}

`title.template`は、**子**ルートセグメントで定義された`titles`にプレフィックスまたはサフィックスを追加するために使用できます。

> **Good to know**:
>
> - `title.template`は**子**ルートセグメントに適用され、定義されているセグメント自体には適用されません。つまり：
>
>   - `title.template`を追加する際には、`title.default`が**必要**です。
>   - `layout.js`で定義された`title.template`は、同じルートセグメントの`page.js`で定義された`title`には適用されません。
>   - `page.js`で定義された`title.template`は効果がありません。なぜなら、pageは常に終端セグメントであるため、子供のルートセグメントを持たないからです。
>
> - ルートに`title`または`title.default`が定義されていない場合、`title.template`は**何の効果もありません**。

> **Good to know**:
>
> - `layout.js`
>
>   - `title` (文字列) と `title.default` は子セグメントのデフォルトのタイトルを定義し、`title.template`が存在する場合、それに補強します。
>   - `title.absolute` は子セグメントのデフォルトタイトルを定義し、親セグメントの`title.template`を無視します。
>   - `title.template` は子セグメントの新しいタイトルテンプレートを定義します。
>
> - `page.js`
>   - ページが独自のタイトルを定義していない場合、最も近い親の解決されたタイトルが使用されます。
>   - `title` (文字列) はルートのタイトルを定義し、最も近い親セグメントの`title.template`がある場合は補強します。
>   - `title.absolute` はルートタイトルを定義し、親セグメントの`title.template`を無視します。
>   - `title.template` はルートの終端セグメントであるため、`page.js`では効果がありません。

### `metadataBase` {#metadatabase}

`metadataBase`は完全修飾URLを必要とする`metadata`フィールドのために、ベースURLプレフィックスを設定するための便利なオプションです。

- `metadataBase`を使用すると、**現在のルートセグメントおよびその下に定義された**URLベースの`metadata`フィールドが、絶対URLの代わりに**相対パス**を使用できます。
- フィールドの相対パスは、`metadataBase`と合成されて完全修飾URLになります。
- `metadataBase`が設定されていない場合、[デフォルト値](#default-value)で**自動的に設定されます**。

```jsx title="layout.js | page.js"
export const metadata = {
  metadataBase: new URL('https://acme.com'),
  alternates: {
    canonical: '/',
    languages: {
      'en-US': '/en-US',
      'de-DE': '/de-DE',
    },
  },
  openGraph: {
    images: '/og-image.png',
  },
}
```

```html title="<head> output" hideLineNumbers
<link rel="canonical" href="https://acme.com" />
<link rel="alternate" hreflang="en-US" href="https://acme.com/en-US" />
<link rel="alternate" hreflang="de-DE" href="https://acme.com/de-DE" />
<meta property="og:image" content="https://acme.com/og-image.png" />
```

> **Good to know**:
>
> - `metadataBase`は、すべてのルートにわたるURLベースの`metadata`フィールドに適用されるように、rootの`app/layout.js`に通常は設定されます。
> - 完全なURLを必要とするすべてのURLベースの`metadata`フィールドは、`metadataBase`オプションで設定できます。
> - `metadataBase`はサブドメインを含むことができます（例：`https://app.acme.com`）やベースパス（例：`https://acme.com/start/from/here`）
> - `metadata`フィールドが絶対URLを提供する場合、`metadataBase`は無視されます。
> - URLベースの`metadata`フィールドで相対パスを使用し、`metadataBase`が設定されていない場合、ビルドエラーが発生します。
> - Next.jsは`metadataBase`（例：`https://acme.com/`）とフィールド（例：`/path`）の間の重複スラッシュを1つのスラッシュ（例：`https://acme.com/path`）に正規化します。

#### デフォルト値 {#default-value}

`metadataBase`が設定されていない場合、**デフォルト値**を持ちます。

> Vercelで：
>
> - 本番デプロイメントでは、`VERCEL_PROJECT_PRODUCTION_URL`が使用されます。
> - プレビューのデプロイメントでは、優先的に`VERCEL_BRANCH_URL`が取られ、存在しない場合は`VERCEL_URL`にフォールバックします。
>
> これらの値が存在する場合、それが`metadataBase`の**デフォルト値**として使用され、存在しない場合は、`http://localhost:${process.env.PORT || 3000}`にフォールバックします。これにより、Open Graph画像がローカルビルド、Vercelプレビュー、および本番デプロイメントの両方で動作するようになります。デフォルトを上書きする場合、ローカル開発、ステージング、および本番環境向けの異なるURLを設定できるように、環境変数を使用してURLを計算することをお勧めします。
>
> これらの環境変数の詳細については、[システム環境変数](https://vercel.com/docs/concepts/projects/environment-variables/system-environment-variables)のドキュメントをご覧ください。

#### URLの合成 {#url-composition}

URLの合成では、デフォルトのディレクトリ移動セマンティクスよりも開発者の意図を優先します。

- `metadataBase`と`metadata`フィールドの間の末尾のスラッシュは正規化されます。
- `metadata`フィールド内の「絶対」パス（通常は全URLパスを置き換える）は、「相対」パス（`metadataBase`の終わりから始まる）として扱われます。

たとえば、以下の`metadataBase`があるとします：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher
export const metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

</TabItem>
</Tabs>

上記の`metadataBase`を継承し、独自の値を設定した`metadata`フィールドは、以下のように解決されます：

| `metadata`フィールド             | 解決されたURL                    |
| -------------------------------- | -------------------------------- |
| `/`                              | `https://acme.com`               |
| `./`                             | `https://acme.com`               |
| `payments`                       | `https://acme.com/payments`      |
| `/payments`                      | `https://acme.com/payments`      |
| `./payments`                     | `https://acme.com/payments`      |
| `../payments`                    | `https://acme.com/payments`      |
| `https://beta.acme.com/payments` | `https://beta.acme.com/payments` |

## サポートされていないメタデータ {#unsupported-metadata}

現在組み込みのサポートがないメタデータのタイプは以下の通りです。ただし、これらはレイアウトまたはページ自体にレンダリングできます。

| メタデータ                    | 推奨事項                                                                                                                                                                                                                                            |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<meta http-equiv="...">`     | 適切なHTTPヘッダーを使用しています[`redirect()`](/docs/app/api-reference/functions/redirect), [Middleware](/docs/app/building-your-application/routing/middleware#nextresponse), [Security Headers](/docs/app/api-reference/next-config-js/headers) |
| `<base>`                      | タグをレイアウトまたはページ自体にレンダリングしてください。                                                                                                                                                                                        |
| `<noscript>`                  | タグをレイアウトまたはページ自体にレンダリングしてください。                                                                                                                                                                                        |
| `<style>`                     | Next.jsでの[スタイリングについて詳しく学びます](/docs/app/building-your-application/styling/css)。                                                                                                                                                  |
| `<script>`                    | [スクリプトの使用方法についてさらに学びます](/docs/app/building-your-application/optimizing/scripts)。                                                                                                                                              |
| `<link rel="stylesheet" />`   | レイアウトまたはページ自体にスタイルシートを直接`import`します。                                                                                                                                                                                    |
| `<link rel="preload />`       | [ReactDOM preload method](#link-relpreload)を使用                                                                                                                                                                                                   |
| `<link rel="preconnect" />`   | [ReactDOM preconnect method](#link-relpreconnect)を使用                                                                                                                                                                                             |
| `<link rel="dns-prefetch" />` | [ReactDOM prefetchDNS method](#link-reldns-prefetch)を使用                                                                                                                                                                                          |

### リソースのヒント {#resource-hints}

`<link>`要素には、外部リソースが必要になる可能性が高いことをブラウザにヒントするために使用できるいくつかの`rel`キーワードがあります。ブラウザは、この情報を使用して、キーワードに応じたプリロード最適化を適用します。

メタデータAPIはこれらのヒントを直接サポートしていませんが、新しい[`ReactDOM`メソッド](https://github.com/facebook/react/pull/26237)を使用して、ドキュメントの`<head>`に安全に挿入できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/preload-resources.tsx" switcher
'use client'

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/preload-resources.js" switcher

  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
</Tabs>

##### `<link rel="preload">` {#link-rel-preload}

ページレンダリング（ブラウザ）ライフサイクルの早い段階でリソースの読み込みを開始します。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preload)。

```tsx
ReactDOM.preload(href: string, options: { as: string })
```

```html title="<head> output" hideLineNumbers
<link rel="preload" href="..." as="..." />
```

##### `<link rel="preconnect">` {#link-rel-preconnect}

オリジンへの接続を事前に開始します。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preconnect)。

```tsx
ReactDOM.preconnect(href: string, options?: { crossOrigin?: string })
```

```html title="<head> output" hideLineNumbers
<link rel="preconnect" href="..." crossorigin />
```

##### `<link rel="dns-prefetch">` {#link-rel-dns-prefetch}

リソースの要求が送信される前に、ドメイン名の解決を試みます。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/dns-prefetch)。

```tsx
ReactDOM.prefetchDNS(href: string)
```

```html title="<head> output" hideLineNumbers
<link rel="dns-prefetch" href="..." />
```

> **Good to know**:
>
> - これらのメソッドは現在、クライアントコンポーネントでのみサポートされており、初回ページロード時には依然としてサーバーサイドでレンダリングされます。
> - Next.jsの組み込み機能である`next/font`、`next/image`、`next/script`は、関連するリソースヒントを自動的に処理します。

## 型 {#types}

`Metadata`型を使用して、メタデータに型の安全性を追加できます。IDEで[組み込みのTypeScriptプラグイン](/docs/app/api-reference/config/typescript)を使用している場合、型を手動で追加する必要はありませんが、必要に応じて明示的に追加できます。

### `metadata`オブジェクト {#metadata-object}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Next.js',
}
```

### `generateMetadata`関数 {#generatemetadata-function}

#### 通常の関数 {#regular-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export function generateMetadata(): Metadata {
  return {
    title: 'Next.js',
  }
}
```

#### 非同期関数 {#async-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export async function generateMetadata(): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### セグメントprops付き {#with-segment-props}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export function generateMetadata({ params, searchParams }: Props): Metadata {
  return {
    title: 'Next.js',
  }
}

export default function Page({ params, searchParams }: Props) {}
```

#### 親メタデータ付き {#with-parent-metadata}

```tsx title="layout.tsx | page.tsx"
import type { Metadata, ResolvingMetadata } from 'next'

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### JavaScriptプロジェクト {#javascript-projects}

JavaScriptプロジェクトの場合、JSDocを使用して型の安全性を追加できます。

```js title="layout.js | page.js"
/** @type {import("next").Metadata} */
export const metadata = {
  title: 'Next.js',
}
```
