**not-found** ファイルは、[`notFound`](/docs/app/api-reference/functions/not-found) 関数がルートセグメント内でスローされたときにUIをレンダリングするために使用されます。カスタムUIを提供するだけでなく、Next.jsはストリーミングレスポンスには `200` HTTPステータスコードを返し、非ストリーミングレスポンスには `404` を返します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>要求されたリソースが見つかりませんでした</p>
      <Link href="/">ホームに戻る</Link>
    </div>
  )
}
```

## リファレンス {#reference}

### Props {#props}

`not-found.js` コンポーネントは、いかなるpropsも受け付けません。

> **Good to know**: 期待される `notFound()` エラーを捕捉することに加えて、root `app/not-found.js` ファイルはアプリケーション全体で一致しないURLも処理します。これは、アプリで処理されないURLを訪れたユーザーが、`app/not-found.js` ファイルがエクスポートしたUIを表示することを意味します。

## 例 {#examples}

### データフェッチ {#data-fetching}

デフォルトでは、`not-found` はServer Componentです。データをフェッチして表示するために`async`としてマークすることができます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>要求されたリソースが見つかりませんでした</p>
      <p>
        <Link href="/blog">すべての記事を見る</Link>
      </p>
    </div>
  )
}
```

</TabItem>
