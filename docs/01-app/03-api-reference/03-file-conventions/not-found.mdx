**not-found** ファイルは、ルートセグメント内で [`notFound`](/docs/app/api-reference/functions/not-found) 関数がスローされたときに UI をレンダリングするために使用されます。カスタム UI の提供に加えて、Next.js はストリームされたレスポンスの場合に `200` HTTP ステータスコードを返し、非ストリームされたレスポンスの場合には `404` を返します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'

// "見つかりませんでした" を表示する UI をレンダリングする
export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

## 参考 {#reference}

### Props {#props}

`not-found.js` コンポーネントは、いかなる props も受け取りません。

> **Good to know**: 期待される `notFound()` エラーをキャッチすることに加えて、ルート `app/not-found.js` ファイルはアプリケーション全体で一致しない URL も処理します。これは、アプリケーションで処理されていない URL にアクセスするユーザーに対して、`app/not-found.js` ファイルがエクスポートする UI が表示されることを意味します。

## 例 {#examples}

### データフェッチング {#data-fetching}

デフォルトでは、`not-found` は Server Component です。データをフェッチして表示するために `async` としてマークすることができます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

// 非同期関数として定義し、ヘッダー情報を取得する
export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

