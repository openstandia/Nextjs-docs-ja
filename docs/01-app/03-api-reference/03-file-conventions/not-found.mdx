**not-found**ファイルは、ルートセグメント内で[`notFound`](/docs/app/api-reference/functions/not-found)関数がスローされたときにUIをレンダリングするために使用されます。カスタムUIを提供するのと共に、Next.jsはストリーミングレスポンスには`200`のHTTPステータスコードを返し、非ストリーミングレスポンスには`404`を返します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'

// NotFoundコンポーネントの定義
export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>要求されたリソースが見つかりませんでした</p>
      <Link href="/">ホームに戻る</Link>
    </div>
  )
}
```

## リファレンス {#reference}

### Props {#props}

`not-found.js`コンポーネントは、いかなるpropsも受け付けません。

> **Good to know**: 予期された`notFound()`エラーをキャッチすることに加えて、rootの`app/not-found.js`ファイルはアプリケーション全体の一致しないすべてのURLも処理します。つまり、アプリケーションで処理されないURLを訪れるユーザーには、`app/not-found.js`ファイルでエクスポートされるUIが表示されます。

## 例 {#examples}

### データ取得 {#data-fetching}

デフォルトでは、`not-found`はServer Componentです。これを`async`としてマークすることで、データを取得して表示できます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

// 非同期関数としてのNotFoundコンポーネント
export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>要求されたリソースが見つかりませんでした</p>
      <p>
        <Link href="/blog">すべての投稿を表示</Link>
      </p>
    </div>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

