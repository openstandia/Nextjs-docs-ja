**not-found** ファイルは、ルートセグメント内で [`notFound`](/docs/app/api-reference/functions/not-found) 関数が投げられたときに UI をレンダリングするために使用されます。カスタム UI を提供することに加えて、Next.js はストリームされたレスポンスに対しては `200` の HTTP ステータスコードを返し、ストリームされていないレスポンスに対しては `404` を返します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

## リファレンス {#reference}

### Props {#props}

`not-found.js` コンポーネントは、いかなる props も受け取りません。

> **Good to know**: 予期された `notFound()` エラーをキャッチすることに加えて、root の `app/not-found.js` ファイルは、アプリ全体の未一致の URL も処理します。つまり、アプリによって処理されない URL を訪れるユーザーには、`app/not-found.js` ファイルによってエクスポートされた UI が表示されます。

## 例 {#examples}

### データフェッチング {#data-fetching}

デフォルトでは、`not-found` は Server Component です。データを取得して表示するために `async` としてマークできます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

</TabItem>
