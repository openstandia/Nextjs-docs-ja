**not-found**ファイルは、route segment内で[`notFound`](/docs/app/api-reference/functions/not-found)関数が投げられた際にUIをレンダリングするために使用されます。カスタムUIを提供するだけでなく、Next.jsはストリームレスポンスには`200`のHTTPステータスコードを返し、非ストリームレスポンスには`404`を返します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'

// リクエストされたリソースは見つかりませんでした
export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

## リファレンス {#reference}

### Props {#props}

`not-found.js`コンポーネントはどのようなpropsも受け取りません。

> **Good to know**: 期待される`notFound()`エラーをキャッチすることに加え、rootの`app/not-found.js`ファイルはアプリケーション全体で一致しないURLも処理します。したがって、アプリが処理していないURLを訪れたユーザーには、`app/not-found.js`ファイルによってエクスポートされたUIが表示されます。

## 例 {#examples}

### データフェッチング {#data-fetching}

デフォルトでは、`not-found`はServer Componentです。データをフェッチして表示するために`async`としてマークできます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

// リクエストされたリソースは見つかりませんでした
export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">


`usePathname`のようなClient Componentフックを使用してパスに基づいたコンテンツを表示する必要がある場合は、クライアント側でデータをフェッチする必要があります。
