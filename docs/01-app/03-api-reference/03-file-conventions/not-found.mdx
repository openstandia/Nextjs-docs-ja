**not-found** ファイルは、ルートセグメント内で [`notFound`](/docs/app/api-reference/functions/not-found) 関数が投げられた際にUIをレンダリングするために使用されます。カスタム UI を提供するだけでなく、Next.js はストリーミングされたレスポンスに対して `200` HTTP ステータスコードを返し、ストリーミングされていないレスポンスに対して `404` を返します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'

// リクエストされたリソースが見つからない場合に表示するUI
export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

## リファレンス {#reference}

### Props {#props}

`not-found.js` コンポーネントは props を受け入れません

> **Good to know**: 期待された `notFound()` エラーをキャッチすることに加えて、root `app/not-found.js` ファイルはアプリケーション全体の一致しない URL も処理します。つまり、アプリで処理されていない URL を訪れたユーザーには、`app/not-found.js` ファイルでエクスポートされている UI が表示されます。

## 例 {#examples}

### データ取得 {#data-fetching}

デフォルトでは、`not-found` はServer Componentです。データを取得して表示するために `async` としてマークできます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

// 非同期関数としてデータを取得して表示
export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

