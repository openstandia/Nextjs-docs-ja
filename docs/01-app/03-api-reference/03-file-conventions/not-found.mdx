**not-found**ファイルは、ルートセグメント内で[`notFound`](/docs/app/api-reference/functions/not-found)関数が投げられたときにUIをレンダリングするために使用されます。カスタムUIの提供に加えて、Next.jsはストリームされたレスポンスに対して`200`HTTPステータスコードを返し、非ストリームされたレスポンスに対して`404`を返します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>リクエストされたリソースが見つかりませんでした</p>
      <Link href="/">ホームに戻る</Link>
    </div>
  )
}
```

## リファレンス {#reference}

### Props {#props}

`not-found.js`コンポーネントは、いかなるpropsも受け付けません。

> **Good to know**: 期待される`notFound()`エラーをキャッチすることに加えて、root `app/not-found.js`ファイルはアプリ全体で一致しないURLも処理します。つまり、アプリが処理しないURLを訪れたユーザーには`app/not-found.js`ファイルでエクスポートされたUIが表示されます。

## 例 {#examples}

### データフェッチング {#data-fetching}

デフォルトで、`not-found`はServer Componentです。データを取得して表示するためには`async`としてマークすることができます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/not-found.tsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>リクエストされたリソースが見つかりませんでした</p>
      <p>
        <Link href="/blog">すべての投稿を見る</Link>
      </p>
    </div>
  )
}
```

</TabItem>
