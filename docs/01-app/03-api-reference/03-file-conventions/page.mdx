`page` ファイルは、Next.jsアプリケーションでページを定義するために使用されます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  return <h1>My Page</h1>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default function Page({ params, searchParams }) {
  return <h1>My Page</h1>
}
```

</TabItem>
</Tabs>

## リファレンス {#reference}

### プロパティ {#props}

#### `params` (オプショナル) {#params-optional}

rootセグメントからそのページまでの[dynamic route parameters](/docs/app/building-your-application/routing/dynamic-routes)を含むオブジェクトに解決されるプロミス。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const slug = (await params).slug
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/[slug]/page.js" switcher
export default async function Page({ params }) {
  const slug = (await params).slug
}
```

</TabItem>
</Tabs>

| 例のルート                           | URL         | `params`                                |
| ------------------------------------ | ----------- | --------------------------------------- |
| `app/shop/[slug]/page.js`            | `/shop/1`   | `Promise<{ slug: '1' }>`                |
| `app/shop/[category]/[item]/page.js` | `/shop/1/2` | `Promise<{ category: '1', item: '2' }>` |
| `app/shop/[...slug]/page.js`         | `/shop/1/2` | `Promise<{ slug: ['1', '2'] }>`         |

- `params` プロパティはプロミスであるため、値にアクセスするには`async/await`またはReactの[`use`](https://react.dev/reference/react/use)関数を使用する必要があります。
  - バージョン14以前では、`params`は同期プロパティでした。後方互換性のために、Next.js 15では同期的にアクセスできますが、この動作は将来廃止される予定です。

#### `searchParams` (オプショナル) {#searchparams-optional}

現在のURLの[検索パラメーター](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)を含むオブジェクトに解決されるプロミス。例えば：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const filters = (await searchParams).filters
}
```

</TabItem>
</Tabs>

| 例のURL         | `searchParams`                |
| --------------- | ----------------------------- |
| `/shop?a=1`     | `Promise<{ a: '1' }>`         |
| `/shop?a=1&b=2` | `Promise<{ a: '1', b: '2' }>` |
| `/shop?a=1&a=2` | `Promise<{ a: ['1', '2'] }>`  |

- `searchParams` プロパティはプロミスであるため、値にアクセスするには`async/await`またはReactの[`use`](https://react.dev/reference/react/use)関数を使用する必要があります。
  - バージョン14以前では、`searchParams`は同期プロパティでした。後方互換性のために、Next.js 15では同期的にアクセスできますが、この動作は将来廃止される予定です。
- `searchParams` は事前に値を知ることができない[Dynamic API](/docs/app/building-your-application/rendering/server-components#server-rendering-strategies#dynamic-apis)です。これを使用するとページはリクエスト時の[動的レンダリング](/docs/app/building-your-application/rendering/server-components#dynamic-rendering)に切り替わります。
- `searchParams` はPlainなJavaScriptオブジェクトであり、`URLSearchParams`のインスタンスではありません。

## 例 {#examples}

### `params`に基づいたコンテンツの表示 {#displaying-content-based-on-params}

[dynamic route segments](/docs/app/building-your-application/routing/dynamic-routes)を使用することで、`params` プロパティに基づいてページに特定のコンテンツを表示または取得できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

</TabItem>
</Tabs>

### `searchParams`を使ったフィルタリングの処理 {#handling-filtering-with-searchparams}

`searchParams` プロパティを使用して、URLのクエリ文字列に基づいたフィルタリングやページネーション、ソートを処理できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

</TabItem>
</Tabs>

### Client Componentでの`searchParams`と`params`の読み取り {#reading-searchparams-and-params-in-client-components}

Client Component（`async`を使用できない）で`searchParams`と`params`を使用するために、Reactの[`use`](https://react.dev/reference/react/use)関数を使用してプロミスを読み取ることができます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
'use client'

import { use } from 'react'

export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/page.js" switcher
'use client'

import { use } from 'react'

export default function Page({ params, searchParams }) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

</TabItem>
</Tabs>
