`page` ファイルは、Next.js アプリケーション内のページを定義するために使用します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  return <h1>My Page</h1>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default function Page({ params, searchParams }) {
  return <h1>My Page</h1>
}
```

</TabItem>
</Tabs>

## 参照 {#reference}

### Props {#props}

#### `params` (オプション) {#params-optional}

このプロミスは、root segment からそのページまでの [dynamic route parameters](/docs/app/building-your-application/routing/dynamic-routes) を含むオブジェクトに解決します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const slug = (await params).slug
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/[slug]/page.js" switcher
export default async function Page({ params }) {
  const slug = (await params).slug
}
```

</TabItem>
</Tabs>

| 例のルート                           | URL         | `params`                                |
| ------------------------------------ | ----------- | --------------------------------------- |
| `app/shop/[slug]/page.js`            | `/shop/1`   | `Promise<{ slug: '1' }>`                |
| `app/shop/[category]/[item]/page.js` | `/shop/1/2` | `Promise<{ category: '1', item: '2' }>` |
| `app/shop/[...slug]/page.js`         | `/shop/1/2` | `Promise<{ slug: ['1', '2'] }>`         |

- `params` prop はプロミスであるため、値を取得するには `async/await` または React の [`use`](https://react.dev/reference/react/use) 関数を使用しなければなりません；
  - バージョン 14 以前では、`params` は同期的な prop でした； 後方互換性を考慮し、Next.js 15 では同期的にアクセス可能ですが、この動作は将来廃止される予定です；

#### `searchParams` (optional) {#searchparams-optional}

このプロミスは、現在の URL の [search parameters](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters) を含むオブジェクトに解決します。例：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const filters = (await searchParams).filters
}
```

</TabItem>
</Tabs>

| 例のURL         | `searchParams`                |
| --------------- | ----------------------------- |
| `/shop?a=1`     | `Promise<{ a: '1' }>`         |
| `/shop?a=1&b=2` | `Promise<{ a: '1', b: '2' }>` |
| `/shop?a=1&a=2` | `Promise<{ a: ['1', '2'] }>`  |

- `searchParams` prop はプロミスであるため、値を取得するには `async/await` または React の [`use`](https://react.dev/reference/react/use) 関数を使用しなければなりません；
  - バージョン 14 以前では、`searchParams` は同期的な prop でした；後方互換性を考慮し、Next.js 15 では同期的にアクセス可能ですが、この動作は将来廃止される予定です；
- `searchParams` は、その値を事前には知ることができない**[Dynamic API](/docs/app/building-your-application/rendering/server-components#server-rendering-strategies#dynamic-apis)**です；これを使用すると、要求時にページを**[dynamic rendering](/docs/app/building-your-application/rendering/server-components#dynamic-rendering)**に設定します；
- `searchParams` は `URLSearchParams` インスタンスではなく、純粋な JavaScript オブジェクトです；

## 例 {#examples}

### `params` に基づいてコンテンツを表示する {#displaying-content-based-on-params}

[dynamic route segments](/docs/app/building-your-application/routing/dynamic-routes) を使用して、`params` prop に基づいてページに特定のコンテンツを表示または取得できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

</TabItem>
</Tabs>

### `searchParams` を使用してフィルタリングを処理する {#handling-filtering-with-searchparams}

`searchParams` prop を使用して、フィルタリング、ページネーション、または URL のクエリストリングに基づくソートを処理できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

</TabItem>
</Tabs>

### Client Component で `searchParams` と `params` を読み取る {#reading-searchparams-and-params-in-client-components}

Client Component（`async` にできない）で `searchParams` と `params` を使用するためには、React の [`use`](https://react.dev/reference/react/use) 関数を使用してプロミスを読み取れます；

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
'use client'

import { use } from 'react'

export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/page.js" switcher
'use client'

import { use } from 'react'

export default function Page({ params, searchParams }) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

</TabItem>
</Tabs>
