`page` ファイルは、Next.js アプリケーション内でページを定義するために使用されます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  return <h1>My Page</h1>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default function Page({ params, searchParams }) {
  return <h1>My Page</h1>
}
```

</TabItem>
</Tabs>

## 参照 {#reference}

### Props {#props}

#### `params` (オプション) {#params-optional}

このプロミスは、root セグメントからそのページまでの[動的ルートパラメーター](/docs/app/building-your-application/routing/dynamic-routes)を含むオブジェクトに解決されます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const slug = (await params).slug
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/[slug]/page.js" switcher
export default async function Page({ params }) {
  const slug = (await params).slug
}
```

</TabItem>
</Tabs>

| ルートの例                           | URL         | `params`                                |
| ------------------------------------ | ----------- | --------------------------------------- |
| `app/shop/[slug]/page.js`            | `/shop/1`   | `Promise<{ slug: '1' }>`                |
| `app/shop/[category]/[item]/page.js` | `/shop/1/2` | `Promise<{ category: '1', item: '2' }>` |
| `app/shop/[...slug]/page.js`         | `/shop/1/2` | `Promise<{ slug: ['1', '2'] }>`         |

- `params` prop はプロミスなので、`async/await` やReact の [`use`](https://react.dev/reference/react/use) 関数を使用して値にアクセスする必要があります。
  - バージョン 14 以前では、`params` は同期的な prop でした。後方互換性を考慮し、Next.js 15 でも同期的にアクセスできますが、この動作は将来的に非推奨となります。

#### `searchParams` (オプション) {#searchparams-optional}

このプロミスは、現在の URL の[検索パラメーター](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)を含むオブジェクトに解決されます。例：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const filters = (await searchParams).filters
}
```

</TabItem>
</Tabs>

| URL の例        | `searchParams`                |
| --------------- | ----------------------------- |
| `/shop?a=1`     | `Promise<{ a: '1' }>`         |
| `/shop?a=1&b=2` | `Promise<{ a: '1', b: '2' }>` |
| `/shop?a=1&a=2` | `Promise<{ a: ['1', '2'] }>`  |

- `searchParams` prop はプロミスなので、`async/await` や React の [`use`](https://react.dev/reference/react/use) 関数を使用して値にアクセスする必要があります。
  - バージョン 14 以前では、`searchParams` は同期的な prop でした。後方互換性を考慮し、Next.js 15 でも同期的にアクセスできますが、この動作は将来的に非推奨となります。
- `searchParams` は、値を事前に知ることができない**[動的 API](/docs/app/building-your-application/rendering/server-components#server-rendering-strategies#dynamic-apis)**です。これを使用すると、ページがリクエスト時に**[動的レンダリング](/docs/app/building-your-application/rendering/server-components#dynamic-rendering)**を行うようになります。
- `searchParams` は `URLSearchParams` インスタンスではなく、通常の JavaScript オブジェクトです。

## 例 {#examples}

### `params` をもとにコンテンツを表示する {#displaying-content-based-on-params}

[動的ルートセグメント](/docs/app/building-your-application/routing/dynamic-routes)を使用し、`params` prop をもとに、ページの特定のコンテンツを表示またはフェッチできます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

</TabItem>
</Tabs>

### `searchParams` を使用したフィルタリングの処理 {#handling-filtering-with-searchparams}

`searchParams` prop を使用して、URL のクエリ文字列に基づくフィルタリング、ページネーション、またはソートを処理できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

</TabItem>
</Tabs>

### Client Components で `searchParams` と `params` を読み取る {#reading-searchparams-and-params-in-client-components}

Client Component（`async` にできない）で `searchParams` と `params` を使用するには、React の [`use`](https://react.dev/reference/react/use) 関数を使用してプロミスを読み取ります：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
'use client'

import { use } from 'react'

export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/page.js" switcher
'use client'

import { use } from 'react'

export default function Page({ params, searchParams }) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

</TabItem>
</Tabs>
