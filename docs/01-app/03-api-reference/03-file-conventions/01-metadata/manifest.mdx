ブラウザに対してウェブアプリケーションの情報を提供するために、`app` ディレクトリの**root**に、[Web Manifest Specification](https://developer.mozilla.org/docs/Web/Manifest)に準拠した`manifest.(json|webmanifest)`ファイルを追加または生成します。

## 静的マニフェストファイル {#static-manifest-file}

```json title="app/manifest.json | app/manifest.webmanifest"
{
  "name": "My Next.js Application",
  "short_name": "Next.js App",
  "description": "An application built with Next.js",
  "start_url": "/"
  // ...
}
```

## マニフェストファイルを生成する {#generate-a-manifest-file}

[`Manifest` オブジェクト](#manifest-object)を返す `manifest.js` または `manifest.ts` ファイルを追加します。

> 知っておくと良いこと：`manifest.js`は特別なRoute Handlersで、[Dynamic API](/docs/app/building-your-application/caching#dynamic-apis)または[dynamic config](/docs/app/building-your-application/caching#segment-config-options)オプションを使用しない限り、デフォルトでキャッシュされます。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="app/manifest.ts" switcher
import type { MetadataRoute } from 'next'

export default function manifest(): MetadataRoute.Manifest {
  return {
    name: 'Next.js App',
    short_name: 'Next.js App',
    description: 'Next.js App',
    start_url: '/',
    display: 'standalone',
    background_color: '#fff',
    theme_color: '#fff',
    icons: [
      {
        src: '/favicon.ico',
        sizes: 'any',
        type: 'image/x-icon',
      },
    ],
  }
}
```

### マニフェストオブジェクト {#manifest-object}

マニフェストオブジェクトには、ウェブ標準の更新に伴い更新される可能性のある多数のオプションが含まれています。すべての現在のオプションに関する情報は、[TypeScript](https://nextjs.org/docs/app/api-reference/config/typescript#ide-plugin)を使用している場合はコードエディタ内の`MetadataRoute.Manifest`型を参照するか、[MDN](https://developer.mozilla.org/docs/Web/Manifest)のドキュメントをご覧ください。
