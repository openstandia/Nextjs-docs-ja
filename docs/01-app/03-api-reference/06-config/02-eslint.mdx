Next.js は、ESLint プラグインである [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next) を提供しており、これはベース設定にすでに組み込まれています。これにより、Next.js アプリケーションで一般的な問題をキャッチすることが可能です。

## リファレンス {#reference}

以下の ESLint プラグインからの推奨ルールセットはすべて `eslint-config-next` 内で使用されています:

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これが `next.config.js` の設定に優先します。

## 例 {#examples}

### カスタムディレクトリとファイルのリント {#linting-custom-directories-and-files}

デフォルトでは、Next.js は `pages/`、`app/`、`components/`、`lib/`、および `src/` ディレクトリ内のすべてのファイルに対して ESLint を実行します。ただし、`next.config.js` の `eslint` 設定で `dirs` オプションを使用することで、どのディレクトリを対象とするかを指定できます（プロダクション構築の際に）：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // プロダクションビルド (next build) 中に 'pages' と 'utils' ディレクトリのみに対して ESLint を実行
  },
}
```

同様に、`next lint` に対して `--dir` および `--file` フラグを使用して特定のディレクトリやファイルにリントを適用できます：

```bash title="Terminal"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内でのルートディレクトリの指定 {#specifying-a-root-directory-within-a-monorepo}

もし、Next.js がルートディレクトリにインストールされていないプロジェクト（例えばモノレポ）で `eslint-plugin-next` を使用している場合、次のように `.eslintrc` の `settings` プロパティを使って `eslint-plugin-next` に Next.js アプリケーションがどこにあるかを知らせることができます：

```json title=".eslintrc.json"
{
  "extends": "next",
  "settings": {
    "next": {
      "rootDir": "packages/my-app/"
    }
  }
}
```

`rootDir` はパス（相対または絶対）、グロブ（例：`"packages/*/"`）、またはパスと/またはグロブの配列にすることができます。

### キャッシュの無効化 {#disabling-the-cache}

パフォーマンスを向上させるために、ESLint が処理したファイルの情報はデフォルトでキャッシュされます。これは `.next/cache` または定義した [ビルドディレクトリ](/docs/app/api-reference/next-config-js/distDir) に保存されます。ESLint のルールが単一のソースファイル以上の内容に依存しており、キャッシュを無効化する必要がある場合は、`next lint` で `--no-cache` フラグを使用してください。

```bash title="Terminal"
next lint --no-cache
```

### ルールの無効化 {#disabling-rules}

サポートされているプラグイン（`react`、`react-hooks`、`next`）によって提供されるルールを変更または無効化したい場合、`.eslintrc` の `rules` プロパティを使って直接変更できます：

```json title=".eslintrc.json"
{
  "extends": "next",
  "rules": {
    "react/no-unescaped-entities": "off",
    "@next/next/no-page-custom-font": "off"
  }
}
```

### Core Web Vitals との連携 {#with-core-web-vitals}

`next/core-web-vitals` ルールセットは、`next lint` が初めて実行され、**strict** オプションが選択されたときに有効になります。

```json title=".eslintrc.json"
{
  "extends": "next/core-web-vitals"
}
```

`next/core-web-vitals` は、[Core Web Vitals](https://web.dev/vitals/) に影響を与える場合にデフォルトで警告となっているルールの一部をエラーにするために、`eslint-plugin-next` を更新します。

> `next/core-web-vitals` エントリポイントは、[Create Next App](/docs/app/api-reference/cli/create-next-app) で構築された新しいアプリケーションに自動的に含まれます。

### TypeScript と一緒に {#with-typescript}

Next.js ESLint のルールに加えて、`create-next-app --typescript` は `next/typescript` を使って TypeScript 固有のリントルールを設定に追加します：

```json title=".eslintrc.json"
{
  "extends": ["next/core-web-vitals", "next/typescript"]
}
```

これらのルールは、[`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended) に基づいています。
詳しくは [typescript-eslint > Configs](https://typescript-eslint.io/linting/configs) を参照してください。

### Prettier と一緒に {#with-prettier}

ESLint はまた、コードフォーマットのルールを含んでおり、これは既存の [Prettier](https://prettier.io/) 設定と競合する可能性があります。ESLint と Prettier を共存させるには [eslint-config-prettier](https://github.com/prettier/eslint-config-prettier) を ESLint 設定に含めることをお勧めします。

まず、依存関係をインストールします：

```bash title="Terminal"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

その後、既存の ESLint 設定に `prettier` を追加します：

```json title=".eslintrc.json"
{
  "extends": ["next", "prettier"]
}
```

### ステージングされたファイルへのリントの実行 {#running-lint-on-staged-files}

ステージングされた git ファイルに対して `lint-staged` を使用して `next lint` を実行する場合、プロジェクトのルートにある `.lintstagedrc.js` ファイルに次の設定を追加して、`--file` フラグの使用を指定する必要があります。

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## プロダクションビルド中のリントの無効化 {#disabling-linting-during-production-builds}

`next build` 中に ESLint を実行したくない場合は、`next.config.js` で `eslint.ignoreDuringBuilds` オプションを `true` に設定します：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  eslint: {
    // 警告：これにより、プロジェクトに ESLint エラーがあってもプロダクションビルドが正常に完了します。
    ignoreDuringBuilds: true,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
const nextConfig = {
  eslint: {
    // 警告：これにより、プロジェクトに ESLint エラーがあってもプロダクションビルドが正常に完了します。
    ignoreDuringBuilds: true,
  },
}

export default nextConfig
```

</TabItem>

### 既存の設定の移行 {#migrating-existing-config}

アプリケーションで既に ESLint が設定されている場合、いくつかの条件を満たしていない限り、`eslint-config-next` を含むのではなく、このプラグインから直接拡張することをお勧めします。

#### 推奨プラグインルールセット {#recommended-plugin-ruleset}

次の条件が真の場合：

- 以下のプラグインのいずれかを既にインストールしている（`airbnb` や `react-app` のような別の設定を通じて、または個別に）：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- Babel が Next.js で設定されたものと異なる `parserOptions` を特定している（Babel の設定を [カスタマイズしていない限り](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)は推奨されません）
- Node.js および/または TypeScript [リゾルバ](https://github.com/benmosher/eslint-plugin-import#resolvers) を定義する `eslint-plugin-import` をインストールしている

これらの場合、[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js) 内でこれらのプロパティが設定された方法を好む場合は、これらの設定を削除するか、Next.js ESLint プラグインから直接拡張することをお勧めします：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

プラグインは通常、プロジェクトにインストールして `next lint` を実行せずに使用できます：

```bash title="Terminal"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これは、複数の設定で同じプラグインやパーサーをインポートすることによって発生する可能性のある衝突やエラーのリスクを排除します。

#### 追加の設定 {#additional-configurations}

すでに別の ESLint 設定を使用している場合で `eslint-config-next` を含めたい場合、他の設定の後に最後に拡張されることを確認してください。例えば：

```json title=".eslintrc.json"
{
  "extends": ["eslint:recommended", "next"]
}
```

`next` 設定は、`parser`、`plugins`、`settings` プロパティのデフォルト値を設定する処理をすでに行っています。異なる設定が必要な場合を除き、これらのプロパティを手動で再宣言する必要はありません。

他の共有可能な設定を含める場合は、**これらのプロパティが上書きまたは変更されないことを確認する必要があります**。それ以外の場合は、`next` 設定と同じ動作を共有する設定を削除するか、上で述べたように Next.js ESLint プラグインから直接拡張することをお勧めします。
