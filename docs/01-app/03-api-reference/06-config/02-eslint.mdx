Next.jsは、Next.jsアプリケーションで一般的な問題を検出できるESLintプラグイン、[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)を提供しており、これはベース構成に既にバンドルされています。

## 参照 {#reference}

次のESLintプラグインの推奨ルールセットはすべて`eslint-config-next`内で使用されています：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これは`next.config.js`の構成より優先されます。

## 例 {#examples}

### カスタムディレクトリおよびファイルのリント {#linting-custom-directories-and-files}

デフォルトで、Next.jsは`pages/`、`app/`、`components/`、`lib/`、および`src/`ディレクトリ内のすべてのファイルに対してESLintを実行します。しかし、`next.config.js`内の`eslint`設定で`dirs`オプションを使用して、プロダクションビルド時にどのディレクトリに対してリントを実行するかを指定できます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // プロダクションビルド（next build）中に'pages'および'utils'ディレクトリでのみESLintを実行する
  },
}
```

同様に、特定のディレクトリおよびファイルに対してリントを行うために、`next lint`に`--dir`および`--file`フラグを使用できます：

```bash title="ターミナル"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内のrootディレクトリを指定する {#specifying-a-root-directory-within-a-monorepo}

Next.jsがrootディレクトリにインストールされていないプロジェクト（例：モノレポ）で`eslint-plugin-next`を使用している場合、`.eslintrc`の`settings`プロパティを使用してNext.jsアプリケーションがどこにあるかを`eslint-plugin-next`に教えることができます：

```json title=".eslintrc.json"
{
  "extends": "next",
  "settings": {
    "next": {
      "rootDir": "packages/my-app/"
    }
  }
}
```

`rootDir`はパス（相対または絶対）、グロブ（例：`"packages/*/"`）、またはパスやグロブの配列にできます。

### キャッシュを無効にする {#disabling-the-cache}

パフォーマンスを向上させるために、ESLintで処理されたファイルの情報はデフォルトでキャッシュされます。これは`.next/cache`または定義された[ビルドディレクトリ](/docs/app/api-reference/next-config-js/distDir)に保存されます。キャッシュを無効にする必要があるESLintルールを含める場合、`next lint`で`--no-cache`フラグを使用します。

```bash title="ターミナル"
next lint --no-cache
```

### ルールを無効にする {#disabling-rules}

サポートされているプラグイン（`react`、`react-hooks`、`next`）によって提供されるルールを変更または無効にしたい場合は、`.eslintrc`内の`rules`プロパティを使用して直接設定を変更できます：

```json title=".eslintrc.json"
{
  "extends": "next",
  "rules": {
    "react/no-unescaped-entities": "off",
    "@next/next/no-page-custom-font": "off"
  }
}
```

### コアウェブバイタルと共に {#with-core-web-vitals}

`next/core-web-vitals`ルールセットは、`next lint`を最初に実行し、**strict**オプションが選択されたときに有効になります。

```json title=".eslintrc.json"
{
  "extends": "next/core-web-vitals"
}
```

`next/core-web-vitals`は`eslint-plugin-next`を更新して、[Core Web Vitals](https://web.dev/vitals/)に影響を与える場合、デフォルトで警告に設定された多くのルールをエラーにします。

> 新しいアプリケーションが[Create Next App](/docs/app/api-reference/cli/create-next-app)で構築された場合、`next/core-web-vitals`エントリーポイントは自動的に含まれます。

### TypeScriptと共に {#with-typescript}

Next.js ESLintルールに加えて、`create-next-app --typescript`は`next/typescript`を用いて、TypeScript固有のリントルールも設定に追加します：

```json title=".eslintrc.json"
{
  "extends": ["next/core-web-vitals", "next/typescript"]
}
```

これらのルールは[`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)に基づいています。詳細については、[typescript-eslint > Configs](https://typescript-eslint.io/linting/configs)を参照してください。

### Prettierと共に {#with-prettier}

ESLintはコードフォーマットルールも含み、既存の[Prettier](https://prettier.io/)セットアップと競合することがあります。ESLintとPrettierを連携させるために、[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier)をESLintの設定に含めることをお勧めします。

まず、依存関係をインストールします：

```bash title="ターミナル"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

次に、既存のESLint設定に`prettier`を追加します：

```json title=".eslintrc.json"
{
  "extends": ["next", "prettier"]
}
```

### ステージングされたファイルでlintを実行する {#running-lint-on-staged-files}

[lint-staged](https://github.com/okonet/lint-staged)と共に`next lint`を使用して、ステージされたgitファイルでリントを実行したい場合、プロジェクトのルートにある`.lintstagedrc.js`ファイルに次の内容を追加し、`--file`フラグの使用を指定する必要があります：

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## プロダクションビルド中のlintの無効化 {#disabling-linting-during-production-builds}

`next build`中にESLintを実行したくない場合は、`next.config.js`の`eslint.ignoreDuringBuilds`オプションを`true`に設定できます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  eslint: {
    // 警告: これにより、プロジェクトにESLintエラーがあってもプロダクションビルドが正常に完了できるようになります。
    ignoreDuringBuilds: true,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
const nextConfig = {
  eslint: {
    // 警告: これにより、プロジェクトにESLintエラーがあってもプロダクションビルドが正常に完了できるようになります。
    ignoreDuringBuilds: true,
  },
}

export default nextConfig
```

</TabItem>
</Tabs>

### 既存の設定を移行する {#migrating-existing-config}

すでにアプリケーションにESLintを設定している場合、以下のいくつかの条件が満たされない限り、`eslint-config-next`を含める代わりにこのプラグインから直接拡張することをお勧めします。

#### 推奨プラグインルールセット {#recommended-plugin-ruleset}

次の条件が真である場合：

- 次のプラグインのいずれかがすでにインストールされている（個別または`airbnb`や`react-app`のような異なる構成を通じて）：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- BabelがNext.js内で設定されている方法と異なる特定の`parserOptions`を定義している（Babelの設定をカスタマイズしていない限り、これは推奨されません）
- `eslint-plugin-import`をNode.jsおよび/またはTypeScriptの[リゾルバ](https://github.com/benmosher/eslint-plugin-import#resolvers)を定義してインストールして、インポートを処理している

この場合、これらのプロパティが[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js)内でどのように設定されたかを好む場合は、これらの設定を削除することをお勧めします。または、代わりにNext.js ESLintプラグインから直接拡張します：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

プラグインは、`next lint`を実行する必要なく、通常のプロジェクトでインストールできます：

```bash title="ターミナル"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これにより、複数の構成で同じプラグインやパーサーをインポートすることによる衝突やエラーのリスクが排除されます。

#### 追加の構成 {#additional-configurations}

すでに別のESLint構成を使用していて、`eslint-config-next`を含めたい場合は、それが他の構成の後に最後に拡張されていることを確認してください。例えば：

```json title=".eslintrc.json"
{
  "extends": ["eslint:recommended", "next"]
}
```

`next`構成は、`parser`、`plugins`、および`settings`プロパティのデフォルト値の設定を既に処理しています。これらのプロパティを手動で再宣言する必要はありませんが、必要に応じて異なる構成を使用する場合を除きます。

もし他の共有可能な構成を含める場合、**これらのプロパティが上書きされたり変更されたりしないようにする必要があります**。それ以外の場合は、`next`構成と共通する動作をする構成を削除するか、上記のようにNext.js ESLintプラグインから直接拡張することをお勧めします。
