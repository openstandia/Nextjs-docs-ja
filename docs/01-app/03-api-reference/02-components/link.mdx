`<Link>`はReactコンポーネントであり、[プリフェッチ](/docs/app/building-your-application/routing/linking-and-navigating#2-prefetching)とクライアント側でのルート間のナビゲーションを提供するためにHTMLの`<a>`要素を拡張しています。これは、Next.jsでルート間をナビゲートする主な方法です。

基本的な使い方：

```tsx title="app/page.tsx" switcher
export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

## リファレンス {#reference}

`<Link>`コンポーネントに渡すことのできる次のpropsがあります：

<PagesOnly>

| Prop                                | 例                      | 型                  | 必須 |
| ----------------------------------- | ----------------------- | ------------------- | ---- |
| [`href`](#href-required)            | `href="/dashboard"`     | StringまたはObject  | はい |
| [`replace`](#replace)               | `replace={false}`       | Boolean             | -    |
| [`scroll`](#scroll)                 | `scroll={false}`        | Boolean             | -    |
| [`prefetch`](#prefetch)             | `prefetch={false}`      | Boolean             | -    |
| [`legacyBehavior`](#legacybehavior) | `legacyBehavior={true}` | Boolean             | -    |
| [`passHref`](#passhref)             | `passHref={true}`       | Boolean             | -    |
| [`shallow`](#shallow)               | `shallow={false}`       | Boolean             | -    |
| [`locale`](#locale)                 | `locale="fr"`           | StringまたはBoolean | -    |

</PagesOnly>

<AppOnly>

| Prop                     | 例                  | 型                 | 必須 |
| ------------------------ | ------------------- | ------------------ | ---- |
| [`href`](#href-required) | `href="/dashboard"` | StringまたはObject | はい |
| [`replace`](#replace)    | `replace={false}`   | Boolean            | -    |
| [`scroll`](#scroll)      | `scroll={false}`    | Boolean            | -    |
| [`prefetch`](#prefetch)  | `prefetch={false}`  | Booleanまたはnull  | -    |

</AppOnly>

> **Good to know**: `<a>`タグの属性（例えば、`className`や`target="_blank"`など）は`<Link>`にpropsとして追加でき、基盤の`<a>`要素に渡されます。

### `scroll` {#scroll}

**デフォルトでは`true`です。** Next.jsの`<Link>`のデフォルトのスクロール動作は、ブラウザが前後のナビゲーションを処理するのと同様に**スクロール位置を保持することです。**新しい[ページ](/docs/app/building-your-application/routing/pages)にナビゲートすると、ページがビューポートに表示されている限り、スクロール位置は同じままになります。ただし、ページがビューポートに表示されていない場合、Next.jsは最初のページ要素の上にスクロールします。

`scroll = {false}`のとき、Next.jsは最初のページ要素にスクロールしようとしません。

> **Good to know**: Next.jsはスクロール動作を管理する前に`scroll: false`かどうかを確認します。スクロールが有効な場合、ナビゲーションの関連するDOMノードを識別し、各トップレベル要素を検査します。スクロールできない要素やレンダリングされたHTMLを持たない要素はすべてバイパスされます。これは、位置が固定された要素や、`getBoundingClientRect`で計算される非表示の要素を含みます。その後、Next.jsはビューポートに表示されるスクロール可能な要素が見つかるまで兄弟要素を通過します。

```tsx title="app/page.tsx" switcher
  return (
    <Link href="/dashboard" scroll={false}>
      Dashboard
    </Link>
  )
}
```

</TabItem>

### `prefetch` {#prefetch}

<AppOnly>

プリフェッチは、`<Link />`コンポーネントがユーザーのビューポートに入ったとき（最初またはスクロールを介して）に発生します。Next.jsはリンク先のルート（`href`で示される）とそのデータをバックグラウンドでプリフェッチおよびロードし、クライアント側のナビゲーションのパフォーマンスを向上させます。プリフェッチされたデータがユーザーが`<Link />`にホバーするまでに期限切れの場合、Next.jsは再度プリフェッチしようとします。**プリフェッチはプロダクション環境でのみ有効です。**

`prefetch` propに渡すことのできる値は次のとおりです：

- **`null`（デフォルト）**： ルートが静的か動的であるかに応じてプリフェッチの動作が異なります。静的ルートの場合、ルート全体がプリフェッチされます（そのすべてのデータを含む）。動的ルートの場合、[`loading.js`](/docs/app/building-your-application/routing/loading-ui-and-streaming#instant-loading-states) 境界のある最も近いセグメントまでの部分的なルートがプリフェッチされます。
- `true`： 静的および動的ルートの両方で、ルート全体がプリフェッチされます。
- `false`： ビューポートへの進入時およびホバー時ともに、プリフェッチは行われません。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher

  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</TabItem>
</Tabs>

</AppOnly>

<PagesOnly>

プリフェッチは、`<Link />`コンポーネントがユーザーのビューポートに入ったとき（最初またはスクロールを介して）に発生します。Next.jsはリンク先のルート（`href`で示される）とデータをバックグラウンドでプリフェッチおよびロードし、クライアント側のナビゲーションのパフォーマンスを向上させます。**プリフェッチはプロダクション環境でのみ有効です。**

`prefetch` propに渡すことのできる値は以下の通りです：

- **`true`（デフォルト）**： ルート全体とそのデータがプリフェッチされます。
- `false`： ビューポートへの進入時にはプリフェッチが行われませんが、ホバー時には行われます。ホバー時のフェッチも完全に削除したい場合は、`<a>`タグを使用するか、ホバー時のプリフェッチを無効にできるApp Routerを[増分的に採用する](/docs/app/building-your-application/upgrading/app-router-migration)ことを検討してください。

### `legacyBehavior` {#legacybehavior}

`<Link>`の子要素として`<a>`要素は不要になりました。旧式の動作を使用するには`legacyBehavior` propを追加するか、アップグレードするために`<a>`要素を削除します。コードを自動的にアップグレードするための[コーデモッドが利用可能](/docs/app/building-your-application/upgrading/codemods#new-link)です。

> **Good to know**：`legacyBehavior`が`true`に設定されていない場合、すべての[`アンカータグ`](https://developer.mozilla.org/docs/Web/HTML/Element/a)のプロパティが `next/link` に渡されることもできます。例えば、`className`、`onClick` などです。

### `passHref` {#passhref}

`Link`がその子要素に`href`プロパティを送信するように強制します。デフォルトは`false`です。詳細については、[関数コンポーネントをネストさせる例](#nesting-a-functional-component)をご覧ください。

### `scroll` {#scroll}

ナビゲーション後にページの上部にスクロールします。デフォルトは`true`です。

### アクティブリンクのチェック {#checking-active-links}

[usePathname()](/docs/app/api-reference/functions/use-pathname)を使用して、リンクがアクティブかどうかを判断できます。例えば、アクティブなリンクにクラスを追加するには、現在の`pathname`がリンクの`href`と一致するかどうかを確認します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/ui/nav-links.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'
import Link from 'next/link'

export function Links() {
  const pathname = usePathname()

  return (
    <nav>
      <Link className={`link ${pathname === '/' ? 'active' : ''}`} href="/">
        Home
      </Link>

      <Link
        className={`link ${pathname === '/about' ? 'active' : ''}`}
        href="/about"
      >
        About
      </Link>
    </nav>
  )
}
```

</TabItem>

### `id`へスクロールする {#scrolling-to-an-id}

ナビゲーション時に特定の`id`にスクロールしたい場合、URLに`#`リンクを追加するか、単に`href`プロップにリンクを渡せば可能です。これは、`<Link>`が`<a>`要素をレンダリングするためです。

```jsx
<Link href="/dashboard#settings">Settings</Link>

// 出力
<a href="/dashboard#settings">Settings</a>
```

> **Good to know**:
>
> - ナビゲーション時に[ページ](/docs/app/building-your-application/routing/pages)がビューポートに表示されていない場合、Next.jsはページにスクロールします。

### 子要素が`<a>`タグをラップするカスタムコンポーネントの場合 {#if-the-child-is-a-custom-component-that-wraps-an-a-tag}

<AppOnly>

`Link`の子要素が`<a>`タグをラップするカスタムコンポーネントである場合、`Link`に`passHref`を追加しなければなりません。これは、[styled-components](https://styled-components.com/)のようなライブラリを使用している場合に必要です。これがないと、`<a>`タグに`href`属性が含まれず、サイトのアクセシビリティに影響を及ぼし、SEOも悪化する可能性があります。[ESLint](https://nextjs.org/docs/canary/pages/api-reference/config/eslint)を使用している場合、`passHref`を正しく使用するための組み込みルール`next/link-passhref`があります。

</AppOnly>

### 関数コンポーネントをネストさせる {#nesting-a-functional-component}

`Link`の子要素が関数コンポーネントである場合、`passHref`および`legacyBehavior`に加えて、コンポーネントを[`React.forwardRef`](https://react.dev/reference/react/forwardRef)でラップする必要があります：

<AppOnly>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import Link from 'next/link'
import React from 'react'

// MyButtonのpropsタイプを定義します
interface MyButtonProps {
  onClick?: React.MouseEventHandler<HTMLAnchorElement>
  href?: string
}

// React.ForwardRefRenderFunctionを使用して転送されたrefを正しく型付け
const MyButton: React.ForwardRefRenderFunction<
  HTMLAnchorElement,
  MyButtonProps
> = ({ onClick, href }, ref) => {
  return (
    <a href={href} onClick={onClick} ref={ref}>
      Click Me
    </a>
  )
}

// React.forwardRefを使用してコンポーネントをラップ
const ForwardedMyButton = React.forwardRef(MyButton)

export default function Page() {
  return (
    <Link href="/about" passHref legacyBehavior>
      <ForwardedMyButton />
    </Link>
  )
}
```

</TabItem>


### ページの上部へのスクロールを無効にする {#disable-scrolling-to-the-top-of-the-page}

<AppOnly>

Next.jsの`<Link>`のデフォルトのスクロール動作は、ブラウザが前後のナビゲーションを処理するのと同様に**スクロール位置を保持することです。** 新しい[ページ](/docs/app/building-your-application/routing/pages)にナビゲートすると、ページがビューポートに表示されている限り、スクロール位置は同じままになります。

ただしページがビューポートに表示されていない場合、Next.jsは最初のページ要素の上にスクロールします。この動作を無効にしたい場合は、`<Link>`コンポーネントに`scroll={false}`を渡すか、`router.push()`または`router.replace()`に`scroll: false`を渡せば良いです。

<Tabs>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher

  return (
    <Link href="/#hashid" scroll={false}>
      Disables scrolling to the top
    </Link>
  )
}
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/#hashid" scroll={false}>
      Disables scrolling to the top
    </Link>
  )
}
```

</TabItem>
</Tabs>

```jsx

```

</AppOnly>

### Middlewareでのリンクのプリフェッチ {#prefetching-links-in-middleware}

[Middleware](/docs/app/building-your-application/routing/middleware)は認証やその他の目的でユーザーを異なるページにリダイレクトするためによく使用されます。`<Link />`コンポーネントがミドルウェア経由で書き換えられたリンクを適切にプリフェッチするためには、Next.jsに表示するURLとプリフェッチするURLの両方を伝える必要があります。これは、適切なルートをプリフェッチするためにミドルウェアへの不必要なフェッチを避けるために必要です。

たとえば、認証されたビューと訪問者ビューのある`/dashboard`ルートを提供したい場合、Middlewareに次のコードを追加して、ユーザーを正しいページにリダイレクトできます：

```ts title="middleware.ts"
import { NextResponse } from 'next/server'

export function middleware(request: Request) {
  const nextUrl = request.nextUrl
  if (nextUrl.pathname === '/dashboard') {
    if (request.cookies.authToken) {
      return NextResponse.rewrite(new URL('/auth/dashboard', request.url))
    } else {
      return NextResponse.rewrite(new URL('/public/dashboard', request.url))
    }
  }
}
```

```js title="middleware.js"
import { NextResponse } from 'next/server'

export function middleware(request) {
  const nextUrl = request.nextUrl
  if (nextUrl.pathname === '/dashboard') {
    if (request.cookies.authToken) {
      return NextResponse.rewrite(new URL('/auth/dashboard', request.url))
    } else {
      return NextResponse.rewrite(new URL('/public/dashboard', request.url))
    }
  }
}
```

この場合、以下のコードを`<Link />`コンポーネントで使用します：
