`<Link>` は、HTML の `<a>` 要素を拡張して、ルート間の[プリフェッチ](/docs/app/building-your-application/routing/linking-and-navigating#2-prefetching)とクライアントサイドのナビゲーションを提供する React コンポーネントです。これは、Next.js におけるルート間の移動の主な方法です。

基本的な使い方：

<AppOnly>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">


## Reference {#reference}

以下の props を `<Link>` コンポーネントに渡すことができます：

<PagesOnly>

| Prop                                | 例                      | 型                | 必須 |
| ----------------------------------- | ----------------------- | ----------------- | ---- |
| [`href`](#href-required)            | `href="/dashboard"`     | String or Object  | はい |
| [`replace`](#replace)               | `replace={false}`       | Boolean           | -    |
| [`scroll`](#scroll)                 | `scroll={false}`        | Boolean           | -    |
| [`prefetch`](#prefetch)             | `prefetch={false}`      | Boolean           | -    |
| [`legacyBehavior`](#legacybehavior) | `legacyBehavior={true}` | Boolean           | -    |
| [`passHref`](#passhref)             | `passHref={true}`       | Boolean           | -    |
| [`shallow`](#shallow)               | `shallow={false}`       | Boolean           | -    |
| [`locale`](#locale)                 | `locale="fr"`           | String or Boolean | -    |

</PagesOnly>

<AppOnly>

| Prop                     | 例                  | 型               | 必須 |
| ------------------------ | ------------------- | ---------------- | ---- |
| [`href`](#href-required) | `href="/dashboard"` | String or Object | はい |
| [`replace`](#replace)    | `replace={false}`   | Boolean          | -    |
| [`scroll`](#scroll)      | `scroll={false}`    | Boolean          | -    |
| [`prefetch`](#prefetch)  | `prefetch={false}`  | Boolean or null  | -    |

</AppOnly>

> **Good to know**: `<a>` タグの属性である `className` や `target="_blank"` などを `<Link>` に props として追加でき、それらは基になる `<a>` 要素に引き継がれます。

### `scroll` {#scroll}

**デフォルトは `true`。** Next.js の `<Link>` のデフォルトのスクロール動作は、ブラウザのバックとフォワードのナビゲーションの処理に似ており、**スクロール位置を保持します**。新しい[ページ](/docs/app/building-your-application/routing/pages)に移動する際、ページがビューポートに表示される限り、スクロール位置は同じになります。しかし、ページがビューポートに表示されない場合、Next.js は最初のページ要素の上部にスクロールします。

`scroll = {false}` の場合、Next.js は最初のページ要素にスクロールしようとしません。

> **Good to know**: Next.js はスクロール動作を管理する前に `scroll: false` の確認を行います。スクロールが有効になっている場合、ナビゲーション用の関連する DOM ノードを特定し、各トップレベル要素を検査します。スクロール不可能な要素やレンダリングされた HTML がない要素はすべてバイパスされ、これには固定位置の要素や表示されていない要素、`getBoundingClientRect` で計算された要素、が含まれます。その後、Next.js は兄弟要素を通じて、ビューポートに表示されるスクロール可能な要素を特定します。

```tsx title="app/page.tsx" switcher
  return (

      Dashboard

}
```

### `prefetch` {#prefetch}

<AppOnly>

プリフェッチは、`<Link />` コンポーネントがユーザーのビューポートに入ったとき（最初またはスクロールで）に発生します。Next.js はリンクされたルート（`href` で示されたもの）とそのデータをバックグラウンドでプリフェッチして、クライアントサイドのナビゲーションのパフォーマンスを向上させます。プリフェッチされたデータがリンクの上にマウスをホバーする時点で期限切れの場合、Next.js は再びプリフェッチを試みます。**プリフェッチは本番環境でのみ有効です。**

`prefetch` prop には次の値を渡すことができます：

- **`null` (デフォルト)**：ルートが静的か動的かに応じてプリフェッチの動作が異なります。静的ルートの場合、完全なルートがプリフェッチされます（すべてのデータを含む）。動的ルートの場合、[`loading.js`](https://nextjs.org/docs/app/building-your-application/routing/loading-ui-and-streaming#instant-loading-states) 境界を持つ最も近いセグメントまでの部分ルートがプリフェッチされます。
- `true`：静的ルートと動的ルートの両方で完全なルートがプリフェッチされます。
- `false`：ビューポートへの進入時およびホバー時にプリフェッチが行われることはありません。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</TabItem>
</Tabs>

</AppOnly>

<PagesOnly>

プリフェッチは、`<Link />` コンポーネントがユーザーのビューポートに入ったとき（最初またはスクロールで）に発生します。Next.js はリンクされたルート（`href` で示されたもの）とデータをバックグラウンドでプリフェッチして、クライアントサイドのナビゲーションのパフォーマンスを向上させます。**プリフェッチは本番環境でのみ有効です。**

`prefetch` prop には次の値を渡すことができます：

- **`true` (デフォルト)**：完全なルートとそのデータがプリフェッチされます。
- `false`：ビューポートへの進入時にはプリフェッチが行われませんが、ホバー時にはプリフェッチが行われます。ホバー時のフェッチを完全に削除したい場合は、`<a>` タグまたは [app router の増分採用](/docs/app/building-your-application/upgrading/app-router-migration)を検討してください。これにより、ホバー時のプリフェッチも無効にすることができます。

### `legacyBehavior` {#legacybehavior}

`<a>` 要素は `<Link>` の子としてもはや必要ありません。レガシー動作を使用するには `legacyBehavior` prop を追加するか、アップグレードするために `<a>` を削除してください。[自動的にコードをアップグレードするための codemod が利用可能です](/docs/app/building-your-application/upgrading/codemods#new-link)。

> **Good to know**: `legacyBehavior` が `true` に設定されていない場合、`next/link` に `className`, `onClick` などのすべての [`anchor`](https://developer.mozilla.org/docs/Web/HTML/Element/a) タグのプロパティを渡すことができます。

### `passHref` {#passhref}

`Link` に `href` プロパティをその子に送信させます。デフォルトは `false`。詳細については、[関数型コンポーネントを渡す](#nesting-a-functional-component)の例を参照してください。

### `scroll` {#scroll}

ナビゲーション後にページのトップにスクロールします。デフォルトは `true`。

### `locale` {#locale}

アクティブなロケールは自動的にプレフィックスとして追加されます。`locale` は異なるロケールを提供することを可能にします。`false` の場合、デフォルトの動作は無効化されるため、`href` はロケールを含まなければなりません。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="pages/index.tsx" switcher
import Link from 'next/link'

export default function Home() {
  return (
    <>
      {/* デフォルトの動作: ロケールがプレフィックスとして追加されます */}
      <Link href="/dashboard">Dashboard (with locale)</Link>

      {/* ロケールのプレフィックスを無効化する */}
      <Link href="/dashboard" locale={false}>
        Dashboard (without locale)
      </Link>

      {/* 異なるロケールを指定する */}
      <Link href="/dashboard" locale="fr">
        Dashboard (French)
      </Link>
    </>
  )
}
```

</TabItem>

### dynamic route segment へのリンク {#linking-to-dynamic-segments}

[dynamic segment](/docs/app/building-your-application/routing/dynamic-routes) へのリンク時に、リンクのリストを生成するために [テンプレートリテラルと補間](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Template_literals)を使用できます。たとえば、ブログ投稿のリストを生成するには次のようにします：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/post-list.tsx" switcher
import Link from 'next/link'

interface Post {
  id: number
  title: string
  slug: string
}

export default function PostList({ posts }: { posts: Post[] }) {
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>
          <Link href={`/blog/${post.slug}`}>{post.title}</Link>
        </li>
      ))}
    </ul>
  )
}
```

### アクティブなリンクのチェック {#checking-active-links}

[`usePathname()`](/docs/app/api-reference/functions/use-pathname) を使用してリンクがアクティブかどうかを確認できます。たとえば、アクティブなリンクにクラスを追加するには、現在の `pathname` がリンクの `href` と一致するかを確認します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/ui/nav-links.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'
import Link from 'next/link'

export function Links() {
  const pathname = usePathname()

  return (
    <nav>
      <Link className={`link ${pathname === '/' ? 'active' : ''}`} href="/">
        Home
      </Link>

      <Link
        className={`link ${pathname === '/about' ? 'active' : ''}`}
        href="/about"
      >
        About
      </Link>
    </nav>
  )
}
```

</TabItem>

### 特定の`id`へのスクロール {#scrolling-to-an-id}

ナビゲーション時に特定の `id` までスクロールしたい場合、URL 末尾に `#` ハッシュリンクを追加するか、`href` prop にハッシュリンクを渡すことができます。これは、`<Link>` が `<a>` 要素にレンダリングされるため可能です。

```jsx
<Link href="/dashboard#settings">Settings</Link>

// 出力
<a href="/dashboard#settings">Settings</a>
```

> **Good to know**:
>
> - ナビゲーション時にビューポートに表示されていない場合、Next.js は [ページ](/docs/app/building-your-application/routing/pages) までスクロールします。

</AppOnly>

### 子が `<a>` タグをラップするカスタムコンポーネントである場合 {#if-the-child-is-a-custom-component-that-wraps-an-a-tag}

<AppOnly>

子が `<a>` タグをラップするカスタムコンポーネントである場合、`Link` に `passHref` を追加する必要があります。これは [styled-components](https://styled-components.com/) などのライブラリを使用する場合に必要です。これがないと、`<a>` タグには `href` 属性が含まれず、サイトのアクセシビリティに悪影響を及ぼし、SEO にも影響を与える可能性があります。[ESLint](https://nextjs.org/docs/canary/pages/api-reference/config/eslint) を使用している場合、`passHref` の正しい使用を確保するための組み込み規則 `next/link-passhref` があります。

</AppOnly>

### 関数型コンポーネントのネスト {#nesting-a-functional-component}

`Link` の子が関数型コンポーネントである場合、`passHref` と `legacyBehavior` を使用するだけでなく、[`React.forwardRef`](https://react.dev/reference/react/forwardRef) でコンポーネントをラップする必要があります：

<AppOnly>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import Link from 'next/link'
import React from 'react'

// MyButtonのpropsタイプを定義します
interface MyButtonProps {
  onClick?: React.MouseEventHandler<HTMLAnchorElement>
  href?: string
}

// React.ForwardRefRenderFunctionを使用して、転送されたrefを適切にタイプします
const MyButton: React.ForwardRefRenderFunction<
  HTMLAnchorElement,
  MyButtonProps
> = ({ onClick, href }, ref) => {
  return (
    <a href={href} onClick={onClick} ref={ref}>
      Click Me
    </a>
  )
}

// React.forwardRefを使用してコンポーネントをラップします
const ForwardedMyButton = React.forwardRef(MyButton)

export default function Page() {
  return (
    <Link href="/about" passHref legacyBehavior>
      <ForwardedMyButton />
    </Link>
  )
}
```

### ページのトップへのスクロールを無効にする {#disable-scrolling-to-the-top-of-the-page}

<AppOnly>

Next.js の `<Link>` のデフォルトのスクロール動作は、ブラウザのバックとフォワードのナビゲーションの処理に似ており、**スクロール位置を保持します**。新しい[ページ](/docs/app/building-your-application/routing/pages)に移動する際、ページがビューポートに表示される限り、スクロール位置は同じになります。

しかし、ページがビューポートに表示されない場合、Next.js は最初のページ要素の上部にスクロールします。この動作を無効にしたい場合、`<Link>` コンポーネントまたは `router.push()` または `router.replace()` に `scroll={false}` を渡すことができます。

### Middlewareでのリンクのプリフェッチ {#prefetching-links-in-middleware}

[Middleware](/docs/app/building-your-application/routing/middleware) を認証やユーザーを別のページにリダイレクトする他の目的で使用することが一般的です。`<Link />` コンポーネントが Middleware 経由でリダイレクトされるリンクを正しくプリフェッチするために、Next.js に表示する URL とプリフェッチする URL の両方を伝える必要があります。これは、プリフェッチする正しいルートを Middleware に無駄にフェッチしないようにするために必要です。

たとえば、認証されたビューと訪問者ビューを持つ `/dashboard` ルートを提供したい場合、Middleware に次のコードを追加して正しいページにリダイレクトすることができます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="middleware.ts" switcher
import { NextResponse } from 'next/server'

export function middleware(request: Request) {
  const nextUrl = request.nextUrl
  if (nextUrl.pathname === '/dashboard') {
    if (request.cookies.authToken) {
      return NextResponse.rewrite(new URL('/auth/dashboard', request.url))
    } else {
      return NextResponse.rewrite(new URL('/public/dashboard', request.url))
    }
  }
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

