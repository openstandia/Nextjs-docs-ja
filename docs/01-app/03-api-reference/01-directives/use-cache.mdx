`use cache`ディレクティブは、コンポーネントや関数をキャッシュするために指定します。これにより、ファイルの先頭で指定すると、ファイル内のすべてのエクスポートがキャッシュ可能であることを示します。また、関数やコンポーネントの先頭で指定すると、Next.jsに対して戻り値をキャッシュし、後続のリクエストで再利用するべきであることを指示します。これは実験的なNext.jsの機能であり、[`use client`](/docs/app/api-reference/directives/use-client)や[`use server`](/docs/app/api-reference/directives/use-server)のようなネイティブなReact機能ではありません。

## 使用法 {#usage}

`use cache`ディレクティブのサポートを有効にするには、`next.config.ts`ファイルで[`dynamicIO`](/docs/app/api-reference/config/next-config-js/dynamicIO)フラグを設定します：

```ts title="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
  },
}

export default nextConfig
```

その後、ファイル、コンポーネント、または関数レベルで`use cache`ディレクティブを使用できます：

```tsx
// ファイルレベル
'use cache'

export default async function Page() {
  // ...
}

// コンポーネントレベル
export async function MyComponent() {
  'use cache'
  return <></>
}

// 関数レベル
export async function getData() {
  'use cache'
  const data = await fetch('/api/data')
  return data
}
```

## Good to know {#good-to-know}

- `use cache`は実験的なNext.jsの機能で、ネイティブなReactの[`use client`](/docs/app/api-reference/directives/use-client)や[`use server`](/docs/app/api-reference/directives/use-server)のような機能ではありません。
- キャッシュされた関数に渡されたすべての[直列化可能な](https://react.dev/reference/rsc/use-server#serializable-parameters-and-return-values)引数（またはprops）や、親スコープから読み取った直列化可能な値は、JSONのような形式に変換され、キャッシュキーの一部になります。
- 直列化できない引数、props、クローズドオーバーされた値は、キャッシュされた関数内では不透明な参照に変わり、通過させることはできますが、検査や変更はできません。これらの直列化できない値はリクエスト時に埋め込まれ、キャッシュキーの一部にはなりません。
  - たとえば、キャッシュされた関数はJSXを`children`として受け取り、`<div>{children}</div>`を返すことができますが、実際の`children`オブジェクトを内省することはできません。
- キャッシュ可能な関数の戻り値も直列化可能である必要があります。これにより、キャッシュされたデータが正しく保存および取得されます。
- `use cache`ディレクティブを使用する関数は、状態変更、DOMの直接操作、一定時間ごとにコードを実行するためのタイマーの設定などの副作用を持ってはいけません。
- [`Partial Prerendering`](/docs/app/building-your-application/rendering/partial-prerendering)と一緒に使用した場合、`use cache`を持つセグメントは静的HTMLシェルの一部としてプレンダーされます。
- `use cache`ディレクティブは将来的には[`dynamicIO`](/docs/app/api-reference/config/next-config-js/dynamicIO)フラグから独立して利用可能になる予定です。
- [`unstable_cache`](/docs/app/api-reference/functions/unstable_cache)がJSONデータのみをサポートするのに対して、`use cache`はReactが描画できる任意の直列化可能なデータ、コンポーネントのレンダー出力をキャッシュできます。

## 例 {#examples}

### `use cache`でルート全体をキャッシュする {#caching-entire-routes-with-use-cache}

ルート全体をプレンダーするには、`layout`および`page`ファイルの先頭に`use cache`を追加します。これらのセグメントはアプリケーション内の別々のエントリポイントとして扱われ、独立してキャッシュされます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher
"use cache"
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default Layout({children}: {children: ReactNode}) {
  return <div>{children}</div>
}
```

</TabItem>
</Tabs>
<Tabs>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.tsx" switcher
"use cache"
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default Layout({ children }) {
  return <div>{children}</div>
}
```

</TabItem>
</Tabs>

`page`ファイル内にインポートおよびネストされたコンポーネントは`page`のキャッシュ動作を継承します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher

import { unstable_cacheLife as cacheLife } from 'next/cache'

async function Users() {
  const users = await fetch('/api/users');
  // ユーザーをループ

export default Page() {
  return (
    <main>
      <Users/>
    </main>

}
```

### `use cache`でコンポーネント出力をキャッシュする {#caching-component-output-with-use-cache}

`use cache`をコンポーネントレベルで使用して、そのコンポーネント内で行われたフェッチや計算をキャッシュできます。アプリケーション全体でコンポーネントを再利用する際、propsの構造が維持されている限り、同じキャッシュエントリを共有できます。

propsは直列化され、キャッシュキーの一部を形成し、直列化されたpropsが各インスタンスで同じ値を生成する限り、キャッシュエントリが再利用されます。

```tsx title="app/components/bookings.tsx" highlight={2}
export async function Bookings({ type = 'haircut' }: BookingsProps) {
  'use cache'
  async function getBookingsData() {
    const data = await fetch(`/api/bookings?type=${encodeURIComponent(type)}`)
    return data
  }
  return //...
}

interface BookingsProps {
  type: string
}
```

```jsx title="app/components/bookings.js" highlight={2}
export async function Bookings({ type = 'haircut' }) {
  'use cache'
  async function getBookingsData() {
    const data = await fetch(`/api/bookings?type=${encodeURIComponent(type)}`)
    return data
  }
  return //...
}
```

### `use cache`で関数出力をキャッシュする {#caching-function-output-with-use-cache}

`use cache`を任意の非同期関数に追加できるため、コンポーネントやルートのみならず多くの部分をキャッシュできます。ネットワークリクエストやデータベースクエリをキャッシュしたり、非常に遅い計算をキャッシュしたい場合があります。このタイプの作業を含む関数に`use cache`を追加することで、キャッシュ可能になり、再利用時には同じキャッシュエントリを共有します。

```tsx title="app/actions.ts" highlight={2} switcher
export async function getData() {
  'use cache'

  const data = await fetch('/api/data')
  return data
}
```

</TabItem>

### 再検証 {#revalidating}

デフォルトでは、`use cache`ディレクティブを使用すると、Next.jsは**15分の再検証期間**を設定します。Next.jsはほぼ無限の有効期限を設定し、頻繁な更新を必要としないコンテンツに適しています。

この再検証期間は、あまり変更されないと思われるコンテンツに有用ですが、`cacheLife`および`cacheTag`APIを使用してキャッシュの動作を設定できます：

- [`cacheLife`](/docs/app/api-reference/functions/cacheLife)： 時間に基づいた再検証期間のため
- [`cacheTag`](/docs/app/api-reference/functions/cacheTag)：オンデマンドでの再検証のため

これらのAPIは、クライアントとサーバーのキャッシング層全体で統合されており、一箇所でキャッシングの意味論を設定し、すべてに適用できます。

[`cacheLife`](/docs/app/api-reference/functions/cacheLife)と[`cacheTag`](/docs/app/api-reference/functions/cacheTag)のドキュメントを参照してください。

### インターリーブ {#interleaving}

キャッシュ可能な関数に直列化できない引数を渡す必要がある場合は、それらを`children`として渡すことができます。これにより、`children`参照が変更されても、キャッシュエントリに影響を与えません。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
async function Page() {
  const uncachedData = await getData()
  return (
    <CacheComponent>
      <DynamicComponent data={uncachedData} />
    </CacheComponent>
  )
}

async function CacheComponent({ children }) {
  'use cache'
  const cachedData = await fetch('/api/cached-data')
  return (
    <div>
      <PrerenderedComponent data={cachedData} />
      {children}
    </div>
  )
}
```

</TabItem>
</Tabs>

サーバーアクションをキャッシュされたコンポーネント経由でクライアントコンポーネントに渡し、キャッシュ可能な関数内で呼び出さずに行うこともできます。
