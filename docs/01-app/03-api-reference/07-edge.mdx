## 非対応のAPI {#unsupported-apis}

Edge Runtimeにはいくつかの制約があります、以下はその例です：

- ネイティブのNode.js APIは**サポートされていません**。たとえば、ファイルシステムの読み書きはできません。
- `node_modules` が使用できますが、ES Modulesを実装しており、ネイティブのNode.js APIを使用していない場合のみです。
- `require`を直接呼び出すことは**許可されていません**。代わりにES Modulesを使ってください。

次のJavaScriptの言語機能は無効化されており、**動作しません：**

| API                                                                                                                             | 説明                                                                          |
| ------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| [`eval`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/eval)                                       | 文字列として表されているJavaScriptコードを評価します                          |
| [`new Function(evalString)`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Function)               | 引数として提供されるコードを使って新しい関数を作成します                      |
| [`WebAssembly.compile`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/compile)         | バッファソースからWebAssemblyモジュールをコンパイルします                     |
| [`WebAssembly.instantiate`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/instantiate) | バッファソースからWebAssemblyモジュールをコンパイルおよびインスタンス化します |

まれに、コードがランタイムで到達できない、またはトリーシェイクを通じて削除できない動的コード評価ステートメントを含む（またはインポートする）可能性があります。
ミドルウェアの設定で指定ファイルを許可するチェックを緩めることができます：

```javascript title="middleware.ts"
export const config = {
  unstable_allowDynamic: [
    // 単一ファイルを許可
    '/lib/utilities.js',
    // グロブを使ってfunction-bindという3rdパーティーモジュールのすべてを許可
    '**/node_modules/function-bind/**',
  ],
}
```

`unstable_allowDynamic`は[glob](https://github.com/micromatch/micromatch#matching-features)、もしくは特定のファイルのために動的コード評価を無視するグロブの配列です。これらのグロブはアプリケーションrootフォルダに相対します。

注意：これらのステートメントがEdgeで実行されると、*ランタイムエラーを投げて実行が失敗する*可能性があります。
