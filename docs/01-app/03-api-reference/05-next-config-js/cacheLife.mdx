`cacheLife` オプションを使用することで、コンポーネントや関数内で[`cacheLife`](/docs/app/api-reference/functions/cacheLife) 関数を利用する場合、または [`use cache` directive](/docs/app/api-reference/directives/use-cache) の範囲内で **カスタムキャッシュプロファイル** を定義することができます。

## 使用方法 {#usage}

プロファイルを定義するには、[`dynamicIO` フラグ](/docs/app/api-reference/next-config-js/dynamicIO) を有効にし、`cacheLife` オブジェクトにキャッシュプロファイルを `next.config.js` ファイルに追加します。例えば、`blog` プロファイルを次のように設定します：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
    cacheLife: {
      blog: {
        stale: 3600, // 1時間
        revalidate: 900, // 15分
        expire: 86400, // 1日
      },
    },
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
module.exports = {
  experimental: {
    dynamicIO: true,
    cacheLife: {
      blog: {
        stale: 3600, // 1時間
        revalidate: 900, // 15分
        expire: 86400, // 1日
      },
    },
  },
}
```

</TabItem>
</Tabs>

このカスタム `blog` 設定をコンポーネントや関数で次のように利用できます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/actions.ts" highlight={4,5} switcher
import { unstable_cacheLife as cacheLife } from 'next/cache'

export async function getCachedData() {
  'use cache'
  cacheLife('blog')
  const data = await fetch('/api/data')
  return data
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/actions.js" highlight={4,5} switcher
import { unstable_cacheLife as cacheLife } from 'next/cache'

export async function getCachedData() {
  'use cache'
  cacheLife('blog')
  const data = await fetch('/api/data')
  return data
}
```

</TabItem>
</Tabs>

## リファレンス {#reference}

設定オブジェクトは以下のフォーマットでキーと値を持ちます：

| **プロパティ** | **値**   | **説明**                                                                         | **要件**                                    |
| -------------- | -------- | -------------------------------------------------------------------------------- | ------------------------------------------- |
| `stale`        | `number` | クライアントがサーバーを確認せずに値をキャッシュすべき期間。                     | 任意                                        |
| `revalidate`   | `number` | キャッシュがサーバーで更新される頻度；更新中は古い値が提供されることがあります。 | 任意                                        |
| `expire`       | `number` | 対応しきれなくなるまでの最大期間。                                               | 任意 - `revalidate`より長い必要があります。 |
