`dynamicIO`フラグは、Next.jsにおける実験的な機能で、App Router内でのデータ取得操作が明示的にキャッシュされない限り、プリレンダリングから除外されるようにします。これは、サーバーコンポーネントでの動的なデータ取得のパフォーマンスを最適化するのに役立ちます。

これは、プリレンダーされたキャッシュから提供するのではなく、ランタイム中に新しいデータ取得が必要なアプリケーションにとって有用です。

特定のアプリケーションのパーツをページ、関数、またはコンポーネントレベルで `use cache` を使用してキャッシュとして定義しない限り、デフォルトでランタイムにデータ取得を実行するために[`use cache`](/docs/app/api-reference/directives/use-cache)と共に使用されることを想定しています。

## 使用法 {#usage}

`dynamicIO`フラグを有効にするには、`next.config.ts`ファイルの `experimental` セクションでそれを `true` に設定します：

```ts title="next.config.ts"
import type { NextConfig } from 'next'

// Next.jsの設定ファイルです
const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true, // dynamicIOフラグを有効にします
  },
}

export default nextConfig // 設定をエクスポートします
```

`dynamicIO`が有効になっている場合、次のキャッシュ関数と設定を使用できます：

- [`use cache`ディレクティブ](/docs/app/api-reference/directives/use-cache)
- `cacheLife` 項が `use cache`との併用
- [`cacheTag` 関数](/docs/app/api-reference/functions/cacheTag)

## 注意点 {#notes}

- `dynamicIO`は実行時に新しいデータ取得を保証することによりパフォーマンスを最適化できますが、プリレンダーされたコンテンツを提供する場合に比べて追加の遅延が発生する可能性もあります。
