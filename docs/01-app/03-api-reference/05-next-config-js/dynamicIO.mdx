`dynamicIO`フラグは、Next.jsの実験的な機能で、App Router内のデータフェッチ操作が明示的にキャッシュされない限り、事前レンダリングから除外されるようにします。これは、server componentでの動的なデータフェッチのパフォーマンスを最適化するのに役立ちます。

アプリケーションが、事前レンダリングされたキャッシュではなく、ランタイム中に新鮮なデータフェッチを必要とする場合に役立ちます。

[`use cache`](/docs/app/api-reference/directives/use-cache)と組み合わせて使用することが期待されています。これにより、特定のアプリケーション部分をページ、関数、またはコンポーネントレベルで`use cache`を使用してキャッシュするように定義しない限り、データフェッチがデフォルトでランタイムに行われるようになります。

## Usage {#usage}

`dynamicIO`フラグを有効にするには、`next.config.ts`ファイルの`experimental`セクションで`true`に設定します：

```ts title="next.config.ts"
import type { NextConfig } from 'next'

// dynamicIOフラグを有効にする
const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
  },
}

export default nextConfig
```

`dynamicIO`が有効になっているときは、次のキャッシュ機能および設定を使用できます：

- [`use cache`ディレクティブ](/docs/app/api-reference/directives/use-cache)
- `use cache`との[`cacheLife`関数](/docs/app/api-reference/next-config-js/cacheLife)
- [`cacheTag`関数](/docs/app/api-reference/functions/cacheTag)

## Notes {#notes}

- `dynamicIO`は、ランタイムでの新鮮なデータフェッチを保証することでパフォーマンスの最適化に役立ちますが、事前レンダリングされたコンテンツを提供する場合と比べて追加の遅延が発生する可能性があります；
