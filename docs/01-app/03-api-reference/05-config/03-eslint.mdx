Next.js は ESLint プラグイン、[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next) を提供しており、これは Next.js アプリケーション内で共通の問題や不具合を発見することができるベース設定内にすでに組み込まれています。

## 参照 {#reference}

以下の ESLint プラグインの推奨ルールセットはすべて `eslint-config-next` 内で使用されています：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これは `next.config.js` の設定よりも優先されます。

## 例 {#examples}

### カスタムディレクトリとファイルのリント {#linting-custom-directories-and-files}

デフォルトでは、Next.js は `pages/`、`app/`、`components/`、`lib/`、および `src/` ディレクトリ内のすべてのファイルに対して ESLint を実行します。ただし、`next.config.js` で `eslint` コンフィグの `dirs` オプションを使用して、どのディレクトリを指定するかを決定できます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // 本番ビルド中（next build）に 'pages' と 'utils' ディレクトリのみで ESLint を実行
  },
}
```

同様に、特定のディレクトリやファイルに対して `next lint` コマンドを使用するためには、`--dir` 及び `--file` フラグを使用できます：

```bash title="ターミナル"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内の root ディレクトリの指定 {#specifying-a-root-directory-within-a-monorepo}

もし `eslint-plugin-next` を Next.js が root ディレクトリにインストールされていないプロジェクト（モノレポなど）で使用する場合、`.eslintrc` の `settings` プロパティを使用して Next.js アプリケーションの場所を指定できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  }),
]

export default eslintConfig
```

`rootDir` はパス（相対または絶対）、グロブ（例：`"packages/*/"`）またはパスおよび/またはグロブの配列にすることができます。

### キャッシュの無効化 {#disabling-the-cache}

パフォーマンスを向上させるために、ESLint によって処理されたファイルの情報はデフォルトでキャッシュされます。これは `.next/cache` または定義された[ビルドディレクトリ](/docs/app/api-reference/config/next-config-js/distDir)に保存されます。ソースファイルの内容以外に依存する ESLint ルールを含み、キャッシュを無効にする必要がある場合は、`next lint` に `--no-cache` フラグを使用します。

```bash title="ターミナル"
next lint --no-cache
```

### ルールの無効化 {#disabling-rules}

対応するプラグイン（`react`、`react-hooks`、`next`）が提供するルールを変更または無効にしたい場合、`.eslintrc` の `rules` プロパティを使用して直接変更できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  }),
]

export default eslintConfig
```

### Core Web Vitals と一緒に使用する {#with-core-web-vitals}

`next/core-web-vitals` ルールセットは、`next lint` が初めて実行され、**strict** オプションが選択されたときに有効になります。

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals'],
  }),
]

export default eslintConfig
```

`next/core-web-vitals` は [Core Web Vitals](https://web.dev/vitals/) に影響する場合、`eslint-plugin-next` を更新して多数のデフォルトの警告ルールにエラーを出力します。

> [Create Next App](/docs/app/api-reference/cli/create-next-app) で構築された新しいアプリケーションには、`next/core-web-vitals` のエントリポイントが自動的に含まれます。

### TypeScript と一緒に使用する {#with-typescript}

Next.js ESLint ルールに加えて、`create-next-app --typescript` は `next/typescript` とともに TypeScript 固有のリントルールを設定に追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals', 'next/typescript'],
  }),
]

export default eslintConfig
```

これらのルールは [`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended) に基づいています。詳細は [typescript-eslint > Configs](https://typescript-eslint.io/linting/configs) を参照してください。

### Prettier と一緒に使用する {#with-prettier}

ESLint にもコードフォーマットルールが含まれており、既存の [Prettier](https://prettier.io/) の設定と競合する可能性があります。ESLint と Prettier を一緒に使うために、[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier) を ESLint 設定に追加することをお勧めします。

最初に依存関係をインストールします：

```bash title="ターミナル"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

その後、既存の ESLint 設定に `prettier` を追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next', 'prettier'],
  }),
]

export default eslintConfig
```

### ステージングされたファイルでのリントの実行 {#running-lint-on-staged-files}

ステージングされた git ファイルでリントを実行するために [lint-staged](https://github.com/okonet/lint-staged) と `next lint` を使用する場合、以下をプロジェクトの root にある `.lintstagedrc.js` ファイルに追加して `--file` フラグを使用するように指定する必要があります。

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## 本番ビルド中のリントを無効化する {#disabling-linting-during-production-builds}

`next build` 中に ESLint を実行したくない場合は、`next.config.js` で `eslint.ignoreDuringBuilds` オプションを `true` に設定できます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  eslint: {
    // 警告：これにより、プロジェクトに ESLint エラーがある場合でも本番ビルドが正常に完了します。
    ignoreDuringBuilds: true,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
const nextConfig = {
  eslint: {
    // 警告：これにより、プロジェクトに ESLint エラーがある場合でも本番ビルドが正常に完了します。
    ignoreDuringBuilds: true,
  },
}

export default nextConfig
```

### 既存のコンフィグの移行 {#migrating-existing-config}

アプリケーションで ESLint をすでに設定している場合、いくつかの条件が満たされない限り、`eslint-config-next` を含めるのではなく、このプラグインから直接拡張することをお勧めします。

#### 推奨されるプラグインルールセット {#recommended-plugin-ruleset}

次の条件が真である場合：

- 以下のプラグインの1つ以上をすでにインストールしている（`airbnb` や `react-app` などの他の設定を通じてでも）：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- Babel が Next.js 内で設定されている方法と異なる特定の `parserOptions` を定義している（[Babel の設定をカスタマイズしている](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)場合を除き、これは推奨されません）
- Node.js または TypeScript [リゾルバ](https://github.com/benmosher/eslint-plugin-import#resolvers)をインポート処理用に定義して `eslint-plugin-import` をインストールしている

この場合、[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js) 内でこれらのプロパティが設定されている方法を好む場合はこれらの設定を削除するか、代わりに Next.js ESLint プラグインから直接拡張することをお勧めします：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

このプラグインは `next lint` を実行することなく通常通りプロジェクト内にインストールできます：

```bash title="ターミナル"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これにより、同じプラグインやパーサを複数の設定でインポートすることによる衝突やエラーのリスクが排除されます。

#### 追加の設定 {#additional-configurations}

`eslint-config-next` を含めたい場合は、別の ESLint 設定を使用する場合、他の設定の後で最後に拡張されるようにしてください。例えば：

```mjs title="eslint.config.mjs"
import js from '@eslint/js'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
  recommendedConfig: js.configs.recommended,
})

const eslintConfig = [
  ...compat.config({
    extends: ['eslint:recommended', 'next'],
  }),
]

export default eslintConfig
```

`next` 設定はすでに `parser`、`plugins`、および `settings` プロパティにデフォルト値を設定しているため、これらのプロパティを手動で再宣言する必要はありません。別の構成が必要な場合を除いて。

他の共有可能な設定を含む場合は、**これらのプロパティが上書きまたは変更されないようにする必要があります**。そうでない場合、上述のように `next` 構成と共通の振る舞いを共有する構成を削除するか、Next.js ESLint プラグインから直接拡張することをお勧めします。
