Next.jsは、基本設定に組み込まれたESLintプラグイン、[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)を提供し、Next.jsアプリケーションでよくある問題や課題を検出することが可能です。

## 参照 {#reference}

以下のESLintプラグインからの推奨ルールセットはすべて`eslint-config-next`内で使用されています：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これは`next.config.js`の設定よりも優先されます。

## 例 {#examples}

### カスタムディレクトリとファイルのリント {#linting-custom-directories-and-files}

デフォルトでは、Next.jsは`pages/`、`app/`、`components/`、`lib/`、および`src/`ディレクトリ内のすべてのファイルに対してESLintを実行します。しかし、プロダクションビルドのために`next.config.js`の`eslint`設定で`dirs`オプションを使用して特定のディレクトリを指定できます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // プロダクションビルド中にESLintを'pages'と'utils'ディレクトリでのみ実行します（next build）
  },
}
```

同様に、`next lint`のために`--dir`および`--file`フラグを使用して、特定のディレクトリとファイルをリントすることができます：

```bash title="ターミナル"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内のrootディレクトリの指定 {#specifying-a-root-directory-within-a-monorepo}

Next.jsがrootディレクトリにインストールされていないプロジェクト（例えばモノレポ）の場合、`.eslintrc`の`settings`プロパティを使用して`eslint-plugin-next`にNext.jsアプリケーションの場所を指定できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  }),
]

export default eslintConfig
```

`rootDir`はパス（相対または絶対）、グロブ（例："packages/\*/"）、またはパスおよび/またはグロブの配列であることができます。

### キャッシュの無効化 {#disabling-the-cache}

パフォーマンスを向上させるため、ESLintによって処理されたファイルの情報はデフォルトでキャッシュされます。これは`.next/cache`または定義された[ビルドディレクトリ](/docs/app/api-reference/config/next-config-js/distDir)に格納されます。単一のソースファイルの内容以上のものに依存するESLintルールを含め、キャッシュを無効にする必要がある場合は、`next lint`で`--no-cache`フラグを使用します。

```bash title="ターミナル"
next lint --no-cache
```

### ルールの無効化 {#disabling-rules}

サポートされているプラグイン（`react`、`react-hooks`、`next`）で提供されたルールのいずれかを変更または無効にしたい場合は、`.eslintrc`の`rules`プロパティを使用して直接変更できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  }),
]

export default eslintConfig
```

### Core Web Vitalsと共に {#with-core-web-vitals}

`next/core-web-vitals`ルールセットは、`next lint`が初めて実行され、**strict**オプションが選択されたときに有効になります。

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals'],
  }),
]

export default eslintConfig
```

`next/core-web-vitals`は、[Core Web Vitals](https://web.dev/vitals/)に影響を与える場合、デフォルトで警告となるいくつかのルールを`eslint-plugin-next`にエラーとして更新します。

> `next/core-web-vitals`エントリポイントは、[Create Next App](/docs/app/api-reference/cli/create-next-app)で作成された新しいアプリケーションに自動的に含まれます。

### TypeScriptと共に {#with-typescript}

Next.js ESLintルールに加えて、`create-next-app --typescript`は`next/typescript`と共にTypeScript特有のリントルールを設定に追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals', 'next/typescript'],
  }),
]

export default eslintConfig
```

これらのルールは[`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)に基づいています。
詳細については、[typescript-eslint > Configs](https://typescript-eslint.io/linting/configs)を参照してください。

### Prettierと共に {#with-prettier}

ESLintはコードフォーマットルールも含んでおり、これが既存の[Prettier](https://prettier.io/)設定と衝突する可能性があります。ESLintとPrettierを協働させるために[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier)をESLint設定に含めることをお勧めします。

まず、依存関係をインストールします：

```bash title="ターミナル"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

次に、既存のESLint設定に`prettier`を追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next', 'prettier'],
  }),
]

export default eslintConfig
```

### ステージングされたファイルでのリントの実行 {#running-lint-on-staged-files}

ステージングされたgitファイルでリンターを実行するために[lint-staged](https://github.com/okonet/lint-staged)と`next lint`を使用したい場合は、`.lintstagedrc.js`ファイルをプロジェクトのrootに追加し、`--file`フラグの使用を指定する必要があります。

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

### 既存の設定の移行 {#migrating-existing-config}

すでにアプリケーションでESLintを設定している場合、いくつかの条件が満たされない限り`eslint-config-next`を含めるのではなく、このプラグインから直接拡張することをお勧めします。

#### 推奨プラグインルールセット {#recommended-plugin-ruleset}

次の条件が真である場合：

- 以下のプラグインのいずれかを既にインストールしている（一部または別の設定を通じて`airbnb`や`react-app`など）：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- BabelがNext.js内でどのように設定されているかとは異なる特定の`parserOptions`を定義している（これを行うには、[Babel設定をカスタマイズした場合](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)以外では推奨されません）
- `eslint-plugin-import`をNode.jsおよび/またはTypeScriptの[リゾルバ](https://github.com/benmosher/eslint-plugin-import#resolvers)と共にインストールして、インポートを処理している

これらのプロパティを[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js)内でどのように構成されているかに合わせたい場合は、これらの設定を削除するか、Next.js ESLintプラグインから直接拡張することをお勧めします：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

プラグインは`next lint`を実行することなく通常通りプロジェクトにインストールできます：

```bash title="ターミナル"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これにより、複数の設定間で同じプラグインまたはパーサーをインポートしたことによる衝突やエラーのリスクがなくなります。

#### 追加設定 {#additional-configurations}

すでに別のESLint設定を使用しており、`eslint-config-next`を含めたい場合は、他の設定の後に最後に拡張されるようにしてください。例えば：

```mjs title="eslint.config.mjs"
import js from '@eslint/js'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
  recommendedConfig: js.configs.recommended,
})

const eslintConfig = [
  ...compat.config({
    extends: ['eslint:recommended', 'next'],
  }),
]

export default eslintConfig
```

`next`構成はすでに`parser`、`plugins`、および`settings`プロパティのデフォルト値を設定しています。これらのプロパティを手動で再宣言する必要はありません。ただし、使用目的に応じて異なる設定が必要な場合を除きます。他の共有可能な設定を含める場合、このプロパティが上書きまたは変更されないようにする必要があります。さもなければ、上記のようにNext.js ESLintプラグインから直接拡張することをお勧めします。
