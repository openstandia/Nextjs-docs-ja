Next.jsはESLintプラグイン、[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)を基本設定内にバンドルして提供しており、Next.jsアプリケーションで共通の問題をキャッチすることができます。

## リファレンス {#reference}

以下のESLintプラグインの推奨ルールセットはすべて`eslint-config-next`に含まれています：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これは`next.config.js`の設定より優先されます。

## 例 {#examples}

### カスタムディレクトリとファイルのLint {#linting-custom-directories-and-files}

デフォルトでは、Next.jsは`pages/`、`app/`、`components/`、`lib/`、および`src/`ディレクトリ内のすべてのファイルに対してESLintを実行します。ただし、プロダクションビルド用に`next.config.js`の`eslint`設定内で`dirs`オプションを使用して、どのディレクトリを対象とするかを指定できます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // プロダクションビルド（next build）時に'pages'と'utils'ディレクトリでのみESLintを実行
  },
}
```

同様に、特定のディレクトリとファイルに対して`next lint`を実行するために`--dir`または`--file`フラグを使用できます：

```bash title="Terminal"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内のrootディレクトリを指定 {#specifying-a-root-directory-within-a-monorepo}

`eslint-plugin-next`を使用しているプロジェクトで、Next.jsがrootディレクトリにインストールされていない（モノレポのような）場合、`.eslintrc`内の`settings`プロパティを使用して、Next.jsアプリケーションの場所を`eslint-plugin-next`に指示できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  }),
]

export default eslintConfig
```

`rootDir`は、パス（相対または絶対）、グロブ（例えば`"packages/*/"`）またはパスとグロブの配列を指定できます。

### キャッシュを無効化 {#disabling-the-cache}

パフォーマンスを向上させるために、ESLintで処理されたファイルの情報はデフォルトでキャッシュされます。これは`.next/cache`もしくはあなたが定義した[ビルドディレクトリ](/docs/app/api-reference/config/next-config-js/distDir)に保存されます。1つのソースファイルのコンテンツ以上に依存するESLintルールを含め、キャッシュを無効化する必要がある場合は、`next lint`とともに`--no-cache`フラグを使用します：

```bash title="Terminal"
next lint --no-cache
```

### ルールの無効化 {#disabling-rules}

サポートされているプラグイン（`react`、`react-hooks`、`next`）によって提供されるルールを変更または無効化したい場合、`.eslintrc`内の`rules`プロパティを使用して直接変更できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  }),
]

export default eslintConfig
```

### Core Web Vitalsとともに {#with-core-web-vitals}

`next/core-web-vitals`ルールセットは`next lint`が初めて実行され、**厳しい**オプションが選択されたときに有効になります：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals'],
  }),
]

export default eslintConfig
```

`next/core-web-vitals`は[Core Web Vitals](https://web.dev/vitals/)に影響を与える場合、デフォルトで警告である多くのルールをエラーに設定するように`eslint-plugin-next`を更新します。

> `next/core-web-vitals`エントリポイントは[Create Next App](/docs/app/api-reference/cli/create-next-app)で新しく作成されたアプリケーションに自動的に含まれます。

### TypeScriptとともに {#with-typescript}

Next.jsのESLintルールに加えて、`create-next-app --typescript`は`next/typescript`と共に、TypeScript特有のリントルールもあなたの設定に追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals', 'next/typescript'],
  }),
]

export default eslintConfig
```

これらのルールは[`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)に基づいています。
詳細は [typescript-eslint > Configs](https://typescript-eslint.io/linting/configs)を参照してください。

### Prettierとともに {#with-prettier}

ESLintにはコードフォーマットのルールも含まれており、既存の[Prettier](https://prettier.io/)セットアップと競合する可能性があります。ESLintとPrettierを共に機能させるために、[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier)をESLintの設定に含めることを推奨します。

まず、依存関係をインストール：

```bash title="Terminal"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

次に、既存のESLint設定に`prettier`を追加：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next', 'prettier'],
  }),
]

export default eslintConfig
```

### ステージングされたファイルに対するLintの実行 {#running-lint-on-staged-files}

[lint-staged](https://github.com/okonet/lint-staged)とともに`next lint`を使って、ステージングされたgitファイル上でリンターを実行したい場合、`--file`フラグの使用を指定するために、プロジェクトのrootにある`.lintstagedrc.js`ファイルに次のコードを追加します：

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## プロダクションビルド中のLintの無効化 {#disabling-linting-during-production-builds}

  eslint: {
    // 警告: これにより、プロジェクトにESLintエラーがあってもプロダクションビルドを完了できるようになります。

  },

### 既存の設定の移行 {#migrating-existing-config}

すでにアプリケーションでESLintを設定している場合、いくつかの条件を満たさない限り、`eslint-config-next`を含めるのではなくこのプラグインを直接拡張することをお勧めします。

#### 推奨されるプラグインルールセット {#recommended-plugin-ruleset}

次の条件が真である場合：

- 次のプラグインの1つ以上がすでにインストールされている（別々または`airbnb`や`react-app`などの他の設定を通じて）：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- Next.js内でBabelが設定されている方法と異なる特定の`parserOptions`を定義している（[Babel設定をカスタマイズしていない限り](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)推奨されません）
- インポートを処理するために`eslint-plugin-import`がNode.jsやTypeScriptの[リゾルバ](https://github.com/benmosher/eslint-plugin-import#resolvers)とともにインストールされている

これらの条件が満たされている場合、[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js)内でこれらのプロパティが設定されている方法が好ましい場合、これらの設定を削除するか、代わりにNext.js ESLintプラグインから直接拡張することをお勧めします：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

プラグインは通常、`next lint`を実行することなくプロジェクトにインストールできます：

```bash title="Terminal"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これにより、複数の設定で同じプラグインやパーサーをインポートすることによる衝突やエラーが発生するリスクが排除されます。

#### 追加の設定 {#additional-configurations}

すでに別のESLint設定を使用していて`eslint-config-next`を含めたい場合、他の設定の後に最後に拡張されていることを確認してください。例として：

```mjs title="eslint.config.mjs"
import js from '@eslint/js'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能
  baseDirectory: import.meta.dirname,
  recommendedConfig: js.configs.recommended,
})

const eslintConfig = [
  ...compat.config({
    extends: ['eslint:recommended', 'next'],
  }),
]

export default eslintConfig
```

`next`設定はすでに`parser`、`plugins`、および`settings`プロパティのデフォルト値の設定を処理しています。これらのプロパティを手動で再宣言する必要はありませんが、特定のユースケースのために異なる設定が必要な場合は別です。共有可能な設定を含める場合、**これらのプロパティが上書きまたは変更されないようにする必要があります。**それ以外の場合、`next`設定と挙動を共有する設定を削除するか、上述の通り、Next.js ESLintプラグインから直接拡張することを推奨します。
