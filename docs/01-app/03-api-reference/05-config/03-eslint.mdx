Next.jsは、Next.jsアプリケーション内の一般的な問題をキャッチすることを可能にするESLintプラグイン、[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)を基礎設定に既にバンドルして提供しています。

## 参照 {#reference}

以下のESLintプラグインからの推奨ルールセットはすべて`eslint-config-next`内で使用されています：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これは`next.config.js`からの設定よりも優先されます。

## 例 {#examples}

### カスタムディレクトリとファイルのリンティング {#linting-custom-directories-and-files}

デフォルトでは、Next.jsは`pages/`、`app/`、`components/`、`lib/`、および`src/`ディレクトリ内のすべてのファイルに対してESLintを実行します。しかし、プロダクションビルド用に`next.config.js`の`eslint`設定で`dirs`オプションを使用してディレクトリを指定できます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // プロダクションビルド（next build）時に 'pages' と 'utils' ディレクトリでのみESLintを実行します
  },
}
```

同様に、`next lint`に`--dir`と`--file`フラグを使用して特定のディレクトリとファイルをリンティングすることができます：

```bash title="Terminal"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内のrootディレクトリの指定 {#specifying-a-root-directory-within-a-monorepo}

Next.jsがrootディレクトリ内にインストールされていないプロジェクト（例：モノレポ）で`eslint-plugin-next`を使用している場合は、`.eslintrc`の`settings`プロパティを使用して`eslint-plugin-next`にNext.jsアプリケーションをどこで見つけるかを教えることができます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  }),
]

export default eslintConfig
```

`rootDir`はパス（相対または絶対）、グロブ（例："packages/\*/"）、またはパスおよび/またはグロブの配列を指定できます。

### キャッシュの無効化 {#disabling-the-cache}

パフォーマンスを向上するために、ESLintによって処理されたファイルの情報はデフォルトでキャッシュされます。これは`.next/cache`または定義された[ビルドディレクトリ](/docs/app/api-reference/config/next-config-js/distDir)に保存されます。単一のソースファイルの内容以上に依存するESLintルールを含む場合、キャッシュを無効にするには、`next lint`で`--no-cache`フラグを使用します：

```bash title="Terminal"
next lint --no-cache
```

### ルールの無効化 {#disabling-rules}

サポートされているプラグイン（`react` 、`react-hooks` 、`next` ）が提供するルールを変更または無効化したい場合は、`.eslintrc`の`rules`プロパティを使用して直接変更できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  }),
]

export default eslintConfig
```

### Core Web Vitalsと共に {#with-core-web-vitals}

初めて`next lint`を実行した際に**strict**オプションを選択すると、`next/core-web-vitals`ルールセットが有効になります：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals'],
  }),
]

export default eslintConfig
```

`next/core-web-vitals`は、[Core Web Vitals](https://web.dev/vitals/)に影響を与えるためデフォルトで警告されるいくつかのルールに対して`eslint-plugin-next`をエラーに更新します。

> `next/core-web-vitals`のエントリーポイントは、[Create Next App](/docs/app/api-reference/cli/create-next-app)を使用して構築された新しいアプリケーションに自動的に含まれます。

### TypeScriptと共に {#with-typescript}

Next.js ESLintのルールに加えて、`create-next-app --typescript`はTypeScript固有のリントルールも`next/typescript`と共に設定に追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals', 'next/typescript'],
  }),
]

export default eslintConfig
```

これらのルールは、[`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)に基づいています。詳細については、[typescript-eslint > Configs](https://typescript-eslint.io/linting/configs)を参照してください。

### Prettierと共に {#with-prettier}

ESLintはコードフォーマットのルールも含んでおり、既存の[Prettier](https://prettier.io/)の設定と競合する可能性があります。ESLintとPrettierを共存させるために、[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier)をESLintの設定に含めることをお勧めします。

まず、依存関係をインストールします：

```bash title="Terminal"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

次に、既存のESLint設定に`prettier`を追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next', 'prettier'],
  }),
]

export default eslintConfig
```

### ステージングされたファイルでのリントの実行 {#running-lint-on-staged-files}

ステージングされたgitファイルでリンターを実行するために[lint-staged](https://github.com/okonet/lint-staged)で`next lint`を使用したい場合、`--file`フラグの使用を指定するために、プロジェクトのrootにある`.lintstagedrc.js`ファイルに以下を追加する必要があります：

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## プロダクションビルド中のリンティングの無効化 {#disabling-linting-during-production-builds}

`next build`中にESLintを実行したくない場合、 `next.config.js`で`eslint.ignoreDuringBuilds`オプションを`true`に設定できます：

const nextConfig: NextConfig = {
  eslint: {
    // 警告: これにより、プロダクションビルドが成功するために
    // プロジェクトにESLintエラーがあっても完了します。
    ignoreDuringBuilds: true,
  },

### 既存の設定の移行 {#migrating-existing-config}

既にアプリケーションでESLintが設定されている場合、いくつかの条件が満たされない限り、`eslint-config-next`を含めるのではなく、このプラグインから直接拡張することをお勧めします。

#### 推奨プラグインルールセット {#recommended-plugin-ruleset}

以下の条件が真である場合：

- `react`、`react-hooks`、`jsx-a11y`、`import`のいずれかのプラグインを既にインストールしている（別個に、または`airbnb`や`react-app`などの異なる設定を通して）
- BabelがNext.js内で設定されている方法と異なる特定の`parserOptions`を定義している（[Babelの設定をカスタマイズしている場合](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)以外は推奨されません）
- `eslint-plugin-import`をNode.jsおよび/またはTypeScriptの[リゾルバー](https://github.com/benmosher/eslint-plugin-import#resolvers)をインストールしてインポートを処理している

設定の構成が[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js)でどのように構成されているかを好む場合、これらの設定を削除するか、代わりにNext.js ESLintプラグインから直接拡張することをお勧めします：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

プラグインは`next lint`を実行する必要なく、通常どおりプロジェクトにインストールできます：

```bash title="Terminal"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これにより、複数の設定間で同じプラグインまたはパーサーをインポートすることによる衝突やエラーのリスクを排除します。

#### 追加の設定 {#additional-configurations}

既に別のESLint設定を使用していて、`eslint-config-next`を含めたい場合は、他の設定の後に最後に拡張されるようにする必要があります。例えば：

```mjs title="eslint.config.mjs"
import js from '@eslint/js'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
  recommendedConfig: js.configs.recommended,
})

const eslintConfig = [
  ...compat.config({
    extends: ['eslint:recommended', 'next'],
  }),
]

export default eslintConfig
```

`next`設定は、既に`parser`、`plugins`、および`settings`プロパティのデフォルト値を設定しています。これらのプロパティを異なる設定のために手動で再宣言する必要はありません。

他の共有設定を含める場合、これらのプロパティが上書きまたは変更されないことを確認する必要があります。それ以外の場合は、上記で述べたように、`next`設定と同じ動作を共有する設定を削除するか、Next.js ESLintプラグインから直接拡張することをお勧めします。
