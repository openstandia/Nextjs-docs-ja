Next.js は ESLint プラグイン [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next) を提供しており、基本設定にすでにバンドルされているため、Next.js アプリケーションの一般的な問題を検出することができます。

## リファレンス {#reference}

以下の ESLint プラグインから推奨されるルールセットが、すべて `eslint-config-next` 内で使用されています:

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これは、`next.config.js` からの設定よりも優先されます。

## 例 {#examples}

### カスタムディレクトリとファイルのリント {#linting-custom-directories-and-files}

デフォルトでは、Next.js は `pages/`、`app/`、`components/`、`lib/`、`src/` ディレクトリ内のすべてのファイルに対して ESLint を実行します。ただし、本番ビルドのための `next.config.js` 内の `eslint` 設定で `dirs` オプションを使用して、どのディレクトリを対象にするかを指定できます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // 本番ビルド（next build）中に 'pages' と 'utils' ディレクトリのみで ESLint を実行します
  },
}
```

同様に、`next lint` に対して `--dir` および `--file` フラグを使用して特定のディレクトリとファイルをリントすることもできます：

```bash title="Terminal"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内での root ディレクトリの指定 {#specifying-a-root-directory-within-a-monorepo}

モノレポのように、プロジェクト内で Next.js がインストールされているディレクトリが root ではない場合、`.eslintrc` の `settings` プロパティを使用して、Next.js アプリケーションの場所を `eslint-plugin-next` に教えることができます：

```json title=".eslintrc.json"
{
  "extends": "next",
  "settings": {
    "next": {
      "rootDir": "packages/my-app/"
    }
  }
}
```

`rootDir` は、パス（相対または絶対）、グロブ（例: `"packages/*/"`）、またはパスと/またはグロブの配列にすることができます。

### キャッシュの無効化 {#disabling-the-cache}

パフォーマンスを向上させるために、ESLint が処理したファイルの情報はデフォルトでキャッシュされます。これは `.next/cache` または定義された[ビルドディレクトリ](/docs/app/api-reference/config/next-config-js/distDir)に保存されます。複数のソースファイルの内容以上を依存する ESLint ルールを含めた場合、キャッシュを無効にする必要がある場合は、`next lint` に `--no-cache` フラグを使用します。

```bash title="Terminal"
next lint --no-cache
```

### ルールの無効化 {#disabling-rules}

サポートされているプラグイン（`react`、`react-hooks`、`next`）によって提供されるルールを変更または無効にしたい場合は、`.eslintrc` の `rules` プロパティを使用して直接変更できます：

```json title=".eslintrc.json"
{
  "extends": "next",
  "rules": {
    "react/no-unescaped-entities": "off",
    "@next/next/no-page-custom-font": "off"
  }
}
```

### Core Web Vitals とともに {#with-core-web-vitals}

`next/core-web-vitals` ルールセットは `next lint` が初めて実行されたときに有効になり、**strict** オプションが選択されます。

```json title=".eslintrc.json"
{
  "extends": "next/core-web-vitals"
}
```

`next/core-web-vitals` は、[Core Web Vitals](https://web.dev/vitals/) に影響を与えるいくつかのルールでデフォルトでは警告であるものをエラーに更新します。

> `next/core-web-vitals` のエントリーポイントは、[Create Next App](/docs/app/api-reference/cli/create-next-app) で作成された新しいアプリケーションに自動的に含まれます。

### TypeScript とともに {#with-typescript}

Next.js の ESLint ルールに加えて、`create-next-app --typescript` は設定に TypeScript 固有のリントルールも追加します：

```json title=".eslintrc.json"
{
  "extends": ["next/core-web-vitals", "next/typescript"]
}
```

これらのルールは [`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended) に基づいています。詳細は [typescript-eslint > Configs](https://typescript-eslint.io/linting/configs) を参照してください。

### Prettier とともに {#with-prettier}

ESLint はコードのフォーマットルールも含んでいるため、既存の [Prettier](https://prettier.io/) の設定と衝突することがあります。ESLint と Prettier が一緒に動作するようにするには、[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier) を ESLint 設定に含めることをお勧めします。

まず、依存関係をインストールします：

```bash title="Terminal"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

次に、既存の ESLint 設定に `prettier` を追加します：

```json title=".eslintrc.json"
{
  "extends": ["next", "prettier"]
}
```

### ステージングされたファイルでのリントの実行 {#running-lint-on-staged-files}

ステージングされた git ファイルに対して linter を実行するために [lint-staged](https://github.com/okonet/lint-staged) とともに `next lint` を使用したい場合、`--file` フラグの使用を指定するためにプロジェクトの root にある `.lintstagedrc.js` ファイルに以下を追加する必要があります。

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## 本番ビルド中のリントの無効化 {#disabling-linting-during-production-builds}

`next build` 中に ESLint を実行したくない場合、`next.config.js` の `eslint.ignoreDuringBuilds` オプションを `true` に設定できます：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  eslint: {
    // 警告: この設定により、プロジェクトに ESLint エラーがある場合でも本番ビルドは正常に完了します。
    ignoreDuringBuilds: true,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
const nextConfig = {
  eslint: {
    // 警告: この設定により、プロジェクトに ESLint エラーがある場合でも本番ビルドは正常に完了します。
    ignoreDuringBuilds: true,
  },
}

export default nextConfig
```

</TabItem>
</Tabs>

### 既存の設定の移行 {#migrating-existing-config}

すでにアプリケーションに ESLint を設定している場合、`eslint-config-next` を含むより直接的にこのプラグインから拡張することをお勧めします。

#### 推奨プラグインルールセット {#recommended-plugin-ruleset}

以下の条件が真である場合:

- 以下のいずれか、または複数のプラグインがすでにインストールされている（`airbnb` や `react-app` などの他の設定を通じてもインストールされている場合も含む）:
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- Babel が Next.js で設定されている方法とは異なる特定の `parserOptions` を定義している（これは推奨されません [Babel の設定をカスタマイズした場合](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel) を除く）
- Node.js および/または TypeScript の [リゾルバ](https://github.com/benmosher/eslint-plugin-import#resolvers) が定義されている `eslint-plugin-import` をインストールしている

これらの場合は、[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js) 内でこれらのプロパティがどのように設定されているかを好む場合、これらの設定を削除するか、Next.js の ESLint プラグインから直接拡張することをお勧めします：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

`next lint` を実行する必要なく、プラグインをプロジェクトに通常通りインストールすることができます：

```bash title="Terminal"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これは、複数の設定間で同じプラグインやパーサーをインポートすることによって発生する可能性のある衝突やエラーのリスクを排除します。

#### 追加設定 {#additional-configurations}

すでに別の ESLint 設定を使用していて、`eslint-config-next` を含めたい場合、他の設定の後に最後に拡張されることを確認してください。例えば：

```json title=".eslintrc.json"
{
  "extends": ["eslint:recommended", "next"]
}
```

`next` 設定は、`parser`、`plugins`、`settings` プロパティのデフォルト値を設定することをすでに処理しています。これらのプロパティを別途宣言する必要はありませんが、別の設定が必要な場合を除きます。

他の共有設定を含める場合、**これらのプロパティが上書きされたり変更されたりしないようにする必要があります**。それ以外の場合、上で述べたように Next.js の ESLint プラグインから直接拡張するか、`next` 設定と行動を共有する設定を削除することをお勧めします。
