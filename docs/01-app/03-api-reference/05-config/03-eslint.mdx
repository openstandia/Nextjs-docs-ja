Next.jsはESLintプラグイン[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)を提供しており、これがベースの設定に既にバンドルされています。このプラグインを使用することで、Next.jsアプリケーションにおける一般的な問題をキャッチすることが可能です。

## 参考資料 {#reference}

以下のESLintプラグインから推奨されるルールセットがすべて`eslint-config-next`内で使用されています：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これにより、`next.config.js`の設定よりも優先されます。

## 例 {#examples}

### カスタムディレクトリとファイルのリンティング {#linting-custom-directories-and-files}

デフォルトでは、Next.jsは`pages/`、`app/`、`components/`、`lib/`、`src/`ディレクトリ内のすべてのファイルに対してESLintを実行します。しかし、`next.config.js`の`eslint`設定で`dirs`オプションを使用して、どのディレクトリを対象にするかを指定することができます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // 本番ビルド時に'pages'と'utils'ディレクトリでのみESLintを実行する（next build）
  },
}
```

同様に、`--dir`および`--file`フラグを使用して特定のディレクトリとファイルを`next lint`でリンティングすることができます：

```bash title="Terminal"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内でrootディレクトリを指定する {#specifying-a-root-directory-within-a-monorepo}

もしあなたがNext.jsがrootディレクトリにインストールされていないプロジェクト（モノレポなど）で`eslint-plugin-next`を使用している場合、`.eslintrc`で`settings`プロパティを使って`eslint-plugin-next`にNext.jsアプリケーションの場所を知らせることができます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  }),
]

export default eslintConfig
```

`rootDir`はパス（相対または絶対）、グロブ（例："packages/\*/"）、またはパスと/あるいはグロブの配列にできます。

### キャッシュを無効にする {#disabling-the-cache}

パフォーマンスを向上させるため、ESLintによって処理されたファイルの情報はデフォルトでキャッシュされます。これは`.next/cache`またはあなたが定義した[ビルドディレクトリ](/docs/app/api-reference/config/next-config-js/distDir)に保存されます。もし、単一のソースファイルの内容以上に依存するESLintルールが含まれており、キャッシュを無効にする必要があれば、`next lint`で`--no-cache`フラグを使用してください。

```bash title="Terminal"
next lint --no-cache
```

### ルールを無効にする {#disabling-rules}

サポートされているプラグイン（`react`、`react-hooks`、`next`）によって提供されるルールのいずれかを変更または無効にしたい場合、`.eslintrc`の`rules`プロパティを使用してそれらを直接変更できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  }),
]

export default eslintConfig
```

### Core Web Vitalsと共に {#with-core-web-vitals}

`next/core-web-vitals`ルールセットは、`next lint`が初めて実行され、**厳格（strict）**オプションが選択されたときに有効になります。

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals'],
  }),
]

export default eslintConfig
```

`next/core-web-vitals`は[Core Web Vitals](https://web.dev/vitals/)に影響を与える場合、デフォルトでは警告とされるいくつかのルールで`eslint-plugin-next`をエラーに更新します。

> `next/core-web-vitals`エントリポイントは、[Create Next App](/docs/app/api-reference/cli/create-next-app)で作成された新しいアプリケーションに自動的に含まれます。

### TypeScriptと共に {#with-typescript}

Next.js ESLintルールに加えて、`create-next-app --typescript`はTypeScript用のリントルールを`next/typescript`を使用して設定に追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals', 'next/typescript'],
  }),
]

export default eslintConfig
```

これらのルールは[`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)に基づいています。詳細については、[typescript-eslint > Configs](https://typescript-eslint.io/linting/configs)を参照してください。

### Prettierと共に {#with-prettier}

ESLintはコードフォーマッティングルールも含んでおり、既存の[Prettier](https://prettier.io/)セットアップと矛盾する可能性があります。ESLintとPrettierが一緒に動作するようにするために、[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier)をESLint設定に含めることをお勧めします。

まず、次の依存関係をインストールします：

```bash title="Terminal"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

次に、`prettier`を既存のESLint構成に追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next', 'prettier'],
  }),
]

export default eslintConfig
```

### ステージされたファイルでリントを実行する {#running-lint-on-staged-files}

ステージングされたgitファイルでリンターを実行するために`lint-staged`と共に`next lint`を使用したい場合、`.lintstagedrc.js`ファイルに以下を追加して`--file`フラグの使用を指定する必要があります。

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## 本番ビルド中にリンティングを無効にする {#disabling-linting-during-production-builds}

`next build`中にESLintを実行したくない場合は、`next.config.js`の`eslint.ignoreDuringBuilds`オプションを`true`に設定します：

  eslint: {

### 既存の設定を移行する {#migrating-existing-config}

すでにアプリケーションでESLintが設定されている場合、`eslint-config-next`を含める代わりにこのプラグインから直接拡張することをお勧めします。ただし、いくつかの条件が満たされている場合を除きます。

#### 推奨されるプラグインルールセット {#recommended-plugin-ruleset}

以下の条件が当てはまる場合：

- 次のプラグインの1つ以上をすでに（`airbnb`や`react-app`などの別の設定を含む）インストールしている：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- 定義している特定の`parserOptions`がNext.js内でBabelが設定された方法と異なる（これは[カスタムのBabel設定](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)を行っている場合を除いて推奨されません）
- `eslint-plugin-import`をインストールしており、Node.jsおよび/またはTypeScriptの[リゾルバー](https://github.com/benmosher/eslint-plugin-import#resolvers)でインポートを処理するために定義されている

その場合、これらのプロパティが[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js)内でどのように設定されているかが好ましい場合は、これらの設定を削除することをお勧めします。または、Next.js ESLintプラグインから直接拡張してください：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

プラグインは通常の方法でプロジェクトにインストールでき、`next lint`を実行する必要はありません：

```bash title="Terminal"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これにより、複数の構成で同じプラグインやパーサーをインポートすることによる衝突やエラーのリスクが排除されます。

#### 追加の設定 {#additional-configurations}

すでに別のESLint設定を使用していて`eslint-config-next`を含めたい場合、他の設定の後に最後に拡張されるようにしてください。例えば：

```mjs title="eslint.config.mjs"
import js from '@eslint/js'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で使用可能
  baseDirectory: import.meta.dirname,
  recommendedConfig: js.configs.recommended,
})

const eslintConfig = [
  ...compat.config({
    extends: ['eslint:recommended', 'next'],
  }),
]

export default eslintConfig
```

`next`構成は、`parser`、`plugins`および`settings`プロパティのデフォルト値を設定する処理をすでに行っています。これらのプロパティを手動で再宣言する必要はありませんが、使用ケースに対して異なる設定が必要な場合は除きます。

他の共有可能な構成を含める場合は、**これらのプロパティが上書きまたは変更されないようにする必要があります**。さもなければ、`next`構成と動作を共有する構成を削除するか、前述のNext.js ESLintプラグインから直接拡張することをお勧めします。
