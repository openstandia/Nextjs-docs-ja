Next.jsは、一般的な問題や課題を検出するために、基本設定に組み込まれているESLintプラグイン、[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)を提供しています。

## 参照 {#reference}

次のESLintプラグインからの推奨ルールセットはすべて `eslint-config-next` 内で使用されます：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これは `next.config.js` の設定よりも優先されます。

## 例 {#examples}

### カスタムディレクトリとファイルのLint {#linting-custom-directories-and-files}

Next.jsはデフォルトで、`pages/`、`app/`、`components/`、`lib/`、および `src/` ディレクトリ内のすべてのファイルに対してESLintを実行します。ただし、プロダクションビルドで `next.config.js` の `eslint` コンフィグにある `dirs` オプションを使用して、特定のディレクトリを指定できます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // 本番ビルド中にESLintを 'pages' と 'utils' ディレクトリだけで実行します (next build)
  },
}
```

同様に、特定のディレクトリとファイルに対して `next lint` を行うには、`--dir` と `--file` フラグを使用できます：

```bash title="Terminal"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内のrootディレクトリの指定 {#specifying-a-root-directory-within-a-monorepo}

もし `eslint-plugin-next` を使用しており、Next.jsがrootディレクトリにインストールされていないプロジェクト（モノレポなど）で使用している場合は、`.eslintrc` の `settings` プロパティを使用して `eslint-plugin-next` にNext.jsアプリケーションの位置を教えられます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  }),
]

export default eslintConfig
```

`rootDir` には、パス（相対または絶対）、グロブ（例： `"packages/*/"`）、またはパスやグロブの配列を指定できます。

### キャッシュの無効化 {#disabling-the-cache}

パフォーマンスを向上させるために、ESLintで処理されたファイル情報はデフォルトでキャッシュされます。これは `.next/cache` に保存されているか、定義された[ビルドディレクトリー](/docs/app/api-reference/config/next-config-js/distDir)に保存されます。1つのソースファイルの内容を超えるESLintルールを含んでおり、キャッシュを無効にする必要がある場合は、`next lint` に `--no-cache` フラグを使用します：

```bash title="Terminal"
next lint --no-cache
```

### ルールの無効化 {#disabling-rules}

サポートするプラグイン（`react`、`react-hooks`、`next`）が提供するルールを変更または無効化したい場合、`.eslintrc` の `rules` プロパティを使って直接変更できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  }),
]

export default eslintConfig
```

### Core Web Vitalsと共に {#with-core-web-vitals}

`next/core-web-vitals` ルールセットは、初めて `next lint` が実行され、**strict** オプションが選択されると有効になります：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals'],
  }),
]

export default eslintConfig
```

`next/core-web-vitals` は、[Core Web Vitals](https://web.dev/vitals/)に影響する場合、デフォルトで警告となるいくつかのルールに`eslint-plugin-next`をエラーを発するように更新します。

> `next/core-web-vitals` のエントリポイントは、[Create Next App](/docs/app/api-reference/cli/create-next-app)で構築された新しいアプリケーションには自動的に含まれています。

### TypeScriptと共に {#with-typescript}

Next.js ESLintルールに加えて、`create-next-app --typescript` は、タイプスクリプト専用のlintルールも `next/typescript` で追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals', 'next/typescript'],
  }),
]

export default eslintConfig
```

これらのルールは[`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)に基づいています。詳細については、[typescript-eslint > Configs](https://typescript-eslint.io/linting/configs)を参照してください。

### Prettierと共に {#with-prettier}

ESLintにはコードのフォーマットルールも含まれていますが、これは既存の[Prettier](https://prettier.io/)設定と競合する可能性があります。ESLintとPrettierを共に使用するために[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier)をESLintの設定に含めることをおすすめします。

まず、依存関係をインストールします：

```bash title="Terminal"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

次に、既存のESLint設定に `prettier` を追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next', 'prettier'],
  }),
]

export default eslintConfig
```

### ステージされたファイルに対するLintの実行 {#running-lint-on-staged-files}

`next lint` を [lint-staged](https://github.com/okonet/lint-staged) と共に使用して、ステージされたgitファイルに対してリンターを実行したい場合、`--file` フラグの使用を指定するために、プロジェクトのrootにある `.lintstagedrc.js` ファイルに次を追加します：

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

### 既存の設定の移行 {#migrating-existing-config}

すでにアプリケーションにESLintを設定している場合、いくつかの条件が満たされない限り、`eslint-config-next` を含めるのではなく、このプラグインを直接拡張することをおすすめします。

#### 推奨されるプラグインルールセット {#recommended-plugin-ruleset}

次の条件が真である場合：

- 次のプラグインのいずれかをすでにインストールしている（`airbnb` や `react-app` などの異なる設定を通じて、または個別に）：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- BabelがNext.js内でどのように設定されているかとは異なる特定の `parserOptions` を定義している場合（[Babel設定をカスタマイズ](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)していない限り推奨されません）
- Node.jsおよび/またはTypeScriptでのインポートを処理するために定義された[解決方法](https://github.com/benmosher/eslint-plugin-import#resolvers)を持つ `eslint-plugin-import` をインストールしている

この場合、これらのプロパティが [`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js) 内でどのように構成されているかが好ましい場合、これらの設定を削除するか、Next.js ESLintプラグインから直接拡張することをおすすめします：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

プラグインは通常、`next lint` を実行せずにプロジェクトにインストールできます：

```bash title="Terminal"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これにより、複数の設定で同じプラグインまたはパーサをインポートすることによって発生する衝突やエラーのリスクが排除されます。

#### 追加の設定 {#additional-configurations}

すでに別のESLint設定を使用しており、`eslint-config-next` を含めたい場合、それが他の設定の後、最後に拡張されることを確認してください。例えば：

```mjs title="eslint.config.mjs"
import js from '@eslint/js'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirnameはNode.js v20.11.0以降で利用可能です
  baseDirectory: import.meta.dirname,
  recommendedConfig: js.configs.recommended,
})

const eslintConfig = [
  ...compat.config({
    extends: ['eslint:recommended', 'next'],
  }),
]

export default eslintConfig
```

`next` 設定は `parser`、`plugins`、および `settings` プロパティのデフォルト値を設定することをすでに処理しています。これらのプロパティを再宣言する必要はありませんが、特定のユースケースに異なる設定が必要な場合は別です。

他の共有可能な設定を含む場合は、**これらのプロパティが上書きまたは変更されないようにする必要があります**。それ以外の場合は、`next` 設定と動作を共有する設定を削除するか、上記のようにNext.js ESLintプラグインから直接拡張することをおすすめします。
