Next.jsは、[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)というESLintプラグインを提供しており、これはベース設定にすでにバンドルされており、Next.jsアプリケーションで一般的な問題を検出できるようにします。

## 参考 {#reference}

以下のESLintプラグインの推奨ルールセットはすべて`eslint-config-next`内で使用されています：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

これは、`next.config.js`の設定よりも優先されます。

## 例 {#examples}

### カスタムディレクトリとファイルのリンティング {#linting-custom-directories-and-files}

デフォルトで、Next.js は `pages/`、`app/`、`components/`、`lib/`、`src/` ディレクトリ内のすべてのファイルに対して ESLint を実行します。ただし、`next.config.js` の `eslint` 設定で `dirs` オプションを使用して、どのディレクトリを設定するか指定できます：

```js title="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // 本番ビルド時に'pages'と'utils'ディレクトリに対してのみESLintを実行します（next build）
  },
}
```

同様に、`next lint` に対して `--dir` および `--file` フラグを使用して特定のディレクトリとファイルをリンティングすることもできます：

```bash title="Terminal"
next lint --dir pages --dir utils --file bar.js
```

### モノレポ内でのルートディレクトリの指定 {#specifying-a-root-directory-within-a-monorepo}

Next.js がルートディレクトリにインストールされていないプロジェクト（例えばモノレポ）で `eslint-plugin-next` を使用している場合、`.eslintrc` の `settings` プロパティを使って `eslint-plugin-next` に Next.js アプリケーションの場所を伝えることができます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  }),
]

export default eslintConfig
```

`rootDir` はパス（相対または絶対）、グロブ（例：`"packages/*/"`）、またはパスやグロブの配列とすることができます。

### キャッシュの無効化 {#disabling-the-cache}

パフォーマンスを向上させるために、ESLintによって処理されたファイルの情報はデフォルトでキャッシュされます。これは `.next/cache` または定義された[ビルドディレクトリ](/docs/app/api-reference/config/next-config-js/distDir)に保存されます。複数のソースファイルの内容に依存するESLintルールを含める場合、キャッシュを無効にするには、`next lint`に`--no-cache`フラグを使用してください。

```bash title="Terminal"
next lint --no-cache
```

### ルールの無効化 {#disabling-rules}

サポートされているプラグイン（`react`、`react-hooks`、`next`）によって提供されるルールを変更または無効化したい場合、`.eslintrc` の `rules` プロパティを使用して直接変更できます：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  }),
]

export default eslintConfig
```

### Core Web Vitals と共に使用する場合 {#with-core-web-vitals}

`next/core-web-vitals`ルールセットは、`next lint`が初めて実行され、**厳格** モードが選択されたときに有効になります。

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals'],
  }),
]

export default eslintConfig
```

`next/core-web-vitals`は、[Core Web Vitals](https://web.dev/vitals/)に影響を与える場合にデフォルトで警告となるいくつかのルールについて、`eslint-plugin-next`をエラーに更新します。

> `next/core-web-vitals`エントリーポイントは、[Create Next App](/docs/app/api-reference/cli/create-next-app)で構築された新しいアプリケーションに自動的に含まれます。

### TypeScriptと共に使用する場合 {#with-typescript}

Next.js ESLintルールに加えて、`create-next-app --typescript`は `next/typescript`を使ってTypeScript特有のリントルールも設定に追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals', 'next/typescript'],
  }),
]

export default eslintConfig
```

これらのルールは[`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)に基づいています。詳細については、[typescript-eslint > Configs](https://typescript-eslint.io/linting/configs)を参照してください。

### Prettierと共に使用する場合 {#with-prettier}

ESLintにはコードフォーマットルールも含まれており、既存の[Prettier](https://prettier.io/)の設定と競合することがあります。 ESLintとPrettierを連携させるために、[eslint-config-prettier](https://github.com/prettier/eslint-config-prettier) を以来の中に含めることをお勧めします。

まず、次の依存関係をインストールします：

```bash title="Terminal"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

次に、既存のESLint設定に`prettier`を追加します：

```mjs title="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next', 'prettier'],
  }),
]

export default eslintConfig
```

### ステージされたファイルにlintを実行 {#running-lint-on-staged-files}

[lint-staged](https://github.com/okonet/lint-staged)を使って `next lint`を使用し、ステージングされたgitファイルに対してlinterを実行する場合は、プロジェクトのルートにある `.lintstagedrc.js` ファイルに `--file` フラグの使用を指定するために次の内容を追加する必要があります。

```js title=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## 本番ビルド時にリンティングを無効にする {#disabling-linting-during-production-builds}

`next build`中にESLintの実行を避けたい場合は、`next.config.js`内で `eslint.ignoreDuringBuilds`オプションを`true`に設定できます：

const nextConfig: NextConfig = {
  eslint: {
    // 警告: ESLintエラーがあるプロジェクトでも本番ビルドが成功することを許可します。

  },
}

### 既存の設定を移行する {#migrating-existing-config}

もしアプリケーションにすでにESLintが設定されている場合、`eslint-config-next`を含めるのではなく、このプラグインを直接拡張することをお勧めします。ただし、いくつかの条件を満たさない限り、これにはメリットがあります。

#### 推奨プラグインルールセット {#recommended-plugin-ruleset}

次の条件がすべて真の場合：

- 次のプラグインのうち1つ以上をすでにインストールしている（`airbnb` や `react-app` などの異なる設定を介している場合も含む）：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- BabelがNext.js内でどのように設定されているかから異なる特定の`parserOptions`を定義している（[Babel設定をカスタマイズしたことがない限りはお勧めしません](https://nextjs.org/docs/canary/pages/building-your-application/configuring/babel)）
- インポートを処理するために`eslint-plugin-import`をNode.jsおよび/またはTypeScript[リゾルバ](https://github.com/benmosher/eslint-plugin-import#resolvers)をインストールしている

それなら、[`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js)の中でこれらのプロパティがどのように設定されているかが好みのままであるという場合は、それらの設定を削除するか、Next.js ESLintプラグインから直接拡張することをお勧めします。

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

このプラグインは、`next lint`を実行することなく、通常どおりプロジェクトにインストールできます：

```bash title="Terminal"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

これにより、複数の構成で同じプラグインやパーサーをインポートすることによって発生する衝突やエラーのリスクが排除されます。

#### 追加の構成{#additional-configurations}

すでに別のESLint設定を使用しており、`eslint-config-next`を含めたい場合、他の設定の後に最後に拡張されていることを確認してください。たとえば：

```mjs title="eslint.config.mjs"
import js from '@eslint/js'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname は Node.js v20.11.0 以降で使用可能
  baseDirectory: import.meta.dirname,
  recommendedConfig: js.configs.recommended,
})

const eslintConfig = [
  ...compat.config({
    extends: ['eslint:recommended', 'next'],
  }),
]

export default eslintConfig
```

`next`の構成はすでに`parser`、`plugins`、`settings`プロパティにデフォルト値を設定することを処理します。これらのプロパティを手動で再宣言する必要はありません。使用ケースに異なる設定が必要な場合を除きます。

他の共有可能な構成を含める場合、これらのプロパティが上書きされないよう確認する必要があります。そうでない場合は、`next`構成と動作を共有する構成を削除するか、上記のようにNext.js ESLintプラグインから直接拡張することをお勧めします。
