CSS Chunkingは、CSSファイルをチャンクに分割し、並べ替えてWebアプリケーションのパフォーマンスを向上させる戦略です。これにより、アプリケーション全体のCSSを一度に読み込むのではなく、特定のルートに必要なCSSのみを読み込むことができます。

`next.config.js`ファイルの`experimental.cssChunking`オプションを使用して、CSSファイルのチャンク化を制御できます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig = {
  experimental: {
    cssChunking: true, // デフォルト
  },
} satisfies NextConfig

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    cssChunking: true, // デフォルト
  },
}

module.exports = nextConfig
```

</TabItem>
</Tabs>

## オプション {#options}

- **`true` (デフォルト)**: Next.jsは可能な限りCSSファイルをマージし、インポート順からファイル間の明示的および暗黙的な依存関係を決定してチャンクとリクエストの数を減らすようにします
- **`false`**: Next.jsはCSSファイルをマージまたは並べ替えようとしません
- **`'strict'`**: Next.jsはインポートされた順序通りにCSSファイルを読み込みます。これにより、チャンクとリクエストが増える可能性があります

予期しないCSSの動作が発生した場合は、`'strict'`を使用することを検討してください。たとえば、異なるファイル内で異なるインポート順で`a.css`と`b.css`をインポートする場合（`a`を先に、または`b`を先に）、`true`だと任意の順序でファイルをマージし、間に依存関係がないと仮定します。ただし、`b.css`が`a.css`に依存している場合、ファイルがマージされるのを防ぎ、インポート順でファイルを読み込むために`'strict'`を使用したいかもしれません。これはチャンクとリクエストが増える結果をもたらす可能性があります。

ほとんどのアプリケーションでは、リクエストが少なくなりパフォーマンスが向上するため、`true`を推奨します。
