Next.jsを開発モードで実行する際に、ログのレベルやコンソールに記録される完全なURLを設定できます。

現在、`logging`は`fetch` APIを使用したデータ取得にのみ適用されます。他のNext.js内部のログにはまだ適用されていません。

```js title="next.config.js"
module.exports = {
  logging: {
    fetches: {
      fullUrl: true,
    },
  },
}
```

[Server Components HMR cache](/docs/app/api-reference/config/next-config-js/serverComponentsHmrCache)から復元される`fetch`リクエストは、デフォルトではログに記録されません。ただし、`logging.fetches.hmrRefreshes`を`true`に設定することで有効にできます。

```js title="next.config.js"
module.exports = {
  logging: {
    fetches: {
      hmrRefreshes: true,
    },
  },
}
```

さらに、`logging`を`false`に設定することで開発時のログを無効にすることもできます。

```js title="next.config.js"
module.exports = {
  logging: false,
}
```
