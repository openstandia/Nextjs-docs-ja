---
title: 'staleTimes'
description: 'Client Router Cache の無効化時間を上書きする方法を学びます。'
version: 'experimental'
---

`staleTimes`は、ページセグメントを[クライアントサイドのrouter cache](/docs/app/building-your-application/caching#client-side-router-cache)にキャッシュすることを可能にする実験的な機能です。

この実験的な機能を有効にして、カスタムの再検証時間を提供するには、experimentalな`staleTimes`フラグを設定します。

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

`static`と`dynamic`のプロパティは、異なる種類の[リンクプリフェッチ](/docs/app/api-reference/components/link#prefetch)に基づいた期間（秒単位）に対応します。

- `dynamic`プロパティは、ページが静的に生成されておらず、完全にプリフェッチされていない（例：`prefetch={true}`の場合）際に使用されます
  - デフォルト：0秒（キャッシュされない）
- `static`プロパティは、静的に生成されたページ、または`Link`の`prefetch`プロパティが`true`に設定されている場合、または[`router.prefetch`](/docs/app/building-your-application/caching#routerprefetch)を呼び出す際に使用されます
  - デフォルト：5分

> **Good to know:**
>
> - [Loading boundaries](/docs/app/api-reference/file-conventions/loading)は、この設定で定義された`static`期間の間再利用可能と見なされます。
> - これは、[partial rendering](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering)に影響を与えません。**つまり、共有レイアウトは自動的に再フェッチされることはなく、変更されたページセグメントだけが更新されます。**
> - レイアウトのシフトを防ぎ、ブラウザーのスクロール位置を失わないように、[back/forward caching](/docs/app/building-your-application/caching#client-side-router-cache)の挙動は変更されません。

Client Router Cache についての詳細は[こちら](/docs/app/building-your-application/caching#client-side-router-cache)をご覧ください。
