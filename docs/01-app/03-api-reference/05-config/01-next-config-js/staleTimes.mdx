description: 'Client Router Cacheの無効化時間を上書きする方法を学びましょう。'
version: 'experimental'
---

`staleTimes`は、[client-side router cache](/docs/app/building-your-application/caching#client-side-router-cache)内でページセグメントをキャッシュするための実験的機能です。

この実験的機能を有効にし、カスタムな再検証時間を提供するには、実験的な`staleTimes`フラグを設定します：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

`static`および`dynamic`プロパティは、異なる種類の[リンクプリフェッチ](/docs/app/api-reference/components/link#prefetch)に基づく期間（秒数）に対応しています。

- `dynamic`プロパティは、ページが静的に生成されず、完全にプリフェッチされていない場合に使用されます（例：`prefetch={true}`が設定されていない状態）。
  - デフォルト：0秒（キャッシュされない）
- `static`プロパティは、静的に生成されたページや、`Link`の`prefetch`プロップが`true`に設定されている場合、または[`router.prefetch`](/docs/app/building-your-application/caching#routerprefetch)を呼び出す場合に使用されます。
  - デフォルト：5分

> **Good to know:**
>
> - [Loading boundaries](/docs/app/api-reference/file-conventions/loading)は、この設定で定義されている`static`期間中は再利用可能と見なされます。
> - これは[partial rendering](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering)には影響を与えません。つまり、共有レイアウトはナビゲーションのたびに自動的に再フェッチされるのではなく、変更されたページセグメントのみが再フェッチされます。
> - これは、レイアウトシフトを防ぎ、ブラウザスクロール位置を失わないために、[back/forward caching](/docs/app/building-your-application/caching#client-side-router-cache)の動作を変更しません。

Client Router Cacheについての詳細は[こちら](/docs/app/building-your-application/caching#client-side-router-cache)で学べます。
