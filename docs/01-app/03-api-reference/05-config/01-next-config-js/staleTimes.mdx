title: 'staleTimes'
description: 'Client Router Cacheの無効化時間を上書きする方法を学びます'

---

`staleTimes`は実験的機能で、[クライアントサイドのルーターキャッシュ](/docs/app/building-your-application/caching#client-side-router-cache) のページセグメントのキャッシュを可能にします。

この実験的機能を有効にし、カスタムの再検証時間を提供するには、実験的な `staleTimes` フラグを設定します：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

`static` と `dynamic` プロパティは、さまざまなタイプの[リンクプリフェッチ](/docs/app/api-reference/components/link#prefetch)に基づいた時間周期（秒数）に対応しています。

- ページが静的に生成されておらず、完全にプリフェッチされていない場合（例：`prefetch={true}`を使用している場合）、`dynamic` プロパティが使用されます
  - デフォルト：0秒（キャッシュされない）
- 静的に生成されたページ、または`Link`の`prefetch`プロップが`true`に設定されている場合、または[`router.prefetch`](/docs/app/building-your-application/caching#routerprefetch)を呼び出した場合に使用されるのが`static` プロパティです
  - デフォルト：5分

> **Good to know:**
>
> - [Loading boundaries](/docs/app/api-reference/file-conventions/loading)は、この設定で定義された `static` 期間内で再利用可能と見なされます
> - これにより[部分レンダリング](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering)には影響はありません。**つまり、共有レイアウトは自動的に再フェッチされず、変更されるページセグメントだけがフェッチされます。**
> - レイアウトシフトを防ぎ、ブラウザのスクロール位置を失わないために、[前後のキャッシュ](/docs/app/building-your-application/caching#client-side-router-cache)の動作は変更されません

クライアントルーターキャッシュについての詳細を[こちら](/docs/app/building-your-application/caching#client-side-router-cache)で学ぶことができます。
