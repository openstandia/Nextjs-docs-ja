---
title: 'staleTimes'
description: 'Client Router Cacheの無効化時間を上書きする方法を学びましょう。'
version: 'experimental'
---

`staleTimes`は、ページセグメントを[クライアントサイドのrouter cache](/docs/app/building-your-application/caching#client-side-router-cache)でキャッシュすることを可能にする実験的な機能です。

この実験的な機能を有効にし、カスタムの再検証時間を設定するには、experimental `staleTimes`フラグをセットします：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

`static`および`dynamic`のプロパティは、異なる種類の[リンクプリフェッチ](/docs/app/api-reference/components/link#prefetch)に基づく時間（秒単位）を対応しています。

- `dynamic`プロパティは、ページが静的に生成されていないか完全にプリフェッチされていない（例：`prefetch={true}`の場合）ときに使用されます。
  - デフォルト：0秒（キャッシュされない）
- `static`プロパティは、静的に生成されたページ、または`Link`の`prefetch`プロップが`true`に設定されている場合、あるいは[`router.prefetch`](/docs/app/building-your-application/caching#routerprefetch)を呼び出す場合に使用されます。
  - デフォルト：5分

> **Good to know:**
>
> - [Loading boundary](/docs/app/api-reference/file-conventions/loading)は、この設定で定義された`static`期間中再利用可能と見なされます；
> - これは[部分レンダリング](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering)に影響を与えません。つまり、共有レイアウトはすべてのナビゲーションで自動的に再フェッチされることはなく、変更されるページセグメントのみが対象となります；
> - これは[戻る/進むキャッシュ](/docs/app/building-your-application/caching#client-side-router-cache)の動作を変更しません。レイアウトシフトを防ぎ、ブラウザのスクロール位置が失われないようにします；

Client Router Cacheについての詳細は[こちら](/docs/app/building-your-application/caching#client-side-router-cache)をご覧ください。
