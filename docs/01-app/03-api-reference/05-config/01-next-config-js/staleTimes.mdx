title: 'staleTimes'
description: 'Client Router Cacheの無効化時間を上書きする方法を学びます'
version: 'experimental'
---

`staleTimes` は、ページのセグメントを[クライアントサイドのrouter cache](/docs/app/building-your-application/caching#client-side-router-cache)にキャッシュすることを可能にする実験的な機能です。

この実験的な機能を有効にし、カスタムなリバリデーション時間を設定するには、実験的な `staleTimes` フラグを設定します：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

`static` と `dynamic` のプロパティは、異なる種類の[リンクプリフェッチ](/docs/app/api-reference/components/link#prefetch)に基づいて、時間（秒単位）を指定します。

- `dynamic` プロパティは、ページが静的に生成されておらず、完全にプリフェッチされていない場合に使用されます（例： `prefetch={true}` を指定しない場合）
  - デフォルト：0秒（キャッシュされません）
- `static` プロパティは、静的に生成されたページや、`Link` の `prefetch` prop が `true` に設定されている場合、または[`router.prefetch`](/docs/app/building-your-application/caching#routerprefetch)を呼び出す場合に使用されます
  - デフォルト：5分

> **Good to know:**
>
> - [Loading boundaries](/docs/app/api-reference/file-conventions/loading) は、この設定で定義された `static` 期間に再利用されるとみなされます
> - これは[部分描画](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering)に影響を与えません。**共有レイアウトは、ナビゲーションごとに自動的に再取得されず、変更されるページセグメントのみが再取得されます。**
> - [戻る・進むキャッシュ](/docs/app/building-your-application/caching#client-side-router-cache)の動作は変わらず、レイアウトシフトを防ぎ、ブラウザーのスクロール位置を失わないようにします

Client Router Cacheについてもっと知りたい方は[こちら](/docs/app/building-your-application/caching#client-side-router-cache)をご覧ください。
