[Markdown](https://daringfireball.net/projects/markdown/syntax) は、テキストをフォーマットするために使用される軽量マークアップ言語です。プレーンテキストの構文を使用して書き、それを構造的に有効なHTMLに変換できます。Webサイトやブログのコンテンツを書く際に一般的に使用されます。

次のように書きます...

```md
I **love** using [Next.js](https://nextjs.org/)
```

出力:

```html
<p>I <strong>love</strong> using <a href="https://nextjs.org/">Next.js</a></p>
```

[MDX](https://mdxjs.com/) は、マークダウンのスーパーセットで、[JSX](https://react.dev/learn/writing-markup-with-jsx)をマークダウンファイル内で直接書くことができます。これは、コンテンツ内に動的なインタラクティビティを追加し、Reactコンポーネントを埋め込む強力な方法です。

Next.jsはアプリケーション内のローカルMDXコンテンツ、およびサーバー上で動的にフェッチされたリモートMDXファイルの両方をサポートできます。Next.jsのプラグインは、マークダウンとReactコンポーネントをHTMLに変換する処理を担当し、Server Componentsでの使用もサポートしています（App Routerではデフォルト）。

> **Good to know**: 完成した動作例については、[Portfolio Starter Kit](https://vercel.com/templates/next.js/portfolio-starter-kit)のテンプレートを参照してください。

## 依存関係のインストール {#install-dependencies}

`@next/mdx`パッケージおよび関連パッケージは、Next.jsを設定し、マークダウンとMDXを処理できるようにするために使用されます。**このパッケージはローカルファイルからデータを取得します** ので、`.md`または`.mdx`拡張子のページを、`/pages`または`/app`ディレクトリに直接作成できます。

これらのパッケージをインストールしてNext.jsでMDXをレンダリングします：

```bash title="Terminal"
npm install @next/mdx @mdx-js/loader @mdx-js/react @types/mdx
```

## `next.config.mjs`の設定 {#configure-next-config-mjs}

MDXを使用できるように、プロジェクトのルートにある`next.config.mjs`ファイルを更新します：

```js title="next.config.mjs"
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  // `pageExtensions`を設定してマークダウンとMDXファイルを含めます
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
  // 必要に応じて、他のNext.jsの設定を以下に追加します
}

const withMDX = createMDX({
  // 必要に応じてマークダウンプラグインをここに追加
})

// MDXの設定をNext.jsの設定とマージ
export default withMDX(nextConfig)
```

これにより、`.md`と`.mdx`ファイルをアプリケーション内でページ、ルート、またはインポートとして機能させることができます。

## `mdx-components.tsx`ファイルの追加 {#add-an-mdx-components-tsx-file}

プロジェクトのルートに`mdx-components.tsx`（または`.js`）ファイルを作成し、グローバルMDXコンポーネントを定義します。例えば、`pages` あるいは `app` の同じレベル、または適用可能な場合には `src` 内で作成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'

export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    ...components,
  }
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="mdx-components.js" switcher
export function useMDXComponents(components) {
  return {
    ...components,
  }
}
```

</TabItem>
</Tabs>

> **Good to know**:
>
> - `mdx-components.tsx`はApp Routerで`@next/mdx`を利用するために**必須**であり、これなしでは動作しません。
> - [`mdx-components.tsx`ファイルの規約](/docs/app/api-reference/file-conventions/mdx-components)についてさらに学びます。
> - [カスタムスタイルとコンポーネントを使用する方法](#using-custom-styles-and-components)を学びます。

## MDXのレンダリング {#rendering-mdx}

Next.jsのファイルベースのルーティングを使用するか、ページにMDXファイルをインポートしてMDXをレンダリングできます。

### ファイルベースのルーティングを使用 {#using-file-based-routing}

ファイルベースのルーティングを使用する場合、MDXページを他のページと同じように使用できます。

<AppOnly>

App Routerのアプリでは、[metadata](/docs/app/building-your-application/optimizing/metadata)を使用することも可能です。

`/app`ディレクトリに新しいMDXページを作成します：

```txt
  my-project
  ├── app
  │   └── mdx-page
  │       └── page.(mdx/md)
  |── mdx-components.(tsx/js)
  └── package.json
```

</AppOnly>

<PagesOnly>

`/pages`ディレクトリに新しいMDXページを作成します：

```txt
  my-project
  |── mdx-components.(tsx/js)
  ├── pages
  │   └── mdx-page.(mdx/md)
  └── package.json
```

</PagesOnly>

これらのファイルでMDXを使用し、ReactコンポーネントをMDXページ内に直接インポートすることも可能です：

```mdx
import { MyComponent } from 'my-component'

# Welcome to my MDX page! {#welcome-to-my-mdx-page}

This is some **bold** and _italics_ text.

This is a list in markdown:

- One
- Two
- Three

Checkout my React component:

<MyComponent />
```

`/mdx-page`ルートに移動すると、レンダリングされたMDXページが表示されます。

## カスタムスタイルとコンポーネントの使用 {#using-custom-styles-and-components}

レンダリング時にマークダウンはネイティブのHTML要素にマップされます。例えば、次のようにマークダウンを書きます：

```md
## This is a heading {#this-is-a-heading}

This is a list in markdown:

- One
- Two
- Three
```

これにより、以下のようなHTMLが生成されます：

```html
<h2>This is a heading</h2>

<p>This is a list in markdown:</p>

<ul>
  <li>One</li>
  <li>Two</li>
  <li>Three</li>
</ul>
```

マークダウンにスタイルを適用するために、生成されたHTML要素にマップするカスタムコンポーネントを提供できます。スタイルとコンポーネントは、グローバル、ローカル、共有レイアウトと共に実装できます。

### グローバルスタイルとコンポーネント {#global-styles-and-components}

`mdx-components.tsx`にスタイルとコンポーネントを追加すると、アプリケーション内の*すべての* MDXファイルに影響します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'
import Image, { ImageProps } from 'next/image'

// このファイルでは、MDXファイルで使用されるカスタムReactコンポーネントを提供できます。
// インラインスタイルや他のライブラリからのコンポーネントなど、任意のReactコンポーネントを
// インポートして使用することができます。

export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    // ビルトインコンポーネントをカスタマイズし、スタイリングを追加できます。
    h1: ({ children }) => (
      <h1 style={{ color: 'red', fontSize: '48px' }}>{children}</h1>
    ),
    img: (props) => (
      <Image
        sizes="100vw"
        style={{ width: '100%', height: 'auto' }}
        {...(props as ImageProps)}
      />
    ),
    ...components,
  }
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="mdx-components.js" switcher

// このファイルでは、MDXファイルで使用されるカスタムReactコンポーネントを提供できます。

// インポートして使用することができます。

  return {
    // ビルトインコンポーネントをカスタマイズし、スタイリングを追加できます。

```

</TabItem>

### Tailwindのタイポグラフィプラグインの使用 {#using-tailwind-typography-plugin}

## Frontmatter {#frontmatter}

Frontmatterは、ページに関するデータを格納するのに使用できるYAML形式のキー/値のペアです。`@next/mdx`はデフォルトでfrontmatterをサポートしていませんが、FrontmatterをMDXコンテンツに追加するための多くのソリューションがあります：

- [remark-frontmatter](https://github.com/remarkjs/remark-frontmatter)
- [remark-mdx-frontmatter](https://github.com/remcohaszing/remark-mdx-frontmatter)
- [gray-matter](https://github.com/jonschlinkert/gray-matter)

`@next/mdx`は、他のJavaScriptコンポーネントと同じようにエクスポートを使用することを**許可してい**ます：

<Tabs>
<TabItem value="mdx" label="mdx">

```mdx title="content/blog-post.mdx" switcher
export const metadata = {
  author: 'John Doe',
}

# Blog post {#blog-post}
```

</TabItem>
</Tabs>

これの一般的なユースケースは、MDXのコレクションを繰り返し処理してデータを抽出したいときです。例えば、すべてのブログ投稿からブログインデックスページを作成する場合です。 [Nodeの`fs`モジュール](https://nodejs.org/api/fs.html)または[globby](https://www.npmjs.com/package/globby)などのパッケージを使用して、投稿のディレクトリを読み取り、メタデータを抽出できます。

## remarkとrehypeプラグイン {#remark-and-rehype-plugins}

MDXコンテンツを変換するために、オプションでremarkおよびrehypeプラグインを提供できます。

例えば、[`remark-gfm`](https://github.com/remarkjs/remark-gfm)を使用して、GitHub風のマークダウンをサポートできます。

remarkとrehypeのエコシステムはESMのみなので、設定ファイルとして`next.config.mjs`または`next.config.ts`を使用する必要があります。

```js title="next.config.mjs"
import remarkGfm from 'remark-gfm'
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  // .mdx拡張子のファイルを許可
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
  // 必要に応じて、他のNext.jsの設定を以下に追加します
}

const withMDX = createMDX({
  // 必要に応じてマークダウンプラグインをここに追加
  options: {
    remarkPlugins: [remarkGfm],
    rehypePlugins: [],
  },
})

// MDXとNext.jsの設定を結合
export default withMDX(nextConfig)
```

### Turbopackでのプラグイン使用 {#using-plugins-with-turbopack}

[Turbopack](/docs/architecture/turbopack)でプラグインを使用するには、最新の`@next/mdx`にアップグレードし、文字列を使ってプラグイン名を指定します：

```js title="next.config.mjs"
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
}

const withMDX = createMDX({
  options: {
    remarkPlugins: [],
    rehypePlugins: [['rehype-katex', { strict: true, throwOnError: true }]],
  },
})

export default withMDX(nextConfig)
```

> **Good to know**:
>
> remarkやrehypeプラグインは、シリアライズ可能なオプションなしでは、JavaScript関数をRustに渡すことができないため、[Turbopack](/docs/architecture/turbopack)ではまだ使用できません； 詳細については、[問題の説明](https://github.com/vercel/next.js/issues/71819#issuecomment-2461802968)を参照してください。

## リモートMDX {#remote-mdx}

MDXファイルまたはコンテンツが他の場所にある場合、サーバー上で動的に取得することができます。これは、別のローカルフォルダ、CMS、データベース、または他のどこかに保存されているコンテンツに便利です。この用途によく使われる人気のあるコミュニティパッケージが[`next-mdx-remote`](https://github.com/hashicorp/next-mdx-remote#react-server-components-rsc--nextjs-app-directory-support)です。

> **Good to know**: 注意して進んでください。MDXはJavaScriptにコンパイルされ、サーバーで実行されます。 信頼できるソースからのみMDXコンテンツを取得する必要があります； そうしないと、リモートコード実行（RCE）につながる可能性があります。

## 深堀り: マークダウンをHTMLに変換する方法 {#deep-dive-how-do-you-transform-markdown-into-html}

Reactはマークダウンをネイティブに理解しません。マークダウンのプレーンテキストを最初にHTMLに変換する必要があります。これを`remark`と`rehype`を使って実現できます。

`remark`は、マークダウンを中心としたツールのエコシステムです。`rehype`はHTMLに関して同様のものです。例として、次のコードスニペットは、マークダウンをHTMLに変換します：

```js
import { unified } from 'unified'
import remarkParse from 'remark-parse'
import remarkRehype from 'remark-rehype'
import rehypeSanitize from 'rehype-sanitize'
import rehypeStringify from 'rehype-stringify'

main()

async function main() {
  const file = await unified()
    .use(remarkParse) // マークダウンASTに変換
    .use(remarkRehype) // HTML ASTに変換
    .use(rehypeSanitize) // HTML入力をサニタイズ
    .use(rehypeStringify) // ASTをシリアライズされたHTMLに変換
    .process('Hello, Next.js!')

  console.log(String(file)) // <p>Hello, Next.js!</p>
}
```

`remark`と`rehype`のエコシステムは、[構文ハイライト](https://github.com/atomiks/rehype-pretty-code)、[見出しのリンク](https://github.com/rehypejs/rehype-autolink-headings)、[目次の生成](https://github.com/remarkjs/remark-toc)などのプラグインを含んでいます。

上記のように`@next/mdx`を使用する場合、`remark`や`rehype`を直接使用する必要は**ありません**； この作業はあなたの代わりに処理されます。ここでは、`@next/mdx`パッケージが内部で行っていることについての深い理解を得るために説明しています。

## RustベースのMDXコンパイラの使用（実験的） {#using-the-rust-based-mdx-compiler-experimental}

Next.jsはRustで書かれた新しいMDXコンパイラをサポートしています。このコンパイラはまだ実験的で、プロダクションでの使用は推奨されません。新しいコンパイラを使用するには、`next.config.js`を設定し、`withMDX`に渡します：

```js title="next.config.js"
module.exports = withMDX({
  experimental: {
    mdxRs: true,
  },
})
```

`mdxRs`はMDXファイルを変換する方法を設定するオブジェクトも許容します。

```js title="next.config.js"
module.exports = withMDX({
  experimental: {
    mdxRs: {
      jsxRuntime?: string            // カスタムjsxランタイム
      jsxImportSource?: string       // カスタムjsxインポートソース
      mdxType?: 'gfm' | 'commonmark' // 解析＆変換に使用するMDX構文の設定
    },
  },
})
```
