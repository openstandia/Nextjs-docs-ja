[Markdown](https://daringfireball.net/projects/markdown/syntax)は、テキストをフォーマットするために使用される軽量なマークアップ言語です。プレーンテキストの構文を使用して書き、構造的に有効なHTMLに変換することができます。ウェブサイトやブログのコンテンツを書くためによく使用されます。

以下のように書きます...

```md
I **love** using [Next.js](https://nextjs.org/)
```

出力:

```html
<p>I <strong>love</strong> using <a href="https://nextjs.org/">Next.js</a></p>
```

[MDX](https://mdxjs.com/)は、Markdownのスーパーセットであり、[JSX](https://react.dev/learn/writing-markup-with-jsx)を直接Markdownファイル内に書くことができます。これは、動的な対話機能を追加し、Reactコンポーネントをコンテンツ内に埋め込むための強力な方法です。

Next.js は、アプリケーション内のローカルなMDXコンテンツだけでなく、サーバーで動的に取得したリモートMDXファイルもサポートできます。Next.jsプラグインは、MarkdownとReactコンポーネントをHTMLに変換する処理をします。そしてServer Components（App Routerでのデフォルト）での使用もサポートしています。

> **Good to know**: 完全な動作例については、[Portfolio Starter Kit](https://vercel.com/templates/next.js/portfolio-starter-kit)テンプレートを参照してください。

## 依存関係をインストールする {#install-dependencies}

`@next/mdx` パッケージと関連パッケージは、Next.js を構成してMarkdownとMDXを処理できるようにするために使用されます。**これはローカルファイルからデータを取得します**ので、`/pages`や`/app`ディレクトリ内に`.md`または`.mdx`の拡張子を持つページを作成できます。

これらのパッケージをインストールして、Next.js でMDXをレンダリングします:

```bash title="Terminal"
npm install @next/mdx @mdx-js/loader @mdx-js/react @types/mdx
```

## `next.config.mjs`を設定する {#configure-next-config-mjs}

プロジェクトのrootにある`next.config.mjs`ファイルを更新してMDXを使用するように設定します：

```js title="next.config.mjs"
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  // `pageExtensions`を設定してMarkdownとMDXファイルを含めます
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
  // 必要に応じて、以下に他のNext.js設定を追加してください
}

const withMDX = createMDX({
  // 必要に応じて、Markdownプラグインをここに追加します
})

// MDXの設定をNext.jsの設定とマージします
export default withMDX(nextConfig)
```

これにより、`.md`や`.mdx`ファイルがアプリケーション内でページ、ルート、またはインポートとして機能できます。

## `mdx-components.tsx`ファイルを追加する {#add-an-mdx-components-tsx-file}

プロジェクトのrootに`mdx-components.tsx`または`.js`ファイルを作成し、グローバルなMDXコンポーネントを定義します。例えば、`pages`や`app`と同じレベル、または該当する場合は`src`内に作成します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'

export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    ...components,
  }
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="mdx-components.js" switcher
export function useMDXComponents(components) {
  return {
    ...components,
  }
}
```

</TabItem>
</Tabs>

> **Good to know**:
>
> - `mdx-components.tsx`はApp Routerで`@next/mdx`を使用するために**必須**であり、これがなければ動作しません。
> - [`mdx-components.tsx`ファイル規約](/docs/app/api-reference/file-conventions/mdx-components)についてさらに学びましょう。
> - [カスタムスタイルとコンポーネントを使用する](#using-custom-styles-and-components)方法について学びましょう。

## MDXのレンダリング {#rendering-mdx}

Next.js のファイルベースのルーティングを使用するか、MDXファイルを他のページにインポートすることで、MDXをレンダリングできます。

### ファイルベースのルーティングを使用する {#using-file-based-routing}

ファイルベースのルーティングを使用する場合、他のページと同様にMDXページを使用できます。

<AppOnly>

App Router アプリでは、[metadata](/docs/app/building-your-application/optimizing/metadata)も使用できます。

`/app` ディレクトリ内に新しいMDXページを作成します：

```txt
  my-project
  ├── app
  │   └── mdx-page
  │       └── page.(mdx/md)
  |── mdx-components.(tsx/js)
  └── package.json
```

</AppOnly>

<PagesOnly>

`/pages` ディレクトリ内に新しいMDXページを作成します：

```txt
  my-project
  |── mdx-components.(tsx/js)
  ├── pages
  │   └── mdx-page.(mdx/md)
  └── package.json
```

</PagesOnly>

これらのファイルでMDXを使用し、さらにはReactコンポーネントをインポートして、MDXページ内に直接埋め込むことができます：

```mdx
import { MyComponent } from 'my-component'

# Welcome to my MDX page! {#welcome-to-my-mdx-page}

これは**太字**と*イタリック*のテキストです。

これはMarkdownで書かれたリストです：

- One
- Two
- Three

Reactコンポーネントをチェックしてください：

<MyComponent />
```

`/mdx-page`ルートに移動すると、レンダリングされたMDXページが表示されるはずです。

## カスタムスタイルとコンポーネントを使用する {#using-custom-styles-and-components}

Markdownは、レンダリング時にネイティブのHTML要素にマップされます。例えば、次のようなMarkdownを記述します：

```md
## This is a heading {#this-is-a-heading}

これはMarkdownで書かれたリストです：

- One
- Two
- Three
```

次のHTMLが生成されます：

```html
<h2>This is a heading</h2>

<p>これはMarkdownで書かれたリストです：</p>

<ul>
  <li>One</li>
  <li>Two</li>
  <li>Three</li>
</ul>
```

Markdown をスタイリングするために、生成された HTML 要素にマップするカスタム コンポーネントを提供できます。スタイルとコンポーネントは、グローバル、ローカル、および共有レイアウトで実装できます。

### グローバルスタイルとコンポーネント {#global-styles-and-components}

`mdx-components.tsx` にスタイルとコンポーネントを追加すると、アプリ内の*すべての* MDX ファイルに影響します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'
import Image, { ImageProps } from 'next/image'

// このファイルにより、MDX ファイルで使用されるカスタム React コンポーネントを提供できます。
// インライン スタイル、他のライブラリのコンポーネントなど、あらゆる React コンポーネントをインポートして使用できます。

export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    // 組み込みコンポーネントをカスタマイズして、スタイリングを追加できるようにします。
    h1: ({ children }) => (
      <h1 style={{ color: 'red', fontSize: '48px' }}>{children}</h1>
    ),
    img: (props) => (
      <Image
        sizes="100vw"
        style={{ width: '100%', height: 'auto' }}
        {...(props as ImageProps)}
      />
    ),
    ...components,
  }
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="mdx-components.js" switcher
import Image from 'next/image'

// このファイルにより、MDX ファイルで使用されるカスタム React コンポーネントを提供できます。
// インライン スタイル、他のライブラリのコンポーネントなど、あらゆる React コンポーネントをインポートして使用できます。

export function useMDXComponents(components) {
  return {
    // 組み込みコンポーネントをカスタマイズして、スタイリングを追加できるようにします。
    h1: ({ children }) => (
      <h1 style={{ color: 'red', fontSize: '48px' }}>{children}</h1>
    ),
    img: (props) => (
      <Image
        sizes="100vw"
        style={{ width: '100%', height: 'auto' }}
        {...props}
      />
    ),
    ...components,
  }
}
```

</TabItem>
</Tabs>

### Tailwind typography プラグインを使用する {#using-tailwind-typography-plugin}

アプリケーションのスタイル設定に[Tailwind](https://tailwindcss.com)を使用する場合、[`@tailwindcss/typography` プラグイン](https://tailwindcss.com/docs/plugins#typography)を使用することで、Tailwindの構成とスタイルをマークダウン ファイルで再利用できます。

## Frontmatter {#frontmatter}

Frontmatterは、ページに関するデータを格納するために使用されるYAML形式のキー/値のペアリングです。`@next/mdx` はデフォルトで frontmatter をサポートしていませんが、frontmatter を MDX コンテンツに追加するための多くの解決策があります:

- [remark-frontmatter](https://github.com/remarkjs/remark-frontmatter)
- [remark-mdx-frontmatter](https://github.com/remcohaszing/remark-mdx-frontmatter)
- [gray-matter](https://github.com/jonschlinkert/gray-matter)

`@next/mdx` は、JavaScriptコンポーネントと同様にエクスポートを使用することができます:

<Tabs>
<TabItem value="mdx" label="mdx">

```mdx title="content/blog-post.mdx" switcher
export const metadata = {
  author: 'John Doe',
}

# Blog post {#blog-post}
```

</TabItem>
</Tabs>

これが一般的に使用されるケースは、一連のMDXを反復し、データを抽出したい場合です。たとえば、すべてのブログ投稿からブログインデックスページを作成する場合です。Nodeの[`fs` モジュール](https://nodejs.org/api/fs.html)や[globby](https://www.npmjs.com/package/globby)などのパッケージを使用して投稿のディレクトリを読み取り、メタデータを抽出できます。

> **Good to know**:
>
> - `fs`、`globby` などはサーバー サイドでのみ使用できます。
> - 完全な動作例については、[Portfolio Starter Kit](https://vercel.com/templates/next.js/portfolio-starter-kit)テンプレートを参照してください。

## remarkとrehypeプラグイン {#remark-and-rehype-plugins}

remark と rehype プラグインをオプションで提供して、MDX コンテンツを変換できます。

たとえば、GitHub Flavored Markdown をサポートするために[`remark-gfm`](https://github.com/remarkjs/remark-gfm)を使用できます。

remark と rehype のエコシステムはESM専用であるため、構成ファイルとして `next.config.mjs` または `next.config.ts` を使用する必要があります。

```js title="next.config.mjs"
import remarkGfm from 'remark-gfm'
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  // ファイル拡張子に .mdx を許可する
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
  // 必要に応じて、以下に他の Next.js 設定を追加してください
}

const withMDX = createMDX({
  // 必要に応じて、Markdown プラグインをここに追加します
  options: {
    remarkPlugins: [remarkGfm],
    rehypePlugins: [],
  },
})

// MDX と Next.js の構成を組み合わせる
export default withMDX(nextConfig)
```

### Turbopackを使用したプラグインの使用 {#using-plugins-with-turbopack}

[Turbopack](/docs/architecture/turbopack)でプラグインを使用するには、最新の`@next/mdx`にアップグレードし、プラグイン名を文字列で指定します：

```js title="next.config.mjs"
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
}

const withMDX = createMDX({
  options: {
    remarkPlugins: [],
    rehypePlugins: [['rehype-katex', { strict: true, throwOnError: true }]],
  },
})

export default withMDX(nextConfig)
```

> **Good to know**:
>
> remark および rehype のプラグインは、シリアル化可能なオプションなしでは [Rust に JavaScript 関数を渡すことができない](https://github.com/vercel/next.js/issues/71819#issuecomment-2461802968)ため、まだ [Turbopack](/docs/architecture/turbopack) と一緒に使用することはできません。

## 深堀り: MarkdownをHTMLにどのように変換しますか？ {#deep-dive-how-do-you-transform-markdown-into-html}

React はネイティブでMarkdownを理解していません。Markdownのプレーンテキストは、最初にHTMLに変換する必要があります。これは `remark` と `rehype` を使用して実現できます。

`remark` は、Markdown周りのツール群であり、`rehype` はHTML用のものです。たとえば、次のコードスニペットで、MarkdownをHTMLに変換します：

```js
import { unified } from 'unified'
import remarkParse from 'remark-parse'
import remarkRehype from 'remark-rehype'
import rehypeSanitize from 'rehype-sanitize'
import rehypeStringify from 'rehype-stringify'

main()

async function main() {
  const file = await unified()
    .use(remarkParse) // マークダウン AST に変換する
    .use(remarkRehype) // HTML AST に変換する
    .use(rehypeSanitize) // HTML 入力をサニタイズする
    .use(rehypeStringify) // AST をシリアル化された HTML に変換する
    .process('Hello, Next.js!')

  console.log(String(file)) // <p>Hello, Next.js!</p>
}
```

`remark` と `rehype` のエコシステムには、[構文ハイライト](https://github.com/atomiks/rehype-pretty-code)、[リンク見出し](https://github.com/rehypejs/rehype-autolink-headings)、[目次の生成](https://github.com/remarkjs/remark-toc)などのためのプラグインが含まれています。

上記のように `@next/mdx` を使用する場合、`remark` や `rehype` を直接使用する必要は**ありません**。これらは自動的に処理されます。ここでは、`@next/mdx` パッケージが内部で行っていることを深く理解するために説明しています。

## Rust ベースの MDX コンパイラーの使用（実験的） {#using-the-rust-based-mdx-compiler-experimental}

Next.js は、Rustによって書かれた新しいMDXコンパイラをサポートしています。このコンパイラはまだ実験的なものであり、生産用途には推奨されません。新しいコンパイラーを使用するには、`withMDX` に渡すときに `next.config.js` を構成する必要があります:

```js title="next.config.js"
module.exports = withMDX({
  experimental: {
    mdxRs: true,
  },
})
```

`mdxRs` は、mdxファイルの変換方法を構成するオブジェクトも受け入れます。

```js title="next.config.js"
module.exports = withMDX({
  experimental: {
    mdxRs: {
      jsxRuntime?: string            // カスタム JSX ランタイム
      jsxImportSource?: string       // カスタム JSX インポート元
      mdxType?: 'gfm' | 'commonmark' // 解析および変換に使用される mdx 構文の種類を構成する
    },
  },
})
```
