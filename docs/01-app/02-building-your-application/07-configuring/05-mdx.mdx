[Markdown](https://daringfireball.net/projects/markdown/syntax)は、テキストをフォーマットするために使用される軽量なマークアップ言語です。プレーンテキストの構文を使用して書き、それを構造的に有効なHTMLに変換できます。主にウェブサイトやブログにコンテンツを書くために使用されています。

次のように書きます...

```md
I **love** using [Next.js](https://nextjs.org/)
```

出力:

```html
<p>I <strong>love</strong> using <a href="https://nextjs.org/">Next.js</a></p>
```

[MDX](https://mdxjs.com/)は、markdownのスーパーセットで、[JSX](https://react.dev/learn/writing-markup-with-jsx)をmarkdownファイルに直接書くことができます。これにより、コンテンツ内にReactコンポーネントを埋め込んで動的なインタラクティビティを追加する強力な方法となります。

Next.jsは、アプリケーション内のローカルなMDXコンテンツだけでなく、サーバーで動的に取得されるリモートMDXファイルもサポートすることができます。Next.jsプラグインは、markdownとReactコンポーネントの変換をHTMLに処理し、Server Component（App Routerでデフォルト）での使用もサポートします。

> **Good to know**: 完全な作業例として[Portfolio Starter Kit](https://vercel.com/templates/next.js/portfolio-starter-kit)テンプレートをご覧ください。

## 依存関係のインストール {#install-dependencies}

`@next/mdx`パッケージと関連パッケージを使用して、Next.jsがmarkdownとMDXを処理できるように構成します。**ローカルファイルからデータをソース**し、`/pages`または`/app`ディレクトリに直接`.md`または`.mdx`拡張子のページを作成できます。

次のパッケージをインストールして、Next.jsでMDXをレンダリングします：

```bash title="Terminal"
npm install @next/mdx @mdx-js/loader @mdx-js/react @types/mdx
```

## `next.config.mjs`を構成する {#configure-next-config-mjs}

プロジェクトのルートにある`next.config.mjs`ファイルを更新して、MDXを使用するように構成します：

```js title="next.config.mjs"
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  // `pageExtensions`を設定して、markdownとMDXファイルを含める
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
  // 必要に応じて、その他のNext.jsの設定も追加
}

const withMDX = createMDX({
  // 必要に応じて、お好きなmarkdownプラグインを追加
})

// MDX設定をNext.js設定とマージ
export default withMDX(nextConfig)
```

これにより、`.md`および`.mdx`ファイルがアプリケーション内のページ、ルート、またはインポートとして動作できるようになります。

## `mdx-components.tsx`ファイルを追加する {#add-an-mdx-components-tsx-file}

グローバルなMDXコンポーネントを定義するために、プロジェクトのルートに`mdx-components.tsx`（または`.js`）ファイルを作成します。例えば、`pages`や`app`と同じレベル、または該当する場合は`src`内部に配置します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'

export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    ...components,
  }
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="mdx-components.js" switcher
export function useMDXComponents(components) {
  return {
    ...components,
  }
}
```

</TabItem>
</Tabs>

> **Good to know**:
>
> - `mdx-components.tsx`は、`@next/mdx`をApp Routerと共に使用するために**必須**であり、これなしでは動作しません。
> - [`mdx-components.tsx`ファイルの規約](/docs/app/api-reference/file-conventions/mdx-components)を詳しく学ぶことができます。
> - [カスタムスタイルとコンポーネントの使用](#using-custom-styles-and-components)について学ぶことができます。

## MDXのレンダリング {#rendering-mdx}

Next.jsのファイルベースルーティングを使用するか、他のページにMDXファイルをインポートすることで、MDXをレンダリングできます。

### ファイルベースルーティングを使用する {#using-file-based-routing}

ファイルベースルーティングを使用する場合、他のページと同様にMDXページを使用できます。

<AppOnly>

App Routerアプリケーションでは、[メタデータ](/docs/app/building-your-application/optimizing/metadata)を使用できることも含まれます。

`/app`ディレクトリ内に新しいMDXページを作成します：

```txt
  my-project
  ├── app
  │   └── mdx-page
  │       └── page.(mdx/md)
  |── mdx-components.(tsx/js)
  └── package.json
```

</AppOnly>

<PagesOnly>

`/pages`ディレクトリ内に新しいMDXページを作成します：

```txt
  my-project
  |── mdx-components.(tsx/js)
  ├── pages
  │   └── mdx-page.(mdx/md)
  └── package.json
```

</PagesOnly>

これらのファイルでMDXを使用し、MDXページ内で直接Reactコンポーネントをインポートすることもできます：

```mdx
import { MyComponent } from 'my-component'

# Welcome to my MDX page! {#welcome-to-my-mdx-page}

これはいくつかの**太字**および*イタリック*のテキストです。

これはmarkdownのリストです：

- One
- Two
- Three

私のReactコンポーネントをチェックしてください：

<MyComponent />
```

`/mdx-page`ルートに移動すると、レンダリングされたMDXページが表示されるはずです。

## カスタムスタイルとコンポーネントを使用する {#using-custom-styles-and-components}

Markdownがレンダリングされると、ネイティブHTML要素にマップされます。例えば、次のmarkdownを書くと：

```md
## This is a heading {#this-is-a-heading}

これはmarkdownのリストです：

- One
- Two
- Three
```

次のHTMLが生成されます：

```html
<h2>This is a heading</h2>

<p>これはmarkdownのリストです：</p>

<ul>
  <li>One</li>
  <li>Two</li>
  <li>Three</li>
</ul>
```

markdownをスタイル設定するために、生成されたHTML要素にマップされるカスタムコンポーネントを提供できます。スタイルとコンポーネントは、グローバル、ローカル、および共有レイアウトで実装できます。

### グローバルなスタイルとコンポーネント {#global-styles-and-components}

`mdx-components.tsx`にスタイルとコンポーネントを追加すると、アプリケーション内の*すべての* MDXファイルに影響があります。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'
import Image, { ImageProps } from 'next/image'

// このファイルはカスタムReactコンポーネントをMDXファイルで使用するために提供できます。
// 任意のReactコンポーネントをインポートして使用することができ、インラインスタイル,
// 他のライブラリのコンポーネントなども可能です。

export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    // 組み込みコンポーネントのカスタマイズを許可し、例えばスタイリングを追加
    h1: ({ children }) => (
      <h1 style={{ color: 'red', fontSize: '48px' }}>{children}</h1>
    ),
    img: (props) => (
      <Image
        sizes="100vw"
        style={{ width: '100%', height: 'auto' }}
        {...(props as ImageProps)}
      />
    ),
    ...components,
  }
}
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="mdx-components.js" switcher
import Image from 'next/image'

// このファイルはカスタムReactコンポーネントをMDXファイルで使用するために提供できます。
// 任意のReactコンポーネントをインポートして使用することができ、インラインスタイル,
// 他のライブラリのコンポーネントなども可能です。

export function useMDXComponents(components) {
  return {
    // 組み込みコンポーネントのカスタマイズを許可し、例えばスタイリングを追加
    h1: ({ children }) => (
      <h1 style={{ color: 'red', fontSize: '48px' }}>{children}</h1>
    ),
    img: (props) => (
      <Image
        sizes="100vw"
        style={{ width: '100%', height: 'auto' }}
        {...props}
      />
    ),
    ...components,
  }
}
```

</TabItem>
</Tabs>

## Frontmatter {#frontmatter}

Frontmatterは、ページに関するデータを格納するために使用されるYAMLのようなキー/バリューのペアリングです。`@next/mdx`はデフォルトでfrontmatterをサポートしていませんが、多くの解決策があり、以下のような方法でMDXコンテンツにfrontmatterを追加できます：

- [remark-frontmatter](https://github.com/remarkjs/remark-frontmatter)
- [remark-mdx-frontmatter](https://github.com/remcohaszing/remark-mdx-frontmatter)
- [gray-matter](https://github.com/jonschlinkert/gray-matter)

`@next/mdx`は、他のJavaScriptコンポーネントと同様にエクスポートを使用できることが**できます**：

<Tabs>
<TabItem value="mdx" label="mdx">

```mdx title="content/blog-post.mdx" switcher
export const metadata = {
  author: 'John Doe',
}

# Blog post {#blog-post}
```

</TabItem>
</Tabs>

メタデータは、MDXファイルの外部で参照できます：

この一般的なユースケースは、MDXのコレクションを反復し、データを抽出したいときに発生します。たとえば、すべてのブログ投稿からブログインデックスページを作成する場合です。Nodeの[`fs`モジュール](https://nodejs.org/api/fs.html)や[globby](https://www.npmjs.com/package/globby)のようなパッケージを使用して、投稿のディレクトリを読み込み、メタデータを抽出できます。

> - `fs`、`globby`などはサーバーサイドでのみ使用できます。

## remarkとrehypeプラグイン {#remark-and-rehype-plugins}

MDXコンテンツを変換するために、オプションでremarkとrehypeのプラグインを提供できます。

たとえば、[`remark-gfm`](https://github.com/remarkjs/remark-gfm)を使用してGitHub Flavored Markdownをサポートできます。

remarkとrehypeのエコシステムはESMのみのため、設定ファイルとして`next.config.mjs`または`next.config.ts`を使用する必要があります。

```js title="next.config.mjs"
import remarkGfm from 'remark-gfm'
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  // .mdx拡張子をファイルに許可
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
  // 必要に応じて、その他のNext.jsの設定も追加
}

const withMDX = createMDX({
  // 必要に応じて、お好きなmarkdownプラグインを追加
  options: {
    remarkPlugins: [remarkGfm],
    rehypePlugins: [],
  },
})

// MDXとNext.jsの設定を組み合わせる
export default withMDX(nextConfig)
```

### Turbopackでプラグインを使用する {#using-plugins-with-turbopack}

[Turbopack](/docs/architecture/turbopack)でプラグインを使用するには、最新の`@next/mdx`にアップグレードし、プラグイン名を文字列で指定します：

```js title="next.config.mjs"
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
}

const withMDX = createMDX({
  options: {
    remarkPlugins: [],
    rehypePlugins: [['rehype-katex', { strict: true, throwOnError: true }]],
  },
})

export default withMDX(nextConfig)
```

> **Good to know**:
>
> 記録不可能なオプションを持つremarkおよびrehypeプラグインは、[JavaScript関数をRustに渡すことができない](https://github.com/vercel/next.js/issues/71819#issuecomment-2461802968)ため、まだ[Turbopack](/docs/architecture/turbopack)では使用できません。

## リモートMDX {#remote-mdx}

MDXファイルやコンテンツが他の場所にある場合は、サーバーで動的にフェッチできます。これは、別のローカルフォルダ、CMS、データベース、または他の任意の場所に保存されたコンテンツに便利です。この用途に人気のあるコミュニティパッケージとして、[`next-mdx-remote`](https://github.com/hashicorp/next-mdx-remote#react-server-components-rsc--nextjs-app-directory-support)があります。

> **Good to know**: 注意が必要です。MDXはJavaScriptにコンパイルされ、サーバー上で実行されます。信頼できるソースからのみMDXコンテンツを取得する必要があります。そうしないとリモートコード実行（RCE）につながる可能性があります。

## 深堀り: MarkdownをHTMLに変換する方法 {#deep-dive-how-do-you-transform-markdown-into-html}

Reactはネイティブにmarkdownを理解しません。Markdownのプレーンテキストは最初にHTMLに変換される必要があります。これは`remark`と`rehype`を使用して達成できます。

`remark`はMarkdown周辺のツールのエコシステムであり、`rehype`はHTMLのための同様のものです。例えば、次のコードスニペットはMarkdownをHTMLに変換します：

```js
import { unified } from 'unified'
import remarkParse from 'remark-parse'
import remarkRehype from 'remark-rehype'
import rehypeSanitize from 'rehype-sanitize'
import rehypeStringify from 'rehype-stringify'

main()

async function main() {
  const file = await unified()
    .use(remarkParse) // Markdown ASTに変換
    .use(remarkRehype) // HTML ASTに変換
    .use(rehypeSanitize) // HTML入力をサニタイズ
    .use(rehypeStringify) // ASTを直列化されたHTMLに変換
    .process('Hello, Next.js!')

  console.log(String(file)) // <p>Hello, Next.js!</p>
}
```

`remark`と`rehype`エコシステムには、[シンタックスハイライト](https://github.com/atomiks/rehype-pretty-code)、[見出しのリンク化](https://github.com/rehypejs/rehype-autolink-headings)、[目次の生成](https://github.com/remarkjs/remark-toc)などのプラグインがあります。

上記のように`@next/mdx`を使用する場合、`remark`や`rehype`を直接使用する必要は**ありません**、なぜならそれはあなたのために処理されるからです。ここでは、`@next/mdx`パッケージが内部で何をしているのかの理解を深めるために説明しています。

## RustベースのMDXコンパイラを使用する（実験的） {#using-the-rust-based-mdx-compiler-experimental}

Next.jsはRustで書かれた新しいMDXコンパイラをサポートしています。このコンパイラはまだ実験的であり、本番環境での使用は推奨されていません。新しいコンパイラを使用するには、`next.config.js`で`withMDX`に渡すときに以下のように構成します：

```js title="next.config.js"
module.exports = withMDX({
  experimental: {
    mdxRs: true,
  },
})
```

`mdxRs`はMDXファイルを変換する方法を構成するためのオブジェクトも受け入れます。

```js title="next.config.js"
module.exports = withMDX({
  experimental: {
    mdxRs: {
      jsxRuntime?: string            // カスタムjsxランタイム
      jsxImportSource?: string       // カスタムjsxインポートソース
      mdxType?: 'gfm' | 'commonmark' // パース＆変換で使用するMDX構文の種類を構成
    },
  },
})
```
