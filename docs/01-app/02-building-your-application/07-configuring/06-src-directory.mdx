プロジェクトの root にある特別な Next.js の `app` または `pages` ディレクトリの代わりに、Next.js は `src` ディレクトリの下にアプリケーションコードを配置する一般的なパターンもサポートしています。

この方法はプロジェクトの root にあるプロジェクト構成ファイルからアプリケーションコードを分離するものであり、一部の個人やチームに好まれています。

`src` ディレクトリを使用するには、`app` Router フォルダまたは `pages` Router フォルダをそれぞれ `src/app` または `src/pages` に移動してください。

> **Good to know**:
>
> - `/public` ディレクトリはプロジェクトの root に留めておくべきです
> - `package.json`、 `next.config.js`、`tsconfig.json` などの設定ファイルはプロジェクトの root に留めておくべきです
> - `.env.*` ファイルはプロジェクトの root に留めておくべきです
> - `src/app` または `src/pages` は、root ディレクトリに `app` または `pages` が存在する場合、無視されます
> - `src` を使用する場合、他のアプリケーションフォルダ（例`/components` や `/lib`）も移動することになるでしょう
> - Middleware を使用する場合、それが `src` ディレクトリ内に配置されていることを確認してください
> - Tailwind CSS を使用している場合、`tailwind.config.js` ファイルの[コンテンツセクション](https://tailwindcss.com/docs/content-configuration)に `/src` プレフィックスを追加する必要があります
> - `@/*` などのインポートに TypeScript パスを使用している場合、`tsconfig.json` 内の `paths` オブジェクトを更新し、`src/` を含めるべきです
