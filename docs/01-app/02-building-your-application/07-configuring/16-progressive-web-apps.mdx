Progressive Web Application（PWA）は、Webアプリケーションの持つリーチとアクセスの容易さに、ネイティブモバイルアプリの機能とユーザー体験を組み合わせたものです。Next.jsを使用すると、複数のコードベースやアプリストアの承認を必要とせずに、すべてのプラットフォームでシームレスなアプリのような体験を提供するPWAを作成できます。

PWAを使用することで：

- アプリストアの承認を待たずに即座にアップデートを展開できます
- 単一のコードベースでクロスプラットフォームアプリケーションを作成できます
- ホーム画面へのインストールやプッシュ通知などのネイティブな機能を提供できます

## Next.jsでPWAを作成する {#creating-a-pwa-with-next-js}

### 1. Webアプリのマニフェストを作成する {#1-creating-the-web-app-manifest}

Next.jsは、app routerを使用しての[webアプリマニフェスト](/docs/app/api-reference/file-conventions/metadata/manifest)の作成をサポートしています。静的または動的なマニフェストファイルを作成できます。

例として、`app/manifest.ts` または `app/manifest.json` ファイルを作成します：

このファイルには、名前、アイコン、ユーザーのデバイス上でアイコンとして表示される方法に関する情報が含まれるべきです。これによりユーザーは、ネイティブアプリのような体験を提供するために、PWAをホーム画面にインストールできるようになります。

### 2. Webプッシュ通知の実装 {#2-implementing-web-push-notifications}

Webプッシュ通知は、以下のすべての最新のブラウザでサポートされています：

- ホーム画面にインストールされたアプリケーション向けのiOS 16.4以降
- macOS 13以降のSafari 16
- Chromiumベースのブラウザ
- Firefox

これにより、PWAはネイティブアプリの代替としても適しています。特に、オフラインサポートを必要とせずにインストールプロンプトをトリガーすることが可能です。

Webプッシュ通知により、ユーザーがアプリをアクティブに使用していないときでも再エンゲージすることができます。Next.jsアプリケーションでの実装方法は次のとおりです：

まず、`app/page.tsx`にメインページコンポーネントを作成します。理解しやすいように、これを小さな部分に分けます。まず、必要なインポートやユーティリティを追加します。参照されるServer Actionsはまだ存在しないですが構いません：

```tsx switcher
'use client'

import { useState, useEffect } from 'react'
import { subscribeUser, unsubscribeUser, sendNotification } from './actions'

function urlBase64ToUint8Array(base64String: string) {
  const padding = '='.repeat((4 - (base64String.length % 4)) % 4)
  const base64 = (base64String + padding)
    .replace(/\\-/g, '+')
    .replace(/_/g, '/')

  const rawData = window.atob(base64)
  const outputArray = new Uint8Array(rawData.length)

  for (let i = 0; i < rawData.length; ++i) {
    outputArray[i] = rawData.charCodeAt(i)
  }
  return outputArray
}
```

次に、プッシュ通知の購読、購読解除、および送信を管理するコンポーネントを追加しましょう。

```tsx switcher
function PushNotificationManager() {
  const [isSupported, setIsSupported] = useState(false)
  const [subscription, setSubscription] = useState<PushSubscription | null>(
    null
  )
  const [message, setMessage] = useState('')

  useEffect(() => {
    if ('serviceWorker' in navigator && 'PushManager' in window) {
      setIsSupported(true)
      registerServiceWorker()
    }
  }, [])

  async function registerServiceWorker() {
    const registration = await navigator.serviceWorker.register('/sw.js', {
      scope: '/',
      updateViaCache: 'none',
    })
    const sub = await registration.pushManager.getSubscription()
    setSubscription(sub)
  }

  async function subscribeToPush() {
    const registration = await navigator.serviceWorker.ready
    const sub = await registration.pushManager.subscribe({
      userVisibleOnly: true,
      applicationServerKey: urlBase64ToUint8Array(
        process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!
      ),
    })
    setSubscription(sub)
    await subscribeUser(sub)
  }

  async function unsubscribeFromPush() {
    await subscription?.unsubscribe()
    setSubscription(null)
    await unsubscribeUser()
  }

  async function sendTestNotification() {
    if (subscription) {
      await sendNotification(message)
      setMessage('')
    }
  }

  if (!isSupported) {
    return <p>Push notifications are not supported in this browser.</p>
  }

  return (
    <div>
      <h3>Push Notifications</h3>
      {subscription ? (
        <>
          <p>You are subscribed to push notifications.</p>
          <button onClick={unsubscribeFromPush}>Unsubscribe</button>
          <input
            type="text"
            placeholder="Enter notification message"
            value={message}
            onChange={(e) => setMessage(e.target.value)}
          />
          <button onClick={sendTestNotification}>Send Test</button>
        </>
      ) : (
        <>
          <p>You are not subscribed to push notifications.</p>
          <button onClick={subscribeToPush}>Subscribe</button>
        </>
      )}
    </div>
  )
}
```

### 4. VAPIDキーを生成する {#4-generating-vapid-keys}

WebプッシュAPIを使用するには、[VAPID](https://vapidkeys.com/)キーを生成する必要があります。最も簡単な方法は、web-push CLIを直接使用することです：

まず、web-pushをグローバルにインストールします：

```bash title="端末"
npm install -g web-push
```

VAPIDキーを生成するには、次のコマンドを実行します：

```bash title="端末"
web-push generate-vapid-keys
```

出力結果をコピーし、キーを`.env`ファイルに貼り付けます：

```.env
NEXT_PUBLIC_VAPID_PUBLIC_KEY=your_public_key_here
VAPID_PRIVATE_KEY=your_private_key_here
```

### 5. サービスワーカーを作成する {#5-creating-a-service-worker}

サービスワーカー用に`public/sw.js`ファイルを作成します：

```js title="public/sw.js"
self.addEventListener('push', function (event) {
  if (event.data) {
    const data = event.data.json()
    const options = {
      body: data.body,
      icon: data.icon || '/icon.png',
      badge: '/badge.png',
      vibrate: [100, 50, 100],
      data: {
        dateOfArrival: Date.now(),
        primaryKey: '2',
      },
    }
    event.waitUntil(self.registration.showNotification(data.title, options))
  }
})

self.addEventListener('notificationclick', function (event) {
  console.log('Notification click received.')
  event.notification.close()
  event.waitUntil(clients.openWindow('<https://your-website.com>'))
})
```

このサービスワーカーは、カスタム画像と通知をサポートしています。着信プッシュイベントと通知クリックを処理します。

- `icon`および`badge`プロパティを使用して、通知のカスタムアイコンを設定できます。
- `vibrate`パターンを調整して、対応するデバイスでのカスタムバイブレーションアラートを作成できます。
- `data`プロパティを使用して、通知に追加のデータを添付することができます。

サービスワーカーが期待通りに動作するか、異なるデバイスやブラウザで十分にテストすることを忘れないでください。また、`notificationclick`イベントリスナーの中の`'https://your-website.com'`リンクを、あなたのアプリケーションの適切なURLに更新することを忘れないでください。

### 6. ホーム画面への追加 {#6-adding-to-home-screen}

ステップ2で定義した`InstallPrompt`コンポーネントは、iOSデバイスにホーム画面にインストールするよう指示するメッセージを表示します。

アプリケーションがモバイルホーム画面にインストールできるようにするには、以下の要件を満たす必要があります：

1. 有効なウェブアプリマニフェスト（ステップ1で作成済み）
2. HTTPS経由で提供されるWebサイト

これらの条件が満たされると、現代のブラウザは自動的にユーザーにインストールプロンプトを表示します。[`beforeinstallprompt`](https://developer.mozilla.org/en-US/docs/Web/API/Window/beforeinstallprompt_event)を使ってカスタムインストールボタンを提供できますが、これはSafari iOSでは動作せず、クロスブラウザおよびプラットフォームの点で問題があるため、推奨されません。

### 7. ローカルでのテスト {#7-testing-locally}

ローカルで通知を確認できるようにするために、以下に注意してください：

- [ローカルでHTTPSを使用しながら実行する](/docs/app/api-reference/cli/next#using-https-during-development)
  - テストには`next dev --experimental-https`を使用します
- ブラウザ（Chrome, Safari, Firefox）で通知が有効になっています
  - ローカルでプロンプトが表示されたときに、通知を使用するための許可を受け入れます
  - ブラウザ全体で通知が無効になっていないことを確認してください
  - 通知がまだ表示されない場合、デバッグのために別のブラウザを試してください

### 8. アプリケーションのセキュリティ {#8-securing-your-application}

セキュリティは、特にPWAにおいてあらゆるWebアプリケーションにおいて重要な要素です。Next.jsでは`next.config.js`ファイルを使用してセキュリティヘッダーを構成できます。例えば：

```js title="next.config.js"
module.exports = {
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          {
            key: 'X-Content-Type-Options',
            value: 'nosniff',
          },
          {
            key: 'X-Frame-Options',
            value: 'DENY',
          },
          {
            key: 'Referrer-Policy',
            value: 'strict-origin-when-cross-origin',
          },
        ],
      },
      {
        source: '/sw.js',
        headers: [
          {
            key: 'Content-Type',
            value: 'application/javascript; charset=utf-8',
          },
          {
            key: 'Cache-Control',
            value: 'no-cache, no-store, must-revalidate',
          },
          {
            key: 'Content-Security-Policy',
            value: "default-src 'self'; script-src 'self'",
          },
        ],
      },
    ]
  },
}
```

各オプションを詳しく見てみましょう：

1. グローバルヘッダー（すべてのルートに適用）：
   1. `X-Content-Type-Options: nosniff`：MIMEタイプスニッフィングを防ぎ、悪意あるファイルのアップロードのリスクを軽減します
   2. `X-Frame-Options: DENY`：クリックジャッキング攻撃を防ぐために、サイトがiframe内で埋め込まれるのを防ぎます
   3. `Referrer-Policy: strict-origin-when-cross-origin`：リクエストに含まれるリファラー情報の量を制御し、セキュリティと機能性をバランスします
2. サービスワーカー用の特定ヘッダー：
   1. `Content-Type: application/javascript; charset=utf-8`：サービスワーカーが正しくJavaScriptとして解釈されるようにします
   2. `Cache-Control: no-cache, no-store, must-revalidate`：サービスワーカーのキャッシュを防ぎ、常に最新バージョンがユーザーに提供されるようにします
   3. `Content-Security-Policy: default-src 'self'; script-src 'self'`：サービスワーカーの厳格なコンテンツセキュリティポリシーを実装し、同じオリジンからのスクリプトのみを許可します

Next.jsを使った[コンテンツセキュリティポリシー](/docs/app/building-your-application/configuring/content-security-policy)の定義方法について、さらに詳しく学んでください。

## 次のステップ {#next-steps}

1. **PWAの機能を探る**：PWAは、Web APIを活用して高度な機能を提供できます。バックグラウンド同期、定期的なバックグラウンド同期、またはファイルシステムアクセスAPIなどの機能を探ることで、アプリケーションを強化することを検討してください。PWAの機能や、最新の情報についてのリソースとしては、[What PWA Can Do Today](https://whatpwacando.today/)があります。
2. **静的エクスポート**：サーバーを実行せずに、ファイルの静的エクスポートを使用する必要がある場合、Next.jsの設定を更新してこの変更を有効にできます。詳しくは[Next.js Static Export documentation](/docs/app/building-your-application/deploying/static-exports)を参照してください。ただし、Server Actionsから外部APIの呼び出しに移行し、定義したヘッダーをプロキシに移動する必要があります。
3. **オフラインサポート**：オフライン機能を提供するための1つのオプションは、Next.jsとともに[Serwist](https://github.com/serwist/serwist)を使用することです。Next.jsとSerwistを統合する方法の例については、彼らの[ドキュメント](https://github.com/serwist/serwist/tree/main/examples/next-basic)を参照してください。**注意**：このプラグインには、現時点でwebpackの設定が必要です。
4. **セキュリティの考慮**：サービスワーカーが適切にセキュリティ保護されていることを確認してください。これには、HTTPSを使用し、プッシュメッセージのソースを検証し、適切なエラーハンドリングを実装することが含まれます。
5. **ユーザー体験**：ユーザーのブラウザが特定のPWA機能をサポートしていない場合でも、アプリがうまく動作するように、進化的な強化技術を実装することを検討してください。
