## 14から15へのアップグレード {#upgrading-from-14-to-15}

Next.jsのバージョン15にアップデートするには、`upgrade` codemodを使用できます；

```bash title="Terminal"
npx @next/codemod@canary upgrade latest
```

手動で行いたい場合は、最新のNext & React RCをインストールしていることを確認してください。例えば：

```bash title="Terminal"
npm i next@latest react@rc react-dom@rc eslint-config-next@latest
```

> **Good to know:**
>
> - ピア依存警告が表示された場合は、`react`と`react-dom`を推奨バージョンに更新するか、`--force`または`--legacy-peer-deps`フラグを使用して警告を無視することができます。Next.js 15とReact 19が安定版になれば、これらは必要なくなります。

## React 19 {#react-19}

- `react`と`react-dom`の最低バージョンは19になりました。
- `useFormState`は`useActionState`に置き換えられました。`useFormState`フックはReact 19でも利用可能ですが、非推奨とされており、今後のリリースで削除される予定です。`useActionState`の使用が推奨され、`pending`状態を直接読み取るなどの追加プロパティが含まれています。[詳細はこちら](https://react.dev/reference/react/useActionState)
- `useFormStatus`は、`data`や`method`、`action`といった追加キーを含むようになりました。React 19を使用していない場合は、`pending`キーのみが利用可能です。[詳細はこちら](https://react.dev/reference/react-dom/hooks/useFormStatus)
- [React 19 アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide)をお読みください。

> **Good to know:** TypeScriptを使用している場合は、`@types/react`と`@types/react-dom`を最新バージョンにアップグレードしておいてください。

## 非同期リクエストAPI （破壊的変更） {#async-request-apis-breaking-change}

以前は同期的に処理されていた、ランタイム情報に依存する Dynamic API は**非同期**になりました；

- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`draftMode`](/docs/app/api-reference/functions/draft-mode)
- [`layout.js`](/docs/app/api-reference/file-conventions/layout)、[`page.js`](/docs/app/api-reference/file-conventions/page)、[`route.js`](/docs/app/api-reference/file-conventions/route)、[`default.js`](/docs/app/api-reference/file-conventions/default)、[`opengraph-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`twitter-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)、そして[`apple-icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)内の`params`
- [`page.js`](/docs/app/api-reference/file-conventions/page)内の`searchParams`

移行の負担を軽減するために、プロセスを自動化するための[codemodが利用可能です](/docs/app/building-your-application/upgrading/codemods#150)；また、APIは一時的に同期的にアクセスできます。

### `cookies` {#cookies}

#### 推奨非同期使用法 {#recommended-async-usage}

```tsx
import { cookies } from 'next/headers'

// Before
const cookieStore = cookies()
const token = cookieStore.get('token')

// After
const cookieStore = await cookies()
const token = cookieStore.get('token')
```

## `runtime`設定（破壊的変更） {#runtime-configuration-breaking-change}

`runtime` [segment config](/docs/app/api-reference/file-conventions/route-segment-config#runtime)は以前、`edge`に加えて`experimental-edge`の値もサポートしていました。両方の設定は同じものを指しているため、選択肢を簡素化するために`experimental-edge`を使用するとエラーが発生します。これを修正するには、`runtime`設定を`edge`に更新してください。[codemod](/docs/app/building-your-application/upgrading/codemods#app-dir-runtime-config-experimental-edge)が自動的にこの変更を行います。

</AppOnly>

## `fetch`リクエスト {#fetch-requests}

[`fetch`リクエスト](/docs/app/api-reference/functions/fetch)はデフォルトでキャッシュされなくなりました。

特定の`fetch`リクエストをキャッシュに含めるためには、`cache: 'force-cache'`オプションを渡すことができます；

```js title="app/layout.js"
export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされない
  const b = await fetch('https://...', { cache: 'force-cache' }) // キャッシュされる

  // ...
}
```

レイアウトやページ内のすべての`fetch`リクエストをキャッシュに含めるには、`export const fetchCache = 'default-cache'` [segment config](/docs/app/api-reference/file-conventions/route-segment-config)オプションを使用できます。個別の`fetch`リクエストが`cache`オプションを指定している場合は、それが優先されます；

```js title="app/layout.js"
// これはroot レイアウトであるため、アプリ全体のfetchリクエスト
// で自身のキャッシュオプションを設定しないものはキャッシュされる。
export const fetchCache = 'default-cache'

export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされる
  const b = await fetch('https://...', { cache: 'no-store' }) // キャッシュされない

  // ...
}
```

## Route Handlers {#route-handlers}

[Route Handlers](/docs/app/api-reference/file-conventions/route)内の`GET`関数は、デフォルトでキャッシュされなくなりました。`GET`メソッドをキャッシュに含めるには、Route Handlerファイルに `export const dynamic = 'force-static'` といった[ルート config](/docs/app/api-reference/file-conventions/route-segment-config)オプションを使用できます；

```js title="app/api/route.js"
export const dynamic = 'force-static'

export async function GET() {}
```

## クライアント側ルーターキャッシュ {#client-side-router-cache}

`<Link>`や`useRouter`でページを行き来する際、[ページ](/docs/app/api-reference/file-conventions/page)セグメントはクライアント側のrouter cacheから再利用されなくなりました。ただし、ブラウザのバックワード及びフォワードナビゲーションや共有レイアウトの際には、引き続き再利用されます。

ページセグメントをキャッシュに含めるには、[`staleTimes`](/docs/app/api-reference/config/next-config-js/staleTimes) configオプションを使用できます：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

[レイアウト](/docs/app/api-reference/file-conventions/layout)と[ローディング状態](/docs/app/api-reference/file-conventions/loading)は、依然としてキャッシュされ、ナビゲーション時に再利用されます。

## `next/font` {#next-font}

`@next/font`パッケージは組み込みの[`next/font`](/docs/app/api-reference/components/font)に置き換えられました。[codemodが利用可能です](/docs/app/building-your-application/upgrading/codemods#built-in-next-font)で、インポートを安全かつ自動的にリネームします。

```js title="app/layout.js"
// Before
import { Inter } from '@next/font/google'

// After
import { Inter } from 'next/font/google'
```

## `NextRequest` ジオロケーション {#nextrequest-geolocation}

`NextRequest`の`geo`と`ip`プロパティは削除されました。これらの値はホスティングプロバイダーによって提供されます。自動的にこの移行を行う[codemod](/docs/app/building-your-application/upgrading/codemods#150)が利用可能です。

Vercelを使用している場合は、代替として[`@vercel/functions](https://vercel.com/docs/functions/vercel-functions-package)の`geolocation`と`ipAddress`関数を使用できます：

```ts title="middleware.ts"
import { geolocation } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const { city } = geolocation(request)

  // ...
}
```

```ts title="middleware.ts"
import { ipAddress } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const ip = ipAddress(request)

  // ...
}
```
