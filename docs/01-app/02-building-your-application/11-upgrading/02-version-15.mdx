## バージョン14から15へのアップグレード {#upgrading-from-14-to-15}

Next.js バージョン15にアップデートするには、`upgrade` codemodを使います：

```bash title="Terminal"
npx @next/codemod@canary upgrade latest
```

手動で行いたい場合は、最新のNextおよびReact RCをインストールしていることを確認してください。例えば：

```bash title="Terminal"
npm i next@latest react@rc react-dom@rc eslint-config-next@latest
```

> **Good to know:**
>
> - ピア依存関係の警告が表示された場合は、`react`および`react-dom`を提案されたバージョンに更新する必要があるか、`--force`または`--legacy-peer-deps`フラグを使って警告を無視することができます。この操作はNext.js 15およびReact 19が安定版になると不要になります。
> - TypeScriptを使用している場合は、一時的にReact型を上書きする必要があります。詳細については、[React 19 RCアップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide#installing)を参照してください。

## React 19 {#react-19}

- `react`と`react-dom`の最低バージョンは19になりました。
- `useFormState`は`useActionState`に置き換えられました。`useFormState`フックはReact 19でも利用可能ですが、非推奨であり将来のリリースで削除されます。`pending`状態を直接読み取るなどの追加プロパティが含まれている`useActionState`を使用することが推奨されます。[詳細はこちら](https://react.dev/reference/react/useActionState)
- `useFormStatus`には、`data`、`method`、`action`などの追加のキーが含まれるようになりました。React 19を使用していない場合、利用可能なのは`pending`キーだけです。[詳細はこちら](https://react.dev/reference/react-dom/hooks/useFormStatus)
- [React 19 アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide)でさらに詳しく知ることができます。

## 非同期リクエストAPI (破壊的変更) {#async-request-apis-breaking-change}

以前は同期的であったランタイム情報に依存するDynamic APIが**非同期**になりました：

- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`draftMode`](/docs/app/api-reference/functions/draft-mode)
- `params` は[`layout.js`](/docs/app/api-reference/file-conventions/layout)、[`page.js`](/docs/app/api-reference/file-conventions/page)、[`route.js`](/docs/app/api-reference/file-conventions/route)、[`default.js`](/docs/app/api-reference/file-conventions/default)、[`opengraph-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`twitter-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)、[`apple-icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)などで使用されます。
- `searchParams` は[`page.js`](/docs/app/api-reference/file-conventions/page)で使用されます。

移行負担を軽減するために、[codemodが利用可能です](/docs/app/building-your-application/upgrading/codemods#150)。これらのAPIは一時的に同期的にアクセスできます。

### `cookies` {#cookies}

#### 推奨される非同期使用法 {#recommended-async-usage}

```tsx
import { cookies } from 'next/headers'

// 以前
const cookieStore = cookies()
const token = cookieStore.get('token')

// 以後
const cookieStore = await cookies()
const token = cookieStore.get('token')
```

## `runtime` 構成 (破壊的変更) {#runtime-configuration-breaking-change}

`runtime` [segment 配置](/docs/app/api-reference/file-conventions/route-segment-config#runtime)は以前、`edge`に加えて`experimental-edge`の値をサポートしていました。両方の構成は同じことを指しており、オプションを簡素化するため、`experimental-edge`が使用されるとエラーになります。これを修正するには、`runtime`構成を`edge`に更新します。[codemod](/docs/app/building-your-application/upgrading/codemods#app-dir-runtime-config-experimental-edge)がこれを自動的に行います。

</AppOnly>

## `fetch`リクエスト {#fetch-requests}

[`fetch`リクエスト](/docs/app/api-reference/functions/fetch)はデフォルトでキャッシュされなくなりました。

特定の`fetch`リクエストをキャッシュに登録するには、`cache: 'force-cache'`オプションを渡します。

```js title="app/layout.js"
export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされない
  const b = await fetch('https://...', { cache: 'force-cache' }) // キャッシュされる

  // ...
}
```

レイアウトまたはページ内のすべての`fetch`リクエストをキャッシュに登録するには、`export const fetchCache = 'default-cache'` [segment 構成オプション](/docs/app/api-reference/file-conventions/route-segment-config)を使用することができます。個別の`fetch`リクエストが`cache`オプションを指定している場合は、それが使用されます。

```js title="app/layout.js"
// これはroot レイアウトであるため、アプリ内のすべてのfetchリクエストは
// 自分のキャッシュオプションを設定しない限り、キャッシュされます。
export const fetchCache = 'default-cache'

export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされる
  const b = await fetch('https://...', { cache: 'no-store' }) // キャッシュされない

  // ...
}
```

## Route Handlers {#route-handlers}

[Route Handlers](/docs/app/api-reference/file-conventions/route) の `GET` 関数はデフォルトでキャッシュされなくなりました。`GET`メソッドをキャッシュに登録するには、`export const dynamic = 'force-static'`のような[ルート構成オプション](/docs/app/api-reference/file-conventions/route-segment-config)を使用することができます。

```js title="app/api/route.js"
export const dynamic = 'force-static'

export async function GET() {}
```

## クライアントサイドRouter Cache {#client-side-router-cache}

`<Link>`や`useRouter`を介してページ間を移動するとき、[ページ](/docs/app/api-reference/file-conventions/page)セグメントはクライアントサイドのrouter cacheから再利用されなくなりました。しかし、ブラウザーのバックワードおよびフォワードナビゲーション中や、共有レイアウトでは依然として再利用されます。

ページセグメントをキャッシュに登録するには、[`staleTimes`](/docs/app/api-reference/next-config-js/staleTimes)構成オプションを使用します：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

[Layouts](/docs/app/api-reference/file-conventions/layout) および [loading states](/docs/app/api-reference/file-conventions/loading) は依然としてキャッシュされ、ナビゲーション時には再利用されます。

## `next/font` {#next-font}

`@next/font`パッケージは、組み込みの[`next/font`](/docs/app/api-reference/components/font)に置き換えられました。[codemodが利用可能です](/docs/app/building-your-application/upgrading/codemods#built-in-next-font) これにより安全かつ自動的にインポートをリネームします。

```js title="app/layout.js"
// 以前
import { Inter } from '@next/font/google'

// 以後
import { Inter } from 'next/font/google'
```

## bundlePagesRouterDependencies {#bundlepagesrouterdependencies}

`experimental.bundlePagesExternals`は安定版となり、`bundlePagesRouterDependencies`にリネームされました。

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  // 以前

    bundlePagesExternals: true,
  },

  // 以後

module.exports = nextConfig
```

## Speed Insights {#speed-insights}

Next.js 15では、Speed Insightsの自動計装が削除されました。

Speed Insightsの使用を続けるには、[Vercel Speed Insights Quickstart](https://vercel.com/docs/speed-insights/quickstart)ガイドに従ってください。

## `NextRequest` ジオロケーション {#nextrequest-geolocation}

`NextRequest`の`geo`と`ip`プロパティは削除されました。これらの値はホスティングプロバイダーによって提供されます。[codemodが利用可能です](/docs/app/building-your-application/upgrading/codemods#150)でこの移行を自動化します。

Vercelを使用している場合は、代わりに[`@vercel/functions`](https://vercel.com/docs/functions/vercel-functions-package)から`geolocation`および`ipAddress`関数を使用することもできます：

```ts title="middleware.ts"
import { geolocation } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const { city } = geolocation(request)

  // ...
}
```

```ts title="middleware.ts"
import { ipAddress } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const ip = ipAddress(request)

  // ...
}
```
