## 14から15へのアップグレード {#upgrading-from-14-to-15}

Next.jsバージョン15に更新するには、`upgrade` codemodを使用できます：

```bash title="ターミナル"
npx @next/codemod@canary upgrade latest
```

手動で行う方がよい場合は、最新のNextとReactのRCをインストールすることを確認してください。例：

```bash title="ターミナル"
npm i next@latest react@rc react-dom@rc eslint-config-next@latest
```

> **Good to know:**
>
> - ピア依存関係の警告が表示された場合、`react`と`react-dom`を推奨バージョンに更新する必要があるか、警告を無視するために`--force`または`--legacy-peer-deps`フラグを使用することができます。Next.js 15とReact 19が安定すると、この必要はなくなります。
> - TypeScriptを使用している場合、Reactの型を一時的に上書きする必要があります。詳細は[React 19 RC アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide#installing)を参照してください。

## React 19 {#react-19}

- `react`と`react-dom`の最小バージョンは19になりました。
- `useFormState`は`useActionState`に置き換えられました。`useFormState`フックはReact 19でも利用可能ですが、非推奨とされており将来のリリースで削除されます。`useActionState`が推奨され、`pending`状態を直接読み取るなどの追加のプロパティが含まれています。[詳細はこちら](https://react.dev/reference/react/useActionState)。
- `useFormStatus`には`data`、`method`、`action`といった追加のキーが含まれました。他のReact 19を使用していない場合、利用可能なキーは`pending`のみです。[詳細はこちら](https://react.dev/reference/react-dom/hooks/useFormStatus)。
- 詳細は[React 19 アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide)を参照してください。

## 非同期リクエストAPI（差し替えが必要な変更） {#async-request-apis-breaking-change}

以前は同期的なランタイム情報に依存していたDynamic APIが、今は**非同期**になっています：

- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`draftMode`](/docs/app/api-reference/functions/draft-mode)
- [`layout.js`](/docs/app/api-reference/file-conventions/layout)、[`page.js`](/docs/app/api-reference/file-conventions/page)、[`route.js`](/docs/app/api-reference/file-conventions/route)、[`default.js`](/docs/app/api-reference/file-conventions/default)、[`opengraph-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`twitter-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)、[`apple-icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)の`params`
- [`page.js`](/docs/app/api-reference/file-conventions/page)の`searchParams`

移行の負担を軽減するために、プロセスを自動化するための[codemodが利用可能です](/docs/app/building-your-application/upgrading/codemods#150)で、APIを一時的に同期的にアクセスすることもできます。

### `cookies` {#cookies}

#### 推奨される非同期使用法 {#recommended-async-usage}

```tsx
import { cookies } from 'next/headers'

// Before
const cookieStore = cookies()
const token = cookieStore.get('token')

// After
const cookieStore = await cookies()
const token = cookieStore.get('token')
```

## `runtime`設定（差し替えが必要な変更） {#runtime-configuration-breaking-change}

`runtime` [segment config](/docs/app/api-reference/file-conventions/route-segment-config#runtime)は以前、`edge` に加えて `experimental-edge` の値をサポートしていました。両方の設定は同じものを指しており、選択肢を簡素化するため、`experimental-edge`の場合はエラーが発生するようになりました。これを修正するには、`runtime` 設定を `edge`に更新します。[codemod](/docs/app/building-your-application/upgrading/codemods#app-dir-runtime-config-experimental-edge)が自動的にこれを行うために利用可能です。

</AppOnly>

## `fetch`リクエスト {#fetch-requests}

[`fetch`リクエスト](/docs/app/api-reference/functions/fetch)は既定ではキャッシュされなくなりました。

特定の`fetch`リクエストをキャッシュに組み込むには、`cache: 'force-cache'`オプションを渡すことができます。

```js title="app/layout.js"
export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされない
  const b = await fetch('https://...', { cache: 'force-cache' }) // キャッシュされる

  // ...
}
```

レイアウトやページ内のすべての`fetch`リクエストをキャッシュに組み込むには、`export const fetchCache = 'default-cache'` [segment configオプション](/docs/app/api-reference/file-conventions/route-segment-config)を使用します。個々の`fetch`リクエストが`cache`オプションを指定している場合、それが優先されます。

```js title="app/layout.js"
// これはrootレイアウトなので、アプリ内のすべてのfetchリクエスト
// が、独自のキャッシュオプションを設定していない場合、キャッシュされます。
export const fetchCache = 'default-cache'

export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされる
  const b = await fetch('https://...', { cache: 'no-store' }) // キャッシュされない

  // ...
}
```

## route handler {#route-handlers}

[route handler](/docs/app/api-reference/file-conventions/route)の`GET`関数は、既定ではキャッシュされなくなりました。`GET`メソッドをキャッシュに組み込むには、`export const dynamic = 'force-static'`などの[route 設定オプション](/docs/app/api-reference/file-conventions/route-segment-config)を使用できます。

```js title="app/api/route.js"
export const dynamic = 'force-static'

export async function GET() {}
```

## クライアントサイドのrouter cache {#client-side-router-cache}

`<Link>`または `useRouter`を通じてページ間を移動する場合、[ページ](/docs/app/api-reference/file-conventions/page)セグメントはクライアントサイドのrouter cacheから再利用されなくなりました。 ただし、ブラウザの戻ると進むのナビゲーション中、および共有レイアウトでは、引き続き再利用されます。

ページセグメントをキャッシュに組み込むには、[`staleTimes`](/docs/app/api-reference/next-config-js/staleTimes)設定オプションを使用します：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

[レイアウト](/docs/app/api-reference/file-conventions/layout)や[ローディング状態](/docs/app/api-reference/file-conventions/loading)は、依然としてキャッシュされ、ナビゲーション時に再利用されます。

## `next/font` {#next-font}

`@next/font`パッケージは、組み込みの[`next/font`](/docs/app/api-reference/components/font)に置き換えられました。[codemod](/docs/app/building-your-application/upgrading/codemods#built-in-next-font)が、安全かつ自動的にインポートをリネームするために利用可能です。

```js title="app/layout.js"
// Before
import { Inter } from '@next/font/google'

// After
import { Inter } from 'next/font/google'
```

## Speed Insights {#speed-insights}

Speed Insightsの自動インストルメンテーションはNext.js 15で削除されました。

Speed Insightsを引き続き使用するには、[Vercel Speed Insights Quickstart](https://vercel.com/docs/speed-insights/quickstart)ガイドに従ってください。

## `NextRequest` のジオロケーション {#nextrequest-geolocation}

`NextRequest`の`geo`と`ip`プロパティは削除されました。これらの値はホスティングプロバイダーによって提供されます。[codemod](/docs/app/building-your-application/upgrading/codemods#150)が、この移行を自動化するために利用可能です。

Vercelを使用している場合、代わりに[`@vercel/functions`](https://vercel.com/docs/functions/vercel-functions-package)から`geolocation`および`ipAddress`関数を使用することもできます：

```ts title="middleware.ts"
import { geolocation } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const { city } = geolocation(request)

  // ...
}
```

```ts title="middleware.ts"
import { ipAddress } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const ip = ipAddress(request)

  // ...
}
```
