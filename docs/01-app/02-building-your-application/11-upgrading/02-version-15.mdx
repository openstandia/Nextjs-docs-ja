## バージョン 14 から 15 へのアップグレード {#upgrading-from-14-to-15}

Next.js バージョン 15 にアップデートするには、`upgrade` codemod を使用できます：

```bash title="Terminal"
npx @next/codemod@canary upgrade latest
```

手動で行うことを選ぶ場合は、最新の Next & React RC をインストールすることを確認してください、例：

```bash title="Terminal"
npm i next@latest react@rc react-dom@rc eslint-config-next@latest
```

> **Good to know:**
>
> - ピア依存関係の警告が表示された場合、`react` と `react-dom` を提案されたバージョンに更新する必要があるか、警告を無視するために `--force` または `--legacy-peer-deps` フラグを使用することができます。これは、Next.js 15 と React 19 の両方が安定版になると不要になります。

## React 19 {#react-19}

- `react` と `react-dom` の最小バージョンは、現在 19 です。
- `useFormState` は `useActionState` に置き換えられました。`useFormState` フックは React 19 ではまだ利用可能ですが、非推奨であり、将来のリリースで削除されます。`useActionState` の使用が推奨されており、直接 `pending` 状態を読み取るなどの追加プロパティが含まれています。[詳細はこちら](https://react.dev/reference/react/useActionState)
- `useFormStatus` は `data`、`method`、`action` のような追加のキーを含むようになりました。React 19 を使用していない場合は、`pending` キーのみが利用可能です。[詳細はこちら](https://react.dev/reference/react-dom/hooks/useFormStatus)
- [React 19 アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide)で詳細を読むことができます。

> **Good to know:** TypeScript を使用している場合は、`@types/react` と `@types/react-dom` も最新バージョンにアップグレードしてください。

## 非同期リクエストAPI（破壊的変更） {#async-request-apis-breaking-change}

以前は同期的だった、ランタイム情報に依存する動的APIが、**非同期**になりました：

- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`draftMode`](/docs/app/api-reference/functions/draft-mode)
- [`layout.js`](/docs/app/api-reference/file-conventions/layout)、[`page.js`](/docs/app/api-reference/file-conventions/page)、[`route.js`](/docs/app/api-reference/file-conventions/route)、[`default.js`](/docs/app/api-reference/file-conventions/default)、[`opengraph-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`twitter-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)、[`apple-icon`](/docs/app/api-reference/file-conventions/metadata/app-icons) 内の `params`
- [`page.js`](/docs/app/api-reference/file-conventions/page) 内の `searchParams`

移行の負担を軽減するために、プロセスを自動化する [codemod が利用可能](/docs/app/building-your-application/upgrading/codemods#150) であり、API は一時的に同期的にアクセス可能です。

### `cookies` {#cookies}

#### 推奨される非同期使用法 {#recommended-async-usage}

```tsx
import { cookies } from 'next/headers'

// 以前
const cookieStore = cookies()
const token = cookieStore.get('token')

// 以後
const cookieStore = await cookies()
const token = cookieStore.get('token')
```

## `runtime` 設定（破壊的変更） {#runtime-configuration-breaking-change}

以前、`runtime` [segment config](/docs/app/api-reference/file-conventions/route-segment-config#runtime) は `experimental-edge` と `edge` の値をサポートしていました。両方の設定は同じものを指しており、オプションを簡素化するため、`experimental-edge` が使用されている場合はエラーが発生します。これを修正するには、`runtime` 設定を `edge` に変更してください。[codemod](/docs/app/building-your-application/upgrading/codemods#app-dir-runtime-config-experimental-edge) が自動的にこれを行います。

</AppOnly>

## `fetch` リクエスト {#fetch-requests}

[`fetch` リクエスト](/docs/app/api-reference/functions/fetch) はデフォルトでキャッシュされなくなりました。

特定の `fetch` リクエストをキャッシュするには、`cache: 'force-cache'` オプションを渡すことができます。

```js title="app/layout.js"
export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされない
  const b = await fetch('https://...', { cache: 'force-cache' }) // キャッシュされる

  // ...
}
```

レイアウトやページ内のすべての `fetch` リクエストをキャッシュするには、`export const fetchCache = 'default-cache'` という [segment configオプション](/docs/app/api-reference/file-conventions/route-segment-config) を使用できます。個々の `fetch` リクエストが `cache` オプションを指定している場合は、それが使用されます。

```js title="app/layout.js"
// これは root レイアウトであるため、アプリ内のすべての `fetch` リクエストで
// 自分の `cache` オプションを設定していないものはキャッシュされます。
export const fetchCache = 'default-cache'

export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされる
  const b = await fetch('https://...', { cache: 'no-store' }) // キャッシュされない

  // ...
}
```

## Route Handlers {#route-handlers}

[Route Handlers](/docs/app/api-reference/file-conventions/route) 内の `GET` 関数はデフォルトでキャッシュされなくなりました。`GET` メソッドをキャッシュするために、Route Handler ファイル内で `export const dynamic = 'force-static'` などの [route config オプション](/docs/app/api-reference/file-conventions/route-segment-config) を使用できます。

```js title="app/api/route.js"
export const dynamic = 'force-static'

export async function GET() {}
```

## クライアントサイドルーターキャッシュ {#client-side-router-cache}

`<Link>` または `useRouter` を介してページ間をナビゲートする際、[page](/docs/app/api-reference/file-conventions/page) セグメントはもはやクライアントサイドルーターキャッシュから再利用されません。ただし、ブラウザの前後の移動と共有レイアウトでは引き続き再利用されます。

ページセグメントをキャッシュするには、[`staleTimes`](/docs/app/api-reference/config/next-config-js/staleTimes) 設定オプションを使用できます：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

[レイアウト](/docs/app/api-reference/file-conventions/layout) と [ローディングステート](/docs/app/api-reference/file-conventions/loading) は引き続きキャッシュされ、ナビゲーション時に再利用されます。

## `next/font` {#next-font}

`@next/font` パッケージは組み込みの [`next/font`](/docs/app/api-reference/components/font) に置き換えられました。安全にインポートを自動的にリネームする [codemod が利用可能](/docs/app/building-your-application/upgrading/codemods#built-in-next-font) です。

```js title="app/layout.js"
// 以前
import { Inter } from '@next/font/google'

// 以後
import { Inter } from 'next/font/google'
```

const nextConfig = {

## Speed Insights {#speed-insights}

Speed Insights の自動計測は Next.js 15 で削除されました。

Speed Insights を使用し続けるには、[Vercel Speed Insights クイックスタート](https://vercel.com/docs/speed-insights/quickstart) ガイドに従ってください。

## `NextRequest` の地理情報 {#nextrequest-geolocation}

`NextRequest` の `geo` および `ip` プロパティは、ホスティングプロバイダーによって提供されるこれらの値として削除されました。この移行を自動化する[コーデモッド](/docs/app/building-your-application/upgrading/codemods#150)が利用可能です。

Vercel を使用している場合は、代わりに [`@vercel/functions`](https://vercel.com/docs/functions/vercel-functions-package) から `geolocation` および `ipAddress` 関数を使用できます：

```ts title="middleware.ts"
import { geolocation } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const { city } = geolocation(request)

  // ...
}
```

```ts title="middleware.ts"
import { ipAddress } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const ip = ipAddress(request)

  // ...
}
```
