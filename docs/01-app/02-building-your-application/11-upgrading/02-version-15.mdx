## 14から15へのアップグレード {#upgrading-from-14-to-15}

Next.jsバージョン15にアップデートするには、`upgrade` codemodを使用できます：

```bash title="ターミナル"
npx @next/codemod@canary upgrade latest
```

手動で行うことを希望する場合は、最新のNextとReact RCをインストールしていることを確認してください。例：

```bash title="ターミナル"
npm i next@latest react@rc react-dom@rc eslint-config-next@latest
```

> **Good to know:**
>
> - ピア依存関係の警告が表示された場合には、提案されたバージョンに`react`と`react-dom`を更新するか、警告を無視するために`--force`または`--legacy-peer-deps`フラグを使用する必要があるかもしれません；これは、Next.js 15およびReact 19の両方が安定した状態になると不要になります。

## React 19 {#react-19}

- `react`と`react-dom`の最小バージョンは19になりました。
- `useFormState`は`useActionState`に置き換えられました。`useFormState`フックはReact 19でも利用可能ですが、非推奨となり、将来のリリースで削除されます。`useActionState`が推奨されており、`pending`状態を直接読むことができるような追加プロパティを含んでいます。 [こちらで詳しく学ぶ](https://react.dev/reference/react/useActionState)。
- `useFormStatus`には、`data`、`method`、`action`のような追加のキーが含まれています。React 19を使用していない場合は、`pending`キーのみが利用可能です。 [こちらで詳しく学ぶ](https://react.dev/reference/react-dom/hooks/useFormStatus)。
- [React 19 アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide)でさらに詳しく読むことができます。

> **Good to know:** TypeScriptを使用している場合には、最新バージョンに`@types/react`と`@types/react-dom`もアップグレードしてください。

## 非同期リクエストAPI（破壊的変更） {#async-request-apis-breaking-change}

以前は同期的だったランタイム情報に依存する動的APIが、現在は**非同期**になっています：

- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`draftMode`](/docs/app/api-reference/functions/draft-mode)
- [`layout.js`](/docs/app/api-reference/file-conventions/layout), [`page.js`](/docs/app/api-reference/file-conventions/page), [`route.js`](/docs/app/api-reference/file-conventions/route), [`default.js`](/docs/app/api-reference/file-conventions/default), [`opengraph-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image), [`twitter-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image), [`icon`](/docs/app/api-reference/file-conventions/metadata/app-icons), および [`apple-icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)での`params`。
- [`page.js`](/docs/app/api-reference/file-conventions/page)での`searchParams`

移行の負担を軽減するためには、このプロセスを自動化するための[codemodが利用可能です](/docs/app/building-your-application/upgrading/codemods#150)；APIは一時的に同期的にアクセスできます。

### `cookies` {#cookies}

#### 推奨される非同期使用法 {#recommended-async-usage}

```tsx
import { cookies } from 'next/headers'

// 変更前
const cookieStore = cookies()
const token = cookieStore.get('token')

// 変更後
const cookieStore = await cookies()
const token = cookieStore.get('token')
```

## `runtime` 設定（破壊的変更） {#runtime-configuration-breaking-change}

`runtime` の[segment 設定](/docs/app/api-reference/file-conventions/route-segment-config#runtime)は、以前は`edge`に加えて`experimental-edge`の値をサポートしていました。両方の設定は同じことを指し、オプションを単純化するために、`experimental-edge`が使用されている場合にエラーが発生するようになりました。これを修正するには、`runtime`の設定を`edge`に更新します。これは自動的に行うための[codemod](/docs/app/building-your-application/upgrading/codemods#app-dir-runtime-config-experimental-edge)が利用可能です。

</AppOnly>

## `fetch` リクエスト {#fetch-requests}

[`fetch` リクエスト](/docs/app/api-reference/functions/fetch)は、デフォルトでキャッシュされなくなりました。

特定の`fetch`リクエストをキャッシュに登録するには、`cache: 'force-cache'`オプションを渡すことができます。

```js title="app/layout.js"
export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされない
  const b = await fetch('https://...', { cache: 'force-cache' }) // キャッシュされる

  // ...
}
```

レイアウトまたはページ内のすべての`fetch`リクエストをキャッシュに登録するには、`export const fetchCache = 'default-cache'`を使用できます[segment 設定オプション](/docs/app/api-reference/file-conventions/route-segment-config)。個々の`fetch`リクエストが`cache`オプションを指定している場合は、そのオプションが使用されます。

```js title="app/layout.js"
// これはroot レイアウトですので、アプリ内のすべてのfetchリクエストは
// 自身のキャッシュオプションが設定されていない限り、キャッシュされます。
export const fetchCache = 'default-cache'

export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされる
  const b = await fetch('https://...', { cache: 'no-store' }) // キャッシュされない

  // ...
}
```

## ルートハンドラ {#route-handlers}

[ルートハンドラ](/docs/app/api-reference/file-conventions/route)内の`GET`関数は、デフォルトでキャッシュされなくなりました。`GET`メソッドをキャッシュに登録するには、[routes 設定オプション](/docs/app/api-reference/file-conventions/route-segment-config)のような`export const dynamic = 'force-static'`をルートハンドラファイルに使用できます。

```js title="app/api/route.js"
export const dynamic = 'force-static'

export async function GET() {}
```

## クライアント側ルーターキャッシュ {#client-side-router-cache}

`<Link>`または`useRouter`を介してページ間を移動する際、[ページ](/docs/app/api-reference/file-conventions/page)セグメントはクライアント側ルーターキャッシュから再利用されなくなりました。ただし、ブラウザのバックワードとフォワードのナビゲーション、および共有レイアウトに対しては依然として再利用されます。

ページセグメントをキャッシュに登録するには、[`staleTimes`](/docs/app/api-reference/config/next-config-js/staleTimes)設定オプションを使用できます：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

[レイアウト](/docs/app/api-reference/file-conventions/layout)と[読み込み状態](/docs/app/api-reference/file-conventions/loading)は依然としてキャッシュされ、ナビゲーションで再利用されます。

## `next/font` {#next-font}

`@next/font`パッケージはビルトインの[`next/font`](/docs/app/api-reference/components/font)に置き換えられました。インポート名を安全に自動的にリネームするための[codemodが利用可能です](/docs/app/building-your-application/upgrading/codemods#built-in-next-font)。

```js title="app/layout.js"
// 変更前
import { Inter } from '@next/font/google'

// 変更後
import { Inter } from 'next/font/google'
```

## Speed Insights {#speed-insights}

Speed Insightsの自動インストゥルメンテーションは、Next.js 15で削除されました。

Speed Insightsを使用し続けるには、[Vercel Speed Insights クイックスタート](https://vercel.com/docs/speed-insights/quickstart)ガイドに従ってください。

## `NextRequest` のジオロケーション {#nextrequest-geolocation}

`NextRequest`上の`geo`および`ip`プロパティは、これらの値がホスティングプロバイダーによって提供されるため、削除されました。この移行を自動化するための[codemodが利用可能です](/docs/app/building-your-application/upgrading/codemods#150)。

Vercelを使用している場合は、代わりに[`@vercel/functions`](https://vercel.com/docs/functions/vercel-functions-package)から`geolocation`および`ipAddress`関数を使用することができます：

```ts title="middleware.ts"
import { geolocation } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const { city } = geolocation(request)

  // ...
}
```

```ts title="middleware.ts"
import { ipAddress } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const ip = ipAddress(request)

  // ...
}
```
