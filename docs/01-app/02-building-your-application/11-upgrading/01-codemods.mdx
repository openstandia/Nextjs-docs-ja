---
title: 'Codemods'
description: '新機能がリリースされた際にNext.jsコードベースをアップグレードするためにcodemodsを使用します。'
---

Codemodsは、コードベース上でプログラム的に実行される変換です。これにより、多数の変更をすべてのファイルを手動で確認することなくプログラム的に適用できます。

Next.jsは、APIが更新または廃止された際にNext.jsコードベースをアップグレードするためのCodemod変換を提供しています。

## 使用法 {#usage}

ターミナルで、プロジェクトのフォルダに移動し（`cd`）、次のコマンドを実行します：

```bash title="Terminal"
npx @next/codemod <transform> <path>
```

`<transform>` と `<path>` に適切な値を入れてください。

- `transform` - 変換の名前
- `path` - 変換するファイルまたはディレクトリ
- `--dry` テスト実行を行い、コードは編集されません
- `--print` 変更された出力を比較のために表示します

## Codemods {#codemods}

### 15.0 {#15-0}

#### ルートセグメントの設定`runtime`値を`experimental-edge`から`edge`に変換 {#transform-app-router-route-segment-config-runtime-value-from-experimental-edge-to-edge}

##### `app-dir-runtime-config-experimental-edge` {#app-dir-runtime-config-experimental-edge}

> **注**: このcodemodはApp Router専用です。

```bash title="Terminal"
npx @next/codemod@latest app-dir-runtime-config-experimental-edge .
```

このcodemodは、[Route Segment Config `runtime`](https://nextjs.org/docs/app/api-reference/file-conventions/route-segment-config#runtime) の値`experimental-edge`を`edge`に変換します。

例えば：

```ts
export const runtime = 'experimental-edge'
```

以下に変換されます：

```ts
export const runtime = 'edge'
```

#### 非同期Dynamic APIへの移行 {#migrate-to-async-dynamic-apis}

従来同期アクセスをサポートしていたDynamic RenderingにオプトインしたAPIは、現在非同期です。この破壊的変更については、[アップグレードガイド](/docs/app/building-your-application/upgrading/version-15)をお読みください。

##### `next-async-request-api` {#next-async-request-api}

```bash title="Terminal"
npx @next/codemod@latest next-async-request-api .
```

このcodemodは、非同期になったDynamic API（`cookies()`、`headers()`、および`draftMode()` from `next/headers`）を、適用可能であれば適切にawaitまたは`React.use()`でラップするように変換します。自動移行が不可能な場合、codemodはユーザーに手動で確認と更新が必要であることを通知するために、型キャスト（TypeScriptファイルの場合）またはコメントを追加します。

例えば：

```tsx
import { cookies, headers } from 'next/headers'
const token = cookies().get('token')

function useToken() {
  const token = cookies().get('token')
  return token
}

export default function Page() {
  const name = cookies().get('name')
}

function getHeader() {
  return headers().get('x-foo')
}
```

以下に変換されます：

```tsx
import { use } from 'react'
import {
  cookies,
  headers,
  type UnsafeUnwrappedCookies,
  type UnsafeUnwrappedHeaders,
} from 'next/headers'
const token = (cookies() as unknown as UnsafeUnwrappedCookies).get('token')

function useToken() {
  const token = use(cookies()).get('token')
  return token
}

export default async function Page() {
  const name = (await cookies()).get('name')
}

function getHeader() {
  return (headers() as unknown as UnsafeUnwrappedHeaders).get('x-foo')
}
```

ページ / ルートエントリー (`page.js`、`layout.js`、`route.js`、または `default.js`)や、`generateMetadata` / `generateViewport` APIの中での`params`または`searchParams`のpropsへのプロパティアクセスを検出した場合、同期関数から非同期関数への呼び出しサイト変換を試み、プロパティアクセスをawaitします。クライアントコンポーネントなどで非同期にできない場合、`React.use`を使用してpromiseを展開します。

例えば：

```tsx
// page.tsx
export default function Page({
  params,
  searchParams,
}: {
  params: { slug: string }
  searchParams: { [key: string]: string | string[] | undefined }
}) {
  const { value } = searchParams
  if (value === 'foo') {
    // ...
  }
}

export function generateMetadata({ params }: { params: { slug: string } }) {
  const { slug } = params
  return {
    title: `My Page - ${slug}`,
  }
}
```

以下に変換されます：

```tsx
// page.tsx
export default async function Page(props: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const searchParams = await props.searchParams
  const { value } = searchParams
  if (value === 'foo') {
    // ...
  }
}

export async function generateMetadata(props: {
  params: Promise<{ slug: string }>
}) {
  const params = await props.params
  const { slug } = params
  return {
    title: `My Page - ${slug}`,
  }
}
```

> **Good to know:** このcodemodが手動介入が必要な箇所を識別するものの、正確な修正を特定できない場合、コードにコメントまたは型キャストを追加して、ユーザーに手動での更新が必要であることを通知します。これらのコメントは**@next/codemod**でプレフィックスされており、型キャストは`UnsafeUnwrapped`でプレフィックスされています。これらのコメントを明示的に削除しない限り、ビルドはエラーになります。[詳細はこちら](https://nextjs.org/docs/messages/sync-dynamic-apis)。

#### `NextRequest` の `geo` および `ip` プロパティを `@vercel/functions` に置き換え {#replace-geo-and-ip-properties-of-nextrequest-with-vercel-functions}

##### `next-request-geo-ip` {#next-request-geo-ip}

```bash title="Terminal"
npx @next/codemod@latest next-request-geo-ip .
```

このcodemodは、`@vercel/functions`をインストールし、`NextRequest` の`geo` および `ip` プロパティを対応する `@vercel/functions` 機能に変換します。

例えば：

```ts
import type { NextRequest } from 'next/server'

export function GET(req: NextRequest) {
  const { geo, ip } = req
}
```

以下に変換されます：

```ts
import type { NextRequest } from 'next/server'
import { geolocation, ipAddress } from '@vercel/functions'

export function GET(req: NextRequest) {
  const geo = geolocation(req)
  const ip = ipAddress(req)
}
```

#### `viewport` エクスポートを使用 {#use-viewport-export}

```bash title="Terminal"
npx @next/codemod@latest metadata-to-viewport-export .
```

### 13.0 {#13-0}

#### Next Image インポートの名前変更 {#rename-next-image-imports}

##### `next-image-to-legacy-image` {#next-image-to-legacy-image}

```bash title="Terminal"
npx @next/codemod@latest next-image-to-legacy-image .
```

Next.js 10、11、または12アプリケーションにおける`next/image`インポートを、Next.js 13で`next/legacy/image`に安全に名前変更します。また、`next/future/image`を`next/image`に名前変更します。

例えば：

```jsx title="pages/index.js"
import Image1 from 'next/image'
import Image2 from 'next/future/image'

export default function Home() {
  return (
    <div>
      <Image1 src="/test.jpg" width="200" height="300" />
      <Image2 src="/test.png" width="500" height="400" />
    </div>
  )
}
```

以下に変換されます：

```jsx title="pages/index.js"
// 'next/image' は 'next/legacy/image' になります
import Image1 from 'next/legacy/image'
// 'next/future/image' は 'next/image' になります
import Image2 from 'next/image'

export default function Home() {
  return (
    <div>
      <Image1 src="/test.jpg" width="200" height="300" />
      <Image2 src="/test.png" width="500" height="400" />
    </div>
  )
}
```

#### 新しい画像コンポーネントへの移行 {#migrate-to-the-new-image-component}

##### `next-image-experimental` {#next-image-experimental}

```bash title="Terminal"
npx @next/codemod@latest next-image-experimental .
```

`next/legacy/image` から 新しい `next/image` への危険な移行を行い、インラインスタイルを追加し、未使用のpropsを削除します。

- `layout` propsを削除し、`style`を追加
- `objectFit` propsを削除し、`style`を追加
- `objectPosition` propsを削除し、`style`を追加
- `lazyBoundary` propsを削除
- `lazyRoot` propsを削除

#### リンクコンポーネントから `<a>` タグを削除 {#remove-a-tags-from-link-components}

##### `new-link` {#new-link}

```bash title="Terminal"
npx @next/codemod@latest new-link .
```

<AppOnly>

[リンクコンポーネント](/docs/app/api-reference/components/link)内の`<a>`タグを削除するか、自動修正できないリンクに`legacyBehavior` propsを追加します。

</AppOnly>

<PagesOnly>

[リンクコンポーネント](https://nextjs.org/docs/canary/pages/api-reference/components/link)内の`<a>`タグを削除するか、自動修正できないリンクに`legacyBehavior` propsを追加します。

</PagesOnly>

例えば：

```jsx
<Link href="/about">
  <a>About</a>
</Link>
// 以下に変換
<Link href="/about">
  About
</Link>

<Link href="/about">
  <a onClick={() => console.log('clicked')}>About</a>
</Link>
// 以下に変換
<Link href="/about" onClick={() => console.log('clicked')}>
  About
</Link>
```

自動修正できない場合には、`legacyBehavior` props が追加されます。これにより、その特定のリンクに対して旧動作を使用してアプリが動作を保つことができます。

```jsx
const Component = () => <a>About</a>

<Link href="/about">
  <Component />
</Link>
// 以下に変換
<Link href="/about" legacyBehavior>
  <Component />
</Link>
```

### 11 {#11}

#### CRAからの移行 {#migrate-from-cra}

##### `cra-to-next` {#cra-to-next}

```bash title="Terminal"
npx @next/codemod cra-to-next
```

Create React App プロジェクトをNext.jsに移行し、Pages Routerと動作を一致させるための必要な設定を作成します。当初はクライアントサイドでのみレンダリングを活用し、SSR中の`window`使用による互換性の破壊を防止し、Next.js特定の機能を段階的に採用できるようにシームレスに有効化できます。

この変換に関連するフィードバックを[このディスカッション](https://github.com/vercel/next.js/discussions/25858)で共有してください。

### 10 {#10}

#### Reactインポートの追加 {#add-react-imports}

##### `add-missing-react-import` {#add-missing-react-import}

```bash title="Terminal"
npx @next/codemod add-missing-react-import
```

Reactをインポートしていないファイルを変換して、[新しいReact JSXトランスフォーム](https://reactjs.org/blog/2020/09/22/introducing-the-new-jsx-transform.html)が動作するようにインポートを含めるようにします。

例えば：

```jsx title="my-component.js"

```

以下に変換されます：

```jsx title="my-component.js"
import React from 'react'
export default class Home extends React.Component {
  render() {
    return <div>Hello World</div>
  }
}
```

### 9 {#9}

#### 匿名コンポーネントを名前付きコンポーネントに変換 {#transform-anonymous-components-into-named-components}

##### `name-default-component` {#name-default-component}

```bash title="Terminal"
npx @next/codemod name-default-component
```

**バージョン9以上。**

匿名コンポーネントをFast Refreshで動作するように名前付きコンポーネントに変換します。

例えば：

```jsx title="my-component.js"
export default function () {
  return <div>Hello World</div>
}
```

以下に変換されます：

```jsx title="my-component.js"
export default function MyComponent() {
  return <div>Hello World</div>
}
```

コンポーネントにはファイル名に基づいたキャメルケースの名前が付き、アロー関数でも動作します。

### 8 {#8}

#### AMP HOCをページ設定に変換 {#transform-amp-hoc-into-page-config}

##### `withamp-to-config` {#withamp-to-config}

```bash title="Terminal"
npx @next/codemod withamp-to-config
```

`withAmp` HOCをNext.js 9のページ設定に変換します。

例えば：

```js
// Before
import { withAmp } from 'next/amp'

function Home() {
  return <h1>My AMP Page</h1>
}

export default withAmp(Home)
```

```js
// After
export default function Home() {
  return <h1>My AMP Page</h1>
}

export const config = {
  amp: true,
}
```

### 6 {#6}

#### `withRouter`を使用 {#use-withrouter}

##### `url-to-withrouter` {#url-to-withrouter}

```bash title="Terminal"
npx @next/codemod url-to-withrouter
```

トップレベルのページに自動で注入されている`url`プロパティを、`withRouter`とそれが注入する`router`プロパティを使用するように変換します。詳細はこちら：[https://nextjs.org/docs/messages/url-deprecated](https://nextjs.org/docs/messages/url-deprecated)

例えば：

```js title="From"
import React from 'react'
export default class extends React.Component {
  render() {
    const { pathname } = this.props.url
    return <div>Current pathname: {pathname}</div>
  }
}
```

```js title="To"
import React from 'react'
import { withRouter } from 'next/router'
export default withRouter(
  class extends React.Component {
    render() {
      const { pathname } = this.props.router
      return <div>Current pathname: {pathname}</div>
    }
  }
)
```

これは一例です。変換（およびテスト済み）のすべてのケースは[`__testfixtures__`ディレクトリ](https://github.com/vercel/next.js/tree/canary/packages/next-codemod/transforms/__testfixtures__/url-to-withrouter)で見つけることができます。
