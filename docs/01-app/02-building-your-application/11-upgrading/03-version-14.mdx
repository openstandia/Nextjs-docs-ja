## バージョン 13 から 14 にアップグレードする {#upgrading-from-13-to-14}

Next.js のバージョン 14 にアップデートするには、お好みのパッケージマネージャーを使用して次のコマンドを実行してください：

```bash title="端末"
npm i next@next-14 react@latest react-dom@latest && npm i eslint-config-next@next-14 -D
```

```bash title="端末"
yarn add next@next-14 react@latest react-dom@latest && yarn add eslint-config-next@next-14 -D
```

```bash title="端末"
pnpm i next@next-14 react@latest react-dom@latest && pnpm i eslint-config-next@next-14 -D
```

```bash title="端末"
bun add next@next-14 react@latest react-dom@latest && bun add eslint-config-next@next-14 -D
```

> **Good to know:** TypeScript を使用している場合は、`@types/react` および `@types/react-dom` も最新バージョンにアップグレードしてください。

### v14 サマリー {#v14-summary}

- Node.js の最小バージョンが 16.14 から 18.17 に引き上げられました。16.x はサポート終了になりました
- `next export` コマンドは `output: 'export'` 設定に置き換えられました。詳しくは [ドキュメント](https://nextjs.org/docs/app/building-your-application/deploying/static-exports) をご覧ください
- `next/server` の `ImageResponse` インポートは `next/og` に名称が変更されました。[コーデモッドが利用可能です](/docs/app/building-your-application/upgrading/codemods#next-og-import) ので、安全に自動でインポートをリネームできます
- `@next/font` パッケージは組み込みの `next/font` に完全に置き換えられました。[コーデモッドが利用可能です](/docs/app/building-your-application/upgrading/codemods#built-in-next-font) ので、安全に自動でインポートをリネームできます
- `next-swc` の WASM ターゲットが削除されました
