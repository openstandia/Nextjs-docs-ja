## 13から14へのアップグレード {#upgrading-from-13-to-14}

Next.js バージョン 14 に更新するには、お好きなパッケージマネージャーを使用して次のコマンドを実行してください：

```bash title="Terminal"
npm i next@next-14 react@latest react-dom@latest && npm i eslint-config-next@next-14 -D
```

```bash title="Terminal"
yarn add next@next-14 react@latest react-dom@latest && yarn add eslint-config-next@next-14 -D
```

```bash title="Terminal"
pnpm i next@next-14 react@latest react-dom@latest && pnpm i eslint-config-next@next-14 -D
```

```bash title="Terminal"
bun add next@next-14 react@latest react-dom@latest && bun add eslint-config-next@next-14 -D
```

> **Good to know:** TypeScript を使用している場合は、`@types/react` と `@types/react-dom` の最新バージョンにアップグレードしてください。

### v14 概要 {#v14-summary}

- 最小 Node.js バージョンは 16.14 から 18.17 に引き上げられました。これは 16.x がサポート終了を迎えたためです
- `next export` コマンドは `output: 'export'` 設定に置き換えられました。詳細は[ドキュメント](https://nextjs.org/docs/app/building-your-application/deploying/static-exports)をご覧ください
- `ImageResponse` のための `next/server` インポートは `next/og` に名前が変更されました。インポートを安全かつ自動的に名前変更するための[コードモッドが利用可能です](/docs/app/building-your-application/upgrading/codemods#next-og-import)
- `@next/font` パッケージは、組み込みの `next/font` に完全に置き換えられました。インポートを安全かつ自動的に名前変更するための[コードモッドが利用可能です](/docs/app/building-your-application/upgrading/codemods#built-in-next-font)
- `next-swc` の WASM ターゲットが削除されました
