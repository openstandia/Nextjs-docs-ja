## バージョン13から14へのアップグレード {#upgrading-from-13-to-14}

Next.js のバージョン 14 に更新するには、お好みのパッケージマネージャーを使用して、次のコマンドを実行してください：

```bash title="ターミナル"
npm i next@next-14 react@latest react-dom@latest && npm i eslint-config-next@next-14 -D
```

```bash title="ターミナル"
yarn add next@next-14 react@latest react-dom@latest && yarn add eslint-config-next@next-14 -D
```

```bash title="ターミナル"
pnpm i next@next-14 react@latest react-dom@latest && pnpm i eslint-config-next@next-14 -D
```

```bash title="ターミナル"
bun add next@next-14 react@latest react-dom@latest && bun add eslint-config-next@next-14 -D
```

> **Good to know:** TypeScript を使用している場合は、`@types/react` と `@types/react-dom` も最新バージョンにアップグレードしてください。

### v14 概要 {#v14-summary}

- Node.js の最低バージョンが 16.14 から 18.17 に引き上げられました；16.x はサポート終了となっているためです
- `next export` コマンドは、`output: 'export'` 設定を推奨するために削除されました。詳しくは[ドキュメント](https://nextjs.org/docs/app/building-your-application/deploying/static-exports)をご覧ください
- `ImageResponse` のための `next/server` インポートが `next/og` に名前が変更されました。インポートを安全かつ自動でリネームする[コーデモッドが用意されています](/docs/app/building-your-application/upgrading/codemods#next-og-import)。
- `@next/font` パッケージは、組み込みの `next/font` に完全に置き換えられました。インポートを安全かつ自動でリネームする[コーデモッドが用意されています](/docs/app/building-your-application/upgrading/codemods#built-in-next-font)。
- `next-swc` の WASM ターゲットは削除されました。
