Next.jsは、複数の言語をサポートするためにコンテンツのルーティングとレンダリングを設定することを可能にします。異なるロケールに適応するサイトを作ることには、翻訳されたコンテンツ（ローカリゼーション）と国際化されたルートが含まれます。

## 用語 {#terminology}

- **Locale（ロケール）:** 言語と書式設定の好みのセットを示す識別子です。これは通常、ユーザーの希望する言語とおそらくその地理的な地域を含みます
  - `en-US`: アメリカで話されている英語
  - `nl-NL`: オランダで話されているオランダ語
  - `nl`: 特定の地域を指定しないオランダ語

## ルーティングの概要 {#routing-overview}

ユーザーのブラウザの言語設定を使用して、使用するロケールを選択することが推奨されます。希望する言語を変更することで、アプリケーションに送信される`Accept-Language`ヘッダーが変更されます。

例えば、以下のライブラリを使用して、`Request`を受け取り、`Headers`、サポートする予定のロケール、およびデフォルトのロケールに基づいてどのロケールを選択するかを判断できます。

```js title="middleware.js"
import { match } from '@formatjs/intl-localematcher'
import Negotiator from 'negotiator'

let headers = { 'accept-language': 'en-US,en;q=0.5' }
let languages = new Negotiator({ headers }).languages()
let locales = ['en-US', 'nl-NL', 'nl']
let defaultLocale = 'en-US'

match(languages, locales, defaultLocale) // -> 'en-US'
```

ルーティングはサブパス（`/fr/products`）またはドメイン（`my-site.fr/products`）によって国際化することができます。この情報を用いることで、[Middleware](/docs/app/building-your-application/routing/middleware)内でロケールに基づいてユーザーをリダイレクトすることができます。

```js title="middleware.js"
import { NextResponse } from "next/server";

let locales = ['en-US', 'nl-NL', 'nl']

// 上記またはライブラリを使用して、優先されるロケールを取得
function getLocale(request) { ... }

export function middleware(request) {
  // パス名にサポートされているロケールがあるか確認
  const { pathname } = request.nextUrl
  const pathnameHasLocale = locales.some(
    (locale) => pathname.startsWith(`/${locale}/`) || pathname === `/${locale}`
  )

  if (pathnameHasLocale) return

  // ロケールがない場合、リダイレクト
  const locale = getLocale(request)
  request.nextUrl.pathname = `/${locale}${pathname}`
  // 例）受信リクエストが /products の場合
  // 新しいURLは /en-US/products になります
  return NextResponse.redirect(request.nextUrl)
}

export const config = {
  matcher: [
    // 内部パス（_next）をすべてスキップ
    '/((?!_next).*)',
    // オプション: ルート（/）URLでのみ実行
    // '/'
  ],
}
```

最後に、`app/`内のすべての特別なファイルが`app/[lang]`の下に入れられるようにしてください。これにより、Next.jsのルーターがルート内の異なるロケールを動的に処理し、`lang`パラメータをレイアウトやページに転送できます。たとえば：

```jsx title="app/[lang]/page.js"
// 現在のロケールにアクセスできるようになります
// 例）/en-US/products -> `lang`は "en-US"
export default async function Page({ params: { lang } }) {
  return ...
}
```

root レイアウトも新しいフォルダ内にネストできます（例：`app/[lang]/layout.js`）。

## ローカリゼーション {#localization}

ユーザーの希望するロケールに基づいて表示されるコンテンツを変更する、すなわちローカリゼーションは、Next.jsに特有のものではありません。以下に説明するパターンは、あらゆるWebアプリケーションでも同様に機能します。

アプリケーション内で英語とオランダ語のコンテンツをサポートしたいとします。2つの異なる「辞書」を保持するかもしれません。これはあるキーからローカライズされた文字列へのマッピングを提供するオブジェクトです。たとえば：

```json title="dictionaries/en.json"
{
  "products": {
    "cart": "Add to Cart"
  }
}
```

```json title="dictionaries/nl.json"
{
  "products": {
    "cart": "Toevoegen aan Winkelwagen"
  }
}
```

次に、要求されたロケールの翻訳を読み込むために`getDictionary`関数を作成できます：

```jsx title="app/[lang]/dictionaries.js"
import 'server-only'

const dictionaries = {
  en: () => import('./dictionaries/en.json').then((module) => module.default),
  nl: () => import('./dictionaries/nl.json').then((module) => module.default),
}

export const getDictionary = async (locale) => dictionaries[locale]()
```

現在選択されている言語をもとに、レイアウトやページ内で辞書を取得できます。

```jsx title="app/[lang]/page.js"
import { getDictionary } from './dictionaries'

export default async function Page({ params: { lang } }) {
  const dict = await getDictionary(lang) // en
  return <button>{dict.products.cart}</button> // Add to Cart
}
```

`app/`ディレクトリ内のすべてのレイアウトとページはデフォルトで[Server Components](/docs/app/building-your-application/rendering/server-components)となるため、翻訳ファイルのサイズがクライアント側のJavaScriptバンドルサイズに影響することを心配する必要はありません。このコードは**サーバーでのみ実行**され、結果として得られるHTMLのみがブラウザに送信されます。

## 静的生成 {#static-generation}

特定のロケールセットに対して静的ルートを生成するには、任意のページまたはレイアウトで`generateStaticParams`を使用できます。たとえば、次のようにグローバルなものにすることができます。

```jsx title="app/[lang]/layout.js"
export async function generateStaticParams() {
  return [{ lang: 'en-US' }, { lang: 'de' }]
}

export default function Root({ children, params }) {
  return (
    <html lang={params.lang}>
      <body>{children}</body>
    </html>
  )
}
```
