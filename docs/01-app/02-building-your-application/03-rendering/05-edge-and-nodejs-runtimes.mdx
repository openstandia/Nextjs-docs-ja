Next.jsには、アプリケーションで使用できる2つのサーバーランタイムがあります：

- **Node.js Runtime**（デフォルト）で、すべてのNode.js APIとエコシステムから対応するパッケージにアクセスできます
- より限られた[APIのセット](/docs/app/api-reference/edge)を含む**Edge Runtime**

## ユースケース {#use-cases}

- Node.js Runtimeは、アプリケーションをレンダリングするために使用されます
- Edge Runtimeは、ミドルウェア（リダイレクト、リライト、ヘッダーの設定などのルーティング規則）で使用されます

## 注意点 {#caveats}

- Edge RuntimeはすべてのNode.js APIをサポートしているわけではありません。いくつかのパッケージは期待どおりに動作しないことがあります。未サポートのAPIについては[Edge Runtime](/docs/app/api-reference/edge#unsupported-apis)を参照してください
- Edge Runtimeはincremental static regeneration（ISR）をサポートしていません
- 両方のランタイムは、デプロイインフラストラクチャに応じて[ストリーミング](/docs/app/building-your-application/routing/loading-ui-and-streaming)をサポートできます
