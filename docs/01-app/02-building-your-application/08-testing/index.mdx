ReactとNext.jsでは、いくつかの異なる種類のテストを書くことができます。それぞれ独自の目的と使用ケースがあります。このページでは、アプリケーションをテストするために使用できるテストの種類と一般的なツールの概要を提供します。

## テストの種類 {#types-of-tests}

- **単体テスト**は、個々の単位（またはコードのブロック）を孤立してテストすることです。Reactでは、単位は単一の関数、フック、またはコンポーネントとなります
  - **コンポーネントテスト**は、単体テストのより焦点を絞ったバージョンで、テストの主な対象がReactコンポーネントになります。これには、コンポーネントがどのようにレンダリングされるか、propsとの相互作用、ユーザーイベントへの応答としての挙動をテストすることが含まれる場合があります
  - **統合テスト**は、複数の単位がどのように連携するかをテストします。これは、コンポーネント、フック、関数の組み合わせになる可能性があります
- **エンドツーエンド（E2E）テスト**は、ブラウザのような実際のユーザーシナリオをシミュレートした環境でユーザーフローをテストします。これは、プロダクションに似た環境で特定のタスク（例：サインアップフロー）をテストすることを意味します
- **スナップショットテスト**は、コンポーネントのレンダリングされた出力をキャプチャし、スナップショットファイルに保存します。テストが実行されると、コンポーネントの現在のレンダリング出力が保存されたスナップショットと比較されます。スナップショットの変化は、予期しない挙動の変化を示すために使用されます

## 非同期Server Components {#async-server-components}

`async` Server Componentsは、Reactエコシステムにとって新しいため、いくつかのツールでは完全にサポートされていません。その間、`async` コンポーネントには**単体テスト**よりも**エンドツーエンドテスト**を使用することをおすすめします。
