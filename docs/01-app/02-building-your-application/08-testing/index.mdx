ReactとNext.jsでは、いくつかの異なる種類のテストを作成できます。それぞれに目的と使用ケースがあります。このページでは、アプリケーションをテストする際に使用できる種類と一般的に使用されるツールの概要を提供します。

## テストの種類 {#types-of-tests}

- **ユニットテスト**は、個々のユニット（またはコードブロック）を分離してテストします。Reactでは、ユニットは単一の関数、フック、またはコンポーネントになります。
  - **コンポーネントテスト**は、テストの主な対象がReactコンポーネントであるユニットテストのより焦点を絞ったバージョンです。これには、コンポーネントのレンダリング方法、propsとのやりとり、ユーザーイベントに対する動作のテストが含まれることがあります
  - **統合テスト**は、複数のユニットがどのように連携して動作するかをテストします。これには、コンポーネント、フック、関数の組み合わせが考えられます
- **エンドツーエンド（E2E）テスト**は、ブラウザのような実際のユーザーシナリオをシミュレートした環境でユーザーフローをテストします。これは、特定のタスク（例：サインアップフロー）を本番環境に近い環境でテストすることを意味します
- **スナップショットテスト**は、コンポーネントのレンダリングされた出力をキャプチャし、スナップショットファイルに保存します。テストが実行されると、コンポーネントの現在のレンダリング出力が保存済みのスナップショットと比較されます。スナップショットの変更は、予期しない動作の変化を示すために使用されます

## 非同期 Server Components {#async-server-components}

`async` server componentはReactのエコシステムにおいて新しいため、一部のツールはそれらを完全にサポートしていません。そのため、`async`コンポーネントには**ユニットテスト**よりも**エンドツーエンドテスト**を使用することをお勧めします。
