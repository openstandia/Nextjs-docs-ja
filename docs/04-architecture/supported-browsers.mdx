Next.jsは**モダンブラウザ**をゼロ構成でサポートしています。

- Chrome 64+
- Edge 79+
- Firefox 67+
- Opera 51+
- Safari 12+

## Browserslist

特定のブラウザや機能をターゲットにしたい場合、Next.jsは`package.json`ファイル内での[Browserslist](https://browsersl.ist)の設定をサポートしています。Next.jsはデフォルトで以下のBrowserslist設定を使用します：

```json title="package.json"
{
  "browserslist": [
    "chrome 64",
    "edge 79",
    "firefox 67",
    "opera 51",
    "safari 12"
  ]
}
```

## Polyfills

[広く使われているポリフィル](https://github.com/vercel/next.js/blob/canary/packages/next-polyfill-nomodule/src/index.js)を注入しています。これには以下が含まれます：

- [**fetch()**](https://developer.mozilla.org/docs/Web/API/Fetch_API) — 置換対象：`whatwg-fetch`と`unfetch`
- [**URL**](https://developer.mozilla.org/docs/Web/API/URL) — 置換対象：[Node.js APIの`url`パッケージ](https://nodejs.org/api/url.html)
- [**Object.assign()**](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Object/assign) — 置換対象：`object-assign`、`object.assign`、`core-js/object/assign`

もし、依存関係の中にこれらのポリフィルが含まれている場合、それらは本番ビルドから自動的に排除され、重複を避けます。

さらに、バンドルサイズを削減するために、Next.jsはこれらのポリフィルを必要とするブラウザにのみロードします。世界的に大半のウェブトラフィックはこれらのポリフィルをダウンロードしません。

### カスタムポリフィル

自身のコードや外部npm依存が対象ブラウザでサポートされていない機能（IE 11など）を必要とする場合、自身でポリフィルを追加する必要があります。

この場合、[カスタム`<App>`](/docs/pages/building-your-application/routing/custom-app)や個々のコンポーネントで、必要な**特定のポリフィル**をトップレベルインポートとして追加するべきです。

## JavaScript言語機能

Next.jsは最新のJavaScript機能をそのまま利用できます。さらに[ES6機能](https://github.com/lukehoban/es6features)に加えて、Next.jsは以下もサポートしています：

- [Async/await](https://github.com/tc39/ecmascript-asyncawait) (ES2017)
- [Object Rest/Spread Properties](https://github.com/tc39/proposal-object-rest-spread) (ES2018)
- [動的`import()`](https://github.com/tc39/proposal-dynamic-import) (ES2020)
- [Optional Chaining](https://github.com/tc39/proposal-optional-chaining) (ES2020)
- [Nullish Coalescing](https://github.com/tc39/proposal-nullish-coalescing) (ES2020)
- [クラスフィールド](https://github.com/tc39/proposal-class-fields) と [静的プロパティ](https://github.com/tc39/proposal-static-class-features) (ES2022)
- その他多数！
