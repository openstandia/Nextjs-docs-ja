## Next.jsとは？

Next.jsは、フルスタックのWebアプリケーションを構築するためのReactフレームワークです。Reactコンポーネントを使用してユーザーインターフェースを構築し、Next.jsを追加機能や最適化のために使用します。

内部的には、Next.jsはバンドリング、コンパイルなどのReactに必要なツールを抽象化し、自動的に設定します。これにより、設定に時間をかけることなくアプリケーションの構築に集中できます。

個別の開発者でも大規模なチームの一員でも、Next.jsはインタラクティブで動的かつ高速なReactアプリケーションを構築するのを助けます。

## 主な特徴

Next.jsの主な特徴には以下のものがあります

| 機能                                                                  | 説明                                                                                                                                                                                      |
| ---------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [ルーティング](/docs/app/building-your-application/routing)            | レイアウト、ネストされたルーティング、ロード状態、エラーハンドリングなどをサポートする、サーバーコンポーネント上に構築されたファイルシステムベースのルーターです                                                                |
| [レンダリング](/docs/app/building-your-application/rendering)          | クライアントサイドとサーバサイドのレンダリングをクライアントおよびサーバコンポーネントで行います。Next.jsによる静的および動的レンダリングでさらに最適化されています。エッジとNode.jsのランタイムでストリーミングします      |
| [データ取得](/docs/app/building-your-application/data-fetching)        | サーバコンポーネントでのasync/awaitによる簡略化されたデータ取得、およびリクエストのメモ化、データキャッシュ、再検証用の拡張された`fetch`API                                                                       |
| [スタイリング](/docs/app/building-your-application/styling)          | CSSモジュール、Tailwind CSS、CSS-in-JSなど、お好みのスタイリング方法をサポート                                                                                                                                                    |
| [最適化](/docs/app/building-your-application/optimizing)              | 画像、フォント、スクリプトの最適化によってアプリケーションのCore Web Vitalsとユーザーエクスペリエンスを向上                                                                                                                  |
| [TypeScript](/docs/app/building-your-application/configuring/typescript) | 改善されたTypeScriptのサポートで、型チェックが向上し、より効率的なコンパイル、さらにカスタムTypeScriptプラグインと型チェッカーがあります                                                                               |

## ドキュメントの使用方法

画面の左側にドキュメントのナビゲーションバーがあります。ドキュメントのページは、基本から応用まで順序に並べられているため、アプリケーションを構築する際にステップバイステップで進めることができます。しかし、どの順序でも読むことができ、自分のユースケースに適したページにスキップすることも可能です。

画面の右側には、ページのセクション間を簡単に移動できる目次が表示されます。ページをすばやく見つけるためには、上部の検索バーやショートカット（`Ctrl+K`または`Cmd+K`）を使用してください。

開始するには、[インストール](/docs/getting-started/installation)ガイドを参照してください。

## App Router vs Pages Router

Next.jsには2つの異なるルーターがあります：App RouterとPages Routerです。App Routerは、サーバーコンポーネントやストリーミングなどReactの最新機能を使用できる新しいルーターです。Pages Routerは、サーバレンダリングされたReactアプリケーションを構築するための元々のNext.jsルーターで、古いNext.jsアプリケーション用として引き続きサポートされています。

サイドバーの上部には、**App Router**と**Pages Router**の機能を切り替えることができるドロップダウンメニューがあります。各ディレクトリに固有の機能があるため、どのタブが選択されているかを確認することが重要です。

ページの上部にあるパンくずリストも、App Routerのドキュメントを見ているのかPages Routerのドキュメントを見ているのかを示します。

## 事前知識

ドキュメントは初心者向けにデザインされていますが、Next.jsの機能に焦点を当てたままにするための基準を確立する必要があります。新しい概念を導入するたびに関連ドキュメントへのリンクを提供します。

ドキュメントを最大限に活用するには、HTML、CSS、Reactの基本的な理解を持っていることを推奨します。Reactのスキルを磨く必要がある場合は、基本を紹介する[React Foundationsコース](/learn/react-foundations)を確認してください。その後、[ダッシュボードアプリケーションを構築すること](/learn/dashboard-app)でNext.jsについてさらに学んでください。

## アクセシビリティ

ドキュメントを読みながらスクリーンリーダーを使用する際の最適なアクセシビリティのためには、FirefoxとNVDA、またはSafariとVoiceOverを使用することを推奨します。
