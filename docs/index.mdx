## Next.jsとは？

Next.jsはフルスタックのWebアプリケーションを構築するためのReactフレームワークです。ユーザーインターフェースを構築するためにReactコンポーネントを使用し、Next.jsは追加機能と最適化を提供します。

Next.jsは、Reactに必要なツール（バンドリング、コンパイルなど）を抽象化し、自動的に設定します。これにより、設定に時間をかけることなく、アプリケーションの構築に集中できます。

個人の開発者でも大規模なチームの一員でも、Next.jsはインタラクティブで動的、そして高速なReactアプリケーションの構築を支援します。

## 主な機能

Next.jsの主な機能には以下が含まれます：

| 機能                                                                     | 説明                                                                                                                                                                              |
| ------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Routing](/docs/app/building-your-application/routing)                   | layoutやネストされたルーティング、ロード状態、エラーハンドリングなどをサポートするServer Components上に構築されたファイルシステムベースのルーター。                                  |
| [Rendering](/docs/app/building-your-application/rendering)               | Client ComponentとServer Componentを用いたクライアントサイドとサーバーサイドのレンダリング。Next.jsでの静的および動的レンダリングによってサーバーでさらに最適化。EdgeとNode.jsランタイムでのストリーミング。 |
| [Data Fetching](/docs/app/building-your-application/data-fetching)       | Server Componentでのasync/awaitを使ったシンプルなデータフェッチングと、リクエストのメモ化、データキャッシング、再検証のための拡張された`fetch` API。                             |
| [Styling](/docs/app/building-your-application/styling)                   | CSS Modules, Tailwind CSS, CSS-in-JSなど、好みのスタイリング方法のサポート                                                                                                         |
| [最適化](/docs/app/building-your-application/optimizing)                 | 画像、フォント、スクリプトの最適化でアプリケーションのCore Web Vitalsとユーザー体験を向上                                                                                                |
| [TypeScript](/docs/app/building-your-application/configuring/typescript) | 型チェックを改善したり、より効率的なコンパイルを実現したりするTypeScriptのサポート向上、カスタムTypeScriptプラグインと型チェッカーも提供                                               |

## これらのドキュメントの使い方

画面の左側にはドキュメントのナビゲーションバーがあります。ドキュメントのページは基本から高度な内容まで順番に整理されているので、アプリケーションを構築する際にステップバイステップで進むことができます。ただし、どの順序で読んでも、自分のケースに適用するページにスキップすることができます。

画面の右側には、ページ内のセクション間を移動しやすくするための目次があります。すぐにページを見つけたい場合は、上部の検索バーや検索ショートカット（`Ctrl+K`または`Cmd+K`）を使用することができます。

始めるには、[インストール](/docs/getting-started/installation)ガイドをご覧ください。

## App Router vs Pages Router

:::warning[ご注意ください]

現在、本サイトでは App Router に関するドキュメントのみ翻訳して公開しております。
そのため、以下に記載のドロップダウンメニューは本サイトにはございません。

:::

Next.jsには2つの異なるルーターがあります：App RouterとPages Router。App Routerはより新しいルーターで、Server ComponentsやStreamingなど、Reactの最新機能を利用できます。Pages Routerは元々のNext.jsのルーターで、サーバーレンダリングされたReactアプリケーションを構築することができ、古いNext.jsアプリケーションのために引き続きサポートされています。

サイドバーの上部には、**App Router**と**Pages Router**の機能を切り替えるためのドロップダウンメニューがあります。それぞれのディレクトリに特有の機能があるので、どのタブが選択されているかを把握することが重要です。

ページの上部にあるパンくずリストでも、App Routerのドキュメントを見ているのか、Pages Routerのドキュメントを見ているのかが分かります。

## 前提知識

私たちのドキュメントは初心者にも優しい設計ですが、Next.jsの機能に集中するための基準を確立する必要があります。新しい概念を紹介する際には、関連するドキュメントへのリンクを提供することを確実に行います。

私たちのドキュメントを最大限に活用するには、HTML、CSS、Reactの基本的な理解を持っていることが推奨されます。Reactのスキルを確認したい場合は、[React Foundationsコース](https://nextjs.org/learn/react-foundations)をご覧ください。これは基礎を紹介します。その後、[ダッシュボードアプリケーションを構築](https://nextjs.org/learn/dashboard-app)して、Next.jsについてさらに学びましょう。
