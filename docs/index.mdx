## Next.jsとは {#what-is-next-js}

Next.jsは、フルスタックWebアプリケーションを構築するためのReactフレームワークです。ユーザーインターフェイスを構築するためにReact Componentsを使用し、Next.jsで追加の機能や最適化を行います。

Next.jsは裏側で、バンドリングやコンパイルなど、Reactに必要なツールを抽象化し、 自動的に設定します。これにより、設定に時間を費やすことなくアプリケーションの構築に集中できます。

個人開発者であれ、大規模なチームの一員であれ、Next.jsはインタラクティブで動的かつ高速なReactアプリケーションの構築を支援します。

## 主な機能 {#main-features}

Next.jsの主な機能には以下があります：

| 機能                                                                    | 説明                                                                                                                                                                                                                     |
| ----------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| [ルーティング](/docs/app/building-your-application/routing)             | Server Components上に構築されたファイルシステムベースのルーターで、レイアウト、ネストされたルーティング、ローディング状態、エラーハンドリングなどをサポートします                                                        |
| [レンダリング](/docs/app/building-your-application/rendering)           | Client ComponentとServer Componentによるクライアントサイド及びサーバーサイドレンダリング。Next.jsによる静的および動的レンダリングでさらに最適化されます。EdgeやNode.jsランタイムでのストリーミングもサポートされています |
| [データフェッチング](/docs/app/building-your-application/data-fetching) | Server Componentsでのasync/awaitを用いたデータフェッチングの簡略化、およびリクエストメモ化、データキャッシング、再検証のための拡張`fetch`APIを提供                                                                       |
| [スタイリング](/docs/app/building-your-application/styling)             | CSS Modules、Tailwind CSS、およびCSS-in-JSを含むお好みのスタイリング方法をサポート                                                                                                                                       |
| [最適化](/docs/app/building-your-application/optimizing)                | 画像、フォント、およびスクリプトの最適化で、アプリケーションのCore Web Vitalsとユーザーエクスペリエンスを向上                                                                                                            |
| [TypeScript](/docs/app/api-reference/config/typescript)                 | TypeScriptサポートの向上、型チェックの改善、および効率的なコンパイル、カスタムTypeScriptプラグインと型チェッカーを提供                                                                                                   |

## ドキュメントの使い方 {#how-to-use-these-docs}

画面の左側には、ドキュメントのナビゲーションバーがあります。ドキュメントのページは、基本から高度なものへと順次整理されているため、アプリケーションを構築する際にステップバイステップでフォローすることができます。ただし、どの順番でも読めますし、特定のユースケースに該当するページに直接飛ぶこともできます。

画面の右側には、ページ内のセクションを簡単に移動できる目次が表示されています。特定のページをすばやく見つけたい場合は、上部の検索バー、または検索ショートカット (`Ctrl+K` や `Cmd+K`) を使用してください。

まずは[インストールガイド](/docs/app/getting-started/installation)を参照してみてください。

## App Router vs Pages Router {#app-router-vs-pages-router}

Next.jsには、App RouterとPages Routerの2つの異なるルーターがあります。App Routerは、Server ComponentsやStreamingのようなReactの最新機能を使用できる新しいルーターです。一方、Pages Routerは、サーバーレンダリングされたReactアプリケーションを構築するためのNext.jsのオリジナルのルーターで、古いNext.jsアプリケーションのサポートを続けています。

サイドバーの上部には、**App Router** と **Pages Router** の機能を切り替えるドロップダウンメニューがあります。それぞれのディレクトリに固有の機能があるため、どのタブが選択されているかを把握しておくことが重要です。

ページ上部のパンくずリストでも、App RouterドキュメントとPages Routerドキュメントのどちらを閲覧しているかを示します。

## 必要な知識 {#pre-requisite-knowledge}

我々のドキュメントは初心者にも優しいよう設計されていますが、Next.jsの機能に集中するための基準を設けています。新しい概念を紹介する際には、関連するドキュメントへのリンクを提供するよう努めます。

このドキュメントを最大限に活用するためには、HTML、CSS、およびReactの基本的な理解が必要です。Reactスキルを向上させたい場合は、基本を紹介する[React Foundationsコース](https://nextjs.org/learn/react-foundations)をご覧ください。そして、[ダッシュボードアプリケーションの構築](https://nextjs.org/learn/dashboard-app)を経て、Next.jsについてより詳しく学んでください。

## アクセシビリティ {#accessibility}

スクリーンリーダーを使用してドキュメントを読む際の最適なアクセシビリティのために、FirefoxとNVDA、またはSafariとVoiceOverの使用をお勧めします。
