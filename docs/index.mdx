## Next.js とは？ {#what-is-next-js}

Next.js は、フルスタック Web アプリケーションを構築するための React フレームワークです。React コンポーネントを使用してユーザーインターフェイスを構築し、Next.js で追加の機能と最適化を行います。

Next.js は内部で、バンドルやコンパイルといった、React に必要なツールの設定を抽象化し自動的に行います。これにより、設定に時間を費やすことなくアプリケーションの構築に集中できます。

個人開発者でも大規模チームの一員でも、Next.js はインタラクティブで動的かつ高速な React アプリケーションの構築を助けます。

## 主な機能 {#main-features}

Next.js の主な機能には、次のものがあります。

| 機能                                                               | 説明                                                                                                                                                                                                               |
| ------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| [Routing](/docs/app/building-your-application/routing)             | layouts、ネストされたルーティング、読み込み状態、エラー処理などをサポートした Server Components 上に構築されたファイルシステムベースのルーターです                                                                 |
| [Rendering](/docs/app/building-your-application/rendering)         | Client および Server Components を使用したクライアントサイドおよびサーバーサイドレンダリング。Next.js でさらに最適化されたサーバーでの静的および動的レンダリング。Edge および Node.js ランタイムでのストリーミング |
| [Data Fetching](/docs/app/building-your-application/data-fetching) | サーバーコンポーネントでの async/await を使用した簡略化されたデータフェッチ、および request memoization、データキャッシング、再検証のための拡張 `fetch` API                                                        |
| [Styling](/docs/app/building-your-application/styling)             | CSS Modules、Tailwind CSS、および CSS-in-JS を含む希望のスタイリング手法をサポート                                                                                                                                 |
| [Optimizations](/docs/app/building-your-application/optimizing)    | アプリケーションのコアウェブバイタルおよびユーザー体験を向上させるための画像、フォント、およびスクリプトの最適化                                                                                                   |
| [TypeScript](/docs/app/api-reference/config/typescript)            | 型チェックの改善、より効率的なコンパイル、カスタム TypeScript プラグインおよび型チェッカーによる TypeScript のサポートの向上                                                                                       |

## ドキュメントの使い方 {#how-to-use-these-docs}

画面の左側には、ドキュメントのナビゲーションバーがあります。ドキュメントのページは、アプリケーションを構築する際にステップバイステップで進められるように、基本から応用へと順序よく整理されています。ただし、どの順番でも読んでいただくことができ、必要に応じてページを飛ばすこともできます。

画面の右側には、ページのセクション間を簡単に移動できる目次が表示されます。ページをすばやく見つけたい場合は、画面上部の検索バーや検索ショートカット（`Ctrl+K` または `Cmd+K`）を使用できます。

まずは、[インストールガイド](/docs/app/getting-started/installation)をご覧ください。

## App Router と Pages Router {#app-router-vs-pages-router}

Next.js には、App Router と Pages Router の 2 つの異なるルーターがあります。App Router は、Server Components や Streaming など React の最新機能を利用できる新しいルーターです。Pages Router は、サーバーレンダリングされた React アプリケーションを構築できるオリジナルの Next.js ルーターであり、古い Next.js アプリケーションで引き続きサポートされています。

サイドバーの上部には、**App Router** と **Pages Router** の機能を切り替えるためのドロップダウンメニューがあります。各ディレクトリに固有の機能があるため、選択されているタブを確認することが重要です。

ページ上部のパンくずリストも、App Router のドキュメントを見ているのか Pages Router のドキュメントを見ているのかを示します。

## 前提知識 {#pre-requisite-knowledge}

私たちのドキュメントは初心者に優しくデザインされていますが、Next.js の機能に焦点を当て続けるために、基礎を確立する必要があります。新しいコンセプトを紹介する際には、関連するドキュメントへのリンクを提供します。

ドキュメントを最大限に活用するには、HTML、CSS、React の基本的な理解があると良いでしょう。React のスキルを身につける必要がある場合は、[React Foundations コース](https://nextjs.org/learn/react-foundations)をご覧ください。基本を紹介しています。その後、[ダッシュボードアプリケーションの構築](https://nextjs.org/learn/dashboard-app)を通じて Next.js についての理解を深めてください。

## アクセシビリティ {#accessibility}

スクリーンリーダーを使用してドキュメントを読む際の最適なアクセシビリティを確保するために、Firefox と NVDA、または Safari と VoiceOver を使用することをお勧めします。
