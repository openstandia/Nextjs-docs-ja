ReactおよびNext.jsでは、いくつかの異なる種類のテストを書くことができ、それぞれに目的と使用ケースがあります。このページでは、アプリケーションをテストするために使用できる種類と一般的に使用されるツールの概要を提供します。

## テストの種類 {#types-of-tests}

- **単体テスト**は、個々のユニット（またはコードブロック）を独立してテストします。Reactでは、ユニットは単一の関数、フック、またはコンポーネントであることがあります
  - **コンポーネントテスト**は、ユニットテストの中でも、主にReactコンポーネントを対象とするテストです。これには、コンポーネントのレンダリング方法、propsとの相互作用、ユーザーイベントに応じた動作のテストが含まれることがあります
  - **統合テスト**は、複数のユニットが一緒にどのように動作するかをテストします。これにはコンポーネント、フック、関数の組み合わせを含むことがあります
- **エンドツーエンド（E2E）テスト**は、ブラウザーのような実際のユーザーシナリオを模擬した環境でユーザーフローをテストします。これにより、特定のタスク（例：サインアップフロー）を本番環境に近い環境でテストします
- **スナップショットテスト**は、コンポーネントのレンダリング出力をキャプチャし、それをスナップショットファイルに保存します。テストを実行する際、コンポーネントの現在のレンダリング出力は保存されたスナップショットと比較されます。スナップショットの変化は、動作が予期せぬ変更を示すために使用されます

## 非同期サーバーコンポーネント {#async-server-components}

`async` server componentがReactエコシステムにおいて新しいため、一部のツールでは完全にサポートされていません。現時点では、`async`コンポーネントには**エンドツーエンドテスト**を単体テストより優先して使用することをお勧めします。
