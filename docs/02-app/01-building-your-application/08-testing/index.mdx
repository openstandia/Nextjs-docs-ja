React と Next.js では、目的と使用ケースに応じて複数の種類のテストを書くことができます。このページでは、アプリケーションをテストするために使用できるテストの種類と、一般的に使用されるツールの概要を提供します。

## テストの種類

- **ユニットテスト** は、個々の単位（またはコードのブロック）を独立してテストするものです。React では、単位は単一の関数、フック、またはコンポーネントにすることができます
  - **コンポーネントテスト** は、ユニットテストのより集中した形であり、テストの主な対象は React コンポーネントになります。これは、コンポーネントのレンダリング方法、propsとの相互作用、ユーザーイベントに対するコンポーネントの反応をテストすることを含む場合があります
  - **統合テスト** は、複数の単位がどのように連携するかをテストするものです。これは、コンポーネント、フック、および関数の組み合わせになることがあります
- **エンドツーエンド（E2E）テスト** は、実際のユーザーシナリオをシミュレートした環境（ブラウザなど）でユーザーフローをテストするものです。これにより、実際の環境に近い環境で特定のタスク（例：サインアップフローなど）をテストすることができます
- **スナップショットテスト** は、コンポーネントのレンダリングされた出力をキャプチャしてスナップショットファイルに保存します。テストが実行されると、コンポーネントの現在のレンダリングされた出力が保存されたスナップショットと比較されます。スナップショットの変化は、予期しない動作の変化を示すために使用されます

## Async Server Components

`async` Server Components は React のエコシステムにおいて新しいため、一部のツールでは完全にサポートされていません。そのため、当面の間、`async` コンポーネントに対しては**ユニットテスト**ではなく**エンドツーエンドテスト**を使用することをお勧めします。
