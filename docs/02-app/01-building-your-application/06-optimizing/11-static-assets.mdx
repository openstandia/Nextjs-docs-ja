Next.js は、ルートディレクトリ内の `public` というフォルダにある画像のような静的ファイルを提供できます。`public` 内のファイルは、ベースURL (`/`) から開始してコード内で参照されます。

たとえば、ファイル `public/avatars/me.png` は `/avatars/me.png` パスを訪問することで見ることができます。その画像を表示するコードは次のようになるでしょう：

```jsx title="avatar.js"
import Image from 'next/image'

// 画像を表示するコンポーネント
export function Avatar({ id, alt }) {
  return <Image src={`/avatars/${id}.png`} alt={alt} width="64" height="64" />
}

// 私の画像を表示するコンポーネント
export function AvatarOfMe() {
  return <Avatar id="me" alt="A portrait of me" />
}
```

## キャッシュ

Next.js は `public` フォルダ内のアセットを安全にキャッシュすることができません。なぜなら、それらは変更される可能性があるからです。デフォルトで適用されるキャッシュヘッダーは次のとおりです：

```jsx
Cache-Control: public, max-age=0
```

## ロボット、ファビコン、およびその他

<PagesOnly>

このフォルダは `robots.txt`、`favicon.ico`、Google サイト認証、およびその他の静的ファイル (`.html` を含む) にも便利です。ただし、`pages/` ディレクトリ内のファイルと同じ名前の静的ファイルを持たないようにしてください。これによりエラーが発生します。[詳しく読む](https://nextjs.org/docs/messages/conflicting-public-file-page)

</PagesOnly>

<AppOnly>

静的メタデータファイル、例えば `robots.txt`、`favicon.ico` などの場合は、`app` フォルダ内に [特別なメタデータファイル](/docs/app/api-reference/file-conventions/metadata)を使用する必要があります。

</AppOnly>

> Good to know:
>
> - ディレクトリは `public` と名付けられなければなりません。名前は変更できず、静的アセットを提供するために使用される唯一のディレクトリです。
> - [ビルド時](/docs/app/api-reference/cli/next#next-build-options)に `public` ディレクトリにあるアセットのみが Next.js によって提供されます。リクエスト時に追加されたファイルは利用できません。永続的なファイルストレージには、[Vercel Blob](https://vercel.com/docs/storage/vercel-blob?utm_source=next-site&utm_medium=docs&utm_campaign=next-website)のようなサードパーティサービスの使用を推奨します。
