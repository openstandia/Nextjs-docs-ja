Next.jsは、ルートディレクトリにある`public`というフォルダー内で、画像などの静的ファイルをサーブすることができます。`public`内のファイルは、ベースURL (`/`) から始めてコード内で参照できます。

例えば、ファイル`public/avatars/me.png`は`/avatars/me.png`パスで閲覧できます。この画像を表示するコードは次のようになります：

```jsx title="avatar.js"
import Image from 'next/image'

// アバターを表示するコンポーネント
export function Avatar({ id, alt }) {
  return <Image src={`/avatars/${id}.png`} alt={alt} width="64" height="64" />
}

// 自分のアバターを表示するコンポーネント
export function AvatarOfMe() {
  return <Avatar id="me" alt="私の肖像画" />
}
```

## キャッシュ

`public`フォルダー内のアセットは変更される可能性があるため、Next.jsは安全にキャッシュできません。デフォルトのキャッシュヘッダーは次のように適用されます：

```jsx
Cache-Control: public, max-age=0
```

## ロボット、ファビコン、およびその他

<PagesOnly>

このフォルダーは、`robots.txt`、`favicon.ico`、Googleサイト認証、その他の静的ファイル（`.html`を含む）にも便利です。ただし、`pages/`ディレクトリのファイルと同じ名前の静的ファイルを持たないようにしてください。これはエラーの原因となります。[詳しくはこちらをお読みください](/docs/messages/conflicting-public-file-page).

</PagesOnly>

<AppOnly>

`robots.txt`、`favicon.ico` などの静的メタデータファイルには、`app`フォルダー内の[特別なメタデータファイル](/docs/app/api-reference/file-conventions/metadata)を使用する必要があります。

</AppOnly>

> Good to know:
>
> - このディレクトリ名は`public`でなければなりません。この名前は変更できず、静的アセットをサーブするための唯一のディレクトリです。
> - [ビルド時](/docs/app/api-reference/cli/next#next-build-options)に`public`ディレクトリにあるアセットのみがNext.jsによってサーブされます。リクエスト時に追加されたファイルは使用できません。永続的なファイルストレージには、[Vercel Blob](https://vercel.com/docs/storage/vercel-blob?utm_source=next-site&utm_medium=docs&utm_campaign=next-website)などのサードパーティサービスの使用をお勧めします。
