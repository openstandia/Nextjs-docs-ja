Next.js には、アプリケーションの速度や[Core Web Vitals](https://web.dev/vitals/)を改善するためのさまざまな組み込みの最適化機能が用意されています。このガイドでは、ユーザーエクスペリエンスを向上させるために活用できる最適化について説明します。

## 内蔵コンポーネント

内蔵コンポーネントは、一般的なUI最適化を実装する際の複雑さを隠蔽します。以下のコンポーネントがあります：

- **Images**: ネイティブの `<img>` 要素に基づいています。Image コンポーネントは、遅延読み込みやデバイスサイズに基づく自動画像のリサイズによってパフォーマンスを最適化します
- **Link**: ネイティブの `<a>` タグに基づいています。Link コンポーネントはバックグラウンドでページをプリフェッチし、より速く、よりスムーズなページ遷移を提供します
- **Scripts**: ネイティブの `<script>` タグに基づいています。Script コンポーネントは、サードパーティスクリプトの読み込みと実行を制御するためのオプションを提供します

## メタデータ

メタデータは検索エンジンがコンテンツをよりよく理解するのに役立ち（結果として SEO が向上する可能性があります）、ソーシャルメディア上でコンテンツの表示をカスタマイズすることで、さまざまなプラットフォームでのユーザーエクスペリエンスをより魅力的で一貫性のあるものにします。

<AppOnly>

Next.js の Metadata API を使用すると、ページの `<head>` 要素を変更できます。メタデータを設定する方法は2つあります：

- **設定ベースのメタデータ**: `layout.js` または `page.js` ファイルに[静的な `metadata` オブジェクト](/docs/app/api-reference/functions/generate-metadata#metadata-object)または動的な [`generateMetadata` 関数](/docs/app/api-reference/functions/generate-metadata#generatemetadata-function)をエクスポートします
- **ファイルベースのメタデータ**: 静的または動的に生成された特別なファイルをルートセグメントに追加します

さらに、[imageResponse](/docs/app/api-reference/functions/image-response) コンストラクターを用いて、JSX と CSS を使用して動的な Open Graph 画像を生成できます。

</AppOnly>

<PagesOnly>

Next.js の Head コンポーネントを使用すると、ページの `<head>` を変更できます。詳細は [Head コンポーネント](/docs/pages/api-reference/components/head) のドキュメントをご覧ください。

</PagesOnly>

## 静的アセット

Next.js の `/public` フォルダを使用して、画像やフォント、その他のファイルなどの静的アセットを提供できます。`/public` 内のファイルは CDN プロバイダーによってキャッシュされ、高効率で配信されるようにすることもできます。
