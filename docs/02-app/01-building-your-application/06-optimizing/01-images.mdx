[Web Almanac](https://almanac.httparchive.org)によると、画像は典型的なWebサイトの[ページ重量](https://almanac.httparchive.org/en/2022/page-weight#content-type-and-file-formats)の大部分を占め、Webサイトの[LCPパフォーマンス](https://almanac.httparchive.org/en/2022/performance#lcp-image-optimization)に大きな影響を与える可能性があります。

Next.jsのImageコンポーネントは、自動画像最適化の機能を備えたHTML `<img>`要素を拡張します：

- **サイズの最適化:** 各デバイスに適切なサイズの画像を自動的に提供し、WebPやAVIFのような最新の画像フォーマットを使用します
- **視覚的安定性:** 画像が読み込まれるときに[レイアウトシフト](https://nextjs.org/learn/seo/web-performance/cls)を自動的に防ぎます
- **より速いページロード:** 画像は、ビューに入ったときにネイティブブラウザの遅延読み込みを使用して読み込まれます。オプションでぼかしプラセホルダーを使用できます
- **アセットの柔軟性:** リモートサーバーに保存された画像にも対応する、オンデマンドの画像サイズ変更

> **🎥 視聴:** `next/image` の使い方についてもっと学びましょう → [YouTube (9 分)](https://youtu.be/IU_qq_c_lKA)。

## 使用方法 {#usage}

```js
import Image from 'next/image'
```

次に、`src`としてローカルまたはリモート画像を定義できます。

### ローカル画像 {#local-images}

ローカル画像を使用するには、`.jpg`、`.png`、または`.webp`画像ファイルを`import`します。

Next.js は、インポートされたファイルに基づいて画像の固有の`width`と`height`を[自動的に決定](#image-sizing)します。これらの値は、画像比率を決定し、画像の読み込み中に[累積的なレイアウトシフト](https://nextjs.org/learn/seo/web-performance/cls)を防ぐのに使用されます。

<AppOnly>

```jsx title="app/page.js"
import Image from 'next/image'
import profilePic from './me.png'

export default function Page() {
  return (
    <Image
      src={profilePic}
      alt="筆者の写真"
      // width={500} は自動的に提供されます
      // height={500} は自動的に提供されます
      // blurDataURL="data:..." は自動的に提供されます
      // placeholder="blur" // 読み込み中のオプションのぼかし表示
    />
  )
}
```

</AppOnly>

<PagesOnly>

```jsx title="pages/index.js"
import Image from 'next/image'
import profilePic from '../public/me.png'

export default function Page() {
  return (
    <Image
      src={profilePic}
      alt="筆者の写真"
      // width={500} は自動的に提供されます
      // height={500} は自動的に提供されます
      // blurDataURL="data:..." は自動的に提供されます
      // placeholder="blur" // 読み込み中のオプションのぼかし表示
    />
  )
}
```

</PagesOnly>

> **警告:** 動的な `await import()` または `require()` はサポートされていません。`import` はビルド時に解析できるように静的でなければなりません。

特定の画像を許可し、他のすべてをブロックするために、`next.config.js` ファイルで`localPatterns`をオプションで設定することができます。

```js title="next.config.js"
module.exports = {
  images: {
    localPatterns: [
      {
        pathname: '/assets/images/**',
        search: '',
      },
    ],
  },
}
```

### リモート画像 {#remote-images}

リモート画像を使用する場合、`src`プロパティはURL文字列でなければなりません。

Next.js はビルドプロセス中にリモートファイルにアクセスできないため、[`width`](/docs/app/api-reference/components/image#width)、[`height`](/docs/app/api-reference/components/image#height)、およびオプションの[`blurDataURL`](/docs/app/api-reference/components/image#blurdataurl)のpropsを手動で提供する必要があります。

`width`と`height`属性は画像の正しいアスペクト比を推測し、画像の読み込みによるレイアウトシフトを防ぐために使用されます。`width`と`height`は画像ファイルの描画サイズを決定しません。[画像のサイズ設定](#image-sizing)についてさらに学んでください。

```jsx title="app/page.js"
import Image from 'next/image'

export default function Page() {
  return (
    <Image
      src="https://s3.amazonaws.com/my-bucket/profile.png"
      alt="筆者の写真"
      width={500}
      height={500}
    />
  )
}
```

画像の最適化を安全に許可するには、`next.config.js`でサポートされているURLパターンのリストを定義します。不正使用を防ぐためにできるだけ具体的にします。たとえば、以下の設定では特定のAWS S3バケットの画像のみが許可されます：

```js title="next.config.js"
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 's3.amazonaws.com',
        port: '',
        pathname: '/my-bucket/**',
      },
    ],
  },
}
```

[`remotePatterns`](/docs/app/api-reference/components/image#remotepatterns)の設定についてさらに学びましょう。画像`src`に相対URLを使用したい場合は、[`loader`](/docs/app/api-reference/components/image#loader)を使用してください。

### ドメイン {#domains}

リモート画像を最適化したいが、Next.js の組み込みの画像最適化APIを使用したい場合があります。この場合、`loader`をデフォルト設定のままにし、画像`src`プロパティに絶対URLを入力します。

アプリケーションを悪意のあるユーザーから保護するために、`next/image`コンポーネントで使用しようとしているリモートホスト名のリストを定義する必要があります。

> [`remotePatterns`](/docs/app/api-reference/components/image#remotepatterns)の設定についてさらに学びましょう。

### ローダー {#loaders}

[先ほどの例](#local-images)では、ローカル画像に部分URL（`"/me.png"`）が提供されています。これはローダーのアーキテクチャのおかげで可能です。

ローダーは画像のURLを生成する関数です。提供された`src`を修正し、異なるサイズで画像を要求する複数のURLを生成します。これらの複数のURLは自動の[srcset](https://developer.mozilla.org/docs/Web/API/HTMLImageElement/srcset)生成で使用され、サイトの訪問者にはビューポートに適したサイズの画像が提供されます。

Next.jsアプリケーションのデフォルトのローダーは、組み込みの画像最適化APIを使用して、Web上のどこからでも画像を最適化し、それをNext.jsのWebサーバーから直接提供します。CDNや画像サーバーから画像を直接提供したい場合は、JavaScriptの数行で独自のローダー関数を書くことができます。

[`loader` prop](/docs/app/api-reference/components/image#loader)を使用して画像ごとに、またはアプリケーションレベルで[`loaderFile` configuration](/docs/app/api-reference/components/image#loaderfile)を使用してローダーを定義できます。

## 優先順位 {#priority}

各ページの[最大視覚的コンテンツ (LCP) 要素](https://web.dev/lcp/#what-elements-are-considered)となる画像には、`priority`プロパティを追加するべきです。これにより、Next.jsは画像の読み込みを特別に優先し（プリロードタグや優先ヒントなどを通じて）、LCPの大幅な向上をもたらします。

LCP要素は通常、ページのビューポート内に表示される最大の画像またはテキストブロックです。`next dev`を実行すると、LCP要素が`<Image>`であるが`priority`プロパティがない場合、コンソールに警告が表示されます。

LCP画像を特定したら、このようにプロパティを追加できます：

<PagesOnly>

```jsx title="app/page.js"
import Image from 'next/image'

export default function Home() {
  return (
    <>
      <h1>私のホームページ</h1>
      <Image src="/me.png" alt="筆者の写真" width={500} height={500} priority />
      <p>私のホームページへようこそ！</p>
    </>
  )
}
```

</PagesOnly>

<AppOnly>

```jsx title="app/page.js"
import Image from 'next/image'
import profilePic from '../public/me.png'

export default function Page() {
  return <Image src={profilePic} alt="筆者の写真" priority />
}
```

</AppOnly>

[`next/image`コンポーネントドキュメント](/docs/app/api-reference/components/image#priority)にて優先順位についてさらに詳しくご覧ください。

## 画像のサイズ設定 {#image-sizing}

画像がパフォーマンスを最も損なう一般的な方法の一つは、*レイアウトシフト*を引き起こすことです。これは、画像が読み込まれるときにページ上の他の要素を押しのけてしまうことです。このパフォーマンスの問題はユーザーにとって非常に煩わしいため、[累積レイアウトシフト](https://web.dev/cls/)と呼ばれる独自のコアウェブバイタルがあります。画像ベースのレイアウトシフトを回避する方法は、[常に画像のサイズを設定すること](https://web.dev/optimize-cls/#images-without-dimensions)です。これにより、ブラウザは画像が読み込まれる前に、正確にスペースを予約できます。

`next/image`は優れたパフォーマンス結果を保証するために設計されているため、レイアウトシフトに寄与するような方法では使用できず、**次の3つの方法のいずれかでサイズを設定する必要があります**。

1. [静的な import](#local-images) を使用して自動的に設定
2. 画像のアスペクト比を決定するために使用される[`width`](/docs/app/api-reference/components/image#width)および[`height`](/docs/app/api-reference/components/image#height)プロパティを含めて手動で設定
3. [fill](/docs/app/api-reference/components/image#fill)を使用して暗黙的に、親要素を埋める画像を拡張する

> **画像のサイズが分からない場合はどうすればよいですか？**
>
> 画像のサイズが分からないソースから画像にアクセスしている場合、いくつかの方法があります。
>
> **`fill` の使用**
>
> [`fill`](/docs/app/api-reference/components/image#fill) プロパティを使用すると、画像を親要素によってサイズ設定できます。CSSを使用して画像の親要素にページ上のスペースを与え、メディアクエリブレークポイントを一致させるための[`sizes`](/docs/app/api-reference/components/image#sizes)プロパティを考慮してください。また、[`object-fit`](https://developer.mozilla.org/docs/Web/CSS/object-fit) を `fill`、`contain`、`cover`のいずれかと組み合わせて使用し、[`object-position`](https://developer.mozilla.org/docs/Web/CSS/object-position)を使用して画像がそのスペースにどのように収まるべきかを定義できます。
>
> **画像の正規化**
>
> 制御可能なソースから画像を提供している場合、画像パイプラインを変更して画像を特定のサイズに正規化することを検討してください。
>
> **API呼び出しの変更**
>
> アプリケーションがAPI呼び出しを使用して画像URLを取得している場合（たとえばCMSへの呼び出し）、画像のURLと一緒に画像の寸法を返すようにAPI呼び出しを変更できるかもしれません。

提案された方法がどれも画像のサイズ設定に機能しない場合、`next/image`コンポーネントは標準の`<img>`要素と一緒にページ上で正常に動作するように設計されています。

## スタイリング {#styling}

Image コンポーネントのスタイリングは通常の `<img>` 要素のスタイリングと似ていますが、いくつか注意すべきガイドラインがあります：

- `className` または `style` を使用し、`styled-jsx` を使用しない
  - 多くの場合、`className` prop を使用することをお勧めします。これは、インポートされた[CSS モジュール](/docs/app/building-your-application/styling/css)、[グローバルスタイルシート](/docs/app/building-your-application/styling/css#global-styles) などである可能性があります
  - インラインスタイルを割り当てるために `style` prop も使用できます
  - [styled-jsx](/docs/app/building-your-application/styling/css-in-js)を使用することはできません（スタイルをグローバルとしてマークしない限り）、これは現在のコンポーネントにスコープされているためです
- `fill` を使用する場合、親要素には `position: relative` が必要
  - これはそのレイアウトモードで画像要素を正しくレンダリングするために必要です
- `fill` を使用する場合、親要素には `display: block` が必要
  - これは `<div>` 要素のデフォルトですが、そうでない場合は指定する必要があります

## 例 {#examples}

### レスポンシブデザイン {#responsive}

<Image
  alt="親コンテナの幅と高さを満たしているレスポンシブ画像"
  srcLight="/docs/light/responsive-image.png"
  srcDark="/docs/dark/responsive-image.png"
  width="1600"
  height="629"
/>

```jsx
import Image from 'next/image'
import mountains from '../public/mountains.jpg'

export default function Responsive() {
  return (
    <div style={{ display: 'flex', flexDirection: 'column' }}>
      <Image
        alt="山々"
        // 画像をインポートすると幅と高さが自動的に設定されます
        src={mountains}
        sizes="100vw"
        // 画像を全幅で表示します
        style={{
          width: '100%',
          height: 'auto',
        }}
      />
    </div>
  )
}
```

### コンテナを埋める {#fill-container}

<Image
  alt="親コンテナの幅を満たす画像のグリッド"
  srcLight="/docs/light/fill-container.png"
  srcDark="/docs/dark/fill-container.png"
  width="1600"
  height="529"
/>

```jsx
import Image from 'next/image'
import mountains from '../public/mountains.jpg'

export default function Fill() {
  return (
    <div
      style={{
        display: 'grid',
        gridGap: '8px',
        gridTemplateColumns: 'repeat(auto-fit, minmax(400px, auto))',
      }}
    >
      <div style={{ position: 'relative', height: '400px' }}>
        <Image
          alt="山々"
          src={mountains}
          fill
          sizes="(min-width: 808px) 50vw, 100vw"
          style={{
            objectFit: 'cover', // cover, contain, none
          }}
        />
      </div>
      {/* グリッド内の他の画像... */}
    </div>
  )
}
```

### 背景画像 {#background-image}

<Image
  alt="ページの全幅と高さを占有する背景画像"
  srcLight="/docs/light/background-image.png"
  srcDark="/docs/dark/background-image.png"
  width="1600"
  height="427"
/>

```jsx
import Image from 'next/image'
import mountains from '../public/mountains.jpg'

export default function Background() {
  return (
    <Image
      alt="山々"
      src={mountains}
      placeholder="blur"
      quality={100}
      fill
      sizes="100vw"
      style={{
        objectFit: 'cover',
      }}
    />
  )
}
```
