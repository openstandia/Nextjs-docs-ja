ページとは、ルートに**固有の**UIです。`page.js`ファイルからコンポーネントをデフォルトエクスポートすることによってページを定義できます。

例えば、`index`ページを作成するには、`app`ディレクトリ内に`page.js`ファイルを追加します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
// `app/page.tsx` は `/` URL の UI です
export default function Page() {
  return <h1>Hello, Home page!</h1>
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher
// `app/page.js` は `/` URL の UI です
export default function Page() {
  return <h1>Hello, Home page!</h1>
}
```
</TabItem>
</Tabs>


次に、さらにページを作成するには、新しいフォルダを作成し、その中に`page.js`ファイルを追加します。例えば、`/dashboard`ルート用のページを作成するには、`dashboard`という名前の新しいフォルダを作成し、その中に`page.js`ファイルを追加します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/dashboard/page.tsx" switcher
// `app/dashboard/page.tsx` は `/dashboard` URL の UI です
export default function Page() {
  return <h1>Hello, Dashboard Page!</h1>
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/dashboard/page.js" switcher
// `app/dashboard/page.js` は `/dashboard` URL の UI です
export default function Page() {
  return <h1>Hello, Dashboard Page!</h1>
}
```
</TabItem>
</Tabs>


> **Good to know**:
>
> - Pagesには`.js`、`.jsx`、または`.tsx`のファイル拡張子を使用できます
> - ページは常に[ルートサブツリー](/docs/app/building-your-application/routing#terminology)の[leaf](/docs/app/building-your-application/routing#terminology)です
> - ルートセグメントを公開するには`page.js`ファイルが必要です
> - ページはデフォルトで[Server Components](/docs/app/building-your-application/rendering/server-components)ですが、[Client Component](/docs/app/building-your-application/rendering/client-components)に設定することもできます
> - ページはデータを取得できます。詳細については[データ取得](/docs/app/building-your-application/data-fetching)のセクションを参照してください
