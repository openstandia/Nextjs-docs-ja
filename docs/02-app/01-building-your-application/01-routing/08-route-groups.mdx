title: 'Route Groups'

---

`app` ディレクトリ内のネストされたフォルダーは通常、URL パスにマッピングされます。しかし、フォルダーを **Route Group** としてマークすることで、そのフォルダーがルートの URL パスに含まれないようにすることが可能です。

これは、URL パスの構造に影響を与えることなく、ルートのセグメントとプロジェクトのファイルを論理的なグループにまとめることを可能にします。

Route groups は以下の用途に役立ちます：

- [ルートをグループにまとめる](#organize-routes-without-affecting-the-url-path)（例：サイトセクション、目的、チームごと）
- 同じルートセグメントレベルでの [ネストされたレイアウト](/docs/app/building-your-application/routing/layouts-and-templates) の有効化：
  - [同じセグメント内で複数のネストされたレイアウト、複数の root レイアウトを作成する](#creating-multiple-root-layouts)
  - [共通のセグメント内のルートのサブセットにレイアウトを追加する](#opting-specific-segments-into-a-layout)

## 規約

Route group は、フォルダー名を括弧で囲むことで作成できます：`(folderName)`

## 例

### URL パスに影響を与えずにルートを整理する

URL に影響を与えることなくルートを整理するには、関連するルートをまとめるためにグループを作成します。括弧で囲まれたフォルダーは URL から省略されます（例：`(marketing)` や `(shop)`）。

### 特定のセグメントをレイアウトに取り込む

特定のルートをレイアウトに取り込むには、新しい route group を作成し（例：`(shop)`）、同じレイアウトを共有するルートをグループに移動します（例：`account` と `cart`）。グループ外のルートはレイアウトを共有しません（例：`checkout`）。

### 複数の root レイアウトを作成する

複数の [root レイアウト](/docs/app/building-your-application/routing/layouts-and-templates#root-layout-required) を作成するには、トップレベルの `layout.js` ファイルを削除し、各 route group 内に `layout.js` ファイルを追加します。これは、完全に異なる UI やエクスペリエンスを持つセクションにアプリケーションを分割するのに役立ちます。各 root レイアウトに `<html>` と `<body>` タグを追加する必要があります。

上記の例では、`(marketing)` と `(shop)` の両方が独自の root レイアウトを持っています。

---

> **Good to know**:
>
> - Route group の名前には、組織化以外に特別な意味はありません。URL パスには影響を与えません。
> - Route group を含むルートは、他のルートと同じ URL パスに解決されるべき **ではありません**。たとえば、route group が URL 構造に影響を与えないため、`(marketing)/about/page.js` と `(shop)/about/page.js` の両方が `/about` に解決され、エラーが発生します。
> - トップレベルの `layout.js` ファイルがない場合に複数の root レイアウトを使用する場合、ホーム `page.js` ファイルは route group のいずれかに定義されているべきです。例：`app/(marketing)/page.js`。
> - **複数の root レイアウトを横断してナビゲートする**と、**フルページロード**が発生します（クライアントサイドのナビゲーションとは対照的に）。たとえば、`app/(shop)/layout.js` を使用する `/cart` から、`app/(marketing)/layout.js` を使用する `/blog` にナビゲートすると、フルページロードが発生します。これは**複数の root レイアウト**にのみ適用されます。
