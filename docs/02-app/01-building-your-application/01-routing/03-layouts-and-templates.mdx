特別なファイルである[layout.js](#layouts)と[template.js](#templates)を使用することで、複数の[ルート](/docs/app/building-your-application/routing/defining-routes#creating-routes)間で共有されるUIを作成できます。このページでは、これらの特別なファイルをどのように、いつ使用するかを案内します。

## Layouts

レイアウトは複数のルート間で**共有**されるUIです。ナビゲーション時に、レイアウトは状態を保持し、インタラクティブに動作し、再レンダリングされません。レイアウトは[ネスト](#nesting-layouts)することもできます。

`layout.js`ファイルからReactコンポーネントをデフォルトでエクスポートすることで、レイアウトを定義できます。このコンポーネントは、レンダリング中に子レイアウト（存在する場合）やページとなる`children` propを受け取るべきです。

たとえば、レイアウトは`/dashboard`および`/dashboard/settings`ページと共有されます。

### Root Layout (必須)

root レイアウトは`app`ディレクトリのトップレベルで定義され、すべてのルートに適用されます。このレイアウトは**必須**であり、`html`および`body`タグを含める必要があり、サーバーから返される初期のHTMLを変更できるようにします。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="ja">
      <body>
        {/* レイアウト UI */}
        <main>{children}</main>
      </body>
    </html>
  )
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html lang="ja">
      <body>
        {/* レイアウト UI */}
        <main>{children}</main>
      </body>
    </html>
  )
}
```
</TabItem>
</Tabs>


### レイアウトのネスト

デフォルトでは、フォルダ階層内のレイアウトは**ネスト**されます。これは`children` propを通じて子レイアウトをラップすることを意味します。特定のルートセグメント（フォルダ）内に`layout.js`を追加することで、レイアウトをネストすることができます。

たとえば、`/dashboard`ルートのレイアウトを作成するには、`dashboard`フォルダの内部に新しい`layout.js`ファイルを追加します。

```tsx title="app/dashboard/layout.tsx" switcher

  children,
}: {
  children: React.ReactNode

  return <section>{children}</section>
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/dashboard/layout.js" switcher

  return <section>{children}</section>

```
</TabItem>

上記の2つのレイアウトを組み合わせると、root レイアウト（`app/layout.js`）がダッシュボードレイアウト（`app/dashboard/layout.js`）をラップし、`app/dashboard/*`内のルートセグメントをラップします。

> **Good to know**:
> 
> - レイアウトには`.js`、`.jsx`、または`.tsx`ファイル拡張子が使用できます
> - `<html>`および`<body>`タグを含められるのはroot レイアウトのみです
> - 同じフォルダに`layout.js`と`page.js`ファイルが定義されている場合、レイアウトがページをラップします
> - レイアウトはデフォルトで[Server Components](/docs/app/building-your-application/rendering/server-components)ですが、[Client Component](/docs/app/building-your-application/rendering/client-components)に設定できます
> - レイアウトはデータを取得できます。詳細は[Data Fetching](/docs/app/building-your-application/data-fetching)セクションをご覧ください
> - 親レイアウトとその子孫間でデータを渡すことはできません。ただし、あるルートで同じデータを複数回取得することは可能であり、Reactはパフォーマンスに影響を与えずに[リクエストの重複排除](/docs/app/building-your-application/caching#request-memoization)を自動的に行います
> - レイアウトは`pathname`にアクセスできません。([詳細はこちら](/docs/app/api-reference/file-conventions/layout)) ただし、インポートされたクライアントコンポーネントは[`usePathname`](/docs/app/api-reference/functions/use-pathname)フックを使用してパス名にアクセスできます
> - レイアウトはそれ自体の下にあるルートセグメントにアクセスできません。すべてのルートセグメントにアクセスするには、クライアントコンポーネントで[`useSelectedLayoutSegment`](/docs/app/api-reference/functions/use-selected-layout-segment)または[`useSelectedLayoutSegments`](/docs/app/api-reference/functions/use-selected-layout-segments)を使用できます
> - [Route Groups](/docs/app/building-your-application/routing/route-groups)を使用して特定のルートセグメントを共有レイアウトに含めたり除外したりできます
> - [Route Groups](/docs/app/building-your-application/routing/route-groups)を使用して複数のroot レイアウトを作成できます。[こちらの例を参照してください](/docs/app/building-your-application/routing/route-groups#creating-multiple-root-layouts)
> - **`pages`ディレクトリからの移行**: root レイアウトは[`_app.js`](https://nextjs.org/docs/canary/pages/building-your-application/routing/custom-app)および[`_document.js`](https://nextjs.org/docs/canary/pages/building-your-application/routing/custom-document)ファイルを置き換えます。[移行ガイドを確認する](/docs/app/building-your-application/upgrading/app-router-migration#migrating-_documentjs-and-_appjs)

## Templates

Templatesは、子レイアウトやページをラップする点でレイアウトに似ています。異なる点は、ルート全体で状態を維持し、状態を管理するのではなく、ナビゲーションごとに子に対する新しいインスタンスを作成する点です。つまり、テンプレートを共有するルート間をナビゲートすると、子の新しいインスタンスがマウントされ、DOM要素が再作成され、クライアントコンポーネントでは状態が**保存されません**。効果は再同期されます。

これらの特定の動作が必要な場合があり、テンプレートはレイアウトよりも適した選択肢になります。たとえば:

- ナビゲーション時に`useEffect`を再同期するため
- ナビゲーション時に子クライアントコンポーネントの状態をリセットするため

テンプレートは`template.js`ファイルからデフォルトでReactコンポーネントをエクスポートすることによって定義できます。このコンポーネントは、`children` propを受け取るべきです。

テンプレートのネストに関しては、`template.js`はレイアウトとその子孫の間でレンダリングされます。以下は簡略化された出力例です：

```jsx title="Output"
<Layout>
  {/* テンプレートにはユニークなキーが割り当てられます。 */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```

## 例

### メタデータ

`title` や `meta` などの `<head>` HTML要素を[メタデータAPI](/docs/app/building-your-application/optimizing/metadata)を使用して変更できます。

メタデータは、[`metadata`オブジェクト](/docs/app/api-reference/functions/generate-metadata#the-metadata-object)や[`generateMetadata`関数](/docs/app/api-reference/functions/generate-metadata#generatemetadata-function)を[`layout.js`](/docs/app/api-reference/file-conventions/layout)や[`page.js`](/docs/app/api-reference/file-conventions/page)ファイルでエクスポートすることで定義できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Next.js',
}

export default function Page() {
  return '...'
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher
export const metadata = {
  title: 'Next.js',
}

export default function Page() {
  return '...'
}
```
</TabItem>
</Tabs>


> **Good to know**: `<title>` や `<meta>` のような `<head>` タグを手動でroot レイアウトに追加してはいけません。代わりに、[メタデータAPI](/docs/app/api-reference/functions/generate-metadata)を使用してください。これにより、ストリーミングや `<head>` 要素の重複排除など、高度な要件が自動的に処理されます。

利用可能なメタデータオプションについては、[APIリファレンス](/docs/app/api-reference/functions/generate-metadata)をご確認ください。

### アクティブナビリンク

[usePathname()](/docs/app/api-reference/functions/use-pathname) フックを使用してナビリンクがアクティブかどうかを判断できます。

`usePathname()`はクライアントフックであるため、ナビリンクをクライアントコンポーネントに抽出し、それをレイアウトやテンプレートにインポートする必要があります。


<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/ui/nav-links.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'
import Link from 'next/link'

export function NavLinks() {
  const pathname = usePathname()

  return (
    <nav>
      <Link className={`link ${pathname === '/' ? 'active' : ''}`} href="/">
        Home
      </Link>

      <Link
        className={`link ${pathname === '/about' ? 'active' : ''}`}
        href="/about"
      >
        About
      </Link>
    </nav>
  )
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

