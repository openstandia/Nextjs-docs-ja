## ルートの作成

Next.jsは、ファイルシステムベースのルーターを使用しており、**フォルダ**がルートを定義するために使われます。

各フォルダは、**URL** セグメントにマップされる[**ルート**セグメント](/docs/app/building-your-application/routing#route-segments)を表しています。[ネストされたルート](/docs/app/building-your-application/routing#nested-routes)を作成するには、フォルダを入れ子にすることができます。

ルートセグメントを公開するためには、特別な[`page.js`ファイル](/docs/app/building-your-application/routing/pages)を使用します。

この例では、`/dashboard/analytics` URLパスは対応する`page.js`ファイルがないため、公開されていません。このフォルダは、コンポーネント、スタイルシート、画像、または他のコロケートされたファイルを保存するために使用できます。

> **Good to know**: 特別なファイルには`.js`, `.jsx`, `.ts`, または `.tsx` ファイル拡張子を使用できます。

## UIの作成

各ルートセグメントのUIを作成するために、[特別なファイル規約](/docs/app/building-your-application/routing#file-conventions)が使用されます。最も一般的なのは、ルートに固有のUIを表示する[ページ](/docs/app/building-your-application/routing/pages)と、複数のルート間で共有されるUIを表示する[レイアウト](/docs/app/building-your-application/routing/layouts-and-templates#layouts)です。

例えば、最初のページを作成するには、`app`ディレクトリ内に`page.js`ファイルを追加し、Reactコンポーネントをエクスポートします：

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```
