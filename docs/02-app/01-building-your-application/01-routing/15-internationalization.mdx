title: '国際化'
description: '国際化されたルーティングとローカライズされたコンテンツで複数の言語のサポートを追加します。'
---

Next.jsを使用すると、複数の言語をサポートするためにコンテンツのルーティングとレンダリングを構成できます。サイトをさまざまなロケールに適応させるには、翻訳されたコンテンツ（ローカリゼーション）と国際化されたルートを含みます。

## 用語解説

- **ロケール:** 言語と書式設定の好みを示す識別子。通常、ユーザーの希望する言語と地理的な地域が含まれます。
  - `en-US`: アメリカ合衆国で話される英語
  - `nl-NL`: オランダで話されるオランダ語
  - `nl`: 特定の地域を指定しないオランダ語

## ルーティングの概要

ブラウザでのユーザーの言語設定を使用して、どのロケールを使用するか選択することを推奨します。希望する言語を変更すると、アプリケーションに対する`Accept-Language`ヘッダーが変更されます。

たとえば、以下のライブラリを使用して、受信した`Request`を見て、`Headers`、サポートしたいロケール、デフォルトロケールに基づいてどのロケールを選択するかを判断できます。

```js filename="middleware.js"
import { match } from '@formatjs/intl-localematcher'
import Negotiator from 'negotiator'

// ヘッダーを設定
let headers = { 'accept-language': 'en-US,en;q=0.5' }
// 優先される言語を取得
let languages = new Negotiator({ headers }).languages()
// サポートするロケールを設定
let locales = ['en-US', 'nl-NL', 'nl']
// デフォルトロケールを設定
let defaultLocale = 'en-US'

// 使用するロケールを判断
match(languages, locales, defaultLocale) // -> 'en-US'
```

ルーティングはサブパス（`/fr/products`）またはドメイン（`my-site.fr/products`）で国際化できます。この情報を使用して、[Middleware](/docs/app/building-your-application/routing/middleware)内でロケールに基づいてユーザーをリダイレクトすることができます。

```js filename="middleware.js"
import { NextResponse } from "next/server";

let locales = ['en-US', 'nl-NL', 'nl']

// 上記のように、またはライブラリを使用して、希望されるロケールを取得する
function getLocale(request) { ... }

export function middleware(request) {
  // パス名にサポートされるロケールがあるか確認
  const { pathname } = request.nextUrl
  const pathnameHasLocale = locales.some(
    (locale) => pathname.startsWith(`/${locale}/`) || pathname === `/${locale}`
  )

  if (pathnameHasLocale) return

  // ロケールがない場合はリダイレクトする
  const locale = getLocale(request)
  request.nextUrl.pathname = `/${locale}${pathname}`
  // 例: 受信リクエストが /products の場合
  // 新しいURLは /en-US/products になる
  return NextResponse.redirect(request.nextUrl)
}

export const config = {
  matcher: [
    // すべての内部パス（_next）をスキップ
    '/((?!_next).*)',
    // オプション: ルート（/）URLでのみ実行
    // '/'
  ],
}
```

最後に、`app/`内の特別なファイルがすべて`app/[lang]`の下にネストされていることを確認します。これにより、Next.jsのルーターはルート内の異なるロケールを動的に処理し、`lang`パラメーターをすべてのレイアウトやページに転送できます。たとえば：

```jsx filename="app/[lang]/page.js"
// 現在のロケールにアクセスできます
// 例: /en-US/products -> `lang` は "en-US"
export default async function Page({ params: { lang } }) {
  return ...
}
```

root レイアウトも新しいフォルダ（例: `app/[lang]/layout.js`）にネストできます。

## ローカリゼーション

ユーザーの希望ロケールに基づいて表示コンテンツを変更する、すなわちローカリゼーションは、Next.jsに特有のものではありません。以下に説明するパターンはどのWebアプリケーションでも同様に機能します。

アプリケーション内で英語とオランダ語の両方のコンテンツをサポートしたいと仮定しましょう。2つの異なる「辞書」を管理することができます。これは、あるキーからローカライズされた文字列へのマッピングを提供するオブジェクトです。たとえば：

```json filename="dictionaries/en.json"
{
  "products": {
    "cart": "Add to Cart"
  }
}
```

```json filename="dictionaries/nl.json"
{
  "products": {
    "cart": "Toevoegen aan Winkelwagen"
  }
}
```

次に、要求されたロケールの翻訳をロードするための`getDictionary`関数を作成できます。

```jsx filename="app/[lang]/dictionaries.js"
import 'server-only'

const dictionaries = {
  en: () => import('./dictionaries/en.json').then((module) => module.default),
  nl: () => import('./dictionaries/nl.json').then((module) => module.default),
}

export const getDictionary = async (locale) => dictionaries[locale]()
```

現在選択されている言語を考慮して、レイアウトやページ内で辞書を取得できます。

```jsx filename="app/[lang]/page.js"
import { getDictionary } from './dictionaries'

export default async function Page({ params: { lang } }) {
  const dict = await getDictionary(lang) // en
  return <button>{dict.products.cart}</button> // Add to Cart
}
```

`app/`ディレクトリ内のすべてのレイアウトとページはデフォルトで[Server Components](/docs/app/building-your-application/rendering/server-components)ですので、翻訳ファイルのサイズがクライアント側のJavaScriptバンドルのサイズに影響を与えることを心配する必要はありません。このコードは**サーバー上のみで実行され**、結果のHTMLのみがブラウザに送信されます。

## 静的生成

特定のロケールセットの静的ルートを生成するには、`generateStaticParams`を使用してページまたはレイアウトで使用できます。これは、たとえばルートレイアウトでグローバルに設定できます：

```jsx filename="app/[lang]/layout.js"
export async function generateStaticParams() {
  return [{ lang: 'en-US' }, { lang: 'de' }]
}

export default function Root({ children, params }) {
  return (
    <html lang={params.lang}>
      <body>{children}</body>
    </html>
  )
}
```
