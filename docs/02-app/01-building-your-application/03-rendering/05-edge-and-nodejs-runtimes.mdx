Next.jsには、アプリケーションで使用できる2つのサーバーランタイムがあります：

- **Node.js Runtime**（デフォルト）: これはすべてのNode.js APIとエコシステムからの互換性のあるパッケージにアクセスできます
- **Edge Runtime**: より限定された[APIのセット](/docs/app/api-reference/edge)を含みます

## ユースケース {#use-cases}

- Node.js Runtimeはアプリケーションのレンダリングに使用されます
- Edge Runtimeはミドルウェア（リダイレクト、リライト、ヘッダー設定などのルーティングルール）に使用されます

## 注意点 {#caveats}

- Edge RuntimeはすべてのNode.js APIをサポートしているわけではありません。いくつかのパッケージは期待通りに動作しない場合があります。サポートされていないAPIについては、[Edge Runtime](/docs/app/api-reference/edge#unsupported-apis)で詳しく学んでください
- Edge RuntimeはIncremental Static Regeneration（ISR）をサポートしていません
- どちらのランタイムも、デプロイ先のインフラストラクチャに依存して[ストリーミング](/docs/app/building-your-application/routing/loading-ui-and-streaming)をサポートできます
