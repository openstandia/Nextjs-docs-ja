Next.jsでは、アプリケーションで使用できる2つのサーバーランタイムがあります：

- **Node.js Runtime**（デフォルト）：Node.jsのすべてのAPIとエコシステム内の互換パッケージにアクセスできます
- **Edge Runtime**：もっと制限された[一連のAPI](/docs/app/api-reference/edge)を含んでいます

## ユースケース {#use-cases}

- Node.js Runtimeは、アプリケーションのレンダリングに使用されます
- Edge Runtimeは、ミドルウェア（リダイレクト、リライト、ヘッダの設定などのルーティングルール）で使用されます

## 注意点 {#caveats}

- Edge RuntimeはすべてのNode.js APIをサポートしているわけではありません。一部のパッケージは期待どおりに動作しない場合があります。サポートされていないAPIについて、詳しくは[Edge Runtime](/docs/app/api-reference/edge#unsupported-apis)をご確認ください
- Edge RuntimeはIncremental Static Regeneration (ISR)をサポートしていません
- 両方のランタイムは、デプロイメントインフラストラクチャに応じて[ストリーミング](/docs/app/building-your-application/routing/loading-ui-and-streaming)をサポートできます
