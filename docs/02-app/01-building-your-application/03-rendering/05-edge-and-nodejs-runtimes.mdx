Next.jsでは、アプリケーションで使用する2つのサーバーランタイムがあります：

- **Node.js Runtime**（デフォルト）で、すべてのNode.js APIおよびエコシステムからの互換性のあるパッケージにアクセスできます
- より限られた[APIセット](/docs/app/api-reference/edge)を含む**Edge Runtime**

## 使用例 {#use-cases}

- Node.js runtimeは、アプリケーションのレンダリングに使用されます
- Edge runtimeは、ミドルウェア（リダイレクト、リライト、ヘッダーの設定などのルーティングルール）に使用されます

## 注意点 {#caveats}

- Edge RuntimeはすべてのNode.js APIをサポートしているわけではありません。 一部のパッケージは期待どおりに動作しない可能性があります。 サポートされていないAPIについては、[Edge Runtime](/docs/app/api-reference/edge#unsupported-apis)で詳しく学んでください。
- Edge RuntimeはIncremental Static Regeneration (ISR)をサポートしていません。
- 両方のランタイムは、デプロイメントのインフラストラクチャによっては[ストリーミング](/docs/app/building-your-application/routing/loading-ui-and-streaming)をサポートできます。
