## 14 から 15 へのアップグレード {#upgrading-from-14-to-15}

Next.js バージョン 15 にアップデートするには、`upgrade` codemod を使用できます:

```bash title="Terminal"
npx @next/codemod@canary upgrade latest
```

手動で行う場合は、最新の Next と React RC をインストールしてください。例:

```bash title="Terminal"
npm i next@latest react@rc react-dom@rc eslint-config-next@latest
```

> **Good to know:**
>
> - ピア依存関係の警告が表示される場合は、`react` および `react-dom` を推奨されるバージョンに更新するか、`--force` または `--legacy-peer-deps` フラグを使用して警告を無視することができます。これは、Next.js 15 と React 19 が安定版になると不要になります。
> - TypeScript を使用している場合は、一時的に React タイプをオーバーライドする必要があります。詳細は [React 19 RC アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide#installing) を参照してください。

## React 19 {#react-19}

- `react` および `react-dom` の最低バージョンは 19 になりました。
- `useFormState` は `useActionState` に置き換えられました。 `useFormState` フックは React 19 でも利用可能ですが、非推奨とされており、将来のリリースで削除される予定です。 `useActionState` は推奨され、`pending` 状態を直接読み取ることができる追加のプロパティを含んでいます。 [詳細はこちら](https://react.dev/reference/react/useActionState)
- `useFormStatus` には、`data`、`method`、および `action` などの追加キーが含まれるようになりました。React 19 を使用していない場合は、`pending` キーのみが使用可能です。 [詳細はこちら](https://react.dev/reference/react-dom/hooks/useFormStatus)
- その他の情報は [React 19 アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide) を参照してください。

## 非同期リクエスト API（破壊的変更） {#async-request-apis-breaking-change}

以前はランタイム情報に依存していた同期の Dynamic API が、今後は**非同期**になります：

- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`draftMode`](/docs/app/api-reference/functions/draft-mode)
- [`layout.js`](/docs/app/api-reference/file-conventions/layout)、[`page.js`](/docs/app/api-reference/file-conventions/page)、[`route.js`](/docs/app/api-reference/file-conventions/route)、[`default.js`](/docs/app/api-reference/file-conventions/default)、[`opengraph-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`twitter-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)、および [`apple-icon`](/docs/app/api-reference/file-conventions/metadata/app-icons) の `params`
- [`page.js`](/docs/app/api-reference/file-conventions/page) の `searchParams`

移行の負担を軽減するため、プロセスを自動化するための [codemod が利用可能](/docs/app/building-your-application/upgrading/codemods#150) であり、一時的に API に同期的にアクセスすることができます。

### `cookies` {#cookies}

#### 推奨される非同期使用法 {#recommended-async-usage}

```tsx
import { cookies } from 'next/headers'

// 変更前
const cookieStore = cookies()
const token = cookieStore.get('token')

// 変更後
const cookieStore = await cookies()
const token = cookieStore.get('token')
```

## `runtime` 設定（破壊的変更） {#runtime-configuration-breaking-change}

`runtime` [segment config](/docs/app/api-reference/file-conventions/route-segment-config#runtime) には以前は `edge` に加えて `experimental-edge` の値がサポートされていました。両者は同じものを指しており、オプションを簡素化するために、`experimental-edge` を使用するとエラーが発生するようになります。これを修正するには、`runtime` 設定を `edge` に更新してください。これを自動化するための codemod が利用可能です。

</AppOnly>

## `fetch` リクエスト {#fetch-requests}

[`fetch` リクエスト](/docs/app/api-reference/functions/fetch) はデフォルトでキャッシュされなくなりました。

特定の `fetch` リクエストをキャッシュに選ぶには、`cache: 'force-cache'` オプションを渡すことができます。

```js title="app/layout.js"
export default async function RootLayout() {
  const a = await fetch('https://...') // 未キャッシュ
  const b = await fetch('https://...', { cache: 'force-cache' }) // キャッシュ済み

  // ...
}
```

`export const fetchCache = 'default-cache'`という [segment config オプション](/docs/app/api-reference/file-conventions/route-segment-config) を使用することで、レイアウトやページ内のすべての `fetch` リクエストをキャッシュに選ぶことができます。個々の `fetch` リクエストが `cache` オプションを指定している場合は、それが優先して使用されます。

```js title="app/layout.js"
// これは root レイアウトなので、アプリ内のすべての fetch リクエスト
// が独自のキャッシュオプションを設定しない限り、キャッシュされます。
export const fetchCache = 'default-cache'

export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュ済み
  const b = await fetch('https://...', { cache: 'no-store' }) // 未キャッシュ

  // ...
}
```

## Route Handlers {#route-handlers}

[Route Handlers](/docs/app/api-reference/file-conventions/route) の`GET`関数はデフォルトでキャッシュされなくなりました。`GET`メソッドをキャッシュに選ぶには、`export const dynamic = 'force-static'`のような [ルート設定オプション](/docs/app/api-reference/file-conventions/route-segment-config) を Route Handler ファイルで使用できます。

```js title="app/api/route.js"
export const dynamic = 'force-static'

export async function GET() {}
```

## クライアントサイドの Router Cache {#client-side-router-cache}

`<Link>` または `useRouter` を介してページ間を移動する際に、[ページ](/docs/app/api-reference/file-conventions/page)セグメントはクライアントサイドの router cache から再利用されなくなりました。ただし、ブラウザのバックワードおよびフォワードのナビゲーション中、および共有レイアウトに対しては依然として再利用されます。

ページセグメントをキャッシュに選ぶには、[`staleTimes`](/docs/app/api-reference/next-config-js/staleTimes) 設定オプションを利用できます：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

[レイアウト](/docs/app/api-reference/file-conventions/layout)および[読み込み状態](/docs/app/api-reference/file-conventions/loading)は依然としてキャッシュされ、ナビゲーションで再利用されます。

## `next/font` {#next-font}

`@next/font` パッケージはビルトインの [`next/font`](/docs/app/api-reference/components/font) を利用するように削除されました。[codemod が利用可能](/docs/app/building-your-application/upgrading/codemods#built-in-next-font) で、安全かつ自動的にインポートをリネームできます。

```js title="app/layout.js"
// 変更前
import { Inter } from '@next/font/google'

// 変更後
import { Inter } from 'next/font/google'
```

## Speed Insights {#speed-insights}

Speed Insights の自動計測は Next.js 15 で削除されました。

Speed Insights を利用し続けるには、 [Vercel Speed Insights クイックスタート](https://vercel.com/docs/speed-insights/quickstart) ガイドに従ってください。

## `NextRequest` のジオロケーション {#nextrequest-geolocation}

`NextRequest` の `geo` および `ip` プロパティは、ホスティングプロバイダが提供する値であるため削除されました。[codemod が利用可能](/docs/app/building-your-application/upgrading/codemods#150) で、この移行を自動化できます。

Vercel を使用している場合は、代わりに [`@vercel/functions](https://vercel.com/docs/functions/vercel-functions-package) の `geolocation` および `ipAddress` 関数を使用することもできます。

```ts title="middleware.ts"
import { geolocation } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const { city } = geolocation(request)

  // ...
}
```

```ts title="middleware.ts"
import { ipAddress } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const ip = ipAddress(request)

  // ...
}
```
