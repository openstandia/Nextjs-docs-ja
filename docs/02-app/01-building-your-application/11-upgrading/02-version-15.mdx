## バージョン14から15へのアップグレード {#upgrading-from-14-to-15}

Next.jsバージョン15に更新するには、`upgrade` codemodを使用できます。

```bash title="Terminal"
npx @next/codemod@canary upgrade latest
```

手動で行いたい場合、最新のNext & React RCをインストールすることを確認してください。例えば：

```bash title="Terminal"
npm i next@latest react@rc react-dom@rc eslint-config-next@latest
```

> **Good to know:**
>
> - ピア依存警告が表示された場合は、提案されたバージョンに`react`と`react-dom`を更新する必要がある場合があります。または、警告を無視するために`--force`または`--legacy-peer-deps`オプションを使用することができます。この必要は、Next.js 15とReact 19の両方が安定版になると無くなります。
> - TypeScriptを使用している場合は、一時的にReactの型を上書きする必要があります。詳細については、[React 19 RCアップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide#installing)を参照してください。

## React 19 {#react-19}

- `react`と`react-dom`の最低バージョンは19になりました。
- `useFormState`は`useActionState`に置き換えられました。`useFormState`フックはReact 19でも利用可能ですが、非推奨であり、将来のリリースで削除されます。`useActionState`が推奨され、`pending`状態を直接読み取るのような追加のプロパティが含まれています。[詳しくはこちら](https://react.dev/reference/react/useActionState)
- `useFormStatus`には今、`data`、`method`、`action`のような追加のキーが含まれています。React 19を使用していない場合、`pending`キーのみが利用可能です。[詳しくはこちら](https://react.dev/reference/react-dom/hooks/useFormStatus)
- [React 19アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide)で詳細をご覧ください。

## 非同期リクエストAPI（互換性を壊す変更） {#async-request-apis-breaking-change}

以前は、ランタイム情報に依存する同期的な動的APIが、現在は**非同期**になっています。

- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`draftMode`](/docs/app/api-reference/functions/draft-mode)
- [`layout.js`](/docs/app/api-reference/file-conventions/layout)、[`page.js`](/docs/app/api-reference/file-conventions/page)、[`route.js`](/docs/app/api-reference/file-conventions/route)、[`default.js`](/docs/app/api-reference/file-conventions/default)、[`opengraph-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`twitter-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)、および[`apple-icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)での`params`
- [`page.js`](/docs/app/api-reference/file-conventions/page)での`searchParams`

移行の負担を軽減するため、移行を自動化するための[codemodが用意されています](/docs/app/building-your-application/upgrading/codemods#150)。APIは一時的に同期的にアクセスできます。

### `cookies` {#cookies}

#### 推奨される非同期使用法 {#recommended-async-usage}

```tsx
import { cookies } from 'next/headers'

// 以前
const cookieStore = cookies()
const token = cookieStore.get('token')

// 以降
const cookieStore = await cookies()
const token = cookieStore.get('token')
```

## `runtime`構成（互換性を壊す変更） {#runtime-configuration-breaking-change}

`runtime` [セグメント構成](/docs/app/api-reference/file-conventions/route-segment-config#runtime)は以前、`experimental-edge`と`edge`の値をサポートしていました。両方の構成は同じ意味を持ち、オプションを簡素化するために、`experimental-edge`を使用するとエラーが出るようになります。これを修正するには、`runtime`構成を`edge`に更新してください。これを自動的に行う[codemod](/docs/app/building-your-application/upgrading/codemods#app-dir-runtime-config-experimental-edge)が利用可能です。

</AppOnly>

## `fetch`リクエスト {#fetch-requests}

[`fetch`リクエスト](/docs/app/api-reference/functions/fetch)はデフォルトでキャッシュされなくなりました。

特定の`fetch`リクエストをキャッシュするように選択するには、`cache: 'force-cache'`オプションを渡すことができます。

```js title="app/layout.js"
export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされない
  const b = await fetch('https://...', { cache: 'force-cache' }) // キャッシュされる

  // ...
}
```

あるレイアウトやページ内のすべての`fetch`リクエストをキャッシュに選択するには、`export const fetchCache = 'default-cache'` [セグメント設定オプション](/docs/app/api-reference/file-conventions/route-segment-config)を使用できます。個別の`fetch`リクエストが`cache`オプションを指定している場合、それが使用されます。

```js title="app/layout.js"
// これはroot レイアウトなので、アプリ内のすべてのfetchリクエスト
// 自分自身のキャッシュオプションを設定しないものはキャッシュされます。
export const fetchCache = 'default-cache'

export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされる
  const b = await fetch('https://...', { cache: 'no-store' }) // キャッシュされない

  // ...
}
```

## ルートハンドラー {#route-handlers}

ルートハンドラーの`GET`関数はデフォルトでキャッシュされなくなりました。`GET`メソッドをキャッシュに選択するには、[ルート設定オプション](/docs/app/api-reference/file-conventions/route-segment-config)を使用して、たとえばファイル内で`export const dynamic = 'force-static'`を設定できます。

```js title="app/api/route.js"
export const dynamic = 'force-static'

export async function GET() {}
```

## クライアントサイドのルーターキャッシュ {#client-side-router-cache}

`<Link>`または`useRouter`を通じてページ間を移動する際、[page](/docs/app/api-reference/file-conventions/page)セグメントはクライアントサイドのルーターキャッシュから再利用されなくなりました。しかし、ブラウザのバックとフォワードナビゲーション、および共有レイアウトには依然として再利用されます。

ページセグメントをキャッシュに選択するには、[`staleTimes`](/docs/app/api-reference/next-config-js/staleTimes)設定オプションを使用できます。

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

[Layouts](/docs/app/api-reference/file-conventions/layout)と[loading states](/docs/app/api-reference/file-conventions/loading)はナビゲーション中に依然としてキャッシュされ、再利用されます。

## `next/font` {#next-font}

`@next/font`パッケージは組み込みの[`next/font`](/docs/app/api-reference/components/font)に置き換えられ削除されました。安全かつ自動的にインポートをリネームするための[codemodが利用可能です](/docs/app/building-your-application/upgrading/codemods#built-in-next-font)。

```js title="app/layout.js"
// 以前
import { Inter } from '@next/font/google'

// 以降
import { Inter } from 'next/font/google'
```

## Speed Insights {#speed-insights}

Speed Insightsの自動インストゥルメンテーションはNext.js 15で削除されました。

Speed Insightsを引き続き使用するには、[Vercel Speed Insightsクイックスタート](https://vercel.com/docs/speed-insights/quickstart)ガイドに従ってください。

## `NextRequest` の地理情報 {#nextrequest-geolocation}

`NextRequest`の`geo`および`ip`プロパティは削除されました。これらの値はホスティングプロバイダーによって提供されるためです。自動化された移行を実行するための[codemod](/docs/app/building-your-application/upgrading/codemods#150)が利用可能です。

Vercelを使用している場合、代わりに[`@vercel/functions`](https://vercel.com/docs/functions/vercel-functions-package)からの`geolocation`や`ipAddress`関数を使用できます：

```ts title="middleware.ts"
import { geolocation } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const { city } = geolocation(request)

  // ...
}
```

```ts title="middleware.ts"
import { ipAddress } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const ip = ipAddress(request)

  // ...
}
```
