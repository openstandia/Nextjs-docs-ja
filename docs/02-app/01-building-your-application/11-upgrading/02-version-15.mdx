## バージョン 14 から 15 へのアップグレード {#upgrading-from-14-to-15}

Next.js バージョン 15 にアップデートするには、`upgrade` codemod を使用します。

```bash title="Terminal"
npx @next/codemod@canary upgrade latest
```

手動で行う場合は、最新の Next と React リリース候補をインストールしていることを確認してください。例えば：

```bash title="Terminal"
npm i next@latest react@rc react-dom@rc eslint-config-next@latest
```

> **Good to know:**
>
> - ピア依存関係の警告が表示された場合、`react` と `react-dom` を提案されたバージョンに更新するか、`--force` または `--legacy-peer-deps` フラグを使って警告を無視することができます。Next.js 15 と React 19 が安定版になると、これは必要なくなります。
> - TypeScript を使用している場合は、一時的に React タイプを上書きする必要があります。詳細については、[React 19 RC アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide#installing)をご覧ください。

## React 19 {#react-19}

- `react` と `react-dom` の最小バージョンは 19 になりました。
- `useFormState` は `useActionState` に置き換えられました。`useFormState` フックは React 19 でも利用可能ですが、非推奨となり、将来のリリースで削除されます。`useActionState` が推奨されており、直接 `pending` 状態を読み取るなどの追加プロパティが含まれています。[詳細はこちら](https://react.dev/reference/react/useActionState)。
- `useFormStatus` は `data`、`method`、`action` などの追加キーを含むようになりました。React 19 を使用していない場合は、`pending` キーのみ使用可能です。[詳細はこちら](https://react.dev/reference/react-dom/hooks/useFormStatus)。
- 詳細は [React 19 アップグレードガイド](https://react.dev/blog/2024/04/25/react-19-upgrade-guide)をお読みください。

## 非同期リクエストAPI (互換性に影響する変更) {#async-request-apis-breaking-change}

これまで同期的であったランタイム情報に依存するDynamic APIは、**非同期**になりました：

- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`draftMode`](/docs/app/api-reference/functions/draft-mode)
- `params` in [`layout.js`](/docs/app/api-reference/file-conventions/layout)、[`page.js`](/docs/app/api-reference/file-conventions/page)、[`route.js`](/docs/app/api-reference/file-conventions/route)、[`default.js`](/docs/app/api-reference/file-conventions/default)、[`opengraph-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`twitter-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)、および [`apple-icon`](/docs/app/api-reference/file-conventions/metadata/app-icons)。
- `searchParams` in [`page.js`](/docs/app/api-reference/file-conventions/page)

移行の負担を軽減するために、プロセスを自動化する [codemod が利用可能です](/docs/app/building-your-application/upgrading/codemods#150)、API は一時的に同期的にアクセスすることができます。

### `cookies` {#cookies}

#### 推奨される非同期使用法 {#recommended-async-usage}

```tsx
import { cookies } from 'next/headers'

// Before
const cookieStore = cookies()
const token = cookieStore.get('token')

// After
const cookieStore = await cookies()
const token = cookieStore.get('token')
```

## `runtime` 設定 (互換性に影響する変更) {#runtime-configuration-breaking-change}

`runtime` [segment configuration](/docs/app/api-reference/file-conventions/route-segment-config#runtime) は、以前は `edge` に加え、`experimental-edge` の値をサポートしていました。両方の設定は同じものを指しているため、オプションを簡素化するために、`experimental-edge` を使用するとエラーが発生します。これを解決するには、`runtime` 設定を `edge` に更新します。自動的にこの作業を行う [codemod](/docs/app/building-your-application/upgrading/codemods#app-dir-runtime-config-experimental-edge) が利用可能です。

</AppOnly>

## `fetch` リクエスト {#fetch-requests}

[`fetch` リクエスト](/docs/app/api-reference/functions/fetch) は、デフォルトではもはやキャッシュされなくなりました。

特定の `fetch` リクエストをキャッシュに含めるには、`cache: 'force-cache'` オプションを渡すことができます。

```js title="app/layout.js"
export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされない
  const b = await fetch('https://...', { cache: 'force-cache' }) // キャッシュされる

  // ...
}
```

レイアウトやページ内のすべての `fetch` リクエストをキャッシュに組み入れるには、`export const fetchCache = 'default-cache'` [segment config option](/docs/app/api-reference/file-conventions/route-segment-config) を使用できます。個々の `fetch` リクエストが `cache` オプションを指定している場合は、それが使用されます。

```js title="app/layout.js"
// これは root レイアウトなので
// 独自のキャッシュオプションを設定しない限りアプリ内のすべてのfetchリクエストがキャッシュされます。
export const fetchCache = 'default-cache'

export default async function RootLayout() {
  const a = await fetch('https://...') // キャッシュされる
  const b = await fetch('https://...', { cache: 'no-store' }) // キャッシュされない

  // ...
}
```

## Route Handlers {#route-handlers}

[Route Handlers](/docs/app/api-reference/file-conventions/route) の `GET` 関数は、デフォルトではもはやキャッシュされません。`GET` メソッドをキャッシュに含めるには、`export const dynamic = 'force-static'` などの [route config option](/docs/app/api-reference/file-conventions/route-segment-config) を Route Handler ファイルに使用できます。

```js title="app/api/route.js"
export const dynamic = 'force-static'

export async function GET() {}
```

## クライアントサイドルーターキャッシュ {#client-side-router-cache}

`<Link>` または `useRouter` を介してページ間を移動する際、[page](/docs/app/api-reference/file-conventions/page) セグメントはもはやクライアントサイドルーターキャッシュから再利用されません。ただし、ブラウザの戻る、進むナビゲーションと共有レイアウトの場合には引き続き再利用されます。

ページセグメントをキャッシュに含めるには、[`staleTimes`](/docs/app/api-reference/next-config-js/staleTimes) 設定オプションを使用できます：

```js title="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

[レイアウト](/docs/app/api-reference/file-conventions/layout) と [ロード中の状態](/docs/app/api-reference/file-conventions/loading) は引き続きキャッシュされ、ナビゲーション時に再利用されます。

## `next/font` {#next-font}

`@next/font` パッケージは、組み込みの [`next/font`](/docs/app/api-reference/components/font) に置き換えられ削除されました。インポートの名前を安全かつ自動的に変更する [codemod が利用可能です](/docs/app/building-your-application/upgrading/codemods#built-in-next-font)。

```js title="app/layout.js"
// Before
import { Inter } from '@next/font/google'

// After
import { Inter } from 'next/font/google'
```

## bundlePagesRouterDependencies {#bundlepagesrouterdependencies}

/** @type {import('next').NextConfig} */
const nextConfig = {

    bundlePagesExternals: true,

## serverExternalPackages {#serverexternalpackages}

## Speed Insights {#speed-insights}

Speed Insights の自動インスツルメンテーションは Next.js 15 で削除されました。

Speed Insights を引き続き使用するには、[Vercel Speed Insights クイックスタート](https://vercel.com/docs/speed-insights/quickstart)ガイドに従ってください。

## `NextRequest` ジオロケーション {#nextrequest-geolocation}

`NextRequest` の `geo` と `ip` プロパティは、ホスティングプロバイダーから提供されるため削除されました。この移行を自動化するために [codemod](/docs/app/building-your-application/upgrading/codemods#150) が利用可能です。

Vercel を使用している場合は、代わりに[`@vercel/functions`](https://vercel.com/docs/functions/vercel-functions-package) から `geolocation` と `ipAddress` 関数を使用することができます：

```ts title="middleware.ts"
import { geolocation } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const { city } = geolocation(request)

  // ...
}
```

```ts title="middleware.ts"
import { ipAddress } from '@vercel/functions'
import type { NextRequest } from 'next/server'

export function middleware(request: NextRequest) {
  const ip = ipAddress(request)

  // ...
}
```
