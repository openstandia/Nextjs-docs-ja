`create-next-app` CLIは、デフォルトのテンプレートや公開されているGitHubリポジトリの[例](https://github.com/vercel/next.js/tree/canary/examples)を使用して新しいNext.jsアプリケーションを作成できます。Next.jsを始める最も簡単な方法です。

基本的な使用法:

```bash title="Terminal"
npx create-next-app@latest [project-name] [options]
```

## 参照

利用可能なオプションは以下のとおりです:

| オプション                                | 説明                                                        |
| --------------------------------------- | --------------------------------------------------------- |
| `-h` または `--help`                    | すべての利用可能なオプションを表示する                                 |
| `-v` または `--version`                 | バージョン番号を出力する                                          |
| `--no-*`                                | デフォルトオプションを無効にする 例: `--no-eslint`                      |
| `--ts` または `--typescript`            | TypeScriptプロジェクトとして初期化する (デフォルト)                        |
| `--js` または `--javascript`            | JavaScriptプロジェクトとして初期化する                               |
| `--tailwind`                            | Tailwind CSSの設定で初期化する (デフォルト)                             |
| `--eslint`                              | ESLint設定で初期化する                                           |
| `--app`                                 | アプリルータープロジェクトとして初期化する                                |
| `--src-dir`                             | `src/`ディレクトリ内で初期化する                                   |
| `--turbo`                               | 開発においてTurbopackをデフォルトで有効にする                          |
| `--import-alias <alias-to-configure>`   | 使用するインポートエイリアスを指定する (デフォルト "@/\*")                 |
| `--empty`                               | 空のプロジェクトを初期化する                                         |
| `--use-npm`                             | npmを使用してアプリケーションをブートストラップするようにCLIに明示的に指示する     |
| `--use-pnpm`                            | pnpmを使用してアプリケーションをブートストラップするようにCLIに明示的に指示する   |
| `--use-yarn`                            | Yarnを使用してアプリケーションをブートストラップするようにCLIに明示的に指示する   |
| `--use-bun`                             | Bunを使用してアプリケーションをブートストラップするようにCLIに明示的に指示する    |
| `-e` または `--example [name] [github-url]` | アプリをブートストラップするための例                                 |
| `--example-path <path-to-example>`      | 例へのパスを別途指定する                                            |
| `--reset-preferences`                   | 保存された設定をすべてリセットするようにCLIに明示的に指示する                  |
| `--skip-install`                        | パッケージのインストールをスキップするようにCLIに明示的に指示する              |
| `--yes`                                 | すべてのオプションについて前回の設定またはデフォルトを使用する                  |

## 例

### デフォルトテンプレートを使用

デフォルトテンプレートを使用して新しいアプリを作成するには、ターミナルで次のコマンドを実行します:

```bash title="Terminal"
npx create-next-app@latest
```

すると、次のプロンプトが表示されます:

```txt title="Terminal"
What is your project named?  my-app
Would you like to use TypeScript?  No / Yes
Would you like to use ESLint?  No / Yes
Would you like to use Tailwind CSS?  No / Yes
Would you like your code inside a `src/` directory?  No / Yes
Would you like to use App Router? (recommended)  No / Yes
Would you like to use Turbopack for `next dev`?  No / Yes
Would you like to customize the import alias (`@/*` by default)?  No / Yes
```

プロンプトに答えると、選択した設定で新しいプロジェクトが作成されます。

### 公式のNext.js例を使用

公式のNext.js例を使用して新しいアプリを作成するには、`--example`フラグを使用します。例えば:

```bash title="Terminal"
npx create-next-app@latest --example [example-name] [your-project-name]
```

[Next.jsリポジトリ](https://github.com/vercel/next.js/tree/canary/examples)では、すべての利用可能な例とセットアップ手順を確認できます。

### 任意の公開GitHub例を使用

公開GitHub例を使用して新しいアプリを作成するには、GithubリポジトリのURLを`--example`オプションで指定します。例えば:

```bash title="Terminal"
npx create-next-app@latest --example "https://github.com/.../" [your-project-name]
```
