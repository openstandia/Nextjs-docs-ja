`create-next-app` CLI は、デフォルトのテンプレートや、公開されている GitHub リポジトリの[例](https://github.com/vercel/next.js/tree/canary/examples)から、新しい Next.js アプリケーションを作成することができます。Next.js を始めるのに最も簡単な方法です。

基本的な使い方：

```bash title="Terminal"
npx create-next-app@latest [project-name] [options]
```

## リファレンス {#reference}

使用可能なオプションは、次のとおりです：

| オプション                                | 説明                                                              |
| ---------------------------------------- | ----------------------------------------------------------------- |
| `-h` または `--help`                     | 利用可能なすべてのオプションを表示する                              |
| `-v` または `--version`                  | バージョン番号を出力する                                           |
| `--no-*`                                 | デフォルトオプションを否定する。例：`--no-eslint`                 |
| `--ts` または `--typescript`             | TypeScript プロジェクトとして初期化する（デフォルト）               |
| `--js` または `--javascript`             | JavaScript プロジェクトとして初期化する                            |
| `--tailwind`                             | Tailwind CSS の設定で初期化する（デフォルト）                       |
| `--eslint`                               | ESLint の設定で初期化する                                          |
| `--app`                                  | App Router プロジェクトとして初期化する                             |
| `--src-dir`                              | `src/` ディレクトリ内で初期化する                                   |
| `--turbo`                                | 開発のためにデフォルトで Turbopack を有効にする                     |
| `--import-alias <alias-to-configure>`    | 使用するインポートエイリアスを指定する（デフォルト "@/\*"）         |
| `--empty`                                | 空のプロジェクトを初期化する                                        |
| `--use-npm`                              | CLI に明示的に npm を使用してアプリケーションをブートストラップさせる |
| `--use-pnpm`                             | CLI に明示的に pnpm を使用してアプリケーションをブートストラップさせる |
| `--use-yarn`                             | CLI に明示的に Yarn を使用してアプリケーションをブートストラップさせる |
| `--use-bun`                              | CLI に明示的に Bun を使用してアプリケーションをブートストラップさせる  |
| `-e` または `--example [name] [github-url]` | アプリをブートストラップするための例                                |
| `--example-path <path-to-example>`       | 例へのパスを個別に指定する                                         |
| `--reset-preferences`                    | CLI に明示的に保存された設定をリセットするよう指示する               |
| `--skip-install`                         | CLI に明示的にパッケージのインストールをスキップさせる              |
| `--yes`                                  | すべてのオプションで以前の設定またはデフォルトを使用する              |

## 例 {#examples}

### デフォルトテンプレートを使用する場合 {#with-the-default-template}

デフォルトテンプレートを使用して新しいアプリを作成するには、ターミナルで次のコマンドを実行します：

```bash title="Terminal"
npx create-next-app@latest
```

次に、以下のプロンプトが表示されます：

```txt title="Terminal"
What is your project named?  my-app
Would you like to use TypeScript?  No / Yes
Would you like to use ESLint?  No / Yes
Would you like to use Tailwind CSS?  No / Yes
Would you like your code inside a `src/` directory?  No / Yes
Would you like to use App Router? (recommended)  No / Yes
Would you like to use Turbopack for `next dev`?  No / Yes
Would you like to customize the import alias (`@/*` by default)?  No / Yes
```

プロンプトに回答すると、選択した設定で新しいプロジェクトが作成されます。

### 公式の Next.js の例を使用する場合 {#with-an-official-next-js-example}

公式の Next.js の例を使用して新しいアプリを作成するには、`--example` フラグを使用します。例：

```bash title="Terminal"
npx create-next-app@latest --example [example-name] [your-project-name]
```

[Next.js リポジトリ](https://github.com/vercel/next.js/tree/canary/examples)で利用可能なすべての例とセットアップ手順のリストを確認できます。

### 公開されている任意の GitHub の例を使用する場合 {#with-any-public-github-example}

公開されている GitHub の任意の例を使用して新しいアプリを作成するには、`--example` オプションに GitHub リポジトリの URL を指定します。例：

```bash title="Terminal"
npx create-next-app@latest --example "https://github.com/.../" [your-project-name]
```
