`page` ファイルは、Next.js アプリケーション内でページを定義するために使用されます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  return <h1>My Page</h1>
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default function Page({ params, searchParams }) {
  return <h1>My Page</h1>
}
```
</TabItem>
</Tabs>


## リファレンス

### Props

#### `params` (オプション)

[dyn終引 route parameters](/docs/app/building-your-application/routing/dynamic-routes)を含むオブジェクトに解決される promise です。このオブジェクトは root segment からそのページに至るまで含まれます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const slug = (await params).slug
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/[slug]/page.js" switcher
export default async function Page({ params }) {
  const slug = (await params).slug
}
```
</TabItem>
</Tabs>


| ルートの例                           | URL         | `params`                                |
| ------------------------------------ | ----------- | --------------------------------------- |
| `app/shop/[slug]/page.js`            | `/shop/1`   | `Promise<{ slug: '1' }>`                |
| `app/shop/[category]/[item]/page.js` | `/shop/1/2` | `Promise<{ category: '1', item: '2' }>` |
| `app/shop/[...slug]/page.js`         | `/shop/1/2` | `Promise<{ slug: ['1', '2'] }>`         |

- `params` prop は promise です；値にアクセスするには `async/await` か React の [`use`](https://react.dev/reference/react/use) 関数を使用する必要があります
  - バージョン 14 およびそれ以前では、`params` は同期的な prop でした。後方互換性を助けるために、Next.js 15 でも同期的にアクセスできますが、この動作は将来的に非推奨となります。

#### `searchParams` (オプション)

現在の URL の[検索パラメーター](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)を含むオブジェクトに解決される promise です。例えば：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const filters = (await searchParams).filters
}
```
</TabItem>
</Tabs>


| 例の URL     | `searchParams`                |
| --------------- | ----------------------------- |
| `/shop?a=1`     | `Promise<{ a: '1' }>`         |
| `/shop?a=1&b=2` | `Promise<{ a: '1', b: '2' }>` |
| `/shop?a=1&a=2` | `Promise<{ a: ['1', '2'] }>`  |

- `searchParams` prop は promise です；値にアクセスするには `async/await` か React の [`use`](https://react.dev/reference/react/use) 関数を使用する必要があります
  - バージョン 14 およびそれ以前では、`searchParams` は同期的な prop でした。後方互換性を助けるために、Next.js 15 でも同期的にアクセスできますが、この動作は将来的に非推奨となります。
- `searchParams` はその値を事前に知ることができない **[Dynamic API](/docs/app/building-your-application/rendering/server-components#server-rendering-strategies#dynamic-apis)** です。これを使用すると、リクエスト時に **[dynamic rendering](/docs/app/building-your-application/rendering/server-components#dynamic-rendering)** でページがレンダリングされます
- `searchParams` は `URLSearchParams` インスタンスではなく、単なる JavaScript オブジェクトです。

## 例

### `params` に基づいてコンテンツを表示

[dynamic route segments](/docs/app/building-your-application/routing/dynamic-routes) を使用して、`params` prop に基づいて特定のコンテンツを表示または取得できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/blog/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/blog/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```
</TabItem>
</Tabs>


### `searchParams` を使用したフィルタリングの処理

URL のクエリ文字列に基づいて、フィルタリング、ページネーション、またはソートを処理するために `searchParams` prop を使用できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```
</TabItem>
</Tabs>


### クライアントコンポーネントでの `searchParams` および `params` の読み込み

`searchParams` と `params` をクライアントコンポーネントで使用するには（`async` にできない場合）、React の [`use`](https://react.dev/reference/react/use) 関数を使用して promise を読み込みます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
'use client'

import { use } from 'react'

export function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```
</TabItem>
<TabItem value="js" label="JavaScript">

```js title="app/page.js" switcher
'use client'

import { use } from 'react'

export function Page({ params, searchParams }) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```
</TabItem>
</Tabs>
