`mdx-components.js|tsx` ファイルは、[`@next/mdx` with App Router](/docs/app/building-your-application/configuring/mdx)を使用するために**必須**であり、これなしでは動作しません。また、[スタイルをカスタマイズ](/docs/app/building-your-application/configuring/mdx#using-custom-styles-and-components)するために使用することもできます。

`mdx-components.tsx` （または`.js`）ファイルは、プロジェクトのルートに配置して、MDXコンポーネントを定義します。例えば、`pages`や`app`と同じレベル、または該当する場合は`src`内に配置します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'

// MDXコンポーネントを使用する関数をエクスポートします
export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    ...components, // 既存のコンポーネントを展開して返します
  }
}
```
</TabItem>


## エクスポート

### `useMDXComponents` 関数

ファイルは、シングル関数として、デフォルトエクスポートするか`useMDXComponents`という名前でエクスポートする必要があります。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'

// MDXコンポーネントを使用する関数をエクスポートします
export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    ...components, // 既存のコンポーネントを展開して返します
  }
}
```
</TabItem>


## パラメータ

### `components`

MDXコンポーネントを定義する際、エクスポート関数は1つのパラメータ`components`を受け取ります。このパラメータは`MDXComponents`のインスタンスです。

- キーは、上書きするHTML要素の名前です
- 値は、その代わりにレンダリングするコンポーネントです

> **Good to know**: オーバーライドされないすべてのコンポーネント（例：`...components`）を渡すことを忘れないでください。
