`mdx-components.js|tsx`ファイルは、[`@next/mdx`をApp Routerと共に使用するため](/docs/app/building-your-application/configuring/mdx)に**必須**で、これがないと機能しません。さらに、[カスタムスタイルを設定する](/docs/app/building-your-application/configuring/mdx#using-custom-styles-and-components)ためにも利用できます。

プロジェクトのルートに`mdx-components.tsx`（または`.js`）ファイルを使用して、MDXコンポーネントを定義します。例えば、`pages`や`app`と同じレベル、または適用可能であれば`src`内に置きます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'

// すべてのMDXコンポーネントを適用します
export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    ...components,
  }
}
```
</TabItem>
<TabItem value="js" label="JavaScript">

```js title="mdx-components.js" switcher
// すべてのMDXコンポーネントを適用します
export function useMDXComponents(components) {
  return {
    ...components,
  }
}
```
</TabItem>
</Tabs>


## エクスポート

### `useMDXComponents`関数

ファイルは単一の関数を、デフォルトエクスポートまたは`useMDXComponents`という名前でエクスポートする必要があります。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'

// すべてのMDXコンポーネントを適用します
export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    ...components,
  }
}
```
</TabItem>
<TabItem value="js" label="JavaScript">

```js title="mdx-components.js" switcher
// すべてのMDXコンポーネントを適用します
export function useMDXComponents(components) {
  return {

  }
```

</TabItem>
</Tabs>


## パラメーター

### `components`

MDXコンポーネントを定義するとき、エクスポート関数は単一のパラメーター`components`を受け取ります。このパラメーターは`MDXComponents`のインスタンスです。

- キーはオーバーライドするHTML要素の名前です
- 値は代わりにレンダリングするコンポーネントです

> **Good to know**: オーバーライドのない他のすべてのコンポーネント（つまり、`...components`）を必ず渡してください。
