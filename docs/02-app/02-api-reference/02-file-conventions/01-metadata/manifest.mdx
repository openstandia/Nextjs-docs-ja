ブラウザに対してWebアプリケーションの情報を提供するために、`app`ディレクトリの**root**に[Web Manifest Specification](https://developer.mozilla.org/docs/Web/Manifest)に一致する`manifest.(json|webmanifest)`ファイルを追加または生成します。

## Static Manifest ファイル

```json filename="app/manifest.json | app/manifest.webmanifest"
{
  "name": "My Next.js Application",
  "short_name": "Next.js App",
  "description": "An application built with Next.js", // Next.jsで構築されたアプリケーション
  "start_url": "/"
  // ...
}
```

## Manifest ファイルを生成する

[`Manifest`オブジェクト](#manifest-object)を返す`manifest.js`または`manifest.ts`ファイルを追加します。

> Good to know: `manifest.js`は特別なルートハンドラで、[Dynamic API](/docs/app/building-your-application/caching#dynamic-apis)や[dynamic config](/docs/app/building-your-application/caching#segment-config-options)オプションを使用しない限りデフォルトでキャッシュされます。

```ts filename="app/manifest.ts" switcher
import type { MetadataRoute } from 'next'

export default function manifest(): MetadataRoute.Manifest {
  return {
    name: 'Next.js App',
    short_name: 'Next.js App',
    description: 'Next.js App',
    start_url: '/',
    display: 'standalone',
    background_color: '#fff',
    theme_color: '#fff',
    icons: [
      {
        src: '/favicon.ico',
        sizes: 'any',
        type: 'image/x-icon',
      },
    ],
  }
}
```

```js filename="app/manifest.js" switcher
export default function manifest() {
  return {
    name: 'Next.js App',
    short_name: 'Next.js App',
    description: 'Next.js App',
    start_url: '/',
    display: 'standalone',
    background_color: '#fff',
    theme_color: '#fff',
    icons: [
      {
        src: '/favicon.ico',
        sizes: 'any',
        type: 'image/x-icon',
      },
    ],
  }
}
```

### Manifest オブジェクト

manifest オブジェクトには新しいWeb標準に応じて更新される可能性のある豊富なオプションリストが含まれています。すべての現在のオプションについては、[TypeScript](https://nextjs.org/docs/app/building-your-application/configuring/typescript#typescript-plugin)を使用している場合はコードエディタで`MetadataRoute.Manifest`型を参照するか、[MDN](https://developer.mozilla.org/docs/Web/Manifest)ドキュメントを参照してください。
