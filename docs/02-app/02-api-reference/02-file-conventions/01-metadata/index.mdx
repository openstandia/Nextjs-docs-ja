このドキュメントのセクションでは、**メタデータファイルの規約**について説明します。ファイルベースのメタデータは、ルートセグメントに特別なメタデータファイルを追加することで定義できます。

各ファイル規約は、静的ファイル（例: `opengraph-image.jpg`）を使用して定義することも、コードを使ってファイルを生成する動的バリアント（例: `opengraph-image.js`）を使用することもできます。

ファイルが定義されると、Next.js はそのファイルを自動的に提供し（本番環境ではキャッシュのためにハッシュが追加されます）、アセットのURL、ファイルタイプ、画像サイズなどの適切なメタデータで関連するhead要素を更新します。

> **Good to know**:
>
> - 特別なRoute Handlersである[`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap), [`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image), [`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons)、およびその他の[メタデータファイル](/docs/app/api-reference/file-conventions/metadata)はデフォルトでキャッシュされます
> - [`middleware.ts`](/docs/app/api-reference/file-conventions/middleware)とともに使用する場合、メタデータファイルを除外するように[matcherを設定してください](/docs/app/building-your-application/routing/middleware#matcher)
