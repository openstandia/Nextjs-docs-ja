このドキュメントのセクションでは、**メタデータファイルの規約**を取り上げます。ファイルベースのメタデータは、ルートセグメントに特別なメタデータファイルを追加することで定義できます。

各ファイル規約は、静的ファイル（例：`opengraph-image.jpg`）を使用して定義することも、コードを使用してファイルを生成する動的なバリアント（例：`opengraph-image.js`）を使用して定義することもできます。

一度ファイルが定義されると、Next.js はそのファイルを自動的に提供し（キャッシュのために本番環境ではハッシュ付き）、アセットのURL、ファイルタイプ、画像サイズなどの正しいメタデータで関連するhead要素を更新します。

> **Good to know**:
>
> - 特殊なルートハンドラー、例えば [`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap)、[`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons)、その他の [メタデータファイル](/docs/app/api-reference/file-conventions/metadata) はデフォルトでキャッシュされます
> - [`middleware.ts`](/docs/app/api-reference/file-conventions/middleware) と一緒に使用する場合は、メタデータファイルを除外するように[マッチャーを設定](/docs/app/building-your-application/routing/middleware#matcher)してください
