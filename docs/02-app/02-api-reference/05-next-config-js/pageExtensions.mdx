<AppOnly>

デフォルトでは、Next.jsは次の拡張子を持つファイルを受け入れます：`.tsx`、`.ts`、`.jsx`、`.js`。 これを変更して、マークダウン（`.md`、`.mdx`）のような他の拡張子を許可することができます。

```js filename="next.config.js"
const withMDX = require('@next/mdx')()

/** @type {import('next').NextConfig} */
const nextConfig = {
  pageExtensions: ['js', 'jsx', 'ts', 'tsx', 'md', 'mdx'],
}

module.exports = withMDX(nextConfig)
```

</AppOnly>

<PagesOnly>

Next.jsで使用されるデフォルトのページ拡張子（`.tsx`、`.ts`、`.jsx`、`.js`）を拡張できます。`next.config.js`内で、`pageExtensions`構成を追加します：

```js filename="next.config.js"
module.exports = {
  pageExtensions: ['mdx', 'md', 'jsx', 'js', 'tsx', 'ts'],
}
```

これらの値を変更すると、次の.jsのすべてのページに影響を与えます；以下を含みます：

- [`middleware.js`](/docs/pages/building-your-application/routing/middleware)
- [`instrumentation.js`](/docs/pages/building-your-application/optimizing/instrumentation)
- `pages/_document.js`
- `pages/_app.js`
- `pages/api/`

例えば、`.ts`ページ拡張子を`.page.ts`に再構成する場合、`middleware.page.ts`、`instrumentation.page.ts`、`_app.page.ts`などのページをリネームする必要があります。

## `pages`ディレクトリ内への非ページファイルの含み

テストファイルや、コンポーネントで使用される他のファイルを`pages`ディレクトリに配置できます。`next.config.js`内で、`pageExtensions`構成を追加します：

```js filename="next.config.js"
module.exports = {
  pageExtensions: ['page.tsx', 'page.ts', 'page.jsx', 'page.js'],
}
```

次に、ページの名前を`.page`を含むファイル拡張子にリネームします（例：`MyPage.tsx`を`MyPage.page.tsx`にリネームします）。上記で紹介したファイルを含む、すべてのNext.jsページをリネームしてください。

</PagesOnly>
