> **Good to know**: Next.js 13.5.1 以降、Strict モードは `app` router でデフォルトで `true` になっているため、上記の設定は `pages` にのみ必要です。Strict モードを無効にするには、`reactStrictMode: false` を設定することで可能です。

> **Suggested**: Next.js アプリケーションで Strict モードを有効にすることを強くお勧めします。これにより、React の将来に備えたアプリケーションの準備が整います。

React の[Strict モード](https://react.dev/reference/react/StrictMode)は、アプリケーション内の潜在的な問題を強調するための開発モード専用の機能です。これにより、安全でないライフサイクル、レガシー API の使用、およびその他の多くの機能を識別するのに役立ちます。

Next.js のランタイムは Strict モードに準拠しています。Strict モードにオプトインするには、以下のオプションを `next.config.js` で設定します：

```js title="next.config.js"
module.exports = {
  reactStrictMode: true,
}
```

あなたやあなたのチームが、アプリケーション全体で Strict モードを使用する準備ができていない場合でも大丈夫です！ `<React.StrictMode>` を使用して、ページごとに段階的に移行することができます。
