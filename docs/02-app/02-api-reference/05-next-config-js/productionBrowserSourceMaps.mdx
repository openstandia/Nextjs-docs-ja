開発中は、ソースマップはデフォルトで有効になっています。 本番ビルドでは、クライアントにソースを漏らさないように、それらは無効になっています。ただし、設定フラグを使用して、明示的にオプトインする場合を除きます。

Next.jsは、本番ビルド中にブラウザーソースマップ生成を有効にするための設定フラグを提供しています：

```js title="next.config.js"
module.exports = {
  productionBrowserSourceMaps: true,
}
```

`productionBrowserSourceMaps` オプションが有効になっている場合、ソースマップはJavaScriptファイルと同じディレクトリに出力されます。Next.jsはこれらのファイルが要求されたときに自動的にそれらを提供します。

- ソースマップを追加すると、`next build` 時間が増加する
- `next build` 中のメモリ使用量が増加する
