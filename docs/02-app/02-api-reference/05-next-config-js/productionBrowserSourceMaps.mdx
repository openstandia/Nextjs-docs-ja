開発中はソースマップがデフォルトで有効になっています。本番ビルド中は、お客様のソースをクライアントに漏らさないように、特定の設定フラグでオプトインしない限り無効になっています。

Next.jsは、本番ビルド中にブラウザーソースマップ生成を有効にするための設定フラグを提供しています：

```js filename="next.config.js"
module.exports = {
  productionBrowserSourceMaps: true,
}
```

`productionBrowserSourceMaps`オプションが有効な場合、ソースマップはJavaScriptファイルと同じディレクトリに出力されます。Next.jsはリクエストされたときにこれらのファイルを自動的に提供します；

- ソースマップを追加すると`next build`時間が増加します
- `next build`中のメモリ使用量が増加します
