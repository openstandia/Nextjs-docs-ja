`dynamicIO`フラグはNext.jsにおける実験的な機能で、App Router内のデータ取得操作を、明示的にキャッシュされない限りプリレンダリングから除外します。これは、server componentでの動的データ取得のパフォーマンスを最適化するのに役立ちます。

アプリケーションがプリレンダリングされたキャッシュから提供するのではなく、実行時に新しいデータ取得を必要とする場合に便利です。

また、[`use cache`](/docs/app/api-reference/directives/use-cache)と併用することが推奨されます。これにより、アプリケーションの特定の部分をページ、関数、またはコンポーネントレベルで`use cache`を使ってキャッシュするように定義されていない限り、データ取得はデフォルトで実行時に行われます。

## 使用方法 {#usage}

`dynamicIO`フラグを有効にするには、`next.config.ts`ファイルの`experimental`セクションで`true`に設定します：

```ts title="next.config.ts"
import type { NextConfig } from 'next'

// experimentalセクションでdynamicIOフラグをtrueに設定します
const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
  },
}

export default nextConfig
```

`dynamicIO`が有効になると、以下のキャッシュ関数と設定を使用できます：

- [`use cache`ディレクティブ](/docs/app/api-reference/directives/use-cache)
- `use cache`を使用した[`cacheLife`関数](/docs/app/api-reference/next-config-js/cacheLife)
- [`cacheTag`関数](/docs/app/api-reference/functions/cacheTag)

## 注意事項 {#notes}

- `dynamicIO`は、実行時に新しいデータ取得を保証することでパフォーマンスを最適化できますが、プリレンダリングされたコンテンツを提供するのと比べて追加の遅延を引き起こす可能性もあります；
