`dynamicIO`フラグはNext.jsの実験的な機能で、App Routerにおけるデータ取得操作が、明示的にキャッシュされていない限り、プリレンダリングから除外されるようにします。これは、サーバーコンポーネントにおける動的なデータ取得のパフォーマンスを最適化するのに役立ちます。

アプリケーションが実行時に最新のデータ取得を必要とし、プリレンダリングされたキャッシュから提供するのではなくする場合に役立ちます。

この機能は、[`use cache`](/docs/app/api-reference/directives/use-cache)と共に使用されることを想定しています。これにより、データ取得はデフォルトで実行時に行われ、特定の部分に`use cache`をページ、関数、またはコンポーネントレベルで定義しない限り、実行時にデータ取得が行われるようになります。

## 使用方法 {#usage}

`dynamicIO`フラグを有効にするには、`next.config.ts`ファイルの`experimental`セクションで`true`に設定します：

```ts title="next.config.ts"
import type { NextConfig } from 'next'

// experimentalセクションでdynamicIOフラグを有効にする
const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
  },
}

export default nextConfig
```

`dynamicIO`が有効な場合、以下のキャッシュ機能と設定を使用できます：

- [`use cache`ディレクティブ](/docs/app/api-reference/directives/use-cache)
- `use cache`と共に使用する[`cacheLife`関数](/docs/app/api-reference/next-config-js/cacheLife)
- [`cacheTag`関数](/docs/app/api-reference/functions/cacheTag)

## 注意点 {#notes}

- `dynamicIO`は、実行時に最新のデータ取得を保証することでパフォーマンスを最適化できますが、プリレンダリングされたコンテンツを提供する場合と比べて追加の待ち時間をもたらす可能性があります；
