`dynamicIO`フラグは、Next.jsの実験的な機能で、App Router内でのデータフェッチ操作を、明示的にキャッシュされていない限りプリレンダリングから除外するものです。これは、server componentでの動的データフェッチのパフォーマンスを最適化するのに役立ちます。

このフラグは、キャッシュされたプリレンダリングの提供ではなく、実行時に新しいデータフェッチを必要とするアプリケーションにとって便利です。

特定のアプリケーションの一部をページ、関数、またはcomponentレベルで`use cache`を使用してキャッシュするよう定義しない限り、デフォルトで実行時にデータフェッチが行われるようにする[`use cache`](/docs/app/api-reference/directives/use-cache)と併せて使用することが想定されています。

## 使用方法 {#usage}

`dynamicIO`フラグを有効にするには、`next.config.ts`ファイルの`experimental`セクションで`true`に設定します：

```ts title="next.config.ts"
import type { NextConfig } from 'next'

// dynamicIOフラグをtrueに設定
const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
  },
}

export default nextConfig
```

`dynamicIO`が有効になっているときは、次のキャッシュ関数と設定を使用できます：

- [`use cache`ディレクティブ](/docs/app/api-reference/directives/use-cache)
- `use cache`での[`cacheLife`関数](/docs/app/api-reference/next-config-js/cacheLife)
- [`cacheTag`関数](/docs/app/api-reference/functions/cacheTag)

## 注意点 {#notes}

- `dynamicIO`は実行時に新しいデータフェッチを確保することでパフォーマンスを最適化できますが、プリレンダリングされたコンテンツを提供する場合に比べて追加の遅延を引き起こす可能性もあります；
