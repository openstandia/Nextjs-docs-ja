`dynamicIO` フラグは Next.js の実験的な機能で、App Router 内のデータフェッチ操作が明示的にキャッシュされない限り、事前レンダリングから除外されるようにします。これは、サーバーコンポーネントで動的なデータフェッチのパフォーマンスを最適化するのに役立ちます。

ランタイム中に最新のデータフェッチが必要なアプリケーションでは、事前レンダリングされたキャッシュから提供するのではなく、この機能が役立ちます。

この機能は [`use cache`](/docs/app/api-reference/directives/use-cache) と一緒に使用されることを前提としており、特定のページ、関数、またはコンポーネントレベルで `use cache` を使用してキャッシュする部分を定義しない限り、デフォルトではランタイムでデータフェッチが行われます。

## 使用法 {#usage}

`dynamicIO` フラグを有効にするには、`next.config.ts` ファイルの `experimental` セクションで `true` に設定します：

```ts title="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
  },
}

export default nextConfig
```

`dynamicIO` が有効になると、以下のキャッシュ関数と設定を使用できます：

- [`use cache` ディレクティブ](/docs/app/api-reference/directives/use-cache)
- `use cache` と共に使用する [`cacheLife` 関数](/docs/app/api-reference/next-config-js/cacheLife)
- [`cacheTag` 関数](/docs/app/api-reference/functions/cacheTag)

## 注意 {#notes}

- `dynamicIO` は、ランタイム中に最新のデータフェッチを確保することでパフォーマンスを最適化できますが、事前レンダリングされたコンテンツを提供する場合と比較して追加の遅延を引き起こす可能性もあります；
