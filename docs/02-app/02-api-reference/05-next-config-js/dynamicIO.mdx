`dynamicIO` フラグはNext.jsの実験的な機能で、App Routerにおけるデータフェッチ操作が、明示的にキャッシュされない限りプリレンダリングから除外されます。これはサーバーコンポーネントでの動的なデータフェッチのパフォーマンスを最適化するために便利です。

アプリケーションがプリレンダリングされたキャッシュから提供するのではなく、実行時に新しいデータフェッチを必要とする場合に便利です。

[`use cache`](/docs/app/api-reference/directives/use-cache)と併用することが期待されており、特定のアプリケーションの部分をページや関数、コンポーネントレベルで`use cache`を使ってキャッシュするよう定義しない限り、デフォルトで実行時にデータフェッチが行われます。

## 使用法 {#usage}

`dynamicIO` フラグを有効にするには、`next.config.ts` ファイルの `experimental` セクションで `true` に設定します：

```ts title="next.config.ts"
import type { NextConfig } from 'next'

// dynamicIOフラグを有効にする
const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
  },
}

export default nextConfig
```

`dynamicIO` が有効になった場合、以下のキャッシュ関数と設定を使用できます：

- [`use cache` ディレクティブ](/docs/app/api-reference/directives/use-cache)
- `use cache` との組み合わせで使用する [`cacheLife` 関数](/docs/app/api-reference/next-config-js/cacheLife)
- [`cacheTag` 関数](/docs/app/api-reference/functions/cacheTag)

## 注意事項 {#notes}

- `dynamicIO` は実行時に新鮮なデータフェッチを保証することでパフォーマンスを最適化できますが、プリレンダリングされたコンテンツを提供する場合と比べて、追加の遅延を引き起こす可能性もあります；
