Next.jsは、プロジェクトにTypeScriptのエラーが存在する場合、**productionビルド**（`next build`）に失敗します。

エラーがある場合でも、Next.jsが危険を冒してプロダクションコードを生成してほしい場合、組み込みのタイプチェックステップを無効にすることができます。

無効にする場合は、ビルドまたはデプロイのプロセスの一環としてタイプチェックを実行していることを確認してください。そうしないと非常に危険です。

`next.config.js`を開いて、`typescript`設定で`ignoreBuildErrors`オプションを有効にします：

```js title="next.config.js"
module.exports = {
  typescript: {
    // !! 警告 !!
    // プロジェクトに型エラーがある場合でも、プロダクションビルドが正常に完了するように危険を冒します。
    // !! 警告 !!
    ignoreBuildErrors: true,
  },
}
```
