`cacheLife`オプションを使用することで、コンポーネント、関数、ファイル単位で[`use cache`ディレクティブ](/docs/app/api-reference/directives/use-cache)を使ってキャッシュの管理をより細かく制御するためのカスタム設定を定義します。

## 使用方法 {#usage}

`cacheLife`を使用するには、[`dynamicIO`フラグ](/docs/app/api-reference/next-config-js/dynamicIO)を有効にし、`next.config.js`ファイルで下記のように設定を定義します：

```js title="next.config.js"
module.exports = {
  experimental: {
    dynamicIO: true,
    cacheLife: {
      blog: {
        stale: 3600, // 1時間
        revalidate: 900, // 15分
        expire: 86400, // 1日
      },
    },
  },
}
```

これで、コンポーネント、関数、ファイルでこのカスタム`blog`設定を次のように使用できます：

```tsx title="app/actions.ts" highlight={4,5}
import { unstable_cacheLife as cacheLife } from 'next/cache'

export async function getCachedData() {
  'use cache'
  cacheLife('blog')
  const data = await fetch('/api/data')
  return data
}
```

## 設定の構造 {#configuration-structure}

設定オブジェクトには、以下の形式でキーと値があります：

| **プロパティ** | **値**   | **説明**                                                                                         | **要件**                                    |
| -------------- | -------- | ------------------------------------------------------------------------------------------------ | ------------------------------------------- |
| `stale`        | `number` | クライアントがサーバーを確認せずに値をキャッシュする期間                                         | 任意                                        |
| `revalidate`   | `number` | キャッシュがサーバーで更新される頻度；再検証中に古い値を提供することがあります                   | 任意                                        |
| `expire`       | `number` | 値が動的フェッチに切り替わるまで古いままでいられる最大期間；`revalidate`より長くなければならない | 任意 - `revalidate`より長くなければならない |
