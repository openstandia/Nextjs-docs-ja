`cacheLife` オプションは、コンポーネントや関数、[`use cache` 指示文](/docs/app/api-reference/directives/use-cache)のスコープ内で [`cacheLife`](/docs/app/api-reference/functions/cacheLife) 関数を使用する際に、**カスタムキャッシュプロファイル**を定義することを可能にします。

## 使用方法 {#usage}

プロファイルを定義するには、[`dynamicIO` フラグ](/docs/app/api-reference/next-config-js/dynamicIO) を有効にして、`next.config.js` ファイルの `cacheLife` オブジェクト内にキャッシュプロファイルを追加してください。例えば、`blog` プロファイルの場合：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
    cacheLife: {
      blog: {
        stale: 3600, // 1時間
        revalidate: 900, // 15分
        expire: 86400, // 1日
      },
    },
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
module.exports = {
  experimental: {
    dynamicIO: true,
    cacheLife: {
      blog: {
        stale: 3600, // 1時間
        revalidate: 900, // 15分
        expire: 86400, // 1日
      },
    },
  },
}
```

</TabItem>
</Tabs>

これで、このカスタム `blog` 設定をコンポーネントや関数内で次のように使用することができます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/actions.ts" highlight={4,5} switcher
import { unstable_cacheLife as cacheLife } from 'next/cache'

export async function getCachedData() {
  'use cache'
  cacheLife('blog')
  const data = await fetch('/api/data')
  return data
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/actions.js" highlight={4,5} switcher
import { unstable_cacheLife as cacheLife } from 'next/cache'

export async function getCachedData() {
  'use cache'
  cacheLife('blog')
  const data = await fetch('/api/data')
  return data
}
```

</TabItem>
</Tabs>

## リファレンス {#reference}

設定オブジェクトには、以下の形式でキーの値があります：

| **プロパティ** | **値**   | **説明**                                                                         | **要件**                                     |
| -------------- | -------- | -------------------------------------------------------------------------------- | -------------------------------------------- |
| `stale`        | `number` | クライアントがサーバを確認せずに値をキャッシュすべき期間                         | 任意                                         |
| `revalidate`   | `number` | サーバでキャッシュが更新されるべき頻度。再検証中に古い値が提供される可能性がある | 任意                                         |
| `expire`       | `number` | 動的に切り替わる前に値が古いままでいられる最大期間                               | 任意 - `revalidate` より長くなければならない |
