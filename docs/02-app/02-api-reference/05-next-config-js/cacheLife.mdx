`cacheLife`オプションを使用すると、コンポーネントまたは関数内、および[`use cache`ディレクティブ](/docs/app/api-reference/directives/use-cache)の範囲内で[`cacheLife`](/docs/app/api-reference/functions/cacheLife)関数を使用する際に**カスタムキャッシュプロファイル**を定義できます。

## 利用方法 {#usage}

プロファイルを定義するには、[`dynamicIO`フラグ](/docs/app/api-reference/next-config-js/dynamicIO)を有効にし、`next.config.js`ファイルの`cacheLife`オブジェクトにキャッシュプロファイルを追加します。例えば、`blog`プロファイルの場合：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
    cacheLife: {
      blog: {
        stale: 3600, // 1時間
        revalidate: 900, // 15分
        expire: 86400, // 1日
      },
    },
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
module.exports = {
  experimental: {
    dynamicIO: true,
    cacheLife: {
      blog: {
        stale: 3600, // 1時間
        revalidate: 900, // 15分
        expire: 86400, // 1日
      },
    },
  },
}
```

</TabItem>
</Tabs>

このカスタム`blog`設定を次のようにコンポーネントや関数で使用できます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/actions.ts" highlight={4,5} switcher
import { unstable_cacheLife as cacheLife } from 'next/cache'

export async function getCachedData() {
  'use cache'
  cacheLife('blog')
  const data = await fetch('/api/data')
  return data
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/actions.js" highlight={4,5} switcher
import { unstable_cacheLife as cacheLife } from 'next/cache'

export async function getCachedData() {
  'use cache'
  cacheLife('blog')
  const data = await fetch('/api/data')
  return data
}
```

</TabItem>
</Tabs>

## リファレンス {#reference}

設定オブジェクトには、次の形式のキーと値があります：

| **プロパティ** | **値**   | **説明**                                                                                     | **要件**                                      |
| -------------- | -------- | -------------------------------------------------------------------------------------------- | --------------------------------------------- |
| `stale`        | `number` | クライアントがサーバーを確認せずに値をキャッシュする期間。                                   | 任意                                          |
| `revalidate`   | `number` | サーバーでキャッシュをリフレッシュする頻度；リフレッシュ中は古い値を提供することがあります。 | 任意                                          |
| `expire`       | `number` | 動的に切り替える前に値が古くなったままの最大期間。                                           | 任意 - `revalidate`より長くなければなりません |
