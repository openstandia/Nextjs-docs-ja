---
title: 'reactCompiler'
description: 'React Compiler を有効にして、コンポーネントのレンダリングを自動的に最適化します。'
version: 'experimental'
---

Next.js 15 RC では、[React Compiler](https://react.dev/learn/react-compiler) のサポートが導入されました。コンパイラはコンポーネントのレンダリングを自動的に最適化することでパフォーマンスを向上させます。これにより、開発者が `useMemo` や `useCallback` などの API を使用して手動で memoization を行う必要が減少します。

これを使用するには、Next.js 15 にアップグレードし、`babel-plugin-react-compiler` をインストールします：

```bash filename="Terminal"
npm install babel-plugin-react-compiler
```

その後、`next.config.js` に `experimental.reactCompiler` オプションを追加します：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    reactCompiler: true,
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    reactCompiler: true,
  },
}

module.exports = nextConfig
```

オプションで、コンパイラを「オプトイン」モードで実行するように設定できます：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    reactCompiler: {
      compilationMode: 'annotation',
    },
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    reactCompiler: {
      compilationMode: 'annotation',
    },
  },
}

module.exports = nextConfig
```

> **Note:** 現在、React Compiler は Next.js で Babel プラグインを通じてのみ使用可能です。これにより、Next.js のデフォルトの [Rust ベースのコンパイラ](/docs/architecture/nextjs-compiler) をオプトアウトすることになり、ビルド時間が遅くなる可能性があります。React Compiler をデフォルトのコンパイラとしてサポートするよう取り組んでいます。

[React Compiler](https://react.dev/learn/react-compiler) と [利用可能な Next.js の設定オプション](https://react.dev/learn/react-compiler#usage-with-nextjs) についてさらに詳しく学んでください。
