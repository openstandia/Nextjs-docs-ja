`staticGeneration*`オプションを使用すると、Next.jsアプリケーション内で静的生成プロセスを設定できます。これにより、より高度なユースケースでの制御が可能になります。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

// experimentalで静的生成の設定を行います
const nextConfig: NextConfig = {
  experimental: {
    staticGenerationRetryCount: 1,
    staticGenerationMaxConcurrency: 8,
    staticGenerationMinPagesPerWorker: 25,
  },
}

export default nextConfig
```
</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
// experimentalで静的生成の設定を行います
const nextConfig = {
  experimental: {
    staticGenerationRetryCount: 1
    staticGenerationMaxConcurrency: 8
    staticGenerationMinPagesPerWorker: 25
  },
}

export default nextConfig;
```
</TabItem>
</Tabs>

## 設定オプション {#config-options}

利用可能なオプションは次のとおりです:

- `staticGenerationRetryCount`: ページ生成が失敗した場合に、ビルドが失敗する前に再試行する回数
- `staticGenerationMaxConcurrency`: ワーカーごとに処理されるページの最大数
- `staticGenerationMinPagesPerWorker`: 新しいワーカーを開始する前に処理されるページの最小数
