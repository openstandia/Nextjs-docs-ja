Next.jsを開発モードで実行しているときに、ログ出力されるレベルや完全なURLがコンソールに表示されるかどうかを設定できます。

現在、`logging`は`fetch` APIを使用したデータ取得にのみ適用されます。Next.js内の他のログにはまだ適用されていません。

```js title="next.config.js"
module.exports = {
  logging: {
    fetches: {
      fullUrl: true,
    },
  },
}
```

[Server Components HMR cache](/docs/app/api-reference/next-config-js/serverComponentsHmrCache)から復元される`fetch`リクエストは、デフォルトではログ出力されません。ただし、`logging.fetches.hmrRefreshes`を`true`に設定すると、これを有効にできます。

```js title="next.config.js"
module.exports = {
  logging: {
    fetches: {
      hmrRefreshes: true,
    },
  },
}
```

さらに、`logging`を`false`に設定することで、開発時のログ出力を無効にすることもできます。

```js title="next.config.js"
module.exports = {
  logging: false,
}
```
