Next.jsを開発モードで実行する際に、ログレベルとフルURLをコンソールに記録するかどうかを設定できます。

現在、`logging`は`fetch` APIを使用したデータフェッチにのみ適用されます。Next.js内の他のログにはまだ適用されていません。

```js filename="next.config.js"
module.exports = {
  logging: {
    fetches: {
      fullUrl: true,
    },
  },
}
```

[Server Components HMR cache](/docs/app/api-reference/next-config-js/serverComponentsHmrCache)から復元された`fetch`リクエストはデフォルトではログに記録されません。ただし、`logging.fetches.hmrRefreshes`を`true`に設定することで有効にできます。

```js filename="next.config.js"
module.exports = {
  logging: {
    fetches: {
      hmrRefreshes: true,
    },
  },
}
```

さらに、`logging`を`false`に設定することで開発ログを無効にすることもできます。

```js filename="next.config.js"
module.exports = {
  logging: false,
}
```
