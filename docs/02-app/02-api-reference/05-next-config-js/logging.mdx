Next.jsを開発モードで実行する際、コンソールに表示されるロギングレベルや完全なURLを設定可能です。

現在、`logging`は`fetch` API を使用したデータフェッチにのみ適用されます。Next.jsの他のログにはまだ適用されていません。

```js title="next.config.js"
module.exports = {
  logging: {
    fetches: {
      fullUrl: true, // 完全なURLを表示するための設定
    },
  },
}
```

[サーバーコンポーネントのHMRキャッシュ](/docs/app/api-reference/next-config-js/serverComponentsHmrCache) から復元されるすべての`fetch`リクエストはデフォルトではログ出力されません；しかし、`logging.fetches.hmrRefreshes`を`true`に設定することでこの機能を有効にできます。

```js title="next.config.js"
module.exports = {
  logging: {
    fetches: {
      hmrRefreshes: true, // HMRリフレッシュをログ化するための設定
    },
  },
}
```

さらに、`logging`を`false`に設定することで開発用のロギングを無効化することができます；

```js title="next.config.js"
module.exports = {
  logging: false, // ログを無効にするための設定
}
```
