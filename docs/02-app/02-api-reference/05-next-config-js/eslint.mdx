プロジェクト内でESLintが検出されると、Next.jsはエラーがある場合、**本番ビルド**（`next build`）を失敗させます。

アプリケーションにESLintのエラーがあっても、Next.jsが本番コードを生成するようにしたい場合、組み込みのリンティングステップを完全に無効にすることができます。ただし、これはESLintをワークフローの別の部分（例えば、CIやプリコミットフック）で実行するようにすでに設定されている場合を除き、お勧めしません。

`next.config.js` を開き、`eslint` 設定で `ignoreDuringBuilds` オプションを有効にします：

```js title="next.config.js"
module.exports = {
  eslint: {
    // 警告: これにより、プロジェクトにESLintエラーがある場合でも
    // 本番ビルドが正常に完了するようになります。
    ignoreDuringBuilds: true,
  },
}
```
