プロジェクトに ESLint が検出された場合、Next.js はエラーが存在するときに **プロダクションビルド** (`next build`) を失敗させます。

アプリケーションに ESLint のエラーが存在しても Next.js にプロダクションコードを生成させたい場合は、組み込みのリントステップを完全に無効にすることができます。ただし、ESLint をワークフローの別の部分（例えば、CI やプリコミットフック）の一部としてすでに実行するように設定されている場合を除いては推奨されません。

`next.config.js` を開いて、`eslint` 設定で `ignoreDuringBuilds` オプションを有効にします：

```js title="next.config.js"
module.exports = {
  eslint: {
    // 警告: この設定により、プロジェクトに ESLint のエラーがあっても
    // プロダクションビルドが正常に完了するようになります。
    ignoreDuringBuilds: true,
  },
}
```
