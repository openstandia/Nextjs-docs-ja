プロジェクトでESLintが検出された場合、Next.jsはエラーがあると**プロダクションビルド**（`next build`）に失敗します。

アプリケーションにESLintエラーがある場合でも、Next.jsがプロダクションコードを生成するようにしたい場合は、組み込みのリントステップを完全に無効にできます。ただし、これはESLintがすでにワークフローの別の部分（たとえば、CIやプリコミットフック）で実行するように設定されている場合を除き、お勧めしません。

`next.config.js`を開いて、`eslint`の設定で`ignoreDuringBuilds`オプションを有効にします：

```js filename="next.config.js"
module.exports = {
  eslint: {
    // 警告: ESLintエラーがある場合でもプロダクションビルドが正常に完了するようになります。
    ignoreDuringBuilds: true,
  },
}
```
