<AppOnly>

`devIndicators`を使用すると、開発中に現在表示しているルートについてのコンテキストを提供するオンスクリーンインジケーターを設定できます。

```ts title="Types"
  devIndicators: {
    appIsrStatus?: boolean, // デフォルトはtrue
    buildActivity?: boolean, // デフォルトはtrue
    buildActivityPosition?: 'bottom-right'
    | 'bottom-left'
    | 'top-right'
    | 'top-left', // デフォルトは'bottom-right'
  },
```

## `appIsrStatus` (静的インジケーター) {#appisrstatus-static-indicator}

> このオプションはNext.jsのcanaryで利用可能です。

Next.jsは、ビルド時にルートが事前レンダリングされるかどうかを知らせる静的インジケーターを画面の隅に表示します。これにより、ルートが静的か動的かを理解しやすくなり、ルートが[静的レンダリングをオプトアウトしている](#static-route-not-showing-the-indicator)かどうかを特定しやすくなります。

一時的にインジケーターを非表示にするには閉じるボタンをクリックします。これは`localStorage`に1時間保存されます。永久に無効にするには、`next.config.js`で設定オプションを使用します：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  devIndicators: {
    appIsrStatus: false,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  devIndicators: {
    appIsrStatus: false,
  },
}

module.exports = nextConfig
```

</TabItem>
</Tabs>

## `buildActivity` (コンパイルインジケーター) {#buildactivity-compilation-indicator}

コードを編集し、Next.jsがアプリケーションをコンパイルしているとき、コンパイルインジケーターがページの右下隅に表示されます。

> **Good to know**: このインジケーターは開発モードでのみ表示され、本番モードでアプリをビルドおよび実行したときには表示されません。

場合によっては、チャットランチャーと競合するなどしてこのインジケーターがページ上で誤って配置されることがあります。位置を変更するには、`next.config.js`を開き、`devIndicators`オブジェクト内の`buildActivityPosition`を`bottom-right`（デフォルト）、`bottom-left`、`top-right`または`top-left`に設定します：

```js title="next.config.js"
module.exports = {
  devIndicators: {
    buildActivityPosition: 'bottom-right',
  },
}
```

このインジケーターが役に立たない場合もあります。削除するには、`next.config.js`を開き、`devIndicators`オブジェクト内の`buildActivity`設定を無効にします：

```js title="next.config.js"
module.exports = {
  devIndicators: {
    buildActivity: false,
  },
}
```

## トラブルシューティング {#troubleshooting}

### 静的ルートにインジケーターが表示されない {#static-route-not-showing-the-indicator}

ルートが静的であり、インジケーターが有効になっているにもかかわらず表示されない場合、そのルートは静的レンダリングをオプトアウトしている可能性が高いです。

アプリケーションを`next build --debug`でビルドし、ターミナルの出力を確認することで、ルートが[静的](/docs/app/building-your-application/rendering/server-components#static-rendering-default)か[動的](/docs/app/building-your-application/rendering/server-components#dynamic-rendering)かを確認できます。静的（または事前レンダリングされた）ルートは`○`シンボルを表示し、動的ルートは`ƒ`シンボルを表示します。例えば：

```bash title="Build Output"
Route (app)                              Size     First Load JS
┌ ○ /_not-found                          0 B               0 kB
└ ƒ /products/[id]                       0 B               0 kB

○  (Static)   prerendered as static content
ƒ  (Dynamic)  server-rendered on demand
```

ルートが静的レンダリングをオプトアウトする理由は2つあります：

- ランタイム情報に依存する[動的API](/docs/app/building-your-application/rendering/server-components#dynamic-apis)の存在
- ORMやデータベースドライバへの呼び出しのような[未キャッシュのデータリクエスト](/docs/app/building-your-application/data-fetching/fetching)

これらの条件のいずれかをルートで確認し、ルートを静的にレンダリングできない場合は、[`loading.js`](/docs/app/api-reference/file-conventions/loading)または[`<Suspense />`](https://react.dev/reference/react/Suspense)を使用して[ストリーミング](/docs/app/building-your-application/routing/loading-ui-and-streaming#what-is-streaming)を活用することを検討してください。

<PagesOnly>

> **Good to know**: このインジケーターはNext.jsバージョン10.0.1で削除されました。最新のNext.jsへのアップグレードをお勧めします。

ページが[自動静的最適化](https://nextjs.org/docs/canary/pages/building-your-application/rendering/automatic-static-optimization)の対象となる場合、インジケーターを表示して知らせます。

これは、自動静的最適化が非常に有益であり、開発中にページが対象になるかどうかをすぐに知ることが役立つためです。

場合によってはこのインジケーターが役に立たないことがあります。たとえば、Electronアプリケーションで作業するときです。削除するには`next.config.js`を開き、`devIndicators`の`autoPrerender`設定を無効にします：

```js title="next.config.js"
module.exports = {
  devIndicators: {
    autoPrerender: false,
  },
}
```

</PagesOnly>
