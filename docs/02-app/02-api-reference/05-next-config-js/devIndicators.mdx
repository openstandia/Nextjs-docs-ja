`devIndicators` は、開発中に表示される現在のルートに関するコンテキストを提供する画面上の指標を設定できます。

```ts title="Types"
  devIndicators: {
    appIsrStatus?: boolean, // 初期値は true
    buildActivity?: boolean, // 初期値は true
    buildActivityPosition?: 'bottom-right'
    | 'bottom-left'
    | 'top-right'
    | 'top-left', // 初期値は 'bottom-right'
  },
```

## `appIsrStatus` (静的指標) {#appisrstatus-static-indicator}

Next.js は、ルートがビルド時に事前レンダリングされるかどうかを示す静的指標を画面の下隅に表示します。これにより、ルートが静的か動的かを理解しやすくなり、ルートが[静的レンダリングをオプトアウトしているかどうか](#static-route-not-showing-the-indicator)を特定できます。

この指標は一時的に非表示にすることができ、閉じるボタンをクリックすると `localStorage` に1時間の間、好みを記憶します。永久的に無効にする場合は、`next.config.js` の設定オプションを使用します：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  devIndicators: {
    appIsrStatus: false,
  },
}

export default nextConfig
```

</TabItem>
<TabItem value="js" label="JavaScript">

```js title="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  devIndicators: {
    appIsrStatus: false,
  },
}

module.exports = nextConfig
```

## `buildActivity` (コンパイル指標) {#buildactivity-compilation-indicator}

コードを編集すると、Next.js がアプリケーションをコンパイルする際に、コンパイル指標がページの右下隅に表示されます。

> **Good to know**: この指標は開発モードでのみ表示され、本番モードでアプリをビルドおよび実行している際は表示されません；

この指標が、たとえばチャットランチャーと競合する場合などに、ページ上で誤った位置に表示されることがあります。その位置を変更するには、`next.config.js` を開き、`devIndicators` オブジェクト内の `buildActivityPosition` を `bottom-right`（デフォルト）、`bottom-left`、`top-right`、または `top-left` に設定します：

```js title="next.config.js"
module.exports = {
  devIndicators: {
    buildActivityPosition: 'bottom-right',
  },
}
```

この指標が役に立たない場合があります。それを削除するには、`next.config.js` を開き、`devIndicators` オブジェクトで `buildActivity` の設定を無効にします：

```js title="next.config.js"
module.exports = {
  devIndicators: {
    buildActivity: false,
  },
}
```

## トラブルシューティング {#troubleshooting}

### 静的ルートに指標が表示されない {#static-route-not-showing-the-indicator}

ルートが静的で指標が有効であると予想されるのに表示されない場合、そのルートが静的レンダリングをオプトアウトしている可能性があります；

ルートが静的か動的かを確認するには、アプリケーションを `next build --debug` を使用してビルドし、ターミナルの出力を確認します。静的（または事前レンダリングされた）ルートは `○` 記号を表示し、動的ルートは `ƒ` 記号を表示します。例：

```bash title="ビルド出力"
Route (app)                              Size     First Load JS
┌ ○ /_not-found                          0 B               0 kB
└ ƒ /products/[id]                       0 B               0 kB

○  (Static)   prerendered as static content
ƒ  (Dynamic)  server-rendered on demand
```

ルートが静的レンダリングをオプトアウトする理由は2つあります：

- 実行時の情報に依存する[動的 API](/docs/app/building-your-application/rendering/server-components#dynamic-apis)の存在
- ORM やデータベースドライバへの呼び出しのような[キャッシュされていないデータリクエスト](/docs/app/building-your-application/data-fetching/fetching)

これらの条件をルートで確認し、ルートを静的にレンダリングすることができない場合は、[`loading.js`](/docs/app/api-reference/file-conventions/loading) または [`<Suspense />`](https://react.dev/reference/react/Suspense) を使用して[ストリーミング](/docs/app/building-your-application/routing/loading-ui-and-streaming#what-is-streaming)を活用することを検討してください。

</AppOnly>

<PagesOnly>

> **Good to know**: この指標は Next.js バージョン 10.0.1 で削除されました。Next.js の最新バージョンへのアップグレードをお勧めします；

ページが[自動静的最適化](https://nextjs.org/docs/canary/pages/building-your-application/rendering/automatic-static-optimization)に適格である場合、指標を表示して確認できます。

これは、自動静的最適化が非常に有益であり、開発中にページが適格かどうかをすぐに知るのに役立つため便利です。

この指標が役に立たない場合があり、たとえばエレクトロンアプリケーションを開発している場合に該当します。それを削除するには、`next.config.js` を開き、`devIndicators` の中で `autoPrerender` の設定を無効にします：

```js title="next.config.js"
module.exports = {
  devIndicators: {
    autoPrerender: false,
  },
}
```

</PagesOnly>
