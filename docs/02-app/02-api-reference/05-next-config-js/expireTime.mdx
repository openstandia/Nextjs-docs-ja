---

ISR対応ページの`Cache-Control`ヘッダーでCDNが参照するためのカスタム`stale-while-revalidate`の有効期限を指定することができます。

`next.config.js`を開いて、`expireTime`設定を追加します：

```js title="next.config.js"
module.exports = {
  // 1時間を秒で表現
  expireTime: 3600,
}
```

これで、`Cache-Control`ヘッダーを送信する際に、特定のリバリデート期間に応じて期限が計算されます。

例えば、あるパスで15分のリバリデートを設定していて、有効期限が1時間である場合、生成される`Cache-Control`ヘッダーは`s-maxage=900, stale-while-revalidate=2700`となり、設定された有効期限より15分短くなります。
