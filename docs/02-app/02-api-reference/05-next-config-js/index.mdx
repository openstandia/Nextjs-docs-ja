Next.js はプロジェクトディレクトリのルートにある `next.config.js` ファイル（例として、`package.json` の近く）をデフォルトエクスポートで通じて設定できます。

```js title="next.config.js"
// @ts-check

/** @type {import('next').NextConfig} */
const nextConfig = {
  /* 設定オプションはここに記載 */
}

module.exports = nextConfig
```

## ECMAScript モジュール {#ecmascript-modules}

`next.config.js` は通常の Node.js モジュールであり、JSONファイルではありません。Next.js サーバーおよびビルドフェーズで使用され、ブラウザビルドには含まれません。

[ECMAScript モジュール](https://nodejs.org/api/esm.html)が必要な場合は、`next.config.mjs` を使用できます。

```js title="next.config.mjs"
// @ts-check

/**
 * @type {import('next').NextConfig}
 */
const nextConfig = {
  /* 設定オプションはここに記載 */
}

export default nextConfig
```

> **Good to know**: `.cjs`、`.cts`、または `.mts` 拡張子を持つ `next.config` は現在サポートされていません。

## 関数としての設定 {#configuration-as-a-function}

関数も使用できます。

```js title="next.config.mjs"
// @ts-check

export default (phase, { defaultConfig }) => {
  /**
   * @type {import('next').NextConfig}
   */
  const nextConfig = {
    /* 設定オプションはここに記載 */
  }
  return nextConfig
}
```

### 非同期設定 {#async-configuration}

Next.js 12.1.0 以降、非同期関数を使用できます。

```js title="next.config.js"
// @ts-check

module.exports = async (phase, { defaultConfig }) => {
  /**
   * @type {import('next').NextConfig}
   */
  const nextConfig = {
    /* 設定オプションはここに記載 */
  }
  return nextConfig
}
```

### フェーズ {#phase}

`phase` は設定が読み込まれる現在のコンテキストです。[使用可能なフェーズ](https://github.com/vercel/next.js/blob/5e6b008b561caf2710ab7be63320a3d549474a5b/packages/next/shared/lib/constants.ts#L19-L23)を確認できます。フェーズは `next/constants` からインポートできます。

```js title="next.config.js"
// @ts-check

const { PHASE_DEVELOPMENT_SERVER } = require('next/constants')

module.exports = (phase, { defaultConfig }) => {
  if (phase === PHASE_DEVELOPMENT_SERVER) {
    return {
      /* 開発専用の設定オプションはここに記載 */
    }
  }

  return {
    /* 開発以外のすべてのフェーズの設定オプションはここに記載 */
  }
}
```

## TypeScript {#typescript}

プロジェクトで TypeScript を使用している場合、設定に TypeScript を使用するために [`next.config.ts`](/docs/app/building-your-application/configuring/typescript#type-checking-nextconfigts) を使用できます。

```ts title="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  /* 設定オプションはここに記載 */
}

export default nextConfig
```

コメント行は、`next.config.js` で許可されている設定を記載する場所であり、これらは[このファイルで定義](https://github.com/vercel/next.js/blob/canary/packages/next/src/server/config-shared.ts)されています。

しかし、どの設定も必須ではなく、各設定が何をするか理解する必要はありません。このセクションの中で必要な機能を有効または変更するために何を行うかを探してください。

> ターゲットの Node.js バージョンで使用できない新しい JavaScript 機能の使用を避けてください。`next.config.js` は Webpack や Babel で解析されません。

このページでは、利用可能なすべての設定オプションを文書化しています。

## 単体テスト（実験的） {#unit-testing-experimental}

Next.js 15.1 から、`next/experimental/testing/server` パッケージは `next.config.js` ファイルをユニットテストするためのユーティリティを提供します。

`unstable_getResponseFromNextConfig` 関数は、`next.config.js` の [`headers`](/docs/app/api-reference/next-config-js/headers)、[`redirects`](/docs/app/api-reference/next-config-js/redirects)、[`rewrites`](/docs/app/api-reference/next-config-js/rewrites) 関数を提供されたリクエスト情報で実行し、そのルーティングの結果を `NextResponse` として返します。

> `unstable_getResponseFromNextConfig` からの応答は、`next.config.js` のフィールドのみを考慮し、ミドルウェアやファイルシステムルートを考慮しないため、本番環境では単体テストの結果と異なる可能性があります。

```js
import {
  getRedirectUrl,
  unstable_getResponseFromNextConfig,
} from 'next/experimental/testing/server'

const response = await unstable_getResponseFromNextConfig({
  url: 'https://nextjs.org/test',
  nextConfig: {
    async redirects() {
      return [{ source: '/test', destination: '/test2', permanent: false }]
    },
  },
})
expect(response.status).toEqual(307)
expect(getRedirectUrl(response)).toEqual('https://nextjs.org/test2')
```
