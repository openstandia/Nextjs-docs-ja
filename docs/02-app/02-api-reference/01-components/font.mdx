### `src`

フォントファイルのパスは、フォントローダー関数が呼び出されるディレクトリに相対的な、文字列またはオブジェクト（型は `Array<{path: string, weight?: string, style?: string}>`）の配列です。

`next/font/local`で使用されます

- 必須

例：

- `src:'./fonts/my-font.woff2'`：`app` ディレクトリ内の `fonts` というディレクトリに `my-font.woff2` が配置されています
- `src:[{path: './inter/Inter-Thin.ttf', weight: '100',},{path: './inter/Inter-Regular.ttf',weight: '400',},{path: './inter/Inter-Bold-Italic.ttf', weight: '700',style: 'italic',},]`
- フォントローダー関数が `app/page.tsx` で呼び出され、`src:'../styles/fonts/my-font.ttf'` を使用する場合、`my-font.ttf` はプロジェクトのルートにある `styles/fonts` に配置されています

### `weight`

フォントの [`weight`](https://fonts.google.com/knowledge/glossary/weight) は次のいずれかです：

- 文字列：特定のフォントに利用可能なウェイトの値、または [variable](https://fonts.google.com/variablefonts) フォントの場合、値の範囲を持ちます
- フォントが [variable google font](https://fonts.google.com/variablefonts)でない場合、重さの値の配列です。これは `next/font/google` のみに適用されます

`next/font/google` と `next/font/local` で使用されます

- 使用されているフォントが [variable](https://fonts.google.com/variablefonts) で**ない**場合は必須

例：

- `weight: '400'`：単一のウェイト値の文字列 - フォント [`Inter`](https://fonts.google.com/specimen/Inter?query=inter) の場合、可能な値は `'100'`、`'200'`、`'300'`、`'400'`、`'500'`、`'600'`、`'700'`、`'800'`、`'900'` または `'variable'` で、デフォルトは `'variable'` です
- `weight: '100 900'`：variable フォント用の `100` と `900` の間の範囲の文字列
- `weight: ['100','400','900']`：非 variable フォント用の3つの可能な値の配列

### `style`

フォントの [`style`](https://developer.mozilla.org/docs/Web/CSS/font-style) は次のいずれかです：

- デフォルト値は `'normal'` の [value](https://developer.mozilla.org/docs/Web/CSS/font-style#values) の文字列
- フォントが [variable google font](https://fonts.google.com/variablefonts)でない場合、スタイル値の配列です。それは `next/font/google` のみに適用されます

`next/font/google` と `next/font/local` で使用されます

- 省略可能

例：

- `style: 'italic'`：文字列 - `next/font/google` の場合は `normal` または `italic` で指定できます
- `style: 'oblique'`：文字列 - `next/font/local` の場合は任意の値をとることができますが、[標準のフォントスタイル](https://developer.mozilla.org/docs/Web/CSS/font-style)からくると予想されます
- `style: ['italic','normal']`：`next/font/google` の2つの値の配列 - `normal` および `italic` からの値です

### `subsets`

配列の文字列値で定義されるフォントの[`subsets`](https://fonts.google.com/knowledge/glossary/subsetting) で、[preloaded](/docs/app/building-your-application/optimizing/fonts#specifying-a-subset)されるサブセットを指定します。`subsets` で指定されたフォントは、[`preload`](#preload) オプションが true のときに、head にプリロードリンクタグが挿入されます（デフォルトは true です）。

`next/font/google`で使用されます

- 省略可能

例：

- `subsets: ['latin']`：サブセット `latin` の配列

Google Fonts ページでフォントに合わせた全サブセットの一覧を見つけることができます。

### `axes`

一部の variable フォントには追加の `axes` を含めることができます。デフォルトでは、ファイルサイズを小さく保つためにフォントの重さのみが含まれます。`axes` の値の可能性は特定のフォントに依存します。

`next/font/google`で使用されます

- 省略可能

例：

- `axes: ['slnt']`：`slnt` の値を持つ `Inter` variable フォントの配列で、[ここ](https://fonts.google.com/variablefonts?vfquery=inter#font-families)で示されているように `slnt` を追加の `axes` に持ちます。可能な `axes` の値を見つけるには、[Google variable fonts page](https://fonts.google.com/variablefonts#font-families)のフィルターを使用し、`wght` 以外のaxesを探してください

### `display`

フォントの[`display`](https://developer.mozilla.org/docs/Web/CSS/@font-face/font-display) で、可能な文字列 `values` は `'auto'`、 `'block'`、 `'swap'`、 `'fallback'` または `'optional'` で、デフォルト値は `'swap'` です。

`next/font/google` と `next/font/local` で使用されます

- 省略可能

例：

- `display: 'optional'`：`optional` に割り当てられた文字列

### `preload`

フォントを[プリロード](/docs/app/building-your-application/optimizing/fonts#preloading)するかどうかを指定するBoolean値です。デフォルトは `true` です。

`next/font/google` と `next/font/local` で使用されます

- 省略可能

例：

- `preload: false`

### `fallback`

フォントが読み込めない場合に使用するフォントのフォールバックです。フォールバックフォントの文字列配列で、デフォルトはありません。

- 省略可能

`next/font/google` と `next/font/local` で使用されます

例：

- `fallback: ['system-ui', 'arial']`：`system-ui` か `arial` のフォールバックフォントを設定する配列

### `adjustFontFallback`

- `next/font/google` の場合：Boolean値で、[累積レイアウト シフト](https://web.dev/cls/)を減らすために自動フォールバックフォントを使用するかどうかを設定します。デフォルトは `true` です。
- `next/font/local` の場合：文字列またはBoolean `false` 値で、[累積レイアウト シフト](https://web.dev/cls/)を減らすために使用する自動フォールバックフォントを設定します。可能な値は `'Arial'`、`'Times New Roman'` または `false` です。デフォルトは `'Arial'` です。

`next/font/google` と `next/font/local` で使用されます

- 省略可能

例：

- `adjustFontFallback: false`：`next/font/google` 用
- `adjustFontFallback: 'Times New Roman'`：`next/font/local` 用

### `variable`

スタイルを[CSS 変数メソッド](#css-variables)で適用する場合に使用するCSS変数名を定義する文字列値です。

`next/font/google` と `next/font/local` で使用されます

- 省略可能

例：

- `variable: '--my-font'`：CSS変数 `--my-font` が宣言されています

### `declarations`

生成された `@font-face` をさらに定義するフォントフェイス [descriptor](https://developer.mozilla.org/docs/Web/CSS/@font-face#descriptors) キーと値のペアの配列です。

`next/font/local`で使用されます

- 省略可能

例：

- `declarations: [{ prop: 'ascent-override', value: '90%' }]`

## スタイルを適用する

フォントスタイルを適用する方法は3つあります：

- [`className`](#classname)
- [`style`](#style-1)
- [CSS Variables](#css-variables)

### `className`

読み込み済みフォントのCSS `className`を返し、HTML要素に渡します。

```tsx
<p className={inter.className}>Hello, Next.js!</p>
```

### `style`

読み込み済みフォントのCSS `style` オブジェクトを返し、HTML要素に渡します。`style.fontFamily` を含み、フォントファミリー名とフォールバックフォントにアクセスできます。

```tsx
<p style={inter.style}>Hello World</p>
```

### CSS Variables

外部スタイルシートでスタイルを設定し、そこで追加のオプションを指定する場合には、CSS変数メソッドを使用します。

フォントのインポートに加えて、CSS変数が定義されているCSSファイルもインポートし、フォントローダーオブジェクトの変数オプションを次のように設定します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import { Inter } from 'next/font/google'
import styles from '../styles/component.module.css'

const inter = Inter({
  variable: '--font-inter',
})
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher
import { Inter } from 'next/font/google'
import styles from '../styles/component.module.css'

const inter = Inter({
  variable: '--font-inter',
})
```
</TabItem>
</Tabs>


フォントを使用するには、スタイルを適用したいテキストの親コンテナにフォントローダーの `variable` 値を `className` に設定し、テキストの `className` を外部CSSファイルの `styles`プロパティに設定します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
<main className={inter.variable}>
  <p className={styles.text}>Hello World</p>
</main>
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher
<main className={inter.variable}>
  <p className={styles.text}>Hello World</p>
</main>
```
</TabItem>
</Tabs>


`component.module.css` CSSファイルで `text`セレクタークラスを次のように定義します：

```css title="styles/component.module.css"
.text {
  font-family: var(--font-inter);
  font-weight: 200;
  font-style: italic;
}
```

上記の例では、テキスト `Hello World` が `Inter` フォントと生成されたフォールバックフォントを使用してスタイル化され、`font-weight: 200` と `font-style: italic` が適用されています。

## フォント定義ファイルの使用

`localFont` または Google フォント関数を呼び出すたびに、そのフォントはアプリケーション内で1つのインスタンスとしてホストされます。したがって、同じフォントを複数の場所で使用する必要がある場合は、1か所で読み込み、必要な場所で関連するフォントオブジェクトをインポートする必要があります。これはフォント定義ファイルを使用して行われます。

たとえば、アプリディレクトリのルートにある `styles` フォルダーに `fonts.ts` ファイルを作成します。

次に、フォント定義を次のように指定します：

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="styles/fonts.ts" switcher
import { Inter, Lora, Source_Sans_3 } from 'next/font/google'
import localFont from 'next/font/local'

// variable フォントを定義します
const inter = Inter()
const lora = Lora()
// 重さが2つの非 variable フォントを定義します
const sourceCodePro400 = Source_Sans_3({ weight: '400' })
const sourceCodePro700 = Source_Sans_3({ weight: '700' })
// styles フォルダーに配置された GreatVibes-Regular.ttf というカスタムローカルフォントを定義します
const greatVibes = localFont({ src: './GreatVibes-Regular.ttf' })

export { inter, lora, sourceCodePro400, sourceCodePro700, greatVibes }
```
</TabItem>
<TabItem value="js" label="JavaScript">

```js title="styles/fonts.js" switcher
import { Inter, Lora, Source_Sans_3 } from 'next/font/google'
import localFont from 'next/font/local'

// variable フォントを定義します
const inter = Inter()
const lora = Lora()
// 重さが2つの非 variable フォントを定義します
const sourceCodePro400 = Source_Sans_3({ weight: '400' })
const sourceCodePro700 = Source_Sans_3({ weight: '700' })
// styles フォルダーに配置された GreatVibes-Regular.ttf というカスタムローカルフォントを定義します
const greatVibes = localFont({ src: './GreatVibes-Regular.ttf' })

export { inter, lora, sourceCodePro400, sourceCodePro700, greatVibes }
```
</TabItem>
</Tabs>


これらの定義をコード内で次のように使用できます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import { inter, lora, sourceCodePro700, greatVibes } from '../styles/fonts'

export default function Page() {
  return (
    <div>
      <p className={inter.className}>Inter フォントを使ったHello world</p>
      <p style={lora.style}>Lora フォントを使ったHello world</p>
      <p className={sourceCodePro700.className}>
        重さ700のSource_Sans_3 フォントを使ったHello world
      </p>
      <p className={greatVibes.className}>Great Vibes フォントのタイトル</p>
    </div>
  )
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher
import { inter, lora, sourceCodePro700, greatVibes } from '../styles/fonts'

export default function Page() {
  return (
    <div>
      <p className={inter.className}>Inter フォントを使ったHello world</p>
      <p style={lora.style}>Lora フォントを使ったHello world</p>
      <p className={sourceCodePro700.className}>
        重さ700のSource_Sans_3 フォントを使ったHello world
      </p>
      <p className={greatVibes.className}>Great Vibes フォントのタイトル</p>
    </div>
  )
}
```
</TabItem>
</Tabs>


コード内でフォント定義にアクセスしやすくするために、`tsconfig.json` または `jsconfig.json` ファイルでパスエイリアスを次のように定義できます：

```json title="tsconfig.json"
{
  "compilerOptions": {
    "paths": {
      "@/fonts": ["./styles/fonts"]
    }
  }
}
```
