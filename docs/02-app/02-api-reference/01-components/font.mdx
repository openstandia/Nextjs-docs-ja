### `src`

フォントファイルのパスを文字列またはオブジェクトの配列（タイプは`Array<{path: string, weight?: string, style?: string}>`）として、フォントローダー関数が呼び出されるディレクトリに対して相対的に指定します。

`next/font/local`で使用されます

- 必須

例:

- `src:'./fonts/my-font.woff2'` では、`my-font.woff2`が`アプリ`ディレクトリ内の`fonts`という名前のディレクトリに配置されます
- `src:[{path: './inter/Inter-Thin.ttf', weight: '100',},{path: './inter/Inter-Regular.ttf',weight: '400',},{path: './inter/Inter-Bold-Italic.ttf', weight: '700',style: 'italic',},]`
- フォントローダー関数が`app/page.tsx`で呼び出された場合に`src:'../styles/fonts/my-font.ttf'`を使用すると、`my-font.ttf`はプロジェクトのルートにある`styles/fonts`に配置されます

### `weight`

フォントの[`weight`](https://fonts.google.com/knowledge/glossary/weight)で以下の可能性があります：

- 特定のフォントに利用可能なウェイトの値、または[可変](https://fonts.google.com/variablefonts)フォントの場合は、値の範囲の文字列
- フォントが[可変googleフォント](https://fonts.google.com/variablefonts)でない場合のウェイト値の配列。これは`next/font/google`にのみ適用されます。

`next/font/google`および`next/font/local`で使用されます

- 使用されるフォントが[可変](https://fonts.google.com/variablefonts)でない場合は必須

例:

- `weight: '400'`: 単一のウェイト値を示す文字列 - フォント[`Inter`](https://fonts.google.com/specimen/Inter?query=inter)の場合、利用可能な値は`'100'`、`'200'`、`'300'`、`'400'`、`'500'`、`'600'`、`'700'`、`'800'`、`'900'`または`'variable'`で、`'variable'`がデフォルトです）
- `weight: '100 900'`: 可変フォントの`100`と`900`の範囲を示す文字列
- `weight: ['100','400','900']`: 非可変フォントに対する3つの可能な値の配列

### `style`

フォントの[`style`](https://developer.mozilla.org/docs/Web/CSS/font-style)で以下の可能性があります：

- デフォルトで`'normal'`の文字列[値](https://developer.mozilla.org/docs/Web/CSS/font-style#values)
- フォントが[可変googleフォント](https://fonts.google.com/variablefonts)でない場合のスタイル値の配列。これは`next/font/google`にのみ適用されます。

`next/font/google`および`next/font/local`で使用されます

- 任意

例:

- `style: 'italic'`: 文字列 - `next/font/google`用として`normal`または`italic`で指定できます
- `style: 'oblique'`: 文字列 - `next/font/local`用として任意の値を取れますが、通常は[標準フォントスタイル](https://developer.mozilla.org/docs/Web/CSS/font-style)から取られたものが期待されます
- `style: ['italic','normal']`: `next/font/google`用の2つの値の配列 - 値は`normal`および`italic`から取ります

### `subsets`

フォントの[`subsets`](https://fonts.google.com/knowledge/glossary/subsetting)で、文字列値の配列によって定義されます。その文字列値には、[プリロード](/docs/app/building-your-application/optimizing/fonts#specifying-a-subset)したい各サブセットの名前が含まれています。`subsets`で指定されたフォントは、[`preload`](#preload)オプションがtrue（デフォルト）であるときにheadにリンクプリロードタグが挿入されます。

`next/font/google`で使用されます

- 任意

例:

- `subsets: ['latin']`: サブセット`latin`を含む配列

フォントのGoogle Fontsページで、すべてのサブセットのリストを見つけることができます。

### `axes`

一部の可変フォントには追加の`axes`が含まれています。デフォルトでは、ファイルサイズを抑えるためにフォントのウェイトのみが含まれています。`axes`の可能な値は特定のフォントによります。

`next/font/google`で使用されます

- 任意

例:

- `axes: ['slnt']`: 追加の`axes`として`slnt`を持つ`Inter`可変フォントの値`slnt`を含む配列は、[ここ](https://fonts.google.com/variablefonts?vfquery=inter#font-families)に示されています。Googleの可変フォントページのフィルターを使用し、`wght`以外の`axes`を探すことで、フォントの`axes`の可能な値を見つけることができます。

### `display`

フォントの[`display`](https://developer.mozilla.org/docs/Web/CSS/@font-face/font-display)で可能な文字列[値](https://developer.mozilla.org/docs/Web/CSS/@font-face/font-display#values)は`'auto'`、`'block'`、`'swap'`、`'fallback'`または`'optional'`であり、デフォルトは`'swap'`です。

`next/font/google`および`next/font/local`で使用されます

- 任意

例:

- `display: 'optional'`: `optional`値に割り当てられた文字列

### `preload`

フォントを[プリロードする](/docs/app/building-your-application/optimizing/fonts#preloading)かどうかを指定するブール値。デフォルトは`true`です。

`next/font/google`および`next/font/local`で使用されます

- 任意

例:

- `preload: false`

### `fallback`

フォントが読み込めない場合に使用するフォールバックフォント。デフォルトはなく、フォールバックフォントの文字列の配列。

- 任意

`next/font/google`および`next/font/local`で使用されます

例:

- `fallback: ['system-ui', 'arial']`: フォールバックフォントを`system-ui`または`arial`に設定する配列

### `adjustFontFallback`

- `next/font/google`の場合：自動フォールバックフォントを使用して[Cumulative Layout Shift](https://web.dev/cls/)を減少させるかどうかを設定するブール値。デフォルトは`true`です。
- `next/font/local`の場合：自動フォールバックフォントを使用して[Cumulative Layout Shift](https://web.dev/cls/)を減少させるかどうかを設定する文字列または`false`のブール値。可能な値は`'Arial'`、`'Times New Roman'`または`false`です。デフォルトは`'Arial'`です。

`next/font/google`および`next/font/local`で使用されます

- 任意

例:

- `adjustFontFallback: false`: `next/font/google`用
- `adjustFontFallback: 'Times New Roman'`: `next/font/local`用

### `variable`

スタイルを[CSS変数メソッド](#css-variables)を使用して適用する場合、使用するCSS変数名を定義する文字列値。

`next/font/google`および`next/font/local`で使用されます

- 任意

例:

- `variable: '--my-font'`: CSS変数`--my-font`が宣言されます

### `declarations`

生成された`@font-face`をさらに定義するフォントフェイス[ディスクリプタ](https://developer.mozilla.org/docs/Web/CSS/@font-face#descriptors)のキーと値のペアの配列。

`next/font/local`で使用されます

- 任意

例:

- `declarations: [{ prop: 'ascent-override', value: '90%' }]`

## スタイルの適用

フォントスタイルを適用するには、次の3つの方法があります：

- [`className`](#classname)
- [`style`](#style-1)
- [CSS変数](#css-variables)

### `className`

読み込んだフォントの読み取り専用のCSS `className`を取得し、HTML要素に渡します。

```tsx
<p className={inter.className}>Hello, Next.js!</p>
```

### `style`

読み込んだフォントの読み取り専用のCSS `style`オブジェクトを取得し、HTML要素に渡します。このオブジェクトにはフォントファミリ名とフォールバックフォントにアクセスするための`style.fontFamily`が含まれています。

```tsx
<p style={inter.style}>Hello World</p>
```

### CSS変数

外部スタイルシートにスタイルを設定し、そこで追加のオプションを指定したい場合、CSS変数メソッドを使用します。

フォントをインポートするだけでなく、CSS変数が定義されているCSSファイルもインポートし、フォントローダーオブジェクトの変数オプションを次のように設定します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import { Inter } from 'next/font/google'
import styles from '../styles/component.module.css'

const inter = Inter({
  variable: '--font-inter',
})
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher
import { Inter } from 'next/font/google'
import styles from '../styles/component.module.css'

const inter = Inter({
  variable: '--font-inter',
})
```
</TabItem>
</Tabs>


フォントを使用するには、スタイルを適用したいテキストの親コンテナの`className`をフォントローダーの`variable`値に設定し、テキストの`className`を外部CSSファイルの`styles`プロパティに設定します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
<main className={inter.variable}>
  <p className={styles.text}>Hello World</p>
</main>
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher
<main className={inter.variable}>
  <p className={styles.text}>Hello World</p>
</main>
```
</TabItem>
</Tabs>


`component.module.css` CSSファイル内で`text`セレクタークラスを次のように定義します。

```css title="styles/component.module.css"
.text {
  font-family: var(--font-inter);
  font-weight: 200;
  font-style: italic;
}
```

上記の例では、`Inter`フォントと生成されたフォントフォールバックを使用して、テキスト`Hello World`が`font-weight: 200`と`font-style: italic`でスタイリングされています。

## フォント定義ファイルの使用

フォントを`localFont`またはGoogleフォント関数で呼び出すたびに、アプリケーションで1つのインスタンスとしてホストされます。したがって、複数の場所で同じフォントを使用する必要がある場合、1つの場所でロードし、関連フォントオブジェクトを必要な場所にインポートする必要があります。これはフォント定義ファイルを使用して行います。

例えば、`styles`フォルダ内にアプリディレクトリのルートに`fonts.ts`ファイルを作成します。

次に、次のようにフォント定義を指定します。

<Tabs>
<TabItem value="ts" label="TypeScript">

```ts title="styles/fonts.ts" switcher
import { Inter, Lora, Source_Sans_3 } from 'next/font/google'
import localFont from 'next/font/local'

// 可変フォントを定義します
const inter = Inter()
const lora = Lora()
// 非可変フォントの2つのウェイトを定義します
const sourceCodePro400 = Source_Sans_3({ weight: '400' })
const sourceCodePro700 = Source_Sans_3({ weight: '700' })
// GreatVibes-Regular.ttfがstylesフォルダに保存されているカスタムローカルフォントを定義します
const greatVibes = localFont({ src: './GreatVibes-Regular.ttf' })

export { inter, lora, sourceCodePro400, sourceCodePro700, greatVibes }
```
</TabItem>
<TabItem value="js" label="JavaScript">

```js title="styles/fonts.js" switcher
import { Inter, Lora, Source_Sans_3 } from 'next/font/google'
import localFont from 'next/font/local'

// 可変フォントを定義します
const inter = Inter()
const lora = Lora()
// 非可変フォントの2つのウェイトを定義します
const sourceCodePro400 = Source_Sans_3({ weight: '400' })
const sourceCodePro700 = Source_Sans_3({ weight: '700' })
// GreatVibes-Regular.ttfがstylesフォルダに保存されているカスタムローカルフォントを定義します
const greatVibes = localFont({ src: './GreatVibes-Regular.ttf' })

export { inter, lora, sourceCodePro400, sourceCodePro700, greatVibes }
```
</TabItem>
</Tabs>


これらの定義を次のようにコードで使用できます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import { inter, lora, sourceCodePro700, greatVibes } from '../styles/fonts'

export default function Page() {
  return (
    <div>
      <p className={inter.className}>Hello world using Inter font</p>
      <p style={lora.style}>Hello world using Lora font</p>
      <p className={sourceCodePro700.className}>
        Hello world using Source_Sans_3 font with weight 700
      </p>
      <p className={greatVibes.className}>My title in Great Vibes font</p>
    </div>
  )
}
```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher
import { inter, lora, sourceCodePro700, greatVibes } from '../styles/fonts'

export default function Page() {
  return (
    <div>
      <p className={inter.className}>Hello world using Inter font</p>
      <p style={lora.style}>Hello world using Lora font</p>
      <p className={sourceCodePro700.className}>
        Hello world using Source_Sans_3 font with weight 700
      </p>
      <p className={greatVibes.className}>My title in Great Vibes font</p>
    </div>
  )
}
```
</TabItem>
</Tabs>


コードでフォント定義にアクセスしやすくするために、`tsconfig.json`または`jsconfig.json`ファイルで次のようにパスエイリアスを定義できます：

```json title="tsconfig.json"
{
  "compilerOptions": {
    "paths": {
      "@/fonts": ["./styles/fonts"]
    }
  }
}
```

次のようにして任意のフォント定義をインポートできます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/about/page.tsx" switcher

```
</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/about/page.js" switcher

```
</TabItem>
</Tabs>
