> このAPIは将来的に廃止される予定です。バージョン15では、`unstable_noStore`の代わりに[`connection`](/docs/app/api-reference/functions/connection)を使用することをお勧めします。

`unstable_noStore` は静的レンダリングをオプトアウトし、特定のコンポーネントをキャッシュしないように宣言的に示すために使用できます。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```

> **Good to know**:
>
> - `unstable_noStore` は `fetch` の `cache: 'no-store'` と同等です
> - `unstable_noStore` は `export const dynamic = 'force-dynamic'` よりも推奨されます；これはより細かい単位で使用でき、コンポーネントごとに適用できるためです

- [`unstable_cache`](/docs/app/api-reference/legacy-apis/unstable_cache) 内で `unstable_noStore` を使用しても、静的生成からオプトアウトされません；代わりに、キャッシュの結果を保存するかどうかを決定するためにキャッシュの構成に委ねられます。

## 使用法 {#usage}

`fetch` に `cache: 'no-store'`、`next: { revalidate: 0 }` などの追加オプションを渡したくない場合や、`fetch` が利用できない場合には、これらのすべてのユースケースの代替として `noStore()` を使用できます。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```
