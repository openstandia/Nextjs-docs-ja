`headers`は、[Server Component](/docs/app/building-your-application/rendering/server-components)からHTTP受信リクエストヘッダーを**読み取る**ことを可能にする**非同期**関数です。

```tsx filename="app/page.tsx" switcher
import { headers } from 'next/headers'

export default async function Page() {
  const headersList = await headers()
  const userAgent = headersList.get('user-agent')
}
```

```jsx filename="app/page.js" switcher
import { headers } from 'next/headers'

export default async function Page() {
  const headersList = await headers()
  const userAgent = headersList.get('user-agent')
}
```

## リファレンス

### パラメーター

`headers`はパラメーターを受け取りません。

### 戻り値

`headers`は**読み取り専用**の[Web Headers](https://developer.mozilla.org/docs/Web/API/Headers)オブジェクトを返します。

- [`Headers.entries()`](https://developer.mozilla.org/docs/Web/API/Headers/entries): このオブジェクトに含まれるすべてのキー/バリューペアを順に処理するための[`iterator`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Iteration_protocols)を返します
- [`Headers.forEach()`](https://developer.mozilla.org/docs/Web/API/Headers/forEach): この`Headers`オブジェクト内の各キー/バリューペアに対して提供された関数を一度実行します
- [`Headers.get()`](https://developer.mozilla.org/docs/Web/API/Headers/get): 与えられた名前の`Headers`オブジェクト内のヘッダーのすべての値の[`String`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/String)シーケンスを返します
- [`Headers.has()`](https://developer.mozilla.org/docs/Web/API/Headers/has): `Headers`オブジェクトが特定のヘッダーを含んでいるかどうかを示すブール値を返します
- [`Headers.keys()`](https://developer.mozilla.org/docs/Web/API/Headers/keys): このオブジェクトに含まれるすべてのキー/バリューペアのキーを順に処理するための[`iterator`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Iteration_protocols)を返します
- [`Headers.values()`](https://developer.mozilla.org/docs/Web/API/Headers/values): このオブジェクトに含まれるすべてのキー/バリューペアの値を順に処理するための[`iterator`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Iteration_protocols)を返します

## Good to know

- `headers`は**非同期**関数であり、Promiseを返します。`async/await`またはReactの[`use`](https://react.dev/reference/react/use)関数を使用する必要があります。
  - バージョン14以前では、`headers`は同期関数でした。後方互換性を確保するために、Next.js 15ではまだ同期的にアクセスできますが、この動作は将来的にdeprecatedされます。
- `headers`は読み取り専用であるため、送信リクエストヘッダーを`set`または`delete`することはできません。
- `headers`は、事前に返される値を知られない[Dynamic API](/docs/app/building-your-application/rendering/server-components#server-rendering-strategies#dynamic-apis)であり、これを使用すると独自のルートは**[dynamic rendering](/docs/app/building-your-application/rendering/server-components#dynamic-rendering)**に適用されます。

## 例

### Authorizationヘッダーの使用

```jsx filename="app/page.js"
import { headers } from 'next/headers'

export default async function Page() {
  const authorization = (await headers()).get('authorization')
  const res = await fetch('...', {
    headers: { authorization }, // Authorizationヘッダーを転送する
  })
  const user = await res.json()

  return <h1>{user.name}</h1>
}
```
