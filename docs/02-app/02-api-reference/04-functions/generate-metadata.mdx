このページでは、`generateMetadata`と静的metadataオブジェクトを使用した**設定に基づくMetadata**オプションについて説明します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

// 静的metadataの場合
export const metadata: Metadata = {
  title: '...',
}

// または動的metadata
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher
// 静的metadataの場合
export const metadata = {
  title: '...',
}

// または動的metadata
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
</Tabs>

> **Good to know**:
>
> - `metadata`オブジェクトと`generateMetadata`関数のエクスポートは、**server component内でのみサポートされています**。
> - 同一のroute segmentから`metadata`オブジェクトと`generateMetadata`関数の両方をエクスポートすることはできません。
> - 初回ロード時には、`loading.js`からのコンテンツを含む`generateMetadata`が完全に解決するまでストリーミングがブロックされます。

静的metadataを定義するには、`layout.js`または`page.js`ファイルから[`Metadata`オブジェクト](#metadata-fields)をエクスポートします。

## `generateMetadata`関数 {#generatemetadata-function}

動的metadataは、現在のルートパラメータ、外部データ、または親セグメントでの`metadata`など、**動的な情報**に依存します。これを設定するために、[`Metadata`オブジェクト](#metadata-fields)を返す`generateMetadata`関数をエクスポートすることができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[id]/page.tsx" switcher
import type { Metadata, ResolvingMetadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  // ルートパラメータを読み取る
  const id = (await params).id

  // データを取得する
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // オプションとして親metadataをアクセスして拡張する（置き換えるのではなく）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }: Props) {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[id]/page.js" switcher
export async function generateMetadata({ params, searchParams }, parent) {
  // ルートパラメータを読み取る
  const id = (await params).id

  // データを取得する
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // オプションとして親metadataをアクセスして拡張する（置き換えるのではなく）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }) {}
```

</TabItem>
</Tabs>

### パラメータ {#parameters}

`generateMetadata`関数は次のパラメータを受け取ります：

- `props` - 現在のルートのパラメータを含むオブジェクト：

  - `params` - [動的ルートパラメータ](/docs/app/building-your-application/routing/dynamic-routes)オブジェクトをroot segmentから`generateMetadata`が呼び出されているsegmentへ含むオブジェクト。例：

    | Route                           | URL         | `params`                  |
    | ------------------------------- | ----------- | ------------------------- |
    | `app/shop/[slug]/page.js`       | `/shop/1`   | `{ slug: '1' }`           |
    | `app/shop/[tag]/[item]/page.js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |
    | `app/shop/[...slug]/page.js`    | `/shop/1/2` | `{ slug: ['1', '2'] }`    |

  - `searchParams` - 現在のURLの[検索パラメータ](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)を含むオブジェクト。例：

    | URL             | `searchParams`       |
    | --------------- | -------------------- |
    | `/shop?a=1`     | `{ a: '1' }`         |
    | `/shop?a=1&b=2` | `{ a: '1', b: '2' }` |
    | `/shop?a=1&a=2` | `{ a: ['1', '2'] }`  |

- `parent` - 親ルートセグメントから解決されたmetadataのプロミス。

### 戻り値 {#returns}

`generateMetadata`は、1つ以上のmetadataフィールドを含む[`Metadata`オブジェクト](#metadata-fields)を返すべきです。

> **Good to know**:
>
> - メタデータがランタイム情報に依存しない場合、`generateMetadata`の代わりに静的な[`metadata`オブジェクト](#the-metadata-object)を使用して定義するべきです。
> - `fetch`リクエストは同じデータに対して`generateMetadata`、`generateStaticParams`、Layouts、Pages、Server Componentsの間で[自動的にmemo化](/docs/app/building-your-application/caching#request-memoization)されます。`fetch`が利用できない場合はReactの[`cache`関数を使用できます](/docs/app/building-your-application/caching#react-cache-function)。
> - `searchParams`は`page.js`セグメント内でのみ利用可能です。
> - [`redirect()`](/docs/app/api-reference/functions/redirect)や[`notFound()`](/docs/app/api-reference/functions/not-found)といったNext.jsのメソッドは`generateMetadata`内でも使用できます。

## Metadata Fields {#metadata-fields}

### `title` {#title}

`title`属性は、ドキュメントのタイトルを設定するために使用されます。簡単な[文字列](#string)やオプションの[テンプレートオブジェクト](#template-object)として定義することができます。

#### 文字列 {#string}

```jsx title="layout.js | page.js"
}

```

##### Default {#default}

`title.default`は、`title`を定義していない子ルートセグメントに**フォールバックタイトル**を提供するために使用できます。

```tsx title="app/layout.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    default: 'Acme',
  },
}
```

```tsx title="app/about/page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {}

// 出力: <title>Acme</title>
```

##### テンプレート {#template}

`title.template`は、**子**ルートセグメントに定義されている`title`にプレフィックスまたはサフィックスを追加するために使用できます。

```tsx title="app/layout.tsx" switcher

    default: 'Acme', // テンプレートを作成する際はデフォルトが必須です

```

</TabItem>

```jsx title="app/layout.js" switcher

    template: '%s | Acme',

  },

```

</TabItem>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/about/page.tsx" switcher

  title: 'About',

// 出力: <title>About | Acme</title>
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/about/page.js" switcher

  title: 'About',
}

// 出力: <title>About | Acme</title>
```

> **Good to know**:
>
> - `title.template`は、**子**ルートセグメントに適用され、定義されているセグメントには**適用されません**。これには次の意味があります：
>
>   - `title.template`を追加するときは`title.default`が**必須**です。
>   - `layout.js`に定義された`title.template`は同じルートセグメントの`page.js`に定義された`title`には適用されません。
>   - `page.js`に定義された`title.template`は効果がありません。なぜならページは常にターミネーションセグメントであるため、子ルートセグメントを持たないためです。
>
> - ルートが`title`または`title.default`を定義していない場合、`title.template`は**効果がありません**。

> **Good to know**:
>
> - `layout.js`
>
>   - `title`（文字列）と`title.default`は子セグメント（自身の`title`を定義していない）に対するデフォルトのタイトルを定義します。それは、最も近い親セグメントからの`title.template`を補完します。
>   - `title.absolute`は、親セグメントからの`title.template`を無視して子セグメントに対するデフォルトのタイトルを定義します。
>   - `title.template`は、子セグメントに対する新しいタイトルテンプレートを定義します。
>
> - `page.js`
>   - ページが自身のタイトルを定義していない場合、一番近い親の解決されたタイトルが使用されます。
>   - `title`（文字列）はルートのタイトルを定義します。それは、最も近い親セグメントからの`title.template`を補完します。
>   - `title.absolute`は、親セグメントからの`title.template`を無視してルートタイトルを定義します。
>   - `page.js`内での`title.template`は効果を持ちません。ページは常にルートのターミネーションセグメントであるためです。

### `metadataBase` {#metadatabase}

`metadataBase`は、完全修飾URLが必要な`metadata`フィールドにベースURLのプレフィックスを設定するための便利なオプションです。

- `metadataBase`を設定すると、**現在のルートセグメントとその下**に定義されたURLベースの`metadata`フィールドを使用して、これまで必要とされた絶対URLの代わりに**相対パス**が使用できます。
- フィールドの相対パスは`metadataBase`と組み合わせて完全修飾URLを形成します。
- 設定されていない場合、`metadataBase`は[デフォルト値](#default-value)で**自動的に埋め込まれます**。

```jsx title="layout.js | page.js"
export const metadata = {
  metadataBase: new URL('https://acme.com'),
  alternates: {
    canonical: '/',
    languages: {
      'en-US': '/en-US',
      'de-DE': '/de-DE',
    },
  },
  openGraph: {
    images: '/og-image.png',
  },
}
```

```html title="<head> output" hideLineNumbers
<link rel="canonical" href="https://acme.com" />
<link rel="alternate" hreflang="en-US" href="https://acme.com/en-US" />
<link rel="alternate" hreflang="de-DE" href="https://acme.com/de-DE" />
<meta property="og:image" content="https://acme.com/og-image.png" />
```

> **Good to know**:
>
> - `metadataBase`は通常、rootの`app/layout.js`で設定され、すべてのルートでURLベースの`metadata`フィールドに適用されます。
> - 絶対URLを必要とするすべてのURLベースの`metadata`フィールドは、`metadataBase`オプションで設定できます。
> - `metadataBase`には、サブドメイン（例：`https://app.acme.com`）やベースパス（例：`https://acme.com/start/from/here`）を含めることができます
> - `metadata`フィールドが絶対URLを提供する場合、`metadataBase`は無視されます。
> - URLベースの`metadata`フィールドで`metadataBase`を設定せずに相対パスを使用すると、ビルドエラーが発生します。
> - Next.jsは、`metadataBase`（例：`https://acme.com/`）と相対フィールド（例：`/path`）の間で重複するスラッシュを1つのスラッシュ（例：`https://acme.com/path`）に正規化します

#### デフォルト値 {#default-value}

設定されていない場合、`metadataBase`は**デフォルト値**を持ちます。

> Vercel上で：
>
> - プロダクションデプロイメントでは、`VERCEL_PROJECT_PRODUCTION_URL`が使用されます。
> - プレビュー環境では、`VERCEL_BRANCH_URL`が優先され、存在しない場合は`VERCEL_URL`がフォールバックとして使用されます。
>
> これらの値が存在する場合、`metadataBase`の**デフォルト値**として使用されます。そうでない場合、`http://localhost:${process.env.PORT || 3000}`にフォールバックします。これにより、Open Graph画像がローカルビルドとVercelのプレビューおよびプロダクションデプロイメントの両方で機能するようになります。デフォルトを上書きするときは、ローカル開発、ステージング、およびプロダクション環境用にURLを計算するために環境変数を使用することをお勧めします。
>
> これらの環境変数の詳細については、[システム環境変数](https://vercel.com/docs/concepts/projects/environment-variables/system-environment-variables)ドキュメントを参照してください。

#### URLの構成 {#url-composition}

URLの構成は、デフォルトのディレクトリトラバースのセマンティクスよりも開発者の意図を優先します。

- `metadataBase`と`metadata`フィールドの間の末尾のスラッシュは正規化されます。
- `metadata`フィールドの"絶対"パス（通常はURLパス全体を置き換える）は、`metadataBase`の末尾から"相対"パスとして扱われます。

例えば、次の`metadataBase`が与えられた場合：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher
export const metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

</TabItem>
</Tabs>

上記の`metadataBase`を継承し、独自の値を設定する`metadata`フィールドは、次のように解決されます：

| `metadata` フィールド            | 解決されたURL                    |
| -------------------------------- | -------------------------------- |
| `/`                              | `https://acme.com`               |
| `./`                             | `https://acme.com`               |
| `payments`                       | `https://acme.com/payments`      |
| `/payments`                      | `https://acme.com/payments`      |
| `./payments`                     | `https://acme.com/payments`      |
| `../payments`                    | `https://acme.com/payments`      |
| `https://beta.acme.com/payments` | `https://beta.acme.com/payments` |

## サポートされていないMetadata {#unsupported-metadata}

次のmetadataタイプは、現在組み込みのサポートがありません。 ただし、レイアウトやページ自体でレンダリングすることはできます。

| Metadata                      | 推奨事項                                                                                                                                                                                                                                                          |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<meta http-equiv="...">`     | [`redirect()`](/docs/app/api-reference/functions/redirect)、[Middleware](/docs/app/building-your-application/routing/middleware#nextresponse)、[セキュリティヘッダー](/docs/app/api-reference/next-config-js/headers)を介して適切なHTTPヘッダーを使用してください |
| `<base>`                      | タグを自体レイアウトやページ内にレンダリングします。                                                                                                                                                                                                              |
| `<noscript>`                  | タグを自体レイアウトやページ内にレンダリングします。                                                                                                                                                                                                              |
| `<style>`                     | [Next.jsでのスタイリング](/docs/app/building-your-application/styling/css)の詳細を学びます。                                                                                                                                                                      |
| `<script>`                    | [スクリプトの使用](/docs/app/building-your-application/optimizing/scripts)の詳細を学びます。                                                                                                                                                                      |
| `<link rel="stylesheet" />`   | スタイルシートを自体レイアウトやページ内に`import`します。                                                                                                                                                                                                        |
| `<link rel="preload />`       | [ReactDOM preload method](#link-relpreload) を使用します                                                                                                                                                                                                          |
| `<link rel="preconnect" />`   | [ReactDOM preconnect method](#link-relpreconnect) を使用します                                                                                                                                                                                                    |
| `<link rel="dns-prefetch" />` | [ReactDOM prefetchDNS method](#link-reldns-prefetch) を使用します                                                                                                                                                                                                 |

### リソースのヒント {#resource-hints}

`<link>`要素には、ブラウザが外部リソースが必要になる可能性が高いことを示すために使用できるいくつかの`rel`キーワードがあります。 ブラウザはこの情報を使用して、キーワードに応じたプリロード最適化を適用します。

Metadata APIはこれらのヒントを直接サポートしていませんが、新しい[`ReactDOM`メソッド](https://github.com/facebook/react/pull/26237)を使用して、安全にドキュメントの`<head>`に挿入できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/preload-resources.tsx" switcher
'use client'

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/preload-resources.js" switcher

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
</Tabs>

##### `<link rel="preload">` {#link-rel-preload}

ページのレンダリング（ブラウザ）ライフサイクルでリソースを早期にロードし始める。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preload)。

```tsx
ReactDOM.preload(href: string, options: { as: string })
```

```html title="<head> output" hideLineNumbers
<link rel="preload" href="..." as="..." />
```

##### `<link rel="preconnect">` {#link-rel-preconnect}

オリジンへの接続を事前に開始します。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preconnect)。

```tsx
ReactDOM.preconnect(href: string, options?: { crossOrigin?: string })
```

```html title="<head> output" hideLineNumbers
<link rel="preconnect" href="..." crossorigin />
```

##### `<link rel="dns-prefetch">` {#link-rel-dns-prefetch}

リソースが要求される前にドメイン名を解決しようとします。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/dns-prefetch)。

```tsx
ReactDOM.prefetchDNS(href: string)
```

```html title="<head> output" hideLineNumbers
<link rel="dns-prefetch" href="..." />
```

> **Good to know**:
>
> - これらの方法は現在、クライアントコンポーネントでのみサポートされており、初期ページロードでサーバーサイドでレンダリングされています。
> - Next.jsの組み込み機能である`next/font`、`next/image`、`next/script`は、関連するリソースヒントを自動で処理します。

## タイプ {#types}

`Metadata`タイプを使ってmetadataに型の安全性を追加できます。 IDEで[組み込みTypeScriptプラグイン](/docs/app/building-your-application/configuring/typescript)を使用している場合、手動で型を追加する必要はありませんが、必要に応じて明示的に追加することが出来ます。

### `metadata`オブジェクト {#metadata-object}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Next.js',
}
```

### `generateMetadata`関数 {#generatemetadata-function}

#### 通常の関数 {#regular-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export function generateMetadata(): Metadata {
  return {
    title: 'Next.js',
  }
}
```

#### 非同期関数 {#async-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export async function generateMetadata(): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### セグメントプロップと共に {#with-segment-props}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export function generateMetadata({ params, searchParams }: Props): Metadata {
  return {
    title: 'Next.js',
  }
}

export default function Page({ params, searchParams }: Props) {}
```

#### 親Metadataと共に {#with-parent-metadata}

```tsx title="layout.tsx | page.tsx"
import type { Metadata, ResolvingMetadata } from 'next'

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### JavaScript プロジェクト {#javascript-projects}

JavaScriptプロジェクトでは、JSDocを使って型安全性を追加することが出来ます。

```js title="layout.js | page.js"
/** @type {import("next").Metadata} */
export const metadata = {
  title: 'Next.js',
}
```
