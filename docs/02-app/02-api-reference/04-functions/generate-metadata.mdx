このページでは、`generateMetadata`および静的メタデータオブジェクトを使用したすべての**設定ベースのメタデータ**オプションをカバーしています。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

// 静的メタデータ
export const metadata: Metadata = {
  title: '...',
}

// または動的メタデータ
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher
// 静的メタデータ
export const metadata = {
  title: '...',
}

// または動的メタデータ
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
</Tabs>

> **Good to know**:
>
> - `metadata`オブジェクトと`generateMetadata`関数エクスポートは**Server Componentsでのみサポートされています**。
> - 同じルートセグメントから`metadata`オブジェクトと`generateMetadata`関数の両方をエクスポートすることはできません。
> - 初回読み込み時に、`generateMetadata`が完全に解決されるまで、ストリーミングはブロックされます。これには`loading.js`からのコンテンツも含まれます。

## `metadata`オブジェクト {#the-metadata-object}

静的メタデータを定義するには、`layout.js`または`page.js`ファイルから[`Metadata`オブジェクト](#metadata-fields)をエクスポートします。

<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: '...',
  description: '...',
}

export default function Page() {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher

  title: '...',

```

</TabItem>
</Tabs>

サポートされているオプションの完全なリストは[メタデータフィールド](#metadata-fields)を参照してください。

## `generateMetadata`関数 {#generatemetadata-function}

動的メタデータは、現在のルートパラメータ、外部データ、または親セグメントの`metadata`などの**動的情報**に依存します。これは、[`Metadata`オブジェクト](#metadata-fields)を返す`generateMetadata`関数をエクスポートして設定できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[id]/page.tsx" switcher
import type { Metadata, ResolvingMetadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  // ルートパラメータを読み取る
  const id = (await params).id

  // データをフェッチする
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // オプションで親メタデータにアクセスして拡張する（置換せずに）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }: Props) {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[id]/page.js" switcher
export async function generateMetadata({ params, searchParams }, parent) {
  // ルートパラメータを読み取る
  const id = (await params).id

  // データをフェッチする
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // オプションで親メタデータにアクセスして拡張する（置換せずに）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }) {}
```

</TabItem>
</Tabs>

### パラメーター {#parameters}

`generateMetadata`関数は次のパラメーターを受け取ります：

- `props` - 現在のルートのパラメーターを含むオブジェクト：

  - `params` - [dynamic route parameters](/docs/app/building-your-application/routing/dynamic-routes)オブジェクトを含むオブジェクトで、ルートセグメントの起点から`generateMetadata`が呼び出されるセグメントまでを示します。例：

    | ルート                          | URL         | `params`                  |
    | ------------------------------- | ----------- | ------------------------- |
    | `app/shop/[slug]/page.js`       | `/shop/1`   | `{ slug: '1' }`           |
    | `app/shop/[tag]/[item]/page.js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |
    | `app/shop/[...slug]/page.js`    | `/shop/1/2` | `{ slug: ['1', '2'] }`    |

  - `searchParams` - 現在のURLの[search params](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)を含むオブジェクト。例：

    | URL             | `searchParams`       |
    | --------------- | -------------------- |
    | `/shop?a=1`     | `{ a: '1' }`         |
    | `/shop?a=1&b=2` | `{ a: '1', b: '2' }` |
    | `/shop?a=1&a=2` | `{ a: ['1', '2'] }`  |

- `parent` - 親ルートセグメントからの解決済みメタデータのプロミス。

### 戻り値 {#returns}

`generateMetadata`は、1つ以上のメタデータフィールドを含む[`Metadata`オブジェクト](#metadata-fields)を返す必要があります。

> **Good to know**:
>
> - メタデータがランタイム情報に依存しない場合、`generateMetadata`ではなく静的な[`metadata`オブジェクト](#the-metadata-object)を使用して定義する必要があります。
> - `fetch`リクエストは、`generateMetadata`、`generateStaticParams`、Layouts、Pages、およびServer Components全体で同じデータに対して自動的に[メモ化されます](/docs/app/building-your-application/caching#request-memoization)。`fetch`が利用できない場合は、Reactの[`cache`が使用できます](/docs/app/building-your-application/caching#react-cache-function)。
> - `searchParams`は、`page.js`セグメントでのみ使用可能です。
> - Next.jsの[`redirect()`](/docs/app/api-reference/functions/redirect)および[`notFound()`](/docs/app/api-reference/functions/not-found)メソッドは、`generateMetadata`内でも使用できます。

## メタデータフィールド {#metadata-fields}

### `title` {#title}

`title`属性はドキュメントのタイトルを設定するために使用されます。シンプルな[文字列](#string)やオプションの[テンプレートオブジェクト](#template-object)として定義できます。

#### 文字列 {#string}

```jsx title="layout.js | page.js"
export const metadata = {
  title: 'Next.js',
}
```

```html title="<head>出力" hideLineNumbers

```

#### テンプレートオブジェクト {#template-object}

```tsx title="app/layout.tsx" switcher
export const metadata: Metadata = {
  title: {
    template: '...',
    default: '...',
    absolute: '...',
  },
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher

  title: {
    default: '...',
    template: '...',
    absolute: '...',
  },
}
```

</TabItem>
</Tabs>

##### デフォルト {#default}

`title.default`は`title`を定義しない子ルートセグメントに対する**フォールバックタイトル**を提供するために使用できます。

```tsx title="app/layout.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    default: 'Acme',
  },
}
```

```tsx title="app/about/page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {}

// 出力: <title>Acme</title>
```

##### テンプレート {#template}

`title.template`は、**子**ルートセグメントで定義された`titles`に接頭辞や接尾辞を追加するために使用できます。

</TabItem>
<TabItem value="jsx" label="JavaScript">

```tsx title="app/about/page.tsx" switcher
  title: 'About',
```

> **Good to know**:
>
> - `title.template`は、定義されたセグメントではなく、**子**ルートセグメントに適用されます。これを意味します：
>
>   - `title.template`を追加するときに`title.default`が**必須**です。
>   - `layout.js`に定義された`title.template`は、同じルートセグメントの`page.js`に定義された`title`には適用されません。
>   - `page.js`に定義された`title.template`は効果がありません。ページは常に終了セグメントであり、子ルートセグメントを持たないためです。
>
> - ルートが`title`や`title.default`を定義していない場合、`title.template`は**効果がありません**。

> **Good to know**:
>
> - `layout.js`
>
>   - `title` (文字列)と`title.default`は、子セグメントのデフォルトのタイトルを定義します（独自の`title`を定義していない場合）。存在する場合は最も近い親セグメントの`title.template`を強化します。
>   - `title.absolute`は、子セグメントのデフォルトのタイトルを定義します。親セグメントの`title.template`を無視します。
>   - `title.template`は、子セグメントの新しいタイトルテンプレートを定義します。
>
> - `page.js`
>   - ページが独自のタイトルを定義していない場合、最も近い親の解決済みタイトルが使用されます。
>   - `title` (文字列)は、ルートのタイトルを定義します。存在する場合は最も近い親セグメントの`title.template`を強化します。
>   - `title.absolute`は、ルートのタイトルを定義します。親セグメントの`title.template`を無視します。
>   - `page.js`内の`title.template`は効果がありません。ページはルートの終了セグメントであるためです。

### `metadataBase` {#metadatabase}

`metadataBase`は、完全修飾URLを必要とする`metadata`フィールドのための基底URLプレフィックスを設定するための利便性のあるオプションです。

- `metadataBase`は、**現在のルートセグメントおよびその下**に定義されたURLベースの`metadata`フィールドが、絶対 URL の代わりに**相対パス**を使用できるようにします。
- このフィールドの相対パスは`metadataBase`と合成され、完全修飾URLが形成されます。
- 設定されていない場合、`metadataBase`は[デフォルト値](#default-value)で自動的に入力されます。

```jsx title="layout.js | page.js"
export const metadata = {
  metadataBase: new URL('https://acme.com'),
  alternates: {
    canonical: '/',
    languages: {
      'en-US': '/en-US',
      'de-DE': '/de-DE',
    },
  },
  openGraph: {
    images: '/og-image.png',
  },
}
```

```html title="<head>出力" hideLineNumbers
<link rel="canonical" href="https://acme.com" />
<link rel="alternate" hreflang="en-US" href="https://acme.com/en-US" />
<link rel="alternate" hreflang="de-DE" href="https://acme.com/de-DE" />
<meta property="og:image" content="https://acme.com/og-image.png" />
```

> **Good to know**:
>
> - `metadataBase`は通常、ルートの`app/layout.js`に設定され、すべてのルートで URL ベースの `metadata` フィールドに適用されます。
> - 絶対 URL を必要とするすべての URL ベースの `metadata` フィールドは、`metadataBase` オプションで設定できます。
> - `metadataBase`はサブドメインを含むことができます（例: `https://app.acme.com`）またはベースパス（例: `https://acme.com/start/from/here`）
> - `metadata`フィールドが絶対 URL を提供する場合、`metadataBase`は無視されます。
> - `metadataBase`を設定せずに URL ベースの `metadata` フィールドに相対パスを使用すると、ビルド エラーが発生します。
> - Next.js は、`metadataBase` (`https://acme.com/`) とフィールドを組み合わせた際に重複するスラッシュを 1 つのスラッシュ (`https://acme.com/path`) に正規化します。

#### デフォルト値 {#default-value}

設定されていない場合、`metadataBase`は**デフォルト値**を持ちます。

> Vercel上：
>
> - 本番デプロイの場合は、`VERCEL_PROJECT_PRODUCTION_URL`が使用されます。
> - プレビュー デプロイの場合は、`VERCEL_BRANCH_URL`が優先され、存在しない場合は`VERCEL_URL`にフォールバックします。
>
> これらの値が存在する場合、それらは`metadataBase`の**デフォルト値**として使用されます。存在しない場合は`http://localhost:${process.env.PORT || 3000}`にフォールバックします。これにより、Open Graph 画像はローカル ビルドと Vercel のプレビューおよび本番デプロイの両方で機能します。デフォルトをオーバーライドする場合は、環境変数を使用してURLを計算することをお勧めします。これにより、ローカル開発、ステージング、および本番環境用のURLを設定できます。
>
> これらの環境変数の詳細については、[システム環境変数](https://vercel.com/docs/concepts/projects/environment-variables/system-environment-variables)のドキュメントを参照してください。

#### URL 合成 {#url-composition}

URL 合成は、デフォルトのディレクトリ トラバーサル セマンティクスよりも開発者の意図を優先します。

- `metadataBase`と`metadata`フィールドの間の末尾のスラッシュは正規化されます。
- `metadata`フィールド内の「絶対」パス（通常はURLパス全体を置き換える）は、「相対」パスとして扱われます（`metadataBase`のエンドから開始）。

例：以下の`metadataBase`が指定されている場合：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher
export const metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

</TabItem>
</Tabs>

以上の`metadataBase`を継承し、それ自体の値を設定する`metadata`フィールドは次のように解決されます：

| `metadata`フィールド             | 解決されたURL                    |
| -------------------------------- | -------------------------------- |
| `/`                              | `https://acme.com`               |
| `./`                             | `https://acme.com`               |
| `payments`                       | `https://acme.com/payments`      |
| `/payments`                      | `https://acme.com/payments`      |
| `./payments`                     | `https://acme.com/payments`      |
| `../payments`                    | `https://acme.com/payments`      |
| `https://beta.acme.com/payments` | `https://beta.acme.com/payments` |

## サポートされていないメタデータ {#unsupported-metadata}

次のメタデータタイプは現在組み込みのサポートがありません。ただし、レイアウトやページ自体でレンダリングすることは可能です。

| メタデータ                    | 推奨事項                                                                                                                                                                                                                                                      |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<meta http-equiv="...">`     | [`redirect()`](/docs/app/api-reference/functions/redirect)、[Middleware](/docs/app/building-your-application/routing/middleware#nextresponse)、[セキュリティヘッダー](/docs/app/api-reference/next-config-js/headers)を通じて適切なHTTPヘッダーを使用します。 |
| `<base>`                      | レイアウトやページ自体でタグをレンダリングします。                                                                                                                                                                                                            |
| `<noscript>`                  | レイアウトやページ自体でタグをレンダリングします。                                                                                                                                                                                                            |
| `<style>`                     | [Next.jsでのスタイリング](https://developer.x.com/en/docs/twitter-for-websites/cards/overview/markup)の詳細を学びます。                                                                                                                                       |
| `<script>`                    | [スクリプトの使用](https://developer.x.com/en/docs/twitter-for-websites/cards/overview/markup)の詳細を学びます。                                                                                                                                              |
| `<link rel="stylesheet" />`   | スタイルシートをレイアウトやページ自体で`import`します。                                                                                                                                                                                                      |
| `<link rel="preload />`       | [ReactDOM preload メソッド](#link-relpreload)を使用します。                                                                                                                                                                                                   |
| `<link rel="preconnect" />`   | [ReactDOM preconnect メソッド](#link-relpreconnect)を使用します。                                                                                                                                                                                             |
| `<link rel="dns-prefetch" />` | [ReactDOM prefetchDNS メソッド](#link-reldns-prefetch)を使用します。                                                                                                                                                                                          |

### リソースヒント {#resource-hints}

`<link>`要素には、ブラウザに外部リソースが必要になる可能性があることを示すために使用できるいくつかの`rel`キーワードがあります。ブラウザはこの情報を使用して、キーワードに応じたプリロードの最適化を適用します。

メタデータAPIはこれらのヒントを直接サポートしていませんが、新しい[`ReactDOM`メソッド](https://github.com/facebook/react/pull/26237)を使用して、ドキュメントの`<head>`に安全に挿入できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/preload-resources.tsx" switcher
'use client'

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/preload-resources.js" switcher

  return '...'
}
```

</TabItem>
</Tabs>

##### `<link rel="preload">` {#link-rel-preload}

ページレンダリング（ブラウザ）ライフサイクルの早い段階でリソースの読み込みを開始します。[MDNドキュメント](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preload)。

```tsx
ReactDOM.preload(href: string, options: { as: string })
```

```html title="<head>出力" hideLineNumbers
<link rel="preload" href="..." as="..." />
```

##### `<link rel="preconnect">` {#link-rel-preconnect}

オリジンへの接続を事前に開始します。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preconnect)。

```tsx
ReactDOM.preconnect(href: string, options?: { crossOrigin?: string })
```

```html title="<head>出力" hideLineNumbers
<link rel="preconnect" href="..." crossorigin />
```

##### `<link rel="dns-prefetch">` {#link-rel-dns-prefetch}

リソースが要求される前にドメイン名を解決しようとします。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/dns-prefetch)。

```tsx
ReactDOM.prefetchDNS(href: string)
```

```html title="<head>出力" hideLineNumbers
<link rel="dns-prefetch" href="..." />
```

> **Good to know**:
>
> - これらのメソッドは現在、クライアントコンポーネントでのみサポートされていますが、初期ページ読み込み時にはサーバーサイドレンダリングされた状態です。
> - Next.jsの組み込み機能である`next/font`、`next/image`、`next/script`は、関連するリソースヒントを自動的に処理します。

## Types {#types}

`Metadata`型を使用してメタデータに型安全性を追加できます。IDEの[組み込みTypeScriptプラグイン](/docs/app/building-your-application/configuring/typescript)を使用している場合は、タイプを手動で追加する必要はありませんが、それでも必要に応じて明示的に追加できます。

### `metadata`オブジェクト {#metadata-object}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Next.js',
}
```

### `generateMetadata`関数 {#generatemetadata-function}

#### 通常の関数 {#regular-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export function generateMetadata(): Metadata {
  return {
    title: 'Next.js',
  }
}
```

#### 非同期関数 {#async-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export async function generateMetadata(): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### セグメントのpropsとの使用 {#with-segment-props}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export function generateMetadata({ params, searchParams }: Props): Metadata {
  return {
    title: 'Next.js',
  }
}

export default function Page({ params, searchParams }: Props) {}
```

#### 親メタデータとの使用 {#with-parent-metadata}

```tsx title="layout.tsx | page.tsx"
import type { Metadata, ResolvingMetadata } from 'next'

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### JavaScriptプロジェクト {#javascript-projects}

JavaScriptプロジェクトでは、JSDocを使用して型安全性を追加できます。

```js title="layout.js | page.js"
/** @type {import("next").Metadata} */

  title: 'Next.js',
```
