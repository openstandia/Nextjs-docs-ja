このページでは、`generateMetadata` と静的メタデータオブジェクトを使用したすべての **Config-based Metadata** オプションについて説明します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

// 静的メタデータ
export const metadata: Metadata = {
  title: '...',
}

// または動的メタデータ
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher
// 静的メタデータ
export const metadata = {
  title: '...',
}

// または動的メタデータ
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
</Tabs>

> **Good to know**:
>
> - `metadata` オブジェクトと `generateMetadata` 関数エクスポートは**Server Componentsでのみサポートされています**。
> - 同じルートセグメントから `metadata` オブジェクトと `generateMetadata` 関数の両方をエクスポートすることはできません。
> - 初回読み込み時には、`generateMetadata` が完全に解決されるまでストリーミングはブロックされ、`loading.js` のコンテンツを含みます。

## `metadata`オブジェクト {#the-metadata-object}

静的メタデータを定義するには、`layout.js` か `page.js` ファイルから [`Metadata` オブジェクト](#metadata-fields) をエクスポートします。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: '...',
  description: '...',
}

export default function Page() {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher

  title: '...',
  description: '...',
}

export default function Page() {}
```

</TabItem>
</Tabs>

サポートされているオプションの完全なリストは、[Metadata Fields](#metadata-fields) を参照してください。

## `generateMetadata`関数 {#generatemetadata-function}

動的メタデータは、現在のルートパラメータ、外部データ、親セグメントの`metadata` など**動的情報**に依存しています。`Metadata` オブジェクト](#metadata-fields) を返す `generateMetadata` 関数をエクスポートすることで、設定できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[id]/page.tsx" switcher
import type { Metadata, ResolvingMetadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  // ルートパラメータを読み込みます
  const id = (await params).id

  // データを取得する
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // 任意で親メタデータにアクセスして拡張します（置き換えるのではなく）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }: Props) {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[id]/page.js" switcher
export async function generateMetadata({ params, searchParams }, parent) {
  // ルートパラメータを読み込みます
  const id = (await params).id

  // データを取得する
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // 任意で親メタデータにアクセスして拡張します（置き換えるのではなく）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }) {}
```

</TabItem>
</Tabs>

### パラメータ {#parameters}

`generateMetadata`関数は次のパラメータを受け取ります：

- `props` - 現在のルートのパラメータを含むオブジェクト：

  - `params` - [dynamic route parameters](/docs/app/building-your-application/routing/dynamic-routes)オブジェクトをrootセグメントから`generateMetadata`が呼び出されるセグメントまで含むオブジェクトです。例：

    | Route                           | URL         | `params`                  |
    | ------------------------------- | ----------- | ------------------------- |
    | `app/shop/[slug]/page.js`       | `/shop/1`   | `{ slug: '1' }`           |
    | `app/shop/[tag]/[item]/page.js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |
    | `app/shop/[...slug]/page.js`    | `/shop/1/2` | `{ slug: ['1', '2'] }`    |

  - `searchParams` - 現在のURLの[search params](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)を含むオブジェクトです。例：

    | URL             | `searchParams`       |
    | --------------- | -------------------- |
    | `/shop?a=1`     | `{ a: '1' }`         |
    | `/shop?a=1&b=2` | `{ a: '1', b: '2' }` |
    | `/shop?a=1&a=2` | `{ a: ['1', '2'] }`  |

- `parent` - 親ルートセグメントから解決されたメタデータのプロミス。

### 戻り値 {#returns}

`generateMetadata` は、1つ以上のメタデータフィールドを含む[`Metadata` オブジェクト](#metadata-fields)を返す必要があります。

> **Good to know**:
>
> - メタデータがランタイム情報に依存しない場合は、`generateMetadata` ではなく静的な[`metadata` オブジェクト](#the-metadata-object)を使用して定義する必要があります。
> - `fetch`リクエストは`generateMetadata`、`generateStaticParams`、レイアウト、ページ、Server Components全体で同じデータに対して自動的に[メモ化](/docs/app/building-your-application/caching#request-memoization)されます。`fetch` が利用できない場合は、React [`cache` を使用できます](/docs/app/building-your-application/caching#react-cache-function)。
> - `searchParams` は `page.js` セグメントでのみ利用可能です。
> - Next.jsメソッドの[`redirect()`](/docs/app/api-reference/functions/redirect)および[`notFound()`](/docs/app/api-reference/functions/not-found)も`generateMetadata`内で使用できます。

## Metadata Fields {#metadata-fields}

### `title` {#title}

`title`属性はドキュメントのタイトルを設定するために使用されます。シンプルな[string](#string)またはオプションの[templateオブジェクト](#template-object)として定義できます。

#### 文字列 {#string}

```jsx title="layout.js | page.js"
export const metadata = {
  title: 'Next.js',
}
```

```html title="<head> output" hideLineNumbers
<title>Next.js</title>
```

#### テンプレートオブジェクト {#template-object}

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    template: '...',
    default: '...',
    absolute: '...',
  },
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher
export const metadata = {
  title: {
    default: '...',
    template: '...',
    absolute: '...',
  },
}
```

</TabItem>
</Tabs>

##### デフォルト {#default}

`title.default`は、`title`を定義しない子ルートセグメントに**フォールバックタイトル**を提供するために使用できます。

```tsx title="app/layout.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    default: 'Acme',
  },
}
```

```tsx title="app/about/page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {}

// 出力: <title>Acme</title>
```

##### テンプレート {#template}

`title.template`は、**子**ルートセグメントで定義された`titles`にプレフィックスまたはサフィックスを追加するために使用できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    template: '%s | Acme',
    default: 'Acme', // テンプレート作成時にはデフォルトが必要です
  },
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher
export const metadata = {
  title: {
    template: '%s | Acme',
    default: 'Acme', // テンプレート作成時にはデフォルトが必要です
  },
}
```

</TabItem>
</Tabs>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/about/page.tsx" switcher
import type { Metadata } from 'next'

  title: 'About',
}

// 出力: <title>About | Acme</title>
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/about/page.js" switcher
export const metadata = {
  title: 'About',
}

// 出力: <title>About | Acme</title>
```

</TabItem>
</Tabs>

> **Good to know**:
>
> - `title.template`は**子**ルートセグメントに適用され、それが定義されているセグメントには**適用されません**。これは次のことを意味します：
>
>   - `title.default`は、`title.template`を追加した場合、**必要**です。
>   - `layout.js`に定義された`title.template`は、同じルートセグメントの`page.js`に定義された`title`には適用されません。
>   - `page.js`に定義された`title.template`は影響を与えません。なぜなら、ページは常に終端セグメントであり（子ルートセグメントを持たないため）、自分自身のタイトルを定義しない場合は最も近い親の解決済みタイトルが使用されます。
>
> - ルートが`title`や`title.default`を定義していない場合、`title.template`は**影響を与えません**。

`title.absolute`は、親セグメントに設定された`title.template`を**無視**するタイトルを提供するために使用できます。

> **Good to know**:
>
> - `layout.js`
>
>   - `title`（文字列）および`title.default`は子セグメントのデフォルトのタイトルを定義します（自分自身の`title`を定義していない場合）。最も近い親セグメントからの`title.template`を拡張します（存在する場合）。
>   - `title.absolute`は子セグメントのデフォルトのタイトルを定義します。親セグメントからの`title.template`を無視します。
>   - `title.template`は子セグメントの新しいタイトルテンプレートを定義します。
>
> - `page.js`
>   - ページが自分自身のタイトルを定義していない場合、最も近い親の解決済みタイトルが使用されます。
>   - `title`（文字列）はルートのタイトルを定義します。最も近い親セグメントからの`title.template`を拡張します（存在する場合）。
>   - `title.absolute`はルートのタイトルを定義します。親セグメントからの`title.template`を無視します。
>   - ページは常にルートの終端セグメントであるため、`title.template`は`page.js`では影響を与えません。

### `metadataBase` {#metadatabase}

`metadataBase`は、完全に修飾されたURLを必要とする`metadata`フィールドの基本URLプレフィックスを設定するための便利なオプションです。

- `metadataBase`を使うと、**現在のルートセグメントおよびその下**で定義されたURLベースの`metadata`フィールドが、他に必要な絶対URLの代わりに**相対パス**を使用できます。
- フィールドの相対パスは、`metadataBase`と組み合わされて完全なURLを形成します。
- 設定されていない場合、`metadataBase`は[デフォルト値](#default-value)で自動的に設定されます。

```jsx title="layout.js | page.js"
export const metadata = {
  metadataBase: new URL('https://acme.com'),
  alternates: {
    canonical: '/',
    languages: {
      'en-US': '/en-US',
      'de-DE': '/de-DE',
    },
  },
  openGraph: {
    images: '/og-image.png',
  },
}
```

```html title="<head> output" hideLineNumbers
<link rel="canonical" href="https://acme.com" />
<link rel="alternate" hreflang="en-US" href="https://acme.com/en-US" />
<link rel="alternate" hreflang="de-DE" href="https://acme.com/de-DE" />
<meta property="og:image" content="https://acme.com/og-image.png" />
```

> **Good to know**:
>
> - `metadataBase`は通常、root `app/layout.js`で設定され、すべてのルートのURLベースの`metadata`フィールドに適用されます。
> - 絶対URLを必要とするすべてのURLベースの`metadata`フィールドは、`metadataBase`オプションで設定できます。
> - `metadataBase`には、サブドメイン（例：`https://app.acme.com`）や基本パス（例：`https://acme.com/start/from/here`）を含めることができます。
> - `metadata`フィールドが絶対URLを提供する場合、`metadataBase`は無視されます。
> - URLベースの`metadata`フィールドで相対パスを使用し、`metadataBase`を設定しなかった場合、ビルドエラーが発生します。
> - Next.jsは、`metadataBase`（例：`https://acme.com/`）と相対フィールド（例：`/path`）の間の重複スラッシュを通常のスラッシュに正規化します（例：`https://acme.com/path`）。

#### デフォルト値 {#default-value}

設定されていない場合、`metadataBase`には**デフォルト値**があります。

> Vercelの場合：
>
> - 本番環境へのデプロイでは、`VERCEL_PROJECT_PRODUCTION_URL` が使用されます。
> - プレビュー デプロイでは、`VERCEL_BRANCH_URL` が優先され、それが存在しない場合は`VERCEL_URL` にフォールバックします。
>
> これらの値が存在する場合、それらは`metadataBase`の**デフォルト値**として使用されます。さもなければ、`http://localhost:${process.env.PORT || 3000}`にフォールバックします。これにより、ローカル ビルドおよび Vercel のプレビューや本番デプロイメントで Open Graph 画像が機能するようになります。デフォルトをオーバーライドする場合、環境変数を使用して URL を計算することをお勧めします。これにより、ローカル開発、ステージング、本番環境の URL を設定できます。
>
> これらの環境変数の詳細については、[システム環境変数](https://vercel.com/docs/concepts/projects/environment-variables/system-environment-variables) ドキュメントをご覧ください。

#### URL組み合わせ {#url-composition}

URLの組み合わせは、デフォルトのディレクトリ トラバーサルセマンティクスよりも、開発者の意図を優先します。

- `metadataBase`と`metadata`フィールドの間の後続スラッシュが正規化されます。
- `metadata` フィールド内の「絶対」パス（通常は URL パス全体を置き換える）は、「相対」パス（`metadataBase`の末尾から開始）として扱われます。

次の `metadataBase` が与えられたとします：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher
export const metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

</TabItem>
</Tabs>

上記の`metadataBase`を継承し、自分で値を設定する`metadata` フィールドは次のように解決されます：

| `metadata` フィールド            | 解決済みURL                      |
| -------------------------------- | -------------------------------- |
| `/`                              | `https://acme.com`               |
| `./`                             | `https://acme.com`               |
| `payments`                       | `https://acme.com/payments`      |
| `/payments`                      | `https://acme.com/payments`      |
| `./payments`                     | `https://acme.com/payments`      |
| `../payments`                    | `https://acme.com/payments`      |
| `https://beta.acme.com/payments` | `https://beta.acme.com/payments` |

## サポートされていないメタデータ {#unsupported-metadata}

次に示すメタデータ タイプは、現在は組み込みサポートがありません。ただし、それでもレイアウトやページ自体にレンダリングすることができます。

| メタデータ                    | 推奨事項                                                                                                                                                                                                                                                          |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<meta http-equiv="...">`     | [`redirect()`](/docs/app/api-reference/functions/redirect)、[ミドルウェア](/docs/app/building-your-application/routing/middleware#nextresponse)、[セキュリティヘッダー](/docs/app/api-reference/next-config-js/headers) を介して適切なHTTP ヘッダーを使用します。 |
| `<base>`                      | レイアウトやページ自体でタグをレンダリングします。                                                                                                                                                                                                                |
| `<noscript>`                  | レイアウトやページ自体でタグをレンダリングします。                                                                                                                                                                                                                |
| `<style>`                     | [Next.jsでのスタイリング](/docs/app/building-your-application/styling/css)について学びます。                                                                                                                                                                      |
| `<script>`                    | [スクリプトの使用](/docs/app/building-your-application/optimizing/scripts)について学びます。                                                                                                                                                                      |
| `<link rel="stylesheet" />`   | レイアウトやページ自体でスタイルシートを`import`します。                                                                                                                                                                                                          |
| `<link rel="preload />`       | [ReactDOM preload メソッド](#link-relpreload)を使用します。                                                                                                                                                                                                       |
| `<link rel="preconnect" />`   | [ReactDOM preconnect メソッド](#link-relpreconnect)を使用します。                                                                                                                                                                                                 |
| `<link rel="dns-prefetch" />` | [ReactDOM prefetchDNS メソッド](#link-reldns-prefetch)を使用します。                                                                                                                                                                                              |

### リソースヒント {#resource-hints}

`<link>` 要素には、外部リソースが必要とされる可能性があることをブラウザにヒントするために使用できる多数の `rel` キーワードがあります。ブラウザは、この情報を使用して、キーワードに応じてプリロード 最適化を適用します。

メタデータAPIはこれらのヒントを直接サポートしていませんが、新しい[`ReactDOM` メソッド](https://github.com/facebook/react/pull/26237)を使用して、これらをドキュメントの`<head>` に安全に挿入することができます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/preload-resources.tsx" switcher
'use client'

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/preload-resources.js" switcher

  ReactDOM.preload('...', { as: '...' })

  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
</Tabs>

##### `<link rel="preload">` {#link-rel-preload}

ページレンダリング（ブラウザー）ライフサイクルの早い段階でリソースの読み込みを開始します。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preload)。

```tsx
ReactDOM.preload(href: string, options: { as: string })
```

```html title="<head> output" hideLineNumbers
<link rel="preload" href="..." as="..." />
```

##### `<link rel="preconnect">` {#link-rel-preconnect}

オリジンへの接続を事前に開始します。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preconnect)。

```tsx
ReactDOM.preconnect(href: string, options?: { crossOrigin?: string })
```

```html title="<head> output" hideLineNumbers
<link rel="preconnect" href="..." crossorigin />
```

##### `<link rel="dns-prefetch">` {#link-rel-dns-prefetch}

リソースがリクエストされる前にドメイン名を解決しようとします。[MDN Docs](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/dns-prefetch)。

```tsx
ReactDOM.prefetchDNS(href: string)
```

```html title="<head> output" hideLineNumbers
<link rel="dns-prefetch" href="..." />
```

> **Good to know**:
>
> - これらのメソッドは現在、クライアントコンポーネントでのみサポートされていますが、初期ページロード時には依然としてサーバーサイドレンダリングされます。
> - `next/font`、`next/image` および `next/script` などの Next.js 組み込み機能は、関連するリソース ヒントを自動的に処理します。

## 型 {#types}

コンマetadataに`Metadata` 型を使用して型の安全性を追加できます。IDEで組み込まれている[TypeScript プラグイン](/docs/app/building-your-application/configuring/typescript) を使用している場合、手動で型を追加する必要はありませんが、望む場合は明示的に追加することもできます。

### `metadata` オブジェクト {#metadata-object}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Next.js',
}
```

### `generateMetadata` 関数 {#generatemetadata-function}

#### 通常の関数 {#regular-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export function generateMetadata(): Metadata {
  return {
    title: 'Next.js',
  }
}
```

#### 非同期関数 {#async-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export async function generateMetadata(): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### セグメントpropあり {#with-segment-props}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export function generateMetadata({ params, searchParams }: Props): Metadata {
  return {
    title: 'Next.js',
  }
}

export default function Page({ params, searchParams }: Props) {}
```

#### 親のメタデータあり {#with-parent-metadata}

```tsx title="layout.tsx | page.tsx"
import type { Metadata, ResolvingMetadata } from 'next'

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### JavaScript プロジェクト {#javascript-projects}

JavaScript プロジェクトでは、JSDoc を使用して型の安全性を追加できます。

```js title="layout.js | page.js"
/** @type {import("next").Metadata} */
export const metadata = {
  title: 'Next.js',
}
```
