このページでは、`generateMetadata`と静的メタデータオブジェクトを使用したすべての**Config-based Metadata**オプションについて説明します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

// 静的なメタデータ
export const metadata: Metadata = {
  title: '...',
}

// 動的なメタデータ
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher
// 静的なメタデータ
export const metadata = {
  title: '...',
}

// 動的なメタデータ
export async function generateMetadata({ params }) {
  return {
    title: '...',
  }
}
```

</TabItem>
</Tabs>

> **Good to know**:
>
> - `metadata`オブジェクトと`generateMetadata`関数のエクスポートは、**Server Componentsでのみサポートされます**。
> - 同じroute segmentから`metadata`オブジェクトと`generateMetadata`関数の両方をエクスポートすることはできません。
> - 初回ロード時、`loading.js`からのコンテンツを含む`generateMetadata`が完全に解決するまではストリーミングがブロックされます。

## `metadata`オブジェクト {#the-metadata-object}

静的メタデータを定義するには、`layout.js`または`page.js`ファイルから[`Metadata`オブジェクト](#metadata-fields)をエクスポートします。

<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
export const metadata: Metadata = {
  title: '...',
  description: '...',
}

```

</TabItem>
<TabItem value="jsx" label="JavaScript">


</TabItem>
</Tabs>

サポートされているオプションの完全なリストは、[Metadata Fields](#metadata-fields)をご覧ください。

## `generateMetadata`関数 {#generatemetadata-function}

動的メタデータは、現在のルートパラメータや外部データ、親セグメントの`metadata`など、**動的情報**に依存しています。`Metadata`オブジェクトを返す`generateMetadata`関数をエクスポートすることで設定できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/products/[id]/page.tsx" switcher
import type { Metadata, ResolvingMetadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  // ルートパラメータを読み込みます
  const id = (await params).id

  // データを取得します
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // 親メタデータをアクセスして拡張する（置換するのではなく）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }: Props) {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/products/[id]/page.js" switcher
export async function generateMetadata({ params, searchParams }, parent) {
  // ルートパラメータを読み込みます
  const id = (await params).id

  // データを取得します
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // 親メタデータをアクセスして拡張する（置換するのではなく）
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }) {}
```

</TabItem>
</Tabs>

### パラメータ {#parameters}

`generateMetadata`関数は、次のパラメータを受け取ります：

- `props` - 現在のルートのパラメータを含むオブジェクト：

  - `params` - [動的ルートパラメータ](/docs/app/building-your-application/routing/dynamic-routes)オブジェクトをrootセグメントから`generateMetadata`が呼び出されるセグメントに含むオブジェクト。例：

    | ルート                          | URL         | `params`                  |
    | ------------------------------- | ----------- | ------------------------- |
    | `app/shop/[slug]/page.js`       | `/shop/1`   | `{ slug: '1' }`           |
    | `app/shop/[tag]/[item]/page.js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |
    | `app/shop/[...slug]/page.js`    | `/shop/1/2` | `{ slug: ['1', '2'] }`    |

  - `searchParams` - 現在のURLの[検索パラメータ](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)を含むオブジェクト。例：

    | URL             | `searchParams`       |
    | --------------- | -------------------- |
    | `/shop?a=1`     | `{ a: '1' }`         |
    | `/shop?a=1&b=2` | `{ a: '1', b: '2' }` |
    | `/shop?a=1&a=2` | `{ a: ['1', '2'] }`  |

- `parent` - 親ルートセグメントから解決されたメタデータのプロミス。

### 戻り値 {#returns}

`generateMetadata`は、1つ以上のメタデータフィールドを含む[`Metadata`オブジェクト](#metadata-fields)を返す必要があります。

> **Good to know**:
>
> - メタデータがランタイム情報に依存していない場合、静的[`metadata`オブジェクト](#the-metadata-object)を使って定義すべきです。
> - `fetch`リクエストは`generateMetadata`、`generateStaticParams`、Layouts、Pages、およびServer Components全体で同じデータのために自動的に[メモ化されます](/docs/app/building-your-application/caching#request-memoization)。`fetch`が利用できない場合、React [`cache`を使用できます](/docs/app/building-your-application/caching#react-cache-function)。
> - `searchParams`は`page.js`セグメントでのみ使用可能です。
> - Next.jsメソッドである[`redirect()`](/docs/app/api-reference/functions/redirect)と[`notFound()`](/docs/app/api-reference/functions/not-found)も`generateMetadata`の中で使用できます。

## Metadata Fields {#metadata-fields}

### `title` {#title}

`title`属性は、ドキュメントのタイトルを設定するために使用されます。シンプルな[文字列](#string)またはオプションの[テンプレートオブジェクト](#template-object)として定義できます。

#### 文字列 {#string}

```jsx title="layout.js | page.js"

  title: 'Next.js',
}
```



</TabItem>
<TabItem value="jsx" label="JavaScript">

</TabItem>

##### デフォルト {#default}

`title.default`は、`title`を定義していない子ルートセグメントに**フォールバックタイトル**を提供するために使用されます。

```tsx title="app/layout.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    default: 'Acme',
  },
}
```

```tsx title="app/about/page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {}

// 出力: <title>Acme</title>
```

##### テンプレート {#template}

`title.template`は、**子**ルートセグメントで定義された`titles`にプレフィックスまたはサフィックスを追加するために使用できます。

```jsx title="app/layout.js" switcher

    template: '%s | Acme',
    default: 'Acme', // テンプレート作成時にはデフォルトが必要
```

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/about/page.tsx" switcher

```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/about/page.js" switcher
  title: 'About',
```

> **Good to know**:
>
> - `title.template`は、**子**ルートセグメントに適用され、**設定されているセグメントには適用されません**。これは次を意味します：
>
>   - `title.default`は、`title.template`を追加するときに**必須**です。
>   - `layout.js`で定義された`title.template`は、同じルートセグメントの`page.js`で定義された`title`には適用されません。
>   - `title.template`は常に終了セグメントであるため、`page.js`で定義しても効果はありません（子ルートセグメントは持ちません）。
>
> - ルートに`title`や`title.default`が定義されていない場合、`title.template`は**効果がありません**。

> **Good to know**:
>
> - `layout.js`
>
>   - `title`（文字列）と`title.default`は、子セグメント（独自の`title`を定義していないもの）のデフォルトタイトルを定義します。存在する場合、最も近い親セグメントの`title.template`を強化します。
>   - `title.absolute`は、子セグメントのデフォルトタイトルを定義します。親セグメントの`title.template`を無視します。
>   - `title.template`は子セグメントの新しいタイトルテンプレートを定義します。
>
> - `page.js`
>   - ページが自分のタイトルを定義していない場合、最も近い親の解決済みタイトルが使用されます。
>   - `title`（文字列）はルートのタイトルを定義します。存在する場合、最も近い親セグメントの`title.template`を強化します。
>   - `title.absolute`はルートのタイトルを定義します。親セグメントの`title.template`を無視します。
>   - `page.js`で`title.template`は効果がありません。ページは常にルートの終了セグメントだからです。

### `metadataBase` {#metadatabase}

`metadataBase`は、完全修飾URLが必要な`metadata`フィールドのための基本URLプレフィックスを設定する便利なオプションです。

- `metadataBase`により、**現在のルートセグメントとその下**で定義されたURLベースの`metadata`フィールドが、それ以外では絶対URLが必要な場合に**相対パス**を使用できるようになります。
- フィールドの相対パスは、`metadataBase`と組み合わされて完全修飾URLを形成します。
- 設定されていない場合、`metadataBase`は[デフォルト値](#default-value)で**自動的に入力**されます。

```jsx title="layout.js | page.js"
export const metadata = {
  metadataBase: new URL('https://acme.com'),
  alternates: {
    canonical: '/',
    languages: {
      'en-US': '/en-US',
      'de-DE': '/de-DE',
    },
  },
  openGraph: {
    images: '/og-image.png',
  },
}
```

```html title="<head> output" hideLineNumbers
<link rel="canonical" href="https://acme.com" />
<link rel="alternate" hreflang="en-US" href="https://acme.com/en-US" />
<link rel="alternate" hreflang="de-DE" href="https://acme.com/de-DE" />
<meta property="og:image" content="https://acme.com/og-image.png" />
```

> **Good to know**:
>
> - `metadataBase`は、root `app/layout.js`で設定されて、すべてのルートにわたってURLベースの`metadata`フィールドに適用されることが通常です。
> - 絶対URLを必要とするすべてのURLベースの`metadata`フィールドは、`metadataBase`オプションで設定できます。
> - `metadataBase`には、例としてサブドメインを含むことができます。 `https://app.acme.com` またはベースパス例： `https://acme.com/start/from/here`
> - `metadata`フィールドが絶対URLを提供する場合、`metadataBase`は無視されます。
> - URLベースの`metadata`フィールドで相対パスを使用して`metadataBase`を設定しないと、ビルドエラーが発生します。
> - Next.jsは`metadataBase`（例：`https://acme.com/`）とフィールド（例：`/path`）間の重複するスラッシュを単一のスラッシュ（例：`https://acme.com/path`）に正規化します。

#### デフォルト値 {#default-value}

設定されていない場合、`metadataBase`には**デフォルト値**があります。

> Vercelで：
>
> - 本番デプロイメントでは、`VERCEL_PROJECT_PRODUCTION_URL`が使用されます。
> - プレビューデプロイメントでは、`VERCEL_BRANCH_URL`が優先され、存在しない場合は`VERCEL_URL`にフォールバックします。
>
> これらの値が存在する場合、それらが`metadataBase`の**デフォルト値**として使用されます。そうでない場合、`http://localhost:${process.env.PORT || 3000}`にフォールバックします。これにより、ローカルビルドとVercelのプレビューおよび本番デプロイメントの両方でOpen Graphイメージが機能します。デフォルトを上書きする場合、環境変数を使用してURLを計算することをお勧めします。これにより、ローカル開発環境、ステージング環境、および本番環境用にURLを設定できます。
>
> これらの環境変数に関する詳細は、[システム環境変数](https://vercel.com/docs/concepts/projects/environment-variables/system-environment-variables)ドキュメントをご覧ください。

#### URLの構成 {#url-composition}

URLの構成は、デフォルトのディレクトリトラバーサルセマンティクスよりも開発者の意図を優先します。

- `metadataBase`と`metadata`フィールドの間の末尾のスラッシュは正規化されます。
- `metadata`フィールド（通常はURLパス全体を置き換える「絶対」パス）では、「relative」パス（`metadataBase`の終わりから始まる）として扱われます。

次の`metadataBase`を考えると：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/layout.js" switcher
export const metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

</TabItem>
</Tabs>

上記の`metadataBase`を継承し、自分の値を設定する`metadata`フィールドは以下のように解決されます：

| `metadata`フィールド             | 解決されたURL                    |
| -------------------------------- | -------------------------------- |
| `/`                              | `https://acme.com`               |
| `./`                             | `https://acme.com`               |
| `payments`                       | `https://acme.com/payments`      |
| `/payments`                      | `https://acme.com/payments`      |
| `./payments`                     | `https://acme.com/payments`      |
| `../payments`                    | `https://acme.com/payments`      |
| `https://beta.acme.com/payments` | `https://beta.acme.com/payments` |

## Unsupported Metadata {#unsupported-metadata}

次のメタデータタイプは現在、組み込みのサポートを持っていません。ただし、レイアウトまたはページ自体でレンダリングすることはできます。

| メタデータ                    | お薦め                                                                                                                                                                                                                                        |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `<meta http-equiv="...">`     | 適切なHTTPヘッダーを使用[redirect()](/docs/app/api-reference/functions/redirect)、[ミドルウェア](/docs/app/building-your-application/routing/middleware#nextresponse)、[セキュリティヘッダー](/docs/app/api-reference/next-config-js/headers) |
| `<base>`                      | レイアウトまたはページ自体でタグをレンダリングします。                                                                                                                                                                                        |
| `<noscript>`                  | レイアウトまたはページ自体でタグをレンダリングします。                                                                                                                                                                                        |
| `<style>`                     | [Next.jsでのスタイリング](/docs/app/building-your-application/styling/css)についての詳細を学びましょう。                                                                                                                                      |
| `<script>`                    | [スクリプトの使用](/docs/app/building-your-application/optimizing/scripts)についての詳細を学びましょう。                                                                                                                                      |
| `<link rel="stylesheet" />`   | スタイルシートをレイアウトまたはページ自体で直接`import`してください。                                                                                                                                                                        |
| `<link rel="preload />`       | [ReactDOMプリロードメソッド](#link-relpreload)を使用                                                                                                                                                                                          |
| `<link rel="preconnect" />`   | [ReactDOMプレコネクトメソッド](#link-relpreconnect)を使用                                                                                                                                                                                     |
| `<link rel="dns-prefetch" />` | [ReactDOMプリフェッチDNSメソッド](#link-reldns-prefetch)を使用                                                                                                                                                                                |

### リソースのヒント {#resource-hints}

`<link>`要素には、外部リソースが必要になる可能性があることをブラウザにヒントするために使用できるいくつかの`rel`キーワードがあります。ブラウザはこの情報を使用して、キーワードに応じてプレロード最適化を適用します。

メタデータAPIはこれらのヒントを直接サポートしていませんが、新しい[`ReactDOM`メソッド](https://github.com/facebook/react/pull/26237)を使用してドキュメントの`<head>`に安全に挿入できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/preload-resources.tsx" switcher
'use client'

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/preload-resources.js" switcher

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

</TabItem>
</Tabs>

##### `<link rel="preload">` {#link-rel-preload}

ページのレンダリング（ブラウザ）ライフサイクルを早期にリソースを読み込み始めます。[MDNドキュメント](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preload)。

```tsx
ReactDOM.preload(href: string, options: { as: string })
```

```html title="<head> output" hideLineNumbers
<link rel="preload" href="..." as="..." />
```

##### `<link rel="preconnect">` {#link-rel-preconnect}

起源への接続を事前に開始します。[MDNドキュメント](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preconnect)。

```tsx
ReactDOM.preconnect(href: string, options?: { crossOrigin?: string })
```

```html title="<head> output" hideLineNumbers
<link rel="preconnect" href="..." crossorigin />
```

##### `<link rel="dns-prefetch">` {#link-rel-dns-prefetch}

リソースがリクエストされる前にドメイン名を解決しようとします。[MDNドキュメント](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/dns-prefetch)。

```tsx
ReactDOM.prefetchDNS(href: string)
```

```html title="<head> output" hideLineNumbers
<link rel="dns-prefetch" href="..." />
```

> **Good to know**:
>
> - これらのメソッドは現在、初回ページロードでサーバーサイドレンダリングされるクライアントコンポーネントでのみサポートされています。
> - Next.jsの組み込み機能である`next/font`、`next/image`、`next/script`は、関連するリソースヒントを自動的に処理します。

## タイプ {#types}

`Metadata`タイプを使用してメタデータに型安全性を追加できます。IDEで [組み込みのTypeScriptプラグイン](/docs/app/building-your-application/configuring/typescript) を使用している場合、手動で型を追加する必要はありませんが、それでも明示的に追加することができます。

### `metadata`オブジェクト {#metadata-object}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Next.js',
}
```

### `generateMetadata`関数 {#generatemetadata-function}

#### 通常の関数 {#regular-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export function generateMetadata(): Metadata {
  return {
    title: 'Next.js',
  }
}
```

#### 非同期関数 {#async-function}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

export async function generateMetadata(): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### セグメントpropsを使用する場合 {#with-segment-props}

```tsx title="layout.tsx | page.tsx"
import type { Metadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export function generateMetadata({ params, searchParams }: Props): Metadata {
  return {
    title: 'Next.js',
  }
}

export default function Page({ params, searchParams }: Props) {}
```

#### 親のメタデータを使用する場合 {#with-parent-metadata}

```tsx title="layout.tsx | page.tsx"
import type { Metadata, ResolvingMetadata } from 'next'

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  return {
    title: 'Next.js',
  }
}
```

#### JavaScriptプロジェクト {#javascript-projects}

JavaScriptプロジェクトの場合、JSDocを使用して型安全性を追加できます。

```js title="layout.js | page.js"
/** @type {import("next").Metadata} */
export const metadata = {
  title: 'Next.js',
}
```
