`unstable_noStore`は、特定の[サーバーコンポーネント](/docs/app/building-your-application/rendering/server-components)がキャッシュされないように指示し、静的レンダリングを明示的に回避するために使用できます。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```

> **Good to know**:
>
> - `unstable_noStore`は、`fetch`の`cache: 'no-store'`と同等です
> - `unstable_noStore`は`export const dynamic = 'force-dynamic'`よりも細かく制御でき、コンポーネントごとに使用できるため、推奨されます

- [`unstable_cache`](/docs/app/api-reference/functions/unstable_cache)内で`unstable_noStore`を使用しても、静的生成から除外されません。その代わりに、キャッシュ構成に従って結果をキャッシュするかどうかを決定します。

## 使用法

`fetch`に`cache: 'no-store'`、`next: { revalidate: 0 }`を追加で渡したくない場合、または`fetch`が使用できない場合、これらのすべてのユースケースに対する代替として`noStore()`を使用できます。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```
