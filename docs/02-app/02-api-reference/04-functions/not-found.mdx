`notFound` 関数は、ルートセグメント内で [`not-found ファイル`](/docs/app/api-reference/file-conventions/not-found) をレンダリングし `<meta name="robots" content="noindex" />` タグを挿入することを可能にします。

## `notFound()`

`notFound()` 関数を呼び出すと、`NEXT_NOT_FOUND` エラーがスローされ、それがスローされたルートセグメントのレンダリングが終了します。[**not-found** ファイル](/docs/app/api-reference/file-conventions/not-found)を指定することで、セグメント内に「Not Found」UIをレンダリングすることで、このようなエラーを円滑に処理できます。

```jsx title="app/user/[id]/page.js"
import { notFound } from 'next/navigation'

async function fetchUser(id) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined // 「Not Found」UIを表示します。
  return res.json() // ユーザーデータを返します。
}

export default async function Profile({ params }) {
  const user = await fetchUser(params.id)

  if (!user) {
    notFound() // ユーザーが見つからない場合にエラーを処理します。
  }

  // ...
}
```

> **Good to know**: `notFound()` は、TypeScript の [`never`](https://www.typescriptlang.org/docs/handbook/2/functions.html#never) 型を使用しているため、`return notFound()` を使う必要はありません；
