ページの初期ビューポートは、静的な `viewport` オブジェクトや動的な `generateViewport` 関数を使用してカスタマイズできます。

> **知っておくとよいこと**：
>
> - `viewport` オブジェクトと `generateViewport` 関数のエクスポートは、**Server Component** でのみサポートされています。
> - 同じルートセグメントから `viewport` オブジェクトと `generateViewport` 関数の両方をエクスポートすることはできません。
> - `metadata` エクスポートの移行からきている場合は、[metadata-to-viewport-export codemod](/docs/app/building-your-application/upgrading/codemods#metadata-to-viewport-export) を使用して変更を更新できます。

## `viewport` オブジェクト {#the-viewport-object}

ビューポートオプションを定義するには、`layout.jsx` または `page.jsx` ファイルから `viewport` オブジェクトをエクスポートしてください。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
import type { Viewport } from 'next'

export const viewport: Viewport = {
  themeColor: 'black',
}

export default function Page() {}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.jsx | page.jsx" switcher
export const viewport = {
  themeColor: 'black',
}

export default function Page() {}
```

</TabItem>
</Tabs>

## `generateViewport` 関数 {#generateviewport-function}

`generateViewport` は、1つまたは複数のビューポートフィールドを含む [`Viewport` オブジェクト](#viewport-fields) を返す必要があります。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
export function generateViewport({ params }) {
  return {
    themeColor: '...',
  }
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.js | page.js" switcher
export function generateViewport({ params }) {
  return {
    themeColor: '...',
  }
}
```

</TabItem>
</Tabs>

> **知っておくとよいこと**：
>
> - ビューポートがランタイム情報に依存しない場合、静的な [`viewport` オブジェクト](#the-viewport-object) を使用して定義するべきであり、`generateViewport` ではありません。

**シンプルなテーマカラー**

<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher

  themeColor: 'black',
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="layout.jsx | page.jsx" switcher

  themeColor: 'black',
}
```

</TabItem>
</Tabs>

```html title="<head> output" hideLineNumbers

```

### `width`, `initialScale`, `maximumScale` および `userScalable` {#width-initialscale-maximumscale-and-userscalable}

> **知っておくとよいこと**：`viewport` メタタグは自動的に設定され、手動での構成は通常不必要です。デフォルトで十分ですが、情報は完全性のために提供されます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="layout.tsx | page.tsx" switcher
  initialScale: 1,
  maximumScale: 1,
  userScalable: false,
  // Also supported by less commonly used
  // interactiveWidget: 'resizes-visual',
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

## タイプ {#types}

`Viewport` タイプを使用することで、ビューポートオブジェクトに型安全性を付与することができます。IDEで[組み込みのTypeScriptプラグイン](/docs/app/building-your-application/configuring/typescript)を使用している場合、手動で型を追加する必要はありませんが、それでも明示的に追加することは可能です。

### `viewport` オブジェクト {#viewport-object}

```tsx
import type { Viewport } from 'next'

export const viewport: Viewport = {
  themeColor: 'black',
}
```

### `generateViewport` 関数 {#generateviewport-function}

#### 通常の関数 {#regular-function}

```tsx
import type { Viewport } from 'next'

export function generateViewport(): Viewport {
  return {
    themeColor: 'black',
  }
}
```

#### セグメントpropsを使用した場合 {#with-segment-props}

```tsx
import type { Viewport } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export function generateViewport({ params, searchParams }: Props): Viewport {
  return {
    themeColor: 'black',
  }
}

export default function Page({ params, searchParams }: Props) {}
```

#### JavaScriptプロジェクト {#javascript-projects}

JavaScriptプロジェクトの場合、JSDocを使用して型安全性を追加できます。

```js
/** @type {import("next").Viewport} */
export const viewport = {
  themeColor: 'black',
}
```
