`useSearchParams` は現在の URL の **クエリストリング** を読み込むことができる **Client Component** のフックです。

`useSearchParams` は [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams) インターフェースの**読み取り専用**バージョンを返します。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // URL -> `/dashboard?search=my-project`
  // `search` -> 'my-project'
  return <>Search: {search}</>
}
```

</TabItem>

## Parameters {#parameters}

```tsx
const searchParams = useSearchParams()
```

`useSearchParams` はパラメーターを必要としません。

## Returns {#returns}

`useSearchParams` は URL のクエリストリングを読み取るためのユーティリティーメソッドを含む、[`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams) インターフェースの**読み取り専用**バージョンを返します：

- [`URLSearchParams.get()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/get)：検索パラメーターに関連付けられている最初の値を返します。例えば：

  | URL                  | `searchParams.get("a")`                                                                                                                |
  | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
  | `/dashboard?a=1`     | `'1'`                                                                                                                                  |
  | `/dashboard?a=`      | `''`                                                                                                                                   |
  | `/dashboard?b=3`     | `null`                                                                                                                                 |
  | `/dashboard?a=1&a=2` | `'1'` _- すべての値を取得するには [`getAll()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/getAll) を使用してください _ |

- [`URLSearchParams.has()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/has)：指定されたパラメーターが存在するかどうかを示す真偽値を返します。例えば：

  | URL              | `searchParams.has("a")` |
  | ---------------- | ----------------------- |
  | `/dashboard?a=1` | `true`                  |
  | `/dashboard?b=3` | `false`                 |

- [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams)のその他の**読み取り専用**メソッドについての詳細を知りたい場合は、[`getAll()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/getAll)、[`keys()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/keys)、[`values()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/values)、[`entries()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/entries)、[`forEach()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/forEach)、[`toString()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/toString) をご覧ください。

> **Good to know**:
>
> - `useSearchParams` は[Client Component](/docs/app/building-your-application/rendering/client-components)のフックであり、古い値を防ぐために[Server Components](/docs/app/building-your-application/rendering/server-components)では**サポートされていません**。[部分的なレンダリング](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering)です。
> - アプリケーションに`/pages`ディレクトリが含まれている場合、`useSearchParams` は `ReadonlyURLSearchParams | null` を返します。ページが`getServerSideProps`を使用しないプリレンダリング中に検索パラメータを知ることができないため、移行中の互換性のために`null`値が設定されています。

### Static Rendering {#static-rendering}

ルートが[静的にレンダリング](/docs/app/building-your-application/rendering/server-components#static-rendering-default)されている場合、`useSearchParams` を呼び出すと最も近い[`Suspense` バウンダリ](/docs/app/building-your-application/routing/loading-ui-and-streaming#example)までの Client Component ツリーがクライアント側でレンダリングされます。

これにより、`useSearchParams` を使用する動的な部分がクライアント側でレンダリングされる一方で、ルートの一部が静的にレンダリングされるようになります。

`useSearchParams` を使用する Client Component を `<Suspense/>` バウンダリでラップすることをお勧めします。これにより、上の Client Component は静的にレンダリングされ、最初の HTML の一部として送信されます。 [Example](/docs/app/api-reference/functions/use-search-params#static-rendering).

例えば：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // 静的レンダリングを使用しているとき、サーバー上ではログ出力されません
  console.log(search)

  return <>Search: {search}</>
}
```

## Behavior {#behavior}

### Dynamic Rendering {#dynamic-rendering}

ルートが[動的にレンダリング](/docs/app/building-your-application/rendering/server-components#dynamic-rendering)される場合、`useSearchParams` は Client Component の初期サーバーレンダリング中にサーバーで利用可能です。

例えば：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // 初期レンダリング中にサーバーでログ出力され
  // その後のナビゲーションでクライアント上でもログ出力されます。
  console.log(search)

  return <>Search: {search}</>
}
```

### Server Components {#server-components}

#### Pages {#pages}

[Pages](/docs/app/api-reference/file-conventions/page)（Server Components）で検索パラメーターにアクセスするには、[`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) プロップを使用します。

#### Layouts {#layouts}

Pages とは異なり、[Layouts](/docs/app/api-reference/file-conventions/layout)（Server Components）は `searchParams` プロップを受け取りません。これは、共有レイアウトが[ナビゲーション中に再レンダリングされない](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering)ため、ナビゲーション間で古い`searchParams` が残る可能性があるためです。 [詳細な説明](/docs/app/api-reference/file-conventions/layout#layouts-do-not-receive-searchparams)をご覧ください。

代わりに、ページの[`searchParams`](/docs/app/api-reference/file-conventions/page) プロップや、`searchParams`が最新の状態でクライアントで再レンダリングされる Client Component での[`useSearchParams`](/docs/app/api-reference/functions/use-search-params) フックを使用してください。

## Examples {#examples}

### `searchParams` の更新 {#updating-searchparams}

[`useRouter`](/docs/app/api-reference/functions/use-router) や [`Link`](/docs/app/api-reference/components/link) を使用して、新しい `searchParams` を設定できます。ナビゲーションが行われると、現在の [`page.js`](/docs/app/api-reference/file-conventions/page) は更新された [`searchParams` プロップ](/docs/app/api-reference/file-conventions/page#searchparams-optional) を受け取ります。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/example-client-component.tsx" switcher
'use client'

export default function ExampleClientComponent() {
  const router = useRouter()
  const pathname = usePathname()
  const searchParams = useSearchParams()

  // 現在の searchParams を指定されたキー/値ペアと結合することで新しい searchParams 文字列を取得します
  const createQueryString = useCallback(
    (name: string, value: string) => {
      const params = new URLSearchParams(searchParams.toString())
      params.set(name, value)

      return params.toString()
    },
    [searchParams]
  )

  return (
    <>
      <p>Sort By</p>

      {/* using useRouter */}
      <button
        onClick={() => {
          // <pathname>?sort=asc
          router.push(pathname + '?' + createQueryString('sort', 'asc'))
        }}
      >
        ASC
      </button>

      {/* using <Link> */}
      <Link
        href={
          // <pathname>?sort=desc
          pathname + '?' + createQueryString('sort', 'desc')
        }
      >
        DESC
      </Link>
    </>
  )
}
```

</TabItem>
