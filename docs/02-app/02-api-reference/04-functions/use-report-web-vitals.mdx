`useReportWebVitals` フックを使用すると、[Core Web Vitals](https://web.dev/vitals/) をレポートし、分析サービスと組み合わせて使用することができます。

<PagesOnly>

```jsx title="pages/_app.js"
import { useReportWebVitals } from 'next/web-vitals'

function MyApp({ Component, pageProps }) {
  useReportWebVitals((metric) => {
    console.log(metric)
  })

  return <Component {...pageProps} />
}
```

</PagesOnly>

<AppOnly>

```jsx title="app/_components/web-vitals.js"
'use client'

import { useReportWebVitals } from 'next/web-vitals'

export function WebVitals() {
  useReportWebVitals((metric) => {
    console.log(metric)
  })
}
```

```jsx title="app/layout.js"
import { WebVitals } from './_components/web-vitals'

export default function Layout({ children }) {
  return (
    <html>
      <body>
        <WebVitals />
        {children}
      </body>
    </html>
  )
}
```

> `useReportWebVitals` フックは `"use client"` ディレクティブが必要であるため、最もパフォーマンスの高いアプローチは、root レイアウトがインポートする別のコンポーネントを作成することです。これにより、client boundary は `WebVitals` コンポーネントにのみ限定されます。

</AppOnly>

## useReportWebVitals

フックの引数として渡される `metric` オブジェクトは、いくつかのプロパティで構成されています：

- `id`: 現在のページロードのコンテキスト内でのメトリックの一意の識別子
- `name`: パフォーマンス メトリックの名称。可能な値には、Web アプリケーションに特有の [Web Vitals](#web-vitals) メトリック（TTFB、FCP、LCP、FID、CLS） の名前が含まれます。
- `delta`: メトリックの現在の値と前の値の差。通常、この値はミリ秒単位であり、時間経過に伴うメトリックの値の変化を示します。
- `entries`: メトリックに関連付けられた [Performance Entries](https://developer.mozilla.org/docs/Web/API/PerformanceEntry) の配列。これらのエントリは、メトリックに関連するパフォーマンス イベントの詳細情報を提供します。
- `navigationType`: メトリック収集をトリガーした[ナビゲーションのタイプ](https://developer.mozilla.org/docs/Web/API/PerformanceNavigationTiming/type)を示します。可能な値には `"navigate"`、`"reload"`、`"back_forward"`、および `"prerender"` が含まれます。
- `rating`: メトリック値の定性的な評価であり、パフォーマンスの評価を提供します。可能な値は `"good"`、`"needs-improvement"`、および `"poor"` です。評価は通常、メトリック値を受け入れられるか、最適でないかを示す事前定義されたしきい値と比較して決定されます。
- `value`: パフォーマンス エントリの実際の値または期間。通常はミリ秒単位で、測定されたパフォーマンス側面の定量的な指標を提供します。値のソースは、測定される特定のメトリックによって異なり、さまざまな [Performance API](https://developer.mozilla.org/docs/Web/API/Performance_API) から取得されることがあります。

## Web Vitals

[Web Vitals](https://web.dev/vitals/) は、ウェブページのユーザーエクスペリエンスを捉えるための有用な指標セットです。以下の Web Vitals がすべて含まれています：

- [Time to First Byte](https://developer.mozilla.org/docs/Glossary/Time_to_first_byte) (TTFB)
- [First Contentful Paint](https://developer.mozilla.org/docs/Glossary/First_contentful_paint) (FCP)
- [Largest Contentful Paint](https://web.dev/lcp/) (LCP)
- [First Input Delay](https://web.dev/fid/) (FID)
- [Cumulative Layout Shift](https://web.dev/cls/) (CLS)
- [Interaction to Next Paint](https://web.dev/inp/) (INP)

これらのメトリクスのすべての結果を `name` プロパティを使用して処理できます。

<PagesOnly>

```jsx title="pages/_app.js"
import { useReportWebVitals } from 'next/web-vitals'

function MyApp({ Component, pageProps }) {
  useReportWebVitals((metric) => {
    switch (metric.name) {
      case 'FCP': {
        // FCP 結果を処理
      }
      case 'LCP': {
        // LCP 結果を処理
      }
      // ...
    }
  })

  return <Component {...pageProps} />
}
```

</PagesOnly>

## カスタムメトリクス

上記のコアメトリクスに加えて、ページのハイドレートとレンダーにかかる時間を測定する追加のカスタムメトリクスがあります：

- `Next.js-hydration`: ページがハイドレーションを開始および終了するのにかかる時間（ミリ秒単位）
- `Next.js-route-change-to-render`: ルート変更後にページがレンダリングを開始するのにかかる時間（ミリ秒単位）
- `Next.js-render`: ルート変更後にページがレンダリングを完了するのにかかる時間（ミリ秒単位）

これらのメトリクスのすべての結果を個別に処理できます：

```jsx title="pages/_app.js"
import { useReportWebVitals } from 'next/web-vitals'

function MyApp({ Component, pageProps }) {
  useReportWebVitals((metric) => {
    switch (metric.name) {
      case 'Next.js-hydration':
        // ハイドレーション結果を処理
        break
      case 'Next.js-route-change-to-render':
        // ルート変更からレンダーまでの結果を処理
        break
      case 'Next.js-render':
        // レンダー結果を処理
        break
      default:
        break
    }
  })

  return <Component {...pageProps} />
}
```

これらのメトリクスは、[User Timing API](https://caniuse.com/#feat=user-timing) をサポートするすべてのブラウザで動作します。

</PagesOnly>

## Vercelでの使用

[Vercel Speed Insights](https://vercel.com/docs/speed-insights/quickstart) は `useReportWebVitals` を使用せず、代わりに `@vercel/speed-insights` パッケージを使用します。`useReportWebVitals` フックはローカル開発で便利で、または別のサービスを使用して Web Vitals を収集している場合に役立ちます。

## 外部システムへの結果送信

結果を任意のエンドポイントに送信して、サイト上でリアルユーザーパフォーマンスを測定および追跡できます。たとえば：

```js
useReportWebVitals((metric) => {
  const body = JSON.stringify(metric)
  const url = 'https://example.com/analytics'

  // `navigator.sendBeacon()` を利用可能である場合は使い、そうでない場合は `fetch()` を使用します。
  if (navigator.sendBeacon) {
    navigator.sendBeacon(url, body)
  } else {
    fetch(url, { body, method: 'POST', keepalive: true })
  }
})
```

> **Good to know**: [Google アナリティクス](https://analytics.google.com/analytics/web/) を使用する場合、
> `id` 値を使用することで、パーセンタイルを計算するためなどに、メトリックの分布を手動で構築することが可能です。

> ```js
> useReportWebVitals(metric => {
>   // この例のように Google アナリティクスを初期化した場合は `window.gtag` を使用します：
>   // https://github.com/vercel/next.js/blob/canary/examples/with-google-analytics
>   window.gtag('event', metric.name, {
>     value: Math.round(metric.name === 'CLS' ? metric.value * 1000 : metric.value), // 値は整数でなければなりません
>     event_label: metric.id, // 現在のページロードに固有のid
>     non_interaction: true, // 直帰率に影響を与えません。
>   });
> }
> ```
>
> [Google アナリティクスへの結果送信](https://github.com/GoogleChrome/web-vitals#send-the-results-to-google-analytics)についてもっと読む。
