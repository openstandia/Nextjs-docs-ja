`<Link>`は、HTML `<a>`要素を拡張して、[プリフェッチ機能](/docs/app/building-your-application/routing/linking-and-navigating#2-prefetching)とクライアント側でのルート間のナビゲーションを提供するReactコンポーネントです。Next.jsにおけるルート間のナビゲーションの主な方法です。

基本的な使用方法：

```tsx title="app/page.tsx" switcher
export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

## リファレンス {#reference}

以下のpropsを`<Link>`コンポーネントに渡すことができます：

<PagesOnly>

| Prop                                | 例                      | 型                | 必須 |
| ----------------------------------- | ----------------------- | ----------------- | ---- |
| [`href`](#href-required)            | `href="/dashboard"`     | String or Object  | はい |
| [`replace`](#replace)               | `replace={false}`       | Boolean           | -    |
| [`scroll`](#scroll)                 | `scroll={false}`        | Boolean           | -    |
| [`prefetch`](#prefetch)             | `prefetch={false}`      | Boolean           | -    |
| [`legacyBehavior`](#legacybehavior) | `legacyBehavior={true}` | Boolean           | -    |
| [`passHref`](#passhref)             | `passHref={true}`       | Boolean           | -    |
| [`shallow`](#shallow)               | `shallow={false}`       | Boolean           | -    |
| [`locale`](#locale)                 | `locale="fr"`           | String or Boolean | -    |

</PagesOnly>

<AppOnly>

| Prop                     | 例                  | 型               | 必須 |
| ------------------------ | ------------------- | ---------------- | ---- |
| [`href`](#href-required) | `href="/dashboard"` | String or Object | はい |

| [`scroll`](#scroll)      | `scroll={false}`    | Boolean          | -    |

### `scroll` {#scroll}

**デフォルトでは`true`です。** Next.jsでの`<Link>`のデフォルトのスクロール動作は、ブラウザの戻る、進むナビゲーションに似て、スクロール位置を保持します。新しい[Page](/docs/app/building-your-application/routing/pages)に遷移するとき、ページがビューポート内に表示されている限りスクロール位置はそのままです。しかし、ページがビューポート内に表示されていなければ、Next.jsは最初のページ要素のトップにスクロールします。

`scroll = {false}`の場合、Next.jsは最初のページ要素へスクロールを試みません。

> **Good to know**: Next.jsはスクロール動作を管理する前に`scroll: false`を確認します。スクロールが有効である場合、ナビゲーション用に関連するDOMノードを特定し、各トップレベル要素を調査します。すべてのスクロール不可能な要素やHTMLがレンダリングされていないもの（固定された位置の要素や非表示の要素など）はバイパスされ、Next.jsはスクロール可能でビューポート内に見える要素を特定するまで兄弟要素を進み続けます。

### `prefetch` {#prefetch}

<AppOnly>

プリフェッチは、`<Link />`コンポーネントがユーザーのビューポートに入ると発生します（最初またはスクロールによって）。Next.jsは関連するルート（`href`で示される）のデータをバックグラウンドでプリフェッチし、クライアント側のナビゲーションのパフォーマンスを向上させます。プリフェッチされたデータがユーザーが`<Link />`にホバーするまでに期限切れになっている場合、Next.jsは再びプリフェッチを試みます。**プリフェッチは本番環境でのみ有効です。**

`prefetch`prop に渡すことができる値は次のとおりです：

- **`null`（デフォルト）**: ルートが静的か動的かに応じてプリフェッチ動作が異なります。静的ルートの場合、ルート（およびそのデータ全体）がプリフェッチされます。動的ルートの場合、最も近いセグメントとその[`loading.js`](/docs/app/building-your-application/routing/loading-ui-and-streaming#instant-loading-states)バウンダリを含む部分的なルートがプリフェッチされます。
- `true`: 静的ルートでも動的ルートでも、ルート全体がプリフェッチされます。
- `false`: プリフェッチは、ビューポートに入る際にもホバーでさえも、決して発生しません。

```tsx title="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="app/page.js" switcher

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</TabItem>
</Tabs>

</AppOnly>

<PagesOnly>

プリフェッチは、`<Link />`コンポーネントがユーザーのビューポートに入ると発生します（最初またはスクロールによって）。Next.jsは関連するルート（`href`で示される）のデータをバックグラウンドでプリフェッチし、クライアント側のナビゲーションのパフォーマンスを向上させます。**プリフェッチは本番環境でのみ有効です。**

`prefetch`prop に渡すことができる値は次のとおりです：

- **`true`（デフォルト）**: ルート全体とそのデータがプリフェッチされます。

### `legacyBehavior` {#legacybehavior}

`<a>`要素は、`<Link>`の子としてもはや必要ありません。レガシー動作を使用するには`legacyBehavior`propsを追加するか、または`<a>`を削除してアップグレードしてください。コードを自動的にアップグレードする[コードモードが利用可能です](/docs/app/building-your-application/upgrading/codemods#new-link)。

> **Good to know**: `legacyBehavior`が`true`に設定されていない場合、`next/link`に`className`、`onClick`などの[`anchor`](https://developer.mozilla.org/docs/Web/HTML/Element/a)タグのプロパティも渡せます。

### `passHref` {#passhref}

`Link`がその子に`href`プロパティを送信するように強制します。デフォルトでは`false`です。[機能的なコンポーネントを渡す](#nesting-a-functional-component)例を参照してください。

### `locale` {#locale}

アクティブなロケールが自動的に先頭に付けられます。`locale`は、異なるロケールを提供するために使用します。`false`の場合、デフォルトの動作が無効になるため、`href`にロケールを含める必要があります。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="pages/index.tsx" switcher
import Link from 'next/link'

export default function Home() {
  return (
    <>
      {/* デフォルトの動作：ロケールが先頭に付けられる */}
      <Link href="/dashboard">Dashboard (with locale)</Link>

      {/* ロケールの先頭付けを無効にする */}
      <Link href="/dashboard" locale={false}>
        Dashboard (without locale)
      </Link>

      {/* 異なるロケールを指定する */}
      <Link href="/dashboard" locale="fr">
        Dashboard (French)
      </Link>
    </>
  )
}
```

```jsx title="pages/index.js" switcher
  return (
```
```tsx title="app/blog/post-list.tsx" switcher
    <ul>
      {posts.map((post) => (
```

### アクティブリンクの確認 {#checking-active-links}

[`usePathname()`](/docs/app/api-reference/functions/use-pathname)を使用してリンクがアクティブかどうかを確認できます。例えば、現在の`pathname`がリンクの`href`と一致しているかどうかを確認して、アクティブなリンクにクラスを追加することができます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/ui/nav-links.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'
import Link from 'next/link'

export function Links() {
  const pathname = usePathname()

  return (
    <nav>
      <Link className={`link ${pathname === '/' ? 'active' : ''}`} href="/">
        Home
      </Link>

      <Link
        className={`link ${pathname === '/about' ? 'active' : ''}`}
        href="/about"
      >
        About
      </Link>
    </nav>
  )
}
```

</TabItem>

### `id`へのスクロール {#scrolling-to-an-id}

ナビゲーションで特定の`id`にスクロールしたい場合、URLに`#`ハッシュリンクを付加するか、`href`propにハッシュリンクを渡すことができます。これは、`<Link>`が`<a>`要素にレンダリングされるため可能です。

```jsx
<Link href="/dashboard#settings">Settings</Link>

// 出力
<a href="/dashboard#settings">Settings</a>
```

> **Good to know**:
>
> - Next.jsは、ナビゲーション時にビューポート内に表示されていない場合、[Page](/docs/app/building-your-application/routing/pages)にスクロールします。

### 子が<a>タグをラップするカスタムコンポーネントの場合 {#if-the-child-is-a-custom-component-that-wraps-an-a-tag}

<AppOnly>

`Link`の子が`<a>`タグをラップするカスタムコンポーネントである場合、`Link`に`passHref`を追加しなければなりません。これは、[styled-components](https://styled-components.com/)のようなライブラリを使用している場合に必要です。これがないと、`<a>`タグに`href`属性がなくなり、サイトのアクセシビリティに影響を与え、SEOに影響を与える可能性があります。[ESLint](/docs/app/building-your-application/configuring/eslint#eslint-plugin)を使用している場合、`passHref`の正しい使用を保証するための組み込みルール`next/link-passhref`があります。

</AppOnly>

### 機能的なコンポーネントのネスト {#nesting-a-functional-component}

`Link`の子が機能的なコンポーネントである場合、`passHref`と`legacyBehavior`を使用することに加えて、コンポーネントを[`React.forwardRef`](https://react.dev/reference/react/forwardRef)でラップする必要があります：

<AppOnly>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
import Link from 'next/link'
import React from 'react'

// MyButton の props タイプを定義
interface MyButtonProps {
  onClick?: React.MouseEventHandler<HTMLAnchorElement>
  href?: string
}

// React.ForwardRefRenderFunction を使用して転送されたrefを正しく型付けします
const MyButton: React.ForwardRefRenderFunction<
  HTMLAnchorElement,
  MyButtonProps
> = ({ onClick, href }, ref) => {
  return (
    <a href={href} onClick={onClick} ref={ref}>
      Click Me
    </a>
  )
}

// React.forwardRefを使用してコンポーネントをラップします
const ForwardedMyButton = React.forwardRef(MyButton)

export default function Page() {
  return (
    <Link href="/about" passHref legacyBehavior>
      <ForwardedMyButton />
    </Link>
  )
}
```

### ページのトップへのスクロールを無効にする {#disable-scrolling-to-the-top-of-the-page}

<AppOnly>

Next.jsにおける`<Link>`のデフォルトのスクロール動作は、ブラウザの戻る、進むナビゲーションに似て、スクロール位置を保持します。新しい[Page](/docs/app/building-your-application/routing/pages)への遷移時に、ページがビューポート内に表示されている限り、スクロール位置はそのままです。

しかし、ページがビューポート内に表示されていない場合、Next.jsは最初のページ要素の上部にスクロールします。この動作を無効にしたい場合、`<Link>`コンポーネントに`scroll={false}`を渡すか、`scroll: false`を`router.push()`または`router.replace()`に渡すことができます。

```jsx title="app/page.js" switcher
  return (

      ページのトップへのスクロールを無効にする
    </Link>
  )
}
```

</TabItem>

### Middlewareでリンクをプリフェッチする {#prefetching-links-in-middleware}

[Middleware](/docs/app/building-your-application/routing/middleware)を認証やユーザーを異なるページにリダイレクトする目的で使用することが一般的です。Middleware経由でのリライトを使用してリンクを正しくプリフェッチするために、表示するURLとプリフェッチするURLの両方をNext.jsに知らせる必要があります。これは、プリフェッチする正しいルートを知るためにMiddlewareに不要なフェッチを避けるために必要です。

たとえば、認証済みと訪問者の両方のビューを持つ`/dashboard`ルートを提供したい場合、次のようにMiddlewareに追加してユーザーを適切なページにリダイレクトできます：

```ts title="middleware.ts"
import { NextResponse } from 'next/server'

export function middleware(request: Request) {
  const nextUrl = request.nextUrl
  if (nextUrl.pathname === '/dashboard') {
    if (request.cookies.authToken) {
      return NextResponse.rewrite(new URL('/auth/dashboard', request.url))
    } else {
      return NextResponse.rewrite(new URL('/public/dashboard', request.url))
    }
  }
}
```

```js title="middleware.js"
import { NextResponse } from 'next/server'

export function middleware(request) {
  const nextUrl = request.nextUrl
  if (nextUrl.pathname === '/dashboard') {
    if (request.cookies.authToken) {
      return NextResponse.rewrite(new URL('/auth/dashboard', request.url))
    } else {
      return NextResponse.rewrite(new URL('/public/dashboard', request.url))
    }
  }
}
```
