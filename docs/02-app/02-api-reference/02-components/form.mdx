`<Form>` コンポーネントは、HTML `<form>` 要素を拡張し、<AppOnly>[**プレフェッチ**](/docs/app/building-your-application/routing/linking-and-navigating#2-prefetching) と [ローディング UI](/docs/app/building-your-application/routing/loading-ui-and-streaming)、</AppOnly>送信時の**クライアントサイドナビゲーション**、および**プログレッシブエンハンスメント**を提供します。

これは、URLの検索パラメータを更新するフォームに便利です。これにより、上記を実現するために必要なボイラープレートコードが削減されます。

基本的な使用方法：

<AppOnly>

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="/app/ui/search.tsx" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      {/* 送信時に、入力値がURLに追加されます
          例: /search?query=abc */}
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

</TabItem>

## リファレンス {#reference}

`<Form>` コンポーネントの動作は、`action` prop に `string` か `function` が渡されるかによって異なります。

<AppOnly>

- `action` が**文字列**の場合、`<Form>` はネイティブ HTML フォームのように動作し、**`GET`** メソッドを使用します。フォームデータは検索パラメータとしてURLにエンコードされ、フォームが送信されると指定されたURLに移動します。さらに、Next.jsは：
  - フォームが表示されたときにパスを[プレフェッチ](/docs/app/building-your-application/routing/linking-and-navigating#2-prefetching)し、共有UI（例：`layout.js` と `loading.js`）をプレロードします。これにより、ナビゲーションが迅速になります。
  - フォームが送信されると、[クライアントサイドのナビゲーション](/docs/app/building-your-application/routing/linking-and-navigating#5-soft-navigation)を行い、ページ全体のリロードを避けます。これにより、共有UIとクライアントサイドの状態を保持します。
- `action` が**関数**（Server Action）の場合、`<Form>` は [React フォーム](https://react.dev/reference/react-dom/components/form) のように動作し、フォームが送信されたときにそのアクションを実行します。

</AppOnly>

<PagesOnly>

- `action` が**文字列**の場合、`<Form>` はネイティブ HTML フォームのように動作し、**`GET`** メソッドを使用します。フォームデータは検索パラメータとしてURLにエンコードされ、フォームが送信されると指定されたURLに移動します。さらに、Next.jsは：
  - フォームが送信されると、ページ全体のリロードを行わずに[クライアントサイドのナビゲーション](/docs/app/building-your-application/routing/linking-and-navigating#5-soft-navigation)を行います。これにより、共有UIとクライアントサイドの状態を保持します。

</PagesOnly>

### `action`（文字列）Props {#action-string-props}

<PagesOnly>

`action` が文字列の場合、`<Form>` コンポーネントは次の props をサポートします：

| Prop      | 例                 | 型                            | 必須 |
| --------- | ------------------ | ----------------------------- | ---- |
| `action`  | `action="/search"` | `string` (URL または相対パス) | Yes  |
| `replace` | `replace={false}`  | `boolean`                     | -    |
| `scroll`  | `scroll={true}`    | `boolean`                     | -    |

- **`action`**: フォームが送信されたときに移動するURLまたはパス
  - 空文字列 `""` を指定すると、検索パラメータが更新された同じルートに移動します
- **`replace`**: 現在の履歴の状態を置き換え、[ブラウザの履歴](https://developer.mozilla.org/en-US/docs/Web/API/History_API)スタックに新しい履歴をプッシュしないようにします。デフォルトは `false` です
- **`scroll`**: ナビゲーション中のスクロール動作を制御します。デフォルトは `true` で、新しいルートのトップにスクロールし、前後のナビゲーションではスクロール位置を保持します

</PagesOnly>

<AppOnly>

`action` が文字列の場合、`<Form>` コンポーネントは次の props をサポートします：

| Prop       | 例                 | 型                            | 必須 |
| ---------- | ------------------ | ----------------------------- | ---- |
| `action`   | `action="/search"` | `string` (URL または相対パス) | Yes  |
| `replace`  | `replace={false}`  | `boolean`                     | -    |
| `scroll`   | `scroll={true}`    | `boolean`                     | -    |
| `prefetch` | `prefetch={true}`  | `boolean`                     | -    |

- **`action`**: フォームが送信されたときに移動するURLまたはパス
  - 空文字列 `""` を指定すると、検索パラメータが更新された同じルートに移動します
- **`replace`**: 現在の履歴の状態を置き換え、[ブラウザの履歴](https://developer.mozilla.org/en-US/docs/Web/API/History_API)スタックに新しい履歴をプッシュしないようにします。デフォルトは `false` です
- **`scroll`**: ナビゲーション中のスクロール動作を制御します。デフォルトは `true` で、新しいルートのトップにスクロールし、前後のナビゲーションではスクロール位置を保持します
- **`prefetch`**: フォームがユーザーのビューポート内で表示されたとき、ルートがプレフェッチされるかどうかを制御します。デフォルトは `true` です

### `action`（関数）Props {#action-function-props}

`action` が関数の場合、`<Form>` コンポーネントは次の prop をサポートします：

| Prop     | 例                  | 型                         | 必須 |
| -------- | ------------------- | -------------------------- | ---- |
| `action` | `action={myAction}` | `function` (Server Action) | Yes  |

- **`action`**: フォームが送信されたときに呼び出されるServer Action。詳細は [React ドキュメント](https://react.dev/reference/react-dom/components/form#props)を参照してください

> **Good to know**: `action` が関数の場合、`replace` および `scroll` props は無視されます。

</AppOnly>

### 注意点 {#caveats}

<AppOnly>

- **`formAction`**: `<button>` または `<input type="submit">` フィールドで `action` prop をオーバーライドするために使用できます。Next.js はクライアントサイドナビゲーションを実行しますが、このアプローチはプレフェッチをサポートしません。
  - [`basePath`](/docs/app/api-reference/next-config-js/basePath) を使用する場合、`formAction` パスにもこれを含める必要があります。例：`formAction="/base-path/search"`。
- **`key`**: 文字列 `action` に `key` prop を渡すことはサポートされていません。再レンダリングをトリガーしたり、ミューテーションを実行したりするには、代わりに関数 `action` を使用することを検討してください。

</AppOnly>

- **`onSubmit`**: フォームの送信ロジックを処理するために使用できます。ただし、`event.preventDefault()` を呼び出すと、指定されたURLへの移動などの `<Form>` の動作をオーバーライドします。
- **[`method`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#method), [`encType`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#enctype), [`target`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#target)**: これらは `<Form>` の動作をオーバーライドするためサポートされていません。
  - 同様に、`formMethod`、`formEncType`、および `formTarget` は、それぞれ `method`、`encType`、および `target` props をオーバーライドするために使用できますが、それらを使用することでネイティブブラウザの動作にフォールバックすることになります。
  - これらのプロップを使用する必要がある場合は、HTML `<form>` 要素を使用してください。
- **`<input type="file">`**: `action` が文字列の場合、この入力タイプを使用すると、ファイルオブジェクトではなくファイル名を送信するというブラウザの動作に一致します。

<AppOnly>

## 例 {#examples}

### 検索結果ページに移動する検索フォーム {#search-form-that-leads-to-a-search-result-page}

`action` にパスを渡すことで、検索結果ページにナビゲーションする検索フォームを作成できます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="/app/page.tsx" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="/app/page.js" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

</TabItem>
</Tabs>

ユーザーが query 入力フィールドを更新してフォームを送信すると、フォームデータは例のようにURLに検索パラメータとしてエンコードされます `/search?query=abc`。

> **Good to know**: `action` に空文字列 `""` を渡すと、フォームは検索パラメータが更新された同じルートに移動します。

結果ページでは、[`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) `page.js` prop を使用してクエリにアクセスし、外部ソースからデータを取得できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="/app/search/page.tsx" switcher
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({
  searchParams,
}: {
  searchParams: { [key: string]: string | string[] | undefined }
}) {
  const results = await getSearchResults(searchParams.query)

  return <div>...</div>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="/app/search/page.js" switcher
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({ searchParams }) {
  const results = await getSearchResults(searchParams.query)

  return <div>...</div>
}
```

</TabItem>
</Tabs>

`<Form>` がユーザーのビューポートで表示されると、`/search` ページの共有UI（`layout.js`や`loading.js`など）がプレフェッチされます。送信時には、フォームはすぐに新しいルートに移動し、結果が取得されている間にローディングUIを表示します。フォールバックUIの設計には、[`loading.js`](/docs/app/api-reference/file-conventions/loading) を使用できます：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="/app/search/loading.tsx" switcher
export default function Loading() {
  return <div>Loading...</div>
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="/app/search/loading.js" switcher
export default function Loading() {
  return <div>Loading...</div>
}
```

</TabItem>
</Tabs>

共有UIがまだ読み込まれていない場合、[`useFormStatus`](https://react.dev/reference/react-dom/hooks/useFormStatus) を使用してユーザーに対して即時フィードバックを表示できます。

まず、フォームが保留中のときにローディング状態を表示するコンポーネントを作成します：

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="/app/ui/search-button.tsx" switcher
'use client'
import { useFormStatus } from 'react-dom'

export default function SearchButton() {
  const status = useFormStatus()
  return (
    <button type="submit">{status.pending ? 'Searching...' : 'Search'}</button>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="/app/ui/search-button.js" switcher

    <button type="submit">{status.pending ? 'Searching...' : 'Search'}</button>

}
```

</TabItem>
</Tabs>

### サーバーアクションを伴うミューテーション {#mutations-with-server-actions}

`action` prop に関数を渡すことでミューテーションを実行できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="/app/posts/create/page.tsx" switcher
import Form from 'next/form'
import { createPost } from '@/posts/actions'

export default function Page() {
  return (
    <Form action={createPost}>
      <input name="title" />
      {/* ... */}
      <button type="submit">Create Post</button>
    </Form>
  )
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="/app/posts/create/page.js" switcher
import Form from 'next/form'
import { createPost } from '@/posts/actions'

export default function Page() {
  return (
    <Form action={createPost}>
      <input name="title" />
      {/* ... */}
      <button type="submit">Create Post</button>
    </Form>
  )
}
```

</TabItem>
</Tabs>

ミューテーションの後には、新しいリソースにリダイレクトすることが一般的です。`next/navigation` から [`redirect`](/docs/app/building-your-application/routing/redirecting) 関数を使用して、新しいポストページに移動できます。

> **Good to know**: フォーム送信の "目的地" はアクションが実行されるまで不明であるため、`<Form>` は共有UIを自動的にプレフェッチすることはできません。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="/app/posts/actions.ts" switcher
'use server'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  // 新しいポストを作成
  // ...

  // 新しいポストにリダイレクト
  redirect(`/posts/${data.id}`)
}
```

</TabItem>
<TabItem value="jsx" label="JavaScript">

```jsx title="/app/posts/actions.js" switcher
'use server'
import { redirect } from 'next/navigation'

export async function createPost(formData) {
  // 新しいポストを作成
  // ...

  // 新しいポストにリダイレクト

```

</TabItem>
</Tabs>
