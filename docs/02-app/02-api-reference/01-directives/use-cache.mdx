`use cache`ディレクティブは、キャッシュを行うコンポーネントおよび／または関数を指定します。ファイルの冒頭で使用すると、ファイル内のすべてのエクスポートがキャッシュ可能であることを示し、関数やコンポーネントの冒頭で使用すると、Next.jsにリターン値がキャッシュされ、後続のリクエストで再利用されるべきであることを示します。これはNext.jsの実験的な機能であり、Reactのネイティブ機能である`use client`や`use server`のようなものではありません。

## 使用方法 {#usage}

`use cache`ディレクティブのサポートを有効にするために、`next.config.ts`ファイルで`dynamicIO`フラグを有効にしてください：

```ts title="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
  },
}

export default nextConfig
```

その後、`use cache`ディレクティブをファイル、コンポーネント、関数のレベルで使用できます：

```tsx
// ファイルレベル
'use cache'

export default async function Page() {
  // ...
}

// コンポーネントレベル
export async function MyComponent() {
  'use cache'
  return <></>
}

// 関数レベル
export async function getData() {
  'use cache'
  const data = await fetch('/api/data')
  return data
}
```

## 知っておくと良いこと {#good-to-know}

- `use cache`はNext.jsの実験的な機能であり、[`use client`](/docs/app/api-reference/directives/use-client)や[`use server`](/docs/app/api-reference/directives/use-server)のようなReactネイティブの機能ではありません。
- キャッシュされた関数に渡される引数（またはprops）は、Reactによって[シリアル化可能](https://react.dev/reference/rsc/use-server#serializable-parameters-and-return-values)でなければなりません。つまり、JSONのような形式に変換できる必要があります。関数やReact要素のような非シリアル化可能なpropsは、引数として使用できません。
- キャッシュ可能な関数のリターン値もシリアル化可能でなければなりません。これにより、キャッシュされたデータが正しく保存および取得されることを保証します。
- `use cache`ディレクティブを使用する関数は、状態の変更やDOMの直接操作、インターバルでのコード実行のためのタイマー設定などの副作用を持ってはいけません。
- [Partial Prerendering](/docs/app/building-your-application/rendering/partial-prerendering)と一緒に使用する場合、`use cache`を持つセグメントは、静的HTMLシェルの一部としてプリレンダリングされます。
- 将来的には、`use cache`ディレクティブは`dynamicIO`フラグとは別に利用可能になります。
- [`unstable_cache`](/docs/app/api-reference/legacy-apis/unstable_cache)がJSONデータのみをサポートするのに対して、`use cache`はReactがレンダリングできる任意のシリアル化可能なデータ、コンポーネントのレンダリング出力をキャッシュできます。

## 例 {#examples}

### `use cache`を使用したルート全体のキャッシュ {#caching-entire-routes-with-use-cache}

ルート全体をプリレンダリングするには、`layout`と`page`ファイルの両方の冒頭に`use cache`を追加します。これらのセグメントはアプリケーション内の個別のエントリーポイントとして扱われ、それぞれ独立してキャッシュされます。

```tsx title="app/layout.tsx" switcher
  return <div>{children}</div>
}
```

`page`ファイルにインポートされネストされるコンポーネントは、`page`のキャッシュ動作を継承します。

### `use cache`を使用したコンポーネント出力のキャッシュ {#caching-component-output-with-use-cache}

`use cache`をコンポーネントレベルで使用することで、そのコンポーネント内で行われるフェッチや計算をキャッシュできます。アプリケーション全体でコンポーネントを再利用する際、propsが同じ構造を持つ限り同じキャッシュエントリを共有できます。

propsはシリアル化され、キャッシュキーの一部を形成します。シリアル化されたpropsが各インスタンスで同じ値を生成する限り、キャッシュエントリは再利用されます。

```tsx title="app/components/bookings.tsx" highlight={2}
export async function Bookings({ type = 'haircut' }: BookingsProps) {
  'use cache'
  async function getBookingsData() {
    const data = await fetch(`/api/bookings?type=${encodeURIComponent(type)}`)
    return data
  }
  return //...
}

interface BookingsProps {
  type: string
}
```

```jsx title="app/components/bookings.js" highlight={2}
export async function Bookings({ type = 'haircut' }) {
  'use cache'
  async function getBookingsData() {
    const data = await fetch(`/api/bookings?type=${encodeURIComponent(type)}`)
    return data
  }
  return //...
}
```

### `use cache`を使用した関数出力のキャッシュ {#caching-function-output-with-use-cache}

`use cache`を任意の非同期関数に追加できるため、コンポーネントやルートのみに限らずキャッシュできます。ネットワークリクエストやデータベースクエリをキャッシュしたり、非常に遅い計算を行う場合に使用できます。この種の作業を含む関数に`use cache`を追加することで、その関数はキャッシュ可能になり、再利用する際には同じキャッシュエントリを共有します。

```tsx title="app/actions.ts" highlight={2} switcher
  'use cache'

  const data = await fetch('/api/data')
  return data
```

### 再検証 {#revalidating}

デフォルトでは、Next.jsは`use cache`ディレクティブを使用する際に**[再検証期間](/docs/app/building-your-application/data-fetching/fetching#revalidating-cached-data)を15分**に設定します。Next.jsはほぼ無限の有効期限を設定し、頻繁な更新が必要のないコンテンツに適しています。

この再検証期間は、頻繁な変更が予期されないコンテンツに有用ですが、`cacheLife`および`cacheTag`APIを使用してキャッシュ動作を設定することができます：

- [`cacheLife`](/docs/app/api-reference/functions/cacheLife)：時間ベースの再検証期間。
- [`cacheTag`](/docs/app/api-reference/functions/cacheTag)：オンデマンドの再検証。

これらのAPIは、クライアントとサーバのキャッシング層全体に統合されているため、キャッシュのセマンティクスを一箇所で設定し、全体に適用できます。

詳細は[`cacheLife`](/docs/app/api-reference/functions/cacheLife)および[`cacheTag`](/docs/app/api-reference/functions/cacheTag)のドキュメントを参照してください。

### インターリーブ {#interleaving}

キャッシュ可能な関数に非シリアル化可能な引数を渡す必要がある場合、それらを`children`として渡すことができます。これにより、`children`の参照が変更されてもキャッシュエントリに影響を与えずにすみます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
async function Page() {
  const uncachedData = await getData()
  return (
    <CacheComponent>
      <DynamicComponent data={uncachedData} />
    </CacheComponent>
  )
}

async function CacheComponent({ children }) {
  'use cache'
  const cachedData = await fetch('/api/cached-data')
  return (
    <div>
      <PrerenderedComponent data={cachedData} />
      {children}
    </div>
  )
}
```

</TabItem>
</Tabs>

キャッシュ済みのコンポーネントを介して、Server ActionsをClient Componentsに渡すこともできますが、それをキャッシュ可能な関数内で呼び出さずに利用できます。

<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
    "use server"
    // サーバサイドの更新を実行する

  return <CacheComponent performUpdate={performUpdate} />;
}

async function CachedComponent({ performUpdate }) {
```
