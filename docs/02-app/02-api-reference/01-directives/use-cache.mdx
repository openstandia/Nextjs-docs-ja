`use cache`ディレクティブは、コンポーネントおよび/または関数をキャッシュするために指定します。ファイルの冒頭で使用して、ファイル内のすべてのエクスポートがキャッシュ可能であることを示したり、関数やコンポーネントの冒頭でインラインで使用し、Next.jsに返り値をキャッシュし、後続のリクエストで再利用すべきであることを通知したりできます。これは実験的なNext.jsの機能であり、`use client`や`use server`のようなReactのネイティブ機能ではありません。

## 使用方法 {#usage}

`next.config.ts`ファイルの`dynamicIO`フラグを使用して`use cache`ディレクティブのサポートを有効にします：

```ts title="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    dynamicIO: true,
  },
}

export default nextConfig
```

次に、`use cache`ディレクティブをファイル、コンポーネント、または関数レベルで使用できます：

```tsx
// ファイルレベル
'use cache'

export default async function Page() {
  // ...
}

// コンポーネントレベル
export async function MyComponent() {
  'use cache'
  return <></>
}

// 関数レベル
export async function getData() {
  'use cache'
  const data = await fetch('/api/data')
  return data
}
```

## Good to know {#good-to-know}

- `use cache`は実験的なNext.jsの機能であり、[`use client`](/docs/app/api-reference/directives/use-client)や[`use server`](/docs/app/api-reference/directives/use-server)のようなReactのネイティブ機能ではありません。
- キャッシュされた関数に渡される引数（またはprops）は、Reactによって[直列化可能](https://react.dev/reference/rsc/use-server#serializable-parameters-and-return-values)でなければなりません。これは、これらがJSONのような形式に変換できることを意味します。関数やReact要素のような直列化不可能なpropsは引数として使用できません。
- キャッシュ可能な関数の返り値も直列化可能でなければなりません。これにより、キャッシュされたデータが正しく保存および取得されます。
- `use cache`ディレクティブを使用する関数は、状態の変更、DOMの直接操作、インターバルでのコード実行用のタイマー設定などの副作用を持ってはなりません。
- [Partial Prerendering](/docs/app/building-your-application/rendering/partial-prerendering)と併用する場合、`use cache`を含むセグメントは静的HTMLシェルの一部としてプリレンダリングされます。
- `use cache`ディレクティブは将来的に`dynamicIO`フラグとは独立して利用可能になります。
- [`unstable_cache`](/docs/app/api-reference/legacy-apis/unstable_cache)がJSONデータのみをサポートしていたのとは異なり、`use cache`はReactがレンダリングできる任意の直列化可能データをキャッシュできます。コンポーネントのレンダリング出力も含まれます。

## 例 {#examples}

### `use cache`でルート全体をキャッシュする {#caching-entire-routes-with-use-cache}

ルート全体をプリレンダリングするには、`layout`と`page`ファイルの両方の冒頭に`use cache`を追加してください。それぞれのセグメントはアプリケーション内で独立したエントリーポイントとして扱われ、個別にキャッシュされます。

### `use cache`でコンポーネント出力をキャッシュする {#caching-component-output-with-use-cache}

`use cache`をコンポーネントレベルで使用して、そのコンポーネント内で実行されたフェッチや計算をキャッシュすることができます。アプリケーション全体でコンポーネントを再利用する際、propsが同一の構造を保つ限り、同じキャッシュエントリを共有できます。

propsは直列化され、キャッシュキーの一部を形成し、直列化されたpropsが各インスタンスで同じ値を生成する限り、キャッシュエントリが再利用されます。

```tsx title="app/components/bookings.tsx" highlight={2}
export async function Bookings({ type = 'haircut' }: BookingsProps) {
  'use cache'
  async function getBookingsData() {
    const data = await fetch(`/api/bookings?type=${encodeURIComponent(type)}`)
    return data
  }
  return //...
}

interface BookingsProps {
  type: string
}
```

```jsx title="app/components/bookings.js" highlight={2}
export async function Bookings({ type = 'haircut' }) {
  'use cache'
  async function getBookingsData() {
    const data = await fetch(`/api/bookings?type=${encodeURIComponent(type)}`)
    return data
  }
  return //...
}
```

### `use cache`で関数出力をキャッシュする {#caching-function-output-with-use-cache}

`use cache`を任意の非同期関数に追加できるため、コンポーネントやルートのみをキャッシュすることに限定されません。ネットワークリクエストやデータベースクエリをキャッシュしたり、非常に遅い計算をキャッシュしたりすることを望むかもしれません。この種の作業を含む関数に`use cache`を追加することで、それがキャッシュ可能になり、再利用される際には同じキャッシュエントリを共有します。

### 再検証 {#revalidating}

デフォルトでは、Next.jsは`use cache`ディレクティブを使用する際に**[15分間の再検証期間](/docs/app/building-your-application/data-fetching/fetching#revalidating-cached-data)**を設定します。Next.jsはほぼ無限の有効期限を設定し、頻繁な更新が不要なコンテンツに適しています。

この再検証期間は頻繁に変更が予想されないコンテンツには便利かもしれませんが、`cacheLife`や`cacheTag` APIを使用してキャッシュ動作を設定できます：

- [`cacheLife`](/docs/app/api-reference/functions/cacheLife): 時間ベースの再検証期間に使用します。
- [`cacheTag`](/docs/app/api-reference/functions/cacheTag): オンデマンドの再検証に使用します。

これらのAPIはクライアントとサーバーのキャッシング層全体で統合されており、1つの場所でキャッシングセマンティクスを設定し、それをどこでも適用できます。

詳細は、[`cacheLife`](/docs/app/api-reference/functions/cacheLife)および[`cacheTag`](/docs/app/api-reference/functions/cacheTag)のドキュメントを参照してください。

### 交互に組み込む {#interleaving}

キャッシュ可能な関数に直列化不可能な引数を渡す必要がある場合は、それらを`children`として渡すことができます。これにより、`children`の参照はキャッシュエントリに影響を与えることなく変更できます。

<Tabs>
<TabItem value="tsx" label="TypeScript">

```tsx title="app/page.tsx" switcher
async function Page() {
  const uncachedData = await getData()
  return (
    <CacheComponent>
      <DynamicComponent data={uncachedData} />
    </CacheComponent>
  )
}

async function CacheComponent({ children }) {
  'use cache'
  const cachedData = await fetch('/api/cached-data')
  return (
    <div>
      <PrerenderedComponent data={cachedData} />
      {children}
    </div>
  )
}
```

</TabItem>
</Tabs>
